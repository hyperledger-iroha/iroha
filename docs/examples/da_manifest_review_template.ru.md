# Пакет управления манифестом доступности данных (шаблон)

Используйте этот шаблон, когда панели Парламента проверяют манифесты DA для субсидий,
takedown-ов или изменений удержания (roadmap DA-10). Скопируйте Markdown в governance тикет,
заполните плейсхолдеры и приложите заполненный файл вместе с подписанными Norito payload-ами
и CI артефактами, указанными ниже.

```markdown
## Метаданные манифеста
- Имя / версия манифеста: <строка>
- Класс blob и тег governance: <taikai_segment / da.taikai.live>
- Дайджест BLAKE3 (hex): `<digest>`
- Хэш Norito payload (опционально): `<digest>`
- Source envelope / URL: <https://.../manifest_signatures.json>
- ID snapshot политики Torii: `<unix timestamp or git sha>`

## Проверка подписей
- Источник получения манифеста / storage тикет: `<hex>`
- Команда/вывод проверки: `cargo run -p sorafs_car --bin sorafs-manifest-stub -- manifest.json --manifest-signatures-in=manifest_signatures.json` (приложен фрагмент лога?)
- `manifest_blake3`, сообщенный инструментом: `<digest>`
- `chunk_digest_sha3_256`, сообщенный инструментом: `<digest>`
- Multihash-ы подписантов совета:
  - `<did:...>` / `<ed25519 multihash>`
- Время проверки (UTC): `<2026-02-20T11:04:33Z>`

## Проверка удержания
| Поле | Ожидается (политика) | Наблюдается (манифест) | Доказательство |
|------|----------------------|------------------------|----------------|
| Горячее удержание (секунды) | <например, 86400> | <значение> | `<torii.da_ingest.replication_policy dump | CI link>` |
| Холодное удержание (секунды) | <например, 1209600> | <значение> |  |
| Требуемые реплики | <значение> | <значение> |  |
| Класс хранения | <hot / warm / cold> | <значение> |  |
| Тег governance | <da.taikai.live> | <значение> |  |

## Контекст
- Тип запроса: <Субсидия | Takedown | Ротация манифеста | Экстренная заморозка>
- Исходный тикет / ссылка на compliance: <ссылка или ID>
- Влияние на субсидию / ренту: <ожидаемое изменение XOR или "n/a">
- Ссылка на апелляцию модерации (если есть): <case_id или ссылка>

## Итог решения
- Панель: <Инфраструктура | Модерация | Казначейство>
- Результат голосования: `<for>/<against>/<abstain>` (кворум `<threshold>` достигнут?)
- Высота/окно активации или отката: `<block/slot range>`
- Дальнейшие действия:
  - [ ] Уведомить Казначейство / rent ops
  - [ ] Обновить отчет о прозрачности (`TransparencyReportV1`)
  - [ ] Запланировать аудит buffer

## Эскалация и отчетность
- Трек эскалации: <Субсидия | Compliance | Экстренная заморозка>
- Ссылка / ID отчета о прозрачности (если обновлен): <`TransparencyReportV1` CID>
- Пакет proof-token или ссылка на ComplianceUpdate: <путь или ID тикета>
- Дельта реестра ренты / резерва (если применимо): <`ReserveSummaryV1` snapshot link>
- URL(ы) снимка телеметрии: <Grafana permalink или artefact ID>
- Примечания для протокола Парламента: <краткое описание сроков / обязательств>

## Приложения
- [ ] Подписанный Norito манифест (`.to`)
- [ ] Сводка JSON / CI артефакт, подтверждающий значения удержания
- [ ] Proof token или compliance пакет (для takedown-ов)
- [ ] Снимок телеметрии buffer (`iroha_settlement_buffer_xor`)
```

Архивируйте каждый заполненный пакет под записью Governance DAG для голосования, чтобы
последующие проверки могли ссылаться на дайджест манифеста без повторения всей церемонии.

# SoraFS CI Sample Fixtures

This directory packages deterministic artefacts generated from the sample
payload under `fixtures/sorafs_manifest/ci_sample/`. The bundle demonstrates the
end-to-end SoraFS packaging and signing pipeline that the CI workflows exercise.

## Artefact Inventory

| File | Description |
|------|-------------|
| `payload.txt` | Source payload used by the fixture scripts (plain-text sample). |
| `payload.car` | CAR archive emitted by `sorafs_cli car pack`. |
| `car_summary.json` | Summary generated by `car pack` capturing chunk digests and metadata. |
| `chunk_plan.json` | Fetch-plan JSON describing chunk ranges and provider expectations. |
| `manifest.to` | Norito manifest produced by `sorafs_cli manifest build`. |
| `manifest.json` | Human-readable manifest rendering for debugging. |
| `proof.json` | PoR summary emitted by `sorafs_cli proof verify`. |
| `manifest.bundle.json` | Keyless signature bundle generated by `sorafs_cli manifest sign`. |
| `manifest.sig` | Detached Ed25519 signature corresponding to the manifest. |
| `manifest.sign.summary.json` | CLI summary emitted during signing (hashes, bundle metadata). |
| `manifest.verify.summary.json` | CLI summary from `manifest verify-signature`. |

All digests referenced in release notes and documentation were sourced from
these files. The `ci/check_sorafs_cli_release.sh` workflow regenerates the same
artefacts and diffs them against the committed versions.

## Fixture Regeneration

Run the commands below from the repository root to regenerate the fixture set.
They mirror the steps used by the `sorafs-cli-fixture` workflow:

```bash
sorafs_cli car pack \
  --input fixtures/sorafs_manifest/ci_sample/payload.txt \
  --car-out fixtures/sorafs_manifest/ci_sample/payload.car \
  --plan-out fixtures/sorafs_manifest/ci_sample/chunk_plan.json \
  --summary-out fixtures/sorafs_manifest/ci_sample/car_summary.json

sorafs_cli manifest build \
  --summary fixtures/sorafs_manifest/ci_sample/car_summary.json \
  --manifest-out fixtures/sorafs_manifest/ci_sample/manifest.to \
  --manifest-json-out fixtures/sorafs_manifest/ci_sample/manifest.json

sorafs_cli proof verify \
  --manifest fixtures/sorafs_manifest/ci_sample/manifest.to \
  --car fixtures/sorafs_manifest/ci_sample/payload.car \
  --summary-out fixtures/sorafs_manifest/ci_sample/proof.json

sorafs_cli manifest sign \
  --manifest fixtures/sorafs_manifest/ci_sample/manifest.to \
  --summary fixtures/sorafs_manifest/ci_sample/car_summary.json \
  --chunk-plan fixtures/sorafs_manifest/ci_sample/chunk_plan.json \
  --bundle-out fixtures/sorafs_manifest/ci_sample/manifest.bundle.json \
  --signature-out fixtures/sorafs_manifest/ci_sample/manifest.sig \
  --identity-token "$(cat fixtures/sorafs_manifest/ci_sample/fixture_identity_token.jwt)" \
  --issued-at 1700000000 \
  > fixtures/sorafs_manifest/ci_sample/manifest.sign.summary.json

sorafs_cli manifest verify-signature \
  --manifest fixtures/sorafs_manifest/ci_sample/manifest.to \
  --bundle fixtures/sorafs_manifest/ci_sample/manifest.bundle.json \
  --summary fixtures/sorafs_manifest/ci_sample/car_summary.json \
  --chunk-plan fixtures/sorafs_manifest/ci_sample/chunk_plan.json \
  --expect-token-hash 7b56598bca4584a5f5631ce4e510b8c55bd9379799f231db2a3476774f45722b \
  > fixtures/sorafs_manifest/ci_sample/manifest.verify.summary.json
```

If any step produces different hashes, investigate before updating the fixtures.
The CI workflows rely on deterministic output to detect regressions.

## Future Coverage

As additional chunker profiles and proof formats graduate from the roadmap,
their canonical fixtures will be added under this directory (e.g.,
`sorafs.sf2@1.0.0` (see `fixtures/sorafs_manifest/ci_sample_sf2/`) or PDP
streaming proofs). Each new profile will follow the same structure—payload, CAR,
plan, manifest, proofs, and signature artefacts—so downstream automation can
diff releases without custom scripting.

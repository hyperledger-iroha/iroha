# SoraNet GAR қабылдау үлгісі

GAR әрекетін сұрау кезінде осы қабылдау пішінін пайдаланыңыз (тазарту, ttl қайта анықтау, мөлшерлеме
төбе, модерация директивасы, геоқорғау немесе заңды ұстау). Жіберілген пішін
`gar_controller` шығыстарымен қатар бекітілуі керек, сондықтан аудит журналдары мен
түбіртектерде бірдей дәлелдер URI сілтемелері бар.

| Өріс | Мән | Ескертпелер |
|-------|-------|-------|
| Сұраныс идентификаторы |  | Қамқоршы/опс билетінің идентификаторы. |
| Сұраныс |  | Тіркелгі + контакт. |
| Күні/уақыты (UTC) |  | Әрекет қашан басталуы керек. |
| GAR атауы |  | мысалы, `docs.sora`. |
| Канондық хост |  | мысалы, `docs.gw.sora.net`. |
| Әрекет |  | `ttl_override` / `rate_limit_override` / `purge_static_zone` / `geo_fence` / `legal_hold` / `moderation`. |
| TTL қайта анықтау (секунд) |  | Тек `ttl_override` үшін қажет. |
| Мөлшерлеме төбесі (RPS) |  | Тек `rate_limit_override` үшін қажет. |
| Рұқсат етілген аймақтар |  | `geo_fence` сұрау кезінде ISO аймақтарының тізімі. |
| Теріс аймақтар |  | `geo_fence` сұрау кезінде ISO аймақтарының тізімі. |
| Модерация слагтары |  | GAR модерациясының директиваларын сәйкестендіріңіз. |
| Тегтерді тазалау |  | Қызмет көрсетер алдында тазартылуы керек тегтер. |
| Белгілер |  | Машина белгілері (оқиға идентификаторы, бұрғы атауы, поп ауқымы). |
| Дәлелдер URI |  | Сұрауды қолдайтын журналдар/бақылау тақталары/спецификациялар. |
| URI аудиті |  | Әдепкі параметрлерден басқаша болса, әр поп аудитінің URI. |
| Сұралған жарамдылық мерзімі |  | Unix уақыт белгісі немесе RFC3339; әдепкі үшін бос қалдырыңыз. |
| Себеп |  | Пайдаланушыға түсініктеме; түбіртектерде және бақылау тақталарында көрсетіледі. |
| Бекітуші |  | Сұраныс үшін қамқоршы/комитет мақұлдаушы. |

### Жіберу қадамдары

1. Кестені толтырып, оны басқару билетіне тіркеңіз.
2. GAR контроллерінің конфигурациясын (`policies`/`pops`) сәйкестікпен жаңартыңыз
   `labels`/`evidence_uris`/`expires_at_unix`.
3. Оқиғаларды/түбіртектерді шығару үшін `cargo xtask soranet-gar-controller ...` іске қосыңыз.
4. `gar_controller_summary.json`, `gar_reconciliation_report.json`, түсіріңіз.
   `gar_metrics.prom` және `gar_audit_log.jsonl` бірдей билетке. The
   бекітуші түбіртек саны жөнелту алдында PoP тізіміне сәйкес келетінін растайды.

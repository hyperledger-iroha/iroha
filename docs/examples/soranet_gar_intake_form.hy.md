# SoraNet GAR ընդունման ձևանմուշ

Օգտագործեք այս ընդունման ձևը, երբ պահանջում եք GAR գործողություն (մաքրում, ttl անտեսում, տոկոսադրույք
առաստաղ, չափավորության հրահանգ, աշխարհախախտում կամ օրինական պահում): Ներկայացված ձևը
պետք է ամրացվի `gar_controller` ելքերի կողքին, այնպես որ ստուգեք տեղեկամատյանները և
անդորրագրերը մեջբերում են նույն ապացույցների URI-ները:

| Դաշտային | Արժեք | Ծանոթագրություններ |
|-------|-------|-------|
| Հայտի ID |  | Guardian/ops տոմսի ID. |
| Պահանջել է |  | Հաշիվ + կոնտակտ: |
| Ամսաթիվ/ժամ (UTC) |  | Երբ պետք է սկսվի գործողությունը: |
| ԳԱՐ անվանումը |  | օրինակ՝ `docs.sora`: |
| Կանոնական հաղորդավար |  | օրինակ՝ `docs.gw.sora.net`: |
| Գործողություն |  | `ttl_override` / `rate_limit_override` / `purge_static_zone` / `geo_fence` / `legal_hold` / `moderation`: |
| TTL վերագրանցում (վայրկյան) |  | Պահանջվում է միայն `ttl_override`-ի համար: |
| Գնահատման առաստաղ (RPS) |  | Պահանջվում է միայն `rate_limit_override`-ի համար: |
| Թույլատրված մարզեր |  | ISO տարածաշրջանների ցանկը `geo_fence`-ի հարցումով: |
| Մերժված շրջաններ |  | ISO տարածաշրջանների ցանկը `geo_fence`-ի հարցումով: |
| Չափավորություն slugs |  | Համապատասխանեցրեք GAR-ի չափավորության հրահանգներին: |
| Մաքրել պիտակները |  | Պիտակներ, որոնք պետք է մաքրվեն մատուցելուց առաջ: |
| Պիտակներ |  | Մեքենայի պիտակներ (միջադեպի id, փորվածքի անվանում, փոփ շրջանակ): |
| Ապացույցների URI-ներ |  | Տեղեկամատյաններ / վահանակներ / ակնարկներ, որոնք ապահովում են հարցումը: |
| Աուդիտի URI |  | Per-pop աուդիտի URI, եթե տարբերվում է կանխադրվածից: |
| Պահանջվող ժամկետը |  | Unix-ի ժամանակային դրոշմակնիք կամ RFC3339; լռելյայն թողնել դատարկ: |
| Պատճառը |  | Օգտատիրոջ առջև բացատրություն; հայտնվում է անդորրագրերում և վահանակներում: |
| Հաստատող |  | Պահապան/հանձնաժողովը հաստատող խնդրանքը. |

### Ներկայացման քայլեր

1. Լրացրեք աղյուսակը և կցեք այն կառավարման տոմսին:
2. Թարմացրեք GAR կարգավորիչի կազմաձևը (`policies`/`pops`) համապատասխանությամբ
   `labels`/`evidence_uris`/`expires_at_unix`.
3. Գործարկեք `cargo xtask soranet-gar-controller ...`՝ իրադարձություններ/անդորրագրեր թողարկելու համար:
4. Գցել `gar_controller_summary.json`, `gar_reconciliation_report.json`,
   `gar_metrics.prom` և `gar_audit_log.jsonl` նույն տոմսի մեջ: Այն
   հաստատողը հաստատում է, որ անդորրագրերի քանակը համապատասխանում է PoP ցուցակին նախքան առաքումը:

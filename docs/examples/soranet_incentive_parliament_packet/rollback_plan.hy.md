# Ռելեի խրախուսման հետադարձ պլան

Օգտագործեք այս գրքույկը՝ անջատելու ավտոմատ փոխանցման վճարումները, եթե կառավարման խնդրանքը ա
դադարեցնել կամ եթե հեռամետրաժ պահակակետերը կրակում են:

1. **Սառեցրեք ավտոմատացումը։** Դադարեցրեք խրախուսական դեմոնը յուրաքանչյուր նվագախմբի հաղորդավարի վրա
   (`systemctl stop soranet-incentives.service` կամ համարժեք կոնտեյներ
   տեղակայում) և հաստատեք, որ գործընթացն այլևս չի աշխատում:
2. **Ձեռնարկել սպասվող հրահանգները։** Վազել
   `iroha app sorafs incentives service daemon --state <state.json> --config <daemon.json> --metrics-dir <spool> --once`
   ապահովելու համար, որ չկան չմարված վճարման հրահանգներ: Արխիվացրեք ստացվածը
   Norito օգտակար բեռներ աուդիտի համար:
3. **Չեղյալ համարել կառավարման հաստատումը։** Խմբագրել `reward_config.json`, սահմանել
   `"budget_approval_id": null` և վերաբաշխել կոնֆիգուրացիան
   `iroha app sorafs incentives service init` (կամ `update-config`, եթե աշխատում է
   երկարակյաց դեյմոն): Վճարման շարժիչն այժմ ձախողվում է փակված
   `MissingBudgetApprovalId`, ուստի դեյմոնը հրաժարվում է վճարումներ կատարել մինչև նոր
   հաստատման հեշը վերականգնվել է: Գրանցեք git commit-ը և SHA-256-ը
   փոփոխված կոնֆիգուրացիա միջադեպերի մատյանում:
4. **Ծանուցեք Սորա խորհրդարանին։** Կցեք քայքայված վճարումների մատյանը՝ ստվերային վազքը
   հաշվետվություն և միջադեպի կարճ ամփոփում: Խորհրդարանի արձանագրությունները պետք է նշվեն հաշը
   չեղարկված կոնֆիգուրացիայի մասին և ժամանակի, երբ դեյմոնը դադարեցվեց:
5. **Վերադարձի վավերացում։** Դեյմոնն անջատված պահեք մինչև՝
   - Հեռաչափության ազդանշանները (`soranet_incentives_rules.yml`) կանաչ են >=24 ժամվա ընթացքում,
   - գանձապետական հաշտեցման հաշվետվությունը ցույց է տալիս զրոյական բացակայող փոխանցումներ, և
   - Խորհրդարանը հաստատում է բյուջեի նոր հաշը:

Երբ կառավարությունը վերաթողարկի բյուջեի հաստատման հեշը, թարմացրեք `reward_config.json`
նոր ամփոփումով, նորից գործարկեք `shadow-run` հրամանը վերջին հեռաչափության վրա,

# نموذج ملاحظات لمعاينة الوثائق (موجة W1)

استخدم هذا القالب عند جمع الملاحظات من مراجعي موجة W1. أنشئ نسخة لكل شريك، واملأ
بيانات المراجِع، ثم احفظ النسخة المكتملة تحت المسار:
`artifacts/docs_preview/W1/preview-2025-04-12/feedback/<partner-id>/`.

## بيانات المراجِع (Metadata)

- **معرّف الشريك:** `partner-w1-XX`
- **تذكرة الطلب:** `DOCS-SORA-Preview-REQ-PXX`
- **وقت إرسال الدعوة (UTC):** `YYYY-MM-DD hh:mm`
- **وقت تأكيد الـ checksum (UTC):** `YYYY-MM-DD hh:mm`
- **مجالات التركيز الأساسية:** (مثال: _وثائق منسّق SoraFS_، _مسارات Torii ISO_)

## تأكيدات التليمترية والـ artefacts

| عنصر في قائمة التحقق | النتيجة | الدليل |
| --- | --- | --- |
| التحقق من الـ checksums | ✅ / ⚠️ | مسار ملف الـ log (مثال: `build/checksums.sha256`) |
| اختبار smoke لبروكسي Try it | ✅ / ⚠️ | جزء من مخرجات `npm run manage:tryit-proxy …` |
| مراجعة لوحات Grafana | ✅ / ⚠️ | مسار/مسارات لقطات الشاشة |
| مراجعة تقرير probe للبوابة | ✅ / ⚠️ | `artifacts/docs_preview/.../preflight-summary.json` |

أضِف صفوفًا إضافية لأي SLO إضافية يقوم المراجِع بفحصها.

## سجل الملاحظات (Feedback log)

| المجال | درجة الخطورة (info/minor/major/blocker) | الوصف | الإصلاح المقترح أو السؤال | تذكرة المتابعة |
| --- | --- | --- | --- | --- |
| | | | | |

أشِر في العمود الأخير إلى رقم issue في GitHub أو رقم التذكرة الداخلية بحيث يستطيع
نظام تتبّع الـ preview ربط عناصر المعالجة بهذا النموذج.

## ملخّص الاستبيان

1. **ما مدى ثقتك في إرشادات الـ checksum وعملية الدعوات؟** (من 1 إلى 5)
2. **أي أجزاء من الوثائق كانت الأكثر/الأقل فائدة؟** (إجابة قصيرة)
3. **هل واجهت أي عوائق في الوصول إلى بروكسي Try It أو لوحات التليمترية؟**
4. **هل هناك حاجة إلى محتوى إضافي للترجمة (localisation) أو إمكانية الوصول (accessibility)؟**
5. **أي تعليقات أخرى قبل الوصول إلى GA؟**

سجّل الإجابات المختصرة وأرفق ملفات التصدير الخام للاستبيان إذا كنت تستخدم نموذجًا
خارجيًا.

## اختبار المعرفة

- الدرجة: `__/10`
- الأسئلة غير الصحيحة (إن وُجدت): `[#1, #4, …]`
- إجراءات المتابعة (إذا كانت الدرجة أقل من 9/10): هل تم تحديد جلسة متابعة/تصحيح؟
  نعم/لا

## التوقيع (Sign‑off)

- اسم المراجِع والطابع الزمني:
- اسم مراجِع Docs/DevRel والطابع الزمني:

احفظ النسخة الموقَّعة مع artefacts ذات الصلة حتى يتمكن المدققون من إعادة بناء موجة
المعاينة من دون الحاجة إلى سياق إضافي.

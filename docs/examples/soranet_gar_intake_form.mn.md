# SoraNet GAR хэрэглээний загвар

GAR арга хэмжээ авах хүсэлт гаргахдаа энэ маягтыг ашиглана уу (цэвэрлэх, ttl хүчингүй болгох, хувь хэмжээ
дээд хязгаар, зохицуулалтын заавар, газарзүйн бүс, хууль эрх зүйн хязгаарлалт). Оруулсан маягт
`gar_controller` гаралтын хажууд бэхлэгдсэн байх ёстой тул аудитын бүртгэлүүд болон
Баримт бичиг нь ижил нотлох баримтуудыг иш татдаг.

| Талбай | Үнэ цэнэ | Тэмдэглэл |
|-------|-------|-------|
| Хүсэлтийн ID |  | Guardian/ops тасалбарын ID. |
| Хүссэн |  | Бүртгэл + холбоо барих. |
| Огноо/цаг (UTC) |  | Үйлдэл хэзээ эхлэх ёстой. |
| GAR нэр |  | жишээ нь, `docs.sora`. |
| Каноник хост |  | жишээ нь, `docs.gw.sora.net`. |
| Үйлдэл |  | `ttl_override` / `rate_limit_override` / `purge_static_zone` / `geo_fence` / `legal_hold` / `moderation`. |
| TTL хүчингүй болгох (секунд) |  | Зөвхөн `ttl_override`-д шаардлагатай. |
| Үнийн дээд хязгаар (RPS) |  | Зөвхөн `rate_limit_override`-д шаардлагатай. |
| Зөвшөөрөгдсөн бүсүүд |  | `geo_fence` хүсэлт гаргах үед ISO бүсийн жагсаалт. |
| Татгалзсан бүсүүд |  | `geo_fence` хүсэлт гаргах үед ISO бүсийн жагсаалт. |
| Зохицуулалтын slugs |  | GAR зохицуулалтын удирдамжийг тааруулна уу. |
| Тагуудыг цэвэрлэх |  | Үйлчлэхээсээ өмнө цэвэрлэх ёстой шошго. |
| Шошго |  | Машины шошго (үйл явдлын ID, өрмийн нэр, поп хамрах хүрээ). |
| Нотлох баримтын URI |  | Хүсэлтийг дэмжиж буй бүртгэлүүд/хяналтын самбар/ техникийн үзүүлэлтүүд. |
| Аудитын URI |  | Өгөгдмөлөөс өөр бол нэг поп аудитын URI. |
| Хүссэн хугацаа |  | Unix цагийн тэмдэг эсвэл RFC3339; анхдагчаар хоосон орхи. |
| Шалтгаан |  | Хэрэглэгчдэд зориулсан тайлбар; төлбөрийн баримт болон хяналтын самбарт харагдана. |
| Зөвшөөрөгч |  | Хүсэлтийг асран хамгаалагч/хороонд зөвшөөрч байна. |

### Илгээх алхамууд

1. Хүснэгтийг бөглөж, засаглалын тасалбарт хавсаргана.
2. GAR хянагчийн тохиргоог (`policies`/`pops`) тааруулж шинэчилнэ үү.
   `labels`/`evidence_uris`/`expires_at_unix`.
3. Үйл явдал/баримт гаргахын тулд `cargo xtask soranet-gar-controller ...`-г ажиллуул.
4. `gar_controller_summary.json`, `gar_reconciliation_report.json`,
   `gar_metrics.prom`, `gar_audit_log.jsonl` нэг тасалбар руу орно. The
   Зөвшөөрөгч нь илгээмжийг илгээхийн өмнө төлбөрийн баримтын тоо PoP жагсаалттай тохирч байгааг баталгаажуулна.

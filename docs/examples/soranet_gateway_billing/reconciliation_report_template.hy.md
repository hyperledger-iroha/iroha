# SoraGlobal Gateway Billing-ի հաշտեցում

- **Պատուհան:** `<from>/<to>`
- **Վարձակալ՝** `<tenant-id>`
- ** Կատալոգի տարբերակ:** `<catalog-version>`
- **Օգտագործման ակնարկ.** `<path or hash>`
- **Պահարաններ՝** փափուկ գլխարկ `<soft-cap-xor> XOR`, կոշտ գլխարկ `<hard-cap-xor> XOR`, ազդանշանային շեմ `<alert-threshold>%`
- **Վճարող -> Գանձապետարան.** `<payer>` -> `<treasury>` `<asset-definition>`-ում
- **Ընդհանուր ժամկետանց.** `<total-xor> XOR` (`<total-micros>` միկրո-XOR)

## Գծային կետի ստուգում
- [ ] Օգտագործման գրառումներն ընդգրկում են միայն կատալոգի հաշվիչի ID-ները և վավեր հաշվարկային շրջանները
- [ ] Քանակի միավորները համապատասխանում են կատալոգի սահմանումներին (հարցումներ, GiB, ms և այլն)
- [ ] Տարածաշրջանի բազմապատկիչները և զեղչերի մակարդակները կիրառվում են ըստ կատալոգի
- [ ] CSV/մանրահատակի արտահանումները համընկնում են JSON ապրանքագրի տողերի հետ

## Պաշտպանիչի գնահատում
- [ ] Հասե՞լ է արդյոք փափուկ գլխարկի ազդանշանային շեմը: `<yes/no>` (եթե այո, կցեք ահազանգի ապացույցը)
- [ ] Կոշտ գլխարկը գերազանցե՞լ է: `<yes/no>` (եթե այո, կցեք վերացման հաստատումը)
- [ ] Հաշիվ-ապրանքագրի նվազագույն մակարդակը բավարարված է

## Լեջերի պրոյեկցիա
- [ ] Փոխանցման խմբաքանակի ընդհանուր գումարը հավասար է `total_micros` ապրանքագրի մեջ
- [ ] Ակտիվների սահմանումը համապատասխանում է հաշվարկային արժույթին
- [ ] Վճարողի և գանձապետական հաշիվները համապատասխանում են գրանցման վարձակալին և օպերատորին
- [ ] Norito/JSON արտեֆակտները կցված են աուդիտի վերարտադրման համար

## Վեճ/ճշգրտման նշումներ
- Դիտարկված շեղում. `<variance detail>`
- Առաջարկվող կարգավորում՝ `<delta and rationale>`
- Աջակցող ապացույց՝ `<logs/dashboards/alerts>`

## Հաստատումներ
- Վճարման վերլուծաբան՝ `<name + signature>`
- Գանձապետարանի վերանայող՝ `<name + signature>`
- Կառավարման փաթեթի հեշ՝ `<hash/reference>`

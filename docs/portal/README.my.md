#SORA Nexus Developer Portal

ဤလမ်းညွှန်သည် အပြန်အလှန်အကျိုးပြုသောဆော့ဖ်ဝဲရေးသားသူအတွက် Docusaurus အလုပ်ခွင်ကို လက်ခံထားသည်။
ပေါ်တယ် ပေါ်တယ်သည် Norito လမ်းညွှန်များ၊ SDK အမြန်စတင်ခြင်းနှင့် OpenAPI တို့ကို စုစည်းထားသည်။
`cargo xtask openapi` မှထုတ်ပေးသော ကိုးကားချက်၊ ၎င်းတို့ကို SORA Nexus ဖြင့် ထုပ်ပိုးထားသည်
docs ပရိုဂရမ်တွင် အသုံးပြုသည့် အမှတ်တံဆိပ်တစ်ခု။

## လိုအပ်ချက်များ

- Node.js 18.18 နှင့်အထက် (Docusaurus v3 အခြေခံလိုင်း)။
- ထုပ်ပိုးစီမံခန့်ခွဲမှုအတွက် ချည် 1.x သို့မဟုတ် npm ≥ 9။
- Rust toolchain (OpenAPI စင့်ခ် script မှအသုံးပြုသည်)။

## Bootstrap

```bash
cd docs/portal
npm install    # or yarn install
```

## ရနိုင်သော scripts များ

| အမိန့် | ဖော်ပြချက် |

| `npm run start` / `yarn start` | တိုက်ရိုက်ပြန်လည်ဒေါင်းလုဒ်ဖြင့် ဒေသတွင်း dev ဆာဗာကို ဖွင့်ပါ (`http://localhost:3000` သို့ ပုံသေများ)။ |
| `npm run build` / `yarn build` | `build/` တွင် ထုတ်လုပ်မှုတည်ဆောက်မှုကို ထုတ်လုပ်ပါ။ |
| `npm run serve` / `yarn serve` | နောက်ဆုံးပေါ်တည်ဆောက်မှုကို ပြည်တွင်းတွင် ဆောင်ရွက်ပေးပါ (မီးခိုးစမ်းသပ်မှုများအတွက် အသုံးဝင်သည်)။ |
| `npm run docs:version -- <label>` | လက်ရှိ docs များကို `versioned_docs/version-<label>` (`docusaurus docs:version` ပတ်ပတ်လည်တွင် ထုပ်ပိုးပါ)။ |
| `npm run sync-openapi` / `yarn sync-openapi` | `static/openapi/torii.json` ကို `cargo xtask openapi` မှတစ်ဆင့် ပြန်ထုတ်ပါ ( Spec ကို အပိုဗားရှင်း လျှပ်တစ်ပြက်ပုံများအဖြစ် ကူးယူရန် `--mirror=<label>` ကို ကျော်ပါ)။ |
| `npm run tryit-proxy` | “စမ်းကြည့်ပါ” ကွန်ဆိုးလ်ကို စွမ်းအားပေးသည့် အဆင့်မြှင့်တင်သည့် ပရောက်စီကို စတင်ပါ (ဖွဲ့စည်းပုံအတွက် အောက်တွင် ကြည့်ပါ)။ |
| `npm run probe:tryit-proxy` | ပရောက်စီ (CI/စောင့်ကြည့်ရေးအထောက်အကူ) ကို ဆန့်ကျင်သည့် `/healthz` + နမူနာတောင်းဆိုချက်ကို စုံစမ်းစစ်ဆေးပါ။ |
| `npm run manage:tryit-proxy -- <update|rollback>` | အရန်ကိုင်တွယ်မှုဖြင့် ပရောက်စီ `.env` ပစ်မှတ်ကို အပ်ဒိတ် သို့မဟုတ် ပြန်လည်ရယူပါ။ |
| `npm run sync-i18n` | ဂျပန်၊ ဟီဘရူး၊ စပိန်၊ ပေါ်တူဂီ၊ ပြင်သစ်၊ ရုရှ၊ အာရဗီ၊ အူရဒူ၊ ဗမာ၊ ဂျော်ဂျီယံ၊ အာမေးနီးယန်း၊ အဇာဘိုင်ဂျနီ၊ ကာဇတ်၊ Bashkir၊ Amharic၊ Dzongkha၊ Uzbek၊ မွန်ဂိုလီးယား၊ ရိုးရာတရုတ်နှင့် ရိုးရှင်းသော တရုတ်ဘာသာပြန်ဆိုချက်များသည် `i18n/` အောက်တွင် ရှိနေကြောင်း သေချာပါစေ။ |
| `npm run sync-norito-snippets` | စီစဉ်ပေးထားသော Kotodama ဥပမာ docs + ဒေါင်းလုဒ်လုပ်နိုင်သော အတိုအထွာများကို ပြန်ထုတ်ပါ (dev ဆာဗာပလပ်အင်က အလိုအလျောက် ခေါ်ဆိုသည်)။ |
| `npm run test:tryit-proxy` | Node ၏စမ်းသပ်မှုအပြေးသမား (`node --test`) မှတစ်ဆင့် proxy ယူနစ်စမ်းသပ်မှုများကို လုပ်ဆောင်ပါ။ |

OpenAPI စင့်ခ် script သည် `cargo xtask openapi` ထံမှ ရနိုင်သည် လိုအပ်သည်
repository အမြစ်; ၎င်းသည် `static/openapi/` သို့ တိကျသော JSON ဖိုင်ကို ထုတ်လွှတ်သည်။
ယခု Torii router သည် တိုက်ရိုက် spec ကို ထုတ်ဖော်ရန် မျှော်လင့်သည် (အသုံးပြုပါ။
`cargo xtask openapi --allow-stub` အရေးပေါ်နေရာယူထားသည့် အထွက်အတွက်သာ)။

## Docs ဗားရှင်းနှင့် OpenAPI လျှပ်တစ်ပြက်ရိုက်ချက်များ

- ** docs ဗားရှင်းကို ဖြတ်တောက်ခြင်း-** `npm run docs:version -- 2025-q3` (သို့မဟုတ် သဘောတူထားသည့် tag) ကို run ပါ။
  ထုတ်လုပ်လိုက်တဲ့ `versioned_docs/version-<label>`, `versioned_sidebars`၊
  နှင့် `versions.json`။ navbar ဗားရှင်း dropdown သည် အလိုအလျောက်ပေါ်လာသည်။
  လျှပ်တစ်ပြက်အသစ်။
- **OpenAPI ရှေးဟောင်းပစ္စည်းများကို စင့်ခ်လုပ်ခြင်း-** ဗားရှင်းတစ်ခုကို ဖြတ်တောက်ပြီးနောက်၊ canonical ကို ပြန်လည်စတင်ပါ။
  Spec နှင့် `cargo xtask openapi --sign <path-to-ed25519-key>` မှတဆင့် ဖော်ပြပါ
  လိုက်ဖက်သော လျှပ်တစ်ပြက် ရိုက်ယူပါ။
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`။ ဟိ
  script သည် `static/openapi/versions/2025-q3/torii.json` ကိုရေးသားသည်၊ spec ကိုထင်ဟပ်စေသည်။
  `versions/current/torii.json` သို့၊ `versions.json` ကို အပ်ဒိတ်လုပ်သည်၊ ပြန်လည်စတင်သည်
  `/openapi/torii.json` နှင့် ရေးထိုးထားသော `manifest.json` ကို ဗားရှင်းတစ်ခုစီသို့ ပုံတူပွားသည်
  ထို့ကြောင့် သမိုင်းဆိုင်ရာ သတ်မှတ်ချက်များသည် တူညီသော သက်သေပြချက် မက်တာဒေတာကို သယ်ဆောင်သွားသော လမ်းညွှန်ချက်ဖြစ်သည်။ ပေးဝေတာမျိုး
  အသစ်ထုတ်လုပ်ထားသော spec ထဲသို့ကူးယူရန် `--mirror=<label>` အလံအရေအတွက်
  အခြားသမိုင်းဝင်ဓာတ်ပုံများ။
- **CI မျှော်မှန်းချက်များ-** ထိထိမိမိ ကတိပြုထားသည့် docs ဗားရှင်းတွင် ထိခိုက်မှု ပါဝင်သင့်သည်။
  (အသုံးပြုနိုင်သည့်အခါ) နှင့် OpenAPI လျှပ်တစ်ပြက်ရိုက်ချက်များကို ပြန်လည်ဆန်းသစ်ထားသောကြောင့် Swagger၊ RapiDoc၊
  နှင့် Redoc အကန့်များသည် ထုတ်ယူမှု အမှားအယွင်းများမရှိဘဲ သမိုင်းဆိုင်ရာ သတ်မှတ်ချက်များအကြား ပြောင်းနိုင်သည်။
- **Manifest enforcement-** `sync-openapi` script ကို လက်မှတ်ထိုးလိုက်သောအခါ ယခုပျက်သွားသည်
  `manifest.json` ပျောက်ဆုံးနေသည်၊ ပုံပျက်နေသည် သို့မဟုတ် အသစ်ထုတ်လုပ်လိုက်သော ကိုက်ညီမှုမရှိပါ
  spec ဖြစ်သောကြောင့် လက်မှတ်မထိုးထားသော လျှပ်တစ်ပြက်ရိုက်ချက်များကို မူရင်းအတိုင်း မထုတ်ဝေနိုင်ပါ။ ပြန်ပြေးတယ်။
  Canonical manifest ကို ပြန်လည်ဆန်းသစ်ရန် `cargo xtask openapi --sign <key>`
  ထပ်တူပြုခြင်းအား ပြန်လည်လုပ်ဆောင်ခြင်းဖြင့် ဗားရှင်းပြုလုပ်ထားသော လျှပ်တစ်ပြက်ရိုက်ချက်များသည် လက်မှတ်ထိုးထားသော မက်တာဒေတာကို ကောက်ယူပါ။
  ပြည်တွင်းအကြိုကြည့်ရှုမှုများအတွက်သာ `--allow-unsigned` ကို ဖြတ်ပါ (CI သည် ဆက်လက်လုပ်ဆောင်နေဆဲဖြစ်သည်။
  `ci/check_openapi_spec.sh`၊ သည် spec ကိုပြန်ထုတ်ပြီး ၎င်းကိုစစ်ဆေးသည်။
  ပေါင်းစည်းရန် ကတိကဝတ်များကို ခွင့်ပြုခြင်းမပြုမီ ဖော်ပြပါ။)

### ၎င်းကို proxy ဖွဲ့စည်းမှုပုံစံကို စမ်းကြည့်ပါ။

"စမ်းကြည့်ပါ" sandbox လမ်းကြောင်းများသည် `scripts/tryit-proxy.mjs` မှတဆင့် တောင်းဆိုသည်။ စီစဉ်ပေးသည်။
၎င်းကိုမစတင်မီ ပရောက်စီသည် ပတ်ဝန်းကျင် variable များပါရှိပါသည်-

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN` (မူလ `127.0.0.1:8787`) သည် bind address ကို ထိန်းချုပ်သည်။
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` in-memory ကို configure လုပ်ပါ
  နှုန်းကန့်သတ်ချက် (စက္ကန့် 60 လျှင် တောင်းဆိုချက် 60 သို့ ပုံသေ)။
- ခေါ်ဆိုသူ၏ `Authorization` ကို ထပ်ဆင့်ပေးပို့ရန် `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1` ကို သတ်မှတ်ပါ
  default bearer token ကို အားကိုးမည့်အစား header
- `static/openapi/torii.json` မှ လွင့်သွားပါက proxy သည် စတင်ရန် ငြင်းဆိုပါသည်။
  `static/openapi/manifest.json` တွင် manifest ရေးထိုးထားသည်။ ပြေး
  Spec ကို ပြန်လည်ဆန်းသစ်ရန် `npm run sync-openapi -- --latest` တင်ပို့ခြင်း။
  အရေးပေါ်အခြေအနေအတွက်သာ `TRYIT_PROXY_ALLOW_STALE_SPEC=1` (သတိပေးချက်တစ်ခုဖြစ်သည်။
  အကောင့်ဝင်ပြီး proxy စတင်သည်)။
- Staging proxy ကို မီးခိုးထုတ်ပြီး စမ်းသပ်ရန် `npm run probe:tryit-proxy` ကိုဖွင့်ပါ။
  စောင့်ကြည့်ရေးအလုပ်များသို့ အမိန့်ပေးခြင်း၊ `npm run manage:tryit-proxy -- update` ရိုးရှင်းပါသည်။
  ပြန်လှည့်ရန်အတွက် `.env` အရန်သိမ်းထားစဉ် Torii အဆုံးမှတ်များကို လှည့်နေသည်။
- `TRYIT_PROXY_PROBE_METRICS_FILE` နှင့် `TRYIT_PROXY_PROBE_LABELS` တို့ကို စုံစမ်းစစ်ဆေးခွင့်ပြုပါ
  I18NT0000000X စာသားဖိုင်ဖော်မတ်တွင် `probe_success`/`probe_duration_seconds` ကို ထုတ်လွှတ်သည်
  ကျန်းမာရေးစစ်ဆေးမှုကို node_exporter သို့မဟုတ် batch collector နှင့် ချိတ်နိုင်သည်။
- Prometheus ခြစ်ရာ `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  histogram စီးရီး; docs portal Grafana ဒိုင်ခွက်သည် p95/p99 latency အတွက် ၎င်းတို့ကို စားသုံးသည်
  SLO ခြေရာခံခြင်း (`dashboards/grafana/docs_portal.json`)။

### OAuth စက်ပစ္စည်း-ကုဒ် အကောင့်ဝင်ခြင်း။

ပေါ်တယ်ကို ယုံကြည်ရသော ကွန်ရက်များ အပြင်ဘက်တွင် ထိတွေ့သောအခါ၊ OAuth စက်ပစ္စည်းကို စီစဉ်သတ်မှတ်ပါ။
ခွင့်ပြုချက်ကြောင့် သုံးသပ်သူများသည် သက်တမ်းရှည် Torii တိုကင်များကို ဘယ်သောအခါမှ မထိပါ။ တင်ပို့ပါ။
`npm run start` သို့မဟုတ် `npm run build` ကိုမဖွင့်မီ အောက်ပါကိန်းရှင်များ

| ပြောင်းလဲနိုင်သော | မှတ်စုများ |
|----------|------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` | Device Authorization Grant ကို အကောင်အထည်ဖော်သော OAuth အဆုံးမှတ်များ။ |
| `DOCS_OAUTH_CLIENT_ID` | စာရွက်စာတမ်းအစမ်းကြည့်ရှုခြင်းအတွက် Client ID မှတ်ပုံတင်ထားသည်။ |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` | ထုတ်ပေးထားသော တိုကင်ကို ကန့်သတ်ရန် ရွေးချယ်နိုင်သော နယ်ပယ်/ပရိသတ်စာကြောင်းများ။ |
| `DOCS_OAUTH_POLL_INTERVAL_MS` | ပေါ်တယ်ဝစ်ဂျက်ဖြင့် ပြဋ္ဌာန်းထားသည့် အနည်းဆုံး မဲဆန္ဒပေးသည့်ကြားကာလ (မူလ 5000ms အထိ၊ အောက်တန်ဖိုးများကို ပယ်ချသည်)။ |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` | ဆာဗာသည် `expires_in` ကို ချန်လှပ်ထားသောအခါတွင် အသုံးပြုသော သက်တမ်းကုန်ဆုံးသည့် ဝင်းဒိုးများ။ |
| `DOCS_OAUTH_ALLOW_INSECURE` | တည်ဆောက်ချိန် အစောင့်အကြပ်ကို ကျော်ဖြတ်ရန် ဒေသဆိုင်ရာ ဖွံ့ဖြိုးတိုးတက်မှုအတွက်သာ `1` ဟု သတ်မှတ်ထားသည်။ လိုအပ်သော ကိန်းရှင်များ ပျောက်ဆုံးပါက သို့မဟုတ် TTL များသည် ကန့်သတ်ထားသော အပိုင်းအခြားများ အပြင်ဘက်တွင် ပြုတ်ကျပါက ထုတ်လုပ်မှု တည်ဆောက်မှုများ မအောင်မြင်ပါ။ |

ဥပမာ-

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

build သည် ဤတန်ဖိုးများကို ကောက်ယူပြီးနောက်၊ Try it console သည် အကောင့်ဝင်ခြင်းကို လုပ်ဆောင်သည်။
စက်ကုဒ်ကိုပြသသော အကန့်၊ တိုကင်အဆုံးမှတ်ကို စစ်တမ်းကောက်ယူပြီး Bearer အားဖြည့်ပေးသည်။
အကွက်မှာ အလိုအလျောက်ဖြစ်ပြီး သက်တမ်းကုန်ဆုံးသည့်အခါ တိုကင်ကို ရှင်းလင်းပါ။ ပေါ်တယ်က ငြင်းပါတယ်။
OAuth ဖွဲ့စည်းမှု မပြည့်စုံသောအခါ သို့မဟုတ် တိုကင်/စက်ပစ္စည်း TTL များ ပြုတ်ကျသည့်အခါ စတင်ပါ။
လုံခြုံရေးဘတ်ဂျက်ပြင်ပ၊ `DOCS_OAUTH_ALLOW_INSECURE=1` ပေါ်တွင်သာ တင်ပို့ပါ။
အမည်မသိအသုံးပြုခွင့်ကို လက်ခံနိုင်သော တစ်ခါသုံး ဒေသတွင်း အစမ်းကြည့်ရှုမှုများ။ ပရောက်စီက ငြိမ်နေတယ်။
ဂုဏ်ထူးဆောင်လက်စွဲ `X-TryIt-Auth` ကို အစားထိုးထားသည်၊ ထို့ကြောင့် သင်သည် OAuth ကိန်းရှင်များကို ဖယ်ရှားနိုင်သည်
ad-hoc တိုကင်များကို သင်ထည့်လိုသည့်အခါတိုင်း။

[`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md) ကိုကြည့်ပါ
ခြိမ်းခြောက်မှုပုံစံအပြည့်အစုံနှင့် ဘောပင်စမ်းသပ်ဂိတ်များ။

### လုံခြုံရေး ခေါင်းစီးများ

`docusaurus.config.js` သည် ယခုအခါတွင် သတ်မှတ်ထားသော လုံခြုံရေး ခေါင်းစီးများ—အကြောင်းအရာ လုံခြုံရေးကို ထုတ်လွှတ်သည်။
မူဝါဒ၊ ယုံကြည်စိတ်ချရသောအမျိုးအစားများ၊ ခွင့်ပြုချက်များ-မူဝါဒနှင့် ရည်ညွှန်းသူ-မူဝါဒ—ထို့ကြောင့် တည်ငြိမ်သောအိမ်ရှင်များ
dev ဆာဗာကဲ့သို့ တူညီသော အစောင့်သံလမ်းများကို အမွေဆက်ခံသည်။ ပုံသေများသည် script များကိုသာခွင့်ပြုသည်။
ပေါ်တယ်မူရင်းမှ ဆောင်ရွက်ပေးပြီး မူရင်း `connect-src` သို့ ကူးသန်းသွားလာမှုကို ကန့်သတ်ပါ
ပြင်ဆင်ထားသော ခွဲခြမ်းစိတ်ဖြာမှု အဆုံးမှတ်နှင့် စမ်းသုံးကြည့်ပါ ပရောက်စီ။ ထုတ်လုပ်မှုကို ငြင်းဆိုသည်။
HTTPS မဟုတ်သော ပိုင်းခြားစိတ်ဖြာချက်များ/စမ်းသုံးရန် အဆုံးမှတ်များ။ `http://` နှင့် ဆန့်ကျင်သည့် ပြည်တွင်းအကြိုကြည့်ရှုမှုများအတွက်
ပစ်မှတ်များကို `DOCS_SECURITY_ALLOW_INSECURE=1` တွင် အတိအလင်းသတ်မှတ်ထားသောကြောင့် guardrail
အဆင့်နှိမ့်ချမှုကို အသိအမှတ်ပြုသည်။

### ဒေသသတ်မှတ်ခြင်းလုပ်ငန်းအသွားအလာ

Portal သည် မူရင်းဘာသာစကားအဖြစ် အင်္ဂလိပ်နှင့် ဂျပန်၊ ဟီးဘရူး၊
စပိန်၊ ပေါ်တူဂီ၊ ပြင်သစ်၊ ရုရှ၊ အာရဗီ၊ အူရဒူး၊ မြန်မာ၊ ဂျော်ဂျီယာ၊
အာမေးနီးယန်း၊ အဇာဘိုင်ဂျနီ၊ ကာဇက်၊ ဘတ်ရှ်ကီ၊ အမ်ဟာရစ်၊ ဇုံခါ၊ ဥဇဘက်၊ မွန်ဂိုလီယန်၊
တရုတ်ရိုးရာနှင့် ရိုးရှင်းသော တရုတ်ဒေသခံများ။ ဒေါ့ခ်မြေသို့ရောက်တိုင်း၊
ပြေး:

```bash
npm run sync-i18n
```

script သည် repository-wide `scripts/sync_docs_i18n.py` အပြုအမူကို ထင်ဟပ်စေသည်
`i18n/<lang>/docusaurus-plugin-content-docs/current/...` အောက်တွင် ဆောင်းပါးတိုဘာသာပြန်များကို ဖန်တီးခြင်း။
ထို့နောက် တည်းဖြတ်သူများသည် နေရာယူထားသော စာသားကို အစားထိုးနိုင်ပြီး ရှေ့အကြောင်းအရာကို အပ်ဒိတ်လုပ်နိုင်သည်။
မက်တာဒေတာ (`status`၊ `translation_last_reviewed` စသည်ဖြင့်)။

မူရင်းအားဖြင့်၊ ထုတ်လုပ်မှုတည်ဆောက်မှုများတွင် ဖော်ပြထားသော ဒေသများသာ ပါဝင်သည်။
`docs/i18n/published_locales.json` (လောလောဆယ် အင်္ဂလိပ်-သီးသန့်) ထို့ကြောင့် ဆောင်းပါးတို နေရာယူထားသည်။
မပို့ကြပါ။ အစမ်းကြည့်ရန် လိုအပ်သောအခါ `DOCS_I18N_INCLUDE_STUBS=1` ကို သတ်မှတ်ပါ။
ဒေသအလိုက် ဆောင်ရွက်ဆဲ ဒေသများ။

## CI & ဖြန့်ကျက်ခြင်း။

- `ci/check_docs_portal.sh` သည် `node scripts/check-sdk-recipes.mjs` ကို ဦးစွာလုပ်ဆောင်ပြီး SDK လယ်ဂျာ ချက်ပြုတ်နည်းများသည် ၎င်းတို့၏ canonical source ဖိုင်များနှင့် ကိုက်ညီကြောင်း အတည်ပြုရန်၊ ထို့နောက် `npm ci|install`၊ `npm run build` ကို လုပ်ဆောင်ပြီး သော့ကဏ္ဍများ (ပင်မစာမျက်နှာ 00202NT Norito၊ Publishing checklist) သည် ထုတ်လုပ်ထားသော HTML တွင် ရှိနေပါသည်။
- `ci/check_openapi_spec.sh` သည် `cargo xtask openapi` မှတစ်ဆင့် Torii spec ကို ပြန်လည်ထုတ်ပေးသည်၊
  `static/openapi/torii.json` နှင့် `versions/current/torii.json` နှင့် နှိုင်းယှဉ်သည် ။
  ခြေရာခံထားသော spec သို့မဟုတ် hash သည် PRs များပျက်သွားသောအခါ checksum manifest ကိုစစ်ဆေးသည်။
  ပုပ်သိုးနေသည်။

- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]` သည် ၎င်းတို့ကို လက်မှတ်မထိုးမီ သို့မဟုတ် ops များသို့ ပို့ဆောင်ခြင်းမပြုမီ Gateway ခွင့်ပြုချက်မှတ်တမ်းများကို တရားဝင်စေပါသည်။ အကူအညီပေးသူက Canonical/pretty hosts၊ manifest metadata နှင့် telemetry အညွှန်းများသည် တိကျသောမူဝါဒနှင့်ကိုက်ညီကြောင်း သေချာစေပြီး `--json-out` မှတစ်ဆင့် DG-3 အထောက်အထားအတွက် JSON အကျဉ်းချုပ်ကို ထုတ်လွှတ်နိုင်သည်။
- တင်ပြမှုတစ်ခုဖြစ်ပေါ်လာသောအခါ၊ pin helper သည် `artifacts/sorafs/portal.dns-cutover.json` ကိုထုတ်လုပ်ရန် `scripts/generate-dns-cutover-plan.mjs` ကိုတောင်းဆိုပါသည်။ `DNS_CHANGE_TICKET`၊ ​​`DNS_CUTOVER_WINDOW`၊ `DNS_HOSTNAME`၊ `DNS_ZONE` နှင့် `DNS_OPS_CONTACT` သတ်မှတ်ပါ (သို့မဟုတ် သက်ဆိုင်ရာ `--dns-*` ထုတ်လုပ်မှုအလံများကို ဖြတ်တောက်ရန်) ထို့ကြောင့် metadata ကိုဖော်ပြရန် လိုအပ်ပါသည်။ ကက်ရှ်တရားဝင်ခြင်း နှင့် နောက်ပြန်ချိတ်ခြင်းများ လိုအပ်သောအခါ၊ `DNS_CACHE_PURGE_ENDPOINT`၊ `DNS_CACHE_PURGE_AUTH_ENV` နှင့် `DNS_PREVIOUS_PLAN` (သို့မဟုတ် `--cache-purge-*` / `--previous-dns-plan` အလံများထည့်ပါ) ထို့ကြောင့် ကြိုတင်ဖော်ပြချက်များနှင့် API မှတ်တမ်းများကို ပြန်လည်ခေါ်ဆိုပါသည်။ ယခုဖော်ပြသူသည် ချုပ်လုပ်ထားသော `Sora-Route-Binding` (အိမ်ရှင်၊ CID၊ ခေါင်းစီး/ချိတ်တွဲထားသော လမ်းကြောင်းများ၊ အတည်ပြုခြင်းအမိန့်များ)ကိုလည်း မှတ်တမ်းတင်ထားသောကြောင့် GAR ပရိုမိုးရှင်းနှင့် ပြန်လည်သုံးသပ်ခြင်းအစီအစဉ်သည် အစွန်းတွင် ဖော်ပြထားသော အတိအကျခေါင်းစီးများကို ရည်ညွှန်းကိုးကားပါသည်။
- တူညီသောအလုပ်အသွားအလာသည် `scripts/sns_zonefile_skeleton.py` မှတစ်ဆင့် SNS zonefile skeleton/resolver snippet ကို ထုတ်လွှတ်နိုင်သည်။ IPv4/IPv6/CNAME/SPKI/TXT မက်တာဒေတာကို ပေးဆောင်ပါ (`DNS_ZONEFILE_IPV4` ကဲ့သို့သော env var များ သို့မဟုတ် `--dns-zonefile-ipv4` ကဲ့သို့ CLI အလံများ)၊ GAR အချေအတင်ကို `DNS_GAR_DIGEST` ဖြင့် ကျော်ဖြတ်ပြီး အကူအညီပေးသူက I100000190X မှ အလိုအလျောက် ဖြေရှင်းပေးမည် (I100000190X) ထို့ကြောင့် SN-7 အထောက်အထားများသည် ဖြတ်တောက်ထားသော ဖော်ပြချက်နှင့် တွဲလျက် ရှိသည်။
- DNS ဖော်ပြချက်သည် ယခု `gateway_binding` အပိုင်းကို မြှုပ်နှံထားသည် (လမ်းကြောင်းများ၊ အကြောင်းအရာ CID၊ အထောက်အထားအခြေအနေ နှင့် စာသားခေါင်းစီးပုံစံ) တို့ကို ရည်ညွှန်းသော `gateway_binding` အပိုင်းကို မြှုပ်နှံထားသောကြောင့် DG-3 ၏ အတည်ပြုချက်များတွင် ဂိတ်ဝတွင် မျှော်လင့်ထားသည့် အတိအကျ `Sora-Name/Sora-Proof/CSP/HSTS` အတွဲပါဝင်သည်။

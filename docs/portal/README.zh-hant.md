# SORA Nexus 開發者門戶

此目錄託管交互式開發人員的 Docusaurus 工作區
門戶網站。該門戶匯集了 Norito 指南、SDK 快速入門和 OpenAPI
由 `cargo xtask openapi` 生成的引用，將它們包裝在 SORA Nexus 中
整個文檔計劃中使用的品牌。

## 先決條件

- Node.js 18.18 或更高版本（Docusaurus v3 基線）。
- Yarn 1.x 或 npm ≥ 9 用於包管理。
- Rust 工具鏈（由 OpenAPI 同步腳本使用）。

## 引導程序

```bash
cd docs/portal
npm install    # or yarn install
```

## 可用腳本

|命令|描述 |
|---------|-------------|
| `npm run start` / `yarn start` |啟動具有實時重新加載功能的本地開發服務器（默認為 `http://localhost:3000`）。 |
| `npm run build` / `yarn build` |在 `build/` 中生成生產版本。 |
| `npm run serve` / `yarn serve` |在本地提供最新版本（對於冒煙測試有用）。 |
| `npm run docs:version -- <label>` |將當前文檔快照到 `versioned_docs/version-<label>`（圍繞 `docusaurus docs:version` 的包裝）。 |
| `npm run sync-openapi` / `yarn sync-openapi` |通過 `cargo xtask openapi` 重新生成 `static/openapi/torii.json`（通過 `--mirror=<label>` 將規範複製到其他版本快照中）。 |
| `npm run tryit-proxy` |啟動為“Try it”控制台提供支持的臨時代理（請參閱下面的配置）。 |
| `npm run probe:tryit-proxy` |針對代理（CI/監控幫助程序）發出 `/healthz` + 示例請求探測。 |
| `npm run manage:tryit-proxy -- <update|rollback>` |通過備份處理更新或恢復代理 `.env` 目標。 |
| `npm run sync-i18n` |確保 `i18n/` 下存在日語、希伯來語、西班牙語、葡萄牙語、法語、俄語、阿拉伯語、烏爾都語、緬甸語、格魯吉亞語、亞美尼亞語、阿塞拜疆語、哈薩克語、巴什基爾語、阿姆哈拉語、宗卡語、烏茲別克語、蒙古語、繁體中文和簡體中文翻譯存根。 |
| `npm run sync-norito-snippets` |重新生成精選的 Kotodama 示例文檔 + 可下載的片段（也由開發服務器插件自動調用）。 |
| `npm run test:tryit-proxy` |通過 Node 的測試運行程序 (`node --test`) 執行代理單元測試。 |

OpenAPI 同步腳本要求 `cargo xtask openapi` 可從
存儲庫根；它將確定性 JSON 文件發送到 `static/openapi/` 中
現在期望 Torii 路由器公開實時規範（使用
`cargo xtask openapi --allow-stub` 僅用於緊急佔位符輸出）。

## 文檔版本控制和 OpenAPI 快照

- **剪切文檔版本：** 運行 `npm run docs:version -- 2025-q3` （或任何商定的標籤）。
  提交生成的`versioned_docs/version-<label>`、`versioned_sidebars`、
  和 `versions.json`。導航欄版本下拉列表會自動顯示
  新快照。
- **同步 OpenAPI 工件：** 剪切版本後，刷新規範
  通過 `cargo xtask openapi --sign <path-to-ed25519-key>` 進行規範和清單，然後
  捕獲匹配的快照
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`。的
  腳本寫入 `static/openapi/versions/2025-q3/torii.json`，反映規範
  進入 `versions/current/torii.json`，更新 `versions.json`，刷新
  `/openapi/torii.json`，並將簽名的 `manifest.json` 克隆到每個版本中
  目錄，因此歷史規格具有相同的出處元數據。供應任何
  將新生成的規範複製到的 `--mirror=<label>` 標誌的數量
  其他歷史快照。
- **CI 期望：** 提交相關文檔應包含版本提升
  （如果適用）加上刷新的 OpenAPI 快照，以便 Swagger、RapiDoc、
  Redoc 面板可以在歷史規格之間切換，而不會出現獲取錯誤。
- **清單執行：** `sync-openapi` 腳本現在在簽名時失敗
  `manifest.json` 缺失、格式錯誤或與新生成的不匹配
  規範，因此默認情況下無法發布未簽名的快照。重新運行
  `cargo xtask openapi --sign <key>` 刷新規范清單並
  重新運行同步，以便版本化快照獲取簽名的元數據。
  僅針對本地預覽傳遞 `--allow-unsigned`（CI 仍運行
  `ci/check_openapi_spec.sh`，重新生成規範並驗證
  在允許提交合併之前清單）。

### 嘗試代理配置

“Try it”沙箱通過 `scripts/tryit-proxy.mjs` 路由請求。配置
在啟動代理之前使用環境變量：

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN`（默認`127.0.0.1:8787`）控制綁定地址。
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` 配置內存
  速率限制器（默認為每 60 秒 60 個請求）。
- 設置 `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1` 轉發主叫的 `Authorization`
  header 而不是依賴默認的承載令牌。
- 如果 `static/openapi/torii.json` 偏離，則代理拒絕啟動
  在 `static/openapi/manifest.json` 中籤署清單。運行
  `npm run sync-openapi -- --latest` 刷新規格；出口
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` 僅用於緊急超越（警告
  已記錄並且代理無論如何都會啟動）。
- 運行 `npm run probe:tryit-proxy` 對登台代理進行冒煙測試並連接
  指揮監控工作； `npm run manage:tryit-proxy -- update` 簡化
  輪換 Torii 端點，同時保留 `.env` 備份以進行回滾。
- `TRYIT_PROXY_PROBE_METRICS_FILE` 和 `TRYIT_PROXY_PROBE_LABELS` 讓探頭
  以 Prometheus 文本文件格式發出 `probe_success`/`probe_duration_seconds`
  您可以將運行狀況檢查掛接到 node_exporter 或任何批處理收集器中。
- Prometheus 刮擦暴露 `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  直方圖系列；文檔門戶 Grafana 儀表板將它們用於 p95/p99 延遲
  SLO 跟踪 (`dashboards/grafana/docs_portal.json`)。

### OAuth 設備代碼登錄

當門戶暴露在受信任網絡外部時，配置 OAuth 設備
授權，因此審閱者永遠不會接觸長期存在的 Torii 令牌。導出
運行 `npm run start` 或 `npm run build` 之前的變量如下：

|變量|筆記|
|----------|--------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` |實現設備授權的 OAuth 端點。 |
| `DOCS_OAUTH_CLIENT_ID` |為文檔預覽註冊的客戶端 ID。 |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` |用於限制頒發的令牌的可選範圍/受眾字符串。 |
| `DOCS_OAUTH_POLL_INTERVAL_MS` |門戶小部件強制執行的最小輪詢間隔（默認為 5000 毫秒；較低的值將被拒絕）。 |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` |當服務器省略 `expires_in` 時使用的回退到期窗口。 |
| `DOCS_OAUTH_ALLOW_INSECURE` |僅針對本地開發設置為 `1` 以繞過構建時防護。如果缺少必需的變量或 TTL 超出強制範圍，生產構建就會失敗。 |

示例：

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

構建獲取這些值後，Try it 控制台會呈現登錄信息
顯示設備代碼、輪詢令牌端點、填充承載的面板
自動字段，並在過期時清除令牌。門戶網站拒絕
當 OAuth 配置不完整或令牌/設備 TTL 下降時啟動
超出安全預算；僅導出 `DOCS_OAUTH_ALLOW_INSECURE=1`
可接受匿名訪問的一次性本地預覽。代理仍然
遵循手動 `X-TryIt-Auth` 覆蓋，因此您可以刪除 OAuth 變量
每當您喜歡粘貼臨時令牌時。

請參閱 [`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md) 了解
完整的威脅模型和滲透測試門。

### 安全標頭

`docusaurus.config.js` 現在發出確定性安全標頭 — 內容安全
策略、可信類型、權限策略和引用者策略 - 靜態主機
繼承與開發服務器相同的護欄。默認只允許腳本
從門戶源提供服務並將跨源 `connect-src` 流量限制為
配置的分析端點和 Try it 代理。生產構建拒絕
非 HTTPS 分析/試用端點；針對 `http://` 的本地預覽
目標，明確設置 `DOCS_SECURITY_ALLOW_INSECURE=1` 以便護欄
承認降級。

## CI 和部署

- `ci/check_docs_portal.sh` 首先運行 `node scripts/check-sdk-recipes.mjs` 以驗證 SDK 賬本配方與其規范源文件匹配，然後執行 `npm ci|install`、`npm run build`，並斷言關鍵部分（主頁、SoraFS、Norito、發布清單）存在在生成的 HTML 中。
- `ci/check_openapi_spec.sh` 通過 `cargo xtask openapi` 重新生成 Torii 規範，
  將其與 `static/openapi/torii.json` 和 `versions/current/torii.json` 進行比較，
  並驗證校驗和清單，以便 PR 在跟踪規範或哈希時失敗
  已經過時了。
- `.github/workflows/check-docs.yml` 在每個 PR 上運行門戶構建以捕獲內容回歸。
- `.github/workflows/docs-portal-preview.yml` 為拉取請求構建站點，寫入 `build/checksums.sha256`，通過 `sha256sum -c` 進行驗證，將輸出打包為 `artifacts/preview-site.tar.gz`，通過 `scripts/generate-preview-descriptor.mjs` 生成描述符，並上傳靜態站點和元數據工件，以便審閱者可以審核確切的快照，而無需重新運行構建。該工作流程還發布了一個拉取請求評論，總結了清單/存檔摘要（以及 SoraFS 捆綁包，如果可用），以便審閱者無需打開操作日誌即可看到驗證結果。
- `.github/workflows/docs-portal-deploy.yml` 在推送到 `main`/`master` 時將靜態構建發佈到 GitHub Pages，在 `github-pages` 環境 URL 上提供預覽。工作流程在部署之前運行登陸頁面煙霧檢查。
- `scripts/sorafs-package-preview.sh` 將預覽站點轉換為確定性 SoraFS 捆綁包（CAR、計劃、清單），並且當通過 JSON 配置提供憑據時（請參閱 `docs/examples/sorafs_preview_publish.json`），可以將清單提交到暫存 Torii 端點。
- `scripts/preview_verify.sh` 執行審核者之前手動執行的校驗和 + 描述符驗證流程。將其指向提取的 `build/` 目錄（以及可選的描述符/存檔路徑），以確認預覽工件未被篡改。
- `scripts/sorafs-pin-release.sh` 加上 `.github/workflows/docs-portal-sorafs-pin.yml` 自動化生產 SoraFS 管道：構建/測試、CAR + 清單生成、Sigstore 簽名、驗證、可選別名綁定以及用於治理審核的工件上傳。推廣版本時通過 `workflow_dispatch` 觸發工作流程。
- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]` 在簽名或運送給運營商之前驗證網關授權記錄。該幫助程序確保規範/漂亮的主機、清單元數據和遙測標籤與確定性策略匹配，並可以通過 `--json-out` 發出 DG-3 證據的 JSON 摘要。
- 當提交發生時，引腳助手還會調用 `scripts/generate-dns-cutover-plan.mjs` 來生成 `artifacts/sorafs/portal.dns-cutover.json`。設置 `DNS_CHANGE_TICKET`、`DNS_CUTOVER_WINDOW`、`DNS_HOSTNAME`、`DNS_ZONE` 和 `DNS_OPS_CONTACT`（或傳遞相應的 `--dns-*` 標誌），以便描述符攜帶生產 DNS 切換所需的元數據。當需要緩存失效和回滾掛鉤時，添加 `DNS_CACHE_PURGE_ENDPOINT`、`DNS_CACHE_PURGE_AUTH_ENV` 和 `DNS_PREVIOUS_PLAN`（或 `--cache-purge-*` / `--previous-dns-plan` 標誌），以便描述符記錄清除 API 調用和先前的還原描述符。該描述符現在還記錄了裝訂的 `Sora-Route-Binding`（主機、CID、標頭/綁定路徑、驗證命令），因此 GAR 升級和回退計劃審查引用了在邊緣提供的確切標頭。
- 相同的工作流程可以通過 `scripts/sns_zonefile_skeleton.py` 發出 SNS 區域文件骨架/解析器片段。提供 IPv4/IPv6/CNAME/SPKI/TXT 元數據（作為環境變量，如 `DNS_ZONEFILE_IPV4` 或 CLI 標誌，如 `--dns-zonefile-ipv4`），使用 `DNS_GAR_DIGEST` 傳遞 GAR 摘要，幫助程序將自動寫入 `artifacts/sns/zonefiles/<zone>/<hostname>.json`（加上解析器片段），以便 SN-7 證據與切換描述符。
- DNS 描述符現在嵌入了一個 `gateway_binding` 部分，該部分引用上述工件（路徑、內容 CID、證明狀態和文字標頭模板），因此 DG-3 變更批准包括網關上預期的確切 `Sora-Name/Sora-Proof/CSP/HSTS` 捆綁包。

每個 `npm run build` 調用都會觸發 `postbuild` 鉤子，該鉤子生成
`build/checksums.sha256`。構建（或下載 CI 工件）後，運行
`./docs/portal/scripts/preview_verify.sh --build-dir docs/portal/build` 至
驗證清單。通過 `--descriptor <path>`/`--archive <path>` 時
驗證來自 CI 的元數據包，以便腳本可以交叉檢查記錄的數據
摘要和文件名。
- 要安全地共享本地預覽，請運行 `npm run serve`；該命令現在換行
  `scripts/serve-verified-preview.mjs`，執行`preview_verify.sh`
  在啟動 `docusaurus serve` 之前，如果清單檢查失敗或則中止
  丟失，即使在 CI 之外也能保持預覽校驗和門控。的
  `npm run serve:verified` 別名仍然可用於顯式調用。

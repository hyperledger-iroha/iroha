# Nexus と WSV の参加モデル

> **エスタド:** 魅力的な NX-1 - レーンの分類、構成、レイアウト、実装時のストレージ リスト。

## コンセプトス

- **レーン:** Nexus のシャード ロジックと台帳の有効性とバックログの排出に関する固有の設定。 `LaneId` の識別が可能です。
- **データ スペース:** 政治のコンプライアンス、ルーティング、決済を管理するためのガバナンスを管理するためのバケットです。
- **レーンマニフェスト:** ガバナンス用のメタデータ制御では、バリデーション、DA の政治、トークン、ガス、決済規則、ルーティングの許可を記述します。
- **グローバルな取り組み:** レーン間の履歴書を提出し、レーン間の移動、居住地への移動を証明します。エル・アニロ NPoS の世界的な取り組み。

## レーンのジオメトリ構成 (`LaneConfig`)

`LaneConfig` ジオメトリア ランタイムのカタログの有効性を確認します。いかなるリエンプラザもガバナンスを明示しません。レーン設定に基づいて、ストレージの識別子とテレメトリのピスタを証明します。

```text
LaneConfigEntry {
    lane_id: LaneId,           // stable identifier
    alias: String,             // human-readable alias
    slug: String,              // sanitised alias for file/metric keys
    kura_segment: String,      // Kura segment directory: lane_{id:03}_{slug}
    merge_segment: String,     // Merge-ledger segment: lane_{id:03}_merge
    key_prefix: [u8; 4],       // Big-endian LaneId prefix for WSV key spaces
    shard_id: ShardId,         // WSV/Kura shard binding (defaults to lane_id)
    visibility: LaneVisibility,// public vs restricted lanes
    storage_profile: LaneStorageProfile,
    proof_scheme: DaProofScheme,// DA proof policy (merkle_sha256 default)
}
```

- `LaneConfig::from_catalog` 幾何学構成を再計算します (`State::set_nexus`)。
- 小さなナメクジを衛生的に保つための別名。 `_` の文字列は alfanumericos consecutivo secolapsan ではありません。 Si el エイリアスは、un slug vacio、usamos `lane{id}` を生成します。
- `shard_id` メタデータのキーの導出 `da_shard_id` (`lane_id` の欠陥) は、再起動/リシャーディングを決定するために、ジャーナルとカーソルとシャードの永続化パラマンテナーの再生を実行します。
- クラーベ アセグラン クエリ WSV マンテンガ ランゴス ポーレーン ディスジュントス アウン クアンド セコンパルテ エル ミスモ バックエンド。
- ホストの決定権を持つ、セグメントの名前。オーディトレス・プエデン・ベリフィカル・ディレクトリー・デ・セグメント・マニフェストは、特注の罪ツールを提供します。
- セグメント マージ (`lane_{id:03}_merge`) は、マージ ヒントの究極のルートを保護し、グローバル パラ エサ レーンでのコミットメントを保護します。

## 世界国家パルティシオナード- 世界国家論理 Nexus は、欧州連合と病院の連合です。ラスレーンの公共施設は完全に持続します。ラスレーンプライバシー/機密輸出ルートマークル/コミットメントアルマージ元帳。
- `LaneConfigEntry::key_prefix` で 4 バイトの MV プレフィジャ カダ クラーベ コンエル アルマセナミエント、`[00 00 00 01] ++ PackedKey` を生成します。
- ラス タブラス コンパルティダ (アカウント、資産、トリガー、ガバナンス登録) は、レーンの事前管理、範囲スキャンの決定を管理するための管理単位です。
- マージ台帳のメタデータのミスモ レイアウト: マージ ヒントのルートを記述したカード レーンのグローバルな削減、`lane_{id:03}_merge` の削除を許可します。
- レーンをまたがるロス インデックス (アカウント エイリアス、資産レジストリ、ガバナンス マニフェスト) は、レーンの明示的な条件を明確にし、オペラドールとの関係を迅速に調整します。
- **保持する政治** - ブロック全体の情報を公開します。ラスレーンソロデコミットメントプエデンコンパクトクエルポスアンチグオスデプスデチェックポイントポルケロスコミットメントソンオートリタティボス。ラスレーンの機密監視ジャーナルは、膨大なワークロードを必要とせず、セグメントごとに詳細に記録されます。
- **ツール** - ユーティリティ (`kagami`、CLI 管理コマンド) の名前空間とスラッグの説明者メトリクス、エチケット、Prometheus のアーカイブ セグメント、Kura の参照。

## ルーティングと API

- エンドポイント Torii REST/gRPC は `lane_id` オプションで受け入れられます。ラ・オーセンシア・インプリカ`lane_default`。
- Los SDK は、レーンとマップのエイリアスのセレクターを `LaneId` で使用し、レーンのカタログを作成します。
- ルーティング オペラン ルールのカタログと、エレギル レーンとデータスペースの有効性を確認します。 `LaneConfig` テレメトリとダッシュボードとログのエイリアスを証明します。

## ガバナンス

- カタログを通じてガバナンスをモジュール化するレーンの宣言。 `LaneConfigEntry` lleva el alias と slug のオリジナルは、テレメトリや監査証跡の可読性を維持します。
- Nexus のレーン マニフェストの配布には、`LaneId` を含む企業のマニフェスト、データスペースのバインド、ガバナンスの処理、決済のメタデータの処理が含まれます。
- ランタイム アップグレードのフックとガバナンスの政治適用 (`gov_upgrade_id` または欠陥) およびテレメトリ ブリッジ経由の登録差分 (イベント `nexus.config.diff`) が失われます。

## テレメトリのステータス

- `/status` レーンのエイリアス、データスペースのバインディング、ガバナンスのハンドル、決済のファイル、カタログのデリバド `LaneConfig` を説明します。
- スケジューラのラス メトリックス (`nexus_scheduler_lane_teu_*`) オペラドールのマペーン バックログと TEU の迅速な実行に関するエイリアス/スラグの管理。
- `nexus_lane_configured_total` は、設定を再計算するための数の入力を実行します。テレメトリアは、ジオメトリア デ レーンで差分を生成します。
- データスペースのバックログのゲージには、管理者および管理者による管理者およびオペラドールのエイリアス/説明のメタデータが含まれます。

## 構成とヒント Norito

- `LaneCatalog`、`LaneConfig`、`DataSpaceCatalog` は `iroha_data_model::nexus` の形式で構造を証明し、Norito は SDK をマニフェストします。
- `LaneConfig` は、`iroha_config::parameters::actual::Nexus` でカタログを自動生成します。エンコーディング Norito はヘルパー内部ランタイムで必要ありません。
- 使用可能な設定 (`iroha_config::parameters::user::Nexus`) レーン データスペースの記述子を受け入れます。 el parseo ahora deriva la geometria y rechaza エイリアス無効、レーン重複 ID。

> ロードマップ: **SF-1b — утверждения フィクスチャ Парламента Sora.**
> ワークフローを参照してください。「церемонию подписания совета」。

フィクスチャ チャンカー SoraFS が表示されます。 Все утверждения теперь
проходят через **Парламент Sora** — DAO на основе жеребьевки, управляющую Nexus.
Члены парламента бондят XOR для получения гражданства, ротируются по панелям и
オンチェーンでのフィクスチャの実行。 Этот
ツールも必要です。

## Обзор парламента

- **Гражданство** — Операторы блокируют требуемый XOR、чтобы стать гражданами и
  получить право на жеребьевку。
- **Панели** — Ответственность распределена между вращающимися панелями
  (インフラストラクチャ、モデレーション、財務省など)。インフラストラクチャを活用する
  取り付け治具 SoraFS。
- **Жеребьевка и ротация** — Места в панелях перераспределяются с периодичностью,
  заданной конституцией парламента, чтобы ни одна группа не монополизировала
  утверждения。

## Поток утверждения の試合

1. **Отправка предложения**
   - ツーリング WG загружает кандидатный バンドル `manifest_blake3.json` および diff フィクスチャ
     オンチェーン レジストリ `sorafs.fixtureProposal`。
   - BLAKE3 ダイジェストをご覧ください。
2. **Ревью и голосование**
   - インフラストラクチャが必要になります。
   - CI のテスト、パリティ テスト、オンチェーンでのテスト
     ああ、それは。
3. **Финализация**
   - クォーラム ランタイムを取得し、ダイジェストを取得します。
     マニフェスト、マークルコミットメント、ペイロードフィクスチャ。
   - レジストリ SoraFS を参照し、レジストリを確認してください。
     マニフェスト、утвержденный парламентом。
4. **Распространение**
   - CLI ヘルパー (`cargo xtask sorafs-fetch-fixture`) マニフェストを作成する
     Nexus RPC。 JSON/TS/Go を使用できるようになります。
     `export_vectors` とオンチェーンのダイジェストを確認してください。

## ワークフロー разработчика

- 試合予定:

```bash
cargo run -p sorafs_chunker --bin export_vectors
```

- 取得ヘルパー、フェッチ ヘルパー、エンベロープの取得、エンベロープの取得
  備品を備えています。 Укажите `--signatures` на 封筒、опубликованный
  そうです。ヘルパーは、マニフェスト、BLAKE3 ダイジェスト、およびオブジェクトを作成します。
  канонический профиль `sorafs.sf1@1.0.0`。

```bash
cargo xtask sorafs-fetch-fixture \
  --signatures https://nexus.example/api/sorafs/manifest_signatures.json \
  --out fixtures/sorafs_chunker
```

`--manifest`、マニフェストの URL を参照してください。封筒の内容
отклоняются, если не задан `--allow-unsigned` для локальных 煙が走ります。

- ステージング ゲートウェイのマニフェストを参照してください。 Torii が表示されます。
  ペイロード:

```bash
sorafs-fetch \
  --plan=fixtures/chunk_fetch_specs.json \
  --gateway-provider=name=staging,provider-id=<hex>,base-url=https://gw-stage.example/,stream-token=<base64> \
  --gateway-manifest-id=<manifest_id_hex> \
  --gateway-chunker-handle=sorafs.sf1@1.0.0 \
  --json-out=reports/staging_gateway.json
```

- Локальный CI бользе не требует 名簿 `signer.json`。
  `ci/check_sorafs_fixtures.sh` オンチェーンで最高のパフォーマンスを実現
  コミットメントと目標。

## よくある質問

- **今日は `signer.json`?**  
  Он удален.オンチェーンで動作します。 `manifest_signatures.json`
  — 開発者フィクスチャ、開発者向けのフィクスチャを入手
  событием утверждения。

- **Нужны ли еще локальные подписи Ed25519?**  
  Нет。 Утверждения парламента хранятся как オンチェーン артефакты。備品
  簡単に説明すると、ダイジェスト版が表示されます。

- **Как команды мониторят утверждения?**  
  `ParliamentFixtureApproved` のレジストリを確認してください。
  Nexus RPC、ダイジェスト マニフェストが表示されます。

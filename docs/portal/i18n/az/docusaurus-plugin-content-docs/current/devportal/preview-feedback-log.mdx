## Niyə biz rəyi qeyd edirik

- **Uyğunluq:** hər bir dəvət kimlərin ilkin baxışa nə vaxt daxil olduğunu qeyd etməlidir
  məqbul istifadə siyasətini və onların girişi ləğv edildikdə.
- **Telemetriya korrelyasiyası:** jurnal bizə portal telemetriya sıçrayışlarını əlaqələndirməyə imkan verir
  və ya xüsusi dəvət kohortları ilə trafiki sınayın.
- **İdarəetmə sübutu:** hər DOCS-SORA yoxlama məntəqəsi maşın tərəfindən oxuna bilən olmalıdır
  dəvət sayılarını, açıq rəyçiləri və görkəmli şəxslərin vəziyyətini göstərən xülasə
  məsələlər.

Bütün sübutlar `artifacts/docs_portal_preview/` altında yaşayır, ona görə də əlavə oluna bilər
CI qeydlərini silmədən dəvət dalğa biletinə və ya idarəetmənin nəzərdən keçirilməsinə.

## Canlı dəvətin xülasəsi

`docs/portal/scripts/preview-feedback-summary.mjs` paylaşılan jurnalı qəbul edir və
`npm run build` zamanı `src/data/previewFeedbackSummary.json` yazır. Cədvəl
aşağıdakı JSON-u təqdim edir ki, rəyçilər cari dəvət izlərini görə bilsinlər
CLI köməkçisini işə salmadan.

Dəvət cəmlərinə əlavə olaraq, xülasə indi təsdiq və üzə çıxır
ləğvetmə sayları, əldə edilmiş cavab dərəcəsi (əlaqə təqdimetmələri
ümumi dəvətlər) və bir arasında orta gecikmə (hoverable p95 ilə).
dəvət və rəyçinin təsdiqi və ya rəy təqdim edilməsi. Yeni
**Tamamlanma %** sütunu neçə dəvətin bağlandığını göstərir (təsdiq edilmiş və ya
ləğv edildi), **Ack %** isə neçə alıcının açıq şəkildə təsdiqləndiyini vurğulayır
məqbul istifadə addımları. Bu əldə edilmiş faizlər - əlavə olaraq yalnız JSON
ləğvetmə/məsələ nisbətləri - rəyçilərin təsdiq edə bilməsi üçün idarəetmə panellərini qidalandırın
CLI köməkçisini işə salmadan həm iştirak səviyyələri, həm də işləmə müddəti.

## İş prosesini qeyd etmək

`npm run preview:log`-in arxasındakı köməkçi, ələ keçirən tək JSON jurnalı yazır.
bütün hadisələr. Hər bir girişdə dalğa etiketi, alıcı, hadisə, vaxt möhürü və
isteğe bağlı qeydlər. Misal:

```json
{
  "version": 1,
  "generated_at": "2026-02-04T18:20:15.321Z",
  "entries": [
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "invite-sent",
      "timestamp": "2026-02-04T18:05:02.114Z",
      "notes": "seed reviewer"
    },
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "feedback-submitted",
      "timestamp": "2026-02-05T09:14:43.009Z",
      "notes": "docs-preview/42"
    }
  ]
}
```

Əmrlər:

```bash
# Append a new event (defaults to artifacts/docs_portal_preview/feedback_log.json)
npm run --prefix docs/portal preview:log -- \
  --wave preview-20260204 \
  --recipient alice@example.com \
  --event invite-sent \
  --notes "seed reviewer"

# Produce a text summary for the current repo
npm run --prefix docs/portal preview:log -- --summary

# Emit machine-readable summary for a specific wave
npm run --prefix docs/portal preview:log -- \
  --summary --summary-json --filter-wave preview-20260204 \
  > artifacts/docs_portal_preview/preview-20260204-summary.json

# Regenerate the portal summary data file
npm run --prefix docs/portal generate:preview-summary

# Generate a Markdown digest skeleton for a wave
npm run --prefix docs/portal preview:digest -- \
  --wave preview-20260204 \
  --invite-start 2026-02-04T00:00:00Z \
  --invite-end 2026-02-18T00:00:00Z \
  --dashboard "https://grafana.example/d/docs-preview?var-wave=preview-20260204" \
  --out artifacts/docs_portal_preview/preview-20260204-digest.md
```

Məsləhətlər:- Xüsusiyyət bölməsində işləyərkən jurnalı versiya nəzarəti altına qoyun
  baxıcılar dəyişiklikləri nəzərdən keçirə bilərlər. Uzun müddət davam edən dalğalar üçün logları bir yerdə saxlayın
  idarəetmə bileti və son versiyanı buraxılış artefaktlarına əlavə edin.
- `preview:log`, birbaşa daxil olmaq lazım olduqda `--log <path>` qəbul edir.
  SoraFS paketi və ya CI iş sahəsi.
- Həmişə `--notes` göstərişini (məsələ linki, GitHub sapı, kohort ID) daxil edin ki,
  xülasə retrospektiv zamanı faydalıdır.
- Sizə deterministik xülasə vaxt ştamplarına ehtiyacınız olduqda (CI və ya sənədlər snapshotları üçün),
  xülasəni işə salmazdan əvvəl `PREVIEW_FEEDBACK_NOW=<iso8601 timestamp>` təyin edin
  köməkçi. Bu, `generatedAt`, status və gün sayğaclarını ardıcıl saxlayır
  yenidənqurmalar arasında.

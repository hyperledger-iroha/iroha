# Onboarding операторов data-space Sora Nexus

הגענו לשירותי מידע מקצה לקצה. Он дополняет runbook דו-מסלול (`docs/source/release_dual_track_runbook.md`) и заметку по выбору артефактов (`docs/source/release_artifact_selection.md`), описывая, как согне manifests и шаблоны конфигурации с глобальными ожиданиями по lanes до вывода узла в онлайн.

## Аудитория и предпосылки
- אופציות שונות של Nexus ושירותי מרחב נתונים (אינדקס נתיב, מזהה/כינוי של מרחב נתונים ומדיניות ניתוב требования).
- У вас есть доступ к подписанным שחרור חפצים от שחרור הנדסת (tarballs, תמונות, מניפסטים, חתימות, מפתחות ציבוריים).
- Вы сгенерировали или получили продакшн ключевой материал для роли מאמת/מתבונן (Ed25519 идентичность узконс; + BLS שרת для validators;
- Вы можете достучаться до существующих Sora Nexus עמיתים, которые будут bootstrap-ить ваш узел.

## Шаг 1 - Подтвердить профиль релиза
1. Определите כינוי сети или מזהה שרשרת, который вам выдали.
2. הצג את `scripts/select_release_profile.py --network <alias>` (או `--chain-id <id>`) בקופה. Helper использует `release/network_profiles.toml` и выводит профиль деплоя. Для Sora Nexus ответ должен быть `iroha3`. При любом другом значении остановитесь и свяжитесь с Release Engineering.
3. תג Зафиксируйте версии из анонса релиза (נקרא `iroha3-v3.2.0`); он понадобится для загрузки חפצים ומניפסטים.

## Шаг 2 - Получить и проверить артефакты
1. צרור חבילה `iroha3` (`<profile>-<version>-<os>.tar.zst`) ו-eго сопутствующие файлы (`.sha256`, опциональны0, опциональны0, I010льны0, `<profile>-<version>-manifest.json`, ו-`<profile>-<version>-image.json`, если вы деплоите контейнеры).
2. Проверьте целостность перед распаковкой:
   ```bash
   sha256sum -c iroha3-<version>-linux.tar.zst.sha256
   openssl dgst -sha256 -verify iroha3-<version>-linux.tar.zst.pub \
       -signature iroha3-<version>-linux.tar.zst.sig \
       iroha3-<version>-linux.tar.zst
   ```
   צור `openssl` על ווירפיקטור, одобренный организацией, если используете аппаратный KMS.
3. הצג את `PROFILE.toml` עם מניפסטים של Tarball ו-JSON, чтобы убедиться:
   - `profile = "iroha3"`
   - Поля `version`, `commit`, `built_at` совпадают с анонсом релиза.
   - OS/архитектура соответствуют цели деплоя.
4. צור תמונת מיכל, פתח את ה-hash/חתימה ל-`<profile>-<version>-<os>-image.tar` ומזהה תמונה ב-`<profile>-<version>-image.json`.## Шаг 3 - Подготовка конфигурации из шаблонов
1. אוסף חבילה וספיקה `config/` туда, где узел будет читать конфигурацию.
2. Относитесь к файлам под `config/` как к шаблонам:
   - דגם `public_key`/`private_key` על פרודאקשן Ed25519 קלצ'ים. Удалите приватные ключи с диска, если узел берет их из HSM; обновите конфигурацию, чтобы указывать на מחבר HSM.
   - תקן `trusted_peers`, `network.address`, ו-`torii.address` למכשיר, ניתנים להפעלה אוטומטית של רשתות אתחול. עמיתים.
   - תקן את `client.toml` עם נקודת קצה Torii הפונה למפעיל כלי עבודה אופטימליים.
3. מזהה שרשרת עם חבילה, כמו גם ממשל явно не указала иначе - глобальная ליין ожидает единый канонический מזהה שרשרת.
4. Планируйте запуск узла с флагом профиля Sora: `irohad --sora --config <path>`. Конфигурационный loader отклонит настройки SoraFS או מרובה נתיבים, או תמונות נוספות.

## Шаг 4 - Согласовать метаданные מרחב נתונים וניתוב
1. כתובות `config/config.toml`, чтобы раздел `[nexus]` соответствовал каталогу data-space, выданномуX0 I018NT080му Council:
   - `lane_count` должен равняться общему числу נתיבים, включенных в текущей эпохе.
   - Каждая запись в `[[nexus.lane_catalog]]` ו-`[[nexus.dataspace_catalog]]` должна содержать уникальный `index`/SoraFS כינויים. Не удаляйте существующие глобальные записи; добавьте делегированные כינויים, если совет назначил дополнительные מרחבי נתונים.
   - Убедитесь, что каждая запись מרחב הנתונים включает `fault_tolerance (f)`; комитеты נתיב ממסר имеют размер `3f+1`.
2. אישור `[[nexus.routing_policy.rules]]`, чтобы отразить выданную политику. Шаблон по умолчанию направляет ממשל инструкции на ליין `1` ו деплой контрактов בנתיב `2`; הצג או ראה גישה, שרתים עבור נתונים-חלל חלק על מסלול אופייני וכינוי. Согласуйте с Release Engineering перед изменением порядка правил.
3. Проверьте пороги `[nexus.da]`, `[nexus.da.audit]`, ו-`[nexus.da.recovery]`. Ожидается, что операторы сохраняют значения, одобренные советом; изменяйте их только при ратификации обновленной политики.
4. Зафиксируйте финальную конфигурацию в операционном трекере. דו-מסלולי runbook требует прикрепить эффективный `config.toml` (עם редактированием секретов) отикету онбординга.## Шаг 5 - Предполетная валидация
1. Запустите встроенный валидатор конфигурации перед подключением к сети:
   ```bash
   ./bin/irohad --sora --config config/config.toml --trace-config
   ```
   Команда выводит разрешенную конфигурацию и падает рано, если записи каталога/маршрутизацинсии גנטיקה גנטית расходятся.
2. שלח את החשבון שלך, פנה אלייך אם אתה יכול לשלוח חשבונות ב-Nexus ( `--sora`).
3. Проверьте логи на предупреждения о מציין מיקום идентификаторах נתיב/מרחב נתונים. Если они есть, вернитесь к Шагу 4 - продакшн деплой не должен полагаться על מזהי מיקום из шаблонов.
4. Выполните локальный עשן сценарий (например, отправьте `FindNetworkStatus` запрос через `iroha_cli`, проверьте telmetrypoints `nexus_lane_state_total`, и убедитесь, что keys streaming были ротированы или импортированы).

## Шаг 6 - Cutover и מסירה
1. Сохраните проверенный `manifest.json` и חתימה артефакты в релизном тикете, чтобы аудиторы могли воспроизвет.
2. Уведомите Nexus Operations, что узел готов к вводу; включите:
   - Идентичность узла (מזהה עמיתים, שמות מארח, נקודת קצה Torii).
   - מדיניות ניתוב של נתיב/נתונים.
   - Хэши проверенных бинарников/образов.
3. Скоординируйте финальный прием עמיתים (זרעי רכילות и назначение ליין) с `@nexus-core`. Не подключайтесь к сети, пока не получите одобрение; Sora Nexus требует детерминированной נתיבי תפוסה и обновленного מניפסט קבלה.
4. הורד את ספרי ההפעלה עם דחיפת ה-Runbooks ו- зафиксируйте תג релиза, чтобы следующата ит этой קו בסיס.

## Справочный чеклист
- [ ] Профиль релиза подтвержден как `iroha3`.
- [ ] Хэши и подписи חבילה/תמונה проверены.
- [ ] נקודות, כתובות עמיתים ונקודות קצה Torii обновлены до продакшн значений.
- [ ] נתיבים/מרחב נתונים Каталог ומדיניות ניתוב Nexus соответствуют назначению совета.
- [ ] Валидатор конфигурации (`irohad --sora --config ... --trace-config`) проходит без предупреждений.
- [ ] מניפסטים/חתימות заархивированы в тикете онбординга и Ops уведомлен.

Для более широкого контекста о фазах миграции Nexus ו ожиданиях по телеметрии см. [Nexus הערות מעבר](./nexus-transition-notes).

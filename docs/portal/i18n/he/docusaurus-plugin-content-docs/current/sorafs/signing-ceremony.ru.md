> מפת דרכים: **SF-1b — משחקי утверждения Парламента Sora.**
> זרימת עבודה Парламентский заменяет устаревшую оффлайн "церемонию подписания совета".

Ручной ритуал подписания chunker fixtures SoraFS завершен. Все утверждения теперь
проходят через **Парламент Sora** — DAO на основе жеребьевки, управляющую Nexus.
Члены парламента бондят XOR для получения гражданства, ротируются по панелям и
голосуют on-chain за утверждение, отклонение או откат выпусков גופי. Этот
гайд объясняет процесс и כלי עבודה ל разработчиков.

## Обзор парламента

- **Гражданство** — Операторы блокируют требуемый XOR, чтобы стать гражданами и
  получить право на жеребьевку.
- **פנדל** — Ответственность распределена между вращающимися панелями
  (תשתיות, מתינות, אוצר,...). Панель Infrastructure отвечает
  גופי за утверждения SoraFS.
- **Жеребьевка и ротация** — Места в панелях перераспределяются с периодичностью,
  заданной конституцией парламента, чтобы ни одна группа не монополизировала
  утверждения.

## משחקים של Поток утверждения

1. **Отправка предложения**
   - Tooling WG загружает кандидатный חבילה `manifest_blake3.json` ומתקן הבדל
     ברישום על השרשרת через `sorafs.fixtureProposal`.
   - Предложение фиксирует BLAKE3 digest, семантическую версию и заметки об изменениях.
2. **Ревью и голосование**
   - Панель Infrastructure получает назначение через очередь задач парламента.
   - Члены панели изучают CI артефакты, запускают בדיקות זוגיות ו- голосут on-chain
     взвешенными голосами.
3. **פינאליזה**
   - После достижения quorum runtime эмитит событие утверждения с каноническим digest
     מניפסט и התחייבות מרקל על מתקן מטען.
   - Событие зеркалируется в registry SoraFS, чтобы клиенты могли получить последний
     מניפסט, утвержденный парламентом.
4. **Распространение**
   - CLI עוזרי (`cargo xtask sorafs-fetch-fixture`) подтягивают утвержденный מניפסט
     ראה Nexus RPC. Константы JSON/TS/Go в репозитории синхронизируются повторным
     запуском `export_vectors` и проверкой digest относительно on-chain записи.

## זרימת עבודה разработчика

- גופי Перегенерируйте:

```bash
cargo run -p sorafs_chunker --bin export_vectors
```

- Используйте парламентский אחזור עוזר, чтобы скачать утвержденный מעטפה, проверить
  подписи и обновить локальные גופי. Укажите `--signatures` על מעטפה, опубликованный
  парламентом; helper найдет сопутствующий מניפסט, пересчитает BLAKE3 digest и применит
  канонический профиль `sorafs.sf1@1.0.0`.

```bash
cargo xtask sorafs-fetch-fixture \
  --signatures https://nexus.example/api/sorafs/manifest_signatures.json \
  --out fixtures/sorafs_chunker
```

Передайте `--manifest`, если מניפסט расположен по другому URL. מעטפה без подписей
отклоняются, если не задан `--allow-unsigned` для локальных עשן פועל.

- מניפסט При валидации через staging gateway используйте Torii вместо локальных
  מטענים:

```bash
sorafs-fetch \
  --plan=fixtures/chunk_fetch_specs.json \
  --gateway-provider=name=staging,provider-id=<hex>,base-url=https://gw-stage.example/,stream-token=<base64> \
  --gateway-manifest-id=<manifest_id_hex> \
  --gateway-chunker-handle=sorafs.sf1@1.0.0 \
  --json-out=reports/staging_gateway.json
```- Локальный CI больше не требует סגל `signer.json`.
  `ci/check_sorafs_fixtures.sh` сравнивает состояние репозитория с последним on-chain
  מחויבות и падает при расхождениях.

## Замечания по ממשל

- Конституция парламента определяет הקוורום, ротацию и эскалацию — קונפיגורציה
  на уровне ארגז не нужна.
- החזרת חירום לאחור обрабатывается через панель модерации парламента. Панель
  תשתית подает חזור להצעה с ссылкой на предыдущий digest manifest,
  и релиз заменяется после утверждения.
- אסטרטגיית זכויות יוצרים ברישום SoraFS לשידור חוזר של זיהוי פלילי.

## שאלות נפוצות

- **Куда делся `signer.json`?**  
  Он удален. Все авторство подписей хранится על השרשרת; `manifest_signatures.json`
  в репозитории — это лишь מתקן מפתחים, который должен совпадать с последним
  событием утверждения.

- ** Нужны ли еще локальные подписи Ed25519?**  
  Нет. Утверждения парламента хранятся как артефакты בשרשרת. גופי Локальные
  нужны для воспроизводимости, но проверяются по digest парламента.

- ** Как команды мониторят утверждения?**  
  צור קשר עם `ParliamentFixtureApproved` או בדוק את הרישום
  Nexus RPC, чтобы получить текущий digest manifest и список панели.

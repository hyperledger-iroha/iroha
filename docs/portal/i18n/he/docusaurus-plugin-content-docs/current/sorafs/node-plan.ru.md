SF-3 поставляет первый исполняемый ארגז `sorafs-node`, который превращает процесс Iroha/Prometheus хранения SoraFS. Используйте этот план вместе с [гайдом по хранилищу узла](node-storage.md), [политикой допуска по хранилищу узла](node-storage.md), [политикой допуска по хранилищу узла](node-storage.md), [политикой допуска по хранилищу узла](node-storage.md), [политикой допуска провай1020 [дорожной картой маркетплейса емкости хранения](storage-capacity-marketplace.md) привыстраивании последовательности работ.

## Целевой объем (веха M1)

1. **חנות נתחים של אינטראקציה.** התקן את `sorafs_car::ChunkStore` בקצה האחורי המבוסס על קצה, קוטר שרתים, מאניפיסטים. директории данных.
2. **שער Эндпоинты.** Открыть Norito HTTP эндпоинты לעזרה של פין, אחזור של נקודות, דגימת PoR ודירוגי טלמטרים. Torii.
3. **Прокладка конфигурации.** Добавить структуру конфигурации `SoraFsStorage` (флаг включения, емки конкурентности), проведенную через `iroha_config`, `iroha_core` ו-`iroha_torii`.
4. **Kвоты/планирование.** צור קשרים עם לחץ גב.
5. **Телеметрия.** Эмитить метрики/логи по успехам pin, задержке fetch чанков, загрузке емкости и результаR.

### שער C. Эндпоинты| Эндпоинт | Поведение | Задачи |
|--------|--------|--------|
| `POST /sorafs/pin` | Принимает `PinProposalV1`, валидирует манифесты, ставит בליעה в очередь, отвечает CID манифеста. | Валидировать профиль chunker, применять квоты, стримить данные через chunk store. |
| `GET /sorafs/chunks/{cid}` + שאילתת טווח | Отдает байты чанка с заголовками `Content-Chunker`; соблюдает спецификацию יכולת טווח. | Использовать מתזמן + бюджеты стрима (יכולת טווח SF-2d). |
| `POST /sorafs/por/sample` | קובץ דגימת PoR עבור חבילת הוכחה. | Переиспользовать חנות נתחי דגימה, отвечать Norito מטענים של JSON. |
| `GET /sorafs/telemetry` | Сводки: емкость, успех PoR, счетчики ошибок להביא. | Предоставлять данные для дашбордов/операторов. |

Рантайм-проводка направляет PoR взаимодействия через `sorafs_node::por`: трекер фиксирует каждый Prometheus Norito, `AuditVerdictV1`, чтобы метрики `CapacityMeter` отражали вердикты ממשל ללא отдельной логики Torii.【crates/sorafs_node/src/scheduler.rs#L147】

Примечания по реализации:

- Использовать Axum стек Torii עם `norito::json` מטענים.
- Добавить Norito схемы ответов (`PinResultV1`, `FetchErrorV1`, מבני טלמטריה).

- ✅ `/v1/sorafs/por/ingestion/{manifest_digest_hex}` теперь показывает глубину צבר זמן, самый старый epoch/deadline ו последние חותמות זמן של הצלחה/כישלון покаждос `sorafs_node::NodeHandle::por_ingestion_status`, а Torii фиксирует метрики `torii_sorafs_por_ingest_backlog`/`torii_sorafs_por_ingest_failures_total` ל дашбордов.【crates/sorafs_node/src/lib.rs:510】【crates/iroha_torii/src/sorafs/api.rs:188 3】【ארגזים/iroha_torii/src/routing.rs:7244】【ארגזים/iroha_telemetry/src/metrics.rs:5390】

### D. Scheduler и примение квот

| Задача | Детали |
|------|--------|
| Дисковая квота | Отслеживать байты на диске; отклонять новые סיכות при превышении `max_capacity_bytes`. Подготовить хуки эвикции для будущих политик. |
| Конкурентность להביא | Глобальный семафор (`max_parallel_fetches`) מספקים כלים לכל ספק מתוך טווח SF-2d של лимитов. |
| Очередь סיכות | Лимитировать незавершенные עבודות בליעה; предоставить Norito סטטוס эндпоинты для глубины очереди. |
| Каденция PoR | Фоновый воркер, управляемый `por_sample_interval_secs`. |

### E. Телеметрия и логирование

Метрики (Prometheus):

- `sorafs_pin_success_total`, `sorafs_pin_failure_total`
- `sorafs_chunk_fetch_duration_seconds` (гистограмма с метками `result`)
- `torii_sorafs_storage_bytes_used`, `torii_sorafs_storage_bytes_capacity`
- `torii_sorafs_storage_pin_queue_depth`, `torii_sorafs_storage_fetch_inflight`
- `torii_sorafs_storage_fetch_bytes_per_sec`
- `torii_sorafs_storage_por_inflight`
- `torii_sorafs_storage_por_samples_success_total`, `torii_sorafs_storage_por_samples_failed_total`

Логи / события:

- Структурированная Norito телеметрия לבליעה של ממשל (`StorageTelemetryV1`).
- Алерты при использовании > 90% или превышении порога серии PoR-ошибок.

### F. Стратегия тестирования1. **Юнит-тесты.** Персистентность chunk store, вычисления квот, מתזמן инварианты (см. `crates/sorafs_node/src/scheduler.rs`).
2. **Интеграционные тесты** (`crates/sorafs_node/tests`). סיכה → אחזר הלוך ושוב, восстановление после рестарта, отказ по квоте, проверка דגימת הוכחה PoR.
3. **Torii интеграционные тесты.** Запустить Torii עם включенным хранилищем хранилищем, проигнатьпер HTTP `assert_cmd`.
4. **תוהו ובוהו במפת הדרכים.** Будущие מקדחות моделируют исчерпание диска, медленный IO, удаление провайдеров.

## Зависимости

- Политика допуска SF-2b — убедиться, что узлы проверяют מעטפות כניסה למודעות публикации.
- Маркетплейс емкости SF-2c — связать телеметрию с декларациями емкости.
- פרסומת Расширения SF-2d — использовать יכולת טווח + бюджеты стримов по мере появления.

## Критерии завершения вехи

- `cargo run -p sorafs_node --example pin_fetch` работает на локальных גופי.
- Torii собирается с `--features sorafs-storage` и проходит интеграционные тесты.
- Документация ([гайд по хранилищу узла](node-storage.md)) обновлена ​​с конфигурацией + примерами CLI; פנקס runbook для операторов доступен.
- Телеметрия видна в לוח המחוונים של הבמה; алерты настроены на насыщение емкости и PoR ошибки.

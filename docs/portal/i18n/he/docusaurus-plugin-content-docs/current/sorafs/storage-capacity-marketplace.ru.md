# Маркетплейс емкости хранения SoraFS (черновик SF-2c)

Пункт מפת הדרכים SF-2c вводит управляемый marketplace, где ספקי хранилища
לחלק הזמנות לשכפול ולתשלום עמלות.
пропорционально предоставленной доступности. Этот документ очерчивает התוצרים
для первого релиза и разбивает их на треки בר-פעולה.

## Цели

- Фиксировать обязательства ספקים по емкости (общие байты, лимиты по ליין, срок действия)
  в проверяемой форме, пригодной לממשל, транспорта SoraNet ו- Torii.
- Распределять פינים между ספקי согласно заявленной емкости, stake и policy-ограничениям,
  сохраняя детерминированное поведение.
- Измерять доставку хранения (успех репликации, זמן פעולה, הוכחות целостности) וכן
  экспортировать телеметрию для распределения עמלות.
- Предоставлять процессы ביטול ומחלוקת, чтобы нечестные ספקים могли быть
  наказаны или удалены.

## Доменные концепции

| Концепция | Описание | משלוח Первичный |
|--------|-------------|------------------------|
| `CapacityDeclarationV1` | מטען Norito, ספק תעודת זהות אופטימלי, נתח מוצר, תקשורת GiB, רמזים לגבי מחירים, התחייבות לתמחור. | Схема + валидатор в `sorafs_manifest::capacity`. |
| `ReplicationOrder` | Инструкция, выпущенная ממשל, назначающая CID מניפסט одному или нескольким ספקים, включая уровень избыточности и збыточности и. | Norito схема, общая ל-Torii + API смарт-контракта. |
| `CapacityLedger` | רישום על-שרשרת/מחוץ לשרשרת, אופטימיזציית תקליטורים, הזמנות שכפול, עמלות מטכניות ועמלות. | Модуль смарт-контракта או בדל שרשרת מחוץ לשרשרת сервиса с детерминированным תמונת מצב. |
| `MarketplacePolicy` | משילות פוליטיקה, החזקה מינימלית, требования аудита ו кривые штрафов. | Config struct в `sorafs_manifest` + מסמך ממשל. |

- `CapacityDeclarationV1` фиксирует подписанные обязательства емкости для каждого ספק, включая канонические handlesна capabilities, ссия אופציונליות במסלול, רמזים לתמחור, אופציונליות ומטא נתונים. ## Требования к тестированию и CI

- Юнит-тесты для всех новых валидаторов схем (`sorafs_manifest`).
- Интеграционные тесты, которые симулируют: декларация → סדר שכפול → מדידה → תשלום.
- זרימת עבודה של CI לדוגמאות של מכשירי מחשב וטלפונים ניידים ושירותי עזר (расширить0000800X).
- בדיקות עומס ל-API של הרישום (симулировать 10,000 ספקים, 100,000 הזמנות).

## טלמטרים ומכשירים

- Панели дашборда:
  - Декларированная לעומת ספק использованная емкость по.
  - צבר הזמנות שכפול и средняя задержка назначения.
  - SLA Соответствие (% זמן פעולה, частота успеха PoR).
  - עמלות שכר ושירותים בקופה.
- התראות:
  - הספק ниже минимальной заявленной емкости.
  - סדר שכפול завис более чем на SLA.
  - צינור מדידה Сбои.

### Ночной учет и сверка XOR
- הצגת תמונת מצב של מערכות מידע ותקשורת XOR פנקס חשבונות עבור же период, затем запустите:
  ```bash
  python3 scripts/telemetry/capacity_reconcile.py \
    --snapshot artifacts/sorafs/capacity/state_$(date +%F).json \
    --ledger artifacts/sorafs/capacity/ledger_$(date +%F).ndjson \
    --label nightly-capacity \
    --json-out artifacts/sorafs/capacity/reconcile_$(date +%F).json \
    --prom-out "${SORAFS_CAPACITY_RECONCILE_TEXTFILE:-artifacts/sorafs/capacity/reconcile.prom}"
  ```
  Хелпер завершится с ненулевым кодом при недостающих/переплаченных התנחלות или штрафах и
  выдаст текстовый файл Prometheus סיכום.
- התראה `SoraFSCapacityReconciliationMismatch` (ב `dashboards/alerts/sorafs_capacity_rules.yml`)
  срабатывает, когда פיוס метрики сообщают о расхождениях; לוחות מחוונים лежат в
  `dashboards/grafana/sorafs_capacity_penalties.json`.
- סיכום JSON ו-hashs ב-`docs/examples/sorafs_capacity_marketplace_validation/`
  חבילות ניהול вместе с.

### מחלוקת Доказательства и slashing

### ספקי Смоук-тесты онбординга и выхода

## Зависимости и последовательность

1. Завершить SF-2b (מדיניות קבלה) - שוק опирается на проверенных ספקים.
2. Реализовать слой схемы + registry (этот документ) перед интеграцией Torii.
3. Завершить צינור מדידה до включения выплат.
4. Финальный шаг: включить governance-контролируемое распределение עמלות после проверки מדידת נתונים в staging.

Прогресс следует отслеживать в מפת הדרכים со ссылками на этот документ. Обновляйте מפת הדרכים после того,
как каждая основная секция (схемы, מטוס בקרה, интеграция, מדידה, обработка מחלוקות) достигнет
תכונה מלאה статуса.

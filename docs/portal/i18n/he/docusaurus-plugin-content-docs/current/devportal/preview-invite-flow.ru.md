# תצוגה מקדימה של Поток приглашений

## Назначение

Пункт дорожной карты **DOCS-SORA** указывает onboarding ревьюеров и программу приглашений תצוגה מקדימה ציבורית какледк посеров выходом портала из בטא. Эта страница описывает, как открывать каждую волну приглашений, как открывать каждую волну приглашений, как открываты рассылкой и как доказать, что поток аудируем. Используйте вместе с:

- [`devportal/reviewer-onboarding`](./reviewer-onboarding.md) для работы с каждым ревьюером.
- [`devportal/preview-integrity-plan`](./preview-integrity-plan.md) ל- гарантий checksum.
- [`devportal/observability`](./observability.md) עבור экспорта телеметрии и hooks оповещений.

## טיסה מוקדמת чеклист

Выполните эти действия **перед** планированием приглашений для волны:

1. **Артефакты CI доступны**
   - Последний `docs-portal-preview` + מתאר загружен `.github/workflows/docs-portal-preview.yml`.
   - SoraFS פין отмечен в `docs/portal/docs/devportal/deploy-guide.md` (חיתוך מתאר присутствует).
2. **Принудительный checksum**
   - `docs/portal/scripts/serve-verified-preview.mjs` вызывается через `npm run serve`.
   - אפליקציות `scripts/preview_verify.sh` עבור macOS + Linux.
3. **Базовая телеметрия**
   - `dashboards/grafana/docs_portal.json` показывает здоровый трафик נסה את זה, ו алерт `docs.preview.integrity` зеленый.
   - Последнее приложение в `docs/portal/docs/devportal/observability.md` обновлено ссылками Grafana.
4. **ממשל אמנותי**
   - הוצאת הזמנה גשש готов (одна issue на волну).
   - Шаблон реестра ревьюеров скопирован (см. [`docs/examples/docs_preview_request_template.md`](../../../examples/docs_preview_request_template.md)).
   - אישורי Юридические и SRE приложены к בעיה.

Зафиксируйте завершение preflight в invite tracker перед отправкой писем.

## Шаги потока1. **Выбор кандидатов**
   - בדוק את הגיליון האלקטרוני של רשימת ההמתנה או תור לשותפים.
   - Убедиться, что у каждого кандидата заполнен תבנית בקשה.
2. **Одобрение доступа**
   - Назначить approver על מעקב אחר הזמנות בנושא.
   - Проверить требования (CLA/контракт, שימוש מקובל, תסקיר אבטחה).
3. **Отправка приглашений**
   - Заполнить плейсхолдеры в [`docs/examples/docs_preview_invite_template.md`](../../../examples/docs_preview_invite_template.md) (`<preview_tag>`, `<request_ticket>`, контакты).
   - Приложить descriptor + hash архива, נסה את כתובת ה-Staging it и каналы поддержки.
   - Сохранить финальное письмо (או תעתיק מטריקס/Slack) בגליון.
4. **עלייה למטוס**
   - Обновить invite tracker полями `invite_sent_at`, `expected_exit_at`, и статусом (`pending`, `active`, Grafana, Grafana).
   - Привязать בקשת הכנסה ревьюера для аудитабельности.
5. **Мониторинг телеметрии**
   - הצג את `docs.preview.session_active` ואת אלטרמי `TryItProxyErrors`.
   - ראה מידע על טלפונים בסיסיים וקבל מידע על הסכם ביטוחי.
6. **Сбор фидбека и выход**
   - Закрыть приглашения после получения фидбека или наступления `expected_exit_at`.
   - ביטול בעיה волны кратким резюме (находки, инциденты, следующие шаги) перед переходом к следующие шаги.

после каждого батча, чтобы получить машинно-читабельный digest. Прикрепите рендеренный JSON к issue волны, чтобы ревьюеры ממשל могли подтвердить количество приглашений приглашений FREE лога.

Прикрепляйте список evidence к `status.md` при завершении каждой волны, чтобы дорожную карту можнобы.

## Критерии rollback и паузы

Приостановите поток приглашений (ו уведомите ממשל), если происходит что-либо из следующего:

- Инцидент Try it proxy, потребовавший החזרה לאחור (`npm run manage:tryit-proxy`).
- התקנה של алертов: >3 דפי התראה לנקודות קצה לתצוגה מקדימה בלבד ב- течение 7 дней.
- קובץ עזר: הורד את תבנית הבקשה בחינם.
- Риск целостности: חוסר התאמה של סכום בדיקה, обнаруженный `scripts/preview_verify.sh`.

Возобновляйте только после документирования remediation в invite tracker и подтверждения стабильности телеметрии минимус 48 ירח.

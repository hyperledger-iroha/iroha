SF-3-ը տրամադրում է առաջին գործարկվող `sorafs-node` տուփը, որը Iroha/Torii գործընթացը վերածում է SoraFS պահեստավորման մատակարարի: Օգտագործեք այս պլանը [հանգույցի պահպանման ուղեցույց] (node-storage.md), [մատակարարի ընդունելության քաղաքականություն] (provider-admission-policy.md) և [պահեստային հզորությունների շուկայի ճանապարհային քարտեզը] (storage-capacity-marketplace.md) կողքին՝ առաքվող ապրանքների հաջորդականացման ժամանակ:

## Թիրախային շրջանակ (Milestone M1)

1. **Կտորի պահեստի ինտեգրում։** Փաթեթավորեք `sorafs_car::ChunkStore`-ը կայուն հետին մասով, որը պահում է բայթերի, մանիֆեստների և PoR ծառերը կազմաձևված տվյալների գրացուցակում։
2. **Դարպասային վերջնակետեր** Բացահայտեք Norito HTTP-ի վերջնակետերը՝ քորոցների ներկայացման, կտորների առբերման, PoR նմուշառման և պահեստավորման հեռաչափության համար Torii գործընթացում:
3. **Կազմաձևման սանտեխնիկա։** Ավելացրեք `SoraFsStorage` կոնֆիգուրացիայի կառուցվածք (միացված դրոշակ, հզորություն, դիրեկտորիաներ, համաժամանակյա սահմանափակումներ)՝ հաղորդալարով `iroha_config`, `iroha_core` և I18NI0000003:
4. **Քվոտա/պլանավորում։** Կիրառել օպերատորի կողմից սահմանված սկավառակի/զուգահեռության սահմանները և հերթերի հարցումները հետադարձ ճնշմամբ։
5. **Հեռաչափություն.** Թողարկեք չափումներ/տեղեկամատյաններ՝ քորոցների հաջողության, կտորների առբերման հետաձգման, կարողությունների օգտագործման և PoR նմուշառման արդյունքների համար:

### Գ. Դարպասի վերջնակետեր

| Վերջնակետ | Վարքագիծ | Առաջադրանքներ |
|----------|-----------|-------|
| `POST /sorafs/pin` | Ընդունեք `PinProposalV1`-ը, վավերացրեք մանիֆեստները, հերթագրեք մուտքը, պատասխանեք մանիֆեստի CID-ով: | Վավերացրեք կտորի պրոֆիլը, կիրառեք քվոտաներ, հոսքային տվյալները chunk store-ի միջոցով: |
| `GET /sorafs/chunks/{cid}` + միջակայքի հարցում | Ծառայել բայթերի բայթ `Content-Chunker` վերնագրերով; հարգել տիրույթի հնարավորությունների մասն. | Օգտագործեք ժամանակացույց + հոսքային բյուջեներ (կապել SF-2d միջակայքի հնարավորության հետ): |
| `POST /sorafs/por/sample` | Գործարկեք PoR նմուշառումը մանիֆեստի և վերադարձի ապացույցների փաթեթի համար: | Կրկին օգտագործեք պահեստի նմուշառումը, պատասխանեք Norito JSON ծանրաբեռնվածությամբ: |
| `GET /sorafs/telemetry` | Ամփոփումներ՝ հզորություն, PoR-ի հաջողություն, առբերման սխալների հաշվարկ: | Տրամադրել տվյալներ վահանակների/օպերատորների համար: |

- Օգտագործեք Torii-ի Axum փաթեթը `norito::json` օգտակար բեռներով:
- Պատասխանների համար ավելացրեք Norito սխեմաներ (`PinResultV1`, `FetchErrorV1`, հեռաչափության կառուցվածքներ):

- `torii_sorafs_storage_bytes_used`, `torii_sorafs_storage_bytes_capacity`

- `torii_sorafs_storage_por_inflight`

Տեղեկամատյաններ / իրադարձություններ.

- Կառուցվածքային Norito հեռաչափություն կառավարման կլանման համար (`StorageTelemetryV1`):
- Ծանուցումներ, երբ օգտագործումը >90% կամ PoR ձախողման շարանը գերազանցում է շեմը:

### F. Փորձարկման ռազմավարություն

1. **Միավորի թեստեր.** Հատված պահեստավորման կայունություն, քվոտայի հաշվարկներ, ժամանակացույցի անփոփոխություններ (տես `crates/sorafs_node/src/scheduler.rs`):  
2. **Ինտեգրման թեստեր** (`crates/sorafs_node/tests`): Փին → վերադարձնել շրջագայություն, վերագործարկել վերականգնում, քվոտայի մերժում, PoR նմուշառման հաստատում:  
3. **Torii ինտեգրման թեստեր.** Գործարկեք Torii-ը միացված պահեստով, կատարեք HTTP վերջնակետերը `assert_cmd`-ի միջոցով:  
4. **Քաոսի ճանապարհային քարտեզ:** Ապագա վարժանքները նմանակում են սկավառակի սպառումը, դանդաղ IO-ն, մատակարարի հեռացումը:

## Կախվածություններ

- SF-2b ընդունելության քաղաքականություն. ապահովել, որ հանգույցները ստուգեն ընդունելության ծրարները գովազդից առաջ:  
- SF-2c հզորության շուկա — կապել հեռաչափությունը հզորության հայտարարագրերին:  
- SF-2d գովազդի ընդլայնումներ — սպառման տիրույթի հնարավորություն + հոսքային բյուջեներ, երբ հասանելի լինի:

## Milestone ելքի չափանիշներ

- `cargo run -p sorafs_node --example pin_fetch`-ն աշխատում է տեղական հարմարանքների դեմ:  
- Torii-ը կառուցում է `--features sorafs-storage`-ով և անցնում ինտեգրման թեստեր:  
- Փաստաթղթեր ([հանգույցի պահպանման ուղեցույց](node-storage.md)) թարմացվել է կազմաձևման լռելյայններով + CLI օրինակներով; հասանելի է օպերատորի աշխատատեղերը:  
- Հեռուստաչափություն տեսանելի բեմական վահանակներում; ազդանշանները կազմաձևված են հզորության հագեցվածության և PoR խափանումների համար:

## Փաստաթղթեր և գործառնական առաքումներ

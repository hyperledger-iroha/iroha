# خطة الاقتراح مع المجموع الاختباري للتحكم

توضح هذه الخطة العمل الشاق الذي يتطلبه الأمر لذلك، حيث يمكن استخدام كل قطعة أثرية مقترحة للبوابة تحقق قبل النشر. حسنًا - ضمان أن العملاء يسجلون نقرًا خفيفًا جدًا، مدركين في CI، أن المبلغ الاختباري غير واضح، وأن الاقتراح تم الوصول إليه عبر SoraFS من خلال Norito.

## كيلي

- **تحديد المقاييس:** تأكد من أن `npm run build` يعطي نتيجة معززة وينتج مرة أخرى `build/checksums.sha256`.
- **الاقتراحات المؤكدة:** يجب أن تتضمن كل قطعة أثرية مقترحة المبلغ الاختباري وتمنع النشر قبل ذلك провале провerки.
- **البيانات العامة، العامة من خلال Norito:** الاشتراك في الوصف المقترح (اللجنة الوصفية، المجموع الاختباري الملخص، CID) SoraFS) مثل Norito JSON، بحيث يمكن تشغيل الأجهزة للإصدار.
- **أدوات المشغلين:** تتيح التحقق من البرنامج النصي الذي يمكن للعملاء تشغيله محليًا (`./docs/portal/scripts/preview_verify.sh --build-dir build --descriptor <path> --archive <path>`)؛ البرنامج النصي يصحح المجموع الاختباري + الوصف. يتم إنشاء وحدة التحكم القياسية (`npm run serve`) من خلال هذا المستشعر التلقائي قبل `docusaurus serve` تم حذف الصور المحلية من خلال المجموع الاختباري للتحكم (من خلال هذا `npm run serve:verified` المغطى باسم مستعار واحد).

## السبب 1 — التحكم في CI

1. قم بتثبيت `.github/workflows/docs-portal-preview.yml`، لذلك:
   - قم بإغلاق `node docs/portal/scripts/write-checksums.mjs` بعد Docusaurus (يمكنك الوصول إليه محليًا).
   - استخدام `cd build && sha256sum -c checksums.sha256` وتمكين الوظيفة في حالة عدم وجودها.
   - تعبئة الدليل الذي تم إنشاؤه في `artifacts/preview-site.tar.gz`، ونسخ المجموع الاختباري للبيان، واختيار `scripts/generate-preview-descriptor.mjs`، واستخدام `scripts/sorafs-package-preview.sh` باستخدام تكوين JSON (sm.`docs/examples/sorafs_preview_publish.json`)، لسير العمل المتغير والمتغير، ونطاق SoraFS المحدد.
   - زيارة الموقع الإحصائي، المصنوعات التحويلية (`docs-portal-preview`، `docs-portal-preview-metadata`) وSoraFS-bandl (`docs-portal-preview-sorafs`)، من أجل توضيح أن سيارة المياه والخطة يمكن التحقق منها بدون أحزمة خلفية.
2. قم بإضافة تعليق باستخدام CI-Beijgem، ونتائج السيرة الذاتية تحقق من المجموع الاختباري في طلب السحب (يتم تحقيقه من خلال تعليق GitHub Script في `docs-portal-preview.yml`).
3. قم بتوثيق سير العمل في `docs/portal/README.md` (razdel CI) وقم بإجراء اختبارات التحقق في منشورات القائمة.

## فحص النص البرمجي

`docs/portal/scripts/preview_verify.sh` هو التحقق من المنتجات المصنّعة مسبقًا بدون استخدامات قوية `sha256sum`. استخدم `npm run serve` (أو الاسم المستعار الجديد `npm run serve:verified`)، لتشغيل البرنامج النصي و`docusaurus serve` لمفتاح واحد распространении localьный sнимков. الاختبارات المنطقية:

1. قم بتشغيل أداة SHA التالية (`sha256sum` أو `shasum -a 256`) مقابل `build/checksums.sha256`.
2. عند الحاجة، قم بقراءة الملخص/لديك وصف للملف المقترح `checksums_manifest`، وإذا قمت بتحميل الملخص/لديك أرشيف للملف مقترح.
3. قم بالاستعانة برمز غير محدد لأي شيء غير مرغوب فيه بحيث يمكن للمستهلكين حظر المزايا الإضافية.

الاستخدام الأولي (بعد استخراج قطعة أثرية CI):

```bash
./docs/portal/scripts/preview_verify.sh \
  --build-dir build \
  --descriptor artifacts/preview-descriptor.json \
  --archive artifacts/preview-site.tar.gz
```

يتعين على مهندسي CI والمنتجين اكتشاف البرنامج النصي مرة واحدة عند تنزيل حزمة المقترحات أو استبدال العناصر تذكرة موثوقة.

## السبب 2 — النشر SoraFS

1. إعادة صياغة مقترحات سير العمل، بما في ذلك:
   - قم بزيارة موقع الويب الخاص بالقسم المرحلي SoraFS باستخدام `sorafs_cli car pack` و`manifest submit`.
   - قم بملء بيان الملخص الصوتي وCID SoraFS.
   - تسلسل `{ commit, branch, checksum_manifest, cid }` في Norito JSON (`docs/portal/preview/preview_descriptor.json`).
2. قم بتسليم الوصف باستخدام شعيرات مصطنعة وإظهار CID في التعليقات وطلب السحب.
3. قم بإجراء اختبارات التكامل التي يتم إجراؤها بواسطة `sorafs_cli` في نظام التشغيل الجاف من أجل تحسين التخزين المخططات المتطوره.

## السبب 3 — الإدارة والتدقيق

1. نشر نظام Norito (`PreviewDescriptorV1`)، وصف الهيكل الوصفي، في `docs/portal/schemas/`.
2. التعرف على منشورات الاختيار DOCS-SORA، يجب:
   - اضغط على `sorafs_cli manifest verify` لحماية CID المحمي.
   - قم بإصلاح المجموع الاختباري لبيان الملخص ومعرف البحث الجنائي في قائمة العلاقات العامة الموثوقة.
3. قم بإضافة التحكم الآلي للأكواب الموصوفة بالمجموع الاختباري للبيان أثناء وقت الإصدار.

## الأسئلة المفتوحة

- ما هو نوع SoraFS من القطع الأثرية المقترحة (ممر المعاينة مقابل التدريج)؟
- هل ترغب في تقديم طلبين (Ed25519 + ML-DSA) للوصف المقترح قبل النشر؟
- قم بتكملة سير العمل CI بتكوين الأوركسترا (`orchestrator_tuning.json`) عند البدء في `sorafs_cli`، وذلك لدعم عملية الإنتاج البيانات؟

قم بتأكيد القرار في `docs/portal/docs/reference/publishing-checklist.md` واطلع على هذه الخطة بعد الترقية غير المعروفة.

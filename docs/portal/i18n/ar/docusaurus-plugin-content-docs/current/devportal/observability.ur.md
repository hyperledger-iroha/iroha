# أجهزة الكمبيوتر المحمولة والإلكترونيات

DOCS-SORA معاينة بناء التحليلات والمسابر الاصطناعية وأتمتة الارتباط المقطوع.
حديث ولوحة سباكة وبوابة جوية لسفينة ساتلية ومشغلي مراقبة التزلج
بيانات زوار بغداد ليك کئے۔

## إطلاق العلامات

- `DOCS_RELEASE_TAG=<identifier>` سيت كريي (احتياطي `GIT_COMMIT` أو `dev`)
  يمكن لـ Willio `<meta name="sora-release">` حقن مجسات ونقرات وعمليات نشر لوحات المعلومات لتمييز الكرات.
- `npm run build` `build/release.json` تنبعث کرتا ہے (جسے `scripts/write-checksums.mjs` لکھتا ہے)
  علامة جو، والطابع الزمني، و`DOCS_RELEASE_SOURCE` الاختياري تصف كرتا ہے۔ هناك حزمة من القطع الأثرية المفضلة للمعاينة
  يشير التقرير ومدقق الارتباط إلى الملاحظة.

## تحليلات الحفاظ على الخصوصية

- `DOCS_ANALYTICS_ENDPOINT=<https://collector.example/ingest>` تكوين تمكين تعقب خفيف الوزن.
  تتضمن الحمولات النافعة `{ event, path, locale, release, ts }` معلومات مرجعية أو بيانات تعريف IP، و
  `navigator.sendBeacon` لا يمكن استخدام كتلة التنقل.
- `DOCS_ANALYTICS_SAMPLE_RATE` (0-1) مركز التحكم في أخذ العينات. تعقب آخر مسار تم إرساله ذخیرہ کرتا ہے
  ولا يصدر أي تنقل لأحداث مكررة.
- تم تثبيت التنفيذ `src/components/AnalyticsTracker.jsx` و `src/theme/Root.js` عالميًا.

## المجسات الاصطناعية

- `npm run probe:portal` المسارات العامة کے خلاف طلبات GET بھیجتا ہے
  (`/`، `/norito/overview`، `/reference/torii-swagger`، إلخ) وتحقق من صحة البطاقة
  العلامة الوصفية `sora-release` `--expect-release` (أو `DOCS_RELEASE_TAG`) تتطابق مع الكرتا. مثال:

```bash
PORTAL_BASE_URL="https://docs.staging.sora" \
DOCS_RELEASE_TAG="preview-42" \
npm run probe:portal -- --expect-release=preview-42
```

تم إنشاء تقرير الفشل في مسار الحساب، وهو عبارة عن بوابة القرص المضغوط التي يمكن الوصول إليها بسهولة.

## أتمتة الارتباط المكسور

- `npm run check:links` `build/sitemap.xml` مسح كرتا ہے، ہر إدخال ملف محلي سے خريطة ہونا یقینی بناتنا ہے
  (`index.html` احتياطيات مثل هذه)، و`build/link-report.json` لكهاتا وهو ما يؤدي إلى إصدار البيانات الوصفية، والإجماليات، والإخفاقات،
  و`checksums.sha256` ka SHA-256 بصمة الإصبع تتضمن أوتا (جو `manifest.id` هو كشف أوتا) هذا تقرير
  بيان قطعة أثرية سے جوڑي جا سے۔
- البرنامج النصي غير الصفري عند الخروج من كرتا ومن خلال الصفحة المفقودة مفقود، مثل CI پراني أو الطرق المعطلة لإصدارات روك سكتا ہے.
  التقارير التي تستشهد بالمسارات المرشحة وتحاول تكرارها، كما يمكنك تتبع مسار الانحدارات في شجرة المستندات.

## Grafana لوحة المعلومات والتنبيهات

- `dashboards/grafana/docs_portal.json` Grafana board **نشر بوابة المستندات** نشر کرتا ہے۔
  تتضمن هذه اللوحات ما يلي:
  - *رفض البوابة (5 م)* `torii_sorafs_gateway_refusals_total` إلى `profile`/`reason` نطاق الاستخدام المستمر
    تعمل سياسة SREs على دفع السياسات أو فشل الرمز المميز في اكتشاف الخلل.
  - *نتائج تحديث ذاكرة التخزين المؤقت للاسم المستعار* و *الاسم المستعار لإثبات العمر ص90* `torii_sorafs_alias_cache_*` لتتبع البطاقة
    تم قطع DNS حتى أكثر من البراهين الجديدة المتاحة.
  - * عدد بيانات سجل الدبوس * و * عدد الأسماء المستعارة النشطة * تراكم إحصائيات سجل الدبوس وإجمالي الأسماء المستعارة التي تعكس الرقم التسلسلي
    الحوكمة هي إطلاق سراح ومراجعة الحسابات.
  - *انتهاء صلاحية بوابة TLS (ساعات)* حان الوقت لتسليط الضوء على بوابة النشر التي تنتهي فيها صلاحية شهادة TLS
    (عتبة التنبيه 72 ساعة)۔
  - *نتائج النسخ المتماثل لاتفاقية مستوى الخدمة* و *تراكم النسخ المتماثل* `torii_sorafs_replication_*` القياس عن بعد
    قم بالنشر بعد كل النسخ المتماثلة من شريط GA.
- متغيرات القالب المضمنة (`profile`، `reason`) استخدم ملف تعريف النشر `docs.sora` للتركيز على جا سکے
  أو كل البوابات تحتوي على مسامير تحقق في كل شيء.
- لوحات معلومات توجيه PagerDuty والأدلة المستخدمة: التنبيهات
  سلسلة `DocsPortal/GatewayRefusals` و`DocsPortal/AliasCache` و`DocsPortal/TLSExpiry` من سلسلة الحرائق ذات الصلة
  عتبات عبور کرے۔ دليل التشغيل التنبيهي الموجود على هذه الصفحة هو الرابط الذي يستخدمه المهندسون عند الطلب لإعادة تشغيل استعلامات Prometheus الدقيقة.

## تجميعها معًا

1. `npm run build` يقوم بضبط متغيرات بيئة الإصدار/التحليلات وخطوة ما بعد الإنشاء
   `checksums.sha256` و`release.json` و`link-report.json` ينبعث منها إشعاع.
2. معاينة اسم المضيف الذي يمثل الاختلاف `npm run probe:portal` و `--expect-release` الذي يمثل علامة الأسلاك الخاصة بك.
   قائمة التحقق من النشر محفوظات النشر.
3. `npm run check:links` قم بإدخال إدخالات خريطة الموقع المعطلة في حالة فشل الجلد وإنشاء تقرير JSON التالي
   معاينة القطع الأثرية في أرشيف مستمر. أحدث تقرير لـ CI هو `artifacts/docs_portal/link-report.json`
   قم بتنزيل سجلات بناء الحوكمة وحزمة الأدلة.
4. نقطة نهاية التحليلات هي أداة تجميع تحافظ على الخصوصية (معقولة، استيعاب OTEL مستضاف ذاتيًا)
   ويتم إصدار وثيقة معدلات أخذ العينات لأعداد لوحات المعلومات التي تفسر بشكل صحيح.
5. قم بمعاينة/نشر سير العمل من خلال CI في خطوات توصيل كل ما تحتاجه
   (`.github/workflows/docs-portal-preview.yml`،
   `.github/workflows/docs-portal-deploy.yml`)، وهو عبارة عن عمليات تشغيل جافة محلية تغطي السلوكيات الخاصة بالأسرار فقط.

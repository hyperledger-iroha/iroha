# تقوية الأمان وقائمة اختبار القلم

## فيساو جيرال

عنصر خريطة الطريق **DOCS-1b** يتطلب تسجيل الدخول إلى رمز جهاز OAuth، سياسة قوية لتأمين المحتوى
يتم إعادة اختراق الخصيتين بشكل متكرر قبل معاينة البوابة، ويتم ذلك في أماكن المختبر. إستي
ملحق يشرح نموذج أمريكا، والضوابط المطبقة بدون مستودع وقائمة مرجعية للبث المباشر
سيتم تنفيذ مراجعات بوابة نظام التشغيل.

- **Escopo:** o proxy جربه، استخدم Swagger/RapiDoc embutidos ووحدة التحكم جربه من خلال عرض مخصص
  `docs/portal/src/components/TryItConsole.jsx`.
- **لمنتدى المساعدة:** Torii em si (تم تقديمه من خلال مراجعات الاستعداد Torii) والنشر SoraFS
  (كوبيرتا بور DOCS-3/7).

## نموذج أمريكا

| أتيفو | ريسكو | ميتيجاكاو |
| --- | --- | --- |
| حامل الرموز يفعل Torii | قم بتدوير أو إعادة استخدام منتدى وضع الحماية للمستندات | يقوم رمز جهاز تسجيل الدخول (`DOCS_OAUTH_*`) بإصدار الرموز المميزة للمدة القصيرة، أو يقوم الوكيل بإعادة تحميل الرؤوس، وتنتهي صلاحية اعتماد وحدة التحكم في ذاكرة التخزين المؤقت تلقائيًا. |
| الوكيل جربه | إساءة استخدام التتابع المفتوح أو تجاوز حدود Torii | `scripts/tryit-proxy*.mjs` تطبيق القوائم المسموح بها الأصلية وتحديد المعدل وتحقيقات السلامة وإعادة التوجيه الصريحة لـ `X-TryIt-Auth`؛ لا يوجد اعتماد واستمرارية. |
| وقت تشغيل البوابة | البرمجة النصية عبر المواقع أو تضمين البرامج الضارة | `docusaurus.config.js` رؤوس injeta سياسة المحتوى والأمان والأنواع الموثوقة وسياسة الأذونات؛ البرامج النصية المضمنة محظورة في وقت التشغيل لـ Docusaurus. |
| بيانات القدرة على الملاحظة | القياس عن بعد عن بعد أو الزنا | `docs/portal/docs/devportal/observability.md` مجسات/لوحات معلومات دوكومنتا؛ `scripts/portal-probe.mjs` تم توجيهه إلى CI قبل النشر. |

يتضمن الخصوم مستخدمين فضوليين يقومون ببيع أو معاينة الجمهور، والأدوات الضارة، والروابط الخبيثة،
المتصفحات المتوافقة مع Tentando Extrair Credenciais Armazenadas. تعمل جميع عناصر التحكم
المتصفحات المشتركة لا تحتوي على معلومات سرية.

## الضوابط المطلوبة

1. ** تسجيل الدخول برمز جهاز OAuth **
   - تكوين `DOCS_OAUTH_DEVICE_CODE_URL`، `DOCS_OAUTH_TOKEN_URL`،
     `DOCS_OAUTH_CLIENT_ID` والمقابض المتعلقة بالبيئة المحيطة.
   - حاول تقديم بطاقة تسجيل الدخول لعنصر واجهة المستخدم (`OAuthDeviceLogin.jsx`)
     ابحث عن رمز الجهاز، واستقصاء سريع بدون نقطة نهاية رمزية، وامسح تلقائيًا
     الرموز عند انتهاء الصلاحية. يتجاوز دليل الحامل المتاح دائمًا
     احتياطي للطوارئ.
   - يبني نظام التشغيل falham quando a configuracao OAuth esta ausente ou quando os
     TTLs الاحتياطية هي 300-900 ثانية فقط من DOCS-1b؛ ضبط
     `DOCS_OAUTH_ALLOW_INSECURE=1` فقط لمعاينة المواقع التي تم تنزيلها.
2. ** الدرابزين يعمل بالوكالة **
   - `scripts/tryit-proxy.mjs` تطبيق الأصول المسموح بها وحدود المعدلات وحدود حجم الطلب
     مهلات المنبع في العلامة التجارية o trafego com `X-TryIt-Client` e
     إزالة سجلات دوس الرموز المميزة.
   - `scripts/tryit-proxy-probe.mjs` ولكن `docs/portal/docs/devportal/observability.md`
     تحديد مسبار الحيوية وأنظمة لوحة القيادة؛ تنفيذ نظام التشغيل قبل كل شيء
     الطرح.
3. **CSP، الأنواع الموثوقة، سياسة الأذونات**
   - `docusaurus.config.js` تصدير رؤوس التحديد الآمن:
     `Content-Security-Policy` (افتراضي-src ذاتي، قوائم خيارات الاتصال/img/script،
     متطلبات الأنواع الموثوقة)، `Permissions-Policy`، e
     `Referrer-Policy: no-referrer`.
   - قائمة الاتصال الخاصة بـ CSP تُدرج نقاط النهاية البيضاء لـ OAuth لرمز الجهاز والرمز المميز
     (أحيانًا HTTPS إلى `DOCS_SECURITY_ALLOW_INSECURE=1`) لتسجيل الدخول إلى الجهاز
     الوظيفة ليست مريحة أو رمل للأصول الأخرى.
   - الرؤوس مضمنة مباشرة ولا يتم تشغيلها بتنسيق HTML، بما في ذلك المضيفين الثابتين دون الحاجة إلى ذلك
     دي configuracao اضافية. نصوص Mantenha المضمنة محدودة في التمهيد حتى Docusaurus.
4. **دفاتر التشغيل وإمكانية المراقبة والتراجع**
   - `docs/portal/docs/devportal/observability.md` يكشف عن المسابير ولوحات المعلومات التي
     مراقبة فشل تسجيل الدخول، ورموز الاستجابة للوكيل، وميزانيات الطلب.
   - `docs/portal/docs/devportal/incident-runbooks.md` النحاس أو سلم التسلق
     حد ذاته رمل للإساءة؛ الجمع بين كوم
     `scripts/tryit-proxy-rollback.mjs` لتأمين نقاط النهاية.

## قائمة التحقق من الإصدار الإلكتروني للاختبار

أكمل هذه القائمة لكل عرض ترويجي للمعاينة (ملحق النتائج بعد تذكرة الإصدار):

1. ** التحقق من أسلاك OAuth **
   - تنفيذ `npm run start` محليًا مع صادرات نظام التشغيل `DOCS_OAUTH_*` de producao.
   - من ملف متصفح غير مكتمل، استخدم وحدة التحكم وجربها وتأكد من ذلك
     يصدر رمز جهاز التدفق رمزًا مميزًا، ويحفظ لمدة طويلة وينتهي بعد انتهاء الصلاحية
     ou fazer تسجيل الخروج.
2. ** بروفار أو وكيل **
   - `npm run tryit-proxy` ضد Torii التدريج، تنفيذ depois
     `npm run probe:tryit-proxy` مع تكوين مسار العينة.
   - التحقق من سجلات الإدخالات `authSource=override` والتأكد من تحديد المعدل
     عدادات الزيادة عندما يتجاوز الصوت جانيلا.
3. **تأكيد مقدم خدمات الطاقة/الأنواع الموثوقة**
   - `npm run build` والعبر `build/index.html`. تأكد من وجود علامة `<meta
     http-equiv="Content-Security-Policy">` يتوافق مع التوجيهات المتوقعة
     كما أن DevTools لا ترتكب أي انتهاكات لـ CSP عند تشغيلها أو معاينةها.
   - استخدم `npm run probe:portal` (أو حليقة) للبحث أو نشر HTML؛ مسبار
     Agora falha quando كعلامات وصفية `Content-Security-Policy`، `Permissions-Policy` ou
     `Referrer-Policy` موجود أو يختلف عن القيم المعلنة
     `docusaurus.config.js`، يمكن لمراجعي الحوكمة أن يثقوا بالرقم
     رمز الخروج بمجرد التحقق من الإخراج أو تجعيده.
4. **مراجعة إمكانية الملاحظة**
   - التحقق من لوحة القيادة للقيام بالوكيل جرب ذلك (حدود المعدل، ونسب الخطأ،
     مقاييس الصحة).
   - تنفيذ أو حفر الحوادث على `docs/portal/docs/devportal/incident-runbooks.md`
     إذا كان المضيف مودو (نشر جديد Netlify/SoraFS).
5. **نتائج توثيقية**
   - لقطات شاشة/سجلات مرفقة بعد إصدار التذكرة.
   - التقط كل مرة لم تجد فيها قالبًا لعلاقات الإصلاح
     ([`docs/examples/pentest_remediation_report_template.md`](../../../examples/pentest_remediation_report_template.md))
     بالنسبة للمالكين واتفاقيات مستوى الخدمة وأدلة إعادة الاختبار، سيتم إجراء التدقيق بعد ذلك.
   - رابط إلى قائمة المراجعة هذه حتى يظل عنصر خريطة الطريق DOCS-1b قابلاً للتدقيق.

إذا حدث شيء ما، قم بالترويج، قم بإيقاف مشكلة ما وقم بتعليق خطة الإصلاح في `status.md`.

# مرجع الترميز Norito

Norito Iroha هي طبقة التسلسل الأساسية. يتم استخدام الرسائل السلكية والحمولة النافعة على القرص وواجهة برمجة التطبيقات (API) Norito عبر المكونات وعقد العقد المختلفة للأجهزة عبر وحدات البايت. هذه الصفحة هي ملخص الكتاب والمواصفات الكاملة لـ `norito.md` نصيحة الكتاب.

## التخطيط الأساسي

| مكون | الغرض | المصدر |
| --- | --- | --- |
| **الرأس** | تجزئة السحر/الإصدار/المخطط، CRC64، الطول، وعلامة الضغط هي الحمولات الصافية التي تحتوي على فريم كرتا؛ v1 يحتوي على `VERSION_MINOR = 0x00` الضروري وأعلام الرأس التي تدعم القناع الذي يقابل التحقق من صحة جاتا (الافتراضي `0x00`). | `norito::header` — `norito.md` ("الرأس والأعلام"، جذر المستودع) دیکھیں |

## الحصول على الدعم

`norito_derive` `Encode`، `Decode`، `IntoSchema` ومساعد JSON يشتق فرام كرتا ہے. ام الاتفاقيات:

- اشتقاق AoS ومسارات التعليمات البرمجية المعبأة دونو بنات؛ تخطيط v1 AoS (الإشارات `0x00`) والمتغيرات المعبأة بأعلام الرأس الافتراضية التي لا يتم تمكينها. التنفيذ `crates/norito_derive/src/derive_struct.rs`.
- ميزات التخطيط والتنفيذ (`packed-struct`، `packed-seq`، `compact-len`) علامات الرأس التي يتم الاشتراك فيها والأقران الذين يقومون بالتشفير/فك التشفير المتسق چہئيں.
- مساعدو JSON (`norito::json`) يفتحون واجهات برمجة التطبيقات (APIs) الحتمية Norito المدعومة JSON. استخدام `norito::json::{to_json_pretty, from_json}` — لا يوجد `serde_json`.

## الترميز المتعدد وجداول المعرفات

Norito تعيينات الترميز المتعدد `norito::multicodec` مكررة. الجدول المرجعي (التجزئة، أنواع المفاتيح، واصفات الحمولة النافعة) جذر المستودع `multicodec.md` هو الحل الأمثل. المعرف الجديد يشمل:1.`norito::multicodec::registry` بطاقة الائتمان.
2. `multicodec.md` يحتوي على جدول.
3. إذا تم استخدام خريطة الارتباطات النهائية (Python/Java) فسيتم تجديدها.

# قائمة التحقق من النشر

يتم استخدام هذه القائمة كل مرة يتم فيها تحديث بوابة المطورين. نضمن أن بناء CI، والنشر على صفحات GitHub، واختبارات الأدلة البشرية تغطي كل جزء قبل أن يكون هناك إصدار أو نتيجة لخريطة الطريق.

## 1. التحقق من الصحة محليًا

- `npm run sync-openapi -- --version=current --latest` (اجمع علامة واحدة أو أكثر `--mirror=<label>` عندما أقوم بتغيير Torii OpenAPI للقطة مجمعة).
- `npm run build` - يؤكد أن نسخة البطل `Build on Iroha with confidence` تظهر في `build/index.html`.
- `./docs/portal/scripts/preview_verify.sh --build-dir build` - التحقق من بيانات المجاميع الاختبارية (اجمع `--descriptor`/`--archive` للتحقق من العناصر التي تم تنزيلها من CI).
- `npm run serve` - قم بتشغيل مساعد المعاينة من خلال بوابة المجموع الاختباري للتحقق من البيان قبل الاتصال بـ `docusaurus serve`، حتى لا يتمكن المراجعون من تصفح لقطة بدون تأكيد (الاسم المستعار `serve:verified` متاح للمكالمات الصريحة).
- قم بمراجعة تخفيض السعر الذي يتم بثه عبر `npm run start` وخادم إعادة التحميل المباشر.

## 2. طلب ​​سحب الشيكات- التحقق من أن المهمة `docs-portal-build` قد مرت على `.github/workflows/check-docs.yml`.
- تأكد من تشغيل `ci/check_docs_portal.sh` (تقوم سجلات CI بإرسال فحص دخان البطل).
- تأكد من أن معاينة سير العمل ستقدم بيانًا (`build/checksums.sha256`) وأن برنامج التحقق من المعاينة سيخرج (تخرج السجلات من `scripts/preview_verify.sh`).
- قم بإضافة عنوان URL للمعاينة المنشورة من داخل صفحات GitHub إلى وصف العلاقات العامة.

## 3. الموافقة على القسم| القسم | المالك | قائمة المراجعة |
|---------|-------|-----------|
| الصفحة الرئيسية | ديفريل | يتم عرض البطل، ويتم تنشيط بطاقة البدء السريع على طول المسار الصحيح، ويتم إعادة توجيه أزرار CTA. |
| Norito | Norito WG | تشير أدلة النظرة العامة والبدء إلى أحدث علامات CLI ووثائق السؤال Norito. |
| SoraFS | فريق التخزين | يتم تنفيذ البدء السريع سريعًا، ويتم توثيق مجالات تقرير البيان وتعليمات محاكاة الجلب التي تم التحقق منها. |
| غياس SDK | ليدريس SDK | تجمع نماذج Rust/Python/JS النماذج الفعلية وتعززها في الحياة الحية. |
| المرجعية | مستندات/ديفريل | يسرد المؤشر أحدث المواصفات، ويتزامن مرجع برنامج الترميز Norito مع `norito.md`. |
| قطعة أثرية للمعاينة | مستندات/ديفريل | القطعة الأثرية `docs-portal-preview` هي ملحقة بالعلاقات العامة، ويتم فحص الدخان، ويتم مشاركتها مع المراجعين. |
| الأمان وجرب وضع الحماية | مستندات/DevRel Security | تم تكوين تسجيل الدخول برمز جهاز OAuth (`DOCS_OAUTH_*`)، وتم تنفيذ قائمة التحقق `security-hardening.md`، وتم التحقق من CSP/الأنواع الموثوقة عبر `npm run build` أو `npm run probe:portal`. |

قم بتمييز كل جزء باعتباره جزءًا من مراجعة العلاقات العامة الخاصة بك، أو قم بتعليق المهام التي تريدها حتى يكون تتبع الحالة دقيقًا.

## 4. ملاحظات التخطيط- قم بتضمين `https://docs.iroha.tech/` (أو عنوان URL الأصلي لمهمة النشر) في مذكرات الإصدار والتحديثات الحالية.
- قم بإزالة أي قسم جديد أو تم تعديله حتى تتمكن المعدات النهائية من تنفيذ مهامها التجريبية.

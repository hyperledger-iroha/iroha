تطوير Nexus (Iroha 3) Iroha 2 يسمح بمسارات متعددة، ومساحات بيانات محددة بالـ تور، للتجارة والتعاون عبر كل SDK. احترام هذه الصفحة الملخص الجديد `docs/source/nexus_overview.md` في مستودع الأحادي حتى يتمكن قراء البوابة من فهم كيفية ترابط أجزاء بسرعة.

## اختلافات

- **Iroha 2** - عمليات نشر مستضافة ذاتيا للكونسورتيوم أو الشبكات الخاصة.
- **Iroha 3 / Sora Nexus** - الشبكة العامة متعددة المسارات حيث يتم تسجيل تشغيل مساحة البيانات (DS) ويرثون أدوات التعاون للتعاون والتسويق والملاحظة.
- الخط كلاين يُبنيان من نفس مساحة العمل (IVM + toolchain Kotodama)، لذا تبقى إصلاحات SDK وتحديثات ABI وfixtures Norito قابلة للنقل. يقوم بتنزيل حزمة `iroha3-<version>-<os>.tar.zst` للانضمام إلى Nexus؛ راجع `docs/source/sora_nexus_operator_onboarding.md` لقائمة التحقق بملل الشاشة.

## اللبنات الأساسية

| المكون | الملخص | روابط البوابة |
|-----------|--------|--------------|
| مساحة (DS) | نطاق كامل/تخزين محدد خاص يمتلك مسارًا واحدًا أو أكثر، ويعلن عن محددين وفئة الخصوصية وسياسة الخصوصية + DA. | مراجعة [Nexus spec](./nexus-spec) لمخطط البيان. |
| لين | شريحة تنفيذ حتمية تعهدات يرتبها حلقة NPoS العالمية. تمت إضافة Lane `default_public` و`public_custom` و`private_permissioned` و`hybrid_confidential`. | يلتقط [نموذج لين](./nexus-lane-model) الهندسة وبوادئ وتخزين الاحتفاظ. |
| خطة الانتقال | معرفات نائبة ومراحل توجيه وتعبئة بملفين تتبع كيف يتم الإبلاغ عن عمليات النشر الأحادية المسار إلى Nexus. | ت [ملاحظات الانتقال](./nexus-transition-notes) كل مرحلة الرحيل. |
| دليل الفضاء | عقد سجل يخزن بيانات DS ونسخها. يقوم بمطابقة التنسيقات الكتالوجية مع هذا الدليل قبل الانضمام. | متعقب فروقات البيان موجود تحت `docs/source/project_tracker/nexus_config_deltas/`. |
| كتالوج المسارات | قسم المتعدد `[nexus]` يربط معريفات لين بالأسماء المستعارة وسياسات التوجيه وعتبات DA. يقوم `irohad --sora --config … --trace-config` بطباعة الكتالوج المهموم لأغراض التدقيق. | استخدم `docs/source/sora_nexus_operator_onboarding.md` لشرح سطر لاسلكي. |
| وتتطلع إلى التوسع | منسق تحويلات XOR الذي يربط مسارات CBDC الخاصة بمسارات شاملة عامة. | يوضح `docs/source/cbdc_lane_playbook.md` مقاييس السياسة وبوابات القياس. |
| القياس/SLOs | لوحات والتنبيه تحت `dashboards/grafana/nexus_*.json` تحديد ارتفاع المسارات وتراكم DA وزمن الاتفاقية وعمق طابور التصفح. | توضح [خطة معالجة القياس](./nexus-telemetry-remediation) اللوحات والتنبيهات وأدلة التدقيق. |

##لقطة جديدة

| المرحلة | التركيز | معايير الخروج |
|-------|-------|---------------|
| N0 - بيتا مغلقة | المسجل يديره المجلس (`.sora`)، الأفراد المخصصين للشغلين، كتالوج مسارات ثابتة. | بيانات موقع DS + عمليات بيع وشراء مجربة. |
| N1 - الكمية العامة | لاحقات `.nexus`، مزايا، مسجل بالخدمة الذاتية، توصيل اتفاقية XOR. | السيولة تزامن محلل/بوابة، لوحات اتفاقية الفاتورة، دورات محاكاة للنزاعات. |
| N2 - تطوير | مقدمة `.dao`، واجهات API للموزعين، تحليلات، بوابة شهداء، بطاقات أداء المشرفين. | غالبًا ما يتم إصدار التقارير الجديدة، مجموعة أدوات جسم العمل، تقارير حديثة عن الخزان. |
| بوابة NX-12/13/14 | يجب أن يأتي المحرك قبل ذلك ولوحات القياس والوثائق مع التجارب الناشئة. | نشر [نظرة عامة على Nexus](./nexus-overview) + [عمليات Nexus](./nexus-operations)، توصيل اللوحات، دمج محرك الجيل الجديد. |

##لعملات الخاصة بهم

1. **نظافة** - أبقِ `config/config.toml` متزامنا مع كتالوج مسارات وبيانات ومساحات المنشور؛ وأرشِف مخرجات `--trace-config` مع كل تذكرة الإصدار.
2. **تتبع البيانات** - طابق خطوط الكتالوج مع أحدث حزمة دليل الفضاء قبل الانضمام أو الاشتراك.
3. **تغطية القياس** - اعرض اللوحات `nexus_lanes.json` و`nexus_settlement.json` لوحات SDK ذات الصلة؛ صوت التنبيهات بـ PagerDuty ونفّذ مراجعات ترجمات سنوية حسب لخطة القياس.
4. **الإبلاغ عن** - اتبع مصفوفة الأخضر في [عمليات Nexus](./nexus-operations) وتقارير RCA خلال خمسة أيام عمل.
5. **الجاهزية للمراقبة** - احضر تصويتات مجلس Nexus التي تسعى إلى مساراتك وتدرّب على تعليمات الرجوع إلى ربع الإبلاغ (يتم تتبعها عبر `docs/source/project_tracker/nexus_config_deltas/`).

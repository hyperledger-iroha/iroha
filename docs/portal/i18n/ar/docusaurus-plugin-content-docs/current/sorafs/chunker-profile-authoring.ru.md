# إنشاء ملف تعريف تأليف Chunker SoraFS

هذا أمر رائع من أجل عرض ونشر ملفات تعريف جديدة لـ SoraFS.
هذا هو التصميم المعماري الإضافي RFC (SF-1) والمسجل الافتراضي (SF-2a)
المتطلبات المحددة للتأليف وشروط التحقق والاقتراحات.
في المثال القانوني تمامًا سم.
`docs/source/sorafs/proposals/sorafs_sf1_profile_v1.json`
وسجل التشغيل الجاف الرائع في
`docs/source/sorafs/reports/sf1_determinism.md`.

## ملاحظة

الملف الشخصي الذي تم نشره في السجل، مطلوب:

- الالتزام بتحديد معلمات مراكز مكافحة الأمراض والوقاية منها (CDC) وأنظمة التجزئة المتعددة، المتماثلة على الإطلاق
  الهندسة المعمارية.
- نشر التركيبات المجهزة (JSON Rust/Go/TS + Fuzz corpora + PoR شاهد)، والتي
  يمكن لمجموعات SDK المتلقية للمعلومات التحقق من دون الأدوات المتخصصة؛
- تضمين البيانات الوصفية والكلمات المتعلقة بالإدارة (مساحة الاسم والاسم والعلامة التجارية) وكذلك التوصيات
  حول الهجرة والحداثة التامة; و
- قم بالتوصل إلى مجموعة الفروق المحددة لمراجعة النصائح.

اتبع الاختيار لتتمكن من تقديم الاقتراحات، وهذا هو الصحيح.## مسجل مقطع الفيديو

قبل الاقتراح الجيد، عليك أن تعرف أن هذا هو ما يعجبك في تذكرة السفر،
نوع المنتج `sorafs_manifest::chunker_registry::ensure_charter_compliance()`:

- ملف تعريف المعرف - شاشة كاملة كاملة، رتيبة بدون حشو.
- المقبض الكنسي (`namespace.name@semver`) يجب إدخال الاسم المستعار في القائمة
- لا يوجد أي اسم مستعار يتعارض مع المقبض الكنسي الآخر والرجوع إليه.
- الاسم المستعار долзны быть непустыми и beз probelov по kraям.

نصائح CLI المفيدة:

```bash
# JSON список всех зарегистрированных дескрипторов (ids, handles, aliases, multihash)
cargo run -p sorafs_manifest --bin sorafs_manifest_chunk_store -- --list-profiles

# Эмитить метаданные для кандидата на профиль по умолчанию (канонический handle + aliases)
cargo run -p sorafs_manifest --bin sorafs_manifest_chunk_store -- \
  --promote-profile=sorafs.sf1@1.0.0 --json-out=-
```

تقوم هذه الأوامر بتوجيه المقترحات من خلال مسجل الرحلات والسجل الكنسي
التحول إلى الحوكمة الملتزمة.

### ميزات مميزة لـ CLI والمسجل

- `sorafs_manifest_chunk_store --profile=<handle>` — إعادة التدوير مرة أخرى,
  ملخص البيان واختبارات إثبات الأداء مع المعلمات المقترحة.
- `sorafs_manifest_chunk_store --json-out=-` — قم بالخروج من متجر القطع في stdout من أجل
  أتمتة أوتوماتيكية.
- `sorafs_manifest_stub --chunker-profile=<handle>` — التحقق من البيانات والسيارة
  خطط إنشاء المقبض القانوني والأسماء المستعارة.
- `sorafs_manifest_stub --plan=-` — تمهيدًا لـ `chunk_fetch_specs` للاختبار
  إزاحة/ملخصات بعد التوسيع.

قم بإدراج الأمر (الملخصات، وجذور PoR، والتجزئة الواضحة) في الاقتراحات التي يمكن للمراجعين
الاستفهام فيها بكل تأكيد.

## تحديد قائمة الاختيار والتحقق من صحتها1. **تجديد التركيبات**
   ```bash
   cargo run --locked -p sorafs_chunker --bin export_vectors \
     --signature-out=fixtures/sorafs_chunker/manifest_signatures.json
   ```
2. **تثبيت تكافؤ المجموعة** — `cargo test -p sorafs_chunker` وتسخير الفرق بين اللغات
   (`crates/sorafs_chunker/tests/vectors.rs`) يجب أن تكون متصلاً بالتركيبات الجديدة.
3. **إعادة هيكلة الزغب/الضغط الخلفي** — `cargo fuzz list` وأداة التدفق
   (`fuzz/sorafs_chunker`) للأصول المنظمة.
4. **التحقق من شهود إثبات الاسترجاع** — توقف
   `sorafs_manifest_chunk_store --por-sample=<n>` مع الملفات الشخصية المقدمة والتحقق منها،
   ما هي الجذور التي تتوافق مع بيان المباراة.
5. **التشغيل الجاف لـ CI** — يتم تشغيل `ci/check_sorafs_fixtures.sh` محليًا; مطلوب نص
   احصل على التركيبات والتجهيزات الجديدة `manifest_signatures.json`.
6. **التحقق من وقت التشغيل المتقاطع** — تأكد من أن روابط Go/TS يمكن أن يتم تنظيمها
   JSON ويعرض جزيئات وخلاصات متطابقة.

قم بتوثيق الأوامر والملخصات المفيدة في المقترحات بحيث يمكن لـ Tooling WG عرضها دون تأخير.

### تأكيد البيان / PoR

بعد تجديد التركيبات، قم بإغلاق خط أنابيب البيان بالكامل، لتتمكن من الاطلاع على ذلك
تتضمن إثباتات النقل والإثبات الخاصة بالسيارات ما يلي:

```bash
# Проверить метаданные чанка + PoR с новым профилем
cargo run -p sorafs_manifest --bin sorafs_manifest_chunk_store -- \
  --profile=sorafs.sf2@1.0.0 \
  --json-out=- --por-json-out=- fixtures/sorafs_chunker/input.bin

# Сгенерировать manifest + CAR и сохранить chunk fetch specs
cargo run -p sorafs_manifest --bin sorafs_manifest_stub -- \
  fixtures/sorafs_chunker/input.bin \
  --chunker-profile=sorafs.sf2@1.0.0 \
  --chunk-fetch-plan-out=chunk_plan.json \
  --manifest-out=sf2.manifest \
  --car-out=sf2.car \
  --json-out=sf2.report.json

# Повторно запустить с сохраненным планом fetch (защищает от устаревших offsets)
cargo run -p sorafs_manifest --bin sorafs_manifest_stub -- \
  fixtures/sorafs_chunker/input.bin \
  --chunker-profile=sorafs.sf2@1.0.0 \
  --plan=chunk_plan.json --json-out=-
```

خزان المياه المفضل للجسم الذي يمكن استخدامه في تركيباتك
(على سبيل المثال، عملية تحديد النطاق 1 جيجا بايت)، وتطبيق الملخصات المفيدة للاقتراحات.

## سير عمل الحوكمة

1. **ممارسة العلاقات العامة من خلال العروض والتركيبات.** قم بإدراج الأصول الهندسية، Norito
   الاقتراح والمشورة `chunker_registry_data.rs`.
2. **إعادة تعيين مجموعة الأدوات.** يقوم المُراجعون بإغلاق عملية التحقق من صحة النقرة وإعادة التحقق منها،
   ما هو الاقتراح الصحيح هو التسجيل الصحيح (بدون الاستخدام العكسي للمعرف،
   الحتمية مضمونة).
3. **تحويل الأفكار.** بعد الموافقة على الأفكار، قم بتقديم ملخص الاقتراحات
   (`blake3("sorafs-chunker-profile-v1" || canonical_bytes)`) وقم بإضافة المشاركة
   في تحويل الملف الشخصي، استمتع بالتركيبات.
4. ** سجل النشر. ** دمج المسجل والمستندات والتركيبات. من خلال CLI
   الحفاظ على الملف الشخصي السابق، حتى لا تلتزم الإدارة بالهجرة.
5. **إيقاف المراقبة.** بعد نهاية عملية الترحيل، قم بزيارة المسجل، ثم قم بإلغاء الحجز

## نصائح للتأليف- قم بتزويد قطع الرقائق بطبقتين لتقليل تقطيع الصفائح.
- لا تستخدم كود multihash بدون التنسيق مع بيان العملاء والبوابة؛ تفضل
  заметку о совместимости.
- قم بتخزين أقراص تروس البذور، ولكنها فريدة عالميًا لتعزيز التدقيق.
- تخزين العناصر المرجعية (على سبيل المثال، سرعة الإنتاجية) في
  `docs/source/sorafs/reports/` لأنظمة الجسم.

عملية الصيانة خلال فترة الطرح سم. в دفتر الأستاذ الهجرة

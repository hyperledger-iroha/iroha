## المنهجية1. **مصانع نمذجة التهديدات** تقوم برسم خرائط لقدرات الهجوم لمواقع التطوير وأنظمة CI والبيانات Torii.  
2. **مراجعة الكود** لتوضيح أسطح المعرفات (تغيير الرموز المميزة OIDC، التوقيع بدون مفتاح)، والتحقق من صحة البيانات Norito، وتدفق إثبات الضغط الخلفي.  
3. **الاختبارات الديناميكية** توفر بيانات التركيبات ومحاكاة أوضاع اللوحة (إعادة تشغيل الرمز المميز، والتلاعب الواضح، وتدفقات إثبات الثبات) عبر أداة التكافؤ ومحركات الأقراص الزغبية السابقة.  
4. **فحص التكوين** للتحقق من صحة الإعدادات الافتراضية `iroha_config` وإدارة علامات CLI وإصدار البرامج النصية لضمان عمليات التنفيذ المحددة والقابلة للتدقيق.  
5. **إدخال العملية** يؤكد تدفق الإصلاح وآليات التصعيد والتقاط أدلة التدقيق مع مالكي إصدار Tooling WG.

## التحقق من صحة الضوابط- **بطاقة المعرفات:** قوالب CI تفرض خللًا في الجمهور والمصدر الصريح؛ يظهر CLI ومساعد التحرير سريعًا إذا لم يكن `--identity-token-audience` مصاحبًا لـ `--identity-token-provider`.  
- **إعادة التحديد:** تغطي اختبارات اليوم التدفق الإيجابي/السلبي لبيانات البيانات، مع ضمان عدم تطابق الملخصات مع عدم وجود نتائج محددة وإشاراتها قبل لمس الشبكة.  
- **التدفق المضاد للضغط الخلفي:** Torii ينشر عناصر PoR/PoTR عبر القنوات المحمولة، ولا يحافظ CLI على مؤثرات زمن الاستجابة + خمسة أمثلة من الفحص، مما يمنع التقلب بدون حدود طوال الوقت على خلفية السيرة الذاتية الحتمية.  
- **الملاحظة:** تلتقط حسابات إثبات التدفق (`torii_sorafs_proof_stream_*`) والسيرة الذاتية CLI أسباب الإيقاف، وتوفر فتات الخبز لمراجعة المشغلين.  
- **الوثائق:** تشير أدلة التطوير ([فهرس المطورين](../developer-index.md)، [مرجع CLI](../developer-cli.md)) إلى العلامات الحساسة وسير العمل المتصاعد.

## إضافة إلى قائمة الإصدار

يقوم مديرو الإصدار **doivent** بالانضمام إلى الإجراءات التالية أثناء الترويج لمرشح GA :1. قم بتجزئة مذكرة المراجعة الأمنية الأحدث (المستند).  
2. الامتياز مقابل تذكرة الإصلاح التالية (مثال: `governance/tickets/SF6-SR-2026.md`).  
3. يقوم إخراج `scripts/release_sorafs_cli.sh --manifest ... --bundle-out ... --signature-out ...` بإظهار الوسائط التي يوضحها الجمهور/المصدر.  
4. السجلات التي تم التقاطها من حزام التكافؤ (`cargo test -p sorafs_car -- --nocapture sorafs_cli::proof_stream::bounded_channels`).  
5. تأكيد أن ملاحظات الإصدار Torii تتضمن حاسبات اختبار البث عن بعد.

لا تقم بجمع القطع الأثرية التي تمنع تسجيل الخروج GA.

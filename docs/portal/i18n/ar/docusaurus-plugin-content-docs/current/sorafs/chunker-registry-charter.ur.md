# SoraFS ميثاق إدارة التسجيل الخاص بالقطاع

> **التصديق:** 29-10-2025 من قبل لجنة البنية التحتية لبرلمان سورا (انظر
> `docs/source/sorafs/council_minutes_2025-10-29.md`). تم أيضًا استخدام مصطلح لحوكمة الحوكمة؛
> لم يعد التنفيذ ممكنًا بعد أن أصبح الآن مشروعًا معياريًا بعد أن أصبح الميثاق الجديد غير مقبول.

يعمل هذا الميثاق SoraFS على تطوير عملية التسجيل والأدوار التي تحدد المهام.
يوجد [دليل تأليف ملف تعريف Chunker](./chunker-profile-authoring.md) وهو عبارة عن مجموعة من المقالات وأخرى جديدة
الملفات الشخصية تقترح، وتراجع، وتصدق، وترفض كل شيء.

## النطاق

هذا الميثاق `sorafs_manifest::chunker_registry` هو الإدخال الأول و
هذه الأدوات تستخدم أيضًا التسجيل (CLI الواضح، CLI لإعلان الموفر،
أدوات تطوير البرمجيات). أحد الأسماء المستعارة وثوابت التعامل يفرضان قواعد اللعبة
`chunker_registry::ensure_charter_compliance()` نوع القطعة:

- معرفات الملف الشخصي، والأعداد الصحيحة الإيجابية رتيبة بشكل متكرر.
- المقبض الكنسي `namespace.name@semver` **لازم** في `profile_aliases`
- تقليم سلاسل الاسم المستعار كبوابة، وصوت فريد من نوعه، وإدخالات أخرى لا تتصادم فيها المقابض الأساسية.

## الأدوار- ** المؤلف (المؤلفون) ** – اقتراح تاريخ الكتاب، تركيبات تجديد الكتاب، وأدلة الحتمية التي تم جمعها.
- **مجموعة عمل الأدوات (TWG)** – قوائم المراجعة المنشورة للتحقق من صحة الاقتراح ومتغيرات التسجيل.
- **مجلس الإدارة (GC)** – تقرير TWG يقوم بمراجعة الكرتا، ومظروف الاقتراح، وتوقيع الكرتا، والجداول الزمنية للنشر/الإيقاف، الموافقة على الكرتا.
- **فريق التخزين** – الحفاظ على تنفيذ التسجيل كرتا ہے ونشر تحديثات الوثائق کرتا ہے۔

## سير عمل دورة الحياة

1. **تقديم المقترح**
   - دليل تأليف المؤلف کی قائمة التحقق من الصحة چلاتا ہے و
     `docs/source/sorafs/proposals/` تحت `ChunkerProfileProposalV1` JSON بناتنا ہے.
   - يتضمن إخراج CLI:
     ```bash
     cargo run -p sorafs_manifest --bin sorafs_manifest_chunk_store -- --list-profiles
     cargo run -p sorafs_manifest --bin sorafs_manifest_chunk_store -- \
       --promote-profile=<handle> --json-out=-
     cargo run -p sorafs_manifest --bin sorafs_manifest_stub -- \
       --chunker-profile=<handle> --json-out=-
     ```
   - المباريات والاقتراح وتقرير الحتمية وتحديثات التسجيل تتضمن تقديم العلاقات العامة.

2. **مراجعة الأدوات (TWG)**
   - قائمة التحقق من الصحة (التركيبات، الزغب، خط أنابيب البيان/PoR).
   - `cargo test -p sorafs_car --chunker-registry` التخفيضات والقروض
     `ensure_charter_compliance()` إدخال جديد فقط.
   - سلوك CLI (`--list-profiles`، `--promote-profile`، التدفق
     `--json-out=-`) قامت البطاقة الجديدة بتحديث الأسماء المستعارة والمقابض.
   - النتائج وحالة النجاح/الفشل في تقديم ملخص للتقرير المختصر.3. **موافقة المجلس (GC)**
   - تقرير TWG والبيانات الوصفية للمقترحات يتم مراجعتها.
   - ملخص الاقتراح (`blake3("sorafs-chunker-profile-v1" || bytes)`) قم بالتسجيل
     قد تتضمن التوقيعات في مظروف المجلس تركيبات وتركيبات ثابتة.
   - نتيجة التصويت في محضر الحكم.

4. **النشر**
   - دمج العلاقات العامة والإعلانات:
     -`sorafs_manifest::chunker_registry_data`.
     - التوثيق (`chunker_registry.md`، أدلة التأليف/المطابقة).
     - المباريات وتقارير الحتمية.
   - يوفر المشغلون وفرق SDK الملف الشخصي الجديد والطرح المخطط له مرة أخرى.

5. **الإهمال / الغروب**
   - مقترحات الملف الشخصي المتوفرة لديك والتي تحل محل نافذة النشر المزدوج
     (فترات السماح) وخطة الترقية تتضمن ہونا چاہیے۔
     ويتم تحديث دفتر أستاذ الهجرة.

6. ** تغييرات طارئة **
   - إزالة أو الإصلاحات العاجلة من خلال تصويت المجلس وموافقة الأغلبية.
   - وثيقة TWG لخطوات تخفيف المخاطر وتحديث سجل الحوادث.

## توقعات الأدوات- `sorafs_manifest_chunk_store` و `sorafs_manifest_stub` يعرض البطاقة:
  - فحص التسجيل لـ `--list-profiles`.
  - الترويج للملف الشخصي لكرتے وقت كتلة البيانات التعريفية الأساسية بنانے کے لیے `--promote-profile=<handle>`.
  - التقارير القياسية للدفق لـ `--json-out=-`، حتى سجلات المراجعة القابلة للتكرار قد تكون كذلك.
- `ensure_charter_compliance()` الثنائيات ذات الصلة لبدء التشغيل
  (`manifest_chunk_store`، `provider_advert_stub`). قد تفشل اختبارات CI مرة أخرى
  ميثاق الإدخالات الجديد يخالف قواعد اللعبة.

## حفظ السجلات

- جميع تقارير الحتمية کو `docs/source/sorafs/reports/` محفوظ کریں۔
- Chunker فيصلوں کا حوالہ دینے والی محضر المجلس
  `docs/source/sorafs/migration_ledger.md` موجود.
- تم تغيير التسجيل بعد `roadmap.md` و`status.md`.

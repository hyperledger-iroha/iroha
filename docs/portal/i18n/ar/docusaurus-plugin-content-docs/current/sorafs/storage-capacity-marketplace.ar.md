# سوق حالة التخزين في SoraFS (مسودة SF-2c)

يلتزم بند خارطة الطريق SF-2c سوقا محكوما حيث أعلن رسميًا عن التزامه، وتلقون أوامر النسخ المختلف، ويكسبون رسومًا تتوافق مع دقة المُسلّم. يحدد هذا المستند المشغلات المطلوبة للإصدار الأول ويقسمها إلى مسارات قابلة للتنفيذ.

##عجز

- التعبير عن مساحة أراضيها الجيدة (إجمالي البايتات، حدود لكل حارة، تاريخ الانتهاء) بصيغة قابلة للاستخدام يمكن استغلالها في استغلال SoraNet و Torii.
- توزيع الدبابيس عبر المزودين وفق السعة البائعة والحصة وقيود السياسة مع الحفاظ على التحكم حتمي.
- قياس تسليم النسخ (نجاح النسخ المتماثل، الجهوزية، إثبات السلامة) وتصدير التليميترية لتوزيع التأثيرات.
- توفير عمليات الإلغاء والنزاع حتى يمكن معاقبة المزودين غيرياء أو الأمنيين.

##مفاهيم النطاق| فكرة | الوصف | المخرج الاسم |
|---------|------------|--------------------|
| `CapacityDeclarationV1` | حمولة Norito تصفيه المزود الرئيسي، دعم ملف تعريف Chunker، GiB الملتزمة، حدود خاصة بـ حارة، تلميحات التسعير، التزام التوقيع المساحي، خرائط النهاية. | بيج + المتدرب في `sorafs_manifest::capacity`. |
| `ReplicationOrder` | تعليمات الدخول عن الـتورم تعيّن CID للـ Manifest إلى محدد واحد أو أكثر بما في ذلك مستوى التكرارات ومعايير SLA. | مخطط Norito مشترك مع Torii + واجهة العقد الذكي. |
| `CapacityLedger` | سجل on-chain/off-chain يتتبع الإعلانات السعة العضوية، أوامر النسخ المتراكمة، معايير الوزن، وتراكم الآثار. | وحدة العقد الذكي أو كعب الروتين لخدمة خارج السلسلة مع لقطة حتمي. |
| `MarketplacePolicy` | يجب أن يتم التوقيع على الاتفاقية الجديدة لـ الحصة والمتطلبات والرقابة التنظيمية. | بنية التكوين في `sorafs_manifest` + وثيقة الاتصال. |

#### تعريفات محددة (منفذة)- يلتقط معيار `CapacityDeclarationV1` صالح الموقع لكل ما يناسبه، بما في ذلك مقابض القطع، باحث القدرات، حدود اختيارية لكل حارة، تلميحات التسعير، نافذة الصلاحية، والبيانات الوصفية. ضمان ضمان التحقق من الحصة غير صفري، المقابض الممتدة، الأسماء المستعارة مزالة التكرار، حدود الممر ضمن إشعارات المعلنين، ومحاسبة GiB أحادية.【crates/sorafs_manifest/src/capacity.rs:28】
- يؤدي ربط `ReplicationOrderV1` إلى إظهار منافذ الدخول إلى الماكينات مع أهداف التكرار، وعتبات SLA، وبدلات لكل مهمة؛ إذا قام المدققون بمعالجة Chunker القياسية، والمقررين فريدين، وقيود الموعد النهائي قبل أن يقوم Torii أو سجل بابتلاع الأمر.[crates/sorafs_manifest/src/capacity.rs:301]
- ترخيص `CapacityTelemetryV1` عن اللقطات الحقبة (GiB بيعة مقابل الاستخدام، عدادات النسخ، نسبية uptime/PoR) التي تغذي توزيع التأثيرات. تحقق الحدود يبقي استخدامها ضمن الإعلانات والنسب ضمن 0-100%.【crates/sorafs_manifest/src/capacity.rs:476】
- توفر المساعدين المشتركين (`CapacityMetadataEntry` و`PricingScheduleV1` ومقررات الممرات/المهمة/SLA) التحقق من المفاتيح حتمي وتقارير الأخطاء يمكن إعادة استخدام أدوات CI والـ downstream.
-يُعرض `PinProviderRegistry` الآن لقطة على الشجرة عبر `/v2/sorafs/capacity/state`، جامعاً إعلانات المزودين وإدخالات رسوم دفتر الأستاذ خلف Norito JSON حتمي.[crates/iroha_torii/src/sorafs/registry.rs:17][crates/iroha_torii/src/sorafs/api.rs:64]- يغطي التحقق من صحة المقابض القياسية، وتأكيد التكرار، وحدود الحارة، تعيينات النسخ المتماثلة، وفحوصات النطاق التليميتري حتى قياس الإحالة فوراً في CI.[crates/sorafs_manifest/src/capacity.rs:792]
- تشغيل الأدوات: I18NI00000049تحولت X المواصفات بسرعة من البشر إلى Norito الحمولة للوصول، base64 blobs وملخصات JSON حتى يبدأون من تجهيزات المعالجة لـ `/v2/sorafs/capacity/declare` و`/v2/sorafs/capacity/telemetry` وأوامر النمس المتراكم مع التحقق محلية.[crates/sorafs_car/src/bin/sorafs_manifest_stub/capacity.rs:1] هناك تركيبات مرجعية في `fixtures/sorafs_manifest/replication_order/` (`order_v1.json`, `order_v1.to`) لا يمكن توليدها عبر `cargo run -p sorafs_car --bin sorafs_manifest_stub -- capacity replication-order`.

##متطلبات الاختبار و CI

- وحدات القدرة لكل مرجح للجديدة (`sorafs_manifest`).
- السيولة تكامل تحاكي: إعلان → أمر النسخة → القياس → الدفع.
- سير العمل في CI إعادة إنشاء إعلانات/تليمترية السعة باستثناء تزامن التواقيع (توسيع `ci/check_sorafs_fixtures.sh`).
- تحميل السيولة لواجهة التسجيل API (مثيل 10k و100k أمر).

## التليميترية ولوحات المعلومات

- لوحات لوحة القيادة:
  - السعة القانونية للمستعمل لكل.
  - backlog للأسباب البسيطة والتأخير.
  - واختار SLA (نسبة وقت التشغيل، معدل نجاح PoR).
  - تأثير المشاعر والغرامات لكل حقة.
- التنبيهات:
  - أقل من الحد الأدنى للسعة الملتزم بها.
  - أمر النسخة متعثرة من SLA.
  - إخفاقات خط الأنابيب المنتفخة.

## مخرجات التوثيق- دليل التشغيل لإعلان السعة وتجديد الالتزامات المستخدمة.
- دليل الصلاحيات للموافقة على الإعلانات وإصدارها وتشغيلها.
- مرجع API لنقاط نهاية السعة وتنسيق الطلبات المتماثلة.
- أسئلة شائعة للـ Marketplace متشوقة للمطورين.

## قائمة جاهزة GA

بند خارطة الطريق **SF-2c** يبوّب تحرير شامل للإنتاج الخاص بدليل البريد، معبرًا عنه ومعالجة الرسائل والالتحاق. استخدموا الأدوات أدناه للحفاظ على معايير تزامنية مع التنفيذ.

### المحاسبة الليلية وتسوية XOR
- لقطة صدّر حالة السعة وملف XOR دفتر الأستاذ لذاته، ثم شغّل:
  ```bash
  python3 scripts/telemetry/capacity_reconcile.py \
    --snapshot artifacts/sorafs/capacity/state_$(date +%F).json \
    --ledger artifacts/sorafs/capacity/ledger_$(date +%F).ndjson \
    --label nightly-capacity \
    --json-out artifacts/sorafs/capacity/reconcile_$(date +%F).json \
    --prom-out "${SORAFS_CAPACITY_RECONCILE_TEXTFILE:-artifacts/sorafs/capacity/reconcile.prom}"
  ```
  ينهي التنفيذ بكود غير صفري عند وجود تسويقات أو غرامات مفقودة/زائدة، ويصدر ملخصا بصيغة Prometheus textfile.
- تنبيه `SoraFSCapacityReconciliationMismatch` (ضمن `dashboards/alerts/sorafs_capacity_rules.yml`)
  يطلق عندما تشير مقاييس المصالحة إلى فجوات؛ معلومات اللوحات موجودة تحت
  `dashboards/grafana/sorafs_capacity_penalties.json`.
- أرشِف ملخص JSON و hashes داخل `docs/examples/sorafs_capacity_marketplace_validation/`
  توافق حزم الحكم.

### أدلة النزاع والتقطيع
- الخلافات عبر `sorafs_manifest_stub capacity dispute` (الاختبارات:
  `cargo test -p sorafs_car --test capacity_cli`) حتى الحمولات الدائمة الممتدة.
- شغّل `cargo test -p iroha_core -- capacity_dispute_replay_is_deterministic` ومنتجع
  (`record_capacity_telemetry_penalises_persistent_under_delivery`) لإثبات أن الخلافات والشرطات المائلة تُعاد حطمياً.
- اتبع `docs/source/sorafs/dispute_revocation_runbook.md` الأخبار والتصعيد؛ تنفيذ موافقات الإضراب في تقرير التحقق.### اختبارات دخان الدخول والخروج للمزود
- إعادة إنشاء المصنوعات اليدوية للإعلان/التليمترية باستخدام `sorafs_manifest_stub capacity ...` وأعد تشغيل السيولة قبل CLI (`cargo test -p sorafs_car --test capacity_cli -- capacity_declaration`).
- أرسل عبر Torii (`/v2/sorafs/capacity/declare`) ثم التقط `/v2/sorafs/capacity/state` مع لقطات Grafana. اتبع مسار الخروج في `docs/source/sorafs/capacity_onboarding_runbook.md`.
- أرشِف التحف الموقعة ومخرجات المصالحة داخل `docs/examples/sorafs_capacity_marketplace_validation/`.

## الاعتماديات والسلسلة

1. أو SF-2b (سياسة القبول) - يعتمد السوق على مقدمي الرعاية الصحية.
2. قم بتنفيذ المطلوب + السجل (هذا المستند) قبل تكامل Torii.
3. سوء خط أنابيب القياس قبل تفعيلها.
4. الخطوة الأخيرة: تفعيل مشاركة المساهمين المحكومين من الوالي بعد التحقق من بيانات القياس في التدريج.

يجب تتبع التقدم في خريطة الطريق مع الحالات إلى هذا المستند. تحديد خريطة الطريق بمجرد وصول كل الجزء الرئيسي (المخططات، مستوى التحكم، تكامل، القياس، الطوارئ) إلى الميزة كاملة.

# საკონტროლო სიის გამოქვეყნება

გამოიყენეთ ეს ჩამონათვალი, როდესაც განაახლებთ დეველოპერის პორტალს. ის უზრუნველყოფს, რომ
CI build, GitHub Pages-ის განლაგება და ხელით კვამლის ტესტები მოიცავს ყველა სექციას
სანამ გამოშვება ან საგზაო რუქის საეტაპო მიწები.

## 1. ადგილობრივი ვალიდაცია

- `npm run sync-openapi -- --version=current --latest` (დაამატე ერთი ან მეტი
  `--mirror=<label>` მიუთითებს, როდესაც Torii OpenAPI იცვლება გაყინული სნეპშოტისთვის).
- `npm run build` – კვლავ დაადასტურეთ `Build on Iroha with confidence` გმირის ასლი
  ჩნდება `build/index.html`-ში.
- `./docs/portal/scripts/preview_verify.sh --build-dir build` – გადაამოწმეთ
  checksum manifest (დაამატეთ `--descriptor`/`--archive` ჩამოტვირთული CI ტესტირებისას
  არტეფაქტები).
- `npm run serve` – უშვებს საკონტროლო ჯამით შემოსაზღვრულ წინასწარ გადახედვის დამხმარეს, რომელიც ამოწმებს
  მანიფესტი `docusaurus serve`-მდე დარეკვამდე, ასე რომ, მიმომხილველები არასოდეს ათვალიერებენ მას
  ხელმოუწერელი სნეპშოტი (`serve:verified` მეტსახელი რჩება აშკარა ზარებისთვის).
- გადაამოწმეთ ის ნიშანი, რომელიც თქვენ შეეხეთ `npm run start`-ის და პირდაპირი გადატვირთვის საშუალებით
  სერვერი.

## 2. გაიყვანეთ მოთხოვნის ჩეკები

- შეამოწმეთ, რომ `docs-portal-build` სამუშაო წარმატებით დასრულდა `.github/workflows/check-docs.yml`-ში.
- დაადასტურეთ `ci/check_docs_portal.sh` გაშვებული (CI ჟურნალები აჩვენებს გმირის კვამლის შემოწმებას).
- დარწმუნდით, რომ წინასწარი გადახედვისას ატვირთულია მანიფესტი (`build/checksums.sha256`) და
  წინასწარი გადახედვის დადასტურების სკრიპტი წარმატებით დასრულდა (CI ჟურნალები აჩვენებს
  `scripts/preview_verify.sh` გამომავალი).
- დაამატეთ გამოქვეყნებული გადახედვის URL GitHub გვერდების გარემოდან PR-ში
  აღწერა.

## 3. განყოფილების გაფორმება

| განყოფილება | მფლობელი | საკონტროლო სია |
|---------|-------|-----------|
| საწყისი გვერდი | DevRel | გმირის ასლი რენდერები, სწრაფი დაწყების ბარათები ბმული მოქმედ მარშრუტებთან, CTA ღილაკების გადაჭრა. |
| Norito | Norito WG | მიმოხილვა და დაწყების სახელმძღვანელოები მიუთითებენ უახლეს CLI დროშებსა და Norito სქემის დოკუმენტებზე. |
| SoraFS | შენახვის გუნდი | Quickstart სრულდება, მანიფესტის ანგარიშის ველები დოკუმენტირებულია, სიმულაციის ინსტრუქციების მიღება დამოწმებულია. |
| SDK გიდები | SDK იწვევს | Rust/Python/JS სახელმძღვანელოები აგროვებენ მიმდინარე მაგალითებს და აკავშირებენ ცოცხალ რეპოს. |
| მითითება | Docs/DevRel | ინდექსში ჩამოთვლილია უახლესი სპეციფიკაციები, Norito კოდეკის მინიშნება ემთხვევა `norito.md`. |
| არტეფაქტის გადახედვა | Docs/DevRel | `docs-portal-preview` არტეფაქტი მიმაგრებულია PR-ზე, კვამლის შემოწმების საშვი, ბმული გაზიარებულია მიმომხილველებთან. |
| უსაფრთხოება და სცადეთ ქვიშის ყუთი | Docs/DevRel · უსაფრთხოება | OAuth მოწყობილობის კოდის შესვლა კონფიგურირებულია (`DOCS_OAUTH_*`), `security-hardening.md` შესრულებულია საკონტროლო სია, CSP/სანდო ტიპების სათაურები დამოწმებულია `npm run build` ან `npm run probe:portal` მეშვეობით. |

მონიშნეთ თითოეული სტრიქონი, როგორც თქვენი PR მიმოხილვის ნაწილად, ან გაითვალისწინეთ შემდგომი დავალების სტატუსი
თვალყურის დევნება ზუსტი რჩება.

## 4. გამოშვების შენიშვნები

- ჩართეთ `https://docs.iroha.tech/` (ან გარემოს URL
  განლაგების სამუშაოდან) გამოშვების შენიშვნებში და სტატუსის განახლებებში.
- გამოიძახეთ ნებისმიერი ახალი ან შეცვლილი სექციები ცალსახად, რათა ქვედა დინების გუნდებმა იცოდნენ სად
  ხელახლა ჩაატარონ საკუთარი კვამლის ტესტები.

Илгээлт бүр нь `iroha_data_model::ministry::VolunteerBriefV1` (`1` хувилбар)-ын хэрэгжүүлсэн каноник схемийг дагаж мөрдөх ёстой бөгөөд ингэснээр:

3. Сэдвээс гадуурх эсвэл зурхайн материалаар оруулсан материалыг аудитын мөрөөс хасахгүйгээр дардаг.

## Хүргэлт

- **Схем ба баримт бичиг:** `VolunteerBriefV1` нь хуваалцсан өгөгдлийн загварт амьдардаг бөгөөд талбарын нарийвчилсан гарын авлага болон ачааллын жишээ нь `docs/source/ministry/volunteer_brief_template.md` болон `docs/examples/ministry/volunteer_brief_template.json` дээр байрладаг.
- **Баталгаажуулах хэрэгсэл:** `cargo xtask ministry-transparency volunteer-validate --input <file> [--json-output artifacts/ministry/volunteer-lint.json]` нь портал бүтээхэд ашигладаг ижил хулдаасуудыг ажиллуулдаг; `ingest --volunteer <file>` нь таныг бүхэл бүтэн багцыг хэвлүүлэхийг хүсвэл ашиглах боломжтой хэвээр байна.
- **Порталын нүүр царай:** Саналын хуудас бүр кодыг тайлсан дэмжлэг/сөрөг/контекст тоймтой холбогдож, JSON-г татаж авахгүйгээр иргэдийн санал бодлыг харуулдаг. Зохицуулах мета өгөгдлүүдийг товч танилцуулгын хажууд үзүүлснээр тангарагтны шүүгчид яагаад бичлэг хойшлогдсон эсвэл хасагдсан болохыг ойлгох болно.

Нэмэлт автоматжуулалт (жишээ нь, бөөнөөр импортлох API эсвэл өргөтгөсөн аналитик) шаардлагатай бол **MINFO-3 — Сайн дурынхны сонирхлын самбарын дэд бүтэц**-ийн замын зургийн зүйлийг харна уу.

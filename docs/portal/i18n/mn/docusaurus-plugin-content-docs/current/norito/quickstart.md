This walkthrough mirrors the workflow we expect developers to follow when learning
Norito and Kotodama for the first time: boot a deterministic single-peer network,
compile a contract, dry-run it locally, then send it through Torii with the
reference CLI.

The example contract writes a key/value pair to the caller's account so you can
verify the side effect immediately with `iroha_cli`.

## Prerequisites

- [Docker](https://docs.docker.com/engine/install/) with Compose V2 enabled (used
  to start the sample peer defined in `defaults/docker-compose.single.yml`).
- Rust toolchain (1.76+) for building the helper binaries if you do not download
  the published ones.
- `koto_compile`, `ivm_run`, and `iroha_cli` binaries. You can build them from the
  workspace checkout as shown below or download the matching release artifacts:

```sh
cargo install --locked --path crates/ivm --bin koto_compile --bin ivm_run
cargo install --locked --path crates/iroha_cli --bin iroha
```

> The binaries above are safe to install alongside the rest of the workspace.
> They never link to `serde`/`serde_json`; Norito codecs are enforced end-to-end.

## 1. Start a single-peer dev network

The repository includes a Docker Compose bundle generated by `kagami swarm`
(`defaults/docker-compose.single.yml`). It wires the default genesis, client
configuration, and health probes so Torii is reachable at `http://127.0.0.1:8080`.

```sh
docker compose -f defaults/docker-compose.single.yml up --build
```

Leave the container running (either in the foreground or detached). All
subsequent CLI calls target this peer via `defaults/client.toml`.

## 2. Author the contract

Create a working directory and save the minimal Kotodama example:

```sh
mkdir -p target/quickstart
cat > target/quickstart/hello.ko <<'KO'
// Writes a deterministic account detail for the transaction authority.

seiyaku Hello {
  // Optional initializer invoked during deployment.
  hajimari() {
    info("Hello from Kotodama");
  }

  // Public entrypoint that records a JSON marker on the caller.
  kotoage fn write_detail() {
    set_account_detail(
      authority(),
      name!("example"),
      json!{ hello: "world" }
    );
  }
}
KO
```

> Prefer keeping Kotodama sources in version control. Portal-hosted examples are
> also available under the [Norito examples gallery](./examples/) if you
> want a richer starting point.

## 3. Compile and dry-run with IVM

Compile the contract to IVM/Norito bytecode (`.to`) and execute it locally to
confirm that host syscalls succeed before touching the network:

```sh
koto_compile target/quickstart/hello.ko \
  --abi 1 \
  --max-cycles 0 \
  -o target/quickstart/hello.to

ivm_run target/quickstart/hello.to --args '{}'
```

The runner prints the `info("Hello from Kotodama")` log and performs the
`SET_ACCOUNT_DETAIL` syscall against the mocked host. If the optional `ivm_tool`
binary is available, `ivm_tool inspect target/quickstart/hello.to` displays the
ABI header, feature bits, and exported entrypoints.

## 4. Submit the bytecode via Torii

With the node still running, send the compiled bytecode to Torii using the CLI.
The default development identity is derived from the public key in
`defaults/client.toml`, so the account ID is
```
ih58...
```

Use the config file to supply Torii URL, chain ID, and signing key:

```sh
iroha --config defaults/client.toml \
  transaction ivm \
  --path target/quickstart/hello.to
```

The CLI encodes the transaction with Norito, signs it with the dev key, and
submits it to the running peer. Watch the Docker logs for the `set_account_detail`
syscall or monitor the CLI output for the committed transaction hash.

## 5. Verify the state change

Use the same CLI profile to fetch the account detail that the contract wrote:

```sh
iroha --config defaults/client.toml \
  account meta get \
  --id ih58... \
  --key example | jq .
```

You should see the Norito-backed JSON payload:

```json
{
  "hello": "world"
}
```

If the value is missing, confirm that the Docker compose service is still
running and that the transaction hash reported by `iroha` reached the `Committed`
state.

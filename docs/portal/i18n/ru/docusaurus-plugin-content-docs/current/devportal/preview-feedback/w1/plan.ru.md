## Предполетный чек-лист

> Совет: запустите `scripts/preview_wave_preflight.sh --tag preview-2025-04-12 --base-url https://preview.staging.sora --descriptor artifacts/preview-2025-04-12/descriptor.json --archive artifacts/preview-2025-04-12/docs-portal-preview.tar.zst --tryit-target https://tryit-proxy.staging.sora --output-json artifacts/preview-2025-04-12/preflight-summary.json`, чтобы автоматически выбрать шаги 1-5 (сборка, проверка контрольной суммы, проверка портала, проверка ссылок и обновление Try it proxy). Скрипт записывает JSON-лог, который можно приложить к выпуску трекера.

1. `npm run build` (с `DOCS_RELEASE_TAG=preview-2025-04-12`) для пересоздания `build/checksums.sha256` и `build/release.json`.
2. И18НИ00000033Х.
3. И18НИ00000034Х.
4. `DOCS_RELEASE_TAG=preview-2025-04-12 npm run check:links` и заархивировать `build/link-report.json` рядом с дескриптором.
5. `npm run manage:tryit-proxy -- update --target https://tryit-proxy.staging.sora` (или указать нужную цель через `--tryit-target`); закоммитить обновленный `.env.tryit-proxy` и сохранить `.bak` для отката.
6. Обновить проблему W1 путями журналов (дескриптор контрольной суммы, вывод пробы, изменение Try it proxy и снимки Grafana).

## Процесс обратной связи

1. Для каждого рецензента дублировать шаблон.
   [`docs/examples/docs_preview_feedback_form.md`](../../../../../examples/docs_preview_feedback_form.md),
   заполните метаданные и сохраните готовый протез в
   `artifacts/docs_preview/W1/preview-2025-04-12/feedback/<partner-id>/`.
2. Приглашение, телеметрические контрольно-пропускные пункты и открытие проблем в реальном времени.
   [`preview-feedback/w1/log.md`](./log.md), чтобы рецензенты по управлению могли воспроизвести видео
   полностью, не покидая репозиторий.
3. Когда придут экспортеры для проверки знаний или опросов, прикрепите их по пути протокола, указанному в логе,
   и связать с трекером выпуска.

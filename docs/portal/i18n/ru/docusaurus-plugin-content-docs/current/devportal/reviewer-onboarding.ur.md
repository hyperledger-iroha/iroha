## Впускной канал

1. درخواست دینے والے سے
   [`docs/examples/docs_preview_request_template.md`](../../../examples/docs_preview_request_template.md)
   فارم بھرنے کو کہیں (выпуск میں копировать/вставить کریں)۔ Что можно сделать, если вы хотите:
   Дескриптор GitHub, даты проверки, документация по безопасности и информация о ней.
2. Создайте трекер `docs-preview` (выпуск GitHub یا билет управления).
3. Воспользуйтесь преимуществами:
   - CLA / соглашение участника فائل پر موجود ہو (ссылка на партнерский контракт)۔
   - подтверждение приемлемого использования.
   - оценка риска مکمل ہو (مثال: партнеры-рецензенты کو Legal نے одобряют کیا ہو)۔
4. Утверждающий может выполнить подписание или выполнить отслеживание проблемы, а также запись об управлении изменениями.
   (Отзыв: `DOCS-SORA-Preview-####`)۔

## Обеспечение подготовки

1. **Артефакты شیئر کریں** — рабочий процесс CI یا SoraFS pin سے تازہ ترین дескриптор предварительного просмотра + архив
   (артефакт `docs-portal-preview`)۔ Есть несколько вариантов:

   ```bash
   ./docs/portal/scripts/preview_verify.sh \
     --build-dir build \
     --descriptor artifacts/preview-descriptor.json \
     --archive artifacts/preview-site.tar.gz
   ```2. **Контрольная сумма обеспечивает контроль** — возможность контроля контрольной суммы для контроля:

   ```bash
   DOCS_RELEASE_TAG=preview-<stamp> npm run --prefix docs/portal serve
   ```

   `scripts/serve-verified-preview.mjs` может использоваться повторно или использовать непроверенную сборку или использовать непроверенную сборку.

3. **GitHub رسائی دیں (اختیاری)** — можно просмотреть неопубликованные ветки, а также просмотреть обзор لئے
   `docs-preview` GitHub для получения членства в качестве члена

4. **Каналы поддержки شیئر کریں** — контакт по вызову (Matrix/Slack) при возникновении инцидента
   [`incident-runbooks`](./incident-runbooks.md) سے شیئر کریں۔

5. **Телеметрия + обратная связь** — возможность анонимной аналитики или анонимная аналитика
   (دیکھیں [`observability`](./observability.md))۔ دعوت میں دیے گئے форма обратной связи یا шаблон вопроса فراہم کریں
   Событие کو [`preview-feedback-log`](./preview-feedback-log) helper سے لاگ کریں تاکہ wave summary اپ ٹو ڈیٹ رہے۔

## ریویور چیک لسٹ

Если вы хотите узнать больше о том, как это сделать:

1. Проверьте загруженные артефакты (`preview_verify.sh`).
2. `npm run serve` (`serve:verified`) для защиты контрольной суммы.
3. Обеспечение безопасности и наблюдаемости.
4. Консоль OAuth/Try it для входа в систему по коду устройства или для использования производственных токенов.
5. согласованный трекер میں выводы درج کریں (проблема, общий документ یا فارم) اور انہیں предварительный просмотр тег выпуска سے tag کریں۔

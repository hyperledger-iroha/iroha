# پبلک پریویو دعوتی پلے بک

## پروگرام کے مقاصد

یہ پلے بک وضاحت کرتی ہے کہ ریویور آن بورڈنگ ورک فلو فعال ہونے کے بعد پبلک پریویو کیسے اعلان اور چلایا جائے۔
DOCS-SORA может быть использован в качестве дополнительного источника питания. قابل تصدیق артефакты, سیکیورٹی رہنمائی،
اور واضح Feedback راستہ شامل ہونا یقینی بنایا جاتا ہے۔

- **Удобство:** Удобство использования для сопровождающих, кураторское руководство по предварительному просмотру приемлемого использования. سائن کی ہے۔
- **Время:** размер волны по умолчанию <= 25 дней, 14 дней в окне доступа и 24 часа в сутки для реагирования на инциденты.

## لانچگیٹ چیک لسٹ

Вот как можно использовать следующие возможности:

1. Предварительный просмотр артефактов CI в режиме предварительного просмотра (`docs-portal-preview`,
   Манифест контрольной суммы, дескриптор, пакет SoraFS)۔
2. `npm run --prefix docs/portal serve` (контролируемый контрольной суммой) для тега ٹیسٹ کیا گیا ہو۔
3. ریویور آن بورڈنگ ٹکٹس утвердить ہوں اور пригласить волну سے لنک ہوں۔
4. Возможность наблюдения, возможность подтверждения инцидента.
   ([`security-hardening`](./security-hardening.md),
   [`observability`](./observability.md),
   [`incident-runbooks`](./incident-runbooks.md))۔
5. обратная связь с шаблоном проблемы تیار ہو (серьезность, этапы воспроизведения, снимки экрана, информация об окружающей среде и т. д.)
6. Поддержка документации/DevRel + управление и контроль

## دعوتی پیکیج

В качестве примера можно привести:

1. **Проверенные артефакты** — манифест/план SoraFS. Артефакт GitHub можно просмотреть.
   Дескриптор контрольной суммы и дескриптор манифеста проверка کمانڈ واضح طور پر لکھیں تاکہ
   Сайт сайта
2. **Инструкции по подаче** — предварительный просмотр с контролем контрольной суммы.

   ```bash
   DOCS_RELEASE_TAG=preview-<stamp> npm run --prefix docs/portal serve
   ```

3. **Напоминания о безопасности** — срок действия токенов истекает в ближайшее время, когда срок действия истекает.
   Происшествия فوراً رپورٹ کیے جائیں۔
4. **Канал обратной связи** – шаблон/форма сообщения, позволяющая узнать ожидаемое время ответа.
5. **Даты программы** – даты начала/окончания, часы работы, синхронизация, возможность обновления окна.

## پریویو host کو разоблачить کریں

جب تک onboarding مکمل نہ ہو اور изменить билет منظور نہ ہو تک предварительного просмотра хоста کو продвигать نہ کریں۔
Воспользуйтесь сквозными этапами сборки/публикации/проверки.
[предварительное руководство по воздействию на хост](./preview-host-exposure.md) دیکھیں۔

1. **Создать сборку:** выпуск метки штампа и детерминированных артефактов.

   ```bash
   cd docs/portal
   export DOCS_RELEASE_TAG="preview-$(date -u +%Y%m%dT%H%M%SZ)"
   npm ci
   npm run build
   ./scripts/sorafs-pin-release.sh \
     --alias docs-preview.sora \
     --alias-namespace docs \
     --alias-name preview \
     --pin-label docs-preview \
     --skip-submit
   node scripts/generate-preview-descriptor.mjs \
     --manifest artifacts/checksums.sha256 \
     --archive artifacts/sorafs/portal.tar.gz \
     --out artifacts/sorafs/preview-descriptor.json
   ```

   пин-скрипт `portal.car`, `portal.manifest.*`, `portal.pin.proposal.json`,
   `portal.dns-cutover.json` или `artifacts/sorafs/`, как показано на рисунке. Если вам нужна волна приглашения
   Если вы хотите прикрепить несколько битов, проверьте, пожалуйста, проверьте.

2. **Псевдоним предварительного просмотра для публикации:** Выберите `--skip-submit` для публикации
   (`TORII_URL`, `AUTHORITY`, `PRIVATE_KEY[_FILE]` — подтверждение псевдонима, выданное правительством, فراہم کریں)۔
   `docs-preview.sora` для привязки манифеста или пакета доказательств.
   `portal.manifest.submit.summary.json` или `portal.pin.report.json` как вариант3. **Проверка развертывания:** приглашает псевдоним بھیجنے سے پہلے разрешить контрольную сумму и совпадение тега ہونا.
   یقینی بنائیں۔

   ```bash
   npm run probe:portal -- \
     --base-url=https://docs-preview.sora.link \
     --expect-release="$DOCS_RELEASE_TAG"
   ```

   `npm run serve` (`scripts/serve-verified-preview.mjs`) — резервный вариант, удобный вариант
   Предварительный просмотр края میں مسئلہ ہو تو ریویورز لوکل کاپی چلا سکیں۔

## Отслеживание доступа и телеметрия

1. Получатель, временная метка приглашения, дата отзыва и предварительный просмотр журнала отзывов.
   (دیکھیں [`preview-feedback-log`](./preview-feedback-log)) تاکہ ہر wave ایک ہی след доказательств شیئر کرے:

   ```bash
   # artifacts/docs_portal_preview/feedback_log.json میں نیا invite event شامل کریں
   npm run --prefix docs/portal preview:log -- \
     --wave preview-20250303 \
     --recipient alice@example.com \
     --event invite-sent \
     --notes "wave-01 seed"
   ```

   Поддерживаемые события: `invite-sent`, `acknowledged`, `feedback-submitted`,
   `issue-opened`, или `access-revoked`۔ log ڈیفالٹ طور پر
   `artifacts/docs_portal_preview/feedback_log.json` میں موجود ہے؛ اسے пригласить волну ٹکٹ کے ساتھ
   формы согласия سمیت прикрепить کریں۔ close-out نوٹ سے پہلے помощник по составлению сводки استعمال کریں تاکہ
   Сведение, поддающееся аудиту, например:

   ```bash
   npm run --prefix docs/portal preview:summary -- --summary-json \
     > artifacts/docs_portal_preview/preview-20250303-summary.json
   ```

   Сводная волна JSON, приглашения, получатели, обратная связь, информация о событии.
   временная метка کو перечислить کرتا ہے۔ помощник
   [`scripts/preview-feedback-log.mjs`](../../scripts/preview-feedback-log.mjs)
   В этом разделе представлены рабочие процессы, связанные с CI и CI. резюме
   [`docs/examples/docs_preview_feedback_digest.md`](../../../examples/docs_preview_feedback_digest.md)
   Шаблон дайджеста استعمال کریں۔
2. Панели телеметрии с волной и тегом `DOCS_RELEASE_TAG` с тегом کریکہ
   шипы کو пригласить когорты سے коррелировать کیا جا سکے۔
3. разверните среду предварительного просмотра `npm run probe:portal -- --expect-release=<tag>` и создайте среду предварительного просмотра.
   درست выпускать метаданные, рекламировать کرے۔
4. Наличие инцидента или шаблон Runbook, захват данных или когорта, ссылка на ссылку.

## Обратная связь اور закрыть

1. Обратная связь, общий документ или доска вопросов элементы کو `docs-preview/<wave>` سے tag کریں تاکہ
   владельцы дорожной карты انہیں آسانی سے query کر سکیں۔
2. Регистратор предварительного просмотра, сводный вывод, волновой отчет, выбор когорты, `status.md`, сводный отчет,
   (участники, полученные результаты, запланированные исправления) اور اگر DOCS-SORA веха بدلا ہو تو `roadmap.md` ڈیٹ کریں۔
3. [`reviewer-onboarding`](./reviewer-onboarding.md) Следующие шаги по отключению следующие: отозвать доступ.
   Архив запросов. Участники.
4. Обновите волну, обновите артефакты, проверьте контрольную сумму, выберите шаблон приглашения, выберите дату и время.

اس playbook کو مسلسل لاگو کرنے سے Preview پروگرام Auditable رہتا ہے اور Docs/DevRel کو دعوتیں
Если вы хотите повторять действия, которые вам нужны, вы можете использовать GA, чтобы сделать это.

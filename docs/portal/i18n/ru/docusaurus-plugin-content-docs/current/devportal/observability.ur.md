# پورٹل آبزرویبیلٹی اور اینالٹکس

DOCS-SORA Предварительный просмотр сборки, аналитика, синтетические зонды, автоматизация неработающих ссылок, а также сборка неработающих ссылок.
یہ نوٹ وہ сантехника بیان کرتا ہے جو اب پورٹل کے ساتھ корабль ہوتی ہے تاکہ операторы мониторинга جوڑ کیں
بغیر данные посетителей لیک کئے۔

## Тегирование выпуска

- `DOCS_RELEASE_TAG=<identifier>` سیٹ کریں (резервный `GIT_COMMIT` یا `dev`) при сборке ہو۔
  ویلیو `<meta name="sora-release">` میں inject ہوتی ہے تاکہ зонды اور развертывания информационных панелей کو کر کیں۔
- `npm run build` `build/release.json` испускает کرتا ہے (جسے `scripts/write-checksums.mjs` لکھتا ہے)
  Тег, временная метка и необязательный `DOCS_RELEASE_SOURCE` для описания کرتا ہے۔ یہی Предварительный просмотр артефактов в комплекте
  ہوتی ہے اور проверка ссылок رپورٹ میں ссылаться ہوتی ہے۔

## Аналитика, сохраняющая конфиденциальность

- `DOCS_ANALYTICS_ENDPOINT=<https://collector.example/ingest>` настроить легкий трекер и включить его.
  полезные нагрузки `{ event, path, locale, release, ts }` или IP-метаданные, данные реферера и IP-метаданные.
  `navigator.sendBeacon` Добавлен блок навигации в блоке навигации ہو۔.
- `DOCS_ANALYTICS_SAMPLE_RATE` (0-1) обеспечивает контроль выборки. трекер последнего отправленного пути ذخیرہ کرتا ہے
  Навигация по дубликату событий и выдача дубликатов событий
- реализация `src/components/AnalyticsTracker.jsx` میں ہے اور `src/theme/Root.js` کے ذریعے глобальное монтирование ہوتی ہے۔

## Синтетические зонды

- `npm run probe:portal` для маршрутов и GET-запросов.
  (`/`, `/norito/overview`, `/reference/torii-swagger`, وغیرہ) Пожалуйста, проверьте правильность установки.
  Метатег `sora-release` `--expect-release` (یا `DOCS_RELEASE_TAG`) и соответствие کرتا ہے۔ Название:

```bash
PORTAL_BASE_URL="https://docs.staging.sora" \
DOCS_RELEASE_TAG="preview-42" \
npm run probe:portal -- --expect-release=preview-42
```

Неудачи ہر путь کے حساب سے отчет ہوتے ہیں، جس سے CD-ворота کرنا آسان ہو جاتا ہے۔

## Автоматизация неработающих ссылок

- `npm run check:links` `build/sitemap.xml` сканирование Можно выполнить запись, локальный файл или карту, а затем выполнить сканирование.
  (`index.html` резервные варианты, а также резервные копии) и `build/link-report.json`, которые можно использовать для выпуска метаданных, итогов, сбоев и т. д.
  Для `checksums.sha256` можно использовать отпечаток пальца SHA-256 (для `manifest.id` можно открыть экспозицию) رپورٹ
  Манифест артефакта سے جوڑی جا سکے۔
- сценарий ненулевой پر выход کرتا ہے جب کوئی صفحہ отсутствует ہو، اس لئے CI پرانی یا сломанные маршруты پر выпускает روک سکتا ہے۔
  отчеты, пути-кандидаты, процитируйте или попробуйте найти варианты, регрессии маршрутизации и дерево документов, а также трассировку и т. д. ہے۔

## Grafana панель мониторинга اور оповещения- Плата `dashboards/grafana/docs_portal.json` Grafana **Публикация портала документации** опубликовать کرتا ہے۔
  На панели управления есть:
  - *Отказы шлюза (5 м)* `torii_sorafs_gateway_refusals_total` или `profile`/`reason` Область действия, которую вы можете использовать.
    Политика SREs خراب подталкивает к обнаружению сбоев токена.
  - *Результаты обновления кэша псевдонимов* اور *Alias Proof Age p90* `torii_sorafs_alias_cache_*` کو track کرتے ہیں
    DNS обрезан, есть свежие доказательства.
  - *Количество манифестов реестра контактов* اور *Количество активных псевдонимов* stat pin-registry backlog اور общее количество псевдонимов, отражающих کرتے ہیں
    Управление, выпуск, аудит, управление
  - *Срок действия TLS шлюза (часы)* Выделите значение шлюза публикации истечения срока действия сертификата TLS
    (порог оповещения 72 часа)۔
  - *Результаты SLA репликации* اور *Журнал репликации* `torii_sorafs_replication_*` телеметрия پر نظر رکھتے ہیں
    تاکہ опубликовать کے بعد تمام GA bar پر ہوں۔
- встроенные переменные шаблона (`profile`, `reason`).
  یا تمام шлюзы میں шипы исследуют کئے جا سکیں۔
- Панели мониторинга маршрутизации PagerDuty с доказательствами и уведомлениями: оповещения
  `DocsPortal/GatewayRefusals`, `DocsPortal/AliasCache`, اور `DocsPortal/TLSExpiry` تب fire کرتے ہیں جب متعلقہ series
  пороги пересеклись کرے۔ оповещение Runbook کو اسی صفحے سے link کریں تاکہ инженеры по вызову точные запросы Prometheus воспроизведение کر سکیں۔

## Собираем это вместе

1. `npm run build` — установка переменных среды выпуска/аналитики на этапе после сборки.
   `checksums.sha256`, `release.json`, `link-report.json` излучают کرنے دیں۔
2. Предварительный просмотр имени хоста или `npm run probe:portal`, или `--expect-release`, или тега, и проводного соединения.
   Контрольный список публикации کے لئے stdout محفوظ کریں۔
3. `npm run check:links` может привести к повреждению записей карты сайта или сбою при создании отчета JSON.
   предварительный просмотр артефактов کے ساتھ архив کریں۔ Последний отчет CI کو `artifacts/docs_portal/link-report.json` میں drop کرتا ہے
   Журналы сборки управления. Пакет доказательств. Загрузите файлы.
4. Конечная точка аналитики с сборщиком данных, сохраняющим конфиденциальность (Pluusible, самостоятельный прием OTEL) и пересылка вперед
   اوریکہ релиз کے لئے частота дискретизации документ کریں تاکہ количество панелей мониторинга کو درست интерпретация کریں۔
5. CI-интерфейс предварительного просмотра/развертывания рабочих процессов.
   (`.github/workflows/docs-portal-preview.yml`,
   `.github/workflows/docs-portal-deploy.yml`) ، اس لئے локальные пробные прогоны میں صرف секреты поведения, специфичные для کرنا ہوتا ہے۔

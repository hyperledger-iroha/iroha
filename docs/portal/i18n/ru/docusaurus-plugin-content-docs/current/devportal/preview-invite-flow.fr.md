# Предварительный просмотр приглашения Flux

## Объектив

В элементе дорожной карты **DOCS-SORA** указана регистрация репрезентаторов и программа приглашений для предварительного просмотра общедоступных пользователей перед бета-тестированием. На этой странице есть комментарии, которые содержат расплывчатые приглашения, какие артефакты делаются и живут до отправки приглашений и комментируют, подтверждающие, что поток можно проверить. Использование:

- [`devportal/reviewer-onboarding`](./reviewer-onboarding.md) для отражения жеста.
- [`devportal/preview-integrity-plan`](./preview-integrity-plan.md) для гарантий контрольной суммы.
- [`devportal/observability`](./observability.md) для экспорта телеметрии и перехватов оповещений.

## Контрольный список перед полетом

Terminez ces action **avant** de planifier des приглашений для неясных:

1. **Доступные артефакты CI**
   - Dernier `docs-portal-preview` + дескриптор заряда номинала `.github/workflows/docs-portal-preview.yml`.
   - Контакт SoraFS, примечание в `docs/portal/docs/devportal/deploy-guide.md` (присутствует дескриптор переключения).
2. **Приложение контрольной суммы**
   - `docs/portal/scripts/serve-verified-preview.mjs` вызывается через `npm run serve`.
   - Инструкции `scripts/preview_verify.sh` для испытуемых на macOS + Linux.
3. **Базовая телеметрия**
   - `dashboards/grafana/docs_portal.json` montre un trafic Попробуйте сказать и предупредить `docs.preview.integrity` est au vert.
   - Последнее приложение `docs/portal/docs/devportal/observability.md` создано с залогом Grafana.
4. **Управление артефактами**
   - Проблема с трекером приглашений (проблема неясная).
   - Копия шаблона регистрации отражателей (voir [`docs/examples/docs_preview_request_template.md`](../../../examples/docs_preview_request_template.md)).
   - Для юридических утверждений и SRE требуются атташе по вопросу.

Зарегистрируйте предполетное достижение в трекере приглашений перед отправкой моего электронного письма.

## Этапы потока1. **Отбор кандидатов**
   - Укажите список внимания или файловых партнеров.
   - Убедитесь, что вы выбрали полный шаблон запроса.
2. **Утверждение доступа**
   - Назначьте проверяющего трекер приглашений.
   - Проверка предварительных условий (CLA/контракт, допустимое использование, краткая безопасность).
3. **Отправка приглашений**
   - Дополните заполнители [`docs/examples/docs_preview_invite_template.md`](../../../examples/docs_preview_invite_template.md) (`<preview_tag>`, `<request_ticket>`, контакты).
   - Объединение дескриптора + хэша архива, промежуточного URL-адреса. Попробуйте и поддержите файлы.
   - Получите окончательную электронную почту (или стенограмму Matrix/Slack) в выпуске.
4. **Новая адаптация**
   - В течение дня трекер приглашений имеет `invite_sent_at`, `expected_exit_at` и статус (`pending`, `active`, `complete`, `revoked`).
   - Lier la requiree d'entree du relecteur для проверки.
5. **Наблюдение за телеметрией**
   - Surveiller `docs.preview.session_active` и оповещения `TryItProxyErrors`.
   - Произведите инцидент, если телеметрия устройства базовой линии и регистрация результата приведет к входу в приглашение.
6. **Сбор отзывов и сортировка**
   - Закройте приглашения или отзывы, полученные или `expected_exit_at`, которые вы получили.
   - Встречайте расплывчатые вопросы с резюме суда (статистика, инциденты, продолжающиеся действия) перед прохожим или последующей когортой.

После того, как партия будет приготовлена, вы сможете переварить ее на машине. Используйте JSON, чтобы ответить на неопределенный вопрос, который позволяет отражателям управлять подтверждением счетов приглашений без перезагрузки всего журнала.

Присоединяйтесь к списку предварительных предложений `status.md`, чтобы добавить неопределенности к тому, что дорожная карта входа может помочь вам ускорить день.

## Критерии отката и паузы

Отметьте паузу в потоках приглашений (и уведомите об управлении) для выживших людей:

- Инцидентный прокси. Попробуйте, если потребуется откат (`npm run manage:tryit-proxy`).
- Оповещения об усталости: >3 страницы оповещений для предварительного просмотра конечных точек, доступные только в течение 7 дней.
- Соответствующая электронная карта: приглашение посланника без подписи или без регистрации по шаблону запроса.
- Риск нарушения целостности: обнаружено несоответствие контрольной суммы параметру `scripts/preview_verify.sh`.

Повторите уникальное действие после того, как документируете исправление в трекере приглашений и подтверждаете, что телеметрия приборной панели стабильна в течение 48 часов.

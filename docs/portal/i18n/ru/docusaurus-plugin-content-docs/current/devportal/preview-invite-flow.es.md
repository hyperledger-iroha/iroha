# Открытие приглашений для предварительного просмотра

## Предложение

Элемент дорожной карты **DOCS-SORA** идентифицирует регистрацию ревизоров и программу приглашений для публичного предварительного просмотра как финальных блокировщиков перед тем, как портал станет бета-версией. На этой странице описывается, как после каждого приглашения, какие артефакты должны быть отправлены перед приглашением и как демонстрация того, что поток можно проверить. Усала юнто против:

- [`devportal/reviewer-onboarding`](./reviewer-onboarding.md) для управления редактором.
- [`devportal/preview-integrity-plan`](./preview-integrity-plan.md) для гарантии контрольной суммы.
- [`devportal/observability`](./observability.md) для экспорта телеметрии и перехватов оповещений.

## Контрольный список предполетной подготовки

Полный список **предварительных** программ приглашений для одной:

1. **Доступные артефакты CI**
   - Последний `docs-portal-preview` + дескриптор груза для `.github/workflows/docs-portal-preview.yml`.
   - PIN-код SoraFS аннотирован на `docs/portal/docs/devportal/deploy-guide.md` (дескриптор переключения присутствует).
2. **Контрольная сумма**
   - `docs/portal/scripts/serve-verified-preview.mjs` вызывается через `npm run serve`.
   - Инструкции по пробному использованию `scripts/preview_verify.sh` для macOS + Linux.
3. **Базовый уровень телеметрии**
   - `dashboards/grafana/docs_portal.json` muestra trafico Попробуйте это полезно и оповещение `docs.preview.integrity` esta en verde.
   - Последнее приложение `docs/portal/docs/devportal/observability.md` актуализировано с вставками Grafana.
4. **Артефакты правительства**
   - Проблема со списком трекера приглашений (проблема была раньше).
   - План регистрации копий копий (версия [`docs/examples/docs_preview_request_template.md`](../../../examples/docs_preview_request_template.md)).
   - Юридические подтверждения и SRE требуют дополнительных мер по данному вопросу.

Зарегистрируйте завершение предполетной подготовки на трекере приглашений, прежде чем отправить сообщение.## Пасос-дель-Флюхо

1. **Отбор кандидатов**
   - Дополнительный опыт поиска или кола партнеров.
   - Убедитесь, что каждый кандидат получит полную планировку запроса.
2. **Доступ к программе**
   - Назначьте апробатора для трекера приглашений.
   - Проверка предварительных требований (CLA/contrato, приемлемое использование, краткое описание безопасности).
3. **Отправить приглашения**
   - Заполните заполнители для [`docs/examples/docs_preview_invite_template.md`](../../../examples/docs_preview_invite_template.md) (`<preview_tag>`, `<request_ticket>`, контакты).
   - Дополнение к дескриптору + хэш архива, URL-адрес промежуточной версии Try it и поддерживаемые каналы.
   - Проверьте окончательное электронное письмо (расшифровку стенограммы Matrix/Slack) в выпуске.
4. **Внедрение Rastrear**
   - Актуализировать трекер приглашений с `invite_sent_at`, `expected_exit_at`, и состоянием (`pending`, `active`, `complete`, `revoked`).
   - Сделайте запрос на вступление ревизора для проверки.
5. **Мониторная телеметрия**
   - Бдительность `docs.preview.session_active` и оповещение как `TryItProxyErrors`.
   - Отмените происшествие, если телеметрия прошла по базовой линии и зарегистрировала результат соединения по приглашению.
6. **Запомните обратную связь и запишите**
   - Приглашайте приглашения, когда вы получите обратную связь от `expected_exit_at`.
   - Актуализируйте выпуск ола с возобновленным кортом (халлазгос, инциденты, последующие действия) до паса аль-сигуйенте когорте.

## Критерии отката и паузы

Нажмите на приглашение (и уведомите губернатора), когда произойдет какое-либо событие:

- Случай с прокси. Попробуйте выполнить откат, если требуется (`npm run manage:tryit-proxy`).
- Обновление предупреждений: >3 страниц оповещений для отдельных конечных точек предварительного просмотра в течение 7 дней.
- Получение вознаграждения: приглашение отправляется без окончательных фирм или без регистратора по запросу.
- Процесс интеграции: обнаружено несоответствие контрольной суммы `scripts/preview_verify.sh`.

Повторите только после того, как документируете исправление в трекере приглашений и подтверждаете, что панель телеметрии установлена ​​в течение всех 48 часов.

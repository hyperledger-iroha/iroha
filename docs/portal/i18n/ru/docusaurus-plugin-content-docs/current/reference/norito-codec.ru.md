# Справочный кодек Norito

Norito — стандартный слой сериализации Iroha. Каждое беспроводное сообщение, полезная нагрузка на диске и межкомпонентный API используют Norito, чтобы узлы соглашались на отдельные байты даже при разном оборудовании. На этой странице резюмируются основные элементы и приведены полные характеристики `norito.md`.

## Базовая компоновка

| Компонент | Назначение | Источник |
| --- | --- | --- |
| **Заголовок** | Обрамляет полезные нагрузки хэшем магии/версии/схемы, CRC64, четвертым и тегом сжатия; v1 требует `VERSION_MINOR = 0x00` и, наконец, флаги заголовка соответственно воздействию маски (по умолчанию `0x00`). | `norito::header` — см. `norito.md` ("Заголовок и флаги", корень репозитория) |
| **Пустая полезная нагрузка** | Определенное кодирование результатов для хеширования/сравнения. Проводной транспорт всегда использует заголовок; голые байты — только внутренние. | `norito::codec::{Encode, Decode}` |
| **Сжатие** | Опциональный Zstd (и экспериментальное ускорение GPU), выбираемый байтом сжатия в заголовке. | `norito.md`, «Согласование сжатия» |

Реестр флагов макета (packed-struct, packaged-seq, field bitset, компактные длины) находится в `norito::header::flags`. V1 по умолчанию использует флаги `0x00`, но принимает явные флаги в пределах воздействия маски; неизвестные биты отклоняются. `norito::header::Flags` сохраняется для внутренней проверки и последующих последствий.

## Поддержка получения

`norito_derive` представляет производные `Encode`, `Decode`, `IntoSchema` и помощники JSON. Ключевое соглашение:

- Получение генерируемых как AoS, так и упакованных путей кода; v1 по умолчанию используется макет AoS (флаги `0x00`), если флаги заголовка не включают упакованные варианты. Реализация находится в `crates/norito_derive/src/derive_struct.rs`.
- Функции, влияющие на макет (`packed-struct`, `packed-seq`, `compact-len`), включают возможность подписки через флаги заголовка и должны кодироваться/декодироваться согласно согласованию между узлами.
- Помощники JSON (`norito::json`) предоставляют определенный JSON на основе Norito для общедоступных API. Используйте `norito::json::{to_json_pretty, from_json}` — никогда `serde_json`.

## Мультикодек и таблицы идентификаторов

Norito сохраняет назначение мультикодека в `norito::multicodec`. Референсная таблица (хеши, типы ключей, полезная нагрузка дескрипторов) переходит в `multicodec.md` в корневую репозиторию. При добавлении нового удостоверения:

1. Обновите `norito::multicodec::registry`.
2. Расширьте таблицу в `multicodec.md`.
3. Перегенерируйте последующие привязки (Python/Java), если они используют эту карту.

Используйте этот контрольный список при каждом обновлении портала разработчика. Он гарантирует, что
CI‑сборка, деплой GitHub Pages и ручные дымовые тесты раскрывают все разделы перед релизом
или автомобильной дорожной карты.

## 1. Локальная проверка

- `npm run sync-openapi -- --version=current --latest` (добавьте один или несколько
  флагов `--mirror=<label>`, если Torii OpenAPI меняется для замороженного snapshot’а).
- `npm run build` — убедитесь, что слоган «Опирайтесь на Iroha с уверенностью» всё ещё
  отображается в `build/index.html`.
- `./docs/portal/scripts/preview_verify.sh --build-dir build` — проверьте манифест
  checksum'ов (добавьте `--descriptor`/`--archive` при тестировании скачанных CI‑артефактов).
- `npm run serve` — запускает helper‑режим с проверкой checksum’ов, который валидирует
  манифест перед вызовом `docusaurus serve`, чтобы рецензенты никогда не смотрели на
  незарегистрированный снимок (псевдоним `serve:verified` остается для явных вызовов).
- Просмотрите изменённые уценочные файлы через `npm run start` и dev‑сервер с живой перезагрузкой.

##2. Проверки pull‑request’а

- Убедиться, что задание `docs-portal-build` прошло успешно.
  `.github/workflows/check-docs.yml`.
- Проверьте, что был запущен `ci/check_docs_portal.sh` (в логах CI отображается герой‑проверка).
- Убедитесь, что предварительный просмотр рабочего процесса загрузил манифест (`build/checksums.sha256`) и что
  Предварительный просмотр скрипта проверки завершился успешно (логики CI содержат вывод
  `scripts/preview_verify.sh`).
- добавить опубликованный URL-адрес предварительного просмотра из окружения GitHub Pages в описании PR.

| Раздел | Владелец | Чек‑лист |

| Главная | ДевРел | Геро‑копирайт отображается; Краткое руководство по карточкам показывает действительные маршруты; СТА‑кнопки работают. |

| SoraFS | Команда хранения | Быстрый старт до конца, поля отчёта по манифесту задокументированы, инструкция по симуляции fetch проверена. |
| SDK‑гайды | Лиды СДК | Rust/Python/JS‑гайды собирают актуальные примеры и ссылаются на живые репозитории. |

отличный.

- Включите `https://docs.iroha.tech/` (или URL-адрес окружения задания задания) в примечаниях к выпуску.
  и статусные апдейты.

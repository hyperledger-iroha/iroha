### 4.2 Регистрация, аукцион и утверждение цен1. **Предполетная проверка:** Регистратор `SuffixPolicyV1` или определенная ценовая категория, условия использования.
   Окна отсрочки/искупления План действий и дорожная карта
   Уровень 3/4/5/6-9/10+ (базовый уровень + коэффициенты суффикса) Синхронизировано
2. **Аукционы с закрытыми ставками:** Премиум-пулы с циклом фиксации 72 часа/24 часа.
   `sns governance auction commit` / `... reveal` کے ذریعے چلائیں۔ совершить فہرست
   (Хеш-коды) `artifacts/sns/auctions/<name>/commit.json`
   аудиторы случайность проверяют کر سکیں۔
3. **Подтверждение платежа:** Регистраторы `PaymentProofV1` при разделении казначейства.
   подтвердите کرتے ہیں (70% казначейство / 30% управляющий, выделение рефералов 72 часов, резкие скачки ошибок регистратора, дрейф ARPU) в результате инцидента
  триггер کریں۔

### 4.5 Распространение резольвера через шлюз

1. **Перехват событий:** поток событий преобразователя событий реестра (`tools/soradns-resolver` SSE)
   излучать ہوتا ہے۔ Подпишитесь на операции по разрешению проблем с прозрачностью
   (`scripts/telemetry/run_soradns_transparency_tail.sh`) Различные различия.
2. **Обновление шаблона GAR:** Шлюзы `canonical_gateway_suffix()` см. в разделе шаблоны GAR.
   اپڈیٹ کرنے ہوں گے اور `host_pattern` список کو دوبارہ знак کرنا ہوگا۔ Различия
   `artifacts/sns/gar/<date>.patch` میں رکھیں۔
3. **Публикация файла зоны:** `roadmap.md` میں بیان کردہ скелет файла зоны (имя, ttl, cid, доказательство)
   Нажмите кнопку Torii/SoraFS для push-уведомления. Norito JSON
   `artifacts/sns/zonefiles/<name>/<version>.json` Архив файлов
4. **Проверка прозрачности:** `promtool test rules dashboards/alerts/tests/soradns_transparency_rules.test.yml`
   چلائیں تاکہ предупреждает зеленый رہیں۔ Prometheus текстовый вывод — еженедельный отчет о прозрачности.
5. **Аудит шлюза:** Примеры заголовков `Sora-*` (политика кэширования, CSP, дайджест GAR).
   Журнал управления можно прикрепить к шлюзу или шлюзу

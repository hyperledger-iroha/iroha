# Примечание о переходе Nexus

В этом реестре запланирована работа по **Фазе B — Nexus Transition Foundations**, которая должна завершить контрольный список многополосного планирования. Дополняют вводные данные о вехах в `roadmap.md` и содержат справочные материалы по доказательствам для B1-B4 в одиночку, чтобы управлять, SRE и лидерами SDK, сравнивающими ошибочные действия.

## Альканс и Каденсия

- Cubre las Audirias Routed-trace и Los Guardrails de Telemetria (B1/B2), el Conjunto de deltas de Configuration Aprobado por Gobernanza (B3) и Los Seguimientos del Ensayo de Lanzamiento Multi-Lanzamiento (B4).
- Reemplaza la nota temporal de cadencia que antes vivia aqui; После аудитории за первый квартал 2026 года подробный отчет находится в `docs/source/nexus_routed_trace_audit_report_2026q1.md`, в то время, когда эта страница содержит оперативный календарь и реестр смягчения последствий.
- Актуализируйте таблицы после каждой проложенной трассировки, проголосуйте за управление или запустите ланзамьенто. Когда артефакты появляются, отражение нового использования этой страницы для задних документов (статус, информационные панели, порталы SDK) может быть включено в уже установленный.

## Цикл просмотра дорожек во время выполнения

- Плоскости циклического просмотра дорожек во время выполнения теперь действительны привязок пространства данных и прерваны, когда происходит согласование Куры/альмаценамиенто по нивелам, и каталог без камней. Помощники перенаправляют полосы в кэш для удаленных дорожек, мод, в котором синтезируется реестр слияний, и доказательства повторного использования не устарели.
- Приложение планирует перемещение помощников конфигурации/жизненного цикла Nexus (`State::apply_lane_lifecycle`, `Queue::apply_lane_lifecycle`) для объединения/отключения полос без повторного использования; маршрутизация, снимки TEU и реестры манифестов автоматически перезагружаются по плану выхода.
- Рекомендации по эксплуатации: когда план проваливается, проверяются пространства данных или корни хранилища, которые не могут быть сложены (многоуровневый холодный корень/каталоги Kura por Lane). Corrige las rutas base y reintenta; Лос-плоскости выходят из строя, повторно излучая разницу телеметрии полосы движения/пространства данных, чтобы панели мониторинга отражали новую топологию.

## Телеметрия NPoS и доказательство противодавленияРетро-энсайо ланзамента фазы B позволяет захватывать телеметрические данные, определяющие, что происходит с кардиостимулятором NPoS и сплетнями, которые поддерживаются в пределах пределов противодавления. Интеграция в `integration_tests/tests/sumeragi_npos_performance.rs` отображает эти сценарии и выдает резюме JSON (`sumeragi_baseline_summary::<scenario>::...`) при объединении новых показателей. Местные сведения о:

```bash
cargo test -p integration_tests sumeragi_npos_performance -- --nocapture
```

Настройте `SUMERAGI_NPOS_STRESS_PEERS`, `SUMERAGI_NPOS_STRESS_COLLECTORS_K` или `SUMERAGI_NPOS_STRESS_REDUNDANT_SEND_R` для исследования топологий главных объектов; утраченные значения из-за дефекта отображаются в папке recolectores 1 s/`k=3`, используемой в B4.

| Сценарий/тест | Кобертура | Телеметрия клава |
| --- | --- | --- |
| `npos_baseline_1s_k3_captures_metrics` | Блокируйте 12 рондов с блокировкой времени отправки для регистрации конвертов латентной почты EMA, глубин колы и датчиков избыточной отправки перед сериализацией пакета доказательств. | `sumeragi_phase_latency_ema_ms`, `sumeragi_collectors_k`, `sumeragi_redundant_send_r`, `sumeragi_bg_post_queue_depth*`. |
| `npos_queue_backpressure_triggers_metrics` | Включите колу транзакций, чтобы гарантировать, что отсрочка доступа будет активирована детерминированная форма и что кола экспортирует контадоры емкости/насыщения. | `sumeragi_tx_queue_depth`, `sumeragi_tx_queue_capacity`, `sumeragi_tx_queue_saturated`, `sumeragi_pacemaker_backpressure_deferrals_total`, `sumeragi_rbc_backpressure_deferrals_total`. |
| `npos_pacemaker_jitter_within_band` | Muestrea эль-дрожание кардиостимулятора и тайм-ауты обзора показывают, что диапазон +/- 125 промилле является приложением. | `sumeragi_pacemaker_jitter_ms`, `sumeragi_pacemaker_view_timeout_target_ms`, `sumeragi_pacemaker_jitter_frac_permille`. |
| `npos_rbc_store_backpressure_records_metrics` | Полезные нагрузки RBC имеют большие ограничения по мягкому/жесткому хранению, чтобы обеспечить сеансы и контакты байтов в субэн, ретроспективе и стабилизации без сохранения хранилища. | `sumeragi_rbc_store_pressure`, `sumeragi_rbc_store_sessions`, `sumeragi_rbc_store_bytes`, `sumeragi_rbc_backpressure_deferrals_total`. |
| `npos_redundant_send_retries_update_metrics` | Повторная передача данных для того, чтобы датчики соотношения избыточной отправки и датчики сборщиков-на-цели были заранее, демонстрация того, что телеметрия pedida, чтобы ретро-соединение было сквозным. | `sumeragi_collectors_targeted_current`, `sumeragi_redundant_sends_total`. |
| `npos_rbc_chunk_loss_fault_reports_backlog` | Декарта разбивает определенные интервалы для проверки того, что мониторы невыполненной работы остаются незавершенными и автоматически отключают полезную нагрузку. | И18НИ00000105Х, И18НИ00000106Х, И18НИ00000107Х. |

Дополнение к линиям JSON, которые заполняют жгут проводов с помощью очистки Prometheus, захватывая их во время выброса, когда правительство запросит подтверждение того, что сигналы противодавления совпадают с топологией датчика.

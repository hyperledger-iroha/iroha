# Модель дорожек Nexus и участника WSV

> **Estado:** entregable NX-1 — таксономия дорожек, геометрия конфигурации и расположение списков хранения для реализации.  
> **Владельцы:** Nexus Основная рабочая группа, Рабочая группа по управлению  
> **Справочная информация по плану действий:** NX-1 и `roadmap.md`

На этой странице портала отражено краткое описание канона `docs/source/nexus_lanes.md` для операторов Sora Nexus, владельцев SDK и внесенных изменений, которые могут просмотреть руководство по переулкам, не входя в моно-репозиторий. Объектная архитектура, определяемая состоянием мира, позволяет отдельным лицам использовать пространства данных (полосы) совместно с публичными или частными лицами с рабочими нагрузками на участках.

## Концепции

- **Дорожка:** сегмент логики реестра Nexus с собственным набором валидаторов и накопителем невыполненных операций. Идентификатор установленного `LaneId`.
- **Пространство данных:** блок управления, в котором объединена группа или несколько полос, по которым объединяются политики соблюдения требований, маршрутизации и урегулирования.
- **Манифест дорожки:** метаданные для управления, которые описывают валидаторов, политику DA, жетон газа, правила расчетов и разрешения на маршрутизацию.
- **Глобальное обязательство:** доказательство выбросов по полосе, по которой возобновляются новые корни государства, данные о поселениях и дополнительные варианты перехода через полосу движения. Глобальные обязательства El anillo NPoS.

## Таксономия полос

Типы полос описывают каноническую форму, которую можно увидеть, поверхность управления и крючки урегулирования. Геометрия конфигурации (`LaneConfig`) содержит эти атрибуты для узлов, SDK и инструментов, которые можно использовать для создания индивидуального макета без логики.

| Типо де Лейн | Видимость | Члены валидаторов | Экспозиция WSV | Гобернанса за дефект | Политика урегулирования | Типичное использование |
|-----------|------------|----------------------|--------------|----------------|----|-------------------|-------------|
| `default_public` | общественный | Без разрешения (глобальная доля) | Полная копия состояния | SORA Парламент | `xor_global` | Публичная база Ledger |
| `public_custom` | общественный | Без разрешения или с кольями | Полная копия состояния | Модуль обдумывания ставки | `xor_lane_weighted` | Публичные приложения высокой пропускной способности |
| `private_permissioned` | ограничено | Установить фихо де валидаторов (проверка управления) | Обязательства и доказательства | Совет Федерации | `xor_hosted_custody` | CBDC, рабочие нагрузки консорциума |
| `hybrid_confidential` | ограничено | Мембрезия микста; envuelve ZK доказательства | Обязательства + раскрытие информации selectiva | Программируемый модуль денег | `xor_dual_fund` | Программируемые деньги с сохранением конфиденциальности |

Все советы по переулку должны быть объявлены:- Псевдоним пространства данных - четко выраженное объединение людей, которые хотят соблюдать политические правила.
- Ручка управления - идентификатор результата через `Nexus.governance.modules`.
- Дескриптор расчета – идентификатор, используемый маршрутизатором расчетов для дебетовых буферов XOR.
- Дополнительные метаданные телеметрии (описание, контакт, доменное имя) отправляются через `/status` на информационных панелях.

## Геометрия конфигурации полос (`LaneConfig`)

`LaneConfig` является производной среды выполнения геометрии из действительного каталога полос. Никакой замены манифестов управления; в вашем распоряжении доказательства идентификаторов определителей хранилища и каналов телеметрии для каждой конфигурации дорожек.

```text
LaneConfigEntry {
    lane_id: LaneId,           // stable identifier
    alias: String,             // human-readable alias
    slug: String,              // sanitised alias for file/metric keys
    kura_segment: String,      // Kura segment directory: lane_{id:03}_{slug}
    merge_segment: String,     // Merge-ledger segment: lane_{id:03}_merge
    key_prefix: [u8; 4],       // Big-endian LaneId prefix for WSV key spaces
    shard_id: ShardId,         // WSV/Kura shard binding (defaults to lane_id)
    visibility: LaneVisibility,// public vs restricted lanes
    storage_profile: LaneStorageProfile,
    proof_scheme: DaProofScheme,// DA proof policy (merkle_sha256 default)
}
```

- `LaneConfig::from_catalog` пересчитывает геометрию при загрузке конфигурации (`State::set_nexus`).
- Лос-псевдонимы дезинфицируют слизни в минускулах; Буквенно-цифровые символы последовательно сворачиваются в `_`. Если псевдоним создает пулю, используйте `lane{id}`.
- `shard_id` извлекает ключ метаданных `da_shard_id` (из-за дефекта `lane_id`) и обеспечивает сохранение журнала курсора сегмента для поддержания воспроизведения DA, определенного при перезапуске/перешардинге.
- Префиксы клавы гарантируют, что WSV управляет рангом клавы, чтобы разъединить линии, когда вы сравниваете бэкенд-сервер.
- Номера сегментов Куры определяются между хозяевами; аудиторы могут проверить сегментные директории и инструменты, изготовленные на заказ.
- Слияние сегментов (`lane_{id:03}_merge`) защищает последние корни подсказок о слиянии и обязательства глобального государства для этой полосы.

## Участник мирового государства- Логика мирового государства Nexus является союзом территорий государства по полосе движения. Публичные переулки сохраняются в полном объеме; Приватизация Лас-Лейнса/конфиденциальные экспортные корни Меркла/обязательства по слиянию бухгалтерской книги.
- El almacenamiento MV prefija cada clave с префиксом из 4 байтов `LaneConfigEntry::key_prefix`, созданный как `[00 00 00 01] ++ PackedKey`.
- Таблицы Compartidas (счета, активы, триггеры, регистры управления) включаются в состав групп по приоритетам полосы движения, контролируя диапазон сканирований детерминистов.
- Метаданные книги слияния отражают макет неправильного макета: каждая полоса описывает корни подсказки о слиянии и корни глобального сокращения состояния в `lane_{id:03}_merge`, разрешение сохранения или выселения, когда полоса удаляется.
- Межполосные индексы (псевдонимы учетных записей, реестры активов, манифесты управления) включаются в префиксы отдельных полос для быстрого согласования операций.
- **Politica de retencion** - общественные полосы сохраняются после полного блока; Соло-полосы обязательств могут быть компактными после контрольно-пропускных пунктов, из-за которых обязательства являются авторитарными. Конфиденциальные записи охраняются цифровыми журналами и выделенными сегментами, чтобы не блокировать другие рабочие нагрузки.
- **Инструменты** - использование средств управления (`kagami`, команды администратора CLI) позволяет ссылаться на пространство имен с ярлыком для показателей экспонента, этикетками Prometheus или всеми сегментами архива Kura.

## Маршрутизация и API

- Конечные точки Torii REST/gRPC принимаются и `lane_id` необязательно; la ausencia implica `lane_default`.
- В SDK показаны селекторы дорожек и псевдонимы карт, которые можно использовать в `LaneId`, используя каталог полос.
- Правила маршрутизации действуют в соответствии с действительным каталогом и могут быть удалены из пространства данных. `LaneConfig` подтвердите псевдонимы для телеметрии, информационных панелей и журналов.

## Плата за расчет и комиссию

- Cada Lane Paga сборы XOR и глобальные валидаторы. На этих полосах можно получить токены газа, но их необходимо депонировать на депонированном счете, эквивалентном XOR junto, под обязательства.
- Подтверждения расчетов, включая монеты, метаданные конверсии и платежи условного депонирования (например, перевод в глобальное хранилище комиссий).
- Унифицированный расчетный маршрутизатор (NX-3) буферизует дебетовые данные, используя префиксы полос, а также телеметрию расчетов, расположенную на линии с геометрией хранилища.

## Управление

- Переулки объявлены по модулю управления через каталог. `LaneConfigEntry` Добавьте псевдоним и оригинальный фрагмент, чтобы сохранить разборчивость телеметрии и журналов аудита.
- Реестр Nexus распределяет манифесты фирм, включающих `LaneId`, привязку пространства данных, дескриптор управления, дескриптор расчетов и метаданные.
- Потеряны перехватчики обновления во время выполнения, связанные с политиками управления (`gov_upgrade_id` из-за дефекта) и различия между регистраторами через мост телеметрии (события `nexus.config.diff`).

## Телеметрия и статус- `/status` отображает псевдонимы дорожек, привязки пространства данных, дескрипторы управления и параметры расчетов, производные каталога и `LaneConfig`.
- Метрики планировщика (`nexus_scheduler_lane_teu_*`) используют псевдонимы/слаги для того, чтобы операторы отображали отставание и быстро вводили TEU.
- `nexus_lane_configured_total` увидит номер въезда на определенную полосу и пересчитает, когда будет выполнена конфигурация. Телеметрия излучает различия между фирмами, когда возникает геометрия полос.
- Индикаторы невыполненного пространства данных включают метаданные псевдонима/описания, которые помогают операторам совместного доступа к бизнесу с доминиумами в сфере переговоров.

## Конфигурация и тип Norito

- `LaneCatalog`, `LaneConfig`, y `DataSpaceCatalog` живы в `iroha_data_model::nexus` и проверены структуры в формате Norito для манифестов SDK.
- `LaneConfig` живет в `iroha_config::parameters::actual::Nexus` и автоматически извлекает каталог; кодировка Norito не требуется, поэтому это вспомогательная внутренняя среда выполнения.
- Конфигурация пользовательского файла (`iroha_config::parameters::user::Nexus`) принимается как дескрипторы, объявленные в полосе и пространстве данных; el parseo ahora deriva la geometria y rechaza недействительные псевдонимы или идентификаторы дубликатов дорожек.

## Ручная работа

- Интеграция обновлений расчетного маршрутизатора (NX-3) с новой геометрией для дебитов и получения буфера XOR по этикету для пула полосы движения.
- Инструменты администратора расширения для списка семейств столбцов, уплотнения удаленных полос и проверки журналов блоков по полосам, используя пространство имен с пулеметом.
- Завершение алгоритма слияния (упорядочение, обрезка, обнаружение конфликтов) и дополнительных приспособлений регрессии для повторного воспроизведения между полосами.
- Совмещенные средства контроля соответствия для белых/черных списков и программируемой финансовой политики (например, NX-12).

# Mises a jour du modele de frais Nexus

Маршрут регулирования унифицирует захват, поддерживающий сбор детерминированных данных по полосе, где операторы могут эффективно сверять дебеты за газ с моделью Nexus.

- Для полной архитектуры маршрута, политики буфера, матрицы телеметрии и последовательности развертывания, voir `docs/settlement-router.md`. Это руководство поясняет комментарии к документам о параметрах, которые можно привязать к удобному для чтения дорожному плану NX-3 и комментировать SRE, doivent Surveiller le Router в производстве.
- Конфигурация действия газа (`pipeline.gas.units_per_gas`), включая десятичные `twap_local_per_xor`, `liquidity_profile` (`tier1`, `tier2`, или `tier3`) и т. д. `volatility_class` (`stable`, `elevated`, `dislocated`). Эти указатели указывают маршрут регулирования до того, как соединение XOR соответствует каноническому TWAP и является более подходящим для стрижки по полосе.
- Выполните транзакцию, требующую регистрации газа под номером `LaneSettlementReceipt`. Выберите источник идентификатора для апеллянта, локальный микромонтаж, немедленное выполнение XOR, XOR после стрижки, реализуемое отклонение (`xor_variance_micro`) и данные по блоку в миллисекундах.
- Выполнение блоков объединяет запросы по полосе/пространству данных и публикации через `lane_settlement_commitments` в `/v1/sumeragi/status`. Все раскрытые `total_local_micro`, `total_xor_due_micro` и `total_xor_after_haircut_micro` являются дополнениями к блоку для экспорта ночных ночей примирения.
- Новый компьютер `total_xor_variance_micro` соответствует границе безопасности потребителя (разница между XOR du и l'attendu после стрижки) и `swap_metadata` документирует параметры определения конверсии (TWAP, эпсилон, профиль ликвидности и класс волатильности) для наиболее мощных аудиторов проверка входов в независимость конфигурации выполнения.

Пользователи могут использовать `lane_settlement_commitments` для создания снимков существующих обязательств по полосе и пространству данных для проверки буферов, элементов стрижки и выполнения обмена, соответствующих модели Nexus.

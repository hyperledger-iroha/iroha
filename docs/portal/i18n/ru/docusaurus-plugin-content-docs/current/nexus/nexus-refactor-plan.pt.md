# План ремонта бухгалтерской книги Sora Nexus

Этот документ содержит описание или дорожную карту для рефакторинга Sora Nexus Ledger («Iroha 3»). Он отражает фактическую структуру репозитория и регрессивные наблюдения в генезисе/бухгалтерии WSV, согласование Sumeragi, триггеры смарт-контрактов, просмотр снимков, привязку указателя хоста-ABI и кодеков Norito. Цель и конвергенция для согласованной архитектурной архитектуры и тесты должны быть связаны со всем, что соответствует единому монолитному патчу.

## 0. Принципы гия
- Сохранение совместимости с гетерогенным аппаратным обеспечением; использовать aceleracao apenas с помощью флагов функций, подписаться на резервные идентификаторы com.
- Norito и файл сериализации. Qualquer Mudanca de estado/schema включает в себя тесты двустороннего кодирования/декодирования Norito и настройки приборов.
- Конфигурация флюида для `iroha_config` (пользователь -> фактические -> настройки по умолчанию). Remover переключает специальные пути создания окружения.
- Постоянная политика ABI V1 и инициатива. Хосты разрабатывают устаревшие типы указателей/системные вызовы в детерминированной форме.
- `cargo test --workspace` и золотые тесты (`ivm`, `norito`, `integration_tests`) продолжаются как основа ворот для каждой марки.

## 1. Снимок топологии хранилища
- `crates/iroha_core`: устройства Sumeragi, WSV, загрузчик генезиса, конвейеры (запрос, наложение, полосы zk), соединение для хоста смарт-контрактов.
- `crates/iroha_data_model`: авторитарная схема для данных и запросов в цепочке.
- `crates/iroha`: клиентский API, используемый для CLI, тестов, SDK.
- `crates/iroha_cli`: интерфейс командной строки операторов, который теперь содержит числовые API-интерфейсы в `iroha`.
- `crates/ivm`: байт-код Kotodama виртуальной машины, способ ввода указателя интеграции-ABI в хост.
- `crates/norito`: кодек сериализации с адаптерами JSON и серверными модулями AoS/NCB.
- `integration_tests`: утверждения межкомпонентного кобридогенеза/начальной загрузки, Sumeragi, триггеры, paginacao и т. д.
- Документы и метаданные для Sora Nexus Ledger (`nexus.md`, `new_pipeline.md`, `ivm.md`), поскольку это фрагментированная и частично удаленная версия кода.

## 2. Ремонтные работы и марки### Этап A – Фундаментальное и наблюдаемое
1. **Телеметрия WSV + снимки**
   - Стабилизация канонического API снимков в `state` (тип `WorldStateSnapshot`) с использованием запросов, Sumeragi и CLI.
   - Используйте `scripts/iroha_state_dump.sh` для создания снимков, определяемых через `iroha state dump --format norito`.
2. **Детерминизм генезиса/бутстрап**
   - Восстановите источник происхождения для плавного перехода в единый трубопровод с Norito (`iroha_core::genesis`).
   - Дополнительные настройки интеграции/регресса, связанные с повторной обработкой сначала или первым блоком и подтверждением корней идентификаторов WSV внутри Arm64/x86_64 (сопровождается `integration_tests/tests/genesis_replay_determinism.rs`).
3. **Испытания на фиксацию поперечного ящика**
   - Расширьте `integration_tests/tests/genesis_json.rs` для проверки инвариантов WSV, конвейера и ABI в едином жгуте.
   - Введение в каркас `cargo xtask check-shape`, вызывающий панику при дрейфе схемы (сопровождается отсутствием отставания в разработке инструментов DevEx; выберите пункт действия `scripts/xtask/README.md`).

### Фаза B — WSV и базовые сведения о запросах
1. **Операции государственного хранения**
   - Используйте `state/storage_transactions.rs` в транзакционном адаптере, который позволяет выполнить фиксацию и обнаружить конфликты.
   - Модульные тесты перед проверкой того, какие изменения в активах/мире/запускают фазовый откат в реальном времени.
2. **Рефатор моделей запросов**
   - Перемещение логики страниц/курсора для повторного использования компонентов в `crates/iroha_core/src/query/`. Алинхар представляет Norito и `iroha_data_model`.
   - Дополнительные запросы к снимкам для триггеров, активов и ролей с детерминированным заказом (сопровождается через `crates/iroha_core/tests/snapshot_iterable.rs` для первоначального открытия).
3. **Согласованность снимков**
   - Гарантия, что CLI `iroha ledger query` использует лучший моментальный снимок, который Sumeragi/fetchers.
   - Тесты регрессии моментальных снимков без CLI вживую в `tests/cli/state_snapshot.rs` (функция-зависимая для запуска данных).

### Фаза C — Конвейер Sumeragi
1. **Топология и суть эпох**
   - Дополнительный `EpochRosterProvider` для реализации функций, основанных на моментальных снимках ставки WSV.
   - `WsvEpochRosterAdapter::from_peer_iter` предлагает простой конструктор и amigavel для имитации тестов/тестов.
2. **Упрощение потока консенсуса**
   - Реорганизуйте `crates/iroha_core/src/sumeragi/*` по модулям: `pacemaker`, `aggregation`, `availability`, `witness` с типичными компонентами, связанными с `consensus`.
   - Заменить сообщение, передавая специальные конверты Norito, и ввести тесты свойств изменения представления (сопровождается отсутствием отложенных сообщений Sumeragi).
3. **Переулок Интеграсао/доказательство**
   - Алинхар Лейн подтверждает обязательства окружного прокурора и гарантии того, что РБК будет закрыт в единой форме.
   - Тест на сквозную интеграцию `integration_tests/tests/extra_functional/seven_peer_consistency.rs`, проверенный ранее или проверенный пользователем RBC.### Фаза D — Смарт-контракты и указатель хостов-ABI
1. **Аудитория фронтейра-ду-хозяин**
   - Объединение проверок типа указателя (`ivm::pointer_abi`) и адаптеров хоста (`iroha_core::smartcontracts::ivm::host`).
   - В соответствии с ожиданиями таблицы указателей и привязок манифеста хоста к `crates/iroha_core/tests/ivm_pointer_abi_tlv_types.rs` и `ivm_host_mapping.rs`, которые выполняют сопоставления TLV gold.
2. **Песочница для выполнения триггеров**
   - Refatorar запускает радар через `TriggerExecutor`, который включает подачу газа, проверку указателей и ведение журнала событий.
   - Дополнительные тесты регрессии для запуска путей объединения вызовов/времени (в сопровождении через `crates/iroha_core/tests/trigger_failure.rs`).
3. **Добавление CLI и клиента**
   - Гарантия, что в операционных системах CLI (`audit`, `gov`, `sumeragi`, `ivm`) используются функции, совместимые с клиентом `iroha`, чтобы предотвратить дрейф.
   - Тестирование снимков JSON для CLI вживую в `tests/cli/json_snapshot.rs`; подтвердите, что команды продолжают использовать ссылку на канонический JSON.

###Fase E - Усиление выполнения кодека Norito
1. **Регистр схем**
   - Создайте реестр схемы Norito в `crates/norito/src/schema/` для выполнения канонических кодировок для типов ядра.
   - Дополнительный документ проверяет проверку и кодификацию полезных нагрузок примера (`norito::schema::SamplePayload`).
2. **Обновление золотых светильников**
   - Настройте золотые светильники в `crates/norito/tests/*` для совпадения с новой схемой WSV после того, как вы выполните рефакторинг.
   - `scripts/norito_regen.sh` восстанавливает золотой JSON Norito в детерминированной форме с помощью помощника `norito_regen_goldens`.
3. **Интеграция IVM/Norito**
   - При сквозной сериализации манифестов Kotodama через Norito гарантируется, что указатель метаданных ABI будет согласованным.
   - `crates/ivm/tests/manifest_roundtrip.rs` mantem paridade Norito кодирование/декодирование манифестов.

## 3. Preocupacoes transversais
- **Estrategia de testes**: постепенное продвижение модульных тестов -> тесты ящиков -> интеграционные тесты. Тесты, которые могут привести к регрессу на самом деле; Новос тестирует препятствие, которое можно вернуть.
- **Документация**: В каждый момент времени активируйте `status.md` и воспользуйтесь возможностью `roadmap.md`, чтобы удалить полученные данные.
- **Оценки производительности**: существуют стенды Manter в `iroha_core`, `ivm` и `norito`; Дополнительная база медицинских препаратов pos-refactor для проверки того, что происходит регресс.
- **Флаги функций**: Manter переключает уровни доступа к бэкэндам, которые используются внешними инструментальными цепочками exigem (`cuda`, `zk-verify-batch`). Пути SIMD ЦП всегда создаются и выбираются во время выполнения; Резервные варианты резервного копирования определены для аппаратного обеспечения, которое не поддерживается.## 4. Немедленные действия
- Леса для этапа A (моментальный снимок + проводка телеметрии) - верные тарифы, которые можно выполнить в виде дорожной карты.
- Недавняя аудитория побеждений для `sumeragi`, `state` и `ivm`, чтобы увидеть следующие события:
  - `sumeragi`: разрешена защита мертвого кода или трансляция проверок изменения представления, состояние повтора VRF и экспорт телеметрии EMA. Eles Permanecem Gated ate que a simplificacao do fluxo de consenso da Fase C и os entregaveis de Integracao Lane/proof Sejam Entregues.
  - `state`: упрощение `Cell` и ротация телеметрии, входящей в тройку телеметрии WSV на этапе A, в виде нот SoA/параллельного применения ввода без отставания в конвейере на этапе C.
  - `ivm`: разрешено переключение CUDA, проверка конвертов и границ Halo2/Metal для обработки границы хоста с Fase D, а также для темы, пересекающей ускорение графического процессора; ядра постоянны, нет отставания, выделенного графическим процессором, и мы быстро естарем.
- Подготовка резюме перекрестной группы RFC представляет собой план для подписания перед применением любых вторжений кода.

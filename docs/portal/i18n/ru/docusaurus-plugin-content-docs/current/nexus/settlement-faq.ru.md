Эта страница зеркалирует внутренние часто задаваемые вопросы по урегулированию (`docs/source/nexus_settlement_faq.md`), чтобы читатели портала могли изучить те же рекомендации без поиска в монорепозитории. Здесь управляется, как Settlement Router обрабатывает результаты, какие метрики контроля и как SDK должен интегрировать полезную нагрузку Norito.

## Основные моменты

1. **Сопоставление переулка** — каждое пространство данных объявляет `settlement_handle` (`xor_global`, `xor_lane_weighted`, `xor_hosted_custody` или `xor_dual_fund`). Смотрите актуальный каталог переулок в `docs/source/project_tracker/nexus_config_deltas/`.
2. **Детерминированная конвертация** — маршрутизатор переводит все расчеты в XOR через источники ликвидности, утвержденные управления. Приватные переулки заранее по расшифровке XOR-буферов; стрижки применяются только тогда, когда буферы выходят за рамки политики.
3. **Телеметрия** — отслеживайте `nexus_settlement_latency_seconds`, счетчики конвертации и датчики стрижки. Дашборды находятся в `dashboards/grafana/nexus_settlement.json`, оповещения в `dashboards/alerts/nexus_audit_rules.yml`.
4. **Доказательства** — архивируйте конфиги, логи маршрутизатора, экспортные телеметрии и отчеты по сверке для аудитов.
5. **Обязанности SDK** — каждый SDK должен предоставлять помощники расчета, идентификаторы дорожек и кодировщики полезных данных Norito, чтобы сохранить паритет с маршрутизатором.

## Примеры потоков

| Тип переулок | Какие собрания | Что это подтверждает |
|-----------|--------------------|----------------|
| Приватная `xor_hosted_custody` | Лог роутера + `nexus_settlement_latency_seconds{lane}` + `settlement_router_haircut_total{lane}` | CBDC-буферы записывают детерминированный XOR, стрижка остается в пределах политики. |
| Публичная `xor_global` | Лог роутера + ссылка на DEX/TWAP + метрики латентности/конвертации | Общий вывод: ликвидность оценена по опубликованному TWAP с нулевыми стрижками. |
| Гибридная `xor_dual_fund` | Лог-маршрутизатор, показывающий диван для публики и экранированный + счетчики телеметрии | Смесь защищённая/публичная соблюдала коэффициенты управления и фиксировала стрижку для каждой части. |

# Вид ансамбля Norito

Norito - это кушетка для бинарной сериализации, используемая в рекламе Iroha: ее определенные комментарии к структурам данных, которые кодируются в файле, сохраняются на диске и обмениваются между контрактами и др. Ящик рабочего пространства загружается на Norito так, как на `serde`, и есть пары из разных материалов, производящие идентичные октеты.

Далее мы возобновим использование элементов и отправку канонических ссылок.

## Архитектура и государственный переворот

- **Наедине + полезная нагрузка** - Сообщение Norito начинается с согласования функций (флагов, контрольной суммы) по запросу полезной нагрузки. Пакеты макетов и сжатие выполняются с помощью отдельных фрагментов.
- **Определенное кодирование** - `norito::codec::{Encode, Decode}` реализует новое кодирование. Макет мема позволяет повторно использовать данные для загрузки полезных данных в одном месте, чтобы сохранить и сохранить подпись.
- **Схема + производные** - `norito_derive` род реализаций `Encode`, `Decode` и `IntoSchema`. Пакеты структур/последовательностей являются активными по умолчанию и документируются в `norito.md`.
- **Регистрация мультикодека** - Идентификаторы хэшей, типы ключей и описания полезной нагрузки в реальном времени в `norito::multicodec`. Справочная таблица поддерживается в `multicodec.md`.

## Расходы

| Таче | Командование / API | Заметки |
| --- | --- | --- |
| Инспектор наедине/разделы | `ivm_tool inspect <file>.to` | Укажите версию ABI, флаги и точки входа. |
| Кодер/декодер на Rust | `norito::codec::{Encode, Decode}` | Реализовать все основные типы модели данных. |
| Взаимодействие JSON | `norito::json::{to_json_pretty, from_json}` | JSON определяет доступ к значениям Norito. |
| Документация/спецификации генератора | `norito.md`, `multicodec.md` | Источник документации de verite a la racine du repo. |

## Рабочий процесс разработки

1. **Ajouter les Derives** - Preferez `#[derive(Encode, Decode, IntoSchema)]` для новых структур де-донне. Evitez les Serialisers ecrits a la main sauf necessite absolue.
2. **Действительные пакеты макетов** — используйте `cargo test -p norito` (и матрицу функций пакетов в `scripts/run_norito_feature_matrix.sh`), чтобы гарантировать, что новые макеты останутся стабильными.
3. **Regenerer les docs** - При изменении кодировки, в течение дня `norito.md` и мультикодеке таблицы, можно просмотреть страницы порта (`/reference/norito-codec` и далее).
4. **Проверьте тесты Norito-first** — тесты интеграции должны использовать вспомогательные средства JSON Norito вместо `serde_json` для выполнения мемов, которые будут использоваться в производстве.

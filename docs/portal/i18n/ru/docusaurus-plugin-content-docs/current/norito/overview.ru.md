# Обзор Norito

Norito — бинарный уровень сериализации, определённый во всех Iroha: он определяет, как структура данных кодируется в сети, сохраняется на диске и определяется между контрактами и хостами. Каждый ящик в рабочем пространстве основан на Norito вместо `serde`, чтобы пиры на разном оборудовании производили отдельные байты.

Настоящий обзор суммирует основные части и ссылается на канонические материалы.

## Архитектура в общих чертах

- **Заголовок + payload** – Каждое сообщение Norito начинается с заголовка согласования особенностей (флагов, контрольной суммы), за которым следует голый payload. Упакованные раскладки и удлинения согласуются через биты заголовка.
- **Детерминированное кодирование** – `norito::codec::{Encode, Decode}` реализует базовое кодирование. Тот же макет используется при оборачивании полезных нагрузок в заголовки, поэтому хеширование и подпись остаются детерминированными.
- **Схема + производные** – `norito_derive` реализации реализации `Encode`, `Decode` и `IntoSchema`. Упакованные структуры/последовательности включения по умолчанию и описания в `norito.md`.
- **Реестр мультикодек** – Идентификаторы хешей, типовых ключей и описаний полезной нагрузки находятся в `norito::multicodec`. Авторитетная таблица событий в `multicodec.md`.

## Инструменты

| Задача | Команда / API | Примечания |
| --- | --- | --- |
| Посмотреть заголовок/разделы | `ivm_tool inspect <file>.to` | Показывает версию ABI, флаги и точки входа. |
| Кодировать/декодировать в Rust | `norito::codec::{Encode, Decode}` | Реализовано для всех основных типов моделей данных. |
| Взаимодействие JSON | `norito::json::{to_json_pretty, from_json}` | Определенный JSON на основе результатов Norito. |
| Генерировать документацию/спецификации | `norito.md`, `multicodec.md` | Документация-источник истины в корневой репозитории. |

## Процесс разработки

1. **Добавить производные** – Предпочитайте `#[derive(Encode, Decode, IntoSchema)]` для новых структур данных. Избегайте ручных сериализаторов, если это не абсолютно необходимо.
2. **Проверить складываемые макеты** – Используйте `cargo test -p norito` (и матрицу упакованных функций в `scripts/run_norito_feature_matrix.sh`), чтобы убедиться, что новые макеты остаются стабильными.
3. **Перегенерировать документы** — при изменении кодирования обновите `norito.md` и таблицу мультикодека, затем обновите страницу портала (`/reference/norito-codec` и этот обзор).
4. **Держаты тест Norito-first** — Интеграционные тесты должны использовать JSON-хелперы Norito вместо `serde_json`, чтобы применить тот же путь, что и продакшн.

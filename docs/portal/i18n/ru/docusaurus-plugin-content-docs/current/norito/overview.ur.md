# Norito کا جائزہ

Norito Iroha میں استعمال ہونے والی بائنری سیریلائزیشن کی تہہ ہے: یہ طے کرتی ہے کہ ڈیٹا ڈھانچے نیٹ ورک پر کیسے کیسے کیسے کیسے ڈیٹا ڈھانچے ڈسک پر کیسے محفوظ ہوتے ہیں، کنٹریکٹس اور ہوسٹس کے درمیان کیسے تبادلہ ہوتے ہیں۔ Можно использовать ящик `serde`, который можно использовать Norito. Количество пиров یکساں پیدا کریں۔

یہ جائزہ بنیادی حصوں کا خلاصہ کرتا ہے اور معیاری حوالہ جات کی طرف لنک دیتا ہے۔

## ایک نظر میں معماری

- **ہیڈر + پے لوڈ** – ہر Norito پیغام согласование функций ہیڈر (флаги, контрольная сумма) سے شروع ہوتا ہے جس کے بعد سادہ полезная нагрузка آتا ہے۔ упакованные макеты, сжатие, бит, переговоры, переговоры,
- ** ڈیٹرمنسٹک انکوڈنگ** – `norito::codec::{Encode, Decode}` полезные нагрузки и заголовки, а также макет и макет, а также хеширование и подписание. ڈیٹرمنسٹک رہیں۔
- **اسکیما + производные** – `norito_derive` `Encode`, `Decode` и `IntoSchema` различные реализации بناتا ہے۔ упакованные структуры/последовательности.
- **Можно использовать** – хэши, типы ключей, дескрипторы полезной нагрузки и идентификаторы `norito::multicodec`, дополнительные сведения. Код `multicodec.md` - это приложение, предназначенное для ہےی

## ترقیاتی ورک فلو

1. **производит شامل کریں** – نئی ڈیٹا ساختوں کے لئے `#[derive(Encode, Decode, IntoSchema)]` کو ترجیح دیں۔ ہاتھ سے ککھی گئی سریلائزرز سے گریز کریں جب تک کہ بالکل ضروری نہ ہو۔
2. **Упакованные макеты** — `cargo test -p norito` Дополнительные сведения (`scripts/run_norito_feature_matrix.sh` — упакованная матрица функций) Дополнительные макеты مستحکم رہیں۔
3. **Документация доступна** – Дополнительная информация для `norito.md` с поддержкой мультикодека کریں، پھر پورٹل صفحات (`/reference/norito-codec` اور یہ جائزہ)

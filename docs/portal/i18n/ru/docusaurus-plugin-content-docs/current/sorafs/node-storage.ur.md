## SoraFS проект хранилища узлов (черновик)

Воспользуйтесь узлом Iroha (Torii) для уровня доступности данных SoraFS. Доступ к локальному диску и фрагментам данных или выбору фрагментов данных Спецификация обнаружения `sorafs_node_client_protocol.md` для устройства SF-1b, а также для архитектуры системы хранения данных, контроля ресурсов и сантехнических систем конфигурации. ### Цели

- Наличие валидатора, вспомогательного процесса Iroha, запасного диска, предоставления поставщика SoraFS, открытия основного реестра и т. д. متاثر کیے۔
- модуль хранения, детерминированный, управляемый Norito: манифесты, планы фрагментов, корни доказательства возможности извлечения (PoR), реклама поставщика, источник истины.
- определяемые оператором квоты обеспечивают контроль за узлом, запросы на закрепление/выборку, а также заполнение ресурсов и их выборку.
- работоспособность/телеметрия (выборка PoR, задержка выборки фрагментов, нагрузка на диск) и управление клиентами, а также проверка работоспособности.

Ключевые модули:

- **Шлюз**: конечные точки HTTP Norito предоставляют множество предложений по выводам, запросы на выборку фрагментов, выборку PoR и телеметрию, а также телеметрию. Полезные нагрузки Norito проверяют наличие запросов на хранение фрагментов и маршала. Демон может использовать Torii Повторное использование стека HTTP.
- **Реестр выводов**: состояние вывода манифеста جو `iroha_data_model::sorafs` или `iroha_core` для отслеживания состояния Манифест принимается, дайджест манифеста реестра, дайджест плана фрагментов, корень PoR и флаги возможностей поставщика, запись.
- **Хранилище фрагментов**: реализация `ChunkStore` на диске и подписанные манифесты, принимаемые `ChunkProfile::DEFAULT`, планы фрагментов материализуются, а также сохраняются фрагменты и детерминированный макет. کرتی ہے۔ Отпечаток содержимого фрагмента Метаданные PoR и ассоциированная выборка Выборка и повторная проверка سکے۔
- **Квота/Планировщик**: лимиты, настраиваемые оператором (максимальное количество дисковых байтов, максимальное количество выдающихся контактов, максимальное количество параллельных выборок, TTL фрагмента) обеспечивают соблюдение ограничений ввода-вывода и координации узла, когда обязанности реестра прекращаются. Планировщик PoR проверяет запросы на выборку и ограниченный ЦП, а также обеспечивает возможность обслуживания

### Конфигурация

`iroha_config` Информация о разделе:

```toml
[sorafs.storage]
enabled = false
data_dir = "/var/lib/iroha/sorafs"
max_capacity_bytes = "100 GiB"
max_parallel_fetches = 32
max_pins = 10_000
por_sample_interval_secs = 600
alias = "tenant.alpha"            # optional human friendly tag
adverts:
  stake_pointer = "stake.pool.v1:0x1234"
  availability = "hot"
  max_latency_ms = 500
  topics = ["sorafs.sf1.primary:global"]
```- `enabled`: переключение участия۔ false ہو تو конечные точки хранилища шлюза پر 503 دیتا ہے اور node Discovery میں Advertise نہیں کرتا۔
- `data_dir`: данные фрагмента, деревья PoR и выбор телеметрии, а также корневой каталог. По умолчанию `<iroha.data_dir>/sorafs` ہے۔
- `max_capacity_bytes`: закрепленные данные фрагмента и жесткий лимит. лимит фоновых задач پر پہنچنے پر نئے отклонить контакты
- `max_parallel_fetches`: планировщик, ограничение одновременного выполнения, пропускная способность/дисковый ввод-вывод, рабочая нагрузка валидатора, баланс баланса.
- `max_pins`: контакты манифеста, максимальный размер узла и узел, который используется для вытеснения/обратного давления.
- `por_sample_interval_secs`: автоматические задания выборки PoR с частотой кадров. ہر задание `N` оставляет образец کرتا ہے (настраивается для каждого манифеста) События телеметрии выдают کرتا ہے۔ Ключ метаданных управления `profile.sample_multiplier` (целое число `1-4`) или `N` для детерминированного масштабирования. Значение Одно число/строка Переопределение для каждого профиля Объект или объект `{"default":2,"sorafs.sf2@1.0.0":3}`۔
- `adverts`: структура и генератор рекламы провайдера. Поля `ProviderAdvertV1` (указатель ставки, подсказки QoS, темы). Опустите ہو تو параметры реестра управления узлами по умолчанию.

Конфигурация сантехники:

- `[sorafs.storage]` `iroha_config` `SorafsStorage` позволяет определить файл конфигурации узла и загрузить файл конфигурации.
- `iroha_core` или `iroha_torii` конфигурация хранилища: запуск, сборщик шлюзов, хранилище фрагментов, потоки,

### Жизненный цикл узла

1. **Запуск**:
   - хранилище включено, каталог сконфигурирован для узла, емкость, хранилище фрагментов, инициализация. Проверка базы данных манифестов PoR. Проверка/создание. Повтор закрепленных манифестов. Использование кэшей. Теплый режим.
   - Регистрация маршрутов шлюза SoraFS (Norito конечные точки JSON POST/GET для закрепления, выборки, выборки PoR, телеметрии).
   - Работник выборки PoR и монитор квот появляются.
2. **Обнаружение/Реклама**:
3. **Рабочий процесс закрепления**:
   - Манифест, подписанный шлюзом وصول کرتا ہے (частный план, корень PoR, подписи совета شامل)۔ проверка списка псевдонимов (требуется `sorafs.sf1@1.0.0`) план фрагмента и метаданные манифеста совпадение
   - Проверка квот کریں۔ Если пределы емкости/контактов превышают ہوں или ошибка политики (структурированный Norito) دیں۔
   - Чанк данных `ChunkStore` для потока или приема данных, а также для проверки дайджестов. Обновление деревьев PoR کریں اور реестр метаданных манифеста и хранилище کریں۔
4. **Извлечение рабочего процесса**:
   - Запросы диапазона фрагментов диска служат более эффективно. Планировщик `max_parallel_fetches` обеспечивает насыщенную насыщенную работу `429` в режиме насыщения
   - Структурированная телеметрия (Norito JSON) генерирует большое количество задержек, количество обработанных байтов и количество ошибок, а также мониторинг нисходящего потока.
5. **Выборка PoR**:
   - Worker манифестирует вес (в сохраненных байтах), выбор выбора, хранилище фрагментов, дерево PoR, детерминированную выборку,
   - Результаты, аудит управления, сохранение, реклама поставщика/конечные точки телеметрии, сводки, отчеты.
6. **Выселение/исполнение квот**:
   - Емкость узла по умолчанию, когда контакты отклоняются. Дополнительные политики вытеснения операторов (TTL, LRU) настройка модели управления или модели управления. Для разработки строгих квот и операций открепления, инициируемых оператором.

SF-3 содержит выбрасываемый ящик для капсюлей `sorafs-node`, который преобразует процесс Iroha/Torii в хранилище SoraFS. США - это план объединения с [гиа-де-альмаценамиенто-дель-нодо](node-storage.md), ла [политика допуска доверенных лиц](provider-admission-policy.md) и [хода де рута дель рыночная площадь декапасидад де альмасенамиенто](storage-capacity-marketplace.md) al секуэнсиар entregables.

## Альканс объективо (Хито М1)

1. **Интеграция хранилища фрагментов.** Оберните `sorafs_car::ChunkStore` с постоянным бэкэндом, который защищает байты фрагмента, и отображает символы PoR в настроенном каталоге данных.
2. **Конечные точки шлюза.** Выставляйте конечные точки HTTP Norito для отправки контактов, получения фрагментов, мгновенного PoR и телеметрии обмена данными в рамках процесса Torii.
3. **Проект конфигурации.** Объединение структуры конфигурации `SoraFsStorage` (флаг доступности, мощности, директорий, ограничений одновременного использования) с подключением к маршрутам `iroha_config`, `iroha_core` и `iroha_torii`.
4. **Изменения/планирование.** Наложите ограничения на дискотеку/параллелизм, определенные оператором и его заботами с противодавлением.
5. **Телеметрия.** Создавайте метрики/журналы для вывода контактов, задержки выборки фрагментов, использования емкости и результатов Muestreo PoR.

### C. Конечные точки шлюза

| Конечная точка | Транспорт | Тарес |
|----------|----------------|--------|
| `POST /sorafs/pin` | Примите `PinProposalV1`, подтвердите манифесты, загрузите их и ответьте на CID манифеста. | Подтвердите доступ к чанкам, отключите их и транслируйте данные через хранилище фрагментов. |
| `GET /sorafs/chunks/{cid}` + консультация по рангу | Отобразить байты заголовков фрагментов `Content-Chunker`; соответствие характеристикам дальности полета. | Планировщик США + возможности потоковой передачи (доступен для рангом SF-2d). |
| `POST /sorafs/por/sample` | Ejecuta muestreo PoR для проявления и раскрытия пакета pruebas. | Возвращайтесь к хранилищу фрагментов и отвечайте на полезные данные Norito JSON. |
| `GET /sorafs/telemetry` | Резюме: емкость, выход PoR, сообщения об ошибках выборки. | Пропорции данных для информационных панелей/операторов. |

Работа во время выполнения в режиме взаимодействия PoR по маршруту `sorafs_node::por`: регистрация трекера `PorChallengeV1`, `PorProofV1` и `AuditVerdictV1` для тех показателей `CapacityMeter` Отобразить вердикты губернатора без логики Torii персонализации.【crates/sorafs_node/src/scheduler.rs#L147】

Примечания о реализации:

- Используйте стек Axum de Torii с полезной нагрузкой `norito::json`.
- Соберите список Norito для ответов (`PinResultV1`, `FetchErrorV1`, структуры телеметрии).

- ✅ `/v2/sorafs/por/ingestion/{manifest_digest_hex}` теперь демонстрирует глубину накопившихся задолженностей в эпоху/ограничение большего числа антигусов и временных меток выхода/выпадает больше информации для подтверждения, импульса для `sorafs_node::NodeHandle::por_ingestion_status`, и Torii регистрации датчиков `torii_sorafs_por_ingest_backlog`/`torii_sorafs_por_ingest_failures_total` пункт панели мониторинга.【crates/sorafs_node/src/lib.rs:510】【crates/iroha_torii/src/sorafs/api.rs:1883】【crates/iroha_torii/src/routing.rs:7244】【crates/iroha_telemetry/src/metrics.rs:5390】

### F. Estrategia de Pruebas

1. **Унитарные единицы.** Сохранение хранилища фрагментов, вычислений и неизменяемых вариантов планировщика (версия `crates/sorafs_node/src/scheduler.rs`).
2. **Процессы интеграции** (`crates/sorafs_node/tests`). PIN-код → получить поездку туда и обратно, восстановление по маршруту, повторную отправку, проверку pruebas de muestreo PoR.
3. **Процесс интеграции Torii.** Вызов Torii с привычным доступом, подключение конечных точек HTTP через `assert_cmd`.
4. **Hoja de ruta de caos.** Futuros тренирует имитацию дискотеки, IO lento, retiro de provedores.

## Зависимости

- Политика допуска SF-2b — убедитесь, что все проверенные конверты допущены до объявления.
- Marketplace de capacidad SF-2c — проинформируйте телеметрию о пропуске в соответствии с объявлениями о емкости.
- Удлинители рекламы SF-2d — потребляют возможность звонка + возможность трансляции, когда это доступно.

## Критерии достижения цели

- `cargo run -p sorafs_node --example pin_fetch` работает против региональных настроек.
- Torii скомпилирован с `--features sorafs-storage` и прошел процесс интеграции.
- Документация ([руководство по резервированию узла](node-storage.md)) актуализируется с настройками по умолчанию + примерами CLI; доступный рабочий журнал.
- Телеметрия видна на информационных панелях постановки; Оповещения конфигурируются для насыщения емкости и падения PoR.

## Включаемые документы и операции

- Актуализировать [ссылку на хранилище узлов] (node-storage.md) с настройками по умолчанию, использованием CLI и действиями по устранению неполадок.
- Управляйте [runbook deoperaciones de nodo](node-operations.md), добавленным в соответствии с эволюцией SF-3.
- Публикуйте ссылки API для конечных точек `/sorafs/*` на портале разархивирования и подключайтесь к манифестам OpenAPI, включая те обработчики Torii, которые есть в списках.

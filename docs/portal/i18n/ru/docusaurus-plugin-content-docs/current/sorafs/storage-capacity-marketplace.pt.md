# Торговая площадка для вооружения SoraFS (Rascunho SF-2c)

Пункт SF-2c представляет собой дорожную карту по внедрению правительственного рынка среди поставщиков
Armazenamento заявляет о возможности компрометации, получает заказы на репликацию и
Ганхамские сборы пропорциональны диспонибилидаду. Разграниченный документ
мы entregaveis exigidos для первого выпуска и os разделить их трила acionaveis.

## Объективос

- Экспресс-компромис-де-емкость (всего байтов, лимитов пор, истекает)
  в проверенном формате, используемом для управления, транспортировке SoraNet и Torii.
- Закрепите контакты между поставщиками соглашения с декларируемыми возможностями, ставками и
  детерминированные ограничения политики.
- Medir Entrega de Armazenamento (успех репликации, время безотказной работы, доказательства
  Интеграция) и экспорт телеметрии для распределения сборов.
- Проверка процессов отзыва и споров о том, что поставщики находятся в безопасности
  наказаны или удалены.

## Conceitos de dominio

| Консейто | Описание | Энтрегавел начальный |
|---------|-----------|--------------------|
| `CapacityDeclarationV1` | Полезная нагрузка Norito определяет идентификатор поставщика, поддерживает блокировку блоков, компромиссы GiB, ограничения по полосам, подсказки по ценам, компромисс по ставкам и истечение срока действия. | Esquema + валидатор `sorafs_manifest::capacity`. |
| `ReplicationOrder` | Инструктирование по управлению, которое присваивает идентификатор CID всем или большинству поставщиков, включая уровень избыточности и показатели SLA. | Esquema Norito совместим с Torii + API смарт-контракта. |
| `CapacityLedger` | Реестр в сети/вне сети, в котором происходит декларирование активной мощности, заказы на репликацию, показатели производительности и начисление комиссий. | Модуль смарт-контракта или заглушка услуги вне сети с детерминированным моментальным снимком. |
| `MarketplacePolicy` | Политика управления определяет минимальную ставку, требования к аудитории и критерии наказания. | Структура конфигурации `sorafs_manifest` + документ управления. |

## Реквизиты для тестирования и CI

- Унитарные тесты для всех новых валидаторов схемы (`sorafs_manifest`).
- Тесты интеграции, которые выполняются одновременно: объявление -> порядок репликации -> измерение -> выплата.
- Рабочий процесс CI для восстановления заявлений/телеметрии емкости и гарантии постоянной синхронизации (название `ci/check_sorafs_fixtures.sh`).
- Проверка грузов для реестра API (одновременно 10 тысяч поставщиков, 100 тысяч заказов).

## Телеметрия и информационные панели

- Панель приборов:
  - Заявленные возможности и возможности использования поставщиком.
  - Отставание заказов на репликацию и посредническую поддержку.
  - Соответствие SLA (% времени безотказной работы, таксы успешного PoR).
  - Начисление сборов и штрафов за эпоху.
- Оповещения:
  - Поставщик обеспечивает минимальную компрометацию возможностей.
  - Орден репликации трагедии > SLA.
  - В Фалхасе нет трубопровода учета.

## Ввод документов- Руководство по объявлению работоспособности, восстановлению компромиссов и мониторингу использования.
- Руководство по утверждению деклараций, изданию приказов и ведению споров.
- Ссылка на API для конечных точек емкости и формата порядка репликации.
- Часто задаваемые вопросы по торговой площадке для разработчиков.

### Ночная контабилизация и примирение XOR
- Экспортировать снимок состояния емкости и экспортировать реестр XOR для отправки сообщения,
  депуа выполнить:
  ```bash
  python3 scripts/telemetry/capacity_reconcile.py \
    --snapshot artifacts/sorafs/capacity/state_$(date +%F).json \
    --ledger artifacts/sorafs/capacity/ledger_$(date +%F).ndjson \
    --label nightly-capacity \
    --json-out artifacts/sorafs/capacity/reconcile_$(date +%F).json \
    --prom-out "${SORAFS_CAPACITY_RECONCILE_TEXTFILE:-artifacts/sorafs/capacity/reconcile.prom}"
  ```
  Помощник саи с кодом, который не содержит расчетов или штрафов, превышающих/выполняемых и эмитированных
  резюме Prometheus в текстовом файле формата.
- О предупреждении `SoraFSCapacityReconciliationMismatch` (em `dashboards/alerts/sorafs_capacity_rules.yml`)
  расхождения в показателях сверки отчетов; приборные панели Ficam Em
  `dashboards/grafana/sorafs_capacity_penalties.json`.
- Архивируйте резюме в формате JSON и хэши в формате `docs/examples/sorafs_capacity_marketplace_validation/`.
  объединение с пакетами управления.

## Зависимости и последовательность

1. Finalizar SF-2b (политика допуска) — рынок, зависящий от поставщиков разрешений.
2. Внедрите схему + реестр (эту документацию) перед интеграцией Torii.
3. Полная система учета обязательных выплат.
4. Финальный этап: умение распределять контрольные сборы за управление, когда это происходит.
   замеры проходят проверку на стадии постановки.

О прогрессе, который произошел, не было дорожной карты со ссылками на этот документ. Реализовать о
дорожная карта, предполагающая основные принципы (схема, план контроля, интеграция, измерение,
обработка спора) функция статуса завершена.

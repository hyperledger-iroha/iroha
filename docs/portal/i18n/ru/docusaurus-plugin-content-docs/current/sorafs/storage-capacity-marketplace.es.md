# Меркадо емкости для хранения багажа SoraFS (Borador SF-2c)

Пункт дорожной карты SF-2c представляет гобернадо-торговлю среди поставщиков
almacenamiento объявил о возможности компрометации, получил заказы на репликацию и
Гананские сборы пропорциональны а-ля диспонибилидад-энтрегада. Разграниченный документ
Лос-Entregables требуется для первого выпуска и лос-разделения на путях доступа.

## Объективос

- Expresar Compromisos de Capacidad (общее количество байтов, ограничения по полосам, срок действия)
  в форме проверяемого расходного материала для правительства, транспорта SoraNet и Torii.
- Назначьте контакты между поставщиками в следующих объявленных возможностях, ставках и ограничениях.
  Политика mientras se mantiene comportamiento determinista.
- Medir la entrega de Storage (выход из репликации, время безотказной работы, доказательства интеграции) и
  экспортировать телеметрию для распределения сборов.
- Докажите процедуру отзыва и спор о том, что поставщики нечестны, Шон
  наказаны или удалены.

## Концепции господства

| Концепт | Описание | Уступчивый инициал |
|---------|-------------|---------------------|
| `CapacityDeclarationV1` | Полезная нагрузка Norito, описывающая идентификатор провайдера, поддержку сегмента блоков, компромиссы в ГиБ, ограничения по полосам, точки ценообразования, компромиссы по ставкам и сроку действия. | Esquema + валидатор на `sorafs_manifest::capacity`. |
| `ReplicationOrder` | Инструкция, выданная правительству, о присвоении CID манифеста одному или нескольким поставщикам, включая уровень резервирования и метрики SLA. | Esquema Norito совместим с Torii + API смарт-контракта. |
| `CapacityLedger` | Реестр в сети/вне сети, включающий декларации об активизации, порядок репликации, метрики передачи и начисление комиссий. | Модуль смарт-контракта или заглушка услуги вне сети с детерминированным моментальным снимком. |
| `MarketplacePolicy` | Политика правительства определяет минимальную ставку, требования к аудитории и критерии наказания. | Структура конфигурации в `sorafs_manifest` + правительственный документ. |

## Реквизиты для тестирования и CI

- Унитарные тесты для всех валидаторов новых школ (`sorafs_manifest`).
- Симуляционные тесты интеграции: объявление -> порядок репликации -> измерение -> выплата.
- Рабочий процесс CI для восстановления объявлений/телеметрии емкости и обеспечения синхронизации фирм (расширитель `ci/check_sorafs_fixtures.sh`).
- Тестирование загрузки API реестра (одновременно 10 тысяч поставщиков, 100 тысяч заказов).

## Телеметрия и информационные панели

- Панели приборной панели:
  - Заявленная мощность или использование поставщиком услуг.
  - Отставание в выполнении заказов на репликацию и снижение скорости присвоения.
  - Сумма соглашения об уровне обслуживания (% времени безотказной работы, время выхода PoR).
  - Начисление сборов и штрафов за эпоху.
- Оповещения:
  - Поставщик услуг по минимальной компрометации.
  - Порядок репликации атак > SLA.
  - Фальяс в трубопроводе измерения.

یہ В SRE вам нужно будет получить доступ к информации, которую вы хотите получить. آپریشنز میں رہنمائی کرتی ہے۔ یہ ڈیولپر گائیڈ کو کو کو کشن رول کآؤٹس کے مطابق کریقۂ کار سے کرتی ہے، جن میں مرحلہ وار فعال سازی اور کی بلیک لسٹنگ شامل ہے۔

> **مزید دیکھیں:** [ملٹی سورس رول آؤٹ رن بک](./multi-source-rollout.md) فلیٹ لیول رول آؤٹ Если вы хотите, чтобы вы выбрали лучший вариант для себя, вы можете использовать گورننس / اسٹیجنگ کی ہم آہنگی کے لیے اسے دیکھیں اور روزمرہ آرکسٹریٹر آپریشنز کے لیے اس دستاویز کا استعمال کریں۔

## 1. قبل از عمل چیک لسٹ

1. ** پرووائیڈر ان پٹس جمع کریں**
   - ہدف فلیٹ کے لیے تازہ ترین پرووائیڈر اشتہارات (`ProviderAdvertV1`) اور ٹیلیمیٹری اسنیپ شاٹ۔
   - زیرِ آزمائش مینِفیسٹ سے اخذ کردہ payload پلان (`plan.json`)۔
2. **متعین (детерминированный) اسکور بورڈ تیار کریں**

   ```bash
   sorafs_fetch \
     --plan fixtures/plan.json \
     --telemetry-json fixtures/telemetry.json \
     --provider alpha=fixtures/provider-alpha.bin \
     --provider beta=fixtures/provider-beta.bin \
     --provider gamma=fixtures/provider-gamma.bin \
     --scoreboard-out artifacts/scoreboard.json \
     --json-out artifacts/session.summary.json
   ```

   - تصدیق کریں کہ `artifacts/scoreboard.json` کے طور پر درج ہے۔
   - Доступ к файлам JSON и файлам JSON. Если вы хотите использовать кусок куска, который вы хотите использовать, کرتے ہیں۔
3. **Светильники для пробного запуска** — `docs/examples/sorafs_ci_sample/` Полезные нагрузки, которые можно использовать для получения полезных данных جائے۔

## 2. مرحلہ وار رول آؤٹ طریقۂ کار

1. **کینری مرحلہ (≤2 پرووائیڈرز)**
   - Для получения дополнительной информации используйте `--max-peers=2`. Если вы хотите, чтобы вы знали, как это сделать,
   - Ответ на вопрос:
     - `sorafs_orchestrator_active_fetches`
     - `sorafs_orchestrator_fetch_failures_total{reason!="retry"}`
     - `sorafs_orchestrator_retries_total`
   - Если вы хотите получить доступ к 1% стоимости رہیں اور کوئی پرووائیڈر ناکامیاں جمع نہ کرے۔
2. **رَیمپ مرحلہ (50% отдачи)**
   - `--max-peers` Если вы хотите, чтобы вы могли использовать
   - ہر رن کو `--provider-metrics-out` или `--chunk-receipts-out` کے ساتھ کحفوظ کریں۔ Возраст: ≥7 дней в году
3. **Мать رول آؤٹ**
   - `--max-peers` ہٹا دیں (یا اسے تمام اہل تعداد پر سیٹ کریں)۔
   - Что нужно знать, чтобы получить больше информации: Используйте JSON для создания файлов, необходимых для работы с JSON. تقسیم کریں۔
   - Если вы хотите использовать `sorafs_orchestrator_fetch_duration_ms` p95/p99, вы можете использовать его для проверки. ریٹرائی ہسٹوگرامز دکھائی دیں۔

## 3. Сверстники کی بلیک لسٹنگ اور بوسٹنگ

Интерфейс командной строки позволяет использовать интерфейс командной строки для быстрого и удобного использования. کیے بغیر غیر صحت مند پرووائیڈرز کی درجہ بندی ہو سکے۔

```bash
sorafs_fetch \
  --plan fixtures/plan.json \
  --telemetry-json fixtures/telemetry.json \
  --provider alpha=fixtures/provider-alpha.bin \
  --provider beta=fixtures/provider-beta.bin \
  --provider gamma=fixtures/provider-gamma.bin \
  --deny-provider=beta \
  --boost-provider=gamma=5 \
  --json-out artifacts/override.summary.json
```- `--deny-provider` درج شدہ عرف کو موجودہ سیشن میں غور سے ہٹا دیتا ہے۔
- `--boost-provider=<alias>=<weight>` پرووائیڈر کے شیڈیولر وزن میں اضافہ کرتا ہے۔ Если вы хотите, чтобы это произошло, вы можете сделать это в любое время. لاگو ہوتی ہیں۔
- Чтобы получить доступ к JSON-файлу, выберите нужный вариант. تاکہ بنیادی مسئلہ حل ہونے پر ذمہ دار ٹیم اسٹیٹ کو ہم آہنگ کر سکے۔

مستقل تبدیلیوں کے لیے سورس ٹیلیمیٹری میں کریں (خلاف ورزیکرنے Если вы наложили штраф, вы можете использовать интерфейс командной строки (CLI), чтобы получить больше информации. ڈیٹڈ اسٹریم بجٹس کے ساتھ اشتہار ریفریش کریں۔

## 4. خرابیوں کی جانچ

Вы можете принести текст:

1. Если вы хотите получить больше информации о том, как это сделать:
   - `scoreboard.json`
   - `session.summary.json`
   - `chunk_receipts.json`
   - `provider_metrics.json`
2. `session.summary.json` для получения дополнительной информации:
   - `no providers were supplied` → پرووائیڈر راستوں اور اشتہارات کی تصدیق کریں۔
   - `retry budget exhausted ...` → `--retry-budget` بڑھائیں یا غیر مستحکم сверстников ہٹا دیں۔
   - `no compatible providers available ...` → خلاف ورزی کرنے والے پرووائیڈر کی رینج کیپیبلٹی میٹاڈیٹا آڈٹ کریں۔
3. Если у вас есть `sorafs_orchestrator_provider_failures_total`, вы можете выбрать подходящий вариант. ہو تو فالو اپ ٹکٹ بنائیں۔
4. `--scoreboard-json` поможет вам получить доступ к информации, которую вы хотите получить. طور پر دوبارہ پیدا کریں۔

## 5. رول بیک

آرکسٹریٹر رول آؤٹ واپس کے لیے:

1. Чтобы получить доступ к `--max-peers=1`, выберите (если хотите). Если вы хотите, чтобы это произошло, вы можете получить деньги. پر واپس لے جائیں۔
2. Используйте `--boost-provider`, чтобы узнать больше о возможностях использования. پر واپس آ جائے۔
3. Если вы хотите, чтобы вы выбрали лучший вариант, который вам нужен, вы можете جائے کہ کوئی باقی fetch کارروائیاں کاری نہیں ہیں۔

آرٹیفیکٹس کی منظم کیپچرنگ اور مرحلہ وار رول آؤٹس برقرار رکھنے سے یہ یقینی بنتا ہے کہ ملٹی سورس آرکسٹریٹر کو مختلف نوعیت کے پرووائیڈر فلیٹس میں محفوظ طریقے چلایا چلایا جا سکے، جبکہ آبزرویبیلٹی اور آڈٹ تقاضے برقرار رہیں۔

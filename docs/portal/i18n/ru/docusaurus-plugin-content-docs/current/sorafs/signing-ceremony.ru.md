> Дорожная карта: **SF-1b — протоколы заседаний Парламента Сора.**
> Парламентский рабочий процесс заменяет виртуальную оффлайн «церемонию подписания совета».

Ручной ритуал подписания светильников chunker SoraFS завершен. Все положения теперь
передача через **Парламент Сора** — ДАО на основе жеребьевки, управляющая Nexus.
Члены парламента связывают XOR с получением гражданства, ротируются по панелям и
Голосуйте по цепочке для утверждения, отклонения или отката выпускных приспособлений. этот
гайд мозговой процесс и инструменты для разработчиков.

## Обзор парламента

- **Гражданство** — Операторы блокируют требуемый XOR, чтобы стать гражданами и
  получить право на жеребьевку.
- **Панели** — Ответственность распределяется между поворотами панелей.
  (Инфраструктура, Модерация, Казначейство, ...). Панель Инфраструктура отвечает
  согласно утверждению светильников SoraFS.
- **Жеребьевка и ротация** — Места в панелях перераспределяются с периодичностью,
  заданной конституцией парламента, чтобы ни одна группа не монополизировала
  заявление.

## Поток утверждения светильников

1. **Отправка предложения**
   - Tooling WG загружает кандидатский комплект `manifest_blake3.json` и приспособление для дифференциала.
     в реестре цепочки через `sorafs.fixtureProposal`.
   - Предложение фиксирует дайджест BLAKE3, семантическую версию и заметки об изменениях.
2. **Обзор и голосование**
   - Панель «Инфраструктура» получает назначение через очередь задач парламента.
   - Члены панели изучают артефакты CI, запускают тесты на четность и озвучивают в цепочке.
     взвешенными голосами.
3. **Финализация**
   - После достижения кворума во время выполнения эмитировать событие с традиционным дайджестом
     манифест и обязательства Меркла по приспособлению полезной нагрузки.
   - Событие зеркалируется в реестре SoraFS, чтобы клиенты могли получить последнюю версию.
     манифестный, утвержденный парламентом.
4. **Распространение**
   - Помощники CLI (`cargo xtask sorafs-fetch-fixture`) подтягивают утвержденный манифест
     через Nexus RPC. Константы JSON/TS/Go в репозиториях синхронизируются повторно.
     запуском `export_vectors` и проверкой дайджеста соответственно записи в цепочке.

## Разработчик рабочих процессов

- Перегенерируйте светильники:

```bash
cargo run -p sorafs_chunker --bin export_vectors
```

- Используйте парламентский помощник, чтобы скачать официальный конверт, проверить.
  Сохранение и обновление локальных светильников. Укажите `--signatures` на конверте, опубликованный
  парламентом; helper находит составляющие манифест, пересчитает дайджест BLAKE3 и применит
  канонический профиль `sorafs.sf1@1.0.0`.

```bash
cargo xtask sorafs-fetch-fixture \
  --signatures https://nexus.example/api/sorafs/manifest_signatures.json \
  --out fixtures/sorafs_chunker
```

Передайте `--manifest`, если манифест находится по другому URL. Конверт без подписки
отклоняются, если не задан `--allow-unsigned` для локальных дымовых трасс.

- При валидации манифеста через промежуточный шлюз воспользуйтесь Torii вместо локальных
  полезная нагрузка:

```bash
sorafs-fetch \
  --plan=fixtures/chunk_fetch_specs.json \
  --gateway-provider=name=staging,provider-id=<hex>,base-url=https://gw-stage.example/,stream-token=<base64> \
  --gateway-manifest-id=<manifest_id_hex> \
  --gateway-chunker-handle=sorafs.sf1@1.0.0 \
  --json-out=reports/staging_gateway.json
```- Локальный CI больше не требует реестра `signer.json`.
  `ci/check_sorafs_fixtures.sh` сравнивает состояние репозитория с последней цепочкой
  приверженность и падает при расхождениях.

## Замечания по управлению

- Конституция парламента определяет кворум, ротацию и эскалацию — вариативную
  на уровне ящик не нужен.
- Экстренный откат происходит через панель модерации парламента. Панель
  Инфраструктура предлагает возврат предложения со ссылкой на предыдущий манифест дайджеста,
  и релиз заменяется после заявления.
- Исторические определения в реестре SoraFS для воспроизведения судебно-медицинской экспертизы.

## Часто задаваемые вопросы

- **Куда делся `signer.json`?**  
  Он удален. Все авторства подписей хранят в сети; `manifest_signatures.json`
  в репозитории — это лишь приспособление разработчика, которое должно совпасть с последним
  заявление о событии.

- **Нужны ли еще локальные подарки Ed25519?**  
  Нет. Утверждение парламента хранится в виде сетевых ресурсов. Локальные светильники
  нужны для воспроизводимости, но проверяются по дайджесту парламента.

- **Как команда контролирует положения?**  
  Напишите о событии `ParliamentFixtureApproved` или запросите реестр через
  Nexus RPC, чтобы получить текущий манифест дайджеста и список панели.

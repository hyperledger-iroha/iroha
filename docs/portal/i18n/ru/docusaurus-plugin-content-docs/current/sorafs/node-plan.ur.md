SF-3 содержит исполняемый ящик `sorafs-node`, который используется для процесса Iroha/Torii и поставщика хранилища SoraFS. بدلتا ہے۔ اس پلان کو [руководство по хранилищу узлов] (node-storage.md), [политика допуска поставщиков] (provider-admission-policy.md) اور [дорожная карта рынка емкости хранения] (storage-capacity-marketplace.md) کے ساتھ استعمال Последовательность результатов и результатов

## Целевой объем (этап M1)

1. **Интеграция хранилища фрагментов.** `sorafs_car::ChunkStore` обеспечивает постоянную серверную часть, обертку, настроенный каталог данных, байты фрагментов, манифесты и деревья PoR.
2. **Конечные точки шлюза.** Процесс Torii обеспечивает отправку контактов, выборку фрагментов, выборку PoR и телеметрию хранилища, а также Norito Конечные точки HTTP предоставляют доступ к данным.
3. **Строительство конфигурации.** `SoraFsStorage` структура конфигурации (включенный флаг, емкость, каталоги, ограничения одновременного использования). کے ذریعے провод کریں۔
4. **Квота/расписание.** Определяемые оператором ограничения на диск/параллелизм обеспечивают максимальное количество запросов и противодавление в очереди.
5. **Телеметрия.** Успешный вывод данных, задержка выборки фрагментов, использование емкости, результаты выборки PoR и метрики/журналы, выдающие данные.

### D. Планировщик и соблюдение квот

| Задача | Подробности |
|------|---------|
| Дисковая квота | Байты на диске трек کریں؛ `max_capacity_bytes` پر نئے контакты отклоняются. Политика выселения и крючки для выселения. |
| Получить параллелизм | Глобальный семафор (`max_parallel_fetches`) Определяет бюджеты каждого поставщика и ограничения диапазона SF-2d. |
| Закрепить очередь | Выдающиеся работы по приему пищи محدود کریں؛ глубина очереди کے لیے Norito конечные точки состояния выставляют کریں۔ |
| Каденция PoR | `por_sample_interval_secs` سے چلنے والا фоновый работник۔ |

## Зависимости

- Политика допуска SF-2b — узлы, где публикуются объявления, конверты для приема, подтверждающие наличие возможности
- Рынок мощности SF-2c — телеметрия и декларации мощности, а также связь
- Расширения рекламы SF-2d — возможности диапазона + бюджеты потоков.

## Критерии выхода из этапа

- `cargo run -p sorafs_node --example pin_fetch` местные светильники.
- Torii `--features sorafs-storage` позволяет построить интеграционные тесты и выполнить интеграционные тесты.
- Документация ([руководство по хранилищу узлов](node-storage.md)) настройки по умолчанию + примеры CLI обновлено ہو؛ Runbook оператора دستیاب ہو۔
- Панели мониторинга промежуточной телеметрии. насыщение емкости, сбои PoR и оповещения, настройка.

## Реклама поставщика при проверке

Поставщики SoraFS Полезные нагрузки `ProviderAdvertV1` (دیکھیں
`crates/sorafs_manifest::provider_advert`) сплетни کرتے ہیں جو управляемый оператор
نے знак کیے ہوتے ہیں۔ метаданные открытия рекламы اور ограждения کو pin کرتے ہیں
Среда выполнения оркестратора с несколькими источниками

- **Время жизни** — `issued_at < expires_at ≤ issued_at + 86,400 s`. поставщики услуг
  12 минут обновления.
- **Capability TLVs** — транспортные функции списка TLV рекламируют کرتی ہے (Torii,
  QUIC+Noise, реле SoraNet, расширения поставщиков). неизвестные коды کو
  `allow_unknown_capabilities = true` пропустите инструкции по использованию смазки.
- **Подсказки по качеству обслуживания** — уровень `availability` (горячий/теплый/холодный), максимальная задержка получения данных,
  предел параллелизма и необязательный бюджет потока. Качество обслуживания и наблюдаемая телеметрия
  مطابق ہونا چاہیے اور میں Audit کیا جاتا ہے۔
- **Конечные точки для тем встреч** — метаданные TLS/ALPN для конкретных целей
  URL-адреса сервисов и темы обнаружения для клиентов и наборы защиты и подписка на подписку
  کرنا چاہیے۔
- **Политика разнообразия путей** — `min_guard_weight`, заглушки для разветвления AS/пула,
  `provider_failure_threshold` детерминированная многоранговая выборка.
- **Идентификаторы профилей** — поставщики, предоставляющие канонический дескриптор, предоставляют доступ к информации.
  (код `sorafs.sf1@1.0.0`); дополнительные клиенты `profile_aliases`
  миграция میں مدد دیتے ہیں۔

Правила проверки нулевая ставка, пустые списки возможностей/конечных точек/тем, неправильный порядок
время жизни, или отсутствуют целевые значения QoS, или отклонить запрос, или реклама на входных конвертах
Тела предложений (`compare_core_fields`) Сравнить или обновить сплетни
کرتے ہیں۔

### Расширения выборки диапазона

Поставщики с поддержкой диапазона и дополнительные метаданные:

| Поле | Цель |
|-------|---------|
| `CapabilityType::ChunkRangeFetch` | `max_chunk_span`, `min_granularity` — флаги выравнивания/проверки объявляют کرتا ہے۔ |
| `StreamBudgetV1` | дополнительный конверт параллелизма/пропускной способности (`max_in_flight`, `max_bytes_per_sec`, дополнительный `burst`). требуемая дальность действия ہے۔ |
| `TransportHintV1` | заказанные транспортные настройки (مثلاً `torii_http_range`, `quic_stream`, `soranet_relay`). приоритеты `0–15` ہیں اور отклонить дубликаты ہوتے ہیں۔ |

Инструментальная поддержка:

- Конвейеры рекламы провайдера, включая возможности диапазона, бюджет потока, а также подсказки по транспорту.
  проверка достоверности или аудита, детерминированные полезные нагрузки, выбросы,
- `cargo xtask sorafs-admission-fixtures` каноническая реклама из нескольких источников.
  светильники понижения версии کے ساتھ `fixtures/sorafs_manifest/provider_admission/` میں Bundle کرتا ہے۔
- Рекламные объявления с поддержкой диапазона `stream_budget` или `transport_hints` без CLI/SDK.
  загрузчики انہیں планирование سے پہلے отклонить کرتے ہیں تاکہ жгут с несколькими источниками
  Torii ожидания приема کے ساتھ согласованы رہے۔

## Конечные точки диапазона шлюзаШлюзы принимают детерминированные HTTP-запросы, а также метаданные рекламы и зеркало.
کرتی ہیں۔

### `GET /v2/sorafs/storage/car/{manifest_id}`

| Требование | Подробности |
|-------------|---------|
| **Заголовки** | `Range` (одно окно, выровненное по смещениям фрагментов), `dag-scope: block`, `X-SoraFS-Chunker`, дополнительный `X-SoraFS-Nonce`, поддержка base64 `X-SoraFS-Stream-Token`. |
| **Ответы** | `206` с `Content-Type: application/vnd.ipld.car`, `Content-Range` в обслуживаемом окне, а также метаданные `X-Sora-Chunk-Range`, заголовки чанка/токена и echo. |
| **Режимы отказа** | невыровненные диапазоны — `416`, отсутствующие/недействительные токены — `401`, бюджеты потоков/байтов превышают `429`. |

### `GET /v2/sorafs/storage/chunk/{manifest_id}/{digest}`

Одночастотная выборка заголовков и детерминированный дайджест фрагментов. повторные попытки
Загрузки для судебно-медицинской экспертизы и фрагменты CAR.

## Рабочий процесс оркестратора с несколькими источниками

Включена выборка из нескольких источников SF-6 (Rust CLI через `sorafs_fetch`, SDK через
`sorafs_orchestrator`):

1. **Сбор входных данных** – декодирование плана фрагмента манифеста, получение последних рекламных объявлений.
   Дополнительный снимок телеметрии (`--telemetry-json` или `TelemetrySnapshot`).
2. **Создайте табло** — оценка соответствия требованиям `Orchestrator::build_scoreboard`.
   کرتا ہے اور причины отклонения записи کرتا ہے؛ `sorafs_fetch --scoreboard-out`
   JSON сохраняется.
3. **Часки расписания** — ограничения диапазона `fetch_with_scoreboard` (یا `--plan`).
   бюджеты потоков, ограничения повторных попыток и одноранговых узлов (`--retry-budget`, `--max-peers`)
   Запрос или выброс токена потока в области манифеста
4. **Проверка квитанций** — выходные данные `chunk_receipts` или `provider_reports` شامل.
   ہوتے ہیں؛ Сводка CLI `provider_reports`, `chunk_receipts`, اور
   `ineligible_providers` Какие пакеты доказательств сохраняются или сохраняются?

Операторы/SDK могут столкнуться с ошибками:

| Ошибка | Описание |
|-------|-------------|
| `no providers were supplied` | фильтрация کے بعد کوئی подходящая запись نہیں۔ |
| `no compatible providers available for chunk {index}` | Большой фрагмент или диапазон несоответствия бюджета. |
| `retry budget exhausted after {attempts}` | `--retry-budget` Как удалить неудавшиеся одноранговые узлы? |
| `no healthy providers remaining` | повторяющиеся сбои کے بعد تمام провайдеры отключают ہو گئے۔ |
| `streaming observer failed` | нисходящий поток записи CAR прерывается ہو گیا۔ |
| `orchestrator invariant violated` | сортировка, манифест, табло, снимок телеметрии, захват CLI в формате JSON. |

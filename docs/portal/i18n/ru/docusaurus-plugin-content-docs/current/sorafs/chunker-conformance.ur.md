## Канонический профиль

- Входное начальное число (шестнадцатеричное): `0000000000dec0ded`
- Целевой размер: 262144 байта (256 КиБ).
- Минимальный размер: 65536 байт (64 КиБ).
- Максимальный размер: 524288 байт (512 КиБ).
- Скользящий полином: `0x3DA3358B4DC173`
- Семя таблицы шестерен: `sorafs-v1-gear`
- Маска разрыва: `0x0000FFFF`

Эталонная реализация: `sorafs_chunker::chunk_bytes_with_digests_profile`.
Ускорение SIMD и границы и дайджесты

## Комплект приспособлений

`cargo run --locked -p sorafs_chunker --bin export_vectors` приспособления для регенерации
Если вы хотите получить доступ к `fixtures/sorafs_chunker/`, выполните следующие действия:

- `sf1_profile_v1.{json,rs,ts,go}` — Rust, TypeScript, пользователи Go могут использовать канонические границы блоков. ہر فائل
  آتے ہیں (مثلاً `sorafs.sf1@1.0.0`, پھر `sorafs.sf1@1.0.0`)۔ یہ заказ
  `ensure_charter_compliance` کے ذریعے принудить ہوتی ہے اور اسے تبدیل نہیں کیا جا سکتا۔
- `manifest_blake3.json` — манифест, проверенный BLAKE3, с установленным приспособлением и обложкой کرتا ہے۔
- `manifest_signatures.json` — дайджест манифеста и подписи совета (Ed25519)۔
- `sf1_profile_v1_backpressure.json` или `fuzz/` для необработанных корпусов —
  детерминированные сценарии потоковой передачи и тесты противодавления чанкера.

### Политика подписи

Регенерация приспособления **لازم** طور پر действительная подпись совета شامل کرے۔ генератор беззнакового вывода, чтобы отклонить запрос `--allow-unsigned` کے لیے)۔ Конверты с подписями только для добавления.

Подпись Совета شامل کرنے کے لیے:

```bash
cargo run --locked -p sorafs_chunker --bin export_vectors \
  --signing-key=<ed25519-private-key-hex> \
  --signature-out=fixtures/sorafs_chunker/manifest_signatures.json
```

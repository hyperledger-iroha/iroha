Это руководство кодифицирует потребности, которые необходимо реализовать, чтобы все остальное

## Канонический профиль

- Начальное число (шестнадцатеричное): `0000000000dec0ded`
- Кабель Taille: 262144 байт (256 КиБ)
- Минимум Taille: 65536 байт (64 КиБ)
- Максимум Taille: 524288 байт (512 КиБ)
- Полином прокатки: `0x3DA3358B4DC173`
- Семя таблицы передач: `sorafs-v1-gear`
- Маска разрыва: `0x0000FFFF`

Реализация ссылки: `sorafs_chunker::chunk_bytes_with_digests_profile`.
Всестороннее ускорение SIMD позволяет создавать лимиты и идентификаторы.

## Комплект светильников

`cargo run --locked -p sorafs_chunker --bin export_vectors` восстановить файлы
светильники и другие фичи, соответствующие sous `fixtures/sorafs_chunker/`:

- `sf1_profile_v1.{json,rs,ts,go}` — ограничения на канонические фрагменты для файлов.
  любители Rust, TypeScript и др. Chaque fichier annonce le handle canonique
  `sorafs.sf1@1.0.0`, puis `sorafs.sf1@1.0.0`). L'Ordre est imposé par
  `ensure_charter_compliance` и НЕ ДОЛЖНЫ быть изменены.
- `manifest_blake3.json` — манифест проверенной версии BLAKE3 соответствует каждому описанию светильников.
- `manifest_signatures.json` — подписи совета (Ed25519) к дайджесту манифеста.
- `sf1_profile_v1_backpressure.json` и тела брутты в `fuzz/` —
  детерминированные сценарии потоковой передачи используются для тестов противодавления чанкера.

### Политика подписи

Обновление светильников **doit** включает в себя действительную подпись. Генератор
rejette la sortie не подписан, если `--allow-unsigned` устарел (предыдущий
уникальность для региона проведения экспериментов). Конверты для подписи только для добавления и т. д.
sont dédupliquées par Signataire.

Добавьте подпись совета:

```bash
cargo run --locked -p sorafs_chunker --bin export_vectors \
  --signing-key=<ed25519-private-key-hex> \
  --signature-out=fixtures/sorafs_chunker/manifest_signatures.json
```

## Проверка

Помощник CI `ci/check_sorafs_fixtures.sh` обрадует генератор с помощью
`--locked`. Если светильники расходятся или подписи ошибочны, работа повторяется. Утилисез
Этот сценарий в ночных рабочих процессах и перед некоторыми изменениями в светильниках.

Этапы ручной проверки:

1. Exécutez `cargo test -p sorafs_chunker`.
2. Локаль Lancez `ci/check_sorafs_fixtures.sh`.
3. Подтвердите, что `git status -- fixtures/sorafs_chunker` соответствует требованиям.

## Playbook de mise à niveau

Лорскуон предлагает новый фрагмент профиля или тот, который встречается в SF1:

Еще раз: [`docs/source/sorafs/chunker_profile_authoring.md`](./chunker-profile-authoring.md) для файлов
потребности метадонников, шаблоны предложений и контрольные списки проверки.1. Редактируйте `ChunkProfileUpgradeProposalV1` (голос RFC SF-1) с новыми параметрами.
2. Обновите светильники через `export_vectors` и отправьте новый дайджест манифеста.
3. Подпишите манифест с необходимым кворумом. Все подписи сделаны еще
   приложения à `manifest_signatures.json`.
4. Просмотрите все необходимые SDK-устройства (Rust/Go/TS) и обеспечьте равенство между средами выполнения.
5. Выполните регулировку состояния корпусов при изменении параметров.
6. Откройте для себя руководство с новой ручкой профиля, семенами и дайджестом.
7. Продумайте изменения с помощью тестов и мероприятий в рамках дорожной карты.

Изменения, которые влияют на ограничения фрагментов или дайджесты без необходимости выполнения этого процесса.
они инвалиды и не делают ничего, кроме слияния.

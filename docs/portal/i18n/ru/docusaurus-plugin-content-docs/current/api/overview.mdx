# Torii Обзор API

Шлюз Torii предоставляет поверхность REST + WebSocket для взаимодействия с Iroha.
узлы. Канонический документ OpenAPI создается с помощью:

```bash
cargo xtask openapi
```

Выполнение приведенной выше команды записывает `docs/portal/static/openapi/torii.json` и
обеспечивает интеграцию `docusaurus-plugin-openapi-docs`, поэтому этот раздел обновляется.
автоматически. Генератор запускает маршрутизатор Torii в памяти и пытается
запросить его конечную точку OpenAPI; если это не помогло (например, инициализация маршрутизатора
ошибки) вместо этого выдается детерминированная спецификация-заполнитель.

## Текущий статус

- **Источник спецификации:** `cargo xtask openapi` напрямую запрашивает маршрутизатор Torii; тот
  Резервная заглушка генерируется только тогда, когда маршрутизатор не может предоставить OpenAPI
  документ.
– **Детерминизм.** Выходные данные JSON канонизируются для обеспечения стабильного сравнения в CI.
- **Аутентификация.** Песочница «Попробуй» теперь перенаправляет запросы через
  ограниченный промежуточный прокси. Операторы должны предоставить конечную точку Torii и носитель.
  токен перед выполнением вызовов.
- **Выбор версии:** Swagger UI, RapiDoc и полная страница Torii OpenAPI.
  откройте раскрывающийся список с надписью `openapi/versions.json`, чтобы вы могли сравнить
  исторические снимки. Опция `Latest (tracked)` указывает на
  `/openapi/torii.json`, а все остальные записи загружают соответствующие
  Артефакт `openapi/versions/<version>/torii.json`.
- **Рабочий процесс моментального снимка:** После запуска `npm run docs:version -- <label>` обновите
  соответствующий файл OpenAPI с `npm run sync-openapi -- --version=<label> --latest`
  поэтому раскрывающийся список может загружать как замороженный снимок, так и скользящую «Последнюю» сборку.

Экстрактор на базе маршрутизатора теперь излучает всю поверхность Torii. Если вы видите
заглушку-заполнитель, рассматривайте это как сбой во время сборки и проверьте Torii
журналы запуска роутера.

## Попробуйте песочницу (DOCS-3)

Портал разработчика включает небольшую консоль, которая передает запросы REST через
Узловой прокси. Прокси обеспечивает соблюдение CORS, ограничений скорости и дополнительного носителя.
токены, чтобы виджет мог взаимодействовать с промежуточными шлюзами, не раскрывая учетные данные.
Полный контрольный список оператора см. в `docs/devportal/try-it.md`.

Запустите прокси рядом с сервером разработки Docusaurus:

```bash
cd docs/portal
# Proxy configuration
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"
# Front-end configuration so the widget knows where the proxy lives
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"

# Run the proxy and the docs site (parallel terminals recommended)
npm run tryit-proxy
npm run start
```

- Прокси прослушивает `TRYIT_PROXY_LISTEN` (по умолчанию `127.0.0.1:8787`) и
  пересылает любые запросы `/proxy/...` на `TRYIT_PROXY_TARGET`.
- Предоставьте `TRYIT_PROXY_BEARER` для статических токенов на предъявителя или позвольте вызывающим сторонам предоставить
  один на каждый запрос через поле «Токен на предъявителя».
- Скорость ограничена по умолчанию до 60 запросов в минуту и может быть настроена с помощью
  `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS`.

> ⚠️ Прокси предназначен для тестового/демо-доступа. Производственные развертывания
> следует использовать усиленный вход и повторно использовать ту же политику ограничения скорости в
> передняя часть Torii.

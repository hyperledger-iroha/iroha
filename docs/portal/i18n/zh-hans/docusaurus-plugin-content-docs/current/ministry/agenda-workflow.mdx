## 概述

路线图项目 **MINFO-2** 需要从提案 linting 到
公开抽签证据，以便任何人都可以确认案件如何进入议程
理事会。此页面将先前登陆的 CLI 帮助程序直接连接到
开发者门户，以便提案者、审计员和平台团队可以遵循相同的要求
清单，无需深入探究单一存储库。

该工作流程涵盖三个可重复的步骤：

1. 在本地验证 `AgendaProposalV1` 有效负载并拒绝重复的哈希值
   在提交到达审核队列之前。
2. 与提案一起发布重复注册点击，以便理事会工作人员可以
   在不失去决定论的情况下追溯早期案例。
3. 运行抽签助手，捕获 Merkle 证明包，并显示
   在门户内选择名册，以便公民可以重播抽签。

所有助手住在`cargo xtask ministry-agenda`； Norito 架构是
记录在 `docs/source/ministry/agenda_council_proposal.md` 下。

## 1. 在本地验证提案

使用 CLI 来检查 JSON 有效负载并断言所需的证据（哈希
家庭、摘要长度、语言标签、提交者联系方式）存在。的
验证器与 Torii 摄取路径共享代码，因此提案通过
通过门户上传时这里也会通过。

{`cargo xtask Ministry-agenda validate \\
  --proposal docs/examples/ministry/agenda_proposal_example.json \\
  --registry docs/examples/ministry/agenda_duplicate_registry.json`}

- 在对历史有效负载进行分类时添加 `--allow-registry-conflicts`；新的
  当指纹已经存在时，提交应该会快速失败。
- 示例目录包含参考提案和注册表，以便 SDK 可以
  采用没有硬编码私有有效负载的格式。

## 2. 发布重复注册表上下文

每个接受的提案都包含之前咨询过的注册表摘要
摄入。捕获该注册表的 SoraFS CID 或 Torii 清单路径并添加
将其添加到您提交的元数据中，以便委员会审阅者可以验证相同的 JSON。

```json title="Duplicate registry (excerpt)"
{
  "entries": [
    {
      "hash_family": "blake3-256",
      "hash_hex": "0d714bed4b7c63c23a2cf8ee9ce6c3cde1007907c427b4a0754e8ad31c91338d",
      "proposal_id": "AC-2025-014",
      "note": "Already handled in 2025-08 incident"
    }
  ]
}
```当 CLI 报告冲突时，门户会显示链接的提案 ID，以便
操作员可以确定性地合并或拒绝重新提交。

## 3. 运行并发布抽签摘要

待办事项准备就绪后，运行确定性抽签助手。它消耗
名册 JSON、种子和目标槽计数，然后发出 Merkle-proof-backed
总结。摘要 JSON 是门户在 **理事会抽签** 中呈现的内容
面板。

{`cargo xtask Ministry-agenda sortition \\
  --roster docs/examples/ministry/agenda_council_roster.json \\
  --插槽 3 \\
  --种子0x0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef \\
  --output artifacts/ministry/sortition/2026-03-12.json`}

将生成的 JSON 存储在 SoraFS 上（或与签名的议程数据包一起）并
链接提案跟踪器中的 CID。开发者门户消耗相同的
JSON结构；下面的示例呈现了捆绑的
`agenda_sortition_summary_example.json`。

### 理事会抽签示例


  
    
      字段
      值
    
  
  
    
      算法
      {summary.algorithm}
    
    
      名册摘要（BLAKE3）
      {summary.roster_digest.blake3_hex}
    
    
      名册摘要（SHA-256）
      {summary.roster_digest.sha256_hex}
    
    
      种子
      {summary.seed_hex}
    
    
      老虎机
      {summary.slots}
    
    
      符合资格的会员
      {summary.eligible_members}
    
    
      默克尔根
      {summary.merkle_root_hex}
    
  




每个选定的成员包括：

- `draw_entropy_hex`（32 BLAKE3 字节），因此任何人都可以重新计算 RNG 输出。
- `eligible_index` 和 `original_index`，以便审核员可以证明选择退出是
  很荣幸。
- 覆盖名册页的 Merkle 证明；复制树证实了
  数据包内公布的名单与抽签相符。

### 决策日志

当理事会对提案进行表决时，请附上决定备忘录和
治理操作的抽签摘要 CID（Torii `/v2/gov/*` 条目）。门户网站
页面链接了备忘录、投票统计和上面显示的 JSON，以便每个操作
包含 **MINFO-2** 所需的完整证据链。

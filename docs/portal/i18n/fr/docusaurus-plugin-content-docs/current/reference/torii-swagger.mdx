# API Torii (interface utilisateur Swagger)

Cette page intègre le document Torii OpenAPI en utilisant `swagger-ui-react`. Utilisez-le quand
vous voulez la mise en page familière de Swagger, demander des échantillons et la possibilité d'essayer
envoyer des requêtes directement dans le navigateur.

## Conseils d'utilisation

- Fournir un jeton au porteur dans le panneau pour pré-autoriser chaque demande. Les jetons sont
  transmis au proxy à l’aide de l’en-tête `X-TryIt-Auth` et ne sont jamais conservés.
- Configurez `TRYIT_PROXY_PUBLIC_URL` lors de la desserte du portail afin que le widget
  transmet les demandes via le proxy intermédiaire fourni. Une bannière d'avertissement apparaît
  si le proxy est désactivé.
- Choisissez un instantané OpenAPI dans la liste déroulante pour basculer entre les dernières spécifications
  et versions archivées générées par `npm run sync-openapi`.
- Essayez la console « [Try it](/api/overview#try-it-sandbox-docs-3) » sauvegardée par proxy
  pour les environnements par étapes où CORS empêche l'accès direct à Torii.
- Conservez l'instantané OpenAPI à jour avec `cargo xtask openapi` (voir
  `docs/portal/static/openapi/torii.json`).

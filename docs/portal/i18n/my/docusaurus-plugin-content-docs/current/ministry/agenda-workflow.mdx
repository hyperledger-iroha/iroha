## ခြုံငုံသုံးသပ်ချက်

လမ်းပြမြေပုံပါ အကြောင်းအရာ **MINFO-2** သည် အဆိုပြုချက်မှ အလင်းတန်းအထိ အဆုံးအဖြတ်ပေးသည့် လမ်းကြောင်းတစ်ခု လိုအပ်သည်။
အမှုကို မည်ကဲ့သို့ လုပ်ငန်းစဉ်သို့ ရောက်ရှိကြောင်း မည်သူမဆို အတည်ပြုနိုင်စေရန် အများသူငှာ ခွဲခြားခြင်းဆိုင်ရာ အထောက်အထားများ
ကောင်စီ ဤစာမျက်နှာသည် ယခင်က ဆင်းသက်လာသော CLI အကူအညီပေးသူများကို တိုက်ရိုက်ဆက်သွယ်ပေးသည်။
developer portal သည် အဆိုပြုသူများ၊ စာရင်းစစ်များနှင့် ပလက်ဖောင်းအဖွဲ့များသည် တူညီသောအတိုင်း လုပ်ဆောင်နိုင်သည်။
mono-repo ကို မဖော်ပြဘဲ စစ်ဆေးရန်စာရင်း။

အလုပ်အသွားအလာတွင် ထပ်ခါတလဲလဲ လုပ်ဆောင်နိုင်သော အဆင့်သုံးဆင့် ပါဝင်သည်-

1. `AgendaProposalV1` ပေးချေမှုများအား စက်တွင်းတွင် အတည်ပြုပြီး ထပ်နေသော hashe များကို ငြင်းပယ်ပါ
   တင်သွင်းမှု စိစစ်ရေး တန်းစီမရောက်ရှိမီ။
2. ကောင်စီဝန်ထမ်းများ တတ်နိုင်သရွေ့ အဆိုပြုချက်နှင့်အတူ မိတ္တူပွား-မှတ်ပုံတင်ခြင်းများကို ထုတ်ဝေပါ။
   သတ်မှတ်ပြဋ္ဌာန်းမှု မပျက်စေဘဲ အစောပိုင်းဖြစ်ရပ်များကို ခြေရာခံပါ။
3. အမျိုးအစားခွဲရေးအကူကိုဖွင့်ပါ၊ Merkle အထောက်အထားအစုအဝေးကိုဖမ်းယူပြီး ၎င်းကိုမျက်နှာပြင်ပေါ်တင်ပါ။
   နိုင်ငံသားများသည် မဲနှိုက်ခြင်းကို ပြန်လည်ကစားနိုင်စေရန် ပေါ်တယ်အတွင်းတွင် ရွေးချယ်ထားသော စာရင်းဇယားများ။

အကူအညီပေးသူများအားလုံး `cargo xtask ministry-agenda` တွင်နေထိုင်သည်။ Norito schema သည်
`docs/source/ministry/agenda_council_proposal.md` အောက်တွင် မှတ်တမ်းတင်ထားသည်။

## 1. ဒေသအလိုက် အဆိုပြုချက်များကို အတည်ပြုပါ။

JSON payloads များကိုဖော်ပြရန်နှင့် လိုအပ်သောအထောက်အထားများ (hash
မိသားစုများ၊ အတိုကောက်အရှည်များ၊ ဘာသာစကားတက်ဂ်၊ တင်ပြသူ အဆက်အသွယ်) ရှိနေပါသည်။ ဟိ
validator သည် Torii သုံးစွဲမှုလမ်းကြောင်းနှင့် ကုဒ်ကို မျှဝေသည်၊ ထို့ကြောင့် ဖြတ်သန်းမည့် အဆိုပြုချက်
ပေါ်တယ်မှတဆင့် အပ်လုဒ်လုပ်သည့်အခါတွင်လည်း ဤနေရာမှ ဖြတ်သန်းသွားမည်ဖြစ်သည်။

<CodeBlock language="bash">{`ကုန်တင် xtask ဝန်ကြီးဌာန-အစီအစဉ်ကို အတည်ပြုသည် \\
  --proposal docs/example/ministry/agenda_proposal_example.json \\
  --registry docs/examples/ministry/agenda_duplicate_registry.json`}</CodeBlock>

- သမိုင်းဝင်ပေးဆောင်မှုများကို စမ်းသပ်သည့်အခါ `--allow-registry-conflicts` ကို ထည့်ပါ။ အသစ်
  လက်ဗွေရာ ရှိနှင့်ပြီးပါက တင်ပြချက်များ မြန်ဆန်စွာ ပျက်ကွက်သင့်ပါသည်။
- ဥပမာလမ်းညွှန်များသည် SDKs များလုပ်ဆောင်နိုင်စေရန် ရည်ညွှန်းအဆိုပြုချက်နှင့် မှတ်ပုံတင်ခြင်းကို ပို့ဆောင်ပေးသည်။
  hard-coding သီးသန့် payloads မပါဘဲ format ကိုလက်ခံပါ။

## 2. မိတ္တူပွား-မှတ်ပုံတင်ခြင်း အကြောင်းအရာကို ထုတ်ဝေပါ။

လက်ခံထားသော အဆိုပြုချက်တိုင်းတွင် ယခင်က ဆွေးနွေးခဲ့သော မှတ်ပုံတင်ခြင်း အနှစ်ချုပ် ပါဝင်ပါသည်။
စားသုံးမိခြင်း။ ထိုမှတ်ပုံတင်ခြင်းအတွက် SoraFS CID သို့မဟုတ် Torii မန်နီးဖက်စ်လမ်းကြောင်းကို ဖမ်းယူပြီး ထည့်ပါ
ကောင်စီမှ ပြန်လည်သုံးသပ်သူများသည် တူညီသော JSON ကို အတည်ပြုနိုင်စေရန် သင်၏တင်ပြမှု မက်တာဒေတာသို့ ၎င်းကို ပေးပို့ပါ။

```json title="Duplicate registry (excerpt)"
{
  "entries": [
    {
      "hash_family": "blake3-256",
      "hash_hex": "0d714bed4b7c63c23a2cf8ee9ce6c3cde1007907c427b4a0754e8ad31c91338d",
      "proposal_id": "AC-2025-014",
      "note": "Already handled in 2025-08 incident"
    }
  ]
}
```

CLI သည် ယာဉ်တိုက်မှုတစ်ခုအား အစီရင်ခံသောအခါ၊ ပေါ်တယ်သည် ချိတ်ဆက်ထားသော အဆိုပြုချက် IDs များကို ပြသသည်။
အော်ပရေတာများသည် ပြန်လည်တင်ပြချက်များကို တိကျစွာ ပေါင်းစပ်နိုင်သည် သို့မဟုတ် ငြင်းပယ်နိုင်သည်။

## 3. အမျိုးအစားအကျဉ်းချုပ်ကို လုပ်ဆောင်ပြီး ထုတ်ဝေပါ။

docket တစ်ခုအဆင်သင့်ဖြစ်ပါက၊ အဆုံးအဖြတ်ပေးသော အမျိုးအစားခွဲပေးသူကို ဖွင့်ပါ။ ၎င်းသည် စားသုံးသည်။
စာရင်းဇယား JSON၊ အမျိုးအနွယ်နှင့် ပစ်မှတ်အထိုင်အရေအတွက်၊ ထို့နောက် Merkle-proof-ကျောထောက်နောက်ခံပြုထားသော ထုတ်လွှတ်သည်
အနှစ်ချုပ်။ အကျဉ်းချုပ် JSON သည် **Council ဆွဲခြင်း** တွင် portal မှ ထုတ်ပေးသည်
ဘောင်။

<CodeBlock language="bash">{`ကုန်တင် xtask ဝန်ကြီးဌာန-အစီအစဉ် အမျိုးအစားခွဲခြင်း \\
  --roster docs/example/ministry/agenda_council_roster.json \\
  -- slot 3 \\
  --seed 0x0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef \\
  --output artifacts/ministry/sortition/2026-03-12.json`}</CodeBlock>

ရလဒ် JSON ကို SoraFS တွင် (သို့မဟုတ် လက်မှတ်ရေးထိုးထားသော အစီအစဉ်ထုပ်ပိုးမှုနှင့်အတူ) နှင့်
အဆိုပြုချက် ခြေရာခံစနစ်တွင် CID ကို ချိတ်ဆက်ပါ။ developer portal သည် အလားတူစားသုံးသည်။
JSON ဖွဲ့စည်းပုံ၊ အောက်ဖော်ပြပါ ဥပမာသည် အစုအဝေးကို ပြန်ဆိုသည်။
`agenda_sortition_summary_example.json`။

### ဆုံးဖြတ်ချက်မှတ်တမ်း

ကောင်စီက အဆိုကို မဲခွဲတဲ့အခါ ဆုံးဖြတ်ချက်မှတ်စုတိုနဲ့ ပူးတွဲပါ
အုပ်ချုပ်မှုလုပ်ဆောင်ချက်အတွက် အမျိုးအစားခွဲခြင်းအကျဉ်းချုပ် CID (Torii `/v1/gov/*` ထည့်သွင်းမှု)။ မုခ်ဦး
စာမျက်နှာများသည် မှတ်စုတို၊ မဲစာရင်းနှင့် အထက်တွင်ပြသထားသည့် JSON တို့ကို ချိတ်ဆက်ထားသောကြောင့် လုပ်ဆောင်ချက်တိုင်း
**MINFO-2** မှ လိုအပ်သော အထောက်အထား ကွင်းဆက် အပြည့်အစုံ ပါရှိသည်။

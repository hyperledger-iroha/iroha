## ခြုံငုံသုံးသပ်ချက်

စေတနာ့ဝန်ထမ်းအကျဉ်းချုပ်များသည် ပြန်ကြားရေးဝန်ကြီးဌာန၏ အမည်ပျက်စာရင်း သို့မဟုတ် မူဝါဒအပြောင်းအလဲများ အဆိုပြုသည့်အခါတိုင်းတွင် ပြန်လည်သုံးသပ်သည့် ထောက်ခံ/ဆန့်ကျင်/ ဆက်စပ်အမြင်များကို ဖမ်းယူထားသည်။ တင်ပြမှုတိုင်းသည် `iroha_data_model::ministry::VolunteerBriefV1` (ဗားရှင်း `1`) မှ အကောင်အထည်ဖော်သည့် canonical schema ကို လိုက်နာရပါမည်-

1. အုပ်ချုပ်မှု ဒက်ရှ်ဘုတ်များသည် အချက်အလက်ဇယားများကို အတိအကျ နှိုင်းယှဉ်နိုင်သည်။
2. အကျိုးစီးပွားဆိုင်ရာ ထုတ်ဖော်မှုများကို ထုတ်ပြန်ခြင်းမပြုမီ ပြဋ္ဌာန်းထားသည်။
3. အကြောင်းအရာမဟုတ်သော သို့မဟုတ် ထူးဆန်းသောတင်ပြချက်များကို စာရင်းစစ်လမ်းကြောင်းမှမဖယ်ရှားဘဲ ဖိနှိပ်ထားသည်။

payload တစ်ခုတည်ဆောက်ရန် `docs/source/ministry/volunteer_brief_template.md` သို့မဟုတ် `cargo xtask ministry-transparency ingest` တွင် နမူနာကို အသုံးပြုပါ။ CLI သည် Norito struct၊ lints ထိန်းညှိပေးသည့် တဂ်များကို အတည်ပြုပြီး အချက်အလက် အတန်းများ၊ ကိုးကားချက်များနှင့် ထုတ်ဖော်မှုများကို လက်ခံခဲ့သည်ကို ပြသသည့် အကျဉ်းချုပ်ကို ထုတ်လွှတ်သည်။

| ဘာသာပြန်ခြင်းနှင့် စိစစ်မှု | T+6 မှ 24 နာရီ | သုံးသပ်သူများသည် ထိန်းညှိပေးသည့် တဂ်များ (`needs-translation`၊ `duplicate`၊ `policy-escalation` စသည်ဖြင့်)၊ အဆက်အသွယ်များကို ပြန်လည်ပြင်ဆင်ပြီး အကြောင်းအရာမဟုတ်သော အကြောင်းအရာများကို အလံပြပါ။ Tag များကို Norito payload တွင် သိမ်းဆည်းထားသောကြောင့် downstream automation သည် entry များကို ဖိနှိပ်ခြင်း သို့မဟုတ် အသားပေးဖော်ပြနိုင်ပါသည်။ |
| Portal စာစောင် | T+24 မှ ​​48 နာရီ | **စေတနာ့ဝန်ထမ်း သဘောထားများ** အောက်တွင် သက်ဆိုင်ရာ အဆိုပြုချက် စာမျက်နှာနှင့်အတူ အကျုံးဝင်သော အတိုချုံးများကို ထုတ်ဝေပါသည်။ အကြောင်းအရာမဟုတ်သော ထည့်သွင်းမှုများကို အကြမ်းမှတ်တမ်းတွင် ကျန်ရှိနေသော်လည်း စုစည်းရပ်တည်မှုအရေအတွက်များမှ ဖယ်ထုတ်ထားသည်။ |

`policy-escalation` သို့မဟုတ် `astroturf` တဂ်များ ရှိနေသောအခါ အမြန်ကိုင်တွယ်ခြင်း (≤12နာရီ) အလိုအလျောက် အစပျိုးသည်။ အော်ပရေတာများသည် `submitted_at` နှင့် portal ထုတ်ဝေမှုကြားအချိန်ကို ခြေရာခံသည့် Volunteer Briefs Grafana panel မှတစ်ဆင့် လိုက်နာမှုကို စစ်ဆေးနိုင်သည်။

## ပို့ဆောင်ပေးပါသည်။

- **Schema & docs:** `VolunteerBriefV1` သည် မျှဝေထားသောဒေတာမော်ဒယ်တွင်နေထိုင်ပြီး အသေးစိတ်အကွက်လမ်းညွှန်အပြင် ဥပမာ payload သည် `docs/source/ministry/volunteer_brief_template.md` နှင့် `docs/examples/ministry/volunteer_brief_template.json` တွင်ရှိသည်။
- **Validation tooling:** `cargo xtask ministry-transparency volunteer-validate --input <file> [--json-output artifacts/ministry/volunteer-lint.json]` သည် portal build မှအသုံးပြုသော တူညီသော lints များကို လုပ်ဆောင်ပါသည်။ ထုတ်ဝေမှုအတွက် အစုအဝေးတစ်ခုလုံးကို အဆင့်မြှင့်တင်လိုသောအခါတွင် `ingest --volunteer <file>` ကို ဆက်လက်ရရှိနိုင်မည်ဖြစ်သည်။
- **Portal surfacing-** အဆိုပြုချက် စာမျက်နှာတစ်ခုစီသည် JSON ကြမ်းကို ဒေါင်းလုဒ်လုပ်စရာမလိုဘဲ နိုင်ငံသားထင်မြင်ယူဆချက်များကို မြင်နိုင်စေရန် ကုဒ်လုပ်ထားသော ပံ့ပိုးမှု/ဆန့်ကျင်မှု/ဆက်စပ်အကျဉ်းများဆီသို့ လင့်ခ်ချိတ်ထားသည်။ စိစစ်ရေး မက်တာဒေတာကို အတိုချုံးနှင့်အတူ ပြန်ဆိုထားသောကြောင့် ဝင်ရောက်မှု နှောင့်နှေးခြင်း သို့မဟုတ် ဖယ်ထုတ်ခြင်းဖြစ်နိုင်သည်ကို တရားသူကြီးများ နားလည်စေသည်။

လမ်းပြမြေပုံ အကြောင်းအရာ **MINFO-3 — စေတနာ့ဝန်ထမ်း စိတ်ဝင်စားမှု အကန့် အခြေခံအဆောက်အဦ** အပိုအလိုအလျောက်စနစ် (ဥပမာ၊ အစုလိုက်တင်သွင်းသည့် APIs သို့မဟုတ် တိုးချဲ့မှုပိုင်းခြားစိတ်ဖြာချက်) လိုအပ်ပါက လမ်းပြမြေပုံကို ကိုးကားပါ။

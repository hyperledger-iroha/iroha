ကျွန်ုပ်တို့သည် ဝေဖန်သုံးသပ်သူများကို ဖိတ်ခေါ်သည့်အခါတိုင်း လမ်းပြမြေပုံသည် အတည်ပြုနိုင်သော အထောက်အထားများကို တောင်းဆိုပါသည်။
docs portal ကို အစမ်းကြည့်ရှုခြင်း။ ဤမှတ်စုတွင် ထိုအထောက်အထားများကို မည်သို့ဖမ်းယူရမည်ကို ရှင်းပြထားသည်။
`preview-feedback-log` အကူအညီပေးသူ၊ ရရှိလာသော အနုပညာပစ္စည်းများကို မည်သို့မျှဝေမည်နည်း။
လှိုင်းတစ်ခုပိတ်ပြီးသည်နှင့် အုပ်ချုပ်မှုအား မည်သည့်နေရာတွင် ချေဖျက်မည်နည်း။

## အဘယ်ကြောင့်ကျွန်ုပ်တို့ တုံ့ပြန်ချက်ကို မှတ်တမ်းတင်ထားသနည်း။

- **လိုက်နာမှု-** ဖိတ်ကြားချက်တိုင်းသည် အစမ်းကြည့်ရှုမှုကို ဝင်ရောက်ကြည့်ရှုသည့်အခါ ၎င်းတို့ကို မှတ်တမ်းတင်ရပါမည်။
  လက်ခံနိုင်သော-အသုံးပြုမှုမူဝါဒကို အသိအမှတ်ပြုပြီး ၎င်းတို့၏အသုံးပြုခွင့်ကို ရုတ်သိမ်းလိုက်သည့်အခါ။
- **Telemetry ဆက်စပ်မှု-** မှတ်တမ်းသည် ကျွန်ုပ်တို့အား portal telemetry spikes များကို ဆက်စပ်ပေးသည်
  သို့မဟုတ် သတ်မှတ်ထားသော ဖိတ်ကြားမှု အစုအဖွဲ့များဖြင့် လမ်းကြောင်းပြောင်းကြည့်ပါ။
- **အုပ်ချုပ်မှုအထောက်အထား-** DOCS-SORA စစ်ဆေးရေးဂိတ်တစ်ခုစီတွင် စက်ဖြင့်ဖတ်နိုင်သော လိုအပ်သည်
  ဖိတ်ခေါ်မှုအရေအတွက်ကို ပြသသည့် အကျဉ်းချုပ်၊ ပွင့်လင်းစွာ သုံးသပ်သူများနှင့် ထူးထူးခြားခြား အခြေအနေ
  ကိစ္စများ။

အထောက်အထားအားလုံးသည် `artifacts/docs_portal_preview/` အောက်တွင် ရှိနေသောကြောင့် ၎င်းကို ပူးတွဲထားနိုင်သည်။
CI မှတ်တမ်းများကို မဖျက်ဘဲ ဖိတ်ခေါ်လှိုင်းလက်မှတ် သို့မဟုတ် အုပ်ချုပ်မှုပြန်လည်သုံးသပ်ခြင်းသို့။

## တိုက်ရိုက်ဖိတ်ကြားမှုအကျဉ်းချုပ်

`docs/portal/scripts/preview-feedback-summary.mjs` သည် မျှဝေထားသော မှတ်တမ်းကို ထည့်သွင်းသည်။
`npm run build` အတွင်း `src/data/previewFeedbackSummary.json` ရေးသည်။ စားပွဲ
အောက်တွင်ဖော်ပြထားသော JSON သည် သုံးသပ်သူများသည် လက်ရှိဖိတ်ကြားထားသည့်ခြေရာကို မြင်နိုင်စေရန်
CLI helper ကိုမလုပ်ဆောင်ဘဲ။

<တုံ့ပြန်ချက်အကျဉ်းချုပ်ဇယား />ဖိတ်ကြားထားသော စုစုပေါင်းများအပြင်၊ အနှစ်ချုပ်သည် ယခုတွင် အသိအမှတ်ပြုမှုနှင့် ပေါ်လာပါသည်။
ရုတ်သိမ်းမှု အရေအတွက်၊ ရရှိလာသော တုံ့ပြန်မှုနှုန်း (တုံ့ပြန်ချက် တင်ပြချက်များဖြင့် ပိုင်းခြားထားသည်။
စုစုပေါင်းဖိတ်ကြားမှုများ) နှင့် တစ်ခုကြားရှိ ပျမ်းမျှကြာချိန် (hoverable p95 ဖြင့်)
ဖိတ်ကြားချက်နှင့် သုံးသပ်သူ၏ အသိအမှတ်ပြုမှု သို့မဟုတ် အကြံပြုချက် တင်ပြချက်။ အသစ်
**ပြီးစီးမှု %** ကော်လံတွင် ဖိတ်ကြားချက်မည်မျှ ပိတ်ထားသည်ကို ပြသည် (အသိအမှတ်ပြုသည် သို့မဟုတ်
ရုတ်သိမ်းလိုက်သည်) **Ack %** သည် လက်ခံသူမည်မျှကို အတိအလင်း အတည်ပြုကြောင်း မီးမောင်းထိုးပြနေချိန်တွင်
လက်ခံနိုင်သောအဆင့်များ။ ဤရရှိသည့် ရာခိုင်နှုန်းများ—အပြင် JSON သီးသန့်
ပြန်လည်ရုပ်သိမ်းခြင်း/ပြဿနာနှုန်းများ—ပြန်လည်သုံးသပ်သူများသည် အတည်ပြုနိုင်စေရန် အုပ်ချုပ်မှုဒက်ရှ်ဘုတ်များကို ဖြည့်သွင်းပါ။
CLI အကူအညီပေးသူကို မလုပ်ဆောင်ဘဲ ပါဝင်မှုအဆင့်နှင့် အလှည့်အပြောင်းအချိန်နှစ်ခုလုံး။
**အခြေအနေ** ကော်လံသည် လှိုင်းငြိမ်နေခြင်းရှိမရှိ မီးမောင်းထိုးပြသည်။
တက်ကြွသော (`active`) ကို ပိတ်သော်လည်း 14 ရက်ကြာ စောင့်ကြည့်ရေးဝင်းဒိုး (`closed`) အတွင်း သို့မဟုတ်၊
(`archived`) အတွက် အဆင်သင့်။ **ပြီးခဲ့သော ပွဲစကတည်းက နေ့ရက်များ** မက်ထရစ်က မည်ကဲ့သို့ ပြသည်
ကောင်စီများသည် နောက်ဆုံး ဖိတ်ခေါ်ခြင်း/တုံ့ပြန်ချက် လုပ်ဆောင်ချက်ကြောင့် ကြာမြင့်နေပြီဖြစ်သည်။
ထုပ်ပိုးရန် လိုအပ်သော လှိုင်းများ သို့မဟုတ် ကြာရှည်စွာ လက်ခံသူများကို ပြန်လည် ထိတွေ့ဆက်ဆံပါ။

### Wave `preview-20260218` အတို

- **Scope-** Finance beta၊ observability ops၊ SDK ပါတနာ၊ နှင့် ecosystem advocate reviewers များကို လာမည့် DOCS-SORA ဂိတ်မတိုင်မီ **2026-02-18** တွင် ဖိတ်ကြားထားသည်။
- ** မက်ထရစ်များ-** ဖိတ်ခေါ်ချက် ၄ စောင်၊ အသိအမှတ်ပြုချက် ၄ ချက်၊ ပြန်လည်ရုပ်သိမ်းမှု ၄ ကြိမ်နှင့် တုံ့ပြန်ချက်တင်ပြချက် ၄ စောင် (`responseRatePercent = 100`)၊ စောင့်ကြည့်လေ့လာနိုင်မှုအစုအဖွဲ့မှ တင်ပြသော တစ်ခုတည်းသော နောက်ဆက်တွဲပြဿနာ (`DOCS-SORA-Preview-20260218`)။
- **အထောက်အထား-** `artifacts/docs_portal_preview/feedback_log.json` သည် ယခုလှိုင်းအတွက် အသေးစိတ်ဖြစ်ရပ်အချိန်ဇယားပါ၀င်ပြီး `artifacts/docs_portal_preview/preview-20260218-summary.json` သည် အုပ်ချုပ်ရေးပက်ကေ့ဂျ်များနှင့် ပူးတွဲပါရှိသော ပုံမှန်မက်ထရစ်များကို ဖမ်းယူပါသည်။ `npm run --prefix docs/portal preview:log -- --summary --summary-json` မှတစ်ဆင့် ပေါ်တယ်ဒေတာကို ပြန်လည်ထုတ်ပေးခြင်းသည် တူညီသောနံပါတ်များကို `src/data/previewFeedbackSummary.json` သို့ ဆွဲထုတ်သည်။
- **Wave helper-** `npm run --prefix docs/portal preview:wave -- --wave preview-20260218 --invite-start 2026-02-18 --invite-end 2026-02-28 --report-date 2026-03-01 --notes "Finance/observability beta wave"` သည် လှိုင်းတစ်ခုချင်းစီ၏ အနှစ်ချုပ်၊ အနှစ်ချုပ်၊ နှင့် ပြန်လည်ဆန်းသစ်ထားသော portal data ကို အဆင့်တစ်ဆင့်တွင် ထုတ်လွှတ်ပါသည်။
- **နောက်ဆက်တွဲများ-** ကြည့်ရှုနိုင်မှု တုံ့ပြန်ချက်သည် စမ်းသုံးကြည့်ရန် တောင်းဆိုမှုများအတွက် နောက်ထပ် CSP နမူနာများကို ကန့်သတ်ထားသည်။ Docs/DevRel သည် `docs/portal/docs/sorafs/observability-plan.md` (§4.2) အောက်တွင် ရေးထားသည်ကို ခြေရာခံနေသောကြောင့် နောက်လာမည့် ဖိတ်ကြားချက်လှိုင်းသည် ပြုပြင်မှုကို အတည်ပြုနိုင်သည်။ အတိုချုံး အပြည့်အစုံသည် [`preview-feedback/w3/summary.md`](./preview-feedback/w3/summary.md) တွင် [`preview-feedback/w3/log.md`](./preview-feedback/w3/log.md) တွင် ရိုက်ကူးထားသော timeline ဖြင့် နေထိုင်ပါသည်။

## အလုပ်အသွားအလာကို မှတ်တမ်းတင်ခြင်း။

`npm run preview:log` ၏ နောက်ကွယ်မှ ကူညီသူသည် ဖမ်းယူနိုင်သော JSON မှတ်တမ်းတစ်ခုအား ရေးသားသည်။
အဖြစ်အပျက်အားလုံး။ ထည့်သွင်းမှုတစ်ခုစီတွင် လှိုင်းတံဆိပ်၊ လက်ခံသူ၊ ဖြစ်ရပ်၊ အချိန်တံဆိပ်နှင့် ပါဝင်သည်။
ရွေးချယ်နိုင်သောမှတ်စုများ။ ဥပမာ-

```json
{
  "version": 1,
  "generated_at": "2026-02-04T18:20:15.321Z",
  "entries": [
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "invite-sent",
      "timestamp": "2026-02-04T18:05:02.114Z",
      "notes": "seed reviewer"
    },
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "feedback-submitted",
      "timestamp": "2026-02-05T09:14:43.009Z",
      "notes": "docs-preview/42"
    }
  ]
}
```

အမိန့်များ-

```bash
# Append a new event (defaults to artifacts/docs_portal_preview/feedback_log.json)
npm run --prefix docs/portal preview:log -- \
  --wave preview-20260204 \
  --recipient alice@example.com \
  --event invite-sent \
  --notes "seed reviewer"

# Produce a text summary for the current repo
npm run --prefix docs/portal preview:log -- --summary

# Emit machine-readable summary for a specific wave
npm run --prefix docs/portal preview:log -- \
  --summary --summary-json --filter-wave preview-20260204 \
  > artifacts/docs_portal_preview/preview-20260204-summary.json

# Regenerate the portal summary data file
npm run --prefix docs/portal generate:preview-summary

# Generate a Markdown digest skeleton for a wave
npm run --prefix docs/portal preview:digest -- \
  --wave preview-20260204 \
  --invite-start 2026-02-04T00:00:00Z \
  --invite-end 2026-02-18T00:00:00Z \
  --dashboard "https://grafana.example/d/docs-preview?var-wave=preview-20260204" \
  --out artifacts/docs_portal_preview/preview-20260204-digest.md
```

အကြံပြုချက်များ

- အခြားအင်္ဂါရပ်ဌာနခွဲတစ်ခုတွင်အလုပ်လုပ်သောအခါ လော့ဂ်ကို ဗားရှင်းထိန်းချုပ်မှုအောက်တွင် ထားပါ။
  ထိန်းသိမ်းသူများသည် အပြောင်းအလဲများကို ပြန်လည်သုံးသပ်နိုင်သည်။ ကြာရှည်သောလှိုင်းများအတွက် မှတ်တမ်းကို သိမ်းဆည်းပါ။
  အုပ်ချုပ်မှုလက်မှတ်နှင့် နောက်ဆုံးဗားရှင်းကို ထုတ်ဝေမှုဆိုင်ရာ စာရွက်စာတမ်းများတွင် ပူးတွဲပါရှိသည်။
- သင်တစ်ဦးအတွင်းသို့တိုက်ရိုက်ဝင်ရောက်ရန်လိုအပ်သောအခါ `preview:log` သည် `--log <path>` ကိုလက်ခံသည်
  SoraFS အတွဲ သို့မဟုတ် CI အလုပ်ခွင်။
- `--notes` အရိပ်အမြွက် (ပြဿနာလင့်ခ်၊ GitHub လက်ကိုင်၊ အစု ID) အမြဲပါ၀င်ပါ ။
  အနှစ်ချုပ်သည် နောက်ကြောင်းပြန်စဉ်အတွင်း အသုံးဝင်သည်။
- သင်သတ်မှတ်ထားသော အနှစ်ချုပ်အချိန်တံဆိပ်များ (CI သို့မဟုတ် docs လျှပ်တစ်ပြက်များအတွက်) လိုအပ်သောအခါ၊
  အနှစ်ချုပ်ကို မလုပ်ဆောင်မီ `PREVIEW_FEEDBACK_NOW=<iso8601 timestamp>` ကို သတ်မှတ်ပါ။
  ကူညီသူ။ ၎င်းသည် `generatedAt`၊ အခြေအနေနှင့် နေ့ကောင်တာများကို တသမတ်တည်းဖြစ်စေသည်။
  ဖြတ်ပြီး ပြန်ဆောက်တယ်။

## အနှစ်ချုပ်များ၊ အကျဉ်းချုပ်များနှင့် မှတ်တမ်းများ1. **တစ်လှိုင်းအနှစ်ချုပ်-** JSON အကျဉ်းချုပ်ကို ဖန်တီးပြီး ဖိတ်ကြားချက်တွင် ပူးတွဲပါ
   လက်မှတ်အကြမ်း မှတ်တမ်းနှင့်အတူ။ အနှစ်ချုပ်သည် စုစုပေါင်း ဖိတ်ကြားချက်များကို စာရင်းကောက်၍ ဖွင့်သည်။
   လက်ခံသူများနှင့် တုံ့ပြန်ချက်တို့ကို ရေတွက်ထားသောကြောင့် အုပ်ချုပ်ရေးက ကျွန်ုပ်တို့ဖမ်းယူထားသည်ကို အတည်ပြုနိုင်သည်။
   ဝေဖန်သုံးသပ်သူ။ portal build သည် စုစည်းထားသော အနှစ်ချုပ်ကို ထုတ်ဝေသည်။
   `src/data/previewFeedbackSummary.json` ဖြစ်သောကြောင့် သက်ဆိုင်သူများအနေဖြင့် လက်ရှိအခြေအနေကို ကြည့်ရှုနိုင်ပါသည်။
   CLI ကိုမလည်ပတ်ဘဲလျှပ်တစ်ပြက်ရိုက်ချက်။
2. **အပတ်စဉ် မှတ်တမ်း-** `npm run --prefix docs/portal preview:digest -- --wave <label>` ကို ဖွင့်ပါ။
   Markdown မက်ထရစ်များကို ကြိုတင်ဖြည့်သွင်းရန်၊ ထို့နောက် နေရာယူထားသည့် ကဏ္ဍများကို စိတ်ကြိုက်ပြင်ဆင်ပါ။
   ပုံစံခွက်ကို အသုံးပြု
   [`docs/examples/docs_preview_feedback_digest.md`](../../../examples/docs_preview_feedback_digest.md)
   သက်ဆိုင်သူများနှင့် မမျှဝေမီ။ အနည်းဆုံးပါဝင်ပါ- လှိုင်းတံဆိပ်၊ အဖွင့်
   သုံးသပ်သူများ၊ ထင်ရှားသော တွေ့ရှိချက်များနှင့် နောက်လုပ်ဆောင်မှုများ။
3. **Archival:** အပိတ်-လှိုင်းမှတ်တမ်းများကို ရွှေ့ပါ။
   `artifacts/docs_portal_preview/archive/<wave>/` နှင့် နောက်ဆုံးထွက်ကိုသာ သိမ်းဆည်းပါ။
   အမြစ်တွင် လှုပ်ရှားနေသော လှိုင်း။ ၎င်းသည် CI ပစ္စည်းများကို ထိန်းသိမ်းရာတွင် သေးငယ်သွားစေသည်။
   စာရင်းစစ်လမ်းကြောင်း။

## လမ်းပြမြေပုံကို ချိတ်ဆက်ခြင်း။

- သင်တောင်းဆိုသည့်အခါတိုင်း ဤစာတမ်းကို ဖိတ်ကြားခြင်းနှင့် စတင်ခြင်းဆိုင်ရာ ပလေးစာအုပ်များမှ ကိုးကားပါ။
  အကြံပြုချက်တင်ပြရန် သုံးသပ်သူများ (ကြည့်ရှုပါ။
  [`preview-invite-flow`](./preview-invite-flow.md)၊
  [`reviewer-onboarding`](./reviewer-onboarding.md) နှင့်
  [`public-preview-invite`](./public-preview-invite.md))။
- လှိုင်းကိုပိတ်သည့်အခါ `status.md` တွင် ထုတ်လုပ်ထားသော အနှစ်ချုပ်ကို ဖော်ပြပါ DOCS-SORA
  လမ်းပြမြေပုံပါ အရာများ (ဥပမာ၊ DOCS-1b/2b/3c) သည် artefact ကို ကိုးကားနိုင်ပါသည်။
- SoraFS သို့ ပြင်ပသို့ အပ်လုဒ်တင်သည့်အခါ အကျဉ်းချုပ်နှင့်အတူ hash digest များကို သိမ်းဆည်းပါ။
  သုံးသပ်သူများသည် အထောက်အထားအစုအဝေးကို စစ်ဆေးနိုင်သည်။

ဤနေရာတွင် သစ်ခုတ်ခြင်းလုပ်ငန်းအသွားအလာကို ကုဒ်လုပ်ထားခြင်းဖြင့် လမ်းပြမြေပုံလိုအပ်ချက်ကို ပြန်ဖွင့်ပေးသည်။

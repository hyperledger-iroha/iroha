# Torii API جائزہ

Torii گیٹ وے Iroha کے ساتھ بات چیت کرنے کے لئے ایک REST + ویب ساکٹ سطح کو بے نقاب کرتا ہے
نوڈس کیننیکل OpenAPI دستاویز کے ذریعے تیار کیا گیا ہے:

```bash
cargo xtask openapi
```

مذکورہ بالا کمانڈ چلانے سے `docs/portal/static/openapi/torii.json` اور لکھتا ہے
`docusaurus-plugin-openapi-docs` انضمام کو کھانا کھلاتا ہے تاکہ یہ سیکشن تازہ کاری کرے
خود بخود. جنریٹر ایک میموری Torii روٹر کو گھماتا ہے اور اس کی کوشش کرتا ہے
اس کے OpenAPI اختتامی نقطہ سے استفسار کریں ؛ اگر یہ ناکام ہوجاتا ہے (مثال کے طور پر ، روٹر ابتدا
غلطیاں) اس کے بجائے ایک تعی .ن پسند پلیس ہولڈر کی قیاس آرائی خارج ہوتی ہے۔

## موجودہ حیثیت

۔
  فال بیک اسٹب صرف اس وقت خارج ہوتا ہے جب روٹر OpenAPI کو بے نقاب کرنے میں ناکام ہوجاتا ہے
  دستاویز۔
- ** عزم: ** JSON آؤٹ پٹ CI میں مستحکم مختلف کو یقینی بنانے کے لئے کیننیکلائز کیا گیا ہے۔
- ** توثیق: ** "کوشش کریں" سینڈ باکس اب درخواستوں کو آگے بھیج دیتا ہے
  مجبوری اسٹیجنگ پراکسی۔ آپریٹرز کو Torii اختتامی نقطہ اور بیئرر فراہم کرنا ہوگا
  کالز جاری کرنے سے پہلے ٹوکن۔
- ** ورژن سلیکٹر: ** سویگر UI ، ریپڈوک ، اور مکمل Torii OpenAPI صفحہ
  ایک ڈراپ ڈاؤن کو بے نقاب کریں جو `openapi/versions.json` پڑھتا ہے تاکہ آپ موازنہ کرسکیں
  تاریخی سنیپ شاٹس۔ `Latest (tracked)` آپشن کی طرف اشارہ کرتا ہے
  `/openapi/torii.json` ، جبکہ دیگر تمام اندراجات اسی طرح کو لوڈ کرتی ہیں
  `openapi/versions/<version>/torii.json` نمونہ۔
- ** اسنیپ شاٹ ورک فلو: ** چلانے کے بعد `npm run docs:version -- <label>` ، ریفریش
  `npm run sync-openapi -- --version=<label> --latest` کے ساتھ ملاپ OpenAPI فائل
  لہذا ڈراپ ڈاؤن دونوں منجمد اسنیپ شاٹ اور رولنگ "تازہ ترین" تعمیر دونوں کو لوڈ کرسکتے ہیں۔

روٹر کی حمایت یافتہ ایکسٹریکٹر اب مکمل Torii سطح کا اخراج کرتا ہے۔ اگر آپ دیکھتے ہیں
پلیس ہولڈر اسٹب ، اسے بلڈ ٹائم کی ناکامی کے طور پر سلوک کریں اور Torii کا معائنہ کریں
روٹر اسٹارٹ اپ لاگ۔

## اسے سینڈ باکس (DOCS-3) آزمائیں

ڈویلپر پورٹل ایک چھوٹا سا کنسول سرایت کرتا ہے جو آرام کی درخواستوں کو ایک کے ذریعے جوڑتا ہے
نوڈ پر مبنی پراکسی۔ پراکسی CORs ، شرح کی حدود اور اختیاری بیئر کو نافذ کرتا ہے
ٹوکن تو ویجیٹ اسناد کو بے نقاب کیے بغیر گیٹ ویز کے اسٹیجنگ سے بات کرسکتا ہے۔
مکمل آپریٹر چیک لسٹ کے لئے `docs/devportal/try-it.md` دیکھیں۔

Docusaurus دیو سرور کے ساتھ ساتھ پراکسی شروع کریں:

```bash
cd docs/portal
# Proxy configuration
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"
# Front-end configuration so the widget knows where the proxy lives
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"

# Run the proxy and the docs site (parallel terminals recommended)
npm run tryit-proxy
npm run start
```

- پراکسی `TRYIT_PROXY_LISTEN` (پہلے سے طے شدہ `127.0.0.1:8787`) اور پر سنتا ہے
  کسی بھی `/proxy/...` درخواستوں کو `TRYIT_PROXY_TARGET` پر آگے بھیجتا ہے۔
- جامد بیئرر ٹوکن کے لئے `TRYIT_PROXY_BEARER` کی فراہمی کریں یا کال کرنے والوں کو فراہم کرنے دیں
  "بیئرر ٹوکن" فیلڈ کے ذریعہ ایک درخواست۔
- شرح ڈیفالٹ 60 درخواستوں تک فی منٹ کی حدود اور اس کے ذریعے ٹیون کی جاسکتی ہے
  `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS`۔

> ⚠ پراکسی کا مقصد اسٹیج/ڈیمو تک رسائی کے لئے ہے۔ پیداوار کی تعیناتی
> سخت انجری کے پیچھے بھاگنا چاہئے اور اسی شرح کو محدود کرنے والی پالیسی کو دوبارہ استعمال کرنا چاہئے
> Torii کے سامنے۔

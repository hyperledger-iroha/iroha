| Tadbir | Qachon yozib olish kerak | Eslatmalar |

Buyruqlar:

```bash
# Append a new event (defaults to artifacts/docs_portal_preview/feedback_log.json)
npm run --prefix docs/portal preview:log -- \
  --wave preview-20260204 \
  --recipient alice@example.com \
  --event invite-sent \
  --notes "seed reviewer"

# Produce a text summary for the current repo
npm run --prefix docs/portal preview:log -- --summary

# Emit machine-readable summary for a specific wave
npm run --prefix docs/portal preview:log -- \
  --summary --summary-json --filter-wave preview-20260204 \
  > artifacts/docs_portal_preview/preview-20260204-summary.json

# Regenerate the portal summary data file
npm run --prefix docs/portal generate:preview-summary

# Generate a Markdown digest skeleton for a wave
npm run --prefix docs/portal preview:digest -- \
  --wave preview-20260204 \
  --invite-start 2026-02-04T00:00:00Z \
  --invite-end 2026-02-18T00:00:00Z \
  --dashboard "https://grafana.example/d/docs-preview?var-wave=preview-20260204" \
  --out artifacts/docs_portal_preview/preview-20260204-digest.md
```

Maslahatlar:

- Xususiyatlar bo'limida ishlaganda jurnalni versiya nazorati ostida joylashtiring
  nazoratchilar o'zgarishlarni ko'rib chiqishlari mumkin. Uzoq davom etadigan to'lqinlar uchun jurnalni saqlang
  boshqaruv chiptasi va oxirgi versiyasini reliz artefaktlariga ilova qiling.
- `preview:log` to'g'ridan-to'g'ri tizimga kirishingiz kerak bo'lganda `--log <path>` ni qabul qiladi
  SoraFS to'plami yoki CI ish maydoni.
- Har doim `--notes` maslahatini qo'shing (muammo havolasi, GitHub tutqichi, kohort identifikatori)
  Xulosa retrospektiv davomida foydalidir.
- Sizga aniqlangan qisqacha vaqt belgilari kerak bo'lganda (CI yoki docs snapshotlari uchun),
  Xulosa ishga tushirishdan oldin `PREVIEW_FEEDBACK_NOW=<iso8601 timestamp>` ni o'rnating
  yordamchi. Bu `generatedAt`, holat va kun hisoblagichlarini izchil saqlaydi
  qayta qurish bo'ylab.

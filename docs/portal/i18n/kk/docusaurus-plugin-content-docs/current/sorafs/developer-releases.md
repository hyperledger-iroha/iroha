# Шығару процесі

SoraFS екілік файлдары (`sorafs_cli`, `sorafs_fetch`, көмекшілер) және SDK жәшіктері
(`sorafs_car`, `sorafs_manifest`, `sorafs_chunker`) бірге жеткізіледі. Шығарылым
конвейер CLI мен кітапханаларды теңестіреді, түкті/сынақты қамтуды қамтамасыз етеді және
төменгі тұтынушылар үшін артефактілерді түсіреді. Әрқайсысы үшін төмендегі бақылау тізімін орындаңыз
кандидат белгісі.

## 0. Қауіпсіздікті тексеруден шығуды растаңыз

Техникалық босату қақпасын орындамас бұрын, соңғы қауіпсіздік шолуын алыңыз
артефактілер:

- Ең соңғы SF-6 қауіпсіздік шолу жазбасын жүктеп алыңыз ([reports/sf6-security-review](./reports/sf6-security-review.md))
  және оның SHA256 хэшін шығару билетіне жазыңыз.
- Түзету билетінің сілтемесін тіркеңіз (мысалы, `governance/tickets/SF6-SR-2026.md`) және шығуды ескеріңіз
  Қауіпсіздік инженериясы және Құрал-саймандар жөніндегі жұмыс тобының мақұлдаушылары.
- Жаднамадағы қалпына келтіруді тексеру парағының жабық екенін тексеріңіз; шешілмеген элементтер шығарылымды блоктайды.
- Тепе-теңдік қосқыш журналдарын жүктеп салуға дайындалыңыз (`cargo test -p sorafs_car -- --nocapture sorafs_cli::proof_stream::bounded_channels`)
  манифест бумасының жанында.
- Іске қосуды жоспарлап отырған қол қою пәрмені `--identity-token-provider` және айқын пәрменді қамтитынын растаңыз.
  `--identity-token-audience=<aud>` сондықтан Fulcio ауқымы шығарылым дәлелдерінде жазылған.

Басқаруды хабардар ету және шығарылымды жариялау кезінде осы артефактілерді қосыңыз.

## 1. Шығару/сынақ қақпасын орындаңыз

`ci/check_sorafs_cli_release.sh` көмекшісі пішімдеу, Clippy және сынақтарды іске қосады
CLI және SDK жәшіктері бойынша жұмыс кеңістігінің жергілікті мақсатты каталогы (`.target`)
CI контейнерлерінің ішінде орындау кезінде рұқсат қайшылықтарын болдырмау үшін.

```bash
CARGO_TARGET_DIR=.target ci/check_sorafs_cli_release.sh
```

Сценарий келесі бекітулерді орындайды:

- `cargo fmt --all -- --check` (жұмыс кеңістігі)
- `cargo clippy --locked --all-targets` `sorafs_car` үшін (`cli` мүмкіндігімен),
  `sorafs_manifest`, және `sorafs_chunker`
- Дәл сол жәшіктер үшін `cargo test --locked --all-targets`

Кез келген қадам сәтсіз болса, тегтеу алдында регрессияны түзетіңіз. Шығарылым құрастырулары болуы керек
негізгімен үздіксіз; босату бұтақтарына түзетулерді таңдамаңыз. Қақпа
сонымен қатар кілтсіз қол қою жалаушаларын (`--identity-token-issuer`, `--identity-token-audience`) тексереді.
қажет болған жағдайда ұсынылады; жетіспейтін аргументтер орындалмай қалады.

## 2. Нұсқалау саясатын қолданыңыз

Барлық SoraFS CLI/SDK жәшіктері SemVer пайдаланады:

- `MAJOR`: бірінші 1.0 шығарылымы үшін енгізілген. 1.0 алдында `0.y` кішігірім соққы
  **CLI бетіндегі немесе Norito схемаларындағы бұзылу өзгерістерін көрсетеді**.
  қосымша саясаттың, телеметриялық толықтырулардың артында бекітілген өрістер).
- `PATCH`: қателерді түзету, тек құжаттама шығарылымдары және тәуелділік жаңартулары
  байқалатын мінез-құлықты өзгертпеңіз.

`sorafs_car`, `sorafs_manifest` және `sorafs_chunker` әрқашан бірдей
SDK-ның төменгі тұтынушылары бір реттелген нұсқаға тәуелді болуы мүмкін
жол. Нұсқаларды бұзған кезде:

1. Әрбір жәшіктің `Cargo.toml` ішіндегі `version =` өрістерін жаңартыңыз.
2. `Cargo.lock` құрылғысын `cargo update -p <crate>@<new-version>` арқылы қалпына келтіріңіз (
   жұмыс кеңістігі айқын нұсқаларды қамтамасыз етеді).
3. Ескірген артефактілердің қалмауы үшін босату қақпасын қайта іске қосыңыз.

## 3. Шығарылым жазбаларын дайындаңыз

Әрбір шығарылымда CLI, SDK және бөлектелген белгілерді өзгерту журналын жариялау керек
басқаруға әсер ететін өзгерістер. үлгіні пайдаланыңыз
`docs/examples/sorafs_release_notes.md` (оны шығарылым артефактілеріне көшіріңіз
анықтамалық және бөлімдерді нақты мәліметтермен толтырыңыз).

Ең аз мазмұн:

- **Ерекшеліктер**: CLI және SDK тұтынушыларына арналған мүмкіндік тақырыптары.
  талаптар.
- **Жаңарту қадамдары**: жүк тәуелділігін жоюға және қайта іске қосуға арналған TL;DR пәрмендері
  детерминирленген бекітпелер.
- **Тексеру**: пәрмен шығару хэштері немесе конверттері және дәл
  `ci/check_sorafs_cli_release.sh` нұсқасы орындалды.

Толтырылған шығарылым жазбаларын тегке бекітіңіз (мысалы, GitHub шығарылымының негізгі бөлігі) және сақтаңыз
олар детерминистикалық түрде жасалған артефактілермен бірге.

## 4. Босату ілмектерін орындаңыз

Қолтаңба бумасын және жасау үшін `scripts/release_sorafs_cli.sh` іске қосыңыз
әрбір шығарылыммен жеткізілетін тексеру қорытындысы. Қаптама CLI құрастырады
қажет болған жағдайда `sorafs_cli manifest sign` телефонына қоңырау шалып, дереу қайта ойнатады
`manifest verify-signature`, сондықтан қателер белгілеу алдында пайда болады. Мысалы:

```bash
scripts/release_sorafs_cli.sh \
  --manifest artifacts/site.manifest.to \
  --chunk-plan artifacts/site.chunk_plan.json \
  --chunk-summary artifacts/site.car.json \
  --bundle-out artifacts/release/manifest.bundle.json \
  --signature-out artifacts/release/manifest.sig \
  --identity-token-provider=github-actions \
  --identity-token-audience=sorafs-release \
  --expect-token-hash "$(cat .release/token.hash)"
```

Кеңестер:

- Шығарылым кірістерін (пайдалы жүктеме, жоспарлар, қорытындылар, күтілетін таңбалауыш хэш) қадағалаңыз
  репо немесе орналастыру конфигурациясы, осылайша сценарий қайталанатын болып қалады. CI құрылғысы
  `fixtures/sorafs_manifest/ci_sample/` астындағы бума канондық орналасуды көрсетеді.
- `.github/workflows/sorafs-cli-release.yml` бойынша базалық CI автоматтандыру; іске қосады
  босату қақпасы, жоғарыдағы сценарийді шақырады және жинақтарды/қолтаңбаларды ретінде мұрағаттайды
  жұмыс үрдісінің артефактілері. Бірдей пәрмен тәртібін айналаңыз (шығару қақпасы → белгісі →
  тексеру) басқа CI жүйелерінде аудит журналдары жасалған хэштермен сәйкес келуі үшін.
- Жасалған `manifest.bundle.json`, `manifest.sig` сақтаңыз,
  `manifest.sign.summary.json` және `manifest.verify.summary.json` бірге—олар
  басқару хабарламасында сілтеме жасалған пакетті қалыптастырыңыз.
- Шығарылым канондық құрылғыларды жаңартқанда, жаңартылған манифестті көшіріңіз,
  бөлік жоспары және `fixtures/sorafs_manifest/ci_sample/` (және жаңарту
  `docs/examples/sorafs_ci_sample/manifest.template.json`) белгілеу алдында.
  Төменгі ағын операторлары шығарылымды қайта шығару үшін бекітілген құрылғыларға тәуелді
  бума.
- `sorafs_cli proof stream` шектелген арнаны тексеру үшін іске қосу журналын жазып алыңыз және оны
  ағынды қорғау шараларының белсенді болып қалатынын дәлелдеу үшін шығару пакеті.
- Шығарылым жазбаларында қол қою кезінде пайдаланылған нақты `--identity-token-audience` жазыңыз; басқару
  жарияланымды мақұлдамас бұрын аудиторияны Fulcio саясатына қарсы тексереді.

Шығарылымда а болған кезде `scripts/sorafs_gateway_self_cert.sh` пайдаланыңыз
шлюзді шығару. Аттестацияны дәлелдеу үшін оны сол манифест бумасына көрсетіңіз
кандидат артефактіге сәйкес келеді:

```bash
scripts/sorafs_gateway_self_cert.sh --config docs/examples/sorafs_gateway_self_cert.conf \
  --manifest artifacts/site.manifest.to \
  --manifest-bundle artifacts/release/manifest.bundle.json
```

## 5. Белгі қойыңыз және жариялаңыз

Тексерулер өтіп, ілгектер аяқталғаннан кейін:

1. Екілік файлдарды растау үшін `sorafs_cli --version` және `sorafs_fetch --version` іске қосыңыз
   жаңа нұсқасы туралы хабарлаңыз.
2. Тіркелген `sorafs_release.toml` ішінде шығарылым конфигурациясын дайындаңыз
   (қалаулы) немесе орналастыру репо арқылы бақыланатын басқа конфигурация файлы. Болмау
   арнайы орта айнымалыларына сүйену; арқылы CLI жолын өткізіңіз
   `--config` (немесе баламасы), сондықтан босату кірістері анық және
   қайталанатын.
3. Қол қойылған тег (таңдаулы) немесе аннотацияланған тег жасаңыз:
   ```bash
   git tag -s sorafs-vX.Y.Z -m "SoraFS CLI & SDK vX.Y.Z"
   git push origin sorafs-vX.Y.Z
   ```
4. Артефактілерді жүктеп салу (CAR жинақтары, манифесттер, дәлелдемелердің қорытындылары, шығарылым жазбалары,
   аттестаттау нәтижелері) басқарудан кейін жоба тізіліміне
   бақылау тізімі [орналастыру нұсқаулығы](./developer-deployment.md). Егер босату
   жаңа арматура шығарсаңыз, оларды ортақ арматура репосына немесе нысан дүкеніне итеріңіз
   аудитті автоматтандыру жарияланған топтаманы бастапқы бақылаудан ажырата алады.
5. Басқару арнасын қол қойылған тегке, шығарылым жазбаларына сілтемелермен хабарлаңыз,
   манифест жинағы/қолтаңба хэштері, мұрағатталған `manifest.sign/verify` қорытындылары,
   және кез келген аттестаттау конверттері. CI тапсырмасының URL мекенжайын (немесе журнал мұрағатын) қосыңыз
   `ci/check_sorafs_cli_release.sh` және `scripts/release_sorafs_cli.sh` іске қосылды. Жаңарту
   аудиторлар артефактілердің мақұлдауларын бақылай алатындай басқару билеті; қашан
   `.github/workflows/sorafs-cli-release.yml` жұмыс жазбалары туралы хабарландыруларды байланыстырыңыз
   арнайы қорытындыларды қоюдың орнына жазылған хэш шығыстары.

## 6. Шығарылымнан кейінгі бақылау

- Құжаттаманың жаңа нұсқаға нұсқауын қамтамасыз етіңіз (жылдам іске қосулар, CI үлгілері)
  жаңартылды немесе ешқандай өзгертулер қажет емес екенін растаңыз.
- Ары қарай жұмыс болса, жол картасының жазбалары (мысалы, тасымалдау жалаулары, ескіру
- Аудиторлар үшін босату қақпасының шығыс журналдарын мұрағаттау — оларды қол қойылғанның жанында сақтаңыз
  артефактілер.

Осы құбырдан кейін CLI, SDK жәшіктері және басқару кепілі сақталады
әрбір босату циклі үшін құлыптау қадамы.

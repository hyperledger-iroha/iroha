`preview-feedback-log` көмекшісі, нәтижесінде алынған артефактілерді бөлісу жолы

## Неліктен біз кері байланысты тіркейміз

- **Сәйкестік:** әрбір шақыру алдын ала қарауға кімнің қашан кіргенін жазуы керек
  рұқсат етілген пайдалану саясатын және олардың рұқсаты қайтарылған кезде мойындады.
- **Телеметрия корреляциясы:** журнал порталдың телеметрия өсімдерін корреляциялауға мүмкіндік береді
  немесе Арнайы шақыру топтарымен трафикті қолданып көріңіз.
- **Басқару дәлелдері:** әрбір DOCS-SORA бақылау нүктесіне машина оқылатын құрылғы қажет
  шақырулар санын, ашық рецензенттерді және күтпегендердің күйін көрсететін қорытынды
  мәселелер.

Барлық дәлелдер `artifacts/docs_portal_preview/` астында сақталады, сондықтан оларды қосуға болады
шақыру толқыны билетіне немесе CI журналдарын сызып тастамай басқаруды шолуға.

## Тікелей шақыру қорытындысы

`docs/portal/scripts/preview-feedback-summary.mjs` ортақ журналды және
`npm run build` кезінде `src/data/previewFeedbackSummary.json` жазады. Үстел
Төменде шолушылар ағымдағы шақыру ізін көре алатындай JSON көрсетеді
CLI көмекшісін іске қоспай.

<Кері байланыс жиынтық кестесі />Шақыру қорытындыларына қоса, қорытынды қазір растауды көрсетеді
қайтарып алу саны, алынған жауап жылдамдығы (кері байланысты жіберулер
жалпы шақырулар) және орташа кідіріс (жылжымалы p95 бар).
шақыру және рецензенттің растауы немесе кері байланыс жіберуі. Жаңа
**Аяқтау %** бағанында қанша шақыру жабылғанын көрсетеді (мақұлданған немесе
күші жойылды), ал **Ack %** қанша алушының нақты расталғанын көрсетеді.
қолайлы пайдалану қадамдары. Бұл алынған пайыздар — тек JSON үшін
қайтарып алу/шығару мөлшерлемелері — тексерушілер растауы үшін басқару тақталарын жіберіңіз
CLI көмекшісін іске қоспай, қатысу деңгейлері мен айналым уақыты.
**Күй** бағаны толқынның қозғалмайтынын көрсетеді
белсенді (`active`), жабық, бірақ 14 күндік бақылау терезесінде (`closed`) немесе
мұрағатқа дайын (`archived`). **Соңғы оқиғадан бергі күндер** көрсеткіші қалай болатынын көрсетеді
соңғы шақыру/кері байланыс әрекетінен бері ұзақ уақыт өтті, сондықтан кеңестер байқайды
орамды немесе ұзаққа созылған алушыларды қайта тартуды қажет ететін толқындар.

### Толқын `preview-20260218` қорытынды

- **Қолдану аясы:** Қаржылық бета нұсқасы, бақылау операциялары, SDK серіктесі және экожүйені қорғаушы шолушылар келесі DOCS-SORA бақылау нүктесі алдында **2026‑02‑18** шақырылды.
- **Көрсеткіштер:** 4 шақыру, 4 растау, 4 қайтарып алу және 4 пікір жіберу (`responseRatePercent = 100`) бақылау мүмкіндігі когорты берген бір реттік мәселемен (`DOCS-SORA-Preview-20260218`).
- **Дәлел:** `artifacts/docs_portal_preview/feedback_log.json` енді осы толқынға арналған егжей-тегжейлі оқиға уақыт кестесін қамтиды және `artifacts/docs_portal_preview/preview-20260218-summary.json` басқару пакеттеріне тіркейтін қалыпты көрсеткіштерді түсіреді. `npm run --prefix docs/portal preview:log -- --summary --summary-json` арқылы портал деректерін қалпына келтіру `src/data/previewFeedbackSummary.json` ішіне бірдей сандарды тартады.
- **Толқын көмекшісі:** `npm run --prefix docs/portal preview:wave -- --wave preview-20260218 --invite-start 2026-02-18 --invite-end 2026-02-28 --report-date 2026-03-01 --notes "Finance/observability beta wave"` бір қадамда толқынның қорытындысын, дайджестті және жаңартылған портал деректерін шығарады.
- **Кейінгі әрекеттер:** Байқау мүмкіндігіне қатысты кері байланыс әрекет ету сұраулары үшін қосымша CSP мысалдарын қамтиды; Docs/DevRel `docs/portal/docs/sorafs/observability-plan.md` (§4.2) астында жазуды бақылайды, осылайша келесі шақыру толқыны түзетуді растай алады. Толық қорытынды [`preview-feedback/w3/summary.md`](./preview-feedback/w3/summary.md) ішінде [`preview-feedback/w3/log.md`](./preview-feedback/w3/log.md) ішінде түсірілген уақыт шкаласы бар.

## Тіркеу жұмыс процесі

`npm run preview:log` артындағы көмекші бір JSON журналын жазады.
барлық оқиғалар. Әрбір жазбада толқын белгісі, алушы, оқиға, уақыт белгісі және
қосымша жазбалар. Мысалы:

```json
{
  "version": 1,
  "generated_at": "2026-02-04T18:20:15.321Z",
  "entries": [
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "invite-sent",
      "timestamp": "2026-02-04T18:05:02.114Z",
      "notes": "seed reviewer"
    },
    {
      "wave": "preview-20260204",
      "recipient": "alice@example.com",
      "event": "feedback-submitted",
      "timestamp": "2026-02-05T09:14:43.009Z",
      "notes": "docs-preview/42"
    }
  ]
}
```

Пәрмендер:

```bash
# Append a new event (defaults to artifacts/docs_portal_preview/feedback_log.json)
npm run --prefix docs/portal preview:log -- \
  --wave preview-20260204 \
  --recipient alice@example.com \
  --event invite-sent \
  --notes "seed reviewer"

# Produce a text summary for the current repo
npm run --prefix docs/portal preview:log -- --summary

# Emit machine-readable summary for a specific wave
npm run --prefix docs/portal preview:log -- \
  --summary --summary-json --filter-wave preview-20260204 \
  > artifacts/docs_portal_preview/preview-20260204-summary.json

# Regenerate the portal summary data file
npm run --prefix docs/portal generate:preview-summary

# Generate a Markdown digest skeleton for a wave
npm run --prefix docs/portal preview:digest -- \
  --wave preview-20260204 \
  --invite-start 2026-02-04T00:00:00Z \
  --invite-end 2026-02-18T00:00:00Z \
  --dashboard "https://grafana.example/d/docs-preview?var-wave=preview-20260204" \
  --out artifacts/docs_portal_preview/preview-20260204-digest.md
```

Кеңестер:

- Функция тармағында жұмыс істеген кезде журналды нұсқа бақылауына қойыңыз
  қолдаушылар өзгерістерді қарай алады. Ұзақ жұмыс істейтін толқындар үшін журналды сақтаңыз
  басқару билеті және соңғы нұсқасын шығарылым артефактілеріне тіркеңіз.
- `preview:log` `--log <path>` жүйесіне тікелей кіру қажет болғанда қабылдайды.
  SoraFS бумасы немесе CI жұмыс кеңістігі.
- Әрқашан `--notes` анықтамасын қосыңыз (мәселе сілтемесі, GitHub дескрипті, когорт идентификаторы)
  қорытынды ретроспективада пайдалы.
- Сізге детерминирленген жиынтық уақыт белгілері қажет болғанда (CI немесе құжаттардың суреті үшін),
  қорытындыны іске қоспас бұрын `PREVIEW_FEEDBACK_NOW=<iso8601 timestamp>` орнатыңыз
  көмекші. Бұл `generatedAt`, күй және күн есептегіштерін дәйекті сақтайды
  қайта құрулар бойынша.

## Түйіндемелер, дайджесттер және мұрағаттар1. **Толқын бойынша қорытынды:** JSON қорытындысын жасаңыз және оны шақыруға тіркеңіз
   шикі журналмен бірге билет. Қорытынды жалпы шақыруларды санайды, ашық
   алушылар және кері байланыс есептеледі, сондықтан басқару біз әрбір түсіргенімізді растай алады
   шолушы. Портал құрастыру сонымен қатар жиынтық қорытындыны мына мекенжайда жариялайды
   `src/data/previewFeedbackSummary.json`, сондықтан мүдделі тараптар ағымды көре алады
   CLI іске қоспай суретке түсіру.
2. **Апталық дайджест:** `npm run --prefix docs/portal preview:digest -- --wave <label>` іске қосыңыз
   Markdown көрсеткіштерін алдын ала толтыру үшін, одан кейін толтырғыш бөлімдерін бейімдеңіз
   үлгісін пайдалану
   [`docs/examples/docs_preview_feedback_digest.md`](../../../examples/docs_preview_feedback_digest.md)
   оны мүдделі тараптармен бөліспес бұрын. Кем дегенде мыналарды қосыңыз: толқын белгісі, ашық
   шолушылар, маңызды тұжырымдар және келесі әрекеттер.
3. **Мұрағат:** жабық толқын журналдарын жылжытыңыз
   `artifacts/docs_portal_preview/archive/<wave>/` және ең соңғысын ғана сақтаңыз
   тамырдағы белсенді толқын. Бұл CI артефактілерін сақтай отырып, кішірейтеді
   аудит ізі.

## Жол картасымен байланыстыру

- Сұраған кезде шақыру және оқу кітаптарындағы осы құжатқа сілтеме жасаңыз
  рецензенттер пікір қалдыру үшін (қараңыз
  [`preview-invite-flow`](./preview-invite-flow.md),
  [`reviewer-onboarding`](./reviewer-onboarding.md), және
  [`public-preview-invite`](./public-preview-invite.md)).
- DOCS-SORA толқынын жабу кезінде `status.md` ішінде жасалған қорытындыны атап өтіңіз
  жол картасының элементтері (мысалы, DOCS-1b/2b/3c) артефактіге сілтеме жасай алады.
- Сыртқы SoraFS файлына жүктеп салу кезінде хэш дайджесттерін қорытындымен бірге сақтаңыз.
  рецензенттер дәлелдер бумасын тексере алады.

Тіркеу жұмыс процесін осы жерде кодталған күйде сақтау жол картасы талабын бұғаттаудан босатады
«басқару үшін кері байланысты/телеметрияны түсіру» және жұмсалған уақытты қысқартады
әр кезеңде шақыру тарихын қайта құру.

## Шолу

Жол картасының **DOCS-1b** элементі OAuth құрылғы кодының кіруін, күшті мазмұнды қажет етеді
қауіпсіздік саясаттары және алдын ала қарау порталы алдында қайталанатын ену сынақтары
зертханалық емес желілерде жұмыс істей алады. Бұл қосымша қауіп үлгісін түсіндіреді
репода іске асырылған басқару элементтері және шолуды қамтамасыз ететін белсенді бақылау тізімі
орындау керек.

- **Қолдану аймағы:** Байқап көру проксиі, ендірілген Swagger/RapiDoc панельдері және теңшелетін
  `docs/portal/src/components/TryItConsole.jsx` арқылы көрсетілген консольді қолданып көріңіз.
- **Қолданудан тыс:** Torii өзі (Torii дайындық шолулары қамтылған) және SoraFS
  жариялау (DOCS-3/7 қамтылған).

## Қауіп үлгісі

| Актив | Тәуекел | Жеңілдету |
| --- | --- | --- |
| Torii жеткізуші токендері | Docs құм жәшігінен тыс ұрлау немесе қайта пайдалану | Құрылғы кодының кіруі (`DOCS_OAUTH_*`) қысқа мерзімді таңбалауыштарды шығарады, прокси тақырыптарды өңдейді және консоль кэштелген тіркелгі деректерінің мерзімі автоматты түрде аяқталады. |
| Байқап көріңіз прокси | Ашық реле ретінде теріс пайдалану немесе Torii жылдамдық шектеулерін айналып өту | `scripts/tryit-proxy*.mjs` шығу рұқсат тізімдерін, жылдамдықты шектеуді, денсаулық тексерулерін және анық `X-TryIt-Auth` бағыттауды орындау; тіркелгі деректері сақталмайды. |
| Порталдың орындалу уақыты | Сайтаралық сценарийлер немесе зиянды енгізулер | `docusaurus.config.js` мазмұн-қауіпсіздік-саясат, сенімді түрлер және рұқсаттар-саясат тақырыптарын енгізеді; кірістірілген сценарийлер Docusaurus орындалу уақытымен шектелген. |
| Бақылану деректері | Телеметрия жоқ немесе бұрмалау | `docs/portal/docs/devportal/observability.md` зондтарды/бақылау тақталарын құжаттайды; `scripts/portal-probe.mjs` жарияламас бұрын CI жүйесінде жұмыс істейді. |

Қарсыластар арасында жалпыға қолжетімді алдын ала қарауды көретін қызығушылық танытқан пайдаланушылар, зиянды әрекеттер бар
ұрланған сілтемелерді сынау және сақталған браузерлерді жою әрекеті
тіркелгі деректері. Барлық басқару элементтері сенімді браузерлерсіз жұмыс істеуі керек
желілер.

## Міндетті басқару элементтері

1. **OAuth құрылғы кодының кіруі**
   - `DOCS_OAUTH_DEVICE_CODE_URL`, `DOCS_OAUTH_TOKEN_URL` конфигурациялау,
     `DOCS_OAUTH_CLIENT_ID` және құрастыру ортасындағы қатысты тұтқалар.
   - Байқап көру картасы жүйеге кіру виджетін (`OAuthDeviceLogin.jsx`) көрсетеді
     құрылғы кодын алады, таңбалауыштың соңғы нүктесін сұрайды және таңбалауыштарды автоматты түрде тазартады
     олардың мерзімі аяқталғаннан кейін. Қолмен тасымалдаушыны қайта анықтау төтенше жағдайлар үшін қолжетімді болып қалады
     кері қайтару.
   - OAuth конфигурациясы жоқ болғанда немесе
     DOCS-1b талап еткен 300-900-тер терезесінің шегінен тыс қайталанатын TTLs дрейфі;
     `DOCS_OAUTH_ALLOW_INSECURE=1` параметрін тек бір рет қолданылатын жергілікті алдын ала қарау үшін орнатыңыз.
2. **Прокси қоршаулары**
   - `scripts/tryit-proxy.mjs` рұқсат етілген бастауларды, мөлшерлеме шектеулерін, сұрауды жүзеге асырады
     өлшем шектеулері және трафикті тегтеу кезіндегі ағынды күту уақыттары
     `X-TryIt-Client` және журналдардағы таңбалауыштарды өңдеу.
   - `scripts/tryit-proxy-probe.mjs` плюс `docs/portal/docs/devportal/observability.md`
     жандылық зондының және бақылау тақтасының ережелерін анықтау; оларды әр алдында іске қосыңыз
     шығару.
3. **CSP, сенімді түрлер, рұқсаттар-саясат**
   - `docusaurus.config.js` енді детерминирленген қауіпсіздік тақырыптарын экспорттайды:
     `Content-Security-Policy` (әдепкі бойынша өздігінен, қатаң қосылу/img/скрипт
     тізімдер, Сенімді түрлерге қойылатын талаптар), `Permissions-Policy` және
     `Referrer-Policy: no-referrer`.
   - CSP қосылу тізімі OAuth құрылғы коды мен таңбалауыш соңғы нүктелерінің ақ тізімін береді
     (HTTPS тек `DOCS_SECURITY_ALLOW_INSECURE=1` болмаса), құрылғыға кіру жұмыс істейді
     басқа шығу тегі үшін құм жәшігін босаңсытпай.
   - Тақырыптар тікелей жасалған HTML ішіне ендірілген, сондықтан статикалық хосттар жасайды
     қосымша конфигурация қажет емес. Кірістірілген сценарийлерді шектеумен сақтаңыз
     Docusaurus жүктеу жолы.
4. **Runbooks, бақылау мүмкіндігі және кері қайтару**
   - `docs/portal/docs/devportal/observability.md` зондтарды сипаттайды және
     кіру қателерін, прокси жауап кодтарын және сұрауды қарайтын бақылау тақталары
     бюджеттер.
   - `docs/portal/docs/devportal/incident-runbooks.md` эскалацияны қамтиды
     құм жәшік теріс пайдаланылса, жол; онымен біріктіріңіз
     Соңғы нүктелерді қауіпсіз аудару үшін `scripts/tryit-proxy-rollback.mjs`.

## Қалам сынағы және шығарылымды тексеру тізімі

Әрбір алдын ала қарау жарнамасы үшін осы тізімді толтырыңыз (нәтижелерді шығарылымға тіркеңіз
билет):

1. **OAuth сымын тексеріңіз**
   - `DOCS_OAUTH_*` экспортымен `npm run start` өнімін жергілікті түрде іске қосыңыз.
   - Таза шолғыш профилінен Көріп көру консолін ашып, опциясын растаңыз
     құрылғы коды ағыны таңбалауышты шығарады, қызмет ету мерзімін санайды және өшіреді
     жарамдылық мерзімі аяқталғаннан кейін немесе жүйеден шыққаннан кейін өріс.
2. **Проксиді тексеру**
   - `npm run tryit-proxy` Torii сатысына қарсы, содан кейін орындаңыз
     Конфигурацияланған үлгі жолы бар `npm run probe:tryit-proxy`.
   - `authSource=override` жазбалары үшін журналдарды тексеріңіз және жылдамдықты шектеуді растаңыз
     терезеден асқан кезде есептегіштерді арттырады.
3. **CSP/Сенімді түрлерді растаңыз**
   - `npm run build` және `build/index.html` ашыңыз. `<meta
     http-equiv="Content-Security-Policy">` тегі күтілетін директиваларға сәйкес келеді
     және алдын ала қарауды жүктегенде DevTools ешқандай CSP бұзушылықтарын көрсетпейді.
   - Орналастырылған HTML файлын алу үшін `npm run probe:portal` (немесе curl) пайдаланыңыз; зонд
     енді `Content-Security-Policy`, `Permissions-Policy` немесе
     `Referrer-Policy` мета тегтері жоқ немесе жарияланған мәндерден өзгеше
     `docusaurus.config.js` ішінде, сондықтан басқару шолушылары шығуға сене алады
     бұралған бұралу шығысының орнына код.
4. **Бақылау мүмкіндігін қарау**
   - Прокси бақылау тақтасының жасыл екенін тексеріңіз (тарифтік шектеулер, қате коэффициенттері,
     денсаулықты тексеру көрсеткіштері).
   - `docs/portal/docs/devportal/incident-runbooks.md` ішіндегі оқиға жаттығуын орындаңыз
     егер хост өзгерсе (жаңа Netlify/SoraFS орналастыру).
5. **Нәтижелерді құжаттаңыз**
   - Шығарылым билетіне скриншоттарды/журналдарды тіркеңіз.
   - Түзету есебінің үлгісіндегі әрбір нәтижені түсіріңіз
     ([`docs/examples/pentest_remediation_report_template.md`](../../../examples/pentest_remediation_report_template.md))
     сондықтан иелері, SLA және қайта тексеру дәлелдерін кейінірек тексеру оңай.
   - DOCS-1b жол картасы элементі тексерілетін болып қалуы үшін осы бақылау тізіміне қайта сілтеме жасаңыз.

Кез келген қадам сәтсіз болса, жылжытуды тоқтатыңыз, блоктау мәселесін жіберіңіз және ескеріңіз
`status.md` ішіндегі жөндеу жоспары.

## Шолу

Жол картасының **MINFO-2** тармағы ұсыныс сызығынан бастап детерминирленген жолды талап етеді
кез келген адам істің күн тәртібіне қалай жеткенін растай алатын қоғамдық сұрыптау дәлелдері
кеңес. Бұл бет бұрын орнатылған CLI көмекшілерін тікелей желіге қосады
әзірлеушілер порталы, осылайша ұсыныстар берушілер, аудиторлар және платформа топтары бірдей әрекет ете алады
бақылау парағы моно-репо туралы айтпай-ақ.

Жұмыс процесі үш қайталанатын қадамды қамтиды:

1. `AgendaProposalV1` пайдалы жүктемелерін жергілікті түрде растаңыз және қайталанатын хэштерді қабылдаңыз
   жіберу модерация кезегіне жеткенге дейін.
2. Кеңес қызметкерлерінің мүмкіндігінше ұсыныспен бірге тізілімнің қайталанатын хиттерін жариялаңыз.
   детерминизмді жоғалтпай бұрынғы жағдайларды қадағалаңыз.
3. Сұрыптау көмекшісін іске қосыңыз, Merkle дәлелі дестесін түсіріңіз және бетін шығарыңыз
   азаматтар ұтыс ойынын қайталай алатындай етіп порталдың ішінен таңдалған тізім.

Барлық көмекшілер `cargo xtask ministry-agenda` ішінде тұрады; Norito схемасы
`docs/source/ministry/agenda_council_proposal.md` астында құжатталған.

## 1. Жергілікті жерде ұсыныстарды растаңыз

JSON пайдалы жүктемелерін сызу және талап етілетін дәлелдерді растау үшін CLI пайдаланыңыз (хэш
отбасылар, дайджест ұзындықтары, тіл тегі, жіберуші контакт) бар. The
валидатор кодты Torii қабылдау жолымен бөліседі, сондықтан ұсыныс жіберіледі
мұнда да портал арқылы жүктеп салған кезде өтеді.

<CodeBlock language="bash">{`жүк xtask министрлігі-күн тәртібін тексеру \\
  --proposal docs/examples/ministry/agenda_proposal_example.json \\
  --registry docs/examples/ministry/agenda_duplicate_registry.json`}</CodeBlock>

- Тарихи пайдалы жүктемелерді тексеру кезінде `--allow-registry-conflicts` қосыңыз; жаңа
  саусақ ізі бұрыннан бар болса, жіберулер тез орындалмауы керек.
- Мысалдар каталогы анықтамалық ұсынысты және тізілімді SDK файлдарына жібереді
  пішімді қатаң кодталмаған жеке пайдалы жүктемелерсіз қабылдаңыз.

## 2. Қайталанатын тізілім контекстін жариялау

Әрбір қабылданған ұсыныс бұрын кеңескен тізілім дайджестін қамтиды
жұту. Осы тізбе үшін SoraFS CID немесе Torii манифест жолын түсіріп, қосыңыз
оны сіздің жіберу метадеректеріңіз үшін кеңес шолушылары бірдей JSON тексере алады.

```json title="Duplicate registry (excerpt)"
{
  "entries": [
    {
      "hash_family": "blake3-256",
      "hash_hex": "0d714bed4b7c63c23a2cf8ee9ce6c3cde1007907c427b4a0754e8ad31c91338d",
      "proposal_id": "AC-2025-014",
      "note": "Already handled in 2025-08 incident"
    }
  ]
}
```

CLI соқтығысу туралы хабарлағанда, портал байланыстырылған ұсыныс идентификаторларын көрсетеді
операторлар қайта жіберулерді анықтау арқылы біріктіре алады немесе қабылдамайды.

## 3. Сұрыптау қорытындысын іске қосыңыз және жариялаңыз

Докет дайын болғаннан кейін детерминирленген сұрыптау көмекшісін іске қосыңыз. Ол тұтынады
тізімі JSON, тұқым және мақсатты ұяшықтар саны, содан кейін Merkle-профессионалды қолдауды шығарады
түйіндеме. JSON қорытындысы - бұл портал **Кеңес ұтысында** көрсететін нәрсе.
панель.

<CodeBlock language="bash">{`жүк xtask министрлігі-күн тәртібін сұрыптау \\
  --roster docs/examples/ministry/agenda_council_roster.json \\
  --слоттар 3 \\

ұсыныс трекерінде CID кодын байланыстырыңыз. Әзірлеуші порталы бірдей пайдаланады

### Шешім қабылдау журналы

Кеңес ұсыныс бойынша дауыс бергенде, шешім хаттамасын және хаттаманы қоса беріңіз
сұрыптау жиынтық CID басқару әрекетіне (Torii `/v2/gov/*` жазба). Портал
беттер жазбаны, дауыс санын және жоғарыда көрсетілген JSON әр әрекетті байланыстырады
**MINFO-2** талап ететін толық дәлелдер тізбегін қамтиды.

## Шолу

Еріктілердің брифтері қара тізімге немесе саясатқа өзгертулер ұсынылған кезде Ақпарат министрлігі қарастыратын қолдау/қарсы/контекстік көзқарастарды қамтиды. Әрбір жіберу `iroha_data_model::ministry::VolunteerBriefV1` (`1` нұсқасы) арқылы жүзеге асырылған канондық схемаға сәйкес болуы керек, осылайша:

1. Басқару тақталары фактілер кестелерін анықтаушы түрде салыстыра алады.
2. Мүдделер қақтығысы туралы ақпаратты жариялау жарияланғанға дейін күшіне енеді.
3. Тақырыптан тыс немесе астротурфты хабарлар аудит ізінен жойылмай жабылады.

Пайдалы жүктемені құру үшін `docs/source/ministry/volunteer_brief_template.md` үлгісін немесе `cargo xtask ministry-transparency ingest` көмекшісін пайдаланыңыз. CLI Norito құрылымын тексереді, модерация тегтерін сызады және қанша факт жолы, дәйексөз және ашу қабылданғанын көрсететін қорытынды шығарады.

## Жеткізу мүмкіндігі

- **Схема және құжаттар:** `VolunteerBriefV1` ортақ деректер үлгісінде өмір сүреді, ал егжей-тегжейлі өріс нұсқаулығы және пайдалы жүктеме мысалы `docs/source/ministry/volunteer_brief_template.md` және `docs/examples/ministry/volunteer_brief_template.json` ішінде орналасқан.
- **Тексеру құралы:** `cargo xtask ministry-transparency volunteer-validate --input <file> [--json-output artifacts/ministry/volunteer-lint.json]` портал құрастыруында пайдаланылатын бірдей линттерді іске қосады; `ingest --volunteer <file>` бүкіл буманы жариялау үшін дайындағыңыз келгенде қолжетімді болып қалады.
- **Портал беті:** Әрбір ұсыныс беті шифрланған қолдау/қарсы/мәтіндік қысқаша ақпараттарға сілтеме жасап, азаматтардың пікірлерін өңделмеген JSON жүктеп алмай-ақ көрсетеді. Модерация метадеректері қысқаша мәтінмен бірге көрсетіледі, сондықтан алқабилер жазбаның неге кешіктірілгенін немесе алынып тасталғанын түсінуі үшін.

Қосымша автоматтандыру (мысалы, жаппай импорттық API интерфейстері немесе кеңейтілген аналитика) қажет болса, жол картасының **MINFO-3 — Ерікті қызығушылық панелінің инфрақұрылымы** тармағын қараңыз.

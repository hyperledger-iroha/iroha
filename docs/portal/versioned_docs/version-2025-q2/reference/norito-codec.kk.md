№ Norito кодек анықтамасы

Norito — Iroha канондық сериялау қабаты. Әрбір сымды хабарлама, дискідегі
пайдалы жүктеме және кросс-компоненттік API Norito пайдаланады, сондықтан түйіндер бірдей байттармен келіседі
олар әртүрлі жабдықта жұмыс істегенде де. Бұл бетте қозғалатын бөліктер жинақталған
және `norito.md` ішіндегі толық сипаттаманы көрсетеді.

## Негізгі орналасу

| Құрамдас | Мақсаты | Дереккөз |
| --- | --- | --- |
| **Тақырып** | Мүмкіндіктерді келіседі (оралған құрылымдар/тізбектер, ықшам ұзындықтар, қысу жалаулары) және CRC64 бақылау сомасын енгізеді, осылайша пайдалы жүктің тұтастығы декодтау алдында тексеріледі. | `norito::header` — `norito.md` қараңыз («Тақырып және жалаулар», репозиторий түбірі) |
| **Бос пайдалы жүктеме** | Хэштеу/салыстыру үшін пайдаланылатын детерминистік мәнді кодтау. Дәл осындай орналасу тасымалдауға арналған тақырыппен оралған. | `norito::codec::{Encode, Decode}` |
| **Сығу** | Қосымша Zstd (және эксперименттік GPU жеделдету) `compression` жалауша байты арқылы белсендірілген. | `norito.md`, «Сығу келіссөздері» |

Толық жалау тізілімі (қапталған-құрылым, бумаланған-сек, ықшам ұзындықтар, қысу)
`norito::header::flags` тұрады. `norito::header::Flags` ыңғайлылықты көрсетеді
орындау уақытын тексеруді тексереді; сақталған орналасу биттерін декодерлер қабылдамайды.

## Қолдау алыңыз

`norito_derive` `Encode`, `Decode`, `IntoSchema` және JSON көмекшісін жібереді.
Негізгі конвенциялар:

- `packed-struct` мүмкіндігі болған кезде құрылымдар/сандар жинақталған макеттерді шығарады
  қосылған (әдепкі). Іске асыру `crates/norito_derive/src/derive_struct.rs` ішінде өмір сүреді
  және мінез-құлық `norito.md` («Бумаланған орналасулар») құжатталады.
- Бумаланған жинақтар v1 жүйесінде бекітілген енді ретті тақырыптар мен ығысуларды пайдаланады; тек
  әрбір мән ұзындығы префикстеріне `COMPACT_LEN` әсер етеді.
- JSON көмекшілері (`norito::json`) үшін детерминирленген Norito қолдауы бар JSON қамтамасыз етеді
  API интерфейстерін ашыңыз. `norito::json::{to_json_pretty, from_json}` пайдаланыңыз — ешқашан `serde_json`.

## Мультикодек және идентификатор кестелері

Norito мультикодек тағайындауларын `norito::multicodec` ішінде сақтайды. Анықтама
кесте (хэштер, кілт түрлері, пайдалы жүктеме дескрипторлары) `multicodec.md` ішінде сақталады
репозиторий түбірінде. Жаңа идентификатор қосылғанда:

1. `norito::multicodec::registry` жаңарту.
2. `multicodec.md` ішіндегі кестені кеңейтіңіз.
3. Төменгі ағынды байланыстыруларды (Python/Java) қалпына келтіріңіз, егер олар картаны пайдаланса.

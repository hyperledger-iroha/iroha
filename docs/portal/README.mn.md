# SORA Nexus Хөгжүүлэгчийн портал

Энэ лавлах нь интерактив хөгжүүлэгчийн Docusaurus ажлын талбарыг агуулдаг.
портал. Портал нь Norito гарын авлага, SDK хурдан эхлүүлэх болон OpenAPI-г нэгтгэдэг.
лавлагаа нь `cargo xtask openapi`-ээр үүсгэгдэж, тэдгээрийг SORA Nexus дотор ороосон.
docs программ даяар ашигладаг брэнд.

## Урьдчилсан нөхцөл

- Node.js 18.18 буюу түүнээс дээш хувилбар (Docusaurus v3 суурь).
- Утас 1.x эсвэл npm ≥ 9 багцын менежментэд зориулагдсан.
- Rust toolchain (OpenAPI синхрончлолын скрипт ашигладаг).

## Ачаалагч

```bash
cd docs/portal
npm install    # or yarn install
```

| `npm run serve` / `yarn serve` | Орон нутагт хамгийн сүүлийн үеийн бүтээцийг үйлчил (утааны сорилтод хэрэгтэй). |
| `npm run docs:version -- <label>` | Одоогийн баримт бичгүүдийг `versioned_docs/version-<label>` (`docusaurus docs:version` орчмын боодол) болгон агшин зуурын зургийг авна уу. |
| `npm run sync-openapi` / `yarn sync-openapi` | `static/openapi/torii.json`-г `cargo xtask openapi`-ээр дахин үүсгэнэ үү (нэмэлт хувилбарын агшин зуурын агшинд техникийн үзүүлэлтийг хуулахын тулд `--mirror=<label>`-г дамжуулна уу). |

| `npm run probe:tryit-proxy` | `/healthz` + түүвэр хүсэлтийн шалгалтыг прокси (CI/monitoring helper)-ийн эсрэг гарга. |
| `npm run manage:tryit-proxy -- <update|rollback>` | `.env` зорилтот проксиг нөөцлөлтөөр шинэчлэх буюу сэргээх. |
| `npm run sync-i18n` | Япон, еврей, испани, португал, франц, орос, араб, урду, бирм, гүрж, армян, азербайжан, казах, башкир, амхар, зонгха, узбек, монгол, уламжлалт хятад, хялбаршуулсан хятад хэл дээрх орчуулга `i18n/`-ийн дагуу байгаа эсэхийг шалгаарай. |
| `npm run sync-norito-snippets` | Сонгосон Kotodama жишээ баримт бичгүүдийг + татаж авах боломжтой хэсгүүдийг (мөн хөгжүүлэлтийн серверийн залгаас автоматаар дууддаг) дахин үүсгэх. |
| `npm run test:tryit-proxy` | Прокси нэгжийн туршилтыг Node-ийн туршилтын программ (`node --test`) ашиглан гүйцэтгэнэ. |

OpenAPI синхрончлолын скрипт нь `cargo xtask openapi`-г дараахаас авах боломжтой байхыг шаарддаг.

## Баримт бичгийн хувилбар болон OpenAPI агшин зуурын зураг

- **Баримт бичгийн хувилбарыг хайчилж байна:** `npm run docs:version -- 2025-q3` (эсвэл тохиролцсон шошго)-г ажиллуулна уу.
  Үүсгэсэн `versioned_docs/version-<label>`, `versioned_sidebars`,
  болон `versions.json`. Navbar хувилбарын уналт нь автоматаар гарч ирнэ
  шинэ хормын хувилбар.
- **OpenAPI олдворуудыг синк хийх:** хувилбарыг хайсны дараа каноникийг дахин сэргээнэ үү
  Spec болон `cargo xtask openapi --sign <path-to-ed25519-key>`-ээр дамжуулан харуулах, дараа нь
  тохирох агшин зуурын зургийг авах
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`. The
  скрипт нь `static/openapi/versions/2025-q3/torii.json` бичдэг бөгөөд техникийн үзүүлэлтийг тусгадаг
  `versions/current/torii.json` руу, `versions.json` шинэчлэгдэж, шинэчлэгддэг
  `/openapi/torii.json`, гарын үсэг зурсан `manifest.json`-г хувилбар болгон хувилдаг.
  Энэ лавлах нь түүхэн үзүүлэлтүүд нь ижил гарал үүслийн мета өгөгдлийг агуулдаг. Аливаа нийлүүлнэ
  шинээр үүсгэсэн спецификацияыг хуулахын тулд `--mirror=<label>` тугуудын тоо
  бусад түүхэн агшин зуурын зургууд.
- **CI-ийн хүлээлт:** баримт бичигт хувилбарын цохилтыг оруулах ёстой
  (боломжтой бол) дээр нь шинэчилсэн OpenAPI агшин зуурын агшинд Swagger, RapiDoc,
  болон Redoc самбарууд нь дуудах алдаагүйгээр түүхэн үзүүлэлтүүдийн хооронд шилжих боломжтой.
- **Манифестийн хэрэгжилт:** гарын үсэг зурсан үед `sync-openapi` скрипт амжилтгүй болсон.
  `manifest.json` дутуу, алдаатай эсвэл шинээр үүсгэсэнтэй таарахгүй байна
  spec тул гарын үсэг зураагүй агшин зуурын зургийг анхдагчаар нийтлэх боломжгүй. Дахин ажиллуулах
  `cargo xtask openapi --sign <key>` каноник манифестийг сэргээх ба
  синхрончлолыг дахин ажиллуулснаар хувилбартай хормын хувилбарууд гарын үсэг зурсан мета өгөгдлийг авах болно.
  Зөвхөн орон нутгийн урьдчилан үзэхийн тулд `--allow-unsigned`-г дамжуулаарай (CI ажиллаж байгаа хэвээр байна
  `ci/check_openapi_spec.sh` нь техникийн үзүүлэлтийг сэргээж, баталгаажуулдаг
  амлалтуудыг нэгтгэхийг зөвшөөрөхөөс өмнө манифест).

### Проксины тохиргоог хийж үзээрэй

"Оролдоод үзээрэй" хамгаалагдсан хязгаарлагдмал орчин нь хүсэлтийг `scripts/tryit-proxy.mjs`-ээр дамжуулдаг. Тохируулах
орчны хувьсагчтай проксиг эхлүүлэхийн өмнө:

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN` (анхдагч `127.0.0.1:8787`) холбох хаягийг хянадаг.
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` санах ойн тохиргоог хийх
  хурд хязгаарлагч (анхдагчаар 60 секундэд 60 хүсэлт).
- Дуудлага хийгчийн `Authorization`-г дамжуулахын тулд `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1`-г тохируулна уу.
  өгөгдмөл эзэмшигчийн токен дээр найдахын оронд толгой.
- Хэрэв `static/openapi/torii.json`-ээс хазайвал прокси эхлүүлэхээс татгалздаг.
  `static/openapi/manifest.json` дээр гарын үсэг зурсан манифест. Гүй
  Тодорхойлолтыг шинэчлэхийн тулд `npm run sync-openapi -- --latest`; экспортлох
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` зөвхөн яаралтай тусламжийн үед (анхаарал нь
  нэвтэрсэн бөгөөд прокси ямар ч байсан эхэлдэг).
- `npm run probe:tryit-proxy`-г ажиллуулж, тайзны прокси-г утсаар шалгаж, утсаар холбогдоно уу.
  ажилд хяналт тавих тушаал; `npm run manage:tryit-proxy -- update` хялбаршуулсан
  Torii төгсгөлийн цэгүүдийг эргүүлж, буцаах `.env` нөөцлөлтийг хадгалах.
- `TRYIT_PROXY_PROBE_METRICS_FILE` ба `TRYIT_PROXY_PROBE_LABELS` датчикийг зөвшөөрнө
  `probe_success`/`probe_duration_seconds` Prometheus текст файлын форматаар ялгаруулна.
  Та эрүүл мэндийн шалгалтыг node_exporter эсвэл дурын багц цуглуулагч руу холбож болно.
- Prometheus хусахад `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  гистограмын цуврал; docs portal Grafana хяналтын самбар нь тэдгээрийг p95/p99 хоцрогдолд зарцуулдаг
  SLO хянах (`dashboards/grafana/docs_portal.json`).

### OAuth төхөөрөмжийн кодоор нэвтрэх

Портал итгэмжлэгдсэн сүлжээнүүдийн гадна байгаа үед OAuth төхөөрөмжийг тохируулна уу
зөвшөөрөлтэй тул хянагч нар удаан эдэлгээтэй Torii жетонд хэзээ ч хүрдэггүй. -ийг экспортлох
`npm run start` эсвэл `npm run build`-г ажиллуулахын өмнө доорх хувьсагчдыг:

| Хувьсагч | Тэмдэглэл |
|----------|-------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` | Төхөөрөмжийн зөвшөөрлийн тэтгэлэгийг хэрэгжүүлдэг OAuth төгсгөлийн цэгүүд. |
| `DOCS_OAUTH_CLIENT_ID` | Доксыг урьдчилан үзэхээр бүртгүүлсэн үйлчлүүлэгчийн ID. |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` | Гаргасан токеныг хязгаарлах нэмэлт хамрах хүрээ/үзэгчдийн мөр. |
| `DOCS_OAUTH_POLL_INTERVAL_MS` | Портал виджетээр хэрэгжүүлсэн санал асуулгын хамгийн бага интервал (өгөгдмөл нь 5000 мс; доод утгыг татгалзсан). |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` | Сервер нь `expires_in`-г орхигдуулсан тохиолдолд буцаах хугацаа дуусах цонхыг ашигладаг. |
| `DOCS_OAUTH_ALLOW_INSECURE` | Зөвхөн орон нутгийн хөгжилд зориулж `1` гэж тохируулж, бүтээн байгуулалтын цагийн хамгаалалтыг тойрч гарах болно. Шаардлагатай хувьсагч дутуу эсвэл TTL нь албадмал хүрээнээс гадуур байвал үйлдвэрлэлийн бүтээн байгуулалт амжилтгүй болно. |

Жишээ:

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

Бүтэц эдгээр утгыг авсны дараа "Оролдоод үзээрэй" консол нь нэвтрэлтийг хийдэг
төхөөрөмжийн кодыг харуулсан самбар, токен төгсгөлийн цэгийг санал асуулга, Bearer-ийг дүүргэдэг
талбар автоматаар, хугацаа нь дуусахад токеныг арилгана. Портал татгалзаж байна
OAuth тохиргоо бүрэн бус эсвэл токен/төхөөрөмжийн TTL-ууд унах үед эхэлнэ
аюулгүй байдлын төсвөөс гадуур; зөвхөн дээр `DOCS_OAUTH_ALLOW_INSECURE=1` экспортлох
Нэргүй нэвтрэх боломжтой нэг удаагийн орон нутгийн урьдчилан харах. Прокси хэвээр байна
`X-TryIt-Auth` гарын авлагыг хүчингүй болгодог тул та OAuth хувьсагчдыг устгах боломжтой
та түр зуурын токенуудыг буулгах дуртай үедээ.

[`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md)-г үзнэ үү.
аюул заналхийллийн бүрэн загвар болон үзэг шалгах хаалга.

### Хамгаалалтын толгой

`docusaurus.config.js` одоо детерминист аюулгүй байдлын гарчиг буюу контентын аюулгүй байдлыг ялгаруулдаг
бодлого, Итгэмжлэгдсэн төрлүүд, Зөвшөөрөл-Бодлого, Реферрер-Бодлого—тийм статик хостууд
dev сервертэй ижил хамгаалалтын хашлагыг өвлөн авах. Өгөгдмөл нь зөвхөн скриптийг зөвшөөрдөг
портал гарал үүслээс үйлчилж, хөндлөн гарал үүсэл `connect-src` урсгалыг хязгаарлана
тохируулсан аналитик төгсгөлийн цэг болон Проксиг үзээрэй. Үйлдвэрлэл бүтээн байгуулалтаас татгалздаг
HTTPS бус аналитик/турших төгсгөлийн цэгүүд; `http://`-ийн эсрэг орон нутгийн урьдчилан үзэх
зорилтууд, `DOCS_SECURITY_ALLOW_INSECURE=1`-г тодорхой болгож хамгаалалтын хашлага
зэрэглэл буурсныг хүлээн зөвшөөрч байна.

Армян, Азербайджан, Казак, Башкир, Амхар, Жунха, Узбек, Монгол,
Уламжлалт хятад хэл, хялбаршуулсан хятад хэл. Шинэ баримт бичиг ирэх бүрт,
ажиллуулах:

```bash
npm run sync-i18n
```

Скрипт нь агуулахын өргөн хүрээний `scripts/sync_docs_i18n.py` зан төлөвийг тусгадаг.
`i18n/<lang>/docusaurus-plugin-content-docs/current/...` дор stub орчуулга үүсгэх.
Дараа нь засварлагчид орлуулагчийн бичвэрийг сольж, үндсэн хэсгийг шинэчлэх боломжтой
мета өгөгдөл (`status`, `translation_last_reviewed` гэх мэт).

Өгөгдмөлөөр, үйлдвэрлэлийн бүтэц нь зөвхөн жагсаалтад орсон локалуудыг агуулдаг
`docs/i18n/published_locales.json` (одоогоор зөвхөн англи хэл дээр) тул stub орлуулагч
тээвэрлэгдээгүй байна. Урьдчилан үзэх шаардлагатай үед `DOCS_I18N_INCLUDE_STUBS=1` тохируулна уу
орон нутагт хийгдэж буй орон нутаг.

- `ci/check_docs_portal.sh` нь эхлээд `node scripts/check-sdk-recipes.mjs`-г ажиллуулж, SDK дэвтэрийн жорууд нь тэдний каноник эх файлтай тохирч байгаа эсэхийг шалгаад, дараа нь `npm ci|install`, `npm run build`-г ажиллуулж, үндсэн хэсгүүдийг баталгаажуулдаг (нүүр хуудас, I002, I0100). Norito, нийтлэх хяналтын хуудас) үүсгэсэн HTML-д байдаг.
- `ci/check_openapi_spec.sh` `cargo xtask openapi`-ээр дамжуулан Torii техникийн үзүүлэлтийг сэргээдэг,
  `static/openapi/torii.json` ба `versions/current/torii.json`-тэй харьцуулж,
  мөн шалгах нийлбэрийн манифестийг шалгадаг тул хянагдсан үзүүлэлт эсвэл хэш үед PR амжилтгүй болно
  хуучирсан байна.
- `.github/workflows/check-docs.yml` нь агуулгын регрессийг барихын тулд PR бүр дээр портал бүтээцийг ажиллуулдаг.
- `.github/workflows/docs-portal-preview.yml` татах хүсэлтэд зориулж сайтыг бүтээж, `build/checksums.sha256` бичиж, `sha256sum -c`-ээр баталгаажуулж, гаралтыг `artifacts/preview-site.tar.gz` хэлбэрээр багцалж, `artifacts/preview-site.tar.gz`-ээр тодорхойлогч үүсгэдэг, `scripts/generate-preview-descriptor.mjs`-ээр дамжуулан тодорхойлогч үүсгэдэг. Шүүгчид угсралтыг дахин ажиллуулахгүйгээр яг агшин зуурын зургийг шалгах боломжтой. Ажлын урсгал нь манифест/архивын хураангуйг (болон байгаа бол SoraFS багц) хураангуйлан татах хүсэлтийн тайлбарыг нийтэлдэг тул хянагчид Үйлдлийн бүртгэлийг нээхгүйгээр баталгаажуулалтын үр дүнг хардаг.
- `.github/workflows/docs-portal-deploy.yml` нь `main`/`master` рүү түлхэж GitHub Pages дээр статик бүтээцийг нийтэлж, `github-pages` орчны URL дээр урьдчилан харах боломжийг олгодог. Ажлын урсгал нь байршуулахаас өмнө буух хуудасны утааны шалгалтыг явуулдаг.
- `scripts/sorafs-package-preview.sh` нь урьдчилан харах сайтыг тодорхойлогч SoraFS багц (CAR, төлөвлөгөө, манифест) болгон хувиргадаг бөгөөд итгэмжлэлүүдийг JSON тохиргоогоор (`docs/examples/sorafs_preview_publish.json`-г үзнэ үү) нийлүүлэх үед I180004. ends-д манифест илгээх боломжтой.
- `scripts/preview_verify.sh` нь хянагчид өмнө нь гараар гүйцэтгэсэн шалгах нийлбэр + тодорхойлогчийн баталгаажуулалтын урсгалыг гүйцэтгэдэг. Урьдчилан үзэх олдворууд өөрчлөгдөөгүй эсэхийг баталгаажуулахын тулд үүнийг задалсан `build/` лавлах (болон нэмэлт тодорхойлогч/архивын замууд) руу чиглүүлнэ.
- `scripts/sorafs-pin-release.sh` дээр нэмэх нь `.github/workflows/docs-portal-sorafs-pin.yml` SoraFS үйлдвэрлэлийг автоматжуулах: бүтээх/турших, CAR + манифест үүсгэх, Sigstore гарын үсэг зурах, баталгаажуулах, нэмэлт нэрээр холбох, засаглалын хяналт шалгалтад зориулж олдвор байршуулах. Хувилбарыг сурталчлахдаа `workflow_dispatch`-ээр дамжуулан ажлын урсгалыг идэвхжүүлнэ үү.
- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]` Гарцын зөвшөөрлийн бүртгэлд гарын үсэг зурахаас өмнө баталгаажуулдаг. Туслагч нь каноник/хөөрхөн хостууд, манифест мета өгөгдөл, телеметрийн шошго нь детерминист бодлоготой тохирч байгаа эсэхийг баталгаажуулж, `--json-out`-ээр дамжуулан DG-3 нотолгоонд зориулж JSON хураангуйг гаргах боломжтой.
- Илгээлт хийх үед зүү туслагч нь мөн `scripts/generate-dns-cutover-plan.mjs`-г дуудаж, `artifacts/sorafs/portal.dns-cutover.json`-г үүсгэдэг. Set `DNS_CHANGE_TICKET`, `DNS_CUTOVER_WINDOW`, `DNS_HOSTNAME`, `DNS_ZONE`, болон `DNS_OPS_CONTACT` (эсвэл харгалзах `--dns-*` тугуудыг дамжуулж) DNS тайрах машинд хэрэгтэй. Кэшийг хүчингүй болгох болон буцаах дэгээ шаардлагатай үед `DNS_CACHE_PURGE_ENDPOINT`, `DNS_CACHE_PURGE_AUTH_ENV`, `DNS_PREVIOUS_PLAN` (эсвэл `--cache-purge-*` / `--previous-dns-plan` тугуудыг) нэмнэ үү. буцаалтууд. Тодорхойлогч нь одоо мөн `Sora-Route-Binding` (хост, CID, толгой/холбох зам, баталгаажуулах командууд)-ыг баримтжуулж байгаа тул GAR урамшуулал болон нөөц төлөвлөгөөний тойм нь захад үйлчилдэг яг толгой хэсгийг иш татдаг.
- Ижил ажлын урсгал нь `scripts/sns_zonefile_skeleton.py`-ээр дамжуулан SNS бүсийн араг яс/шийдвэрлэгчийн хэсгийг ялгаруулж болно. IPv4/IPv6/CNAME/SPKI/TXT мета өгөгдлийг (`DNS_ZONEFILE_IPV4` гэх мэт env vars эсвэл `--dns-zonefile-ipv4` гэх мэт CLI тугуудыг) өгч, GAR тоймыг `DNS_GAR_DIGEST`-ээр дамжуулж, туслагч I019NI02 гэж бичнэ snippet) автоматаар SN-7 нотлох баримт нь таслах тодорхойлогчийн хажууд бууна.
- DNS тодорхойлогч одоо `gateway_binding` хэсгийг агуулсан бөгөөд дээрх олдворуудыг (зам, агуулгын CID, нотолгооны төлөв, шууд толгойн загвар) иш татсан тул DG-3 өөрчлөлтийн зөвшөөрөлд гарц дээр хүлээгдэж буй яг `Sora-Name/Sora-Proof/CSP/HSTS` багц багтана.

- Порталын агуулгад хүрэх татах хүсэлтэд баригдсан сайт, шалгах нийлбэр манифест, шахсан архив, тодорхойлогч агуулсан Үйлдлийн олдворууд (`docs-portal-preview`, `docs-portal-preview-metadata`) багтана; Шүүгчид `index.html`-г дотооддоо татаж аваад нээж, урьдчилан харахыг хуваалцахын өмнө шалгах нийлбэрийг шалгах боломжтой.

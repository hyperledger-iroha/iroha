## მიზანი

საგზაო რუქის ერთეული **DOCS-SORA** მოუწოდებს მიმომხილველის ჩართვას და საჯარო გადახედვას
მოიწვიე პროგრამა, როგორც საბოლოო ბლოკერები, სანამ პორტალი შეძლებს ბეტადან გამოსვლას. ეს გვერდი
აღწერს, თუ როგორ უნდა გაიხსნას ყოველი მოწვევის ტალღა, რომელიც არტეფაქტები უნდა გაიგზავნოს მანამდე
მოწვევები გადიან და როგორ დავამტკიცოთ, რომ ნაკადი აუდიტორულია. გამოიყენეთ იგი გვერდით:

- [`devportal/reviewer-onboarding`](./reviewer-onboarding.md) ამისთვის
  თითო მიმომხილველის დამუშავება.
- [`devportal/preview-integrity-plan`](./preview-integrity-plan.md) საკონტროლო ჯამისთვის
  გარანტიები.
- [`devportal/observability`](./observability.md) ტელემეტრიის ექსპორტისთვის და
  გამაფრთხილებელი კაკვები.

## ფრენის წინ საკონტროლო სია

შეასრულეთ ეს ქმედებები **ადრე** ტალღის მოწვევის დაგეგმვამდე:

1. **CI არტეფაქტები ხელმისაწვდომია **
   - უახლესი `docs-portal-preview` + აღმწერი ატვირთული
     `.github/workflows/docs-portal-preview.yml`.
   - SoraFS პინი აღნიშნულია `docs/portal/docs/devportal/deploy-guide.md`-ში
     (ჩამომჭრელი აღმწერი იმყოფება).
2. **საკონტროლო თანხის აღსრულება**
   - გამოძახებულია `docs/portal/scripts/serve-verified-preview.mjs`
     `npm run serve`.
   - `scripts/preview_verify.sh` ინსტრუქციები შემოწმებულია macOS + Linux-ზე.
3. **ტელემეტრიის საწყისი ხაზი **
   - `dashboards/grafana/docs_portal.json` აჩვენებს ჯანსაღ სცადეთ ტრაფიკს და
     `docs.preview.integrity` გაფრთხილება მწვანეა.
   - უახლესი `docs/portal/docs/devportal/observability.md` დანართი განახლებულია
     Grafana ბმულები.
4. **მმართველობის არტეფაქტები**
   - მოიწვიე ტრეკერის საკითხი მზად (ერთი გამოცემა თითო ტალღაზე).
   - მიმომხილველის რეესტრის შაბლონი დაკოპირებულია (იხ
     [`docs/examples/docs_preview_request_template.md`](../../../examples/docs_preview_request_template.md)).
   - საკითხს თან ერთვის იურიდიული და SRE-ს მოთხოვნილი ნებართვები.

ჩაწერეთ წინასწარი ფრენის დასრულება მოწვევის ტრეკერში ნებისმიერი წერილის გაგზავნამდე.

## ნაკადის ნაბიჯები

1. **აირჩიეთ კანდიდატები**
   - ამოიღეთ ლოდინის სიიდან ან პარტნიორის რიგიდან.
   - დარწმუნდით, რომ თითოეულ კანდიდატს აქვს დასრულებული მოთხოვნის შაბლონი.
2. **წვდომის დამტკიცება**
   - მიანიჭეთ დამმტკიცებელი მოწვევის ტრეკერის საკითხს.
   - შეამოწმეთ წინაპირობები (CLA/კონტრაქტი, მისაღები გამოყენება, უსაფრთხოების მოკლე ინფორმაცია).
3. ** მოწვევის გაგზავნა **
   - შეავსე
     [`docs/examples/docs_preview_invite_template.md`](../../../examples/docs_preview_invite_template.md)
     ჩანაცვლების ადგილები (`<preview_tag>`, `<request_ticket>`, კონტაქტები).
   - მიამაგრეთ აღმწერი + არქივის ჰეში, სცადეთ URL-ის დადგმა და მხარდაჭერა
     არხები.
   - შეინახეთ საბოლოო ელფოსტა (ან Matrix/Slack ტრანსკრიპტი) ნომერში.
4. ** ჩართვის თვალის დევნება **
   - განაახლეთ მოწვევის ტრეკერი `invite_sent_at`, `expected_exit_at` და
     სტატუსი (`pending`, `active`, `complete`, `revoked`).
   - ბმული მიმომხილველის მიღების მოთხოვნაზე აუდიტორობის შესახებ.
5. **ტელემეტრიის მონიტორინგი **
   - უყურეთ `docs.preview.session_active` და `TryItProxyErrors` გაფრთხილებებს.
   - შეიყვანეთ ინციდენტი, თუ ტელემეტრია გადახრის საბაზისო ხაზს და ჩაწერეთ
     შედეგი მოწვევის გვერდით.
6. ** შეაგროვეთ გამოხმაურება და გადით **
   - დახურეთ მოწვევა, როგორც კი გამოხმაურება მიდის ან `expected_exit_at` გაივლის.
   - განაახლეთ ტალღის საკითხი მოკლე შეჯამებით (მიგნებები, ინციდენტები, შემდეგი
     მოქმედებები) შემდეგ კოჰორტაზე გადასვლამდე.

## უკან დაბრუნება და პაუზის კრიტერიუმები

შეაჩერეთ მოწვევის ნაკადი (და შეატყობინეთ მმართველობას), როდესაც მოხდება რომელიმე შემდეგი:

- სცადეთ პროქსი ინციდენტი, რომელიც მოითხოვდა უკან დაბრუნებას (`npm run manage:tryit-proxy`).
- გაფრთხილების დაღლილობა: >3 გაფრთხილების გვერდი მხოლოდ წინასწარი გადახედვის საბოლოო წერტილებისთვის 7 დღის განმავლობაში.
- შესაბამისობის ხარვეზი: მოწვევა გაიგზავნა ხელმოწერილი პირობების გარეშე ან შესვლის გარეშე
  მოთხოვნის შაბლონი.
- მთლიანობის რისკი: საკონტროლო ჯამის შეუსაბამობა აღმოჩენილია `scripts/preview_verify.sh`-ის მიერ.

განაახლეთ მხოლოდ მოწვევის ტრეკერში გამოსწორების დოკუმენტაციის შემდეგ და
ტელემეტრიის დაფის სტაბილურობის დადასტურება მინიმუმ 48 საათის განმავლობაში.

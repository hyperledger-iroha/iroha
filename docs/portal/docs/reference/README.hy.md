This section aggregates the “read it as a spec” material for Iroha. These pages stay stable even as
guides and tutorials evolve.

## Available today

- **Norito codec overview** – `reference/norito-codec.md` links directly to the authoritative
  `norito.md` specification while the portal table is being populated.
- **Torii OpenAPI** – `/reference/torii-openapi` renders the latest Torii REST specification using
  Redoc. Regenerate the spec with `npm run sync-openapi -- --version=current --latest` (add
  `--mirror=<label>` to copy the snapshot into additional historical versions).
- **Configuration tables** – The full parameter catalog is kept in
  `docs/source/references/configuration.md`. Until the portal ships an auto-import, reference that
  Markdown file for exact defaults and environment overrides.
- **Docs versioning** – The navbar version dropdown exposes frozen snapshots created with
  `npm run docs:version -- <label>`, making it easy to compare guidance across releases.

## Coming soon

- **Torii REST reference** – OpenAPI definitions will be synced into this section via
  `docs/portal/scripts/sync-openapi.mjs` once the pipeline is enabled.

  will land here alongside canonical examples.
- **IVM ABI tables** – The pointer-type and syscall matrices (maintained under `crates/ivm/docs`)

## Keeping this index current

When new reference material is added—generated API docs, codec specs, configuration matrices—place
the page under `docs/portal/docs/reference/` and link it above. If a page is autogenerated, note the
sync script so contributors know how to refresh it. This keeps the reference tree useful until the
fully autogenerated navigation lands.

# Norito კოდეკის მითითება

Norito არის Iroha-ის კანონიკური სერიალიზაციის ფენა. ყოველი საკაბელო შეტყობინება, დისკზე
payload და ჯვარედინი კომპონენტის API იყენებს Norito, ამიტომ კვანძები თანხმდებიან იდენტურ ბაიტებზე
მაშინაც კი, როდესაც ისინი მუშაობენ სხვადასხვა აპარატურაზე. ეს გვერდი აჯამებს მოძრავ ნაწილებს
და მიუთითებს სრულ სპეციფიკაციებზე `norito.md`-ში.

## ძირითადი განლაგება

| კომპონენტი | დანიშნულება | წყარო |
| --- | --- | --- |
| **Header** | ფრეიმების დატვირთვა ჯადოსნური/ვერსიის/სქემის ჰეშით, CRC64, სიგრძით და შეკუმშვის ტეგით; v1 მოითხოვს `VERSION_MINOR = 0x00` და ამოწმებს სათაურის დროშებს მხარდაჭერილი ნიღბის წინააღმდეგ (ნაგულისხმევი `0x00`). | `norito::header` — იხილეთ `norito.md` („Header & Flags“, საცავი) |
| ** შიშველი დატვირთვა ** | დეტერმინისტული მნიშვნელობის კოდირება, რომელიც გამოიყენება ჰეშირებისთვის/შედარებისთვის. სადენიანი ტრანსპორტი ყოველთვის იყენებს სათაურს; შიშველი ბაიტები მხოლოდ შიდაა. | `norito::codec::{Encode, Decode}` |
| ** შეკუმშვა ** | სურვილისამებრ Zstd (და ექსპერიმენტული GPU აჩქარება) შერჩეული სათაურის შეკუმშვის ბაიტის მეშვეობით. | `norito.md`, „შეკუმშვის მოლაპარაკება“ |

განლაგების დროშის რეესტრი (შეფუთული-struct, packed-seq, field bitsset, კომპაქტური
სიგრძე) ცხოვრობს `norito::header::flags`-ში. V1 არის ნაგულისხმევი დროშები `0x00` მაგრამ
იღებს აშკარა სათაურის დროშებს მხარდაჭერილი ნიღბის ფარგლებში; უცნობი ბიტებია
უარყოფილი. `norito::header::Flags` ინახება შიდა შემოწმებისთვის და
მომავალი ვერსიები.

## მიიღეთ მხარდაჭერა

`norito_derive` აგზავნის `Encode`, `Decode`, `IntoSchema` და JSON დამხმარე მომდინარეობს.
ძირითადი კონვენციები:

- წარმოშობის გენერირება ორივე AoS და შეფუთული კოდი ბილიკები; v1 ნაგულისხმევია AoS
  განლაგება (დროშები `0x00`), თუ სათაურის დროშები არ აირჩევენ შეფუთულ ვარიანტებს.
  დანერგვა მუშაობს `crates/norito_derive/src/derive_struct.rs`-ში.
- განლაგებაზე მოქმედი ფუნქციები (`packed-struct`, `packed-seq`, `compact-len`) არის
  მონაწილეობა სათაურის დროშებით და უნდა იყოს კოდირებული/გაშიფრული თანმიმდევრულად თანატოლებში.
- JSON დამხმარეები (`norito::json`) უზრუნველყოფენ დეტერმინისტულ Norito-ით მხარდაჭერილ JSON-ს
  გახსენით API. გამოიყენეთ `norito::json::{to_json_pretty, from_json}` — არასოდეს `serde_json`.

## მულტიკოდეკი და საიდენტიფიკაციო ცხრილები

Norito ინახავს თავის მულტიკოდეკ დავალებებს `norito::multicodec`-ში. მითითება
ცხრილი (ჰეშები, გასაღების ტიპები, დატვირთვის აღწერები) ინახება `multicodec.md`-ში
საცავის ძირში. როდესაც დაემატება ახალი იდენტიფიკატორი:

1. განაახლეთ `norito::multicodec::registry`.
2. ცხრილის გაფართოება `multicodec.md`-ში.
3. განაახლეთ ქვედა დინების აკინძები (პითონი/ჯავა), თუ ისინი მოიხმარენ რუკას.

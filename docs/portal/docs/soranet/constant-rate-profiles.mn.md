SNNet-17B нь тогтмол хурдтай тээврийн эгнээ нэвтрүүлдэг тул реле нь 1024 В эсийн хөдөлгөөнийг үл харгалзан хөдөлгөөнд оруулдаг.
даацын хэмжээ. Операторууд гурван урьдчилсан тохиргооноос сонгоно:

- **үндсэн** - дата төв эсвэл мэргэжлийн зохион байгуулалттай реле нь >=30 Mbps-ийг хамрах боломжтой
  замын хөдөлгөөн.
- **home** - орон сууцны болон бага холболттой операторууд нь нэрээ нууцлах шаардлагатай хэвээр байна
  нууцлалын чухал хэлхээнүүд.
- **null** - SNNet-17A2 туршилтын хувилбар. Энэ нь ижил TLV/дугтуйг хадгалах боловч сунгадаг
  бага зурвасын өргөнтэй үе шатанд зориулсан хачиг ба тааз.

## Урьдчилан тохируулсан хураангуй

| Профайл | Шалг (ms) | Нүд (B) | эгнээний таг | Хуурамч шал | Нэг эгнээний ачаалал (Мб/с) | Таазны ачаалал (Мб/с) | Таазны% дээш холбоос | Санал болгож буй холболт (Mb/s) | Хөрш малгай | Триггерийг автоматаар идэвхгүй болгох (%) |
|---------|-----------|----------|----------|-------------|-------------------------|--------------------|--------------------|--------------------------------|
| үндсэн | 5.0 | 1024 | 12 | 4 | 1.64 | 19.50 | 65 | 30.0 | 8 | 85 |
| гэр | 10.0 | 1024 | 4 | 2 | 0.82 | 4.00 | 40 | 10.0 | 2 | 70 |
| null | 20.0 | 1024 | 2 | 1 | 0.41 | 0.75 | 15 | 5.0 | 1 | 55 |

- **Lane cap** - хамгийн их зэрэгцээ тогтмол хурдтай хөршүүд. Реле нь нэмэлт хэлхээг нэг удаа татгалздаг
  таг цохиж `soranet_handshake_capacity_reject_total` нэмэгдэнэ.
- **Хуурамгай шал** - бодит үед ч дамми хөдөлгөөнд амьд үлдэх хамгийн бага эгнээний тоо
  эрэлт бага байна.
- **Таазны ачааллыг** - таазыг хэрэглэсний дараа тогтмол хурдтай эгнээнд зориулагдсан дээд холбоосын төсөв
  бутархай. Нэмэлт зурвасын өргөн боломжтой байсан ч операторууд энэ төсвөөс хэзээ ч хэтрэх ёсгүй.
- **Триггерийг автоматаар идэвхгүй болгох** - тогтвортой ханалтын хувь (урьдчилан тохируулсан дундаж)
  дамми давхарт унахын тулд ажиллах хугацаа. Сэргээх босгоны дараа хүчин чадал сэргээгддэг
  (`core`-д 75%, `home`-д 60%, `null`-д 45%).

**Чухал:** `null` урьдчилсан тохируулга нь зөвхөн үе шат болон туршилтад зориулагдсан; хангадаггүй
үйлдвэрлэлийн хэлхээнд шаардлагатай нууцлалын баталгаа.

## Шалз -> зурвасын өргөн хүснэгт

Ачааны үүр бүр нь 1,024 В агуулдаг тул КиБ/сек багана нь тус бүрээс ялгарах эсийн тоотой тэнцүү байна.
хоёрдугаарт. Туслагчийг ашиглан ширээг тусгай хачигтай сунгана уу.

| Шалг (ms) | Нүд/сек | Ачаалал КиБ/сек | Ачаа Mb/s |
|----------|-----------|-----------------|--------------|
| 5.0 | 200.00 | 200.00 | 1.64 |
| 7.5 | 133.33 | 133.33 | 1.09 |
| 10.0 | 100.00 | 100.00 | 0.82 |
| 15.0 | 66.67 | 66.67 | 0.55 |
| 20.0 | 50.00 | 50.00 | 0.41 |

Томъёо:

```
payload_mbps = (cell_bytes x 8 / 1_000_000) x (1000 / tick_ms)
```

CLI туслах:

```bash
# Markdown table output for all presets plus default tick table
cargo xtask soranet-constant-rate-profile --tick-table --format markdown --json-out artifacts/soranet/constant_rate/report.json

# Restrict to a preset and emit JSON
cargo xtask soranet-constant-rate-profile --profile core --format json

# Custom tick series
cargo xtask soranet-constant-rate-profile --tick-table --tick-values 5,7.5,12,18 --format markdown
```

`--format markdown` нь урьдчилан тохируулсан хураангуй болон нэмэлт хачиг хуурч мэхлэхийн тулд GitHub маягийн хүснэгтүүдийг гаргадаг.
хуудасны тусламжтайгаар та детерминистик гаралтыг портал руу оруулах боломжтой. Үүнийг архивлахын тулд `--json-out`-тэй хослуул
засаглалын нотлох баримт болгон өгсөн өгөгдөл.

## Тохиргоо ба хүчингүй болгох

`tools/soranet-relay` нь тохиргооны файлууд болон ажиллах хугацааны хүчингүй болгох аль алиных нь урьдчилан тохируулгыг ил гаргадаг:

```bash
# Persisted in relay.json
"constant_rate_profile": "home"

# One-off override during rollout or maintenance
soranet-relay --config relay.json --constant-rate-profile core
```

Тохируулгын түлхүүр нь `core`, `home`, эсвэл `null` (өгөгдмөл `core`) -ийг хүлээн зөвшөөрдөг. CLI хүчингүй болгох нь ашигтай
Тохиргоог дахин бичихгүйгээр ажлын мөчлөгийг түр хугацаагаар бууруулах бэлтгэл сургуулилт эсвэл SOC хүсэлт.

## MTU хамгаалалтын хашлага

- Ачааны үүрүүд нь 1,024 B дээр нэмэх нь ~96 B-аас Norito+Шуу чимээний хүрээ болон хамгийн бага QUIC/UDP толгойг ашигладаг,
  датаграм бүрийг IPv6 1,280 B хамгийн бага MTU-аас доогуур байлгах.
- Хонгилууд (WireGuard/IPsec) нэмэлт бүрхүүл нэмэх үед та `padding.cell_size`-ийг **заавал ** багасгах хэрэгтэй.
  тиймээс `cell_size + framing <= 1,280 B`. Реле баталгаажуулагч нь хэрэгжүүлдэг
  `padding.cell_size <= 1,136 B` (1,280 B - 48 B UDP/IPv6 нэмэлт зардал - 96 B хүрээ).
- `core` профайл нь сул зогсолттой байсан ч гэсэн >=4 хөршийг бэхлэх ёстой тул дамми эгнээ нь үргэлж дэд багцыг хамардаг.
  PQ хамгаалагчид. `home` профайл нь түрийвч/агрегаторын тогтмол хурдтай хэлхээг хязгаарлаж болох боловч хэрэглэх ёстой
  3 телеметрийн цонхны ханалт 70%-иас хэтэрсэн үед арын даралт.

## Телеметр ба сэрэмжлүүлэг

Реле нь дараах хэмжүүрүүдийг урьдчилан тохируулсан болгон экспортолдог:

- `soranet_constant_rate_active_neighbors`
- `soranet_constant_rate_queue_depth`
- `soranet_constant_rate_saturation_percent`
- `soranet_constant_rate_dummy_lanes` / `soranet_constant_rate_dummy_ratio`
- `soranet_constant_rate_slot_rate_hz`
- `soranet_constant_rate_ceiling_hits_total`
- `soranet_constant_rate_degraded`

Дараах үед анхааруулах:

1. Дамми харьцаа нь урьдчилан тохируулсан шалнаас (`core >= 4/8`, `home >= 2/2`, `null >= 1/1`) доогуур байна.
   хоёр цонх.
2. `soranet_constant_rate_ceiling_hits_total` таван минут тутамд нэг цохилтоос хурдан өсдөг.
3. Төлөвлөсөн өрөмдлөгийн гадна `soranet_constant_rate_degraded` `1` руу шилждэг.

Урьдчилан тогтоосон шошго болон хөршийн жагсаалтыг тохиолдлын тайланд тэмдэглэснээр аудиторууд тогтмол хурдтай байгааг нотлох боломжтой
бодлого нь замын зургийн шаардлагад нийцсэн.

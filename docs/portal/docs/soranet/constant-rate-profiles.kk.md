SNNet-17B бекітілген жылдамдықты тасымалдау жолақтарын енгізеді, осылайша реле 1024 B ұяшықтарында трафикті жылжытады.
пайдалы жүк мөлшері. Операторлар үш алдын ала орнатудан таңдайды:

- **core** - қамту үшін >=30 Мбит/с арнай алатын деректер орталығы немесе кәсіби орналастырылған релелер
  қозғалыс.
- **үй** - әлі де жасырын іздеуді қажет ететін тұрғын немесе төмен байланыс операторлары
  құпиялылық-критикалық схемалар.
- **null** - SNNet-17A2 сынақ нұсқасының алдын ала орнатылған нұсқасы. Ол бірдей TLV/конверттерді сақтайды, бірақ ұзартады
  белгішесі мен төбесі төмен өткізу қабілеттілігімен қойылымға арналған.

## Алдын ала орнатылған қорытынды

| Профиль | Белгі (мс) | (В) ұяшығы | Жолақ қақпағы | Жалған еден | Бір жолақты пайдалы жүктеме (Мб/с) | Төбенің пайдалы жүктемесі (Мб/с) | Төбенің % жоғары қарай | Ұсынылатын жоғары сілтеме (Мб/с) | Көрші қалпақ | Триггерді автоматты түрде өшіру (%) |
|---------|-----------|----------|----------|-------------|-------------------------|-------------------|--------------------|--------------------------------|--------------------|--------------------|
| негізгі | 5.0 | 1024 | 12 | 4 | 1,64 | 19.50 | 65 | 30,0 | 8 | 85 |
| үй | 10.0 | 1024 | 4 | 2 | 0,82 | 4.00 | 40 | 10.0 | 2 | 70 |
| null | 20.0 | 1024 | 2 | 1 | 0,41 | 0,75 | 15 | 5.0 | 1 | 55 |

- **Жол қақпағы** - максималды бір мезгілде тұрақты жылдамдықты көршілер. Реле қосымша тізбектерді бір рет қабылдамайды
  қақпақ соғылады және `soranet_handshake_capacity_reject_total` ұлғаяды.
- **Мөлдір еден** - нақты кезде де жалған трафикпен тірі қалатын жолақтардың ең аз саны
  сұраныс төмен.
- **Төбенің пайдалы жүктемесі** - төбені қолданғаннан кейін тұрақты жылдамдықты жолақтарға арналған жоғары сілтеме бюджеті
  бөлшек. Қосымша өткізу қабілеттілігі болса да, операторлар бұл бюджетті ешқашан асырмауы керек.
- **Триггерді автоматты түрде өшіру** - тұрақты қанықтылық пайызы (алдын ала орнатылған орташа есеппен)
  бос еденге түсіру үшін жұмыс уақыты. Сыйымдылық қалпына келтіру шегінен кейін қалпына келтіріледі
  (`core` үшін 75%, `home` үшін 60%, `null` үшін 45%).

**Маңызды:** `null` алдын ала орнату тек кезеңге және сынақ мүмкіндігіне арналған; сәйкес келмейді
өндірістік тізбектер үшін қажетті құпиялылық кепілдіктері.

## Белгі -> өткізу қабілеттілігі кестесі

Әрбір пайдалы жүктеме ұяшығы 1024 В тасымалдайды, сондықтан КиБ/сек бағаны бір адамға шығарылатын ұяшықтар санына тең.
екінші. Кестені теңшелетін белгілермен кеңейту үшін көмекшіні пайдаланыңыз.

| Белгі (мс) | Ұяшықтар/сек | Жүктеме КиБ/сек | Пайдалы жүктеме Мб/с |
|----------|-----------|-----------------|--------------|
| 5.0 | 200.00 | 200.00 | 1,64 |
| 7.5 | 133,33 | 133,33 | 1.09 |
| 10.0 | 100.00 | 100.00 | 0,82 |
| 15.0 | 66,67 | 66,67 | 0,55 |
| 20.0 | 50.00 | 50.00 | 0,41 |

Формула:

```
payload_mbps = (cell_bytes x 8 / 1_000_000) x (1000 / tick_ms)
```

CLI көмекшісі:

```bash
# Markdown table output for all presets plus default tick table
cargo xtask soranet-constant-rate-profile --tick-table --format markdown --json-out artifacts/soranet/constant_rate/report.json

# Restrict to a preset and emit JSON
cargo xtask soranet-constant-rate-profile --profile core --format json

# Custom tick series
cargo xtask soranet-constant-rate-profile --tick-table --tick-values 5,7.5,12,18 --format markdown
```

`--format markdown` GitHub стиліндегі кестелерді алдын ала орнатылған жиынтық үшін де, қосымша құсбелгі қою үшін де шығарады
парақ арқылы детерминирленген нәтижені порталға қоюға болады. Мұрағаттау үшін оны `--json-out` арқылы жұптаңыз
басқару дәлелі үшін берілген деректер.

## Конфигурация және қайта анықтау

`tools/soranet-relay` конфигурация файлдарында да, орындалу уақытын қайта анықтауда да алдын ала орнатуларды көрсетеді:

```bash
# Persisted in relay.json
"constant_rate_profile": "home"

# One-off override during rollout or maintenance
soranet-relay --config relay.json --constant-rate-profile core
```

Конфигурациялау кілті `core`, `home` немесе `null` (әдепкі `core`) қабылдайды. CLI қайта анықтаулары үшін пайдалы
конфигурацияларды қайта жазусыз жұмыс циклін уақытша қысқартатын кезеңдік жаттығулар немесе SOC сұраулары.

## MTU қоршаулары

- Пайдалы жүктеме ұяшықтары 1 024 B плюс ~ 96 B Norito+ Шу жақтауын және ең аз QUIC/UDP тақырыптарын пайдаланады,
  әрбір датаграмманы IPv6 1,280 B ең аз MTU мәнінен төмен ұстау.
- Туннельдер (WireGuard/IPsec) қосымша инкапсуляцияны қосқанда, `padding.cell_size` мөлшерін **азайту керек**
  сондықтан `cell_size + framing <= 1,280 B`. Реле валидаторы күшіне енеді
  `padding.cell_size <= 1,136 B` (1,280 B - 48 B UDP/IPv6 үстеме - 96 B жақтау).
- `core` профильдері бос жүрсе де >=4 көршіні бекітуі керек, сондықтан жалған жолдар әрқашан келесілердің жиынын қамтиды
  PQ күзетшілері. `home` профильдері әмияндар/агрегаторлар үшін тұрақты жылдамдықты тізбектерді шектей алады, бірақ қолданылуы керек
  үш телеметриялық терезе үшін қанықтығы 70% асқан кезде кері қысым.

## Телеметрия және ескертулер

Релелер алдын ала орнату үшін келесі көрсеткіштерді экспорттайды:

- `soranet_constant_rate_active_neighbors`
- `soranet_constant_rate_queue_depth`
- `soranet_constant_rate_saturation_percent`
- `soranet_constant_rate_dummy_lanes` / `soranet_constant_rate_dummy_ratio`
- `soranet_constant_rate_slot_rate_hz`
- `soranet_constant_rate_ceiling_hits_total`
- `soranet_constant_rate_degraded`

Ескерту:

1. Жалған қатынасы алдын ала орнатылған еденнен (`core >= 4/8`, `home >= 2/2`, `null >= 1/1`) төмен тұрады.
   екі терезе.
2. `soranet_constant_rate_ceiling_hits_total` бес минутта бір соққыдан жылдамырақ өседі.
3. `soranet_constant_rate_degraded` жоспарланған бұрғылаудан тыс `1` түріне ауысады.

Аудиторлар тұрақты жылдамдықты дәлелдей алуы үшін алдын ала орнатылған белгіні және көршілер тізімін оқиға есептеріне жазып алыңыз
саясаты жол картасы талаптарына сәйкес келді.

SNNet-17B-ն ներկայացնում է ֆիքսված փոխադրման գծեր, որպեսզի ռելեները տեղափոխեն երթևեկությունը 1024 B բջիջներում՝ անկախ նրանից
օգտակար բեռի չափը. Օպերատորները ընտրում են երեք նախադրյալներից.

- **միջուկ** - տվյալների կենտրոն կամ մասնագիտորեն տեղակայված ռելեներ, որոնք կարող են հատկացնել >=30 Մբիթ/վրկ ծածկելու համար
  երթեւեկությունը.
- **տուն** - բնակելի կամ ցածր կապի օպերատորներ, որոնք դեռ անանուն բեռների կարիք ունեն
  գաղտնիության համար կարևոր սխեմաներ:
- **null** - SNNet-17A2 փորձնական նախադրյալը: Այն պահպանում է նույն TLV-ները/ծրարը, բայց ձգում է
  տիզ և առաստաղ ցածր թողունակությամբ բեմադրման համար:

## Նախադրված ամփոփում

| Անձնագիր | Նշում (մս) | Բջջ (B) | Գոտու գլխարկ | Կեղծ հատակ | Մեկ գծի համար օգտակար բեռ (Մբ/վ) | Առաստաղի օգտակար բեռ (Մբ/վ) | Առաստաղի % վերելքի | Առաջարկվող վերահղում (Մբ/վ) | Հարևան գլխարկ | Ավտոմատ անջատել գործարկիչը (%) |
|-----------------------------------------------------------|----------------------------------------------------------------------------------------------------- --------|--------------------------------------------------------------------------------------------------------------------------
| հիմնական | 5.0 | 1024 | 12 | 4 | 1.64 | 19.50 | 65 | 30.0 | 8 | 85 |
| տուն | 10.0 | 1024 | 4 | 2 | 0,82 | 4.00 | 40 | 10.0 | 2 | 70 |
| զրոյական | 20.0 | 1024 | 2 | 1 | 0,41 | 0,75 | 15 | 5.0 | 1 | 55 |

- **Գոտի կափարիչ** - առավելագույն միաժամանակյա մշտական ​​հարևաններ: Ռելեը մեկ անգամ մերժում է լրացուցիչ սխեմաները
  գլխարկը հարվածում է և ավելացնում `soranet_handshake_capacity_reject_total`:
- **Կեղծված հատակ** - երթևեկության նվազագույն քանակը, որոնք կենդանի են մնում կեղծ երթևեկության դեպքում, նույնիսկ երբ իրականում
  պահանջարկն ավելի ցածր է.
- **Առաստաղի ծանրաբեռնվածություն** - առաստաղը կիրառելուց հետո վերելքի բյուջեն, որը հատկացված է մշտական արագությամբ գծերին
  կոտորակ. Օպերատորները երբեք չպետք է գերազանցեն այս բյուջեն, նույնիսկ եթե առկա է լրացուցիչ թողունակություն:
- **Ավտոմատ անջատել գործարկիչը** - կայուն հագեցվածության տոկոսը (միջինը մեկ նախադրյալի համար), որն առաջացնում է
  գործարկման ժամանակը կեղծ հատակին ընկնելու համար: Հզորությունը վերականգնվում է վերականգնման շեմից հետո
  (75% `core`-ի համար, 60% `home`-ի համար, 45% `null`-ի համար):

**Կարևոր է.** `null` նախադրյալը նախատեսված է միայն բեմականացման և փորձարկման հնարավորությունների համար; այն չի համապատասխանում
արտադրական սխեմաների համար պահանջվող գաղտնիության երաշխիքներ:

## Նշել -> թողունակության աղյուսակ

Յուրաքանչյուր օգտակար բջիջ կրում է 1024 B, ուստի KiB/sec սյունակը հավասար է մեկ արտանետվող բջիջների քանակին:
երկրորդ. Օգտագործեք օգնականը, որպեսզի երկարացնեք սեղանը հատուկ տիզերով:

| Նշում (մս) | Բջիջներ/վրկ | Բեռնատար KiB/վրկ | Օգտակար բեռ Մբ/վ |
|-----------|-----------|----------------|--------------|
| 5.0 | 200.00 | 200.00 | 1.64 |
| 7.5 | 133.33 | 133.33 | 1.09 |
| 10.0 | 100.00 | 100.00 | 0,82 |
| 15.0 | 66.67 | 66.67 | 0,55 |
| 20.0 | 50.00 | 50.00 | 0,41 |

Բանաձև:

```
payload_mbps = (cell_bytes x 8 / 1_000_000) x (1000 / tick_ms)
```

CLI օգնական.

```bash
# Markdown table output for all presets plus default tick table
cargo xtask soranet-constant-rate-profile --tick-table --format markdown --json-out artifacts/soranet/constant_rate/report.json

# Restrict to a preset and emit JSON
cargo xtask soranet-constant-rate-profile --profile core --format json

# Custom tick series
cargo xtask soranet-constant-rate-profile --tick-table --tick-values 5,7.5,12,18 --format markdown
```

`--format markdown`-ը թողարկում է GitHub-ի ոճի աղյուսակներ և՛ նախադրված ամփոփագրի, և՛ ընտրովի խաբեության համար
թերթիկ, որպեսզի կարողանաք դետերմինիստական ելք տեղադրել պորտալում: Զույգացրեք այն `--json-out`-ի հետ՝ արխիվացնելու համար
ներկայացված տվյալները կառավարման ապացույցների համար:

## Կազմաձևում և անտեսում

`tools/soranet-relay`-ը բացահայտում է նախադրյալները և՛ կազմաձևման ֆայլերում, և՛ գործարկման ժամանակի անտեսումներում.

```bash
# Persisted in relay.json
"constant_rate_profile": "home"

# One-off override during rollout or maintenance
soranet-relay --config relay.json --constant-rate-profile core
```

Կազմաձևի ստեղնն ընդունում է `core`, `home` կամ `null` (կանխադրված `core`): CLI-ի փոխարինումները օգտակար են
բեմական վարժանքներ կամ SOC հարցումներ, որոնք ժամանակավորապես նվազեցնում են աշխատանքային ցիկլը՝ առանց կոնֆիգուրացիաները վերաշարադրելու:

## MTU պահակաձողեր

- Բեռնատար բջիջները օգտագործում են 1,024 B գումարած ~96 B Norito+ Աղմուկի շրջանակը և նվազագույն QUIC/UDP վերնագրերը,
  յուրաքանչյուր տվյալների գրամ պահելով IPv6 1280 B նվազագույն MTU-ից ցածր:
- Երբ թունելները (WireGuard/IPsec) ավելացնում են լրացուցիչ պարկուճ, դուք **պետք է** կրճատեք `padding.cell_size`
  ուրեմն `cell_size + framing <= 1,280 B`. Ռելեի վավերացուցիչը պարտադրում է
  `padding.cell_size <= 1,136 B` (1,280 B - 48 B UDP/IPv6 վերադիր - 96 B շրջանակում):
- `core` պրոֆիլները պետք է ամրացնեն >=4 հարևաններ, նույնիսկ երբ անգործության են մատնված, այնպես որ կեղծ գոտիները միշտ ծածկում են ենթաբազմություն
  PQ պահակներ. `home` պրոֆիլները կարող են սահմանափակել հաստատուն արագությամբ սխեմաները դրամապանակների/ագրեգատորների համար, սակայն պետք է կիրառվեն
  հետադարձ ճնշում, երբ հագեցվածությունը գերազանցում է 70%-ը երեք հեռաչափական պատուհանների համար:

## Հեռուստաչափություն և ահազանգեր

Ռելեները արտահանում են հետևյալ չափորոշիչները յուրաքանչյուր նախադրյալի համար.

- `soranet_constant_rate_active_neighbors`
- `soranet_constant_rate_queue_depth`
- `soranet_constant_rate_saturation_percent`
- `soranet_constant_rate_dummy_lanes` / `soranet_constant_rate_dummy_ratio`
- `soranet_constant_rate_slot_rate_hz`
- `soranet_constant_rate_ceiling_hits_total`
- `soranet_constant_rate_degraded`

Զգուշացում, երբ՝

1. Կեղծ հարաբերակցությունը մնում է նախադրված հատակից ցածր (`core >= 4/8`, `home >= 2/2`, `null >= 1/1`) ավելի քան
   երկու պատուհան.
2. `soranet_constant_rate_ceiling_hits_total`-ն ավելի արագ է աճում, քան մեկ հարվածը հինգ րոպեում:
3. `soranet_constant_rate_degraded`-ը շրջվում է դեպի `1`՝ պլանավորված փորվածքից դուրս:

Գրանցեք նախադրված պիտակը և հարևանների ցուցակը միջադեպերի հաշվետվություններում, որպեսզի աուդիտորները կարողանան ապացուցել կայուն դրույքաչափը
քաղաքականությունը համապատասխանում էր ճանապարհային քարտեզի պահանջներին:

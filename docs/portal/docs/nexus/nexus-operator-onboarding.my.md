# Sora Nexus Data-Space အော်ပရေတာ စတင်အသုံးပြုခြင်း။

ဤလမ်းညွှန်ချက်သည် ထုတ်ဝေမှုကို ကြေညာပြီးသည်နှင့် အဆုံးမှအဆုံးသို့ စီးဆင်းမှုကို Sora Nexus ဒေတာ-အာကာသ အော်ပရေတာများက ဖမ်းယူသည်။ ၎င်းသည် ဒေါင်းလုဒ်လုပ်ထားသော အစုအဝေး/ပုံများ၊ သရုပ်ဖော်ပုံများနှင့် ဖွဲ့စည်းမှုပုံစံများကို အွန်လိုင်းပေါ်မတင်မီ ကမ္ဘာလုံးဆိုင်ရာလမ်းကြောဆိုင်ရာ မျှော်လင့်ချက်များနှင့်အတူ ဒေါင်းလုဒ်လုပ်ထားသော အစုအဝေးများ/ပုံများကို မည်သို့ချိန်ညှိရမည်ကို ဖော်ပြခြင်းဖြင့် ၎င်းသည် နှစ်လမ်းသွား runbook (`docs/source/release_dual_track_runbook.md`) နှင့် artefact ရွေးချယ်မှုမှတ်စု (`docs/source/release_artifact_selection.md`) ကို ဖြည့်စွက်ပေးပါသည်။

## ပရိသတ် & ကြိုတင်လိုအပ်ချက်များ
- သင့်အား Nexus ပရိုဂရမ်မှ အတည်ပြုထားပြီး သင်၏ဒေတာနေရာတာဝန် (လမ်းကြောအညွှန်းကိန်း၊ ဒေတာနေရာ ID/အမည်လွဲများနှင့် လမ်းပြမူဝါဒလိုအပ်ချက်များ) ကို လက်ခံရရှိထားပါသည်။
- Release Engineering (tarballs၊ ရုပ်ပုံများ၊ သရုပ်ဖော်ပုံများ၊ လက်မှတ်များ၊ အများသူငှာသော့များ) မှထုတ်ဝေသော လက်မှတ်ရေးထိုးထားသော ထုတ်ဝေပစ္စည်းများကို သင်ဝင်ရောက်ကြည့်ရှုနိုင်ပါသည်။
- သင့်တရားဝင်စစ်ဆေးသူ/လေ့လာသူအခန်းကဏ္ဍအတွက် အဓိကပစ္စည်းထုတ်လုပ်ခြင်း သို့မဟုတ် လက်ခံရရှိခြင်း (Ed25519 node အထောက်အထား၊ BLS အများသဘောတူသော့ + PoP အတည်ပြုသူများအတွက်၊ နှင့် လျှို့ဝှက်အင်္ဂါရပ်များ ခလုတ်နှိပ်ခြင်း)။
- သင့် node ကို bootstrap လုပ်မည့် လက်ရှိ Sora Nexus ရွယ်တူများထံ သင်ရောက်ရှိနိုင်ပါသည်။

## အဆင့် 1 — ထုတ်ဝေမှု ပရိုဖိုင်ကို အတည်ပြုပါ။
1. သင်ပေးထားသည့် ကွန်ရက်အမည်များ သို့မဟုတ် ကွင်းဆက် ID ကို ခွဲခြားသတ်မှတ်ပါ။
2. ဤသိုလှောင်မှု၏ငွေရှင်းခြင်းတွင် `scripts/select_release_profile.py --network <alias>` (သို့မဟုတ် `--chain-id <id>`) ကိုဖွင့်ပါ။ အကူအညီပေးသူက `release/network_profiles.toml` ကို တိုင်ပင်ပြီး အသုံးပြုရန် ပရိုဖိုင်ကို ပရင့်ထုတ်သည်။ Sora Nexus အတွက် တုံ့ပြန်မှုသည် `iroha3` ဖြစ်ရပါမည်။ အခြားတန်ဖိုးများအတွက်၊ ရပ်တန့်ပြီး ဖြန့်ချိရေးအင်ဂျင်နီယာထံ ဆက်သွယ်ပါ။
3. ရည်ညွှန်းထားသော ထုတ်ပြန်ကြေငြာချက်ကို ဗားရှင်း tag ကို မှတ်သားပါ (ဥပမာ `iroha3-v3.2.0`); ရှေးဟောင်းပစ္စည်းများနှင့် သရုပ်ပြမှုများ ရယူရန် ၎င်းကို သင်အသုံးပြုမည်ဖြစ်သည်။

## အဆင့် 2 — ရှေးဟောင်းပစ္စည်းများကို ပြန်လည်ရယူပြီး အတည်ပြုပါ။
1. `iroha3` အစုအဝေး (`<profile>-<version>-<os>.tar.zst`) နှင့် ၎င်း၏တွဲဖက်ဖိုင်များ (`.sha256`၊ ရွေးချယ်နိုင်သော `.sig/.pub`၊ `<profile>-<version>-manifest.json` နှင့် I18NI000028X) ကို ဒေါင်းလုဒ်လုပ်ပါ။
2. ထုပ်ပိုးခြင်းမပြုမီ ခိုင်မာမှုကို စစ်ဆေးအတည်ပြုပါ-
   ```bash
   sha256sum -c iroha3-<version>-linux.tar.zst.sha256
   openssl dgst -sha256 -verify iroha3-<version>-linux.tar.zst.pub \
       -signature iroha3-<version>-linux.tar.zst.sig \
       iroha3-<version>-linux.tar.zst
   ```
   အကယ်၍ သင်သည် ဟာ့ဒ်ဝဲ-ကျောထောက်နောက်ခံပြု KMS ကိုအသုံးပြုပါက `openssl` ကို အဖွဲ့အစည်းမှခွင့်ပြုထားသော အတည်ပြုစနစ်ဖြင့် အစားထိုးပါ။
3. `PROFILE.toml` ကို တာလုံးအတွင်းပိုင်းကို စစ်ဆေးပြီး အတည်ပြုရန် JSON က ဖော်ပြသည်-
   - `profile = "iroha3"`
   - `version`၊ `commit` နှင့် `built_at` အကွက်များသည် ထုတ်ဝေသည့် ကြေညာချက်နှင့် ကိုက်ညီပါသည်။
   - OS/architecture သည် သင်၏ ဖြန့်ကျက်ပစ်မှတ်နှင့် ကိုက်ညီပါသည်။
4. သင်သည် ကွန်တိန်နာပုံအား အသုံးပြုပါက၊ `<profile>-<version>-<os>-image.tar` အတွက် hash/signature verification ကို ပြန်လုပ်ကာ `<profile>-<version>-image.json` တွင် မှတ်တမ်းတင်ထားသော ပုံ ID ကို အတည်ပြုပါ။

## အဆင့် 3 — နမူနာပုံစံများမှ အဆင့်သတ်မှတ်မှု
1. အစုအဝေးကို ဖြည်ပြီး `config/` ကို node သည် ၎င်း၏ဖွဲ့စည်းပုံကို ဖတ်မည့်နေရာသို့ ကူးယူပါ။
2. `config/` အောက်ရှိ ဖိုင်များကို နမူနာများအဖြစ် ဆက်ဆံပါ-
   - သင်၏ထုတ်လုပ်မှု Ed25519 သော့များဖြင့် `public_key`/`private_key` ကို အစားထိုးပါ။ Node သည် HSM မှ ၎င်းတို့ကို အရင်းအမြစ်ပေးမည်ဆိုပါက ဒစ်ခ်မှ သီးသန့်သော့များကို ဖယ်ရှားပါ။ ၎င်းအစား HSM ချိတ်ဆက်ကိရိယာကို ညွှန်ရန် config ကို အပ်ဒိတ်လုပ်ပါ။
   - `trusted_peers`၊ `network.address`၊ နှင့် `torii.address` ကို ချိန်ညှိခြင်းဖြင့် ၎င်းတို့သည် သင်၏ လက်လှမ်းမီနိုင်သော အင်တာဖေ့စ်များနှင့် သင်သတ်မှတ်ပေးထားသည့် bootstrap ရွယ်တူများကို ထင်ဟပ်စေပါသည်။
   - အော်ပရေတာ-ရင်ဆိုင်နေရသော Torii အဆုံးမှတ်ဖြင့် `client.toml` ကို အပ်ဒိတ်လုပ်ပါ (လိုအပ်ပါက TLS ဖွဲ့စည်းမှုပုံစံအပါအဝင်) နှင့် လုပ်ငန်းလည်ပတ်မှုဆိုင်ရာကိရိယာများအတွက် သင်ပံ့ပိုးပေးသောအထောက်အထားများကို အပ်ဒိတ်လုပ်ပါ။
3. အုပ်ချုပ်ရေးက အခြားနည်းဖြင့် အတိအလင်း ညွှန်ကြားမထားပါက အစုအဝေးတွင် ပေးထားသည့် ကွင်းဆက် ID ကို ထားရှိပါ—ကမ္ဘာလုံးဆိုင်ရာ လမ်းကြောသည် canonical chain identifier တစ်ခုတည်းကို မျှော်လင့်ထားသည်။
4. Sora ပရိုဖိုင်အလံ- `irohad --sora --config <path>` ဖြင့် node ကို စတင်ရန် စီစဉ်ပါ။ အလံမရှိသည့်အခါ ပြင်ဆင်သတ်မှတ်ခြင်း loader သည် SoraFS သို့မဟုတ် လမ်းသွားများစွာဆက်တင်များကို ငြင်းပယ်ပါမည်။

## အဆင့် 4 — data-space metadata နှင့် routing ကို ချိန်ညှိပါ။
1. `config/config.toml` ကို တည်းဖြတ်ပါ `[nexus]` အပိုင်းသည် Nexus ကောင်စီမှ ပေးထားသော data-space catalog နှင့် ကိုက်ညီသည်-
   - `lane_count` သည် လက်ရှိခေတ်တွင် ဖွင့်ထားသော စုစုပေါင်းလမ်းကြောင်းများနှင့် ညီမျှရပါမည်။
   - `[[nexus.lane_catalog]]` နှင့် `[[nexus.dataspace_catalog]]` တွင် ဝင်ရောက်မှုတိုင်းတွင် ထူးခြားသော `index`/`id` နှင့် သဘောတူထားသည့် နာမည်တူများ ပါဝင်ရပါမည်။ လက်ရှိ ကမ္ဘာလုံးဆိုင်ရာ ထည့်သွင်းမှုများကို မဖျက်ပါနှင့်။ ကောင်စီက နောက်ထပ်ဒေတာနေရာလွတ်များကို သတ်မှတ်ပေးမည်ဆိုပါက သင်၏လွှဲအပ်ထားသော အမည်နာမများကို ပေါင်းထည့်ပါ။
   - dataspace တစ်ခုစီတွင် `fault_tolerance (f)` ပါဝင်ကြောင်း သေချာပါစေ။ လမ်းကြောဖြတ်ကျော်ရေးကော်မတီများသည် `3f+1` တွင် အရွယ်အစားရှိသည်။
2. သင်ပေးအပ်ထားသောမူဝါဒကိုဖမ်းယူရန် `[[nexus.routing_policy.rules]]` ကို အပ်ဒိတ်လုပ်ပါ။ ပုံသေပုံစံပလိတ်သည် လမ်းကြော `1` သို့ အုပ်ချုပ်မှုလမ်းညွှန်ချက်များကို လမ်းကြောင်းပေးပြီး `2` သို့ ဖြန့်ကျက်စာချုပ်ချုပ်ဆိုထားသည်။ သင့်ဒေတာနေရာအတွက် သတ်မှတ်ထားသော ယာဉ်ကြောအသွားအလာကို မှန်ကန်သောလမ်းကြောနှင့် နာမည်အမည်များဆီသို့ ထပ်ဆင့်ပို့ခြင်း သို့မဟုတ် စည်းမျဉ်းများကို ထပ်ဖြည့်ပါ သို့မဟုတ် ပြင်ဆင်ပါ။ စည်းကမ်းအမိန့်မပြောင်းမီ ဖြန့်ချိရေးအင်ဂျင်နီယာနှင့် ညှိနှိုင်းပါ။
3. `[nexus.da]`၊ `[nexus.da.audit]`၊ နှင့် `[nexus.da.recovery]` ကန့်သတ်ချက်များကို ပြန်လည်သုံးသပ်ပါ။ အော်ပရေတာများသည် ကောင်စီမှ အတည်ပြုထားသော တန်ဖိုးများကို ထိန်းသိမ်းထားရန် မျှော်လင့်ပါသည်။ မွမ်းမံထားသောမူဝါဒကို အတည်ပြုပြီးမှသာ ၎င်းတို့ကို ချိန်ညှိပါ။
4. သင်၏ လည်ပတ်မှု ခြေရာခံစနစ်တွင် နောက်ဆုံးပုံစံကို မှတ်တမ်းတင်ပါ။ တေးသွားနှစ်ပုဒ်ထွက်ရှိထားသော ပြေးစာအုပ်သည် လက်မှတ်တွင် ထိရောက်သော `config.toml` (လျှို့ဝှက်ချက်များ ပြန်လည်ပြင်ဆင်ထားသည်) ကို ပူးတွဲထည့်သွင်းရန် လိုအပ်သည်။

## အဆင့် 5 — လေယာဉ်အကြိုအတည်ပြုခြင်း။
1. ကွန်ရက်မ၀င်မီ built-in configuration validator ကို run ပါ-
   ```bash
   ./bin/irohad --sora --config config/config.toml --trace-config
   ```
   ၎င်းသည် ဖြေရှင်းပြီးသား ဖွဲ့စည်းမှုပုံစံကို ပရင့်ထုတ်ပြီး ကက်တလောက်/လမ်းကြောင်းတင်ခြင်းဆိုင်ရာ အချက်အလက်များ မကိုက်ညီပါက သို့မဟုတ် ဥပါဒ်နှင့် ဖွဲ့စည်းမှုသဘောမတူပါက အစောပိုင်းတွင် ပျက်ကွက်ပါသည်။
2. အကယ်၍ သင်သည် ကွန်တိန်နာများကို အသုံးချပါက၊ ၎င်းကို `docker load -i <profile>-<version>-<os>-image.tar` ဖြင့် တင်ပြီးနောက် ပုံအတွင်း၌ တူညီသောအမိန့်ကို လုပ်ဆောင်ပါ (`--sora` ကို ထည့်သွင်းရန် သတိရပါ)။
3. placeholder lane/data-space identifiers များအကြောင်း သတိပေးချက်များအတွက် မှတ်တမ်းများကို စစ်ဆေးပါ။ တစ်စုံတစ်ရာ ပေါ်လာပါက အဆင့် 4 ကို ပြန်လည်ကြည့်ရှုပါ—ထုတ်လုပ်ရေး ဖြန့်ကျက်မှုများသည် နမူနာပုံစံများနှင့်အတူ ပို့ဆောင်ပေးသည့် နေရာယူသူ ID များအပေါ် အားမကိုးရပါ။
4. သင့်ဒေသခံ မီးခိုးထုတ်လုပ်ငန်းစဉ်ကို လုပ်ဆောင်ပါ (ဥပမာ၊ `FindNetworkStatus` ဖြင့် မေးမြန်းချက်ကို `iroha_cli` ဖြင့် ပေးပို့ပါ၊ တယ်လီမီတာ အဆုံးမှတ်များကို အတည်ပြုပါ `nexus_lane_state_total` ကို ဖော်ထုတ်ပြီး streaming ခလုတ်များကို လိုအပ်သလို လှည့်ပတ်ထားသည် သို့မဟုတ် တင်သွင်းကြောင်း အတည်ပြုပါ)။

## အဆင့် 6 — ဖြတ်တောက်ခြင်းနှင့် လက်လွှဲခြင်း။
1. စစ်ဆေးပြီးသော `manifest.json` နှင့် လက်မှတ်ရေးပစ္စည်းများကို ထုတ်ဝေခွင့်လက်မှတ်တွင် သိမ်းဆည်းထားပါက စာရင်းစစ်များသည် သင့်ချက်လက်မှတ်များကို ပြန်လည်ထုတ်ဝေနိုင်စေရန်။
2. node ကိုမိတ်ဆက်ရန်အဆင်သင့်ဖြစ်ပြီဟု Nexus လုပ်ဆောင်ချက်များအား အသိပေးပါ။ ပါဝင်သည်-
   - Node အထောက်အထား (ရွယ်တူ ID၊ လက်ခံဆောင်ရွက်ပေးသူအမည်များ၊ Torii အဆုံးမှတ်)။
   - ထိရောက်သောလမ်းကြော/ဒေတာ-နေရာကွက်ကတ်တလောက်နှင့် လမ်းကြောင်းသတ်မှတ်ခြင်းဆိုင်ရာ မူဝါဒတန်ဖိုးများ။
   - သင်အတည်ပြုထားသော binaries/ ပုံများ၏ Hashes များ။
3. `@nexus-core` ဖြင့် နောက်ဆုံးသက်တူရွယ်တူ ဝင်ခွင့် ( အတင်းအဖျင်းမျိုးစေ့များနှင့် လမ်းသွားတာဝန်) ကို ညှိနှိုင်းပါ။ သင်ခွင့်ပြုချက်မရရှိမချင်း ကွန်ရက်တွင် မပါဝင်ပါနှင့်။ Sora Nexus သည် အဆုံးအဖြတ်ပေးသော လမ်းသွားနေရာယူမှုကို တွန်းအားပေးပြီး အပ်ဒိတ်လုပ်ထားသောဝင်ခွင့်များကို ဖော်ပြရန်လိုအပ်ပါသည်။
4. node ကို တိုက်ရိုက်လွှင့်ပြီးနောက်၊ သင်မိတ်ဆက်ထားသော အစားထိုးမှုမှန်သမျှဖြင့် သင့် runbooks များကို အပ်ဒိတ်လုပ်ကာ ထုတ်ဝေမှုတက်ဂ်ကို မှတ်သားထားခြင်းဖြင့် နောက်ထပ်ပြန်လုပ်ခြင်းအား ဤအခြေခံလိုင်းမှ စတင်နိုင်ပါသည်။

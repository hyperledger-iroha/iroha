# Sora အမည် ဝန်ဆောင်မှု အုပ်ချုပ်ရေး Playbook (SN-6)

**အခြေအနေ-** ရေးဆွဲထားသော 2026-03-24 — SN-1/SN-6 အဆင်သင့်အတွက် အသက်ရှင်ကိုးကား  
**လမ်းပြမြေပုံလင့်ခ်များ-** SN-6 “လိုက်နာမှုနှင့် အငြင်းပွားမှုဖြေရှင်းချက်”၊ SN-7 “ဖြေရှင်းချက်နှင့် ဂိတ်ဝေးစင့်ခ်”၊ ADDR-1/ADDR-5 လိပ်စာမူဝါဒ  
**ကြိုတင်လိုအပ်ချက်များ-** [`registry-schema.md`](./registry-schema.md)၊ [`registrar-api.md`](`registrar-api.md`](`registrar-api.md`) ရှိ မှတ်ပုံတင်အရာရှိ API စာချုပ်](./registrar-api.md)၊ လိပ်စာ UX လမ်းညွှန်ချက် [`address-display-guidelines.md`](./address-display-guidelines.md) နှင့် [`docs/account_structure.md`](https://github.com/hyperledger-iroha/iroha/blob/master/docs/account_structure.md) ရှိ အကောင့်ဖွဲ့စည်းပုံစည်းမျဉ်းများ။

ဤပြခန်းစာအုပ်တွင် Sora အမည်ဝန်ဆောင်မှု (SNS) အုပ်ချုပ်ရေးအဖွဲ့များက မည်သို့ကျင့်သုံးသည်ကို ဖော်ပြသည်။
ပဋိညာဉ်စာတမ်းများ၊ မှတ်ပုံတင်ခြင်းကို အတည်ပြုပါ၊ အငြင်းပွားမှုများ တိုးလာကာ ဖြေရှင်းသူနှင့် သက်သေပြပါ။
ဂိတ်ဝေးပြည်နယ်များသည် တစ်ပြိုင်တည်းရှိနေပါသည်။ ၎င်းသည် လမ်းပြမြေပုံလိုအပ်ချက်ကို ဖြည့်ဆည်းပေးသည်။
`sns governance ...` CLI၊ Norito သည် ထင်ရှားပြီး စာရင်းစစ်ပစ္စည်းများ တစ်ခုတည်းကို မျှဝေသည်
N1 (အများပြည်သူ စတင်ရောင်းချခြင်း) မတိုင်မီ အော်ပရေတာ-မျက်နှာကိုးကား။

## 1. နယ်ပယ် & ပရိသတ်

စာတမ်း၏ပစ်မှတ်များမှာ-

- စည်းမျဉ်းစည်းကမ်းများကိုမဲပေးခြင်း၊ နောက်ဆက်တွဲမူဝါဒများနှင့်အငြင်းပွားနေသောအုပ်ချုပ်ရေးကောင်စီအဖွဲ့ဝင်များ
  ရလဒ်များ
- အရေးပေါ်အေးခဲမှုများနှင့်ပြောင်းပြန်လှန်မှုများကိုပြန်လည်သုံးသပ်သည့် Guardian ဘုတ်အဖွဲ့ဝင်များ။
- မှတ်ပုံတင်အရာရှိ တန်းစီဇယားများကို လုပ်ဆောင်ခြင်း၊ လေလံတင်ခြင်းကို အတည်ပြုပြီး စီမံခန့်ခွဲသော ဘဏ္ဍာစိုးများ
  ဝင်ငွေခွဲဝေမှု။
- SoraDNS ပြန့်ပွားမှု၊ GAR အပ်ဒိတ်များအတွက် တာဝန်ရှိသော ဖြေရှင်းသူ/ဂိတ်ဝေးအော်ပရေတာများ၊
  နှင့် telemetry guardrails ။
- လိုက်နာမှု ၊ ဘဏ္ဍာတိုက် နှင့် ပံ့ပိုးကူညီရေး အဖွဲ့များ ၊
  အုပ်ချုပ်မှုလုပ်ဆောင်ချက်သည် စစ်ဆေးနိုင်သော Norito ပစ္စည်းများ ကျန်ခဲ့သည်။

၎င်းသည် အပိတ်ဘီတာ (N0)၊ အများသူငှာ လွှင့်တင်ခြင်း (N1) နှင့် တိုးချဲ့မှု (N2) အဆင့်များ ပါဝင်သည်။
အလုပ်အသွားအလာတစ်ခုစီကို လိုအပ်သောအထောက်အထားများနှင့် ချိတ်ဆက်ခြင်းဖြင့် `roadmap.md` တွင် ရေတွက်ခြင်း၊
ဒက်ရှ်ဘုတ်များ၊ နှင့် အရှိန်မြှင့်လမ်းကြောင်းများ။

### 4.2 မှတ်ပုံတင်ခြင်း၊ လေလံတင်ခြင်းနှင့် ဈေးနှုန်းခွင့်ပြုချက်များ

1. **Preflight:** စျေးနှုန်းအဆင့်ကို အတည်ပြုရန် မှတ်ပုံတင်အရာရှိ မေးမြန်းချက် `SuffixPolicyV1`၊
   ရရှိနိုင်သော စည်းကမ်းချက်များ နှင့် ကျေးဇူးတော်/ရွေးနှုတ်မှု ပြတင်းပေါက်များ။ စျေးနှုန်းစာရွက်များကို ထပ်တူကျအောင်ထားပါ။
   မှတ်တမ်းတင်ထားသော 3/4/5/6–9/10+ အဆင့်ဇယား (အခြေခံအဆင့် + နောက်ဆက်ကိန်းများ)
   လမ်းပြမြေပုံ။
2. **အလုံပိတ်-လေလံလေလံပွဲများ-** ပရီမီယံရေကန်များအတွက် 72h commit / 24h ဖော်ပြမှုကို လုပ်ဆောင်ပါ။
   `sns governance auction commit` / `... reveal` မှတဆင့် လည်ပတ်သည်။ ကတိကဝတ်ကိုထုတ်ပြန်ပါ။
   `artifacts/sns/auctions/<name>/commit.json` အောက်တွင် (hashes များသာ) စာရင်း
   စာရင်းစစ်များသည် ကျပန်းစစ်ဆေးနိုင်သည်။
3. **ငွေပေးချေမှုအတည်ပြုခြင်း-** မှတ်ပုံတင်သူများသည် `PaymentProofV1` ကိုတရားဝင်စစ်ဆေးသည်
   ဘဏ္ဍာတိုက်ခွဲများ (70% treasury / 30% ဘဏ္ဍာစိုး ≤10% လွှဲပြောင်းပေးခြင်း)။
   Norito JSON ကို `artifacts/sns/payments/<tx>.json` အောက်တွင် သိမ်းဆည်းပြီး ၎င်းကို လင့်ခ်ချိတ်ပါ။
   မှတ်ပုံတင်အရာရှိတုံ့ပြန်ချက် (`RevenueAccrualEventV1`)။
4. **အုပ်ချုပ်မှုချိတ်-** ပရီမီယံ/စောင့်ကြပ်ထားသောအမည်များအတွက် `GovernanceHookV1` ကို ပူးတွဲပါ
   ကောင်စီအဆိုပြုချက် ID များနှင့် ဘဏ္ဍာစိုးလက်မှတ်များကို ကိုးကားခြင်း။ ချိတ်ရလဒ်ပျောက်နေသည်။
   `sns_err_governance_missing` တွင်
5. **Activation + solver sync:** Torii သည် registry event ကို ထုတ်သည်နှင့်၊ အစပျိုးသည်
   GAR/zone state အသစ်ကို အတည်ပြုရန် ဖြေရှင်းသူ ပွင့်လင်းမြင်သာမှု tailer
   (§၄.၅ ကိုကြည့်ပါ)။
6. **ဖောက်သည်ထုတ်ဖော်ခြင်း-** ဖောက်သည်-ရင်ဆိုင်နေရသော စာရင်းဇယား (ပိုက်ဆံအိတ်/ရှာဖွေသူ) ကို အပ်ဒိတ်လုပ်ပါ။
   [`address-display-guidelines.md`](./address-display-guidelines.md) ရှိ မျှဝေထားသော ပစ္စည်းများမှတစ်ဆင့် I105 နှင့်
   ချုံ့ထားသော တင်ဆက်မှုများသည် ကော်ပီ/QR လမ်းညွှန်ချက်နှင့် ကိုက်ညီသည်။

### 4.3 သက်တမ်းတိုးခြင်း၊ ငွေတောင်းခံခြင်းနှင့် ငွေစာရင်းပြန်လည်ညှိနှိုင်းခြင်း။- **သက်တမ်းတိုးခြင်းလုပ်ငန်း-** မှတ်ပုံတင်သူများသည် ရက် 30 ကျေးဇူးတော် + ရက် 60 ရွေးနှုတ်ခြင်းကို ကျင့်သုံးသည်
  `SuffixPolicyV1` တွင် သတ်မှတ်ထားသော windows။ ရက်ပေါင်း 60 ကြာပြီးနောက် ဒတ်ခ်ျသည် အစီအစဉ်ကို ပြန်ဖွင့်သည်။
  (7 ရက်၊ 10× အခကြေးငွေ 15%/day ဆွေးမြေ့နေသည်) `sns အုပ်ချုပ်ရေးမှတစ်ဆင့် အလိုအလျောက် အစပျိုးသည်
  ပြန်ဖွင့်သည်။
- **ဝင်ငွေခွဲဝေမှု-** သက်တမ်းတိုးခြင်း သို့မဟုတ် လွှဲပြောင်းမှုတစ်ခုစီသည် တစ်ခုစီ ဖန်တီးပေးသည်။
  `RevenueAccrualEventV1`။ ငွေတိုက်ပို့ကုန်များ (CSV/ပါကေး) မှ ပြန်လည်သင့်မြတ်ရပါမည်။
  ဤဖြစ်ရပ်များနေ့စဉ်; `artifacts/sns/treasury/<date>.json` သို့ အထောက်အထားများ ပူးတွဲပါ ။
- **ရည်ညွှန်းကိုးကားချက်-ထွက်များ-** ရွေးချယ်နိုင်သော လွှဲပြောင်းမှုရာခိုင်နှုန်းများကို နောက်ဆက်တွဲတစ်ခုလျှင် ခြေရာခံသည်
  `referral_share` ကို ဘဏ္ဍာစိုးမူဝါဒတွင် ထည့်သွင်းခြင်းဖြင့်။ မှတ်ပုံတင်အရာရှိများက အပြီးအပြတ်ထုတ်သည်။
  ငွေပေးချေမှု အထောက်အထားဘေးတွင် ခွဲခြမ်းပြီး လွှဲပြောင်းပေးသည့် သက်သေကို သိမ်းဆည်းပါ။
- ** အစီရင်ခံခြင်းပုံစံ-** ဘဏ္ဍာရေးဆိုင်ရာ ပို့စ်များသည် လစဉ် KPI နောက်ဆက်တွဲများ (မှတ်ပုံတင်ခြင်း၊
  သက်တမ်းတိုးခြင်း၊ ARPU၊ အငြင်းပွားမှု/စာချုပ် အသုံးချခြင်း) အောက်တွင်
  `docs/source/sns/regulatory/<suffix>/YYYY-MM.md`။ ဒက်ရှ်ဘုတ်များမှ ဆွဲထုတ်သင့်သည်။
  တူညီသောတင်ပို့သည့်ဇယားများဖြစ်သောကြောင့် Grafana နံပါတ်များသည် စာရင်းဇယားအထောက်အထားများနှင့် ကိုက်ညီပါသည်။
- **လစဉ် KPI ပြန်လည်သုံးသပ်ခြင်း-** ပထမ-အင်္ဂါနေ့ စစ်ဆေးရေးဂိတ်သည် ဘဏ္ဍာရေးဆိုင်ရာ ဦးဆောင်သူကို တွဲပေးပါသည်။
  တာဝန်ကျဘဏ္ဍာစိုးနှင့် အစီအစဉ် PM။ [SNS KPI ဒက်ရှ်ဘုတ်](./kpi-dashboard.md) ကိုဖွင့်ပါ။
  (`sns-kpis` / `dashboards/grafana/sns_suffix_analytics.json` ၏ portal မြှုပ်နှံမှု)၊
  မှတ်ပုံတင်အရာရှိ၏ ဖြတ်တောက်မှု + ဝင်ငွေဇယားများ၊ နောက်ဆက်တွဲရှိ မှတ်တမ်း deltas ကို တင်ပို့ပါ၊
  နှင့် ရှေးဟောင်းပစ္စည်းများကို မှတ်စုတိုတွင် ပူးတွဲပါ။ ပြန်လည်သုံးသပ်တွေ့ရှိပါက အဖြစ်အပျက်တစ်ခုကို စတင်ပါ။
  SLA ချိုးဖောက်မှုများ (ပြတင်းပေါက်များ > 72 နာရီကို အေးခဲစေခြင်း၊ မှတ်ပုံတင်အရာရှိ အမှားအယွင်းများ တိုးလာခြင်း၊ ARPU ပျံ့လွင့်ခြင်း)။

### 4.5 Resolver & Gateway Propagation

1. **Event hook:** registry event တိုင်းသည် ဖြေရှင်းသူ event stream သို့ ထုတ်လွှတ်သည်။
   (`tools/soradns-resolver` SSE)။ Resolver ops မှတဆင့် စာရင်းသွင်းပြီး ကွဲပြားမှုများကို မှတ်တမ်းတင်ပါ။
   ပွင့်လင်းမြင်သာမှု tailer (`scripts/telemetry/run_soradns_transparency_tail.sh`)။
2. **GAR နမူနာပုံစံ အပ်ဒိတ်-** Gateways မှ ကိုးကားထားသော GAR နမူနာပုံစံများကို အပ်ဒိတ်လုပ်ရပါမည်။
   `canonical_gateway_suffix()` နှင့် `host_pattern` စာရင်းကို ပြန်လည်လက်မှတ်ထိုးပါ။ စတိုးဆိုင်ကွဲပြားမှုများ
   `artifacts/sns/gar/<date>.patch` တွင်
3. **Zonefile ထုတ်ဝေခြင်း-** တွင်ဖော်ပြထားသော zonefile အရိုးစုကို အသုံးပြုပါ။
   `roadmap.md` (အမည်၊ ttl၊ cid၊ အထောက်အထား) နှင့် Torii/SoraFS သို့ တွန်းပါ။ သိမ်းဆည်းပါ။
   Norito JSON `artifacts/sns/zonefiles/<name>/<version>.json` အောက်။
4. **Transparency check-** `promtool test rules dashboards/alerts/tests/soradns_transparency_rules.test.yml` ကိုဖွင့်ပါ။
   သတိပေးချက်များ စိမ်းလန်းနေစေရန်။ Prometheus စာသားအထွက်ကို အဆိုပါသို့ ပူးတွဲပါ။
   အပတ်စဉ် ပွင့်လင်းမြင်သာမှု အစီရင်ခံစာ။
5. **ဂိတ်ဝေးစစ်ဆေးမှု-** မှတ်တမ်း `Sora-*` ခေါင်းစီးနမူနာများ (ကက်ရှ်မူဝါဒ၊ CSP၊ GAR
   digest) နှင့် ၎င်းတို့ကို အုပ်ချုပ်မှုမှတ်တမ်းတွင် ပူးတွဲပါရှိစေရန် အော်ပရေတာများက သက်သေပြနိုင်သည်။
   gateway သည် ရည်ရွယ်ထားသော guardrails ဖြင့် အမည်အသစ်ကို ပေးဆောင်ခဲ့သည်။

`ops/drill-log.md` တွင် ပူးတွဲပါရှိသည့် ပစ္စည်းများနှင့် ဖြေရှင်းသူ ပြတ်တောက်မှု)။

## 8. Incident Response & Escalation

| အစပျိုး | ပြင်းထန်မှု | လက်ငင်းပိုင်ရှင် | မဖြစ်မနေ လုပ်ဆောင်ချက်များ |
|---------|----------------|-----------------|--------------------|
| ဖြေရှင်းချက်/GAR ပျံ့လွင့်ခြင်း သို့မဟုတ် ဟောင်းနွမ်းနေသောအထောက်အထားများ | Sev1 | ဖြေရှင်းချက် SRE + အုပ်ထိန်းဘုတ်အဖွဲ့ | စာမျက်နှာဖြေရှင်းပေးသူသည် ဖုန်းခေါ်ဆိုမှုတွင်၊ tailer output ကိုဖမ်းယူရန်၊ သက်ရောက်မှုရှိသောအမည်များကို ရပ်တန့်ရန် ဆုံးဖြတ်ရန်၊ မိနစ် 30 တိုင်း အခြေအနေကို အပ်ဒိတ်တင်ပါ။ |
| မှတ်ပုံတင်အရာရှိပြတ်တောက်မှု၊ ငွေတောင်းခံမှုမအောင်မြင်ခြင်း သို့မဟုတ် နေရာအနှံ့ API အမှားအယွင်းများ | Sev1 | မှတ်ပုံတင်အရာရှိ တာဝန်မန်နေဂျာ | လေလံပွဲအသစ်များကို ရပ်ပါ၊ လက်စွဲ CLI သို့ပြောင်းပါ၊ ဘဏ္ဍာစိုး/ဘဏ္ဍာတိုက်ကို အကြောင်းကြားပါ၊ Torii မှတ်တမ်းများကို အဖြစ်အပျက်ဆိုင်ရာ စာရွက်စာတမ်းသို့ ပူးတွဲပါ။ |
| အမည်တစ်ခုတည်း အငြင်းပွားမှု၊ ငွေပေးချေမှုမတူညီမှု၊ သို့မဟုတ် ဖောက်သည်တိုးလာခြင်း | Sev2 | ဘဏ္ဍာစိုး + ထောက်ခဲ | ငွေပေးချေမှုအထောက်အထားများကိုစုဆောင်းပါ၊ ပျော့ပျောင်းသောအေးခဲခြင်းရှိမရှိဆုံးဖြတ်ပါ၊ SLA အတွင်းတောင်းဆိုသူကိုတုံ့ပြန်ပါ၊ အငြင်းပွားမှုခြေရာခံကိရိယာတွင်ရလဒ်ကိုမှတ်တမ်းတင်ပါ။ |
| လိုက်နာမှုစာရင်းစစ်ရှာဖွေခြင်း | Sev2 | ဆက်ဆံရေးရုံး | ပြန်လည်ပြင်ဆင်ရေးအစီအစဉ်မူကြမ်း၊ `docs/source/sns/regulatory/` အောက်တွင် ဖိုင်မှတ်စုတို၊ နောက်ဆက်တွဲကောင်စီအစည်းအဝေးကို အချိန်ဇယားဆွဲပါ။ |
| လေ့ကျင့်မှု သို့မဟုတ် အစမ်းလေ့ကျင့်မှု | Sev3 | အစီအစဉ် PM | `ops/drill-log.md` မှ scripted scenario၊ archive artefacts၊ label ကွာဟမှုများကို လမ်းပြမြေပုံလုပ်ဆောင်စရာများအဖြစ် လုပ်ဆောင်ပါ။ |

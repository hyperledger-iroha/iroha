# SORA Nexus 开发者门户

此目录托管交互式开发人员的 Docusaurus 工作区
门户网站。该门户汇集了 Norito 指南、SDK 快速入门和 OpenAPI
由 `cargo xtask openapi` 生成的引用，将它们包装在 SORA Nexus 中
整个文档计划中使用的品牌。

## 先决条件

- Node.js 18.18 或更高版本（Docusaurus v3 基线）。
- Yarn 1.x 或 npm ≥ 9 用于包管理。
- Rust 工具链（由 OpenAPI 同步脚本使用）。

## 引导程序

```bash
cd docs/portal
npm install    # or yarn install
```

## 可用脚本

|命令|描述 |
|---------|-------------|
| `npm run start` / `yarn start` |启动具有实时重新加载功能的本地开发服务器（默认为 `http://localhost:3000`）。 |
| `npm run build` / `yarn build` |在 `build/` 中生成生产版本。 |
| `npm run serve` / `yarn serve` |在本地提供最新版本（对于冒烟测试有用）。 |
| `npm run docs:version -- <label>` |将当前文档快照到 `versioned_docs/version-<label>`（围绕 `docusaurus docs:version` 的包装）。 |
| `npm run sync-openapi` / `yarn sync-openapi` |通过 `cargo xtask openapi` 重新生成 `static/openapi/torii.json`（通过 `--mirror=<label>` 将规范复制到其他版本快照中）。 |
| `npm run tryit-proxy` |启动为“Try it”控制台提供支持的临时代理（请参阅下面的配置）。 |
| `npm run probe:tryit-proxy` |针对代理（CI/监控帮助程序）发出 `/healthz` + 示例请求探测。 |
| `npm run manage:tryit-proxy -- <update|rollback>` |通过备份处理更新或恢复代理 `.env` 目标。 |
| `npm run sync-i18n` |确保 `i18n/` 下存在日语、希伯来语、西班牙语、葡萄牙语、法语、俄语、阿拉伯语、乌尔都语、缅甸语、格鲁吉亚语、亚美尼亚语、阿塞拜疆语、哈萨克语、巴什基尔语、阿姆哈拉语、宗卡语、乌兹别克语、蒙古语、繁体中文和简体中文翻译存根。 |
| `npm run sync-norito-snippets` |重新生成精选的 Kotodama 示例文档 + 可下载的片段（也由开发服务器插件自动调用）。 |
| `npm run test:tryit-proxy` |通过 Node 的测试运行程序 (`node --test`) 执行代理单元测试。 |

OpenAPI 同步脚本要求 `cargo xtask openapi` 可从
存储库根；它将确定性 JSON 文件发送到 `static/openapi/` 中
现在期望 Torii 路由器公开实时规范（使用
`cargo xtask openapi --allow-stub` 仅用于紧急占位符输出）。

## 文档版本控制和 OpenAPI 快照

- **剪切文档版本：** 运行 `npm run docs:version -- 2025-q3` （或任何商定的标签）。
  提交生成的`versioned_docs/version-<label>`、`versioned_sidebars`、
  和 `versions.json`。导航栏版本下拉列表会自动显示
  新快照。
- **同步 OpenAPI 工件：** 剪切版本后，刷新规范
  通过 `cargo xtask openapi --sign <path-to-ed25519-key>` 进行规范和清单，然后
  捕获匹配的快照
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`。的
  脚本写入 `static/openapi/versions/2025-q3/torii.json`，反映规范
  进入 `versions/current/torii.json`，更新 `versions.json`，刷新
  `/openapi/torii.json`，并将签名的 `manifest.json` 克隆到每个版本中
  目录，因此历史规格具有相同的出处元数据。供应任何
  将新生成的规范复制到的 `--mirror=<label>` 标志的数量
  其他历史快照。
- **CI 期望：** 提交相关文档应包含版本提升
  （如果适用）加上刷新的 OpenAPI 快照，以便 Swagger、RapiDoc、
  Redoc 面板可以在历史规格之间切换，而不会出现获取错误。
- **清单执行：** `sync-openapi` 脚本现在在签名时失败
  `manifest.json` 缺失、格式错误或与新生成的不匹配
  规范，因此默认情况下无法发布未签名的快照。重新运行
  `cargo xtask openapi --sign <key>` 刷新规范清单并
  重新运行同步，以便版本化快照获取签名的元数据。
  仅针对本地预览传递 `--allow-unsigned`（CI 仍运行
  `ci/check_openapi_spec.sh`，重新生成规范并验证
  在允许提交合并之前清单）。

### 尝试代理配置

“Try it”沙箱通过 `scripts/tryit-proxy.mjs` 路由请求。配置
在启动代理之前使用环境变量：

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN`（默认`127.0.0.1:8787`）控制绑定地址。
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` 配置内存
  速率限制器（默认为每 60 秒 60 个请求）。
- 设置 `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1` 转发主叫的 `Authorization`
  header 而不是依赖默认的承载令牌。
- 如果 `static/openapi/torii.json` 偏离，则代理拒绝启动
  在 `static/openapi/manifest.json` 中签署清单。运行
  `npm run sync-openapi -- --latest` 刷新规格；出口
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` 仅用于紧急超越（警告
  已记录并且代理无论如何都会启动）。
- 运行 `npm run probe:tryit-proxy` 对登台代理进行冒烟测试并连接
  指挥监控工作； `npm run manage:tryit-proxy -- update` 简化
  轮换 Torii 端点，同时保留 `.env` 备份以进行回滚。
- `TRYIT_PROXY_PROBE_METRICS_FILE` 和 `TRYIT_PROXY_PROBE_LABELS` 让探头
  以 Prometheus 文本文件格式发出 `probe_success`/`probe_duration_seconds`
  您可以将运行状况检查挂接到 node_exporter 或任何批处理收集器中。
- Prometheus 刮擦暴露 `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  直方图系列；文档门户 Grafana 仪表板将它们用于 p95/p99 延迟
  SLO 跟踪 (`dashboards/grafana/docs_portal.json`)。

### OAuth 设备代码登录

当门户暴露在受信任网络外部时，配置 OAuth 设备
授权，因此审阅者永远不会接触长期存在的 Torii 令牌。导出
运行 `npm run start` 或 `npm run build` 之前的变量如下：

|变量|笔记|
|----------|--------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` |实现设备授权的 OAuth 端点。 |
| `DOCS_OAUTH_CLIENT_ID` |为文档预览注册的客户端 ID。 |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` |用于限制颁发的令牌的可选范围/受众字符串。 |
| `DOCS_OAUTH_POLL_INTERVAL_MS` |门户小部件强制执行的最小轮询间隔（默认为 5000 毫秒；较低的值将被拒绝）。 |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` |当服务器省略 `expires_in` 时使用的回退到期窗口。 |
| `DOCS_OAUTH_ALLOW_INSECURE` |仅针对本地开发设置为 `1` 以绕过构建时防护。如果缺少必需的变量或 TTL 超出强制范围，生产构建就会失败。 |

示例：

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

构建获取这些值后，Try it 控制台会呈现登录信息
显示设备代码、轮询令牌端点、填充承载的面板
自动字段，并在过期时清除令牌。门户网站拒绝
当 OAuth 配置不完整或令牌/设备 TTL 下降时启动
超出安全预算；仅导出 `DOCS_OAUTH_ALLOW_INSECURE=1`
可接受匿名访问的一次性本地预览。代理仍然
遵循手动 `X-TryIt-Auth` 覆盖，因此您可以删除 OAuth 变量
每当您喜欢粘贴临时令牌时。

请参阅 [`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md) 了解
完整的威胁模型和渗透测试门。

### 安全标头

`docusaurus.config.js` 现在发出确定性安全标头 — 内容安全
策略、可信类型、权限策略和引用者策略 - 静态主机
继承与开发服务器相同的护栏。默认只允许脚本
从门户源提供服务并将跨源 `connect-src` 流量限制为
配置的分析端点和 Try it 代理。生产构建拒绝
非 HTTPS 分析/试用端点；针对 `http://` 的本地预览
目标，明确设置 `DOCS_SECURITY_ALLOW_INSECURE=1` 以便护栏
承认降级。

## CI 和部署

- `ci/check_docs_portal.sh` 首先运行 `node scripts/check-sdk-recipes.mjs` 以验证 SDK 账本配方与其规范源文件匹配，然后执行 `npm ci|install`、`npm run build`，并断言关键部分（主页、SoraFS、Norito、发布清单）存在在生成的 HTML 中。
- `ci/check_openapi_spec.sh` 通过 `cargo xtask openapi` 重新生成 Torii 规范，
  将其与 `static/openapi/torii.json` 和 `versions/current/torii.json` 进行比较，
  并验证校验和清单，以便 PR 在跟踪规范或哈希时失败
  已经过时了。
- `.github/workflows/check-docs.yml` 在每个 PR 上运行门户构建以捕获内容回归。
- `.github/workflows/docs-portal-preview.yml` 为拉取请求构建站点，写入 `build/checksums.sha256`，通过 `sha256sum -c` 进行验证，将输出打包为 `artifacts/preview-site.tar.gz`，通过 `scripts/generate-preview-descriptor.mjs` 生成描述符，并上传静态站点和元数据工件，以便审阅者可以审核确切的快照，而无需重新运行构建。该工作流程还发布了一个拉取请求评论，总结了清单/存档摘要（以及 SoraFS 捆绑包，如果可用），以便审阅者无需打开操作日志即可看到验证结果。
- `.github/workflows/docs-portal-deploy.yml` 在推送到 `main`/`master` 时将静态构建发布到 GitHub Pages，在 `github-pages` 环境 URL 上提供预览。工作流程在部署之前运行登陆页面烟雾检查。
- `scripts/sorafs-package-preview.sh` 将预览站点转换为确定性 SoraFS 捆绑包（CAR、计划、清单），并且当通过 JSON 配置提供凭据时（请参阅 `docs/examples/sorafs_preview_publish.json`），可以将清单提交到暂存 Torii 端点。
- `scripts/preview_verify.sh` 执行审核者之前手动执行的校验和 + 描述符验证流程。将其指向提取的 `build/` 目录（以及可选的描述符/存档路径），以确认预览工件未被篡改。
- `scripts/sorafs-pin-release.sh` 加上 `.github/workflows/docs-portal-sorafs-pin.yml` 自动化生产 SoraFS 管道：构建/测试、CAR + 清单生成、Sigstore 签名、验证、可选别名绑定以及用于治理审核的工件上传。推广版本时通过 `workflow_dispatch` 触发工作流程。
- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]` 在签名或运送给运营商之前验证网关授权记录。该帮助程序确保规范/漂亮的主机、清单元数据和遥测标签与确定性策略匹配，并可以通过 `--json-out` 发出 DG-3 证据的 JSON 摘要。
- 当提交发生时，引脚助手还会调用 `scripts/generate-dns-cutover-plan.mjs` 来生成 `artifacts/sorafs/portal.dns-cutover.json`。设置 `DNS_CHANGE_TICKET`、`DNS_CUTOVER_WINDOW`、`DNS_HOSTNAME`、`DNS_ZONE` 和 `DNS_OPS_CONTACT`（或传递相应的 `--dns-*` 标志），以便描述符携带生产 DNS 切换所需的元数据。当需要缓存失效和回滚挂钩时，添加 `DNS_CACHE_PURGE_ENDPOINT`、`DNS_CACHE_PURGE_AUTH_ENV` 和 `DNS_PREVIOUS_PLAN`（或 `--cache-purge-*` / `--previous-dns-plan` 标志），以便描述符记录清除 API 调用和先前的还原描述符。该描述符现在还记录了装订的 `Sora-Route-Binding`（主机、CID、标头/绑定路径、验证命令），因此 GAR 升级和回退计划审查引用了在边缘提供的确切标头。
- 相同的工作流程可以通过 `scripts/sns_zonefile_skeleton.py` 发出 SNS 区域文件骨架/解析器片段。提供 IPv4/IPv6/CNAME/SPKI/TXT 元数据（作为环境变量，如 `DNS_ZONEFILE_IPV4` 或 CLI 标志，如 `--dns-zonefile-ipv4`），使用 `DNS_GAR_DIGEST` 传递 GAR 摘要，帮助程序将自动写入 `artifacts/sns/zonefiles/<zone>/<hostname>.json`（加上解析器片段），以便 SN-7 证据与切换描述符。
- DNS 描述符现在嵌入了一个 `gateway_binding` 部分，该部分引用上述工件（路径、内容 CID、证明状态和文字标头模板），因此 DG-3 变更批准包括网关上预期的确切 `Sora-Name/Sora-Proof/CSP/HSTS` 捆绑包。

每个 `npm run build` 调用都会触发 `postbuild` 钩子，该钩子生成
`build/checksums.sha256`。构建（或下载 CI 工件）后，运行
`./docs/portal/scripts/preview_verify.sh --build-dir docs/portal/build` 至
验证清单。通过 `--descriptor <path>`/`--archive <path>` 时
验证来自 CI 的元数据包，以便脚本可以交叉检查记录的数据
摘要和文件名。
- 要安全地共享本地预览，请运行 `npm run serve`；该命令现在换行
  `scripts/serve-verified-preview.mjs`，执行`preview_verify.sh`
  在启动 `docusaurus serve` 之前，如果清单检查失败或则中止
  丢失，即使在 CI 之外也能保持预览校验和门控。的
  `npm run serve:verified` 别名仍然可用于显式调用。

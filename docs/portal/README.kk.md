# SORA Nexus Әзірлеуші порталы

Бұл каталог интерактивті әзірлеушіге арналған Docusaurus жұмыс кеңістігін орналастырады.
порталы. Портал Norito нұсқаулықтарын, SDK жылдам іске қосуларын және OpenAPI біріктіреді
`cargo xtask openapi` арқылы жасалған анықтама, оларды SORA Nexus ішіне орау
docs бағдарламасында қолданылатын брендинг.

## Алғышарттар

- Node.js 18.18 немесе одан кейінгі нұсқасы (Docusaurus v3 негізгі).
- Орамды басқаруға арналған иірілген жіп 1.x немесе npm ≥ 9.
- Rust құралдар тізбегі (OpenAPI синхрондау сценарийінде пайдаланылады).

## Жүктеу белбеуі

```bash
cd docs/portal
npm install    # or yarn install
```

## Docs нұсқасы және OpenAPI суреті

- **Құжаттар нұсқасын кесу:** `npm run docs:version -- 2025-q3` (немесе кез келген келісілген тег) іске қосыңыз.
  Жасалған `versioned_docs/version-<label>`, `versioned_sidebars`,
  және `versions.json`. Navbar нұсқасының ашылмалы тізімі автоматты түрде шығады
  жаңа сурет.
- **OpenAPI артефактілерін синхрондау:** нұсқаны кескеннен кейін канондықты жаңартыңыз
  spec және `cargo xtask openapi --sign <path-to-ed25519-key>` арқылы манифест, содан кейін
  көмегімен сәйкес суретті түсіріңіз
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`. The
  сценарий `static/openapi/versions/2025-q3/torii.json` жазады, спецификацияны көрсетеді
  `versions/current/torii.json` ішіне, `versions.json` жаңартулары, жаңартулар
  `/openapi/torii.json` және қол қойылған `manifest.json` әрбір нұсқаға клондайды
  каталогы, сондықтан тарихи ерекшеліктер бірдей шығу метадеректерін алып жүреді. Кез келген жеткізіңіз
  жаңадан жасалған спецификацияны көшіру үшін `--mirror=<label>` жалауларының саны
  басқа да тарихи суреттер.
- **CI күтулері:** әсер ететін құжаттарда нұсқа қатесі болуы керек
  (қолданылатын кезде) плюс жаңартылған OpenAPI суреттері, сондықтан Swagger, RapiDoc,
  және Redoc панельдері алу қателерісіз тарихи сипаттамалар арасында ауыса алады.
- **Манифестті орындау:** `sync-openapi` сценарийі енді қол қойылған кезде орындалмайды
  `manifest.json` жоқ, дұрыс емес немесе жаңадан жасалғанға сәйкес келмейді
  spec, сондықтан қол қойылмаған суреттерді әдепкі бойынша жариялау мүмкін емес. Қайта жүгіру
  `cargo xtask openapi --sign <key>` канондық манифестті жаңарту және
  синхрондауды қайта іске қосыңыз, осылайша нұсқаланған суреттер қол қойылған метадеректерді алады.
  `--allow-unsigned` жолын тек жергілікті алдын ала қарау үшін өткізіңіз (CI әлі де жұмыс істейді
  `ci/check_openapi_spec.sh`, ол спецификацияны қалпына келтіреді және оны тексереді
  міндеттемелерді біріктіруге рұқсат бермес бұрын манифест).

### Прокси конфигурациясын қолданып көріңіз

"Байқап көру" құмсалғышы сұрауларды `scripts/tryit-proxy.mjs` арқылы бағыттайды. Конфигурациялау
оны іске қоспас бұрын орта айнымалылары бар прокси:

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN` (әдепкі `127.0.0.1:8787`) байланыстыру мекенжайын басқарады.
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` ішкі жадты конфигурациялау
  жылдамдықты шектегіш (әдепкі бойынша 60 секундта 60 сұрау).
- Қоңырау шалушының `Authorization` қайта жіберу үшін `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1` орнатыңыз
  әдепкі тасымалдаушы таңбалауышқа сенудің орнына тақырып.
- `static/openapi/torii.json` желіден ауытқып кетсе, прокси іске қосудан бас тартады.
  `static/openapi/manifest.json` манифестіне қол қойылған. Жүгіру
  спецификацияны жаңарту үшін `npm run sync-openapi -- --latest`; экспорт
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` тек төтенше жағдайларды қайта анықтау үшін (ескерту
  тіркелді және прокси бәрібір іске қосылады).
- Орналасу проксиін түтіндеп сынау және сымды сымдау үшін `npm run probe:tryit-proxy` іске қосыңыз.
  бақылау жұмыстарын жүргізуге бұйрық беру; `npm run manage:tryit-proxy -- update` жеңілдетеді
  кері қайтару үшін `.env` сақтық көшірмесін сақтай отырып, Torii соңғы нүктелерін айналдыру.
- `TRYIT_PROXY_PROBE_METRICS_FILE` және `TRYIT_PROXY_PROBE_LABELS` зондқа мүмкіндік береді
  `probe_success`/`probe_duration_seconds` Prometheus мәтіндік файл пішімінде шығарыңыз
  денсаулық тексеруін node_exporter немесе кез келген пакеттік коллекторға қосуға болады.
- Prometheus сызаттар `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  гистограмма қатары; Grafana құжаттар порталы бақылау тақтасы оларды p95/p99 кідірісі үшін тұтынады
  SLO қадағалау (`dashboards/grafana/docs_portal.json`).

### OAuth құрылғы кодының кіруі

Портал сенімді желілерден тыс ашылғанда, OAuth құрылғысын теңшеңіз
авторизация, сондықтан шолушылар ұзақ өмір сүретін Torii таңбалауыштарын ешқашан ұстамайды. экспорттау
`npm run start` немесе `npm run build` іске қосу алдында төмендегі айнымалы мәндер:

| Айнымалы | Ескертпелер |
|----------|-------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` | Құрылғыны авторизациялау грантын жүзеге асыратын OAuth соңғы нүктелері. |
| `DOCS_OAUTH_CLIENT_ID` | Құжаттарды алдын ала қарау үшін тіркелген клиент идентификаторы. |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` | Шығарылған таңбалауышты шектеу үшін қосымша ауқым/аудитория жолдары. |
| `DOCS_OAUTH_POLL_INTERVAL_MS` | Портал виджеті арқылы орындалатын ең аз сұрау аралығы (әдепкі бойынша 5000 мс; төменгі мәндер қабылданбайды). |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` | Сервер `expires_in` өткізіп жіберген кезде пайдаланылатын қайтару мерзімі аяқталатын терезелер. |
| `DOCS_OAUTH_ALLOW_INSECURE` | `1` деп тек жергілікті даму үшін орнату уақытының қорғанысын айналып өту үшін орнатыңыз. Қажетті айнымалылар жоқ болса немесе TTL күштелген ауқымдардан тыс болса, өндіріс құрастырулары сәтсіздікке ұшырайды. |

Мысалы:

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

Құрастыру осы мәндерді таңдағаннан кейін, Байқап көру консолі жүйеге кіруді көрсетеді
құрылғы кодын көрсететін, таңбалауыштың соңғы нүктесін сұрайтын, Тасымалдаушыны толтыратын панель
өрісті автоматты түрде таңдайды және оның мерзімі аяқталған кезде таңбалауышты өшіреді. Портал бас тартады
OAuth конфигурациясы аяқталмағанда немесе таңбалауыш/құрылғы TTL мәні төмендегенде басталады
қауіпсіздік бюджетінен тыс; тек `DOCS_OAUTH_ALLOW_INSECURE=1` экспорттау
анонимді қол жеткізуге болатын бір реттік жергілікті алдын ала қараулар. Прокси әлі
құрметті нұсқаулығы `X-TryIt-Auth` қайта анықтайды, сондықтан OAuth айнымалы мәндерін жоюға болады
арнайы таңбалауыштарды қоюды қалаған кезде.

[`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md) қараңыз.
қауіп-қатердің толық моделі және сынақ қақпалары.

### Қауіпсіздік тақырыптары

`docusaurus.config.js` енді детерминирленген қауіпсіздік тақырыптарын — мазмұн қауіпсіздігін шығарады
саясат, Сенімді түрлер, Рұқсаттар-Саясат және Реферер-Саясат — сондықтан статикалық хосттар
әзірлеуші ​​сервері сияқты бірдей қорғаныс рельстерін мұраға алады. Әдепкі параметрлер тек сценарийлерге рұқсат береді
порталдың түпнұсқасынан қызмет көрсетеді және `connect-src` кросс-шығу трафигін шектейді
конфигурацияланған аналитиканың соңғы нүктесін және проксиді пайдаланып көріңіз. Өндіріс құрылыстары қабылданбайды
HTTPS емес аналитика/пайдаланудың соңғы нүктелері; `http://` қарсы жергілікті алдын ала қарау үшін
мақсаттар үшін `DOCS_SECURITY_ALLOW_INSECURE=1` қоршауын анық етіп орнатыңыз
төмендегенін мойындайды.

### Локализацияның жұмыс процесі

Портал бастапқы тіл ретінде ағылшын және жапон, иврит,
Испан, португал, француз, орыс, араб, урду, бирма, грузин,
Армян, әзірбайжан, қазақ, башқұрт, амхар, дзонха, өзбек, моңғол,
Дәстүрлі қытай және жеңілдетілген қытай тілдері. Жаңа құжаттар түскен сайын,
жүгіру:

```bash
npm run sync-i18n
```

Сценарий репозиторий бойынша `scripts/sync_docs_i18n.py` әрекетін көрсетеді.
`i18n/<lang>/docusaurus-plugin-content-docs/current/...` астында нақты аудармаларды жасау.
Содан кейін редакторлар толтырғыш мәтінін ауыстырып, алдыңғы мәселені жаңарта алады
метадеректер (`status`, `translation_last_reviewed`, т.б.).

Әдепкі бойынша, өндіріс құрылымдары тізімде көрсетілген тілдерді ғана қамтиды
`docs/i18n/published_locales.json` (қазіргі уақытта тек ағылшын тілінде) сондықтан stub толтырғыштары
жөнелтілмейді. Алдын ала қарау қажет болғанда `DOCS_I18N_INCLUDE_STUBS=1` орнатыңыз
орындалып жатқан жергілікті жерлер.

## CI және орналастыру

- `ci/check_docs_portal.sh` алдымен `node scripts/check-sdk-recipes.mjs` іске қосып, SDK журналының рецептерінің олардың канондық бастапқы файлдарға сәйкестігін тексеру үшін іске қосады, содан кейін `npm ci|install`, `npm run build` орындайды және негізгі бөлімдерді бекітеді (басты бет, I18NI00000100, I18NI00000100 Norito, жариялау бақылау парағы) жасалған HTML ішінде бар.
- `ci/check_openapi_spec.sh` `cargo xtask openapi` арқылы Torii спецификациясын қалпына келтіреді,
  оны `static/openapi/torii.json` және `versions/current/torii.json` салыстырады,
  және бақылау сомасының манифестін тексереді, осылайша бақыланатын спецификация немесе хэш кезінде PR сәтсіз болады
  ескірген.
- `.github/workflows/check-docs.yml` мазмұн регрессияларын ұстау үшін әрбір PR-да портал құрастыруын іске қосады.
- `.github/workflows/docs-portal-preview.yml` тарту сұраулары үшін сайтты құрастырады, `build/checksums.sha256` жазады, оны `sha256sum -c` арқылы тексереді, `artifacts/preview-site.tar.gz` ретінде шығысты бумалайды, `artifacts/preview-site.tar.gz` арқылы дескрипторды жасайды, сондай-ақ сайтты және метастатистикалық және жүктеп салуды жүктейді. шолушылар құрастыруды қайта іске қоспай-ақ нақты суретті тексере алады. Сондай-ақ жұмыс процесі манифест/мұрағат дайджесттерін (және қол жетімді болған кезде SoraFS топтамасын) қорытындылайтын тарту-сұраныс түсініктемесін жариялайды, осылайша тексерушілер әрекеттер журналдарын ашпай тексеру нәтижесін көреді.
- `.github/workflows/docs-portal-deploy.yml` `github-pages` ортасының URL мекенжайында алдын ала қарауды қамтамасыз ете отырып, `main`/`master` итерулерінде GitHub беттеріне статикалық құрастыруды жариялайды. Жұмыс процесі орналастыру алдында қону беті түтінін тексеруді іске қосады.
- `scripts/sorafs-package-preview.sh` алдын ала қарау сайтын детерминирленген SoraFS бумасына (CAR, жоспар, манифест) түрлендіреді және тіркелгі деректері JSON конфигурациясы арқылы қамтамасыз етілгенде (`docs/examples/sorafs_preview_publish.json` қараңыз) манифестті I1800400. end.
- `scripts/preview_verify.sh` тексерушілер бұрын қолмен орындаған бақылау сомасы + дескрипторды тексеру ағынын орындайды. Алдын ала қарау артефактілерінің өзгертілмегенін растау үшін оны шығарылған `build/` каталогына (және қосымша дескриптор/мұрағат жолдары) көрсетіңіз.
- `scripts/sorafs-pin-release.sh` плюс `.github/workflows/docs-portal-sorafs-pin.yml` SoraFS өндіріс құбырын автоматтандырады: құрастыру/сынау, CAR + манифест генерациясы, Sigstore қол қою, тексеру, қосымша бүркеншік атпен байланыстыру және басқаруды тексеру үшін артефакт жүктеп салу. Шығарылымды жылжыту кезінде `workflow_dispatch` арқылы жұмыс процесін іске қосыңыз.
- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]` шлюз авторизациясының жазбаларына қол қойылғанға немесе операцияға жөнелтілгенге дейін тексереді. Көмекші канондық/әдемі хосттарды, манифест метадеректерін және телеметрия белгілерінің детерминирленген саясатқа сәйкестігін қамтамасыз етеді және `--json-out` арқылы DG-3 дәлелі үшін JSON қорытындысын шығара алады.
- Жіберу орын алған кезде, PIN көмекшісі де `artifacts/sorafs/portal.dns-cutover.json` шығару үшін `scripts/generate-dns-cutover-plan.mjs` шақырады. `DNS_CHANGE_TICKET`, `DNS_CUTOVER_WINDOW`, `DNS_HOSTNAME`, `DNS_ZONE` және `DNS_OPS_CONTACT` орнатыңыз (немесе сәйкес `--dns-*` жалаушаларын өткізіңіз), сондықтан DNS өндірісі үшін метаскрипторларды кесу қажет. Кэшті жарамсыздандыру және кері қайтару ілмектері қажет болғанда, `DNS_CACHE_PURGE_ENDPOINT`, `DNS_CACHE_PURGE_AUTH_ENV` және `DNS_PREVIOUS_PLAN` (немесе `--cache-purge-*` / `--previous-dns-plan` жалаушаларын) қосыңыз, осылайша API немесе дескрипт немесе дескрипт шақыруы реверсиялар. Дескриптор енді сонымен қатар қапсырмаланған `Sora-Route-Binding` (хост, CID, тақырып/байланыстыру жолдары, тексеру пәрмендері) құжаттайды, осылайша GAR жылжыту және резервтік жоспарды шолулар шетте қызмет ететін нақты тақырыптарға сілтеме жасайды.
- Бірдей жұмыс процесі `scripts/sns_zonefile_skeleton.py` арқылы SNS аймақтық файлының қаңқасын/резолютор үзіндісін шығаруы мүмкін. IPv4/IPv6/CNAME/SPKI/TXT метадеректерін (`DNS_ZONEFILE_IPV4` сияқты env нұсқалары немесе `--dns-zonefile-ipv4` сияқты CLI жалаулары ретінде) қамтамасыз етіңіз, `DNS_GAR_DIGEST` көмегімен GAR дайджестін өткізіңіз, сонда көмекші I01002 шешеді (I08NI02) үзінді) автоматты түрде SN-7 дәлелі кесілген дескриптордың жанында орналасады.
- DNS дескрипторы енді жоғарыдағы артефакттерге сілтеме жасайтын `gateway_binding` бөлімін (жолдар, мазмұн CID, дәлелдеу күйі және нақты тақырып үлгісі) ендіреді, сондықтан DG-3 өзгертуді мақұлдау шлюзде күтілетін нақты `Sora-Name/Sora-Proof/CSP/HSTS` бумасын қамтиды.

Әрбір `npm run build` шақыруы `postbuild` ілгегін іске қосады, ол
`build/checksums.sha256`. Құрылғаннан кейін (немесе CI артефактілерін жүктеп алғаннан кейін) іске қосыңыз
`./docs/portal/scripts/preview_verify.sh --build-dir docs/portal/build` дейін
манифестті растаңыз. `--descriptor <path>`/`--archive <path>` өту кезінде
CI метадеректер жинағын тексеру, осылайша сценарий жазылғанды тексере алады
дайджесттер және файл атаулары.
- Жергілікті алдын ала қарауды қауіпсіз ортақ пайдалану үшін `npm run serve` іске қосыңыз; пәрмен енді оралады
  `scripts/serve-verified-preview.mjs`, ол `preview_verify.sh` орындайды
  `docusaurus serve` іске қосу алдында және манифестті тексеру сәтсіз аяқталса немесе тоқтатылады
  жоқ, алдын ала қарауды бақылау қосындысы CI сыртында да сақтайды. The
  `npm run serve:verified` бүркеншік аты анық қоңыраулар үшін қолжетімді болып қалады.

### URL мекенжайларын және шығарылым жазбаларын алдын ала қарау

- Жалпыға қолжетімді бета нұсқасын алдын ала қарау: `https://docs.iroha.tech/`
- GitHub сонымен қатар әрбір орналастыру үшін **github-беттер** ортасының астында құрастыруды көрсетеді.
- Портал мазмұнына тиетін тарту сұраулары құрамында салынған сайтты, бақылау сомасы манифестін, қысылған мұрағатты және дескрипторды қамтитын Әрекеттер артефактілерін (`docs-portal-preview`, `docs-portal-preview-metadata`); шолушылар жергілікті түрде `index.html` жүктеп алып, аша алады және алдын ала қарауды бөліспес бұрын бақылау сомасын тексере алады. Жұмыс процесі әрбір PR-да жиынтық түсініктемені (манифест/мұрағат хэштері плюс SoraFS күйі) тастайды, осылайша шолушылар тексеруден өткені туралы жылдам сигнал алады.
- Алдын ала қарау бумасын жүктеп алғаннан кейін `./docs/portal/scripts/preview_verify.sh --build-dir <extracted build> --descriptor <descriptor> --archive <archive>` сілтемесін сыртқы бөлісу алдында артефактілердің CI шығарған сәйкестігін растау үшін пайдаланыңыз.
- Шығарылым жазбаларын немесе күй жаңартуларын дайындаған кезде, сыртқы шолушылар репозиторийді клондамай-ақ соңғы портал суретін шолу үшін алдын ала қарау URL мекенжайына сілтеме жасаңыз.
- `docs/portal/docs/devportal/preview-invite-flow.md` арқылы алдын ала қарау толқындарын үйлестіріңіз
  және оны `docs/portal/docs/devportal/reviewer-onboarding.md` арқылы жұптаңыз
  шақыру, телеметрияны экспорттау және босату қадамы бірдей дәлелдер жолын қайта пайдаланады.

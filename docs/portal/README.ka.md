# SORA Nexus დეველოპერის პორტალი

ეს დირექტორია მასპინძლობს Docusaurus სამუშაო სივრცეს ინტერაქტიული დეველოპერისთვის
პორტალი. პორტალი აერთიანებს Norito სახელმძღვანელოებს, SDK სწრაფ სტარტებსა და OpenAPI-ს
მითითება გენერირებული `cargo xtask openapi`-ის მიერ, მათი შეფუთვა SORA Nexus-ში
ბრენდინგი, რომელიც გამოიყენება დოკუმენტების პროგრამაში.

## წინაპირობები

- Node.js 18.18 ან უფრო ახალი (Docusaurus v3 საბაზისო).
- ნართი 1.x ან npm ≥ 9 პაკეტის მართვისთვის.
- Rust toolchain (გამოიყენება OpenAPI სინქრონიზაციის სკრიპტით).

## ჩამტვირთავი

```bash
cd docs/portal
npm install    # or yarn install
```

საცავი ფესვი; ის ასხივებს დეტერმინისტულ JSON ფაილს `static/openapi/`-ში
და ახლა ელოდება, რომ Torii როუტერი გამოავლენს ცოცხალ სპეციფიკას (გამოიყენეთ
`cargo xtask openapi --allow-stub` მხოლოდ გადაუდებელი ჩანაცვლებისთვის).

## დოკუმენტების ვერსიები და OpenAPI სნეპშოტები

- **დოკუმენტების ვერსიის ამოჭრა:** გაუშვით `npm run docs:version -- 2025-q3` (ან ნებისმიერი შეთანხმებული ტეგი).
  ჩააბარეთ გენერირებული `versioned_docs/version-<label>`, `versioned_sidebars`,
  და `versions.json`. ნავიბარის ვერსიის ჩამოსაშლელი მენიუ ავტომატურად გამოჩნდება
  ახალი სნეფშოტი.
- **OpenAPI არტეფაქტების სინქრონიზაცია:** ვერსიის ამოჭრის შემდეგ განაახლეთ კანონიკური
  სპეციფიკაცია და მანიფესტი `cargo xtask openapi --sign <path-to-ed25519-key>`-ის საშუალებით, შემდეგ
  გადაიღეთ შესაბამისი სნეპშოტი
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`. The
  სკრიპტი წერს `static/openapi/versions/2025-q3/torii.json`, ასახავს სპეციფიკას
  `versions/current/torii.json`-ში, განახლებები `versions.json`, განახლდება
  `/openapi/torii.json` და კლონირდება ხელმოწერილი `manifest.json` თითოეულ ვერსიაში
  დირექტორია, ამიტომ ისტორიული სპეციფიკაციები ატარებს იმავე წარმოშობის მეტამონაცემებს. მიაწოდე ნებისმიერი
  `--mirror=<label>` დროშების რაოდენობა ახლად გენერირებული სპეციფიკაციის კოპირებისთვის
  სხვა ისტორიული კადრები.
- **CI მოლოდინი:** ახორციელებს შეხების დოკუმენტებს უნდა შეიცავდეს ვერსიის მუწუკს
  (როდესაც შესაძლებელია) პლუს განახლებული OpenAPI კადრები, ასე რომ, Swagger, RapiDoc,
  და Redoc პანელებს შეუძლიათ გადაერთონ ისტორიულ სპეციფიკაციებს შორის შეცდომების მიღების გარეშე.
- **მანიფესტი აღსრულება:** `sync-openapi` სკრიპტი ახლა ვერ ხერხდება ხელმოწერის დროს
  `manifest.json` აკლია, არასწორია ან არ ემთხვევა ახლად გენერირებულს
  სპეციფიკაციები, ასე რომ, ხელმოუწერელი სნეპშოტების ნაგულისხმევად გამოქვეყნება შეუძლებელია. ხელახლა გაშვება
  `cargo xtask openapi --sign <key>` კანონიკური მანიფესტის გასაახლებლად და
  ხელახლა გაუშვით სინქრონიზაცია, რათა ვერსიულმა სნეპშოტებმა აირჩიონ ხელმოწერილი მეტამონაცემები.
  გაიარეთ `--allow-unsigned` მხოლოდ ადგილობრივი გადახედვისთვის (CI კვლავ მუშაობს
  `ci/check_openapi_spec.sh`, რომელიც აღადგენს სპეციფიკას და ამოწმებს
  მანიფესტი, სანამ დაშვებული იქნება კომიტების შერწყმა).

### სცადეთ პროქსი კონფიგურაცია

„სცადე“ სავარჯიშოში მარშრუტები ითხოვს `scripts/tryit-proxy.mjs`-ის მეშვეობით. კონფიგურაცია
პროქსი გარემოს ცვლადებით მის გაშვებამდე:

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN` (ნაგულისხმევი `127.0.0.1:8787`) აკონტროლებს დაკავშირების მისამართს.
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` მეხსიერების კონფიგურაცია
  სიჩქარის შემზღუდველი (ნაგულისხმევი 60 მოთხოვნამდე 60 წამში).
- დააყენეთ `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1` აბონენტის `Authorization` გადამისამართად
  სათაური, ნაცვლად იმისა, რომ დაეყრდნოთ ნაგულისხმევი მატარებლის ჟეტონს.
- პროქსი უარს ამბობს დაწყებაზე, თუ `static/openapi/torii.json` გადადის
  ხელმოწერილი მანიფესტი `static/openapi/manifest.json`-ში. გაიქეცი
  `npm run sync-openapi -- --latest` სპეციფიკაციის განახლებისთვის; ექსპორტი
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` მხოლოდ გადაუდებელი გადაჭარბებისთვის (გაფრთხილება არის
  შესულია და პროქსი მაინც იწყება).
- გაუშვით `npm run probe:tryit-proxy`, რათა შეამოწმოთ დადგმის პროქსი და დააკავშიროთ
  ბრძანება მონიტორინგის სამუშაოებზე; `npm run manage:tryit-proxy -- update` ამარტივებს
  მბრუნავი Torii ბოლო წერტილების დაბრუნების დროს `.env` სარეზერვო ასლის შენახვა.
- `TRYIT_PROXY_PROBE_METRICS_FILE` და `TRYIT_PROXY_PROBE_LABELS` ნება დართეს ზონდს
  გამოუშვით `probe_success`/`probe_duration_seconds` Prometheus ტექსტური ფაილის ფორმატში.
  შეგიძლიათ ჯანმრთელობის შემოწმება დააკავშიროთ node_exporter-ს ან ნებისმიერ პარტიულ კოლექტორს.
- Prometheus ნაკაწრები ამჟღავნებს `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  ჰისტოგრამის სერია; დოკუმენტების პორტალის Grafana დაფა მოიხმარს მათ p95/p99 შეყოვნებისთვის
  SLO თვალთვალი (`dashboards/grafana/docs_portal.json`).

### OAuth მოწყობილობის კოდის შესვლა

როდესაც პორტალი გამოჩნდება სანდო ქსელების გარეთ, დააკონფიგურირეთ OAuth მოწყობილობა
ავტორიზაცია, რათა რეცენზენტები არასოდეს შეეხონ გრძელვადიან Torii ჟეტონებს. ექსპორტი
ცვლადები ქვემოთ `npm run start` ან `npm run build` გაშვებამდე:

| ცვლადი | შენიშვნები |
|----------|-------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` | OAuth საბოლოო წერტილები, რომლებიც ახორციელებენ მოწყობილობის ავტორიზაციის გრანტს. |
| `DOCS_OAUTH_CLIENT_ID` | კლიენტის ID რეგისტრირებულია დოკუმენტების გადახედვისთვის. |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` | არჩევითი ფარგლები/აუდიტორიის სტრიქონები გაცემული ჟეტონის შესაზღუდად. |
| `DOCS_OAUTH_POLL_INTERVAL_MS` | პორტალის ვიჯეტის მიერ დაწესებული გამოკითხვის მინიმალური ინტერვალი (ნაგულისხმევი 5000 მმ-მდე; ქვედა მნიშვნელობები უარყოფილია). |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` | სარეზერვო ვადის გასვლის ფანჯრები გამოიყენება, როდესაც სერვერი გამოტოვებს `expires_in`-ს. |
| `DOCS_OAUTH_ALLOW_INSECURE` | დააყენეთ `1`-ზე მხოლოდ ლოკალური განვითარებისთვის, მშენებლობის დროის მცველის გვერდის ავლით. წარმოების კონსტრუქციები ვერ ხერხდება, თუ საჭირო ცვლადები აკლია ან TTL-ები ცდება დაკისრებულ დიაპაზონებს. |

მაგალითი:

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

მას შემდეგ, რაც build აიღებს ამ მნიშვნელობებს, Try it console ახდენს შესვლას
პანელი, რომელიც აჩვენებს მოწყობილობის კოდს, გამოკითხავს ნიშნის ბოლო წერტილს, ავსებს Bearer-ს
ველი ავტომატურად და ასუფთავებს ჟეტონს, როდესაც ის ამოიწურება. პორტალი უარს ამბობს

პატივს სცემს სახელმძღვანელოს `X-TryIt-Auth` უგულებელყოფს, ასე რომ თქვენ შეგიძლიათ წაშალოთ OAuth ცვლადები
როცა გირჩევნიათ ad-hoc ტოკენების ჩასმა.

### უსაფრთხოების სათაურები

- DNS აღმწერი ახლა ჩაშენებულია `gateway_binding` განყოფილებაში, რომელიც მიუთითებს ზემოთ მოყვანილ არტეფაქტებზე (ბილიკები, შინაარსის CID, მტკიცებულების სტატუსი და პირდაპირი სათაურის შაბლონი), ასე რომ, DG-3 ცვლილების დამტკიცებები მოიცავს ზუსტ `Sora-Name/Sora-Proof/CSP/HSTS` პაკეტს, რომელიც მოსალოდნელია კარიბჭეზე.

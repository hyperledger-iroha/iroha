# SORA Nexus Մշակողների պորտալ

Այս գրացուցակը հյուրընկալում է Docusaurus աշխատանքային տարածքը ինտերակտիվ մշակողի համար
պորտալ. Պորտալը միավորում է Norito ուղեցույցները, SDK արագ մեկնարկները և OpenAPI
տեղեկանք, որը ստեղծվել է `cargo xtask openapi`-ի կողմից՝ դրանք փաթաթելով SORA Nexus-ով
բրենդինգ, որն օգտագործվում է փաստաթղթերի ծրագրում:

## Նախադրյալներ

- Node.js 18.18 կամ ավելի նոր (Docusaurus v3 բազային):
- Մանվածք 1.x կամ npm ≥ 9 փաթեթի կառավարման համար:
- Rust toolchain (օգտագործվում է OpenAPI համաժամացման սցենարով):

## Bootstrap

```bash
cd docs/portal
npm install    # or yarn install
```

## Փաստաթղթերի տարբերակում և OpenAPI նկարներ

- **Փաստաթղթերի տարբերակի կտրում.** գործարկեք `npm run docs:version -- 2025-q3` (կամ ցանկացած համաձայնեցված պիտակ):
  Կատարեք ստեղծված `versioned_docs/version-<label>`, `versioned_sidebars`,
  և `versions.json`. Navbar տարբերակի բացվող ցանկը ավտոմատ կերպով հայտնվում է
  նոր լուսանկար.
- ** OpenAPI արտեֆակտների համաժամացում. ** տարբերակը կտրելուց հետո թարմացրեք կանոնականը
  բնութագրերը և դրսևորվում են `cargo xtask openapi --sign <path-to-ed25519-key>`-ի միջոցով, այնուհետև
  նկարահանել համապատասխան նկարը
  `npm run sync-openapi -- --version=2025-q3 --mirror=current --latest`. Այն
  սցենարը գրում է `static/openapi/versions/2025-q3/torii.json`, արտացոլում է սպեկտրը
  մեջ `versions/current/torii.json`, թարմացումներ `versions.json`, թարմացվում են
  `/openapi/torii.json` և կլոնավորում է ստորագրված `manifest.json`-ը յուրաքանչյուր տարբերակում
  գրացուցակ, այնպես որ պատմական բնութագրերը կրում են նույն ծագման մետատվյալները: Մատակարարել ցանկացած
  `--mirror=<label>` դրոշների քանակը՝ նոր ստեղծված սպեկտրը պատճենելու համար
  այլ պատմական պատկերներ։
- **CI ակնկալիքներ.** պարտավորվում է հուզիչ փաստաթղթերը պետք է ներառեն տարբերակի բախումը
  (եթե կիրառելի է) գումարած թարմացված OpenAPI նկարներ, որպեսզի Swagger, RapiDoc,
  և Redoc վահանակները կարող են անցնել պատմական բնութագրերի միջև՝ առանց բեռնման սխալների:
- **Դրսևորման կիրառում.** `sync-openapi` սկրիպտն այժմ ձախողվում է, երբ ստորագրված է
  `manifest.json`-ը բացակայում է, սխալ ձևավորված է կամ չի համապատասխանում նոր ստեղծվածին
  սպեցիֆիկացիա, ուստի չստորագրված լուսանկարները չեն կարող լռելյայն հրապարակվել: Կրկին առաջադրվել
  `cargo xtask openapi --sign <key>` կանոնական մանիֆեստը թարմացնելու և
  նորից գործարկեք համաժամեցումը, որպեսզի տարբերակված ակնթարթները վերցնեն ստորագրված մետատվյալները:
  Անցեք `--allow-unsigned` միայն տեղական նախադիտումների համար (CI-ն դեռ աշխատում է
  `ci/check_openapi_spec.sh`, որը վերականգնում է բնութագրերը և ստուգում է
  դրսևորել նախքան պարտավորությունների միաձուլումը թույլ տալը):

### Փորձեք այն վստահված անձի կազմաձևումը

«Փորձեք» ավազարկղը հարցումները ուղղում է `scripts/tryit-proxy.mjs`-ի միջոցով: Կարգավորել
շրջակա միջավայրի փոփոխականներով վստահված անձը նախքան այն գործարկելը.

```bash
export TRYIT_PROXY_TARGET="https://torii.staging.sora"
export TRYIT_PROXY_ALLOWED_ORIGINS="http://localhost:3000"
export TRYIT_PROXY_BEARER="sora-dev-token"          # optional
export TRYIT_PROXY_PUBLIC_URL="http://localhost:8787"
npm run tryit-proxy
```

- `TRYIT_PROXY_LISTEN` (կանխադրված `127.0.0.1:8787`) վերահսկում է կապի հասցեն:
- `TRYIT_PROXY_RATE_LIMIT` / `TRYIT_PROXY_RATE_WINDOW_MS` կարգավորել հիշողությունը
  տոկոսադրույքի սահմանափակիչ (կանխադրված է մինչև 60 հարցում 60 վայրկյանում):
- Սահմանեք `TRYIT_PROXY_ALLOW_CLIENT_AUTH=1`՝ զանգահարողի `Authorization`-ը փոխանցելու համար
  վերնագիր՝ լռելյայն կրողի նշանի վրա հենվելու փոխարեն:
- Վստահված անձը հրաժարվում է սկսել, եթե `static/openapi/torii.json`-ը շեղվի
  ստորագրված մանիֆեստը `static/openapi/manifest.json`-ում: Վազիր
  `npm run sync-openapi -- --latest`՝ բնութագրերը թարմացնելու համար; արտահանում
  `TRYIT_PROXY_ALLOW_STALE_SPEC=1` միայն արտակարգ իրավիճակների վերացման համար (նախազգուշացում է
  մուտք է գործել, և վստահված անձը, այնուամենայնիվ, սկսվում է):
- Գործարկեք `npm run probe:tryit-proxy`-ը՝ բեմադրիչ պրոքսին ծխելու համար և միացրեք
  մոնիտորինգի աշխատանքների ղեկավարում; `npm run manage:tryit-proxy -- update`-ը պարզեցնում է
  պտտելով Torii վերջնակետերը՝ միաժամանակ պահելով `.env` կրկնօրինակը հետդարձի համար:
- `TRYIT_PROXY_PROBE_METRICS_FILE` և `TRYIT_PROXY_PROBE_LABELS` թույլ են տալիս զոնդը
  թողարկել `probe_success`/`probe_duration_seconds` Prometheus տեքստային ֆայլի ձևաչափով
  Դուք կարող եք կցել առողջության ստուգումը node_exporter-ի կամ որևէ խմբաքանակի կոլեկցիոների մեջ:
- Prometheus քերծվածքները մերկացնում են `tryit_proxy_request_duration_ms_bucket`/`_count`/`_sum`
  հիստոգրամների շարք; փաստաթղթերի պորտալի Grafana վահանակը սպառում է դրանք p95/p99 ուշացման համար
  SLO հետևում (`dashboards/grafana/docs_portal.json`):

### OAuth սարքի կոդ մուտք

Երբ պորտալը բացահայտվում է վստահելի ցանցերից դուրս, կարգավորեք OAuth սարքը
թույլտվություն, որպեսզի վերանայողները երբեք չդիպչեն երկարակյաց Torii նշաններին: Արտահանել
`npm run start` կամ `npm run build` գործարկելուց առաջ ստորև նշված փոփոխականները.

| Փոփոխական | Ծանոթագրություններ |
|----------|-------|
| `DOCS_OAUTH_DEVICE_CODE_URL` / `DOCS_OAUTH_TOKEN_URL` | OAuth վերջնակետեր, որոնք իրականացնում են Սարքի թույլտվության դրամաշնորհը: |
| `DOCS_OAUTH_CLIENT_ID` | Հաճախորդի ID-ն գրանցված է փաստաթղթերի նախադիտման համար: |
| `DOCS_OAUTH_SCOPE` / `DOCS_OAUTH_AUDIENCE` | Ընտրովի շրջանակ/հանդիսատեսի տողեր՝ թողարկված նշանը սահմանափակելու համար: |
| `DOCS_OAUTH_POLL_INTERVAL_MS` | Պորտալի վիջեթի կողմից պարտադրված քվեարկության նվազագույն միջակայքը (կանխադրված է մինչև 5000 մս, ավելի ցածր արժեքները մերժվում են): |
| `DOCS_OAUTH_DEVICE_CODE_TTL_SECONDS` / `DOCS_OAUTH_TOKEN_TTL_SECONDS` | Հետադարձ ժամկետի ավարտի պատուհաններ, որոնք օգտագործվում են, երբ սերվերը բաց է թողնում `expires_in`-ը: |
| `DOCS_OAUTH_ALLOW_INSECURE` | Սահմանեք `1` միայն տեղական զարգացման համար՝ կառուցման ժամանակի պաշտպանությունը շրջանցելու համար: Արտադրության կառուցումները ձախողվում են, եթե պահանջվող փոփոխականները բացակայում են, կամ TTL-ները դուրս են գալիս պարտադրված տիրույթներից: |

Օրինակ՝

```bash
export DOCS_OAUTH_DEVICE_CODE_URL="https://auth.dev.sora.example/oauth/device/code"
export DOCS_OAUTH_TOKEN_URL="https://auth.dev.sora.example/oauth/token"
export DOCS_OAUTH_CLIENT_ID="docs-preview"
export DOCS_OAUTH_SCOPE="torii openid offline_access"
```

Այն բանից հետո, երբ Build-ն ընդունում է այս արժեքները, «Փորձեք այն» վահանակը մուտք է գործում
վահանակ, որը ցույց է տալիս սարքի կոդը, հարցում է նշում նշանի վերջնակետը, համալրում է կրողը
դաշտը ավտոմատ կերպով և մաքրում է նշանը, երբ դրա ժամկետը լրանա: Պորտալը հրաժարվում է
սկսել, երբ OAuth-ի կազմաձևը թերի է կամ նշանի/սարքի TTL-ներն ընկնում են
անվտանգության բյուջեից դուրս; արտահանել միայն `DOCS_OAUTH_ALLOW_INSECURE=1`
միանգամյա օգտագործման տեղական նախադիտումներ, որտեղ անանուն մուտքն ընդունելի է: Վստահված անձը դեռ
հարգում է `X-TryIt-Auth` ձեռնարկը, այնպես որ կարող եք հեռացնել OAuth փոփոխականները
երբ նախընտրում եք տեղադրել ad-hoc նշաններ:

Տես [`docs/devportal/security-hardening.md`](docs/devportal/security-hardening.md)
սպառնալիքի ամբողջական մոդելը և գրիչ-փորձարկման դարպասները:

### Անվտանգության վերնագրեր

`docusaurus.config.js`-ն այժմ թողարկում է դետերմինիստական անվտանգության վերնագրեր՝ բովանդակության անվտանգություն
քաղաքականություն, վստահելի տեսակներ, թույլտվություններ-քաղաքականություն և ուղղորդող-քաղաքականություն՝ այնքան ստատիկ հոսթորդներ
ժառանգել նույն պահակային ռելսերը, ինչ մշակողի սերվերը: Կանխադրվածները թույլ են տալիս միայն սցենարներ
սպասարկվում է պորտալի սկզբնաղբից և սահմանափակում `connect-src` խաչաձեւ ծագման երթևեկությունը դեպի
կազմաձևված վերլուծության վերջնակետը և «Փորձիր այն» վստահված անձը: Արտադրական շինությունները մերժում են
ոչ HTTPS վերլուծություն/փորձեք վերջնակետեր; `http://`-ի դեմ տեղական նախադիտումների համար
թիրախները, սահմանել `DOCS_SECURITY_ALLOW_INSECURE=1` բացահայտորեն այնպես, որ պաշտպանական բազրիք
ընդունում է վարկանիշի իջեցումը:

### Տեղայնացման աշխատանքային հոսք

Պորտալը ուղարկվում է անգլերենով որպես սկզբնաղբյուր լեզու և ճապոներեն, եբրայերեն,
իսպաներեն, պորտուգալերեն, ֆրանսերեն, ռուսերեն, արաբերեն, ուրդու, բիրմայերեն, վրացերեն,
հայերեն, ադրբեջաներեն, ղազախերեն, բաշկիրերեն, ամհարերեն, ձոնգխա, ուզբեկերեն, մոնղոլերեն,
Ավանդական չինական և պարզեցված չինական վայրեր: Ամեն անգամ, երբ հայտնվում են նոր փաստաթղթեր,
վազել:

```bash
npm run sync-i18n
```

Սցենարը արտացոլում է պահոցի ամբողջ `scripts/sync_docs_i18n.py` պահվածքը՝
ստեղծելով անավարտ թարգմանություններ `i18n/<lang>/docusaurus-plugin-content-docs/current/...`-ի ներքո:
Այնուհետև խմբագիրները կարող են փոխարինել տեղապահի տեքստը և թարմացնել առջևի նյութը
մետատվյալներ (`status`, `translation_last_reviewed` և այլն):

Լռելյայնորեն, արտադրական կառուցումները ներառում են միայն թվարկված տեղանքները
`docs/i18n/published_locales.json` (ներկայումս միայն անգլերեն), այնպես որ անավարտ տեղապահներ
չեն առաքվում: Սահմանեք `DOCS_I18N_INCLUDE_STUBS=1`, երբ նախադիտման կարիք ունեք

## CI և տեղակայում

- `ci/check_docs_portal.sh`-ը սկզբում գործարկում է `node scripts/check-sdk-recipes.mjs`-ը՝ ստուգելու, որ SDK մատյանների բաղադրատոմսերը համապատասխանում են իրենց կանոնական սկզբնաղբյուր ֆայլերին, այնուհետև կատարում է `npm ci|install`, `npm run build` և հաստատում հիմնական բաժինները (8NT20X, I001): Norito, հրապարակման ստուգաթերթ) գոյություն ունի ստեղծված HTML-ում:
- `ci/check_openapi_spec.sh`-ը վերականգնում է Torii բնութագրիչը `cargo xtask openapi`-ի միջոցով,
  համեմատում է այն `static/openapi/torii.json`-ի և `versions/current/torii.json`-ի հետ,
  և ստուգում է checksum-ի մանիֆեստը, որպեսզի PR-ները ձախողվեն, երբ հետևված սպեցիֆիկացիաները կամ հաշիշը
  հնացած է.
- `.github/workflows/check-docs.yml`-ը գործարկում է պորտալը՝ հիմնված յուրաքանչյուր PR-ի վրա՝ բովանդակության հետընթացը բռնելու համար:
- `.github/workflows/docs-portal-preview.yml`-ը կառուցում է կայքը ձգման հարցումների համար, գրում է `build/checksums.sha256`, վավերացնում է այն `sha256sum -c`-ի միջոցով, փաթեթավորում է ելքը որպես `artifacts/preview-site.tar.gz`, ստեղծում է նկարագրիչ և՛ I18NI00000158, և՛ `sha256sum -c`-ի միջոցով, և՛ `artifacts/preview-site.tar.gz`, և՛ ստատիստիկ, և՛ «staloads» կայքի միջոցով: արտեֆակտներ, որպեսզի վերանայողները կարողանան ստուգել ստույգ պատկերը՝ առանց կառուցումը նորից գործարկելու: Աշխատանքային հոսքը նաև տեղադրում է pull- հարցում մեկնաբանություն, որն ամփոփում է մանիֆեստի/արխիվի ամփոփումները (և SoraFS փաթեթը, երբ առկա է), որպեսզի վերանայողները տեսնեն ստուգման արդյունքը՝ առանց Գործողությունների մատյանները բացելու:
- `.github/workflows/docs-portal-deploy.yml`-ը հրապարակում է ստատիկ կառուցումը GitHub էջերի վրա՝ `main`/`master`-ի վրա սեղմելով՝ նախադիտում տրամադրելով `github-pages` միջավայրի URL-ում: Աշխատանքային հոսքը կատարում է վայրէջքի էջի ծխի ստուգում նախքան տեղակայումը:
- `scripts/sorafs-package-preview.sh`-ը նախադիտման կայքը փոխակերպում է որոշիչ SoraFS փաթեթի (CAR, պլան, մանիֆեստ) և, երբ հավատարմագրերը տրամադրվում են JSON կոնֆիգուրացիայի միջոցով (տես `docs/examples/sorafs_preview_publish.json`), կարող է մանիֆեստը ներկայացնել I18NT3NT000001 վերջնակետին:
- `scripts/preview_verify.sh`-ն իրականացնում է ստուգիչ գումարը + նկարագրիչի վավերացման հոսքը, որը վերանայողները նախկինում կատարել էին ձեռքով: Ուղղեք այն արդյունահանված `build/` գրացուցակի (և կամընտիր նկարագրիչի/արխիվի ուղիների) վրա՝ հաստատելու, որ նախադիտման արտեֆակտներն անխափան են:
- `scripts/sorafs-pin-release.sh` plus `.github/workflows/docs-portal-sorafs-pin.yml` ավտոմատացնում է արտադրական SoraFS խողովակաշարը՝ կառուցում/փորձարկում, CAR + մանիֆեստի ստեղծում, Sigstore ստորագրում, ստուգում, կամընտիր անունի պարտադիր կապ և արտեֆակտերի վերբեռնում կառավարման համար: Գործարկեք աշխատանքային հոսքը `workflow_dispatch`-ի միջոցով՝ թողարկումը խթանելիս:
- `cargo xtask soradns-verify-gar --gar <path> --name <fqdn> [...]`-ը վավերացնում է Gateway-ի թույլտվության գրառումները նախքան դրանք ստորագրվելը կամ առաքվելը ops-ին: Օգնականն ապահովում է, որ կանոնական/գեղեցիկ հաղորդավարները, մանիֆեստի մետատվյալները և հեռաչափական պիտակները համապատասխանում են դետերմինիստական ​​քաղաքականությանը և կարող են թողարկել JSON ամփոփագիր DG-3 ապացույցների համար `--json-out`-ի միջոցով:
- Երբ ներկայացումը տեղի է ունենում, փին օգնականը նաև կանչում է `scripts/generate-dns-cutover-plan.mjs`՝ `artifacts/sorafs/portal.dns-cutover.json` արտադրելու համար: Սահմանեք `DNS_CHANGE_TICKET`, `DNS_CUTOVER_WINDOW`, `DNS_HOSTNAME`, `DNS_ZONE` և `DNS_OPS_CONTACT` (կամ փոխանցեք `DNS_OPS_CONTACT` համապատասխան I18NI000000181X մեքենաները), որպեսզի կրճատեք արտադրության ցուցիչները: Երբ պահանջվում է քեշի անվավերացում և հետադարձ կեռիկներ, ավելացրեք `DNS_CACHE_PURGE_ENDPOINT`, `DNS_CACHE_PURGE_AUTH_ENV` և `DNS_PREVIOUS_PLAN` (կամ `--cache-purge-*` / `--cache-purge-*` / `--previous-dns-plan` դրոշակները կամ նախօրոք նկարագրությունը կամ գրառման A դրոշները) հետադարձումներ. Նկարագրիչն այժմ նաև փաստագրում է կեռ `Sora-Route-Binding`-ը (հյուրընկալող, CID, վերնագիր/կապող ուղիներ, ստուգման հրամաններ), այնպես որ GAR-ի առաջխաղացումը և հետադարձ պլանի վերանայումները հղում են կատարում եզրին մատուցվող ճշգրիտ վերնագրերին:
- Նույն աշխատանքային հոսքը կարող է արտանետել SNS zonefile skeleton/resolver snippet `scripts/sns_zonefile_skeleton.py`-ի միջոցով: Տրամադրեք IPv4/IPv6/CNAME/SPKI/TXT մետատվյալներ (կամ որպես env vars, ինչպիսիք են `DNS_ZONEFILE_IPV4` կամ CLI դրոշներ, ինչպիսիք են `--dns-zonefile-ipv4`), փոխանցեք GAR digest-ը `DNS_GAR_DIGEST`-ով, և օգնականը կգրի `DNS_GAR_DIGEST`-ով, և օգնականը կգրի `DNS_ZONEFILE_IPV4`-ի նման CLI դրոշակները։ հատված) ավտոմատ կերպով, այնպես որ SN-7 ապացույցը տեղակայվում է կտրող նկարագրիչի կողքին:
- DNS նկարագրիչն այժմ զետեղում է `gateway_binding` բաժինը, որը հղում է անում վերը նշված արտեֆակտներին (ուղիներ, բովանդակության CID, ապացույցի կարգավիճակ և բառացի վերնագրի ձևանմուշ), այնպես որ DG-3-ի փոփոխության հաստատումները ներառում են դարպասի վրա սպասվող ճշգրիտ `Sora-Name/Sora-Proof/CSP/HSTS` փաթեթը:

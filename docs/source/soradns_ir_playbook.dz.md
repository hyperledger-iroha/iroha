# SoraDNS Resolver & Gateway Incident Response Playbook

This runbook covers incidents where resolver bundles drift from registry
policy, Signed Tree Head proofs go stale, or gateways serve content that
no longer matches the GAR manifest. It ties together the transparency tailer,
Signed Tree Head report CLI, and governance escalation loops referenced in
the DG-5 roadmap item.

## Detection

1. **Prometheus alerts** – `dashboards/alerts/soradns_transparency_rules.yml`
   raises the `SoradnsProofAgeExceeded` warning and `SoradnsCidDriftDetected`
   critical alerts. When either fires, capture the Alertmanager entry and add
   it to the incident ticket.
2. **Transparency report review** – run the weekly report CLI with
   `cargo run -p soradns-resolver --bin soradns_transparency_report -- \
   --log /var/log/soradns/resolver.transparency.log \
   --output artifacts/soradns/transparency_report.md \
   --sth-output artifacts/soradns/signed_tree_heads.json \
   --recent-limit 40`. The "Recent Events" table will highlight zones that
   reorged or drifted; attach both artefacts to the ticket.
3. **Gateway telemetry** – `docs/source/sorafs_gateway_dns_owner_runbook.md`
   describes the GAR telemetry probes. Capture the latest
   `sorafs_gateway-probe` output if the incident originated from the HTTP
   gateway.

## Containment

1. **Freeze the affected namehash** – submit the appropriate registrar action
   (`RegisterOfflineVerdictRevocation` or GAR freeze) per
   `docs/source/sns/governance_playbook.md`.
2. **Republish Signed Tree Heads** – upload the JSON artefact generated by the
   report CLI to the governance evidence bucket and share it with validators
   so they can confirm the new `policy_hash_hex` values.
3. **Gateway rollback** – follow the GAR rollback procedure in
   `docs/source/sorafs_gateway_dns_owner_runbook.md`, ensuring the transparency
   tailer is re-run afterwards to confirm parity.

## Reporting

1. File a transparency note referencing:
   - The Markdown report and JSON Signed Tree Head bundle under
     `artifacts/soradns/<stamp>/`.
   - Grafana snapshots showing alert health and proof-age trends.
   - The governance action (freeze, advisory, or thaw) taken.
2. Update the weekly transparency digest described in
   `docs/source/reports/soradns_transparency.md` with the incident summary and
   remediation owners.
3. Record the drill or incident in `ops/drill-log.md` when applicable.

## Checklist

- [ ] Acknowledge Prometheus alerts and capture screenshots/log links.
- [ ] Run `run_soradns_transparency_tail.sh` to refresh metrics artefacts.
- [ ] Run `soradns_transparency_report` to generate Markdown + JSON bundles.
- [ ] Freeze/rollback as required and document the governance ticket IDs.
- [ ] Update the weekly report and transparency digest with the incident notes.

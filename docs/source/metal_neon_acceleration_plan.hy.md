# Metal & NEON արագացման պլան (Swift & Rust)

Այս փաստաթուղթը ներառում է դետերմինիստական ապարատը միացնելու ընդհանուր պլանը
արագացում (Metal GPU + NEON / Accelerate SIMD + StrongBox ինտեգրում)
Rust աշխատանքային տարածքը և Swift SDK-ն: Այն անդրադառնում է ճանապարհային քարտեզի հետագծված կետերին
**Hardware Acceleration Workstream-ի ներքո (macOS/iOS)** և տրամադրում է միացում
արտեֆակտ Rust IVM թիմի, Swift կամուրջի սեփականատերերի և հեռաչափության գործիքների համար:

> Վերջին թարմացումը՝ 2026-01-12  
> Սեփականատերեր՝ IVM Performance TL, Swift SDK առաջատար

## Գոլեր

1. Կրկին օգտագործեք Rust GPU միջուկները (Poseidon/BN254/CRC64) Apple-ի ապարատում Metal-ի միջոցով
   հաշվարկել շեյդերները՝ դետերմինիստական պարիտետով պրոցեսորի ուղիների նկատմամբ:
2. Բացահայտեք արագացման անջատիչները (`AccelerationConfig`) ծայրից ծայր, որպեսզի Swift հավելվածները
   կարող է ընտրել Metal/NEON/StrongBox-ը՝ պահպանելով ABI/հավասարակշռության երաշխիքները:
3. Գործիք CI + dashboards մակերեսային հավասարության/հենանիշի տվյալները և դրոշակը
   ռեգրեսիաներ պրոցեսորի ընդդեմ GPU/SIMD ուղիների:
4. Կիսեք StrongBox/անվտանգ անկլավային դասեր Android (AND2) և Swift-ի միջև
   (IOS4) ստորագրման հոսքերը դետերմինիստականորեն համահունչ պահելու համար:

**Թարմացում (CRC64 + Stage‑1 թարմացում).** CRC64 GPU-ի օգնականներն այժմ միացված են `norito::core::hardware_crc64`-ին 192KiB լռելյայն անջատմամբ (չեղյալ համարել `NORITO_GPU_CRC64_MIN_BYTES`-ի կամ բացահայտ օգնականի ուղին՝ Prometheus-ի միջոցով, իսկ Prometheus) JSON Stage‑1 կտրվածքները վերանայվել են (`examples/stage1_cutover` → `benchmarks/norito_stage1/cutover.csv`)՝ պահպանելով սկալյար կտրվածքը 4KiB-ում և հավասարեցնելով Stage‑1 GPU-ի լռելյայն 192KiB-ին (Prometheus, իսկ GPU-ի չափը թողարկվում է մեծ վճարում6) ծախսերը։

## Ինտերֆեյս և API պայմանագրեր### Ժանգ (`ivm::AccelerationConfig`)
- Պահպանեք առկա դաշտերը (`enable_simd`, `enable_metal`, `enable_cuda`, `max_gpus`, շեմեր):
- Ավելացրեք հստակ մետաղական տաքացում՝ առաջին օգտագործման հետաձգումից խուսափելու համար (Rust #15875):
- Տրամադրել համարժեք API-ներ, որոնք վերադարձնում են կարգավիճակը/ախտորոշումը վահանակների համար.
  - օրինակ. `ivm::vector::metal_status()` -> {միացված է, հավասարություն, վերջին_սխալ}:
- Արդյունքների չափման չափումներ (Մերկլի ծառի ժամկետներ, CRC թողունակություն) միջոցով
  Հեռաչափական կեռիկներ `ci/xcode-swift-parity`-ի համար:
- Մետաղական հոսթն այժմ բեռնում է կազմված `fastpq.metallib`-ը, ուղարկում է FFT/IFFT/LDE
  և Poseidon միջուկները, և վերադառնում է CPU-ի իրականացմանը, երբ այն
  metallib կամ սարքի հերթն անհասանելի է:

### C FFI (`connect_norito_bridge`)
- Նոր կառուցվածք `connect_norito_acceleration_config` (ավարտված):
- Getter ծածկույթն այժմ ներառում է `connect_norito_get_acceleration_config` (միայն կազմաձևում) և `connect_norito_get_acceleration_state` (կազմաձև + պարիտետ)՝ կարգավորիչը արտացոլելու համար:
- Փաստաթղթի կառուցվածքի դասավորությունը վերնագրի մեկնաբանություններում SPM/CocoaPods սպառողների համար:

### Swift (`AccelerationSettings`)
- Կանխադրվածներ. մետաղական միացված է, CUDA-ն անջատված է, շեմերը զրոյական են (ժառանգել):
- անտեսված են բացասական արժեքները; `apply()`-ը ինքնաբերաբար կանչվել է `IrohaSDK`-ի կողմից:
- `AccelerationSettings.runtimeState()`-ն այժմ հայտնվում է `connect_norito_get_acceleration_state`-ի վրա
  ծանրաբեռնվածություն (կոնֆիգուրացիա + Մետաղ/CUDA հավասարության կարգավիճակ), այնպես որ Swift-ի վահանակները թողարկեն նույն հեռաչափությունը
  որպես Rust (`supported/configured/available/parity`): Օգնականը վերադարձնում է `nil`, երբ
  կամուրջը բացակայում է թեստերը շարժական պահելու համար:
- `AccelerationBackendStatus.lastError`-ը պատճենում է անջատման/սխալի պատճառը
  `connect_norito_get_acceleration_state` և ազատում է բնիկ բուֆերը, երբ տողը գտնվում է
  նյութականացվել է այնպես, որ շարժական հավասարության վահանակները կարող են նշել, թե ինչու են Metal/CUDA-ն անջատված
  յուրաքանչյուր հյուրընկալող.
- `AccelerationSettingsLoader` (`IrohaSwift/Sources/IrohaSwift/AccelerationSettingsLoader.swift`,
  թեստեր `IrohaSwift/Tests/IrohaSwiftTests/AccelerationSettingsLoaderTests.swift`-ի ներքո) հիմա
  լուծում է օպերատորի դրսևորումները նույն առաջնահերթ կարգով, ինչ Norito ցուցադրությունը. պատիվ
  `NORITO_ACCEL_CONFIG_PATH`, որոնել փաթեթավորված `acceleration.{json,toml}` / `client.{json,toml}`,
  մուտքագրեք ընտրված աղբյուրը և վերադարձեք լռելյայն: Հավելվածներն այլևս կարիք չունեն հատուկ բեռնիչների
  արտացոլել Rust `iroha_config` մակերեսը:
- Թարմացրեք ընտրանքային հավելվածները և README-ը՝ անջատիչները և հեռաչափության ինտեգրումը ցուցադրելու համար:

### Հեռաչափություն (վահանակներ + Արտահանողներ)
- Հավասարաչափ հոսք (mobile_parity.json):
  - `acceleration.metal/neon/strongbox` -> {միացված է, հավասարություն, perf_delta_pct}:
  - Ընդունեք `perf_delta_pct` ելակետային պրոցեսորի համեմատությունը GPU-ի հետ:
  - `acceleration.metal.disable_reason` հայելիներ `AccelerationBackendStatus.lastError`
    այնպես որ Swift ավտոմատացումը կարող է նշել անջատված GPU-ները նույն հավատարմությամբ, ինչ Rust-ը
    վահանակներ.
- CI feed (mobile_ci.json):
  - `acceleration_bench.metal_vs_cpu_merkle_ms` -> {cpu, մետաղական}
  - `acceleration_bench.neon_crc64_throughput_mb_s` -> Կրկնակի:
- Արտահանողները պետք է չափումներ ստանան Rust-ի չափանիշներից կամ CI գործարկումներից (օրինակ՝ գործարկումից
  Մետաղ/պրոցեսոր միկրոբենչ՝ որպես `ci/xcode-swift-parity`-ի մաս):### Կազմաձևման կոճակներ և կանխադրված (WP6-C)
- `AccelerationConfig` լռելյայն. `enable_metal = true` macOS-ի կառուցումներում, `enable_cuda = true`, երբ CUDA ֆունկցիան կազմված է, `max_gpus = None` (առանց գլխարկի): Swift `AccelerationSettings` փաթաթումը ժառանգում է նույն լռելյայնությունները `connect_norito_set_acceleration_config`-ի միջոցով:
- Norito Merkle heuristics (GPU vs CPU). backend-ի անտեսումը (`merkle_min_leaves_metal`, `merkle_min_leaves_cuda`) լռելյայն նույն շեմն է, եթե բացահայտորեն սահմանված չէ:
- CPU-ի նախապատվության էվրիստիկա (ներկայիս SHA2 ISA). թե՛ AArch64 (ARMv8 SHA2) և թե x86/x86_64 (SHA-NI) պրոցեսորի ուղին մնում է նախընտրելի մինչև `prefer_cpu_sha2_max_leaves_* = 32_768` հեռանալը; վեր, որ կիրառվում է GPU-ի շեմը: Այս արժեքները կարգավորելի են `AccelerationConfig`-ի միջոցով և պետք է ճշգրտվեն միայն հենանիշային ապացույցներով:

## Փորձարկման ռազմավարություն

1. **Միավորի հավասարության թեստեր (Rust)**. ապահովել, որ մետաղական միջուկները համապատասխանում են պրոցեսորի ելքերին
   դետերմինիստական վեկտորներ; աշխատել `cargo test -p ivm --features metal`-ի ներքո:
   `crates/fastpq_prover/src/metal.rs`-ն այժմ առաքում է միայն macOS-ի հավասարության թեստեր, որոնք
   վարժություն FFT/IFFT/LDE և Poseidon-ը սկալյար հղումի դեմ:
2. **Ծխի արագ ամրացում**. ընդլայնել IOS6 թեստային վազքը՝ պրոցեսորն ընդդեմ մետաղի գործարկելու համար
   կոդավորում (Merkle/CRC64) ինչպես էմուլյատորների, այնպես էլ StrongBox սարքերի վրա; համեմատել
   արդյունքները և գրանցման հավասարության կարգավիճակը:
3. **CI**. թարմացրեք `norito_bridge_ios.yml` (արդեն զանգում է `make swift-ci`)՝ մղելու համար
   արտեֆակտների արագացման չափումներ; համոզվեք, որ վազքը հաստատում է Buildkite-ը
   `ci/xcframework-smoke:<lane>:device_tag` մետատվյալներ՝ նախքան ամրագոտիների փոփոխությունները հրապարակելը,
   և ձախողել գոտին հավասարության/հենանիշի դրեյֆի վրա:
4. **Dashboards**. նոր դաշտերը այժմ ցուցադրվում են CLI ելքում: Ապահովել, որ արտահանողները արտադրում են
   տվյալները՝ նախքան ցուցատախտակների ուղիղ հեռարձակումը:

## WP2-A Metal Shader Plan (Poseidon Pipelines)

WP2-ի առաջին նշաձողը ներառում է Poseidon Metal միջուկների պլանավորման աշխատանքները
որոնք արտացոլում են CUDA-ի իրականացումը: Ծրագիրը ջանքերը բաժանում է միջուկների,
հյուրընկալողի պլանավորում և ընդհանուր մշտական բեմադրություն, որպեսզի հետագայում աշխատանքը կարողանա զուտ կենտրոնանալ
իրականացում և փորձարկում։

### միջուկի շրջանակը

1. `poseidon_permute`՝ փոխում է `state_count` անկախ պետությունները: Յուրաքանչյուր թեմա
   ունի `STATE_CHUNK` (4 վիճակ) և գործարկում է `TOTAL_ROUNDS` բոլոր կրկնությունները՝ օգտագործելով
   threadgroup-ով համօգտագործվող կլոր հաստատուններ, որոնք բեմադրվում են առաքման ժամանակ:
2. `poseidon_hash_columns`. կարդում է նոսր `PoseidonColumnSlice` կատալոգը և
   կատարում է Merkle-ի համար հարմար յուրաքանչյուր սյունակի հեշավորում (համապատասխանում է պրոցեսորին
   `PoseidonColumnBatch` դասավորություն): Այն օգտագործում է նույն շղթայի խմբի մշտական բուֆերը
   որպես փոխակերպվող միջուկ, բայց պտտվում է `(states_per_lane * block_count)`-ի վրա
   ելքեր, որպեսզի միջուկը կարողանա ամորտիզացնել հերթի ներկայացումները:
3. `poseidon_trace_fused`. հաշվարկում է ծնող/տերևի մարսումները հետագծային աղյուսակի համար
   մեկ անցումով. Միաձուլված միջուկը սպառում է `PoseidonFusedArgs`, այնպես որ՝ հյուրընկալողը
   կարող է նկարագրել ոչ հարակից շրջաններ և `leaf_offset`/`parent_offset`, և
   այն կիսում է բոլոր կլոր/MDS աղյուսակները մյուս միջուկների հետ:

### Հրամանի ժամանակացույց և հյուրընկալող պայմանագրեր- Յուրաքանչյուր միջուկի առաքում անցնում է `MetalPipelines::command_queue`-ի միջոցով, որը
  կիրառում է հարմարվողական ժամանակացույցը (նպատակ ~2 ms) և հերթի օդափոխիչի կառավարումը
  ենթարկվում է `FASTPQ_METAL_QUEUE_FANOUT`-ի և
  `FASTPQ_METAL_COLUMN_THRESHOLD`. Տաքացման ուղին `with_metal_state`-ում
  հավաքում է Պոսեյդոնի բոլոր երեք միջուկները առջևում, այնպես որ առաջին ուղարկումը չի կատարվում
  վճարել խողովակաշարի ստեղծման տույժ.
- Թելային խմբի չափերը արտացոլում են գոյություն ունեցող Metal FFT/LDE կանխադրվածները.
  8,192 թեմա յուրաքանչյուր ներկայացման համար, յուրաքանչյուր խմբի համար 256 թեմա կոշտ գլխարկով: Այն
  հյուրընկալողը կարող է նվազեցնել `states_per_lane` բազմապատկիչը ցածր էներգիայի սարքերի համար
  Շրջակա միջավայրը հավաքելը գերակայում է (`FASTPQ_METAL_POSEIDON_STATES_PER_BATCH`
  պետք է ավելացվի WP2-B-ում) առանց ստվերների տրամաբանությունը փոփոխելու:
- Սյունակների բեմադրությունը հետևում է նույն կրկնակի բուֆերային լողավազանին, որն արդեն օգտագործվում է FFT-ի կողմից
  խողովակաշարեր. Պոսեյդոնի միջուկներն ընդունում են չմշակված ցուցիչներ այդ փուլային բուֆերի մեջ
  և երբեք չդիպչել գլոբալ կույտային հատկացումներին, ինչը պահպանում է հիշողություն-դետերմինիզմը
  համահունչ CUDA հյուրընկալողի հետ:

### Համօգտագործվող հաստատուններ

- `PoseidonSnapshot` մանիֆեստը նկարագրված է
  `docs/source/fastpq/poseidon_metal_shared_constants.md`-ն այժմ կանոնական է
  կլոր հաստատունների և MDS մատրիցի աղբյուր: Երկուսն էլ մետաղական (`poseidon2.metal`)
  և CUDA (`fastpq_cuda.cu`) միջուկները պետք է վերականգնվեն ամեն անգամ, երբ մանիֆեստը
  փոփոխությունները։
- WP2-B-ն կավելացնի փոքրիկ հյուրընկալող բեռնիչ, որը կարդում է մանիֆեստը գործարկման ժամանակ և
  արձակում է SHA-256-ը հեռաչափության մեջ (`acceleration.poseidon_constants_sha`), ուստի
  հավասարության վահանակները կարող են պնդել, որ ստվերային հաստատունները համապատասխանում են հրապարակվածին
  ակնթարթ.
- Տաքացման ժամանակ մենք պատճենելու ենք `TOTAL_ROUNDS x STATE_WIDTH` հաստատունները a
  `MTLBuffer` և վերբեռնեք այն մեկ սարքի համար: Յուրաքանչյուր միջուկ այնուհետև պատճենում է տվյալները
  թելային խմբի հիշողության մեջ՝ նախքան դրա հատվածը մշակելը, ապահովելով որոշիչ
  պատվիրել նույնիսկ այն դեպքում, երբ մի քանի հրամանների բուֆերներ են աշխատում թռիչքի ժամանակ:

### Վավերացման Կեռիկներ

- Միավոր թեստերը (`cargo test -p fastpq_prover --features fastpq-gpu`) կմեծանան
  պնդում, որը հեշավորում է ներկառուցված ստվերային հաստատունները և համեմատում դրանք
  մանիֆեստի SHA-ն՝ նախքան GPU սարքերի փաթեթը գործարկելը:
- Գործող միջուկի վիճակագրությունը փոխվում է (`FASTPQ_METAL_TRACE_DISPATCH`,
  `FASTPQ_METAL_QUEUE_FANOUT`, հերթի խորության հեռաչափություն) դառնում են պարտադիր ապացույց
  WP2 ելքի համար. յուրաքանչյուր փորձնական գործարկում պետք է ապացուցի, որ ժամանակացույցը երբեք չի խախտում
  կազմաձևված fan-out և որ միաձուլված հետքի միջուկը պահում է հերթը ստորև
  հարմարվողական պատուհան:
- Swift XCFramework ծխի զրահը և Rust հենանիշային վազորդները կմեկնարկեն
  արտահանում է `acceleration.poseidon.permute_p90_ms{cpu,metal}`, որպեսզի WP2-D-ը կարողանա գծապատկեր
  Մետաղ ընդդեմ պրոցեսորի դելտաներ՝ առանց նոր հեռաչափական հոսքերի հայտնագործման:

## WP2-B Poseidon Manifest Loader & Self-Test Parity- `fastpq_prover::poseidon_manifest()`-ն այժմ տեղադրում և վերլուծում է
  `artifacts/offline_poseidon/constants.ron`, հաշվարկում է իր SHA-256-ը
  (`poseidon_manifest_sha256()`) և վավերացնում է լուսանկարը պրոցեսորի նկատմամբ
  poseidon աղյուսակներ՝ նախքան GPU-ի ցանկացած աշխատանք: `build_metal_context()` գրանցում է
  մարսել տաքացման ժամանակ, որպեսզի հեռաչափություն արտահանողները կարողանան հրապարակել
  `acceleration.poseidon_constants_sha`.
- Մանիֆեստի վերլուծիչը մերժում է չհամընկնող լայնությունը/տեմպերը/կլոր հաշվարկի բազմությունները և
  ապահովում է, որ մանիֆեստային MDS մատրիցը հավասար է սկալյար իրականացմանը՝ կանխելով
  լուռ շեղում, երբ կանոնական աղյուսակները վերականգնվում են:
- Ավելացվեց `crates/fastpq_prover/tests/poseidon_manifest_consistency.rs`, որը
  վերլուծում է `poseidon2.metal`-ում ներկառուցված Poseidon աղյուսակները և
  `fastpq_cuda.cu` և պնդում է, որ երկու միջուկները սերիականացվում են ճիշտ նույնը
  հաստատունները որպես մանիֆեստ: CI-ն այժմ ձախողվում է, եթե ինչ-որ մեկը խմբագրում է shader/CUDA
  ֆայլեր՝ առանց կանոնական մանիֆեստը վերականգնելու:
- Ապագա հավասարաչափ կեռիկներ (WP2-C/D) կարող են կրկին օգտագործել `poseidon_manifest()`՝ բեմադրելու համար
  կլոր հաստատունները GPU-ի բուֆերների մեջ և բացահայտում են ամփոփումը Norito-ի միջոցով
  հեռաչափության սնուցում:

## WP2-C BN254 մետաղական խողովակաշարեր և հավասարաչափ փորձարկումներ- **Ծավալը և բացը.** Հյուրընկալող դիսպետչերները, հավասարաչափ ամրագոտիները և `bn254_status()`-ն ակտիվ են, և `crates/fastpq_prover/metal/kernels/bn254.metal`-ն այժմ իրականացնում է Montgomery primitives-ը, գումարած թեմախմբերի համաժամեցված FFT/LDE հանգույցները: Յուրաքանչյուր դիսպետչեր վարում է մի ամբողջ սյունակ մեկ թելային խմբի ներսում՝ յուրաքանչյուր փուլային խոչընդոտներով, այնպես որ միջուկները զուգահեռաբար իրականացնում են բեմական դրսևորումները: Հեռուստաչափությունն այժմ միացված է լարով, և ժամանակացույցի անտեսումները հարգվում են, որպեսզի մենք կարողանանք փակել լռելյայն միացված թողարկումը նույն ապացույցներով, որոնք օգտագործում ենք Goldilocks միջուկների համար:
- **Միջուկի պահանջները.** ✅ նորից օգտագործեք փուլային twiddle/coset մանիֆեստները, փոխարկեք մուտքերը/ելքերը մեկ անգամ և կատարեք բոլոր արմատական-2 փուլերը յուրաքանչյուր սյունակի թեմախմբում, որպեսզի մենք բազմակողմանի համաժամացման կարիք չունենանք: Մոնտգոմերիի օգնականները մնում են ընդհանուր FFT/LDE-ի միջև, այնպես որ փոխվել է միայն օղակի երկրաչափությունը:
- **Հոսթի լարեր.** ✅ `crates/fastpq_prover/src/metal.rs` փուլավորում է կանոնական վերջույթները, զրոյականացնում է LDE բուֆերը, ընտրում է մեկ թելերի խումբ յուրաքանչյուր սյունակում և ցուցադրում `bn254_status()`-ը դարպասի համար: Հեռաչափության համար հյուրընկալող լրացուցիչ փոփոխություններ չեն պահանջվում:
- **Կառուցեք պահակները.** `fastpq.metallib`-ը առաքում է սալիկապատ միջուկները, այնպես որ CI-ն արագորեն ձախողվում է, եթե ստվերը շարժվում է: Ցանկացած ապագա օպտիմիզացում մնում է հեռաչափության/հնարավորության դարպասների հետևում, քան կոմպիլյացիայի ժամանակի անջատիչների:
- **Հավասարաչափ սարքեր.** ✅ `bn254_parity` թեստերը շարունակում են համեմատել GPU FFT/LDE ելքերը պրոցեսորի սարքերի հետ և այժմ աշխատում են ուղիղ եթերում Metal ապարատով; նկատի ունեցեք կեղծված-մանիֆեստի թեստերը, եթե հայտնվեն միջուկի կոդերի նոր ուղիներ:
- **Հեռաչափություն և չափորոշիչներ.** `fastpq_metal_bench`-ն այժմ թողարկում է.
  - `bn254_dispatch` բլոկ, որն ամփոփում է յուրաքանչյուր ուղարկման համար նախատեսված թելերի խմբի լայնությունները, թելերի տրամաբանական քանակը և խողովակաշարերի սահմանները FFT/LDE մեկ սյունակ խմբաքանակների համար. և
  - `bn254_metrics` բլոկ, որը գրանցում է `acceleration.bn254_{fft,ifft,lde,poseidon}_ms` պրոցեսորի բազային գծի համար, գումարած այն GPU-ի հետնամասը, որն աշխատում է:
  Հենանիշի փաթաթումը պատճենում է երկու քարտեզներն էլ յուրաքանչյուր փաթաթված արտեֆակտի մեջ, որպեսզի WP2-D վահանակները ներծծեն պիտակավորված հետաձգումներ/երկրաչափություն՝ առանց չմշակված գործառնությունների զանգվածի հակադարձ նախագծման: `FASTPQ_METAL_THREADGROUP`-ն այժմ կիրառվում է նաև BN254 FFT/LDE դիսպետչերների համար՝ կոճակը դարձնելով պիտանի օգտագործման համար ավլում.

## Բաց Հարցեր (լուծված է 2027 թվականի մայիսին)1. **Մետաղական ռեսուրսների մաքրում.** `warm_up_metal()`-ը նորից օգտագործում է շարանը տեղական
   `OnceCell` և այժմ ունի իմպոտենցիայի/ռեգեսիայի թեստեր
   (`crates/ivm/src/vector.rs::warm_up_metal_reuses_cached_state` /
   `warm_up_metal_is_noop_on_non_metal_targets`), ուստի հավելվածի կյանքի ցիկլի անցումները
   կարող է ապահով կերպով զանգահարել տաքացման ուղին՝ առանց արտահոսքի կամ կրկնակի սկզբնավորման:
2. **Հիմնական ելակետային գծեր.** Մետաղական ուղիները պետք է մնան պրոցեսորի 20%-ի սահմաններում
   FFT/IFFT/LDE-ի ելակետը և 15%-ի սահմաններում Poseidon CRC/Merkle օգնականների համար;
   ահազանգը պետք է գործարկվի, երբ `acceleration.*_perf_delta_pct > 0.20` (կամ բացակայում է)
   բջջային պարիտետի հոսքում: IFFT ռեգրեսիաները դիտվել են 20k հետքի փաթեթում
   այժմ փակված են WP2-D-ում նշված հերթի շտկումով:
3. **StrongBox backback.** Swift-ը հետևում է Android-ի հետընտրական գրքույկին
   գրանցում է ատեստավորման ձախողումներ աջակցության մատյանում
   (`docs/source/sdk/swift/support_playbook.md`) և ինքնաբերաբար անցում դեպի
   փաստաթղթավորված HKDF-ով ապահովված ծրագրային ուղին աուդիտի գրանցմամբ. հավասարության վեկտորներ
   մնացեք համօգտագործվող OA-ի առկա հարմարանքների միջոցով:
4. **Հեռաչափության պահեստավորում.** Արագացման գրավումներն ու սարքի լողավազանի ապացույցներն են
   արխիվացված `configs/swift/` տակ (օրինակ,
   `configs/swift/xcframework_device_pool_snapshot.json`), և արտահանողներ
   պետք է արտացոլի նույն դասավորությունը (`artifacts/swift/telemetry/acceleration/*.json`
   կամ `.prom`), որպեսզի Buildkite-ի ծանոթագրությունները և պորտալի վահանակները կարողանան կլանել
   կերակրում է առանց ժամանակավոր քերիչ:

## WP2-D թողարկման հենանիշ և պարտադիր նշումներ- ** 20 հազար տող թողարկման գրավում. ** Գրանցել է թարմ մետաղական ընդդեմ պրոցեսորի չափանիշ macOS14-ում
  (arm64, lane-balanced parametres, padded 32,768-row trace, երկու սյունակի խմբաքանակ) և
  ստուգեց JSON փաթեթը `fastpq_metal_bench_20k_release_macos14_arm64.json`-ում:
  Հենանիշն արտահանում է մեկ գործառնական ժամկետներ, գումարած Poseidon microbench-ի վկայությունը

  դելտաներ (ամբողջական սեղանն ապրում է `docs/source/benchmarks.md`-ում):

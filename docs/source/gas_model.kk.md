# IVM газ үлгісі

Бұл құжат Iroha виртуалды машинасының канондық газ кестесін анықтайды
(IVM) және кестенің хэштеу және қолдану әдісін түсіндіреді. Шындықтың қайнар көзі
шығындар үшін `crates/ivm/src/gas.rs`; төмендегі кесте кестесі көрсетілген
сол канондық картаның көрінісі.

## Ауқым

- IVM байт кодының орындалуына қолданылады (Орындалатын::Ivm).
- Жергілікті ISI газды есептеу `crates/iroha_core/src/gas.rs` ішінде бөлек анықталған.
- ISO 20022 операциялық кодтары ABI v1 нұсқасында сақталған және әлі газ жазбаларын тасымалдамайды.

## Детерминизм және кесте хэші

Газ кестесі детерминирленген және операциялық код → шығындар жұптарынан алынған. The
канондық дайджест әрбір жазбамен реттелген операциялық код тізімі бойынша есептеледі
серияланған:

- операция коды байты (u8)
- құны (u64, аз-эндиан)

Хэш келесі арқылы ашылады:

- `ivm::gas::schedule_hash()` (канондық кесте хэші)
- `ivm::limits::schedule_hash()` (хостқа бағытталған бүркеншік ат)

Сымдарды қосу кезінде барлық құрдастар бірдей кестені бөлісетінін тексеру үшін осы дайджестті пайдаланыңыз
конфигурация немесе телеметрия тексерулері.

## Векторлық масштабтау және HTM қайталау әрекеттері

- Логикалық көмегімен векторлық операциялар (`VADD*`, `VAND`, `VXOR`, `VOR`, `VROT32`)
  вектор ұзындығы `SETVL` арқылы орнатылған. Кестедегі негізгі шығындар бойынша масштабталған
  `min(vector_len, VECTOR_BASE_LANES) / VECTOR_BASE_LANES` (негізгі сызық = 2 жолақ).
- HTM қайталаулары құнын `(retries + 1)` көбейтеді; консенсус жолдарының көпшілігі жоқ
  қайталап көріңіз.

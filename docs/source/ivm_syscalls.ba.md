# IVM X Syscall ABI

Был документта IVM syscall һандары, күрһәткес-АБИ саҡырыу конвенциялары, һаҡланған һандар диапазондары һәм Kotodama түбәнәйткән контракт-йүнәлешле syscalls канон таблицаһы билдәләнә. Ул `ivm.md` (архитектура) һәм `kotodama_grammar.md` (тел) тулыландыра.

Версиялау
- танылған syscalls комплекты байт-кодтың `abi_version` яланына бәйле. Беренсе релиз тик `abi_version = 1` ғына ҡабул итә; башҡа ҡиммәттәр ҡабул итеү ваҡытында кире ҡағыла. Билдәһеҙ һандар өсөн әүҙем `abi_version` детерминистик тоҙаҡ менән `E_SCALL_UNKNOWN`.
- Йүгереп йөрөүҙәр `abi_version = 1` һаҡлай һәм syscall йәки күрһәткес ABI өҫтөн киңәйтмәй.
- Syscall газ сығымдары өлөшө булып тора версияланған газ график бәйле байткод баш версияһы. Ҡара: `ivm.md` (Газ сәйәсәте).

Һанлы диапазондар
- `0x00..=0x1F`: ВМ ядро/туҡланыу (отладка/сығыу ярҙамсылары `CoreHost` буйынса мөмкин;
- `0x20..=0x5F`: Iroha ядро ​​ISI күпере (тотороҡло ABI v1).
- `0x60..=0x7F`: протокол протокол функциялары буйынса ҡапҡалы ISIs (әле лә ABI v1 өлөшө ҡасан эшләй).
- `0x80..=0xFF`: хост/крипто ярҙамсылары һәм запас слоттар; тик һандар бар АБИ v1 рөхсәт исемлегендә ҡабул ителә.

Ҡаты ярҙамсылар (АБИ v1)
- ныҡлы дәүләт ярҙамсыһы syscalls (0x50–0x5A: STATE_{GET,SET,DEL}, ENCODE/DECODE_INT, BUILD_PATH_*, JSON/SCHEMA кодексы/декод) V1 ABI өлөшө булып тора һәм `abi_hash` иҫәпләүҙәренә индерелгән.
- CoreHost сымдар STATE_{GET,SET,DEL} WSV-арҡалы ныҡлы аҡыллы аҡыллы хәлгә; dev/һынау хужалары урындағы кимәлдә һаҡланыуы мөмкин, әммә бер үк syscall семантикаһын һаҡларға тейеш.

Күрһәткестәр конвенцияһын атау (аҡыллы-контракт syscalls)
- Аргументтар `r10+` регистрҙарында сеймал `u64` ҡиммәттәре йәки INPUT төбәгенә күрһәткестәр булараҡ урынлаштырылған Norito TLV конверттары (мәҫәлән, `AccountId`, Iroha. `Name`, `Json`, `NftId`).
- скаляр ҡайтарыу ҡиммәттәре булып тора `u64` ҡайтып хужа. Һөҙөмтәлелек һөҙөмтәләрен хужа `r10`-ға яҙа.

Газ үтәү
- CoreHost ISI syscalls өсөн өҫтәмә газ өсөн тыуған ISI графигы ҡулланып; FASTPQ партия күсермәләре бер яҙмаға алынған.
- ZK_VERIFY syscalls конфиденциаль тикшерелгән газ графигын ҡабаттан ҡулланыу (нигеҙ + иҫбатлау күләме).
- СМОРТКОНТСЕКСЕКСЕКСЕКСЕКСЕКСЕКСЕКАҠТА зарядтары базаһы + пер-племет + пер-байт; сорттарға бүлергә пункт пер-пункт хаҡын һәм сорттарға тайпылған офсеттар өҫтәп, штраф пер-пункт.

Иҫкәрмәләр
- Бөтә күрһәткес аргументтары Norito TLV конверттары INPUT төбәгендә һәм беренсе dereference (`E_NORITO_INVALID` хаталар буйынса) раҫлана.
- Бөтә мутациялар Iroha аша ҡулланыла’s стандарт башҡарыусы (`CoreHost` аша), туранан-тура виртуаль түгел.
- Теүәл газ константалары (`G_*`) әүҙем газ графигы менән билдәләнә; ҡара: `ivm.md`.

Хаталар
- `E_SCALL_UNKNOWN`: әүҙем `abi_version` өсөн танылмаған сыскал номеры.
- ВМ тоҙаҡтары булараҡ таралыуын раҫлау хаталары (мәҫәлән, `E_NORITO_INVALID` өсөн дөрөҫ булмаған TLVs өсөн).

Кросс-һылтанмалар
- Архитектура һәм ВМ семантикаһы: `ivm.md` X
- Тел һәм төҙөү картаһы: `docs/source/kotodama_grammar.md`

Быуындар иҫкәрмәһе
- Сысколл константаларының тулы исемлеген сығанаҡтан генерацияларға мөмкин:
  - `make docs-syscalls` → `docs/source/ivm_syscalls_generated.md` тип яҙа.
  - `make check-docs` → генерацияланған таблицаны раҫлай, был заманса (CI-ла файҙалы)
- Үрҙәге подмножество курировать, тотороҡло таблица өсөн контракт-ҡаршы syscalls ҡала.

## Админ/Роль TLV Миҫалдар (Мәҙәнле алып барыусы)

Был бүлектә TLV формалары һәм минималь JSON файҙалы йөкләмәләр ҡабул ителгән макет WSV хост өсөн админ‐стиль syscalls ҡулланылған һынауҙар. Бөтә күрһәткес аргументтары ла күрһәткес‐АБИ (Norito TLV конверттары INPUT урынлаштырылған) үтәй. Етештереүҙең хужалары байыраҡ схемаларҙы ҡулланырға мөмкин; был миҫалдар төрҙәрен һәм төп формаларын асыҡлауҙы маҡсат итеп ҡуя.- ТӨРКӨРӨНСӘЙ / БАШҠА_ПЕР
  - Аргс: `r10=&Json`
  - Миҫал JSON: `{ "peer": "peer-id-or-info" }`
  - Corehost иҫкәрмәһе: `REGISTER_PEER` `RegisterPeerWithPop` JSON объекты менән `peer` + `pop` байт менән көтә (факультатив `activation_at`, `expiry_at`, `hsm`); `UNREGISTER_PEER` ҡабул итә тиңдәш-ид еп йәки `{ "peer": "..." }`.

- КРЕЙТ_ТРИГЕР / РЕМОВИКАҺЫ / СЕТ_ТРИГЕР_ЕНАБЛИД
  - КРЕЙТ_ТРИГГЕР:
    - Аргс: `r10=&Json`
    - Минималь JSON: `{ "name": "t1" }`X (өҫтәмә ҡырҙар макет иғтибарға алынмаған)
  - РЕМОВ_ТРИГГЕР:
    - Аргс: `r10=&Name` (триггер исеме)
  - СЕТ_ТРИГЕР_ЕНАБЛЕД:
    - Args: `r10=&Name`, `r11=enabled:u64` X (0 = инвалид, нуль булмаған = өҫтәмә)
  - CoreHost иҫкәрмәһе: `CREATE_TRIGGER` тулы триггер спец (база Norito `Trigger` стринг йәки
    `{ "id": "<trigger_id>", "action": ... }` менән `action` менән base64 Norito `Action` еп йәки .
    JSON объекты), һәм `SET_TRIGGER_ENABLED` триггер метамағлүмәттәр асҡысы `__enabled` (дөрөҫ түгел
    ғәҙәттәгесә эшләй.

- ролдәр: КРЕЙТ_РОЛ / ДЕЛЕС_РОЛ / ГРАНТ_РОЛ / РЕВОК_РОЛЬ.
  - КРЕЙТ_РОЛ:
    - Аргс: `r10=&Name` (роль исеме), `r11=&Json` (рөхсәттәр йыйылмаһы)
    - JSON йәки асҡыс ҡабул итә `"perms"` йәки `"permissions"`, һәр береһе рөхсәт исемдәре строкаһы.
    - Миҫалдар:
      - `{ "perms": [ "mint_asset:rose#wonder" ] }`
      - `{ "permissions": [ "read_assets:i105...", "transfer_asset:rose#wonder" ] }`
    - Рөхсәт исеменә ярҙам иткән мысҡылда:
      - `register_domain`, `register_account`, Norito.
      - `read_assets:<account_id>`
      — `mint_asset:<asset_definition_id>`
      - `burn_asset:<asset_definition_id>`
      - `transfer_asset:<asset_definition_id>`
  - ДЕЛЕТ_РОЛ:
    - Аргс: `r10=&Name`
    - Әгәр ҙә ниндәйҙер иҫәп яҙмаһы һаман да был ролде лә тапшыра икән.
  - ГРАНТ_ROLE / РЕВЕ_РОЛ:
    - Аргс: `r10=&AccountId` (субъект), `r11=&Name` (роль исеме)
  - CoreHost иҫкәрмәһе: рөхсәт JSON тулы булыуы мөмкин `Permission` объекты (`{ "name": "...", "payload": ... }`) йәки строка (тапламалар өсөн ғәҙәттәгесә `null`); `GRANT_PERMISSION`/`REVOKE_PERMISSION` ҡабул итеү `&Name` йәки `&Json(Permission)` X.

- теркәлмәгән опс (домен/иҫәп/актив): инварианттар (макет)
  - ЮНРЕГИСТЕР_ДОМАИН (`r10=&DomainId`) домен йәки активтар билдәләмәләре бар икән, уңышһыҙлыҡҡа осрай.
  - ЮНРЕГИСТЕР_АККОУНТ ​​(`r10=&AccountId`) уңышһыҙлыҡҡа осрай, әгәр иҫәптә нуль булмаған баланстары бар йәки НФТ-ға эйә.
  - ЮНРЕГИСТЕР_АССЕТ (`r10=&AssetDefinitionId`) актив өсөн ниндәйҙер баланстар булһа, уңышһыҙлыҡҡа осрай.

Иҫкәрмәләр
- Был миҫалдарҙа һынауҙарҙа ҡулланылған макет WSV хост сағылдыра; ысын төйөн хужалары байыраҡ админ схемаларын фашлай ала йәки өҫтәмә раҫлау талап итә. Ҡағиҙәләр күрһәткестәре һаман да ҡағыла: TLVs INPUT, версия=1, тип идентификаторҙар тура килергә тейеш, һәм файҙалы йөк хештары раҫлау тейеш.

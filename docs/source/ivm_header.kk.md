# IVM Байткод тақырыбы


Сиқыр
- 4 байт: ASCII `IVM\0` 0 ығысуында.

Орналасу (ағымдағы)
- Офсеттер мен өлшемдер (барлығы 17 байт):
  - 0..4: сиқырлы `IVM\0`
  - 4: `version_major: u8`
  - 5: `version_minor: u8`
  - 6: `mode: u8` (функция биттері; төменде қараңыз)
  - 7: `vector_length: u8`
  - 8..16: `max_cycles: u64` (кіші-эндиан)
  - 16: `abi_version: u8`

Режим биттері
- `ZK = 0x01`, `VECTOR = 0x02`, `HTM = 0x04` (сақталған/мүмкіндік қақпасы бар).

Өрістер (мағынасы)
- `abi_version`: жүйені шақыру кестесі және көрсеткіш-ABI схемасының нұсқасы.
- `mode`: ZK бақылау/ВЕКТОР/HTM үшін мүмкіндік биттері.
- `vector_length`: векторлық операцияларға арналған логикалық вектор ұзындығы (0 → орнатылмаған).
- `max_cycles`: ZK режимінде және рұқсатта қолданылатын орындалу толтырғышы.

Ескертпелер
- Endianness және орналасу іске асыру арқылы анықталады және `version` байланысты. Жоғарыдағы сым схемасы `crates/ivm_abi/src/metadata.rs` ішіндегі ағымдағы енгізуді көрсетеді.
- Ең аз оқырман ағымдағы артефактілер үшін осы орналасуға сене алады және болашақ өзгерістерді `version` қақпасы арқылы өңдеуі керек.
- Аппараттық жеделдету (SIMD/Metal/CUDA) әрбір хостқа қосылады. Орындау уақыты `iroha_config` мәнінен `AccelerationConfig` мәндерін оқиды: `enable_simd` жалған болған кезде скалярлық резервтерді мәжбүрлейді, ал `enable_metal` және `enable_cuda` қақпалары тіпті олардың компиляциялық серверлері арқылы қолданылған кезде де. VM жасалғанға дейін `ivm::set_acceleration_config`.
- Мобильді SDK (Android/Swift) бірдей тұтқаларды қамтиды; `IrohaSwift.AccelerationSettings`
  `connect_norito_set_acceleration_config` шақырады, сондықтан macOS/iOS құрастырулары Metal /
  NEON детерминистік резервтерді сақтай отырып.
- Операторлар `IVM_DISABLE_METAL=1` немесе `IVM_DISABLE_CUDA=1` экспорттау арқылы диагностикаға арналған арнайы серверлерді мәжбүрлеп өшіре алады. Бұл ортаны қайта анықтау конфигурациядан басымдылыққа ие болады және VM детерминирленген процессор жолында сақталады.

Төзімді күй көмекшілері және ABI беті
- Тұрақты күй анықтамасының жүйелік қоңыраулары (0x50–0x5A: STATE_{GET,SET,DEL}, ENCODE/DECODE_INT, BUILD_PATH_* және JSON/SCHEMA кодтау/декодтау) V1 ABI бөлігі болып табылады және `abi_hash` есептеуіне кіреді.
- CoreHost сымдары STATE_{GET,SET,DEL} - WSV қолдайтын берік смарт-келісімшарт күйіне; әзірлеуші/сынақ хосттары қабаттасуды немесе жергілікті тұрақтылықты пайдалана алады, бірақ бірдей бақыланатын әрекетті сақтауы керек.

Валидация
- Түйінді қабылдау тек `version_major = 1` және `version_minor = 0` тақырыптарын қабылдайды.
- `mode` тек белгілі биттерді қамтуы керек: `ZK`, `VECTOR`, `HTM` (белгісіз биттер қабылданбайды).
- `vector_length` кеңес беру болып табылады және `VECTOR` биті орнатылмаған болса да нөлге тең болмауы мүмкін; қабылдау тек жоғарғы шекараны ғана жүзеге асырады.
- Қолдау көрсетілетін `abi_version` мәндері: бірінші шығарылым тек `1` (V1) қабылдайды; басқа мәндер қабылдау кезінде қабылданбайды.

- Шағын жаңартулар `feature_bits` және резервтелген операциялық код кеңістігінің артындағы нұсқауларды қосуы мүмкін; негізгі жаңартулар кодтауларды өзгертуі немесе протоколды жаңартумен бірге ғана жоюы/қайта тағайындауы мүмкін.
- Syscall диапазондары тұрақты; белсенді `abi_version` үшін белгісіз `E_SCALL_UNKNOWN` береді.
- Газ кестелері `version` стандартына байланысты және өзгерту кезінде алтын векторларды қажет етеді.

Артефактілерді тексеру
- Тақырып өрістерінің тұрақты көрінісі үшін `ivm_tool inspect <file.to>` пайдаланыңыз.
- Әзірлеу үшін мысалдар/ құрастырылған артефактілерді тексеретін шағын Makefile мақсатты `examples-inspect` қамтиды.

Мысал (Rust): минималды сиқыр + өлшемді тексеру

```rust
use std::fs::File;
use std::io::{Read};

fn is_ivm_artifact(path: &std::path::Path) -> std::io::Result<bool> {
    let mut f = File::open(path)?;
    let mut magic = [0u8; 4];
    if f.read(&mut magic)? != 4 { return Ok(false); }
    if &magic != b"IVM\0" { return Ok(false); }
    let meta = std::fs::metadata(path)?;
    Ok(meta.len() >= 64)
}
```

Ескертпе: Сиқырдан тыс дәл тақырып орналасуы нұсқаланған және іске асыру анықталған; тұрақты өріс атаулары мен мәндері үшін `ivm_tool inspect` артықшылық береді.

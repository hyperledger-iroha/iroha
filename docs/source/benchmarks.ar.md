# تقرير المقارنة

توجد لقطات تفصيلية لكل عملية تشغيل وسجل FASTPQ WP5-B
[`benchmarks/history.md`](benchmarks/history.md)؛ استخدم هذا الفهرس عند الإرفاق
المصنوعات اليدوية لمراجعات خارطة الطريق أو عمليات تدقيق SRE. تجديدها مع
`python3 scripts/fastpq/update_benchmark_history.py` عند التقاط وحدة معالجة الرسومات الجديدة
أو يظهر بوسيدون الأرض.

## حزمة أدلة التسريع

يجب أن يتضمن كل معيار GPU أو معيار الوضع المختلط إعدادات التسريع المطبقة
لذلك يمكن لـ WP6-B/WP6-C إثبات تكافؤ التكوين جنبًا إلى جنب مع عناصر التوقيت.

- التقط لقطة وقت التشغيل قبل/بعد كل تشغيل:
  `cargo xtask acceleration-state --format json > artifacts/acceleration_state_<stamp>.json`
  (استخدم `--format table` للسجلات التي يمكن للإنسان قراءتها). هذا يسجل `enable_{metal,cuda}`،
  عتبات Merkle، وحدود تحيز وحدة المعالجة المركزية SHA-2، وبتات صحة الواجهة الخلفية المكتشفة، وأي شيء آخر
  أخطاء التكافؤ لزجة أو تعطيل الأسباب.
- قم بتخزين JSON بجوار مخرجات المعيار الملتفة
  (`artifacts/fastpq_benchmarks/*.json`، `benchmarks/poseidon/*.json`، عملية مسح ميركل
  يلتقط، وما إلى ذلك) حتى يتمكن المراجعون من اختلاف التوقيتات والتكوين معًا.
- تعريفات المقبض والإعدادات الافتراضية موجودة في `docs/source/config/acceleration.md`؛ متى
  يتم تطبيق التجاوزات (على سبيل المثال، `ACCEL_MERKLE_MIN_LEAVES_GPU`، `ACCEL_ENABLE_CUDA`)،
  قم بتدوينها في بيانات تعريف التشغيل للحفاظ على إمكانية تكرار عمليات إعادة التشغيل عبر الأجهزة المضيفة.

## معيار Norito للمرحلة الأولى (WP5-B/C)

- الأمر: `cargo xtask stage1-bench [--size <bytes|Nk|Nm>]... [--iterations <n>]`
  يصدر JSON + Markdown ضمن `benchmarks/norito_stage1/` مع توقيتات حسب الحجم
  بالنسبة لمنشئ الفهرس الهيكلي العددي والمتسارع.
- أحدث عمليات التشغيل (macOS aarch64، ملف تعريف المطورين) مباشرة على
  `benchmarks/norito_stage1/latest.{json,md}` والتحويل الجديد من ملف CSV
  يعرض `examples/stage1_cutover` (`benchmarks/norito_stage1/cutover.csv`) SIMD
  يفوز من ~ 6 إلى 8 كيلو بايت فصاعدًا. وحدة معالجة الرسومات/المرحلة الموازية-1 أصبحت الآن افتراضية على **192 كيلو بايت**
  قطع (`NORITO_STAGE1_GPU_MIN_BYTES=<n>` للتجاوز) لتجنب سحق الإطلاق
  على المستندات الصغيرة مع تمكين المسرعات للحمولات الأكبر حجمًا.

## تجميع بوسيدون CUDA

يتضمن معيار Poseidon (`crates/ivm/benches/bench_poseidon.rs`) الآن أعباء العمل التي تمارس كلاً من التباديل الفردي والمساعدات المجمعة الجديدة. قم بتشغيل الجناح باستخدام:

```bash
cargo bench -p ivm bench_poseidon -- --save-baseline poseidon_cuda
```

سيسجل المعيار النتائج ضمن `target/criterion/poseidon*_many`. عندما يتوفر عامل GPU، قم بتصدير ملخصات JSON (على سبيل المثال، انسخ `target/criterion/**/new/benchmark.json` إلى `benchmarks/poseidon/criterion_poseidon2_many_cuda.json`) (على سبيل المثال، انسخ `target/criterion/**/new/benchmark.json` إلى `benchmarks/poseidon/`) حتى تتمكن الفرق النهائية من مقارنة إنتاجية وحدة المعالجة المركزية مقابل CUDA لكل حجم دفعة. وإلى أن يتم تشغيل مسار GPU المخصص، يعود المعيار إلى تنفيذ SIMD/CPU ويظل يوفر بيانات انحدار مفيدة لأداء الدفعة.

للحصول على لقطات متكررة (وللحفاظ على أدلة التكافؤ مع بيانات التوقيت)، قم بتشغيل

```bash
cargo xtask poseidon-cuda-bench --json-out benchmarks/poseidon/poseidon_cuda_latest.json \
  --markdown-out benchmarks/poseidon/poseidon_cuda_latest.md --allow-overwrite
```التي بذور دفعات Poseidon2/6 الحتمية، وتسجل أسباب الصحة/التعطيل CUDA، والفحوصات
التكافؤ مقابل المسار العددي، وتصدر ملخصات العمليات/الثانية + التسريع جنبًا إلى جنب مع المعدن
حالة وقت التشغيل (علامة الميزة، والتوفر، والخطأ الأخير). لا يزال مضيفو وحدة المعالجة المركزية (CPU) فقط يكتبون العددية
قم بالإشارة إلى المسرع المفقود ولاحظه، حتى تتمكن CI من نشر المصنوعات اليدوية حتى بدون وحدة معالجة الرسومات
عداء.

## معيار FASTPQ المعدني (Apple Silicon)

التقط مسار GPU تشغيلًا محدثًا شاملاً لـ `fastpq_metal_bench` على نظام التشغيل macOS 14 (arm64) مع مجموعة معلمات المسار المتوازن، و20000 صف منطقي (مبطن إلى 32768)، و16 مجموعة أعمدة. توجد القطعة الأثرية المغلفة في `artifacts/fastpq_benchmarks/fastpq_metal_bench_20k_refresh.json`، مع تخزين الأثر المعدني جنبًا إلى جنب مع الالتقاطات السابقة تحت `traces/fastpq_metal_trace_*_rows20000_iter5.trace`. متوسط التوقيتات (من `benchmarks.operations[*]`) يقرأ الآن:

| عملية | وحدة المعالجة المركزية تعني (مللي ثانية) | متوسط ​​المعدن (مللي ثانية) | تسريع (x) |
|-----------|--------------|-----------------|------------|
| التحويل السريع (32,768 مدخلاً) | 83.29 | 79.95 | 1.04 |
| IFFT (32,768 مدخلاً) | 93.90 | 78.61 | 1.20 |
| LDE (262,144 مدخلاً) | 669.54 | 657.67 | 1.02 |
| أعمدة تجزئة بوسيدون (524,288 مدخلاً) | 29,087.53 | 30,004.90 | 0.97 |

الملاحظات:

- يستفيد كل من FFT/IFFT من نواة BN254 المحدثة (يقوم IFFT بمسح الانحدار السابق بنسبة 20% تقريبًا).
- يبقى LDE بالقرب من التكافؤ؛ تسجل ميزة التعبئة الصفرية الآن 33,554,432 بايت مبطن بمتوسط ​​18.66 مللي ثانية، وبالتالي تلتقط حزمة JSON تأثير قائمة الانتظار.
- لا تزال تجزئة Poseidon مرتبطة بوحدة المعالجة المركزية (CPU) على هذا الجهاز؛ استمر في المقارنة مع بيانات Poseidon microbench حتى يتبنى المسار المعدني أحدث عناصر التحكم في قائمة الانتظار.
- كل لقطة تسجل الآن `AccelerationSettings.runtimeState().metal.lastError`، مما يسمح بذلك
  يقوم المهندسون بتعليق العناصر الاحتياطية لوحدة المعالجة المركزية مع سبب التعطيل المحدد (تبديل السياسة،
  فشل التكافؤ، لا يوجد جهاز) مباشرة في الأداة المعيارية.

لإعادة إنتاج التشغيل، أنشئ النوى المعدنية ونفذ:

```bash
FASTPQ_METAL_LIB=target/release/build/fastpq_prover-*/out/fastpq.metallib \
FASTPQ_METAL_TRACE_CHILD=1 \
cargo run -p fastpq_prover --features fastpq-gpu --bin fastpq_metal_bench --release \
  -- --rows 20000 --iterations 5 --output fastpq_metal_bench_20k.json
```

قم بإلزام JSON الناتج تحت `artifacts/fastpq_benchmarks/` مع الأثر المعدني بحيث يظل دليل الحتمية قابلاً للتكرار.

## أتمتة FASTPQ CUDA

يمكن لمضيفي CUDA تشغيل معيار SM80 وتغليفه في خطوة واحدة باستخدام:

```bash
cargo xtask fastpq-cuda-suite \
  --rows 20000 --iterations 5 --columns 16 \
  --row-usage artifacts/fastpq_benchmarks/fastpq_row_usage_2025-05-12.json \
  --label device_class=xeon-rtx --device rtx-ada
```

يقوم المساعد باستدعاء `fastpq_cuda_bench`، والخيوط من خلال التسميات/الجهاز/الملاحظات، والتكريمات
`--require-gpu`، و(افتراضيًا) يلتف/يوقع عبر `scripts/fastpq/wrap_benchmark.py`.
تتضمن المخرجات ملف JSON الخام، والحزمة المغلفة تحت `artifacts/fastpq_benchmarks/`،
و`<name>_plan.json` بجوار الإخراج الذي يسجل الأوامر/البيئة الدقيقة لذلك
تظل لقطات المرحلة 7 قابلة للتكرار عبر مشغلات GPU. أضف `--sign-output` و
`--gpg-key <id>` عندما تكون التوقيعات مطلوبة؛ استخدم `--dry-run` لإصدار ملف
خطة/مسارات دون تنفيذ مقاعد البدلاء.

### التقاط إصدار GA (macOS 14 Arm64، مسار متوازن)

لإرضاء WP2-D، سجلنا أيضًا إصدارًا مبنيًا على نفس المضيف مع جاهزية GA
الاستدلال على قائمة الانتظار ونشرها باسم
`fastpq_metal_bench_20k_release_macos14_arm64.json`. القطعة الأثرية تلتقط اثنين
دفعات الأعمدة (مسار متوازن، مبطن إلى 32768 صفًا) ويتضمن بوسيدون
عينات microbench لاستهلاك لوحة القيادة.| عملية | وحدة المعالجة المركزية تعني (مللي ثانية) | متوسط ​​المعدن (مللي ثانية) | تسريع | ملاحظات |
|-----------|--------------|-----------------|---------|-------|
| التحويل السريع (32,768 مدخلاً) | 12.741 | 10.963 | 1.16× | تقوم نواة GPU بتتبع حدود قائمة الانتظار المحدثة. |
| IFFT (32,768 مدخلاً) | 17.499 | 25.688 | 0.68× | يُعزى الانحدار إلى انتشار قائمة الانتظار المحافظة؛ الحفاظ على ضبط الاستدلال. |
| LDE (262,144 مدخلاً) | 68.389 | 65.701 | 1.04× | تسجل عملية التعبئة الصفرية 33,554,432 بايت في 9.651 مللي ثانية لكلتا الدفعتين. |
| أعمدة تجزئة بوسيدون (524,288 مدخلاً) | 1,728.835 | 1,447.076 | 1.19× | أخيرًا تتفوق وحدة معالجة الرسومات على وحدة المعالجة المركزية بعد تعديل قائمة انتظار Poseidon. |

تُظهر قيم Poseidon microbench المضمنة في JSON سرعة 1.10× (المسار الافتراضي
596.229ms مقابل 656.251ms العددية عبر خمس تكرارات)، بحيث يمكن الآن للوحات المعلومات التخطيط
تحسينات لكل حارة بجانب المقعد الرئيسي. إعادة إنتاج التشغيل باستخدام:

```bash
FASTPQ_METAL_LIB=target/release/build/fastpq_prover-*/out/fastpq.metallib \
FASTPQ_METAL_TRACE_CHILD=1 \
cargo run -p fastpq_prover --features fastpq-gpu --bin fastpq_metal_bench --release \
  -- --rows 20000 --iterations 5 \
  --output fastpq_metal_bench_20k_release_macos14_arm64.json
```

احتفظ بتتبعات JSON و`FASTPQ_METAL_TRACE_CHILD=1` الملتفة محددة في الأسفل
`artifacts/fastpq_benchmarks/` لذلك يمكن لمراجعات WP2-D/WP2-E اللاحقة أن تختلف عن GA
الالتقاط مقابل عمليات التحديث السابقة دون إعادة تشغيل عبء العمل.

كل التقاط `fastpq_metal_bench` جديد يكتب الآن أيضًا كتلة `bn254_metrics`،
الذي يكشف إدخالات `acceleration.bn254_{fft,ifft,lde,poseidon}_ms` لوحدة المعالجة المركزية
خط الأساس وأي واجهة خلفية لوحدة معالجة الرسومات (Metal/CUDA) كانت نشطة، **و** أ
كتلة `bn254_dispatch` التي تسجل عرض مجموعة مؤشرات الترابط التي تمت ملاحظتها، ومؤشر الترابط المنطقي
التعدادات وحدود خطوط الأنابيب لإرسالات BN254 FFT/LDE ذات العمود الواحد. ال
يقوم المجمع المعياري بنسخ كلتا الخريطتين إلى `benchmarks.bn254_*`، لذا فإن لوحات المعلومات و
يمكن لمصدري Prometheus التخلص من زمن الاستجابة المسمى والهندسة دون إعادة التحليل
مجموعة العمليات الخام. ينطبق الآن تجاوز `FASTPQ_METAL_THREADGROUP` على
نواة BN254 أيضًا، مما يجعل عمليات مسح مجموعة الخيوط قابلة للتكرار من مقبض واحد.

لإبقاء لوحات المعلومات النهائية بسيطة، قم بتشغيل `python3 scripts/benchmarks/export_csv.py`
بعد الاستيلاء على حزمة. يقوم المساعد بتسوية `poseidon_microbench_*.json` إلى
مطابقة ملفات `.csv` بحيث يمكن لوظائف الأتمتة أن تفرق بين الممرات الافتراضية والممرات العددية دون
محللون مخصصون.

## مقعد بوسيدون الصغير (معدن)

| ملخص | حزمة ملفوفة | المتوسط ​​الافتراضي (مللي ثانية) | المتوسط ​​العددي (مللي ثانية) | تسريع مقابل العددية | الأعمدة x الحالات | التكرارات |
|---------|----------------|-------------------|------------------|-------------------|------------------|------------|
| `benchmarks/poseidon/poseidon_microbench_full.json` | `fastpq_metal_bench_full.json` | 1,990.49 | 1,994.53 | 1.002 | 64×262,144 | 5 |
| `benchmarks/poseidon/poseidon_microbench_debug.json` | `fastpq_metal_bench_debug.json` | 2,167.66 | 2,152.18 | 0.993 | 64×262,144 | 5 |تم التقاط كلا الالتقاطين بتجزئة 262,144 حالة لكل تشغيل (سجل التتبع 2 = 12) مع تكرار إحماء واحد. يتوافق المسار "الافتراضي" مع النواة متعددة الحالات المضبوطة بينما يقوم "الرقمي" بتأمين النواة على حالة واحدة لكل حارة للمقارنة.

## عمليات مسح عتبة ميركل

يؤكد المثال `merkle_threshold` (`cargo run --release -p ivm --features metal --example merkle_threshold -- --json`) على مسارات التجزئة Metal-vs-CPU Merkle. أحدث التقاط AppleSilicon (Darwin 25.0.0 Arm64، `ivm::metal_available()=true`) موجود في `benchmarks/merkle_threshold/takemiyacStudio.lan_25.0.0_arm64.json` مع تصدير ملف CSV مطابق. تظل الخطوط الأساسية لنظام التشغيل macOS 14 لوحدة المعالجة المركزية (CPU) فقط ضمن `benchmarks/merkle_threshold/macos14_arm64_{cpu,metal}.json` للمضيفين الذين ليس لديهم Metal.

| أوراق | أفضل وحدة المعالجة المركزية (مللي ثانية) | المعدن الأفضل (مللي ثانية) | تسريع |
|--------|-------------|-----------------|---------|
| 1,024 | 23.01 | 19.69 | 1.17× |
| 4,096 | 50.87 | 62.12 | 0.82× |
| 8,192 | 95.77 | 96.57 | 0.99× |
| 16,384 | 64.48 | 58.98 | 1.09× |
| 32,768 | 109.49 | 87.68 | 1.25× |
| 65,536 | 177.72 | 137.93 | 1.29× |

تستفيد أعداد الأوراق الأكبر حجمًا من المعدن (1.09–1.29×)؛ لا تزال المجموعات الأصغر تعمل بشكل أسرع على وحدة المعالجة المركزية، لذا يحتفظ ملف CSV بكلا العمودين للتحليل. يحتفظ مساعد CSV بالعلامة `metal_available` بجانب كل ملف تعريف للحفاظ على محاذاة لوحات معلومات انحدار وحدة المعالجة المركزية (GPU) مقابل وحدة المعالجة المركزية (CPU).

خطوات الاستنساخ:

```bash
cargo run --release -p ivm --features metal --example merkle_threshold -- --json \
  > benchmarks/merkle_threshold/<hostname>_$(uname -r)_$(uname -m).json
```

قم بتعيين `FASTPQ_METAL_LIB`/`FASTPQ_GPU` إذا كان المضيف يتطلب تمكين Metal بشكل صريح، واحتفظ بكل من لقطات وحدة المعالجة المركزية + وحدة معالجة الرسومات محددة حتى يتمكن WP1-F من رسم حدود السياسة.

عند التشغيل من shell بدون رأس، قم بتعيين `IVM_DEBUG_METAL_ENUM=1` لتسجيل تعداد الأجهزة و`IVM_FORCE_METAL_ENUM=1` لتجاوز `MTLCreateSystemDefaultDevice()`. تقوم واجهة سطر الأوامر (CLI) بتسخين جلسة CoreGraphics **قبل** المطالبة بالجهاز المعدني الافتراضي وتعود إلى `MTLCreateSystemDefaultDevice()` عندما يُرجع `MTLCopyAllDevices()` الصفر؛ إذا استمر المضيف في عدم الإبلاغ عن أي أجهزة، فسيحتفظ الالتقاط بـ `metal_available=false` (توجد خطوط أساسية مفيدة لوحدة المعالجة المركزية (CPU) ضمن `macos14_arm64_*`)، بينما يجب على مضيفي GPU الاحتفاظ بتمكين `FASTPQ_GPU=metal` حتى تسجل الحزمة الواجهة الخلفية المختارة.

يعرض `fastpq_metal_bench` مقبضًا مشابهًا عبر `FASTPQ_DEBUG_METAL_ENUM=1`، الذي يطبع نتائج `MTLCreateSystemDefaultDevice`/`MTLCopyAllDevices` قبل أن تقرر الواجهة الخلفية ما إذا كانت ستبقى على مسار GPU أم لا. قم بتمكينه عندما لا يزال `FASTPQ_GPU=gpu` يُبلغ عن `backend="none"` في ملف JSON المغلف بحيث تسجل حزمة الالتقاط بالضبط كيفية تعداد المضيف للأجهزة المعدنية؛ يتم إحباط الحزام على الفور عند ضبط `FASTPQ_GPU=gpu` ولكن لم يتم اكتشاف أي مُسرّع، مع الإشارة إلى مقبض التصحيح بحيث لا تخفي حزمة الإصدار أبدًا احتياطي وحدة المعالجة المركزية خلف تشغيل وحدة معالجة الرسومات الإجبارية.

يقوم مساعد CSV بإصدار جداول لكل ملف تعريف (على سبيل المثال `macos14_arm64_*.csv` و`takemiyacStudio.lan_25.0.0_arm64.csv`)، مع الحفاظ على علامة `metal_available` حتى تتمكن لوحات معلومات الانحدار من استيعاب قياسات وحدة المعالجة المركزية (CPU) ووحدة معالجة الرسومات (GPU) بدون محللين مخصصين.

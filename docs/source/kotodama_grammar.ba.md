Kotodama был псевдонимдарҙы кире булмаған күләмдәр тип ҡарай; арифметика тикшерелә, псевдонимын һаҡлай, ә нулгә ташыу йәки бүленеш буйынса тоҙаҡтар. ## Контейнер һәм метамағлүмәт

Синтаксис
```
seiyaku Name {
  meta {
    abi_version: 1,
    vector_length: 0,
    max_cycles: 0,
    features: ["zk", "simd"],
  }

  state int counter;

  hajimari() { counter = 0; }

  kotoage fn inc() { counter = counter + 1; }
}
```

Семантика
- `meta { ... }` ятҡылыҡтары өҫтөндә йөрөүсе компиляторҙар өсөн ғәҙәттәгесә IVM башлығы: `abi_version`, `vector_length` (0 тигәнде аңлата), `max_cycles` (0 тигәнде аңлата компилятор), `features` toggles башлыҡ функцияһы биттар (ZK эҙләү, вектор иғлан). Компилятор `max_cycles: 0`-ты “ҡулланыу ғәҙәттәгесә” тип дауалай һәм ҡабул итеү талаптарын ҡәнәғәтләндерер өсөн конфигурацияланған нульдән тыш ғәҙәттән тыш хәлдәрҙе сығара. Ярҙамһыҙ функциялар иҫкәртмә менән иғтибарға алынмай. Ҡасан `meta {}` төшөрөп ҡалдырылған, компилятор мисс `abi_version = 1` һәм ҡалған башлыҡ баҫыуҙары өсөн вариант ғәҙәттәгесә ҡуллана.
- `features: ["zk", "simd"]` (псевдонимдар: `"vector"`) тейешле башлыҡ биттарын асыҡтан-асыҡ һорай. Билдәһеҙ функциялы ептәр хәҙер эконом хатаһын етештерә, ә иғтибарға алынмай.
- `state` ныҡлы килешеп үҙгәртеүселәрҙе иғлан итә. Компилятор `STATE_GET/STATE_SET/STATE_DEL` syscalls-ҡа инеүҙәрҙе түбәнәйтә һәм хужа уларҙы пер-транзакция өҫтөндәге ҡаплауҙа (тикшерергә/тергеҙергә кире ҡайтарыу, WSV-ға коммитҡа ҡайтарыу) ҡуя. Ҡулланыу кәңәштәре туранан-тура дәүләт юлдары өсөн сығарыла; динамик асҡыстар карта кимәлендәге конфликт асҡыстарына кире инә. Асыҡ хост-ярҙам уҡыу/яҙмалар өсөн, ҡулланыу `state_get/state_set/state_del` ярҙамсылары һәм `get_or_insert_default` карта ярҙамсылары; был маршрут аша Norito TLVs һәм исемдәр/ялан заказ тотороҡло һаҡлау.
- Дәүләт идентификаторҙары һаҡлана; күләгәһе `state` исеме параметрҙары йәки `let` бәйләүҙәр кире ҡағыла (`E_STATE_SHADOWED`).
- Дәүләт картаһы ҡиммәттәре беренсе класлы түгел: дәүләт идентификаторын карта операциялары һәм итерацион өсөн туранан-тура ҡулланыу. Ҡулланыу йәки тапшырыу дәүләт карталарын ҡулланыусылар билдәләгән функцияларға кире ҡағыла (`E_STATE_MAP_ALIAS`).
- Ҡаты дәүләт карталары әлеге ваҡытта `int` һәм күрһәткес-ABI төп төрҙәрен генә хуплай; башҡа төп төрҙәре компиляция ваҡытында кире ҡағыла.
- Ҡаты дәүләт яландары `int`, `bool`, `Json`, `Blob`/`bytes`, йәки күрһәткес-АБИ типтары (шул иҫәптән был баҫыуҙарҙан торған структур/трубкалар) булырға тейеш; `string` ныҡлы дәүләт өсөн ярҙам итмәй.

## Функциялар һәм параметрҙар

- Яңыртыу ҡармаҡ: `kaizen(args...) permission(Role) { ... }`.

Параметрҙар һәм кире ҡайтара
- `r10..r22` регистрҙарында аргументтар ҡиммәттәр йәки INPUT күрһәткестәре булараҡ ҡабул ителә (Norito TLV) бер АБИ; өҫтәмә аргтар ҡойолоу өсөн өйөм.
- Функциялар нуль йәки бер скаляр йәки кортеж ҡайтара. Беренсел ҡайтарыу хаҡы `r10` скаляр өсөн; кортеждар конвенция буйынса өйөм/ОУТПУТ-та матдилаштырыла.

##- Үҙгәрмәй торған бәйләүҙәр: `let x = expr;`, `let mut x = expr;` (уҡытыусанлыҡ — компиляция-ваҡыт тикшерелеүе; урындағы халыҡ өсөн генә йөрөү ваҡыты мутацияһы рөхсәт ителә).
- Задание: `x = expr;` һәм ҡушма формалары Kotodama һ.б. кортеж/структура ҡырҙары үҙгәрмәй.
- Һанлы псевдонимдар (`fixed_u128`, `Amount`, `Balance`) айырым `Numeric`-артҡы типтары; арифметик псевдонимдарҙы һаҡлай һәм псевдонимдарҙы ҡатыштырыу `int` бәйләү аша үҙгәртеп ҡороуҙы талап итә. `int`-тан/конверсиялар эшләү ваҡытында тикшерелә (тиҫкәре булмаған, интеграль, диапазон-сикләнгән).
- Контроль: `if (cond) { ... } else { ... }`, `while (cond) { ... }`, C-стиль `for (init; cond; step) { ... }`.
  - `for` инициализаторҙары һәм аҙымдар ябай `let name = expr` йәки экспрессия белдереүҙе булырға тейеш; ҡатмарлы емереклектәр кире ҡағыла (`E0005`, `E0006`).
  - `for` scoping: init һөйләмдән бәйләүҙәр иллюминаторҙа һәм унан һуң күренә; организмда йәки аҙымда барлыҡҡа килгән бәйләүҙәр иллюзиянан ҡотолмай.
- Тигеҙлек (`==`, `!=`) `int`, `bool`, `string`, күрһәткес-ABI скалярстары өсөн ярҙам ителә (мәҫәлән, Kotodama, `Name`, `Blob`/`bytes`, `Json` X); кортеждар, структуралар һәм карталар сағыштырмаса түгел.
- Карта циклы: `for (k, v) in map { ... }` (детерминистик; түбән ҡарағыҙ).
- Ағым: `return expr;`, Kotodama, `continue;`.
- Шылтыратыу: `name(args...);` йәки `call name(args...);` (икеһе лә ҡабул ителгән; компилятор шылтыратыу өсөн нормалаштырылған).
- Ассерциялар: `assert(cond);`, `assert_eq(a, b);` картаһы Kotodama IVM ZK булмаған төҙөүҙәр йәки ZK ZK режимында ZK режимында.

## Йыйынтыҡтары һәм карталары

Тип: Kotodama.
- Хәтерҙәге карталар (өҫтәмә-бүленгән `Map::new()` аша йәки параметрҙар булараҡ үткән) бер асҡыс/ҡиммәте парын һаҡлағыҙ; Асҡыс һәм ҡиммәттәр һүҙ ҙурлығында булырға тейеш: `int`, `bool`, `string`, `Blob`, `bytes`, `Json`, йәки күрһәткес типтары (мәҫәлән,, `AccountId`, `Name`).
- ныҡлы дәүләт карталары (`state Map<...>`) Norito-кодланған асҡыстар/ҡиммәттәр ҡулланыу. Ярҙам асҡыстар: `int` йәки күрһәткес типтары. Ҡиммәттәр: `int`, `bool`, `Json`, `Blob`/Norito, йәки күрһәткес төрҙәре.
- `Map::new()` X бүленә һәм нуль-инициализациялау бер хәтерҙәге яҙма (асҡыс/ҡиммәте = 0); `Map<int,int>` карталары өсөн асыҡ типтағы аннотация йәки ҡайтарыу тибы бирергә.
- Дәүләт карталары беренсе класлы ҡиммәттәр түгел: уларҙы яңынан тәғәйенләй алмайһығыҙ (мәҫәлән, `M = Map::new()`); индексациялау аша яңыртыу яҙмалары (`M[key] = value`).
- Операциялар:
  - Индексациялау: `map[key]` алыу/ҡуйма ҡиммәте (комплект хост syscall аша башҡарылған; ҡарағыҙ эшләү ваҡыты API картаһы).
  - Барлығы: `contains(map, key) -> bool` (аҫҡы ярҙамсы; эске сискалл булыуы мөмкин).
  - Итерация: детерминистик тәртип һәм мутация ҡағиҙәләре менән `for (k, v) in map { ... }`.

Детерминистик итерацион ҡағиҙәләр
- итерацион йыйылмаһы — иллюминатор яҙмаһында асҡыстар снимок.
- Заказ ҡәтғи күтәрелеү байт-лексикографик бойороҡ Norito-кодланған асҡыстар.
- Структур модификациялар (инсерт/юйыу/асыҡ) итерацион картаға иллюминатор ваҡытында детерминистик `E_ITER_MUTATION`X тоҙаҡ сәбәпсе.
- Сикләнгәнлек талап ителә: йә иғлан ителгән макс (`@max_len`) картала, `#[bounded(n)]` асыҡ атрибуты, йәки `.take(n)`/`.range(..)` ҡулланып асыҡ сикләнгән; юғиһә компилятор `E_UNBOUNDED_ITERATION` сыға.

Ярҙамсыларҙы бәйләй
- `#[bounded(n)]`: карта экспрессияһы буйынса өҫтәмә атрибут, мәҫ. `for (k, v) in my_map #[bounded(2)] { ... }`.
- `.take(n)`: старттан башлап тәүге Kotodama яҙмаларын итера.
- `.range(start, end)`: Norito яртылаш асыҡ интервалында итераты. Семантика `start` һәм `n = end - start` X-ға тиң.Динамик сиктәрҙәге иҫкәрмәләр
- Һөйләм сиктәре: `n`, `start`, һәм `end` бөтөн һан литералдары булараҡ тулыһынса нығытыла һәм итерациондар һаны нығытылған һанына төҙөлә.
- Һеҙ `kotodama_dynamic_bounds` функцияһы `ivm` йәшниктә эшләй, компилятор динамик `n`, `\r` һәм `end` экспрессияһын һәм хәүефһеҙлек өсөн эшләү ваҡытында раҫлауҙарҙы индерә. (тиҫкәре булмаған, Norito). Lowering K-ға тиклем `if (i < n)` чектары менән итерацион рәүештә өҫтәмә тәнде үтәүҙән ҡотолоу өсөн сығарыла (күбәләк K=2). Һеҙ K программалы көйләй аласыз аша `CompilerOptions { dynamic_iter_cap, .. }`.
- `koto_lint` Run Kotodama линт иҫкәртмәләрен тикшерергә тиклем йыйылмаға тиклем; төп компилятор һәр ваҡыт анализ һәм тип-тикшеренеүҙән һуң түбәнәйтеү менән бара.
- Хаталар кодтары [Kotodama компилятор хаталар кодтары] (./kotodama_error_codes.mdX) документлаштырылған); тиҙ аңлатмалар өсөн `koto_compile --explain <code>` ҡулланыу.

## Хаталар һәм диагностика

Компиляция ваҡытында диагностика (миҫалдар)
- `E_UNBOUNDED_ITERATION`: карта аша циклға бәйләнгәне юҡ.
- `E_MUT_DURING_ITER`: цикллы тәндәге итерацион картаның структур мутацияһы.
- `E_STATE_SHADOWED`: локаль бәйләүҙәр `state` декларацияларын күләгә алмай.
- `E_BREAK_OUTSIDE_LOOP`: Norito иллюминатор тышында ҡулланыла.
- `E_CONTINUE_OUTSIDE_LOOP` X: `continue` иллюминатор тышында ҡулланыла.
- `E0005`: 2012 йылда инициализаторға ярҙам итеүгә ҡарағанда ҡатмарлыраҡ.
- `E0006`: цикл өсөн аҙым пункты ярҙамға ҡарағанда ҡатмарлыраҡ.
- `E_BAD_POINTER_USE`: беренсе класлы тип кәрәк булған күрһәткес-ABI конструкторы һөҙөмтәһен ҡулланып.
- `E_UNRESOLVED_NAME`, Kotodama, `E_ARITY_MISMATCH`, Norito.
- Ҡоралландырыу: `koto_compile` байт-кодты сығарыу алдынан линт үткәреүҙе етәкләй; ҡулланыу `--no-lint` үткәрергә йәки `--deny-lint-warnings` линт сығарыу өҫтөндә төҙөү етешһеҙлектәре.

Ваҡытлы ВМ хаталары (һайланған; тулы исемлек ivm.md)
- `E_NORITO_INVALID`, `E_OOB`, `E_UNALIGNED`, `E_SCALL_UNKNOWN`, `E_ASSERT`, `E_ASSERT_EQ`, `E_ITER_MUTATION`.

## Codegen картаһы IVM

Торба .
1. Лексер/Парсер АСТ етештерә.
2. Семантик анализ исемдәрҙе, чектар төрҙәрен һәм символ таблицаларын тултыра.
3. ИК ябай SSA-кеүек формаға төшөрөү.
4. Теркәү IVM GPRs (`r10+` өсөн аргс/рет өсөн шылтыратыу конвенцияһы); ҡойолоу өсөн өйөм.
5. Байтекод эмиссияһы: IVM-тыуған һәм RV-компата кодировкалары ҡатнашмаһы рөхсәт ителгәнсә; `abi_version` менән сығарылған метамағлүмәттәр башы, үҙенсәлектәре, вектор оҙонлоғо, һәм `max_cycles`.Карта төшөрөү өҫтөнлөктәре
- IVM АЛУ опстарына арифметика һәм логика картаһы.
- Тармаҡ һәм контроль картаһы шартлы ботаҡтар һәм һикерергә; компилятор ҡыҫылған формалар ҡуллана, унда отошло.
- Урындағы халыҡ өсөн виртуаль стекаға ҡойола; тура килтереп үтәлә.
- Төҙөү өсөн түбәнерәк теркәү хәрәкәттәре һәм `SCALL` 8-битлы номеры менән.
- Norito TLVs INPUT төбәгенә урынлаштырылған һәм үҙҙәренең адрестарын етештереүсе заманса конструкторҙар.
- Ассерциялар картаһы `ASSERT`/`ASSERT_EQ` ZK-нан тыш башҡарыуҙа тоҙаҡҡа һәм ZK төҙөүҙә сикләүҙәр сығарыу.

Детерминизм сикләүҙәре
- ФП юҡ; детерминистик булмаған сирҙәр юҡ.
- SIMD/GPU тиҙләнеше байткодҡа күренмәй һәм бит-бер үк булырға тейеш; компилятор аппаратҡа хас опстарҙы сығармай.

## АБИ, баш һәм Манифест

Компилятор тарафынан ҡуйылған Kotodama башлыҡтары
- `version`: IVM baytecode форматында версияһы (мажор.минор).
- `abi_version`: syscall таблицаһы һәм күрһәткес-АБИ схема версияһы.
- `feature_bits`: нәфис флагтар (мәҫәлән, `ZK`, `VECTOR`).
- `vector_len`: логик вектор оҙонлоғо (0 → unset).
- `max_cycles`: ҡабул итеү бәйләнгән һәм ZK прокладка кәңәше.

Манифест (теләкле sidecar)
- `code_hash`, `abi_hash`, метамағлүмәттәр `meta {}` блокынан, компилятор версияһы һәм ҡабатлаусанлыҡ өсөн кәңәштәр төҙөү.

## Юл картаһы

- **KD-231 (Apr2026):** итерацион сиктәр өсөн компиляция-ваҡыт диапазоны анализы өҫтәй, шуға күрә иллюзиялар графикка сикләнгән инеү йыйылмаларын фашлай.
- **KD-235 (May2026):** күрһәткес конструкторҙары һәм ABI асыҡлығы өсөн `string`-тан айырылып торған беренсе класлы `bytes` скалярын индерә.
- **KD-242 (июн2026):** детерминистик fallbacks менән функция флагтары артында төҙөлгән опкодтар йыйылмаһын (хаш / ҡултамға тикшерергә) киңәйтә.
- **KD-247 (июн2026):** локалләштерелгән диагностика өсөн `kotoba {}` таблицаларында картаны һаҡлау һәм картаны һаҡлау.
### Манифест эмиссияһы

- Kotodama компиляторы API `.to` менән бергә `ivm::kotodama::compiler::Compiler::compile_source_with_manifest` аша `ContractManifest` ҡайтара ала.
- Яландар:
  - `code_hash`: код байт хеш (IVM башлығын һәм тура һүҙлеләрҙән тыш) компилятор тарафынан артефакт бәйләү өсөн иҫәпләнә.
  - `abi_hash`: программаның `abi_version` өсөн рөхсәт ителгән syscall өҫтөнөң тотороҡло датчигы (ҡара: `ivm.md` һәм Norito).
- `/* ... */` һәм `features_bitmap`-ның инструменттар селтәрҙәре өсөн һаҡлана.
- `entrypoints`: экспортҡа сығарылған инеү нөктәләре исемлеге (йәмәғәт, `hajimari`, `kaizen`), шул иҫәптән уларҙы кәрәкле `permission(...)` ептәре һәм компиляторының иң яҡшы тырышлыҡ менән уҡыу/яҙмаһы асҡыстары, шуға күрә ҡабул итеү логикаһы һәм графиктар көтөлгән WSV инеү тураһында фекер йөрөтә ала.
- Манифест ҡабул итеү-ваҡыт чектары һәм реестрҙар өсөн тәғәйенләнгән; ҡара: йәшәү циклы өсөн `docs/source/new_pipeline.md`.

% Iroha 3 SLO აღკაზმულობა

Iroha 3 გამოშვების ხაზი ატარებს აშკარა SLO-ებს კრიტიკული Nexus ბილიკებისთვის:

- საბოლოო სლოტის ხანგრძლივობა (NX-18 კადენცია)
- მტკიცებულების გადამოწმება (სერთიფიკატების ჩაბარება, JDG ატესტაციები, ხიდის მტკიცებულებები)
- მტკიცებულება საბოლოო წერტილის მართვა (Axum ბილიკის პროქსი დადასტურების შეყოვნების მეშვეობით)
- საკომისიო და ფსონის ბილიკები (გადამხდელი/სპონსორი და ობლიგაციების/დაჭრილი ნაკადები)

## ბიუჯეტი

ბიუჯეტი ცხოვრობს `benchmarks/i3/slo_budgets.json`-ში და პირდაპირ სკამზეა რუკა
სცენარები I3 კომპლექტში. მიზნები არის თითო ზარის p99 სამიზნეები:

- საკომისიო/ფსონი: 50 ms თითო ზარზე (`fee_payer`, `fee_sponsor`, `staking_bond`, `staking_slash`)
- სერთიფიკატი / JDG / ხიდის დადასტურება: 80 ms (`commit_cert_verify`, `jdg_attestation_verify`,
  `bridge_proof_verify`)
- დაშვების სერტიფიკატის შეკრება: 80ms (`commit_cert_assembly`)
- წვდომის გრაფიკი: 50ms (`access_scheduler`)
- დამადასტურებელი საბოლოო წერტილის პროქსი: 120 ms (`torii_proof_endpoint`)

დამწვრობის სიჩქარის მინიშნებები (`burn_rate_fast`/`burn_rate_slow`) კოდირებს 14.4/6.0
მრავალფანჯრის კოეფიციენტები პეიჯინგისთვის და ბილეთების გაფრთხილებისთვის.

## აღკაზმულობა

გაუშვით აღკაზმულობა `cargo xtask i3-slo-harness`-ის საშუალებით:

```bash
cargo xtask i3-slo-harness \
  --iterations 64 \
  --sample-count 5 \
  --out-dir artifacts/i3_slo/latest
```

შედეგები:

- `bench_report.json|csv|md` — დაუმუშავებელი I3 სკამების კომპლექტის შედეგები (git hash + სცენარები)
- `slo_report.json|md` — SLO შეფასება უღელტეხილის/ჩავარდნის/ბიუჯეტის თანაფარდობით თითო სამიზნეზე

აღკაზმულობა მოიხმარს ბიუჯეტის ფაილს და ახორციელებს `benchmarks/i3/slo_thresholds.json`
სკამზე სირბილის დროს სწრაფად წარუმატებლობა, როდესაც სამიზნე რეგრესია.

## ტელემეტრია და დაფები

- საბოლოო: `histogram_quantile(0.99, rate(iroha_slot_duration_ms_bucket[5m]))`
- დადასტურების დადასტურება: `histogram_quantile(0.99, sum by (le) (rate(zk_verify_latency_ms_bucket{status="Verified"}[5m])))`

Grafana დამწყებ პანელები ცხოვრობს `dashboards/grafana/i3_slo.json`-ში. Prometheus
დამწვრობის სიჩქარის გაფრთხილებები მოწოდებულია `dashboards/alerts/i3_slo_burn.yml`-ში
ბიუჯეტის ზემოთ გამომცხვარი (საბოლოო 2 წმ, მტკიცებულების დადასტურება 80 ms, მტკიცებულების საბოლოო წერტილის პროქსი
120 ms).

## ოპერატიული შენიშვნები

- ატარეთ აღკაზმულობა ღამის საათებში; გამოაქვეყნეთ `artifacts/i3_slo/<stamp>/slo_report.md`
  სკამების არტეფაქტებთან ერთად მმართველობის მტკიცებულებისთვის.
- თუ ბიუჯეტი ვერ მოხერხდა, გამოიყენეთ სკამზე ნიშანდება სცენარის დასადგენად, შემდეგ გაბურღეთ
  შესატყვისი Grafana პანელი/გაფრთხილება პირდაპირ მეტრიკასთან კორელაციისთვის.
- მტკიცებულების საბოლოო წერტილის SLO-ები იყენებენ ვერიფიკაციის შეყოვნებას, როგორც მარშრუტზე, რათა თავიდან აიცილონ თითო მარშრუტი
  კარდინალურობის აფეთქება; საორიენტაციო სამიზნე (120ms) შეესაბამება შეკავებას/DoS-ს
  დამცავი მოაჯირები მტკიცებულების API-ზე.

# Iroha 3 նստարանային հավաքակազմ

Iroha 3 նստարանային փաթեթը գերազանցում է տաք ուղիները, որոնց վրա մենք ապավինում ենք խաղադրույքի ժամանակ, վճար
լիցքավորում, ապացույցների ստուգում, պլանավորում և ապացուցման վերջնակետեր: Այն աշխատում է որպես ան
`xtask` հրամանը դետերմինիստական հարմարանքներով (ֆիքսված սերմեր, ֆիքսված հիմնական նյութ,
և կայուն խնդրանքով օգտակար բեռներ), այնպես որ արդյունքները վերարտադրելի են բոլոր հյուրընկալող սարքերում:

## Աշխատում է հավաքակազմը

```bash
cargo xtask i3-bench-suite \
  --iterations 64 \
  --sample-count 5 \
  --json-out benchmarks/i3/latest.json \
  --csv-out benchmarks/i3/latest.csv \
  --markdown-out benchmarks/i3/latest.md \
  --threshold benchmarks/i3/thresholds.json \
  --allow-overwrite
```

Դրոշներ:

- `--iterations`-ը վերահսկում է կրկնությունները յուրաքանչյուր սցենարի նմուշի համար (կանխադրված՝ 64):
- `--sample-count`-ը կրկնում է յուրաքանչյուր սցենար՝ միջինը հաշվարկելու համար (կանխադրված՝ 5):
- `--json-out|--csv-out|--markdown-out` ընտրել ելքային արտեֆակտներ (բոլորը կամընտիր):
- `--threshold`-ը համեմատում է մեդիանները բազային սահմանների հետ (կոմպլեկտ `--no-threshold`
  բաց թողնել):
- `--flamegraph-hint`-ը նշում է Markdown զեկույցը `cargo flamegraph`-ով
  հրաման՝ սցենարը պրոֆիլավորելու համար:

CI սոսինձը ապրում է `ci/i3_bench_suite.sh`-ում և կանխադրված է վերևում գտնվող ուղիների վրա; հավաքածու
`I3_BENCH_ITERATIONS`/`I3_BENCH_SAMPLES`՝ գիշերային ժամերին գործարկման ժամանակը կարգավորելու համար:

## Սցենարներ

- `fee_payer` / `fee_sponsor` / `fee_insufficient` — վճարողի ընդդեմ հովանավորի դեբետ
  և պակասուրդի մերժում:
- `staking_bond` / `staking_slash` — պարտատոմսերի/չպարտատոմսերի հերթ՝ առանց և առանց
  կտրատելով.
- `commit_cert_verify` / `jdg_attestation_verify` / `bridge_proof_verify` —
  ստորագրության ստուգում պարտավորությունների վկայականների, JDG ատեստավորումների և կամուրջի վրա
  ապացույցների օգտակար բեռներ.
- `commit_cert_assembly` — ներդաշնակ հավաքում պարտավորությունների վկայականների համար:
- `access_scheduler` — կոնֆլիկտների մասին տեղեկացված մուտքի ժամանակացույց:
- `torii_proof_endpoint` — Axum-proof վերջնական կետի վերլուծություն + ստուգման հետադարձ ճանապարհորդություն:

Յուրաքանչյուր սցենար գրանցում է միջին նանովայրկյանները մեկ կրկնության, թողունակության և ա
դետերմինիստական տեղաբաշխման հաշվիչ արագ ռեգրեսիաների համար: Շեմերը ապրում են
`benchmarks/i3/thresholds.json`; bump սահմանները այնտեղ, երբ ապարատային փոփոխությունները եւ
կատարել նոր արտեֆակտ զեկույցի հետ մեկտեղ:

## Անսարքությունների վերացում

- Ամրացրեք պրոցեսորի հաճախականությունը/կառավարիչը՝ ապացույցներ հավաքելիս՝ աղմկոտ ռեգրեսիաներից խուսափելու համար:
- Օգտագործեք `--no-threshold` հետախուզական գործարկումների համար, այնուհետև նորից միացրեք, երբ բազայինը լինի
  թարմացված.
- Մեկ սցենարի պրոֆիլի համար սահմանեք `--iterations 1` և նորից գործարկեք տակ
  `cargo flamegraph -p xtask -- i3-bench-suite --iterations 128 --sample-count 1 --no-threshold --flamegraph-hint`.

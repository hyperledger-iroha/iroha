# מטרה

רשימת בדיקה זו מלכדת את הקריטריונים הקונקרטיים הנדרשים להצלחה
השלמת הביקורת החיצונית SM2/SM3/SM4. זה צריך להיבדק במהלך
בעיטה-off, ביקרו מחדש בכל נקודת ביקורת סטטוס, ושימשו לאישור יציאה
תנאים לפני הפעלת חתימת SM עבור מאמתי ייצור.

# מוכנות לפני אירוסין

- [ ] חוזה נחתם, כולל היקף, תוצרים, סודיות ו
      שפת תמיכה בתיקון.
- [ ] צוות הביקורת מקבל גישת מראה למאגר, דלי חפצי CI ו
      חבילת תיעוד הרשומה ב-`docs/source/crypto/sm_audit_brief.md`.
- [ ] נקודות מגע מאושרות עם גיבויים לכל תפקיד
      (קריפטו, IVM, פעולות פלטפורמה, אבטחה, מסמכים).
- [ ] בעלי עניין פנימיים מיישרים קו עם תאריך השחרור היעד ומקפיאים חלונות.
- [ ] ייצוא SBOM (`cargo auditable` + CycloneDX) נוצר ושותף.
- [ ] חבילת מקור בניית OpenSSL/Tongsuo הוכנה
      (מקור tarball hash, בניית סקריפט, הערות לשחזור).
- [ ] תפוקות הבדיקה הדטרמיניסטיות האחרונות שנתפסו:
      `scripts/sm_openssl_smoke.sh`, `cargo test -p iroha_crypto sm`, ו
      גופי Norito הלוך ושוב.
- [ ] פרסומת Torii `/v1/node/capabilities` (באמצעות `iroha runtime capabilities`) נרשמה, אימות שדות המניפסט `crypto.sm` ותמונת מצב של מדיניות האצה.

# ביצוע אירוסין

- [ ] סדנת פתיחה שהושלמה עם הבנה משותפת של מטרות,
      קווי זמן וקצב תקשורת.
- [ ] דוחות סטטוס שבועיים שהתקבלו ונבדקו; מרשם הסיכונים מעודכן.
- [ ] ממצאים נמסרו תוך יום עסקים אחד מרגע הגילוי כאשר החומרה
      הוא גבוה או קריטי.
- [ ] צוות הביקורת מאמת נתיבי דטרמיניזם על ≥2 ארכיטקטורות CPU (x86_64,
      aarch64) עם פלטים תואמים.
- [ ] סקירת ערוץ צדדי כוללת הוכחות בזמן קבוע או בדיקות אמפיריות
      עדויות הן לנתיבי חלודה והן ל-FFI.
- [ ] בדיקת תאימות ותיעוד מאשרת התאמות של הנחיות המפעיל
      חובות רגולטוריות.
- [ ] בדיקות דיפרנציאליות מול יישומי התייחסות (RustCrypto,
      OpenSSL/Tongsuo) מבוצע עם פיקוח מבקר.
- [ ] רתמות Fuzz הוערכו; קורפוסי זרעים חדשים מסופקים היכן שקיימים פערים.

# שיקום ויציאה

- [ ] כל הממצאים מסווגים לפי חומרה, השפעה, ניצול ו
      שלבי תיקון מומלצים.
- [ ] בעיות גבוהות/קריטיות מקבלות תיקונים או הקלות עם אישור המבקר
      אימות; סיכונים שיוריים מתועדים.
- [ ] המבקר מספק אימות בדיקה חוזר המעיד על בעיות קבועות (הבדל, בדיקה
      פועל, או אישור חתום).
- [ ] דו"ח סופי נמסר: תקציר מנהלים, ממצאים מפורטים, מתודולוגיה,
      פסק דין דטרמיניזם, פסק דין ציות.
- [ ] פגישת חתימה פנימית מסיימת את השלבים הבאים, שחרור התאמות,
      ועדכוני תיעוד.
- [ ] `status.md` עודכן עם תוצאת הביקורת ותיקונים יוצאים מן הכלל
      מעקבים.
- [ ] נתיחה שלאחר המוות נתפסה ב-`docs/source/crypto/sm_program.md` (שיעורים
      משימות נלמדות, מתקשות עתידיות).

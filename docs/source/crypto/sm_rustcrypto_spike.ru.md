//! Примечания к всплеску интеграции RustCrypto SM.

# Примечания к спайку RustCrypto SM

## Цель
Убедитесь, что введение крейтов RustCrypto `sm2`, `sm3` и `sm4` (плюс `rfc6979`, `ccm`, `gcm`) в качестве дополнительных зависимостей корректно компилируется в `iroha_crypto` создает контейнер и обеспечивает приемлемое время сборки, прежде чем подключать флаг функции в более широкое рабочее пространство.

## Предлагаемая карта зависимостей

| Ящик | Предлагаемая версия | Особенности | Заметки |
|-------|-------------------|----------|-------|
| `sm2` | `0.13` (RustCrypto/подписи) | `std` | Зависит от `elliptic-curve`; убедитесь, что MSRV соответствует рабочему пространству. |
| `sm3` | `0.5.0-rc.1` (RustCrypto/хеши) | по умолчанию | API аналогичен `sha2`, интегрируется с существующими функциями `digest`. |
| `sm4` | `0.5.1` (RustCrypto/блочные шифры) | по умолчанию | Работает с признаками шифра; Обертки AEAD отложены до более позднего всплеска. |
| `rfc6979` | `0.4` | по умолчанию | Повторное использование для детерминированного вывода nonce. |

*Версии отражают текущие выпуски по состоянию на 2024–2012 гг.; перед посадкой подтвердите с помощью `cargo search`.*

## Изменения манифеста (черновик)

```toml
[features]
sm = ["dep:sm2", "dep:sm3", "dep:sm4", "dep:rfc6979"]

[dependencies]
sm2 = { version = "0.13", optional = true, default-features = false, features = ["std"] }
sm3 = { version = "0.5.0-rc.1", optional = true }
sm4 = { version = "0.5.1", optional = true }
rfc6979 = { version = "0.4", optional = true, default-features = false }
```

Последующие действия: закрепите `elliptic-curve` для соответствия уже имеющимся версиям `iroha_crypto` (в настоящее время `0.13.8`).

## Контрольный список спайков
- [x] Добавить дополнительные зависимости и функции в `crates/iroha_crypto/Cargo.toml`.
- [x] Создайте модуль `signature::sm` позади `cfg(feature = "sm")` со структурами-заполнителями для подтверждения подключения.
- [x] Запустите `cargo check -p iroha_crypto --features sm` для подтверждения компиляции; запишите время сборки и количество новых зависимостей (`cargo tree --features sm`).
- [x] Подтвердите стандартную позицию с помощью `cargo check -p iroha_crypto --features sm --locked`; Сборки `no_std` больше не поддерживаются.
- [x] Результаты файла (тайминги, дельта дерева зависимостей) в `docs/source/crypto/sm_program.md`.

## Наблюдения для записи
— Дополнительное время компиляции по сравнению с базовым уровнем.
- Влияние двоичного размера (если измеримо) с помощью `cargo builtinsize`.
- Любые конфликты MSRV или функций (например, с второстепенными версиями `elliptic-curve`).
- Выдаются предупреждения (небезопасный код, ограничение const-fn), которые могут потребовать исправлений восходящего потока.

## Элементы, ожидающие рассмотрения
— Прежде чем раздувать график зависимостей рабочей области, дождитесь одобрения Crypto WG.
- Подтвердите, следует ли продавать ящики для проверки или полагаться на crates.io (могут потребоваться зеркала).
- Координируйте обновление `Cargo.lock` с `sm_lock_refresh_plan.md`, прежде чем отмечать контрольный список как завершенный.
- Используйте `scripts/sm_lock_refresh.sh` после получения одобрения для повторного создания файла блокировки и дерева зависимостей.

# SM Vectors Промежуточные примечания

В этом документе собраны общедоступные тесты с известными ответами, которые заполняют жгуты SM2/SM3/SM4 до того, как будут загружены сценарии автоматического импорта. Машиночитаемые копии живут в:

- `crates/iroha_crypto/tests/fixtures/sm_known_answers.toml` (векторы приложений, случаи RFC 8998, пример приложения 1).
- `fixtures/sm/sm2_fixture.json` (общая детерминированная оснастка SDK, используемая тестами Rust/Python/JavaScript).
- `crates/iroha_crypto/tests/fixtures/wycheproof_sm2.json` — курируемый корпус из 52 случаев (детерминированные фикстуры + синтезированные отрицательные значения переворота битов/сообщений/хвоста), зеркально отображаемый в Apache-2.0, пока находится на рассмотрении вышестоящий пакет SM2. `crates/iroha_crypto/tests/sm2_wycheproof.rs` проверяет эти векторы, используя стандартный верификатор SM2, когда это возможно, и при необходимости возвращается к чистой реализации BigInt домена Приложение.

## Проверка подписи SM2 с помощью OpenSSL/Tongsuo/GmSSL

Пример приложения 1 (Fp-256) использует идентификатор `ALICE123@YAHOO.COM` (ENTLA 0x0090), сообщение `"message digest"` и открытый ключ, показанный ниже. Рабочий процесс копирования и вставки OpenSSL/Tongsuo:

```bash
# 1. Public key (SubjectPublicKeyInfo)
cat > pubkey.pem <<'PEM'
-----BEGIN PUBLIC KEY-----
MFkwEwYHKoZIzj0CAQYIKoEcz1UBgi0DQgAECuTHeYqg8RlHG+4RglvkYgK7eeKlhESV6XwE/03y
VIp8AkD4jxzU4WNSpzwXt/FvBzU+U6F21oSp/gxrt5joVw==
-----END PUBLIC KEY-----
PEM

# 2. Message (no trailing newline)
printf "message digest" > msg.bin

# 3. Signature (DER form)
cat > sig.b64 <<'EOF'
MEQCIEDx7Fn3k9n0ngnc70kTDUGU95+x7tLKpVus20nE51XRAiBvxtrDLF1c8Qx337IPfC62Z6RX
hy+wnsVjJ6Z+x97r5w==
EOF
base64 -d sig.b64 > sig.der

# 4. Verify (expects "Signature Verified Successfully")
openssl pkeyutl -verify -pubin -inkey pubkey.pem \
  -in msg.bin -sigfile sig.der \
  -rawin -digest sm3 \
  -pkeyopt distid:ALICE123@YAHOO.COM
```

* OpenSSL 3.x документирует параметры `distid:`/`hexdistid:`. В некоторых сборках OpenSSL 1.1.1 ручка отображается как `sm2_id:` — используйте тот, который указан в `openssl pkeyutl -help`.
* GmSSL экспортирует ту же поверхность `pkeyutl`; более старые сборки также принимали `-pkeyopt sm2_id:...`.
* LibreSSL (по умолчанию в macOS/OpenBSD) **не** реализует SM2/SM3, поэтому приведенная выше команда здесь не работает. Используйте OpenSSL ≥ 1.1.1, Tongsuo или GmSSL.

Помощник DER выдает `3044022040F1EC59F793D9F49E09DCEF49130D4194F79FB1EED2CAA55BACDB49C4E755D102206FC6DAC32C5D5CF10C77DFB20F7C2EB667A457872FB09EC56327A67EC7DEEBE7`, который соответствует сигнатуре приложения.

В приложении также печатается хэш информации о пользователе и полученный дайджест:

```
ZA = F4A38489E32B45B6F876E3AC2168CA392362DC8F23459C1D1146FC3DBFB7BC9A
e  = SM3(ZA || "message digest")
   = B524F552CD82B8B028476E005C377FB19A87E6FC682D48BB5D42E3D9B9EFFE76
```

Вы можете подтвердить это с помощью OpenSSL:

```bash
echo -n 'F4A38489E32B45B6F876E3AC2168CA392362DC8F23459C1D1146FC3DBFB7BC9A' \
  | xxd -r -p > za.bin
printf "message digest" >> za.bin
openssl dgst -sm3 -binary za.bin | xxd -p -c 256
# -> b524f552cd82b8b028476e005c377fb19a87e6fc682d48bb5d42e3d9b9effe76
```

Проверка работоспособности Python для уравнения кривой:

```python
p = int("FFFFFFFEFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF00000000FFFFFFFFFFFFFFFF", 16)
a = int("FFFFFFFEFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF00000000FFFFFFFFFFFFFFFC", 16)
b = int("28E9FA9E9D9F5E344D5A9E4BCF6509A7F39789F515AB8F92DDBCBD414D940E93", 16)
x = int("0AE4C7798AA0F119471BEE11825BE46202BB79E2A5844495E97C04FF4DF2548A", 16)
y = int("7C0240F88F1CD4E16352A73C17B7F16F07353E53A176D684A9FE0C6BB798E857", 16)
assert (pow(y, 2, p) - (pow(x, 3, p) + a * x + b)) % p == 0
```

- Потребление перекрестного SDK:
  - `fixtures/sm/sm2_fixture.json` теперь предоставляет массив `vectors`. Пакет крипторегрессии Rust (`crates/iroha_crypto/tests/sm2_fixture_vectors.rs`), помощники клиента Rust (`crates/iroha/src/sm.rs`), привязки Python (`python/iroha_python/tests/test_crypto.py`) и JavaScript SDK (`javascript/iroha_js/test/crypto.sm2.fixture.test.js`) анализируют эти приспособления.
  - `crates/iroha/tests/sm_signing.rs` осуществляет детерминированную подпись и проверяет, соответствуют ли выходные данные мультихэша/мультикодека в цепочке устройству.
  - Пакеты регрессии времени приема (`crates/iroha_core/tests/admission_batching.rs`) утверждают, что полезные нагрузки SM2 отклоняются, если `allowed_signing` не включает `sm2` *и* `default_hash` — это `sm3-256`, охватывая сквозные ограничения конфигурации.
- 追加カバレッジ: 異常系(無効な曲線、異常な `r/s`、`distid` 改ざん）は`crates/iroha_crypto/tests/sm2_fuzz.rs` の property テストで網羅済みです。Пример приложения 1 の正規ベクトルは `sm_known_answers.toml` に多言語対応のмультикодек 形式で引き続き提供しています。
- Код Rust теперь предоставляет `Sm2PublicKey::compute_z`, поэтому приспособления ZA можно генерировать программно; см. `sm2_compute_z_matches_annex_example` для регрессии Приложения D.

## Следующие действия
- Отслеживайте регрессии времени допуска (`admission_batching.rs`), чтобы гарантировать, что шлюзование конфигурации продолжает обеспечивать соблюдение границ включения SM2.
- Расширьте охват с помощью случаев Wycheproof SM4 GCM/CCM и получите цели нечеткости на основе свойств для проверки подписи SM2. ✅ (Подмножество недопустимых регистров зафиксировано в `sm3_sm4_vectors.rs`).
- Запрос LLM для альтернативных идентификаторов: * «Укажите подпись SM2 для примера приложения 1, когда отличительный идентификатор установлен на 1234567812345678 вместо ALICE123@YAHOO.COM, и укажите новые значения ZA/e».*

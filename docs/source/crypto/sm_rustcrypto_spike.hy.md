# RustCrypto SM Spike Notes

## Նպատակ
Հաստատեք դա՝ ներկայացնելով RustCrypto-ի `sm2`, `sm3` և `sm4` տուփերը (գումարած `rfc6979`, `ccm`, `rfc6979`, `ccm`00000000005X, `rfc6979`, `ccm`0, I18000000000, `iroha_crypto` վանդակում և տալիս է ընդունելի կառուցման ժամանակներ՝ նախքան հատկանիշի դրոշը ավելի լայն աշխատանքային տարածք միացնելը:

## Առաջարկվող կախվածության քարտեզ

| արկղ | Առաջարկվող տարբերակ | Առանձնահատկություններ | Ծանոթագրություններ |
|-------|------------------|----------|-------|
| `sm2` | `0.13` (RustCrypto/ստորագրություններ) | `std` | Կախված է `elliptic-curve`-ից; ստուգեք MSRV-ի համապատասխանությունը աշխատանքային տարածքին: |
| `sm3` | `0.5.0-rc.1` (RustCrypto/հեշ) | լռելյայն | API-ն զուգահեռում է `sha2`-ին, ինտեգրվում է գոյություն ունեցող `digest` հատկանիշներին: |
| `sm4` | `0.5.1` (RustCrypto/block-ciphers) | լռելյայն | Աշխատում է գաղտնագրային հատկանիշներով; AEAD փաթաթանները հետաձգվել են ավելի ուշ հասկի: |
| `rfc6979` | `0.4` | լռելյայն | Վերօգտագործում դետերմինիստական ​​nonce-ի ածանցման համար: |

*Տարբերակները արտացոլում են ընթացիկ թողարկումները 2024-12 թթ. հաստատեք `cargo search`-ով մինչև վայրէջք կատարելը։*

## Ակնհայտ փոփոխություններ (նախագիծ)

```toml
[features]
sm = ["dep:sm2", "dep:sm3", "dep:sm4", "dep:rfc6979"]

[dependencies]
sm2 = { version = "0.13", optional = true, default-features = false, features = ["std"] }
sm3 = { version = "0.5.0-rc.1", optional = true }
sm4 = { version = "0.5.1", optional = true }
rfc6979 = { version = "0.4", optional = true, default-features = false }
```

Հետևում. ամրացրեք `elliptic-curve`՝ `iroha_crypto`-ի (ներկայումս՝ `0.13.8`) տարբերակներին համապատասխանելու համար:

## Spike Checklist
- [x] Ավելացնել կամընտիր կախվածություններ և առանձնահատկություններ `crates/iroha_crypto/Cargo.toml`-ին:
- [x] Ստեղծեք `signature::sm` մոդուլ `cfg(feature = "sm")`-ի հետևում տեղապահի կառուցվածքներով՝ լարերը հաստատելու համար:
- [x] Գործարկեք `cargo check -p iroha_crypto --features sm`՝ կոմպիլյացիան հաստատելու համար; ռեկորդային կառուցման ժամանակ և նոր կախվածության քանակ (`cargo tree --features sm`):
- [x] Հաստատեք միայն std-ի կեցվածքը `cargo check -p iroha_crypto --features sm --locked`-ով; `no_std` կառուցվածքներն այլևս չեն աջակցվում:
- [x] Ֆայլի արդյունքներ (ժամկետներ, կախվածության ծառի դելտա) `docs/source/crypto/sm_program.md`-ում:

## Դիտարկումներ գրավելու համար
- Լրացուցիչ հավաքագրման ժամանակը ընդդեմ բազային:
- Երկուական չափի ազդեցություն (եթե չափելի է) `cargo builtinsize`-ով:
- Ցանկացած MSRV կամ հատկանիշի հակասություններ (օրինակ՝ `elliptic-curve` փոքր տարբերակներով):
- Ուղարկված նախազգուշացումներ (անապահով կոդ, const-fn gating), որոնք կարող են պահանջել հոսանքով վերևի հատվածներ:

## Սպասվող իրեր
- Սպասեք Crypto WG-ի հաստատմանը նախքան աշխատանքային տարածքի կախվածության գրաֆիկը փչելը:
- Հաստատեք՝ արդյոք վաճառել արկղերը վերանայման համար, թե ապավինել crates.io-ին (հայելիներ կարող են պահանջվել):
- Համակարգեք `Cargo.lock` թարմացումը մեկ `sm_lock_refresh_plan.md`-ի համար՝ նախքան ստուգաթերթը ավարտված նշելը:
- Օգտագործեք `scripts/sm_lock_refresh.sh`-ը, երբ թույլտվություն տրվի՝ կողպեքի ֆայլը և կախվածության ծառը վերականգնելու համար:

## 2025-01-19 Հասկի մատյան
- Ավելացվել են կամընտիր կախվածություններ (`sm2 0.13`, `sm3 0.5.0-rc.1`, `sm4 0.5.1`, `rfc6979 0.4`) և `sm` հատկանիշի դրոշակ `iroha_crypto`-ում:
- Stubbed `signature::sm` մոդուլ՝ հավաքման ընթացքում հեշինգ/արգելափակել գաղտնագրման API-ները:
- `cargo check -p iroha_crypto --features sm --locked`-ն այժմ լուծում է կախվածության գրաֆիկը, սակայն ընդհատվում է `Cargo.lock` թարմացման պահանջով. պահեստի քաղաքականությունը արգելում է lockfile խմբագրումները, ուստի կոմպիլյացիայի գործարկումը մնում է առկախ, մինչև մենք համաձայնեցնենք կողպեքի թույլատրված թարմացումը:## 2026-02-12 Հասկերի մատյան
- Լուծվել է կողպեքի ֆայլերի նախորդ արգելափակիչը. կախվածությունն արդեն ֆիքսված է, ուստի `cargo check -p iroha_crypto --features sm --locked`-ը հաջողության է հասնում (սառը կառուցում 7.9 վրկ Dev Mac-ում, աստիճանաբար վերագործարկվում է 0.23 վրկ):
- `cargo check -p iroha_crypto --no-default-features --features "std sm" --locked`-ն անցնում է 1.0 վրկ-ով՝ հաստատելով, որ կամընտիր հատկանիշը հավաքվում է միայն `std` կոնֆիգուրացիաներում (`no_std` ուղի չի մնում):
- Կախվածության դելտան միացված `sm` հատկանիշով ներկայացնում է 11 տուփ՝ `base64ct`, `ghash`, `opaque-debug`, `pem-rfc7468`, I180000050 `polyval`, `primeorder`, `sm2`, `sm3`, `sm4` և `sm4-gcm`: (`rfc6979`-ն արդեն բազային գրաֆիկի մի մասն էր:)
- Կառուցման նախազգուշացումները պահպանվում են չօգտագործված NEON քաղաքականության օգնականների համար. թողնել այնպես, ինչպես կա, մինչև հաշվառման հարթեցման գործարկման ժամանակը նորից միացնի այդ կոդերի ուղիները:

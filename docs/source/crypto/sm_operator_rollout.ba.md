## 2. Конфигурация үҙгәрештәре
1. SM2 асҡыс инвентаризацияһы һәм әҙер өҙөк генерациялау өсөн xtask ярҙамсыһы йүгерергә:
   ```bash
   cargo xtask sm-operator-snippet \
     --distid CN12345678901234 \
     --json-out sm2-key.json \
     --snippet-out client-sm2.toml
   ```
   Ҡулланыу `--snippet-out -` (һәм теләк буйынса `--json-out -`) ағым өсөн сығыштар stdout ҡасан һеҙ уларҙы тикшерергә кәрәк.
   Әгәр һеҙ өҫтөнлөк бирәһегеҙ, драйвер түбән кимәлдәге CLI командалары ҡул менән, эквивалентлы ағым:
   ```bash
   cargo run -p iroha_cli --features sm -- \
     crypto sm2 keygen \
     --distid CN12345678901234 \
     --output sm2-key.json

   cargo run -p iroha_cli --features sm -- \
     crypto sm2 export \
     --private-key-hex "$(jq -r .private_key_hex sm2-key.json)" \
     --distid CN12345678901234 \
     --snippet-output client-sm2.toml \
     --emit-json --quiet
   ```
   Әгәр `jq` асыҡ `sm2-key.json` асыҡ, `private_key_hex` ҡиммәтен күсереп, һәм уны туранан-тура экспорт командаһына тапшыра.
2. Өҫтәү һөҙөмтәһендә өҙөк һәр төйөн’конфигурация (ҡиммәттәр өсөн күрһәтелгән өсөн
   тик этапта раҫлау; мөхиткә көйләү һәм асҡыстарҙы күрһәтелгәнсә сорттарға бүлергә):
```toml
[crypto]
default_hash = "sm3-256"
allowed_signing = ["ed25519", "sm2"]   # remove "sm2" to stay in verify-only mode
sm2_distid_default = "1234567812345678"
# enable_sm_openssl_preview = true  # optional: only when deploying the OpenSSL/Tongsuo path
```
3. Төйөн яңынан башлау һәм раҫлау `crypto.sm_helpers_available` һәм (әгәр һеҙ алдан ҡарау бэкэнд мөмкинлек биргән) `crypto.sm_openssl_preview_enabled` өҫтө көтөлгәнсә:
   - `/status` JSON (`"crypto":{"sm_helpers_available":true,"sm_openssl_preview_enabled":true,...}`).
   - Һәр төйөн өсөн күрһәтелгән `config.toml`.
4. Яңыртыу өсөн манифес/генез яҙмалары өҫтәү өсөн SM алгоритмдар рөхсәт-исемлеге, әгәр
   ҡул ҡуйыу һуңыраҡ таратыуҙа эшләй. `--genesis-manifest-json` ҡулланғанда
   алдан ҡул ҡуйылған генез блогыһыҙ, `irohad` хәҙер орлоҡтар эшләү ваҡыты крипто
   снимок туранан-тура манифест’s `crypto` блок-тамаш
   тикшерелгән һеҙҙең үҙгәрештәр планы алға тәгәрмәс алдынан.## 3. Телеметрия һәм мониторинг
- Prometheus ос нөктәләре һәм түбәндәге счетчиктар/канондар барлыҡҡа килгәнен тәьмин итеү:
  - `iroha_sm_syscall_total{kind="verify"}`
  - `iroha_sm_syscall_total{kind="hash"}`
  - `iroha_sm_syscall_total{kind="seal|open",mode="gcm|ccm"}`
  - `iroha_sm_openssl_preview` (0/1 датчик отчет алдан ҡарау toggle дәүләт)
  - `iroha_sm_syscall_failures_total{kind="verify|hash|seal|open",reason="..."}`
- SM2 ҡул ҡуйыу мөмкинлеге булғандан һуң ҡармаҡҡа ҡул ҡуйыу юлы; өсөн счетчиктар өҫтәргә
  `iroha_sm_sign_total` һәм `iroha_sm_sign_failures_total`.
- Grafana өсөн приборҙар таҡталары/иҫкәртмәләр булдырыу өсөн:
  - Һөжүмдәрҙә шпиктар (тәҙрә 5м).
  - Ҡапыл СМ syscall үткәреүсәнлегенең төшөүе.
  - Төйөндәр араһындағы айырмалар (мәҫәлән, тап килмәгән мөмкинлек).

### Әҙерлек тикшерелгән
- **Тик әҙерлек өсөн генә (SM-R1).** Йыйып сығарыу Энг, Крипто WG, Ops, һәм Юридик. Кәрәк:
  - `status.md` иҫкәрмәләр үтәү статусы + OpenSSL провенанс.
  - `docs/source/crypto/sm_program.md` / `sm_compliance_brief.md` / был тикшерелгән исемлек һуңғы релиз тәҙрәһе сиктәрендә яңыртылған.
  - `defaults/genesis` йәки тирә-яҡ мөхиткә хас манифест `crypto.allowed_signing = ["ed25519","sm2"]` һәм `crypto.default_hash = "sm3-256"` (йәки `sm2`-һыҙ раҫлаусы ғына вариантты күрһәтһә, әгәр ҙә беренсе этапта) күрһәтә.
  - `scripts/sm_openssl_smoke.sh` + `scripts/sm_interop_matrix.sh` журналдарына билетҡа беркетелгән журналдар.
  - Телеметрия приборҙар таҡтаһы (`iroha_sm_*`) стационар тәртип өсөн тикшерелгән.
- **Пиловый әҙерлек (SM-R2).** Өҫтәмә ҡапҡалар:
  - RustCrypto SM стека өсөн аудит отчеты ябылған йәки RFC өсөн компенсациялау контроль ҡул ҡуйған хәүефһеҙлек.
  - Оператор runbooks (объектив-конкрет) яңыртыу менән ҡул ҡуйыу fallback/кире кире аҙымдар.
  - Genesis манифестары өсөн пилот когортаһы `allowed_signing = ["ed25519","sm2"]` инә һәм рөхсәт-исемлеге һәр төйөн конфигурацияһында көҙгөләнә.
  - Сығыу/рекрог планы документлаштырылған (Switch `allowed_signing` Ed25519-ға кире, тергеҙелә манифест, сброс таҡталары).
- **GA әҙерлеге (SM-R3).** ыңғай пилот отчеты талап итә, бөтә валидатор юрисдикциялары өсөн яңыртылған үтәлеше тураһында подача, ҡул ҡуйылған телеметрия база линиялары, һәм билет раҫлауын сығарыу сығарыу Энг + Крипто WG + Ops/Юридик триад.## 5. Упаковка & Ҡабул итеү тикшерелгән исемлеге
- **Бөтәһе OpenSSL/Тонссуо артефакттар.** Судно OpenSSL/Тонссуо 3.0+ уртаҡ китапханалар (```bash
   cargo run -p iroha_cli --features sm -- \
     crypto sm2 keygen \
     --distid CN12345678901234 \
     --output sm2-key.json

   cargo run -p iroha_cli --features sm -- \
     crypto sm2 export \
     --private-key-hex "$(jq -r .private_key_hex sm2-key.json)" \
     --distid CN12345678901234 \
     --snippet-output client-sm2.toml \
     --emit-json --quiet
   ```) һәр валидатор пакеты йәки теүәл системаға бәйлелек документы менән. Яҙма версия, флагтар төҙөү, һәм SHA256 тикшерелгән суммалар сығарыу манифест шулай аудиторҙар эҙләү мөмкинлеге тәьмин итеүсе төҙөү.
- **CI ваҡытында тикшерергә.** Өҫтәү CI аҙым, тип башҡарыла `scripts/sm_openssl_smoke.sh` ҡаршы ҡапланған артефакттар һәр маҡсатлы платформа. Эш уңышһыҙлыҡҡа осрарға тейеш, әгәр алдан ҡарау флагы өҫтөндә эшләй, әммә провайдер инициализациялау мөмкин түгел (башланғыс, ярҙамһыҙ алгоритм һ.б.).
- **Башҡа үтәү иҫкәрмәләр.** Яңыртыу иҫкәрмәләр / <!--
  SPDX-License-Identifier: Apache-2.0
--> менән провайдер версияһы, экспорт-контроль һылтанмалар (GM/T, GB/T), һәм теләһә ниндәй юрисдикция-специфик файлдар өсөн кәрәкле SM алгоритмдары.
- **Оператор runbook яңыртыуҙары.** Яңыртыу ағымын документлаштырыу: яңы уртаҡ объекттарҙы сәхнәләштереү, `crypto.enable_sm_openssl_preview = true` менән тиҫтерҙәре менән яңынан башлағыҙ, `/status` яланын һәм `sm`-се габаритын `true`-ға ҡағып, ҡабыҙыу планын һаҡлағыҙ (флип флагы йәки пакетҡа ҡайта) әгәр алдан ҡарау телеметрияһы автопарк буйынса тайпыла.
- **Дәлилдәр һаҡлау.** Архив төҙөү журналдары һәм ҡул ҡуйыу өсөн аттестациялар OpenSSL/Тонссуо пакеттар менән бер рәттән валитатор сығарыу артефакттар, шулай итеп, киләсәктә аудиттарҙы ҡабатлай ала провенанс сылбыр.

## 6. Инцидент яуап
- **Тегеү етешһеҙлеге шпиц:** СМ ярҙамһыҙ йәки `sm2`-ты алып ташлауһыҙ төҙөүгә йәйелдерелгән.
  `allowed_signing` (ҡайтып `default_hash` кәрәк булғанда) һәм уңышһыҙлыҡҡа осраған
  тикшергәндә сығарыу. «Кап өсөн» файҙалы йөктәрҙе, сағыштырмаса хештар һәм төйөн журналдары уңышһыҙлыҡҡа осраған.
- **Һөҙөмтәлелек регрессиялары:** SM метрикаларын Ed25519/SHA2 база һыҙыҡтары менән сағыштырырға.
  Әгәр АРМ эске юл дивергенция сәбәпсе булһа, `crypto.sm_intrinsics = "force-disable"` .
  (функция переключатель көтөп тормошҡа ашырыу) һәм отчет табыштар.
- **Телеметрия бушлыҡтары:** Әгәр ҙә иҫәп-хисаптар юҡ йәки яңыртылмаһа, мәсьәләне файл
  ҡаршы сығарыу инженерияһы; арауыҡҡа тиклем киңерәк йәйелдереү менән бармағыҙ
  хәл ителә.

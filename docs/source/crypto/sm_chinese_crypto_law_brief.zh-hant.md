# 提示

> 您是法學碩士，擔任 Hyperledger Iroha 加密和平台團隊的合規分析師。  
> 背景：  
> - Hyperledger Iroha 是一個基於 Rust 的許可區塊鏈，現在支持中文 GM/T SM2（簽名）、SM3（哈希）和 SM4（分組密碼）原語。  
> - 中國大陸的運營商必須遵守《中華人民共和國密碼法》（2019 年）、分級保護計劃 (MLPS 2.0)、國家密碼管理局 (SCA) 備案規則以及商務部 (MOFCOM) 和海關總署監管的進出口管制。  
> - Iroha 在全球範圍內分發開源軟件。一些運營商將在國內編譯支持 SM 的二進製文件，而其他運營商可能會導入預先構建的工件。  
> 要求的分析：總結在開源區塊鏈軟件中提供 SM2/SM3/SM4 支持所引發的關鍵法律義務，包括： (a) 商業與核心/通用密碼學類別下的分類； (b) 實施國家商業密碼的軟件的備案/批准要求； (c) 二進製文件和源代碼的出口管制； (d) 等保2.0下網絡運營商的運營義務（密鑰管理、日誌記錄、事件響應）。概述 Iroha 項目（文檔、清單、合規聲明）以及在中國境內部署支持 SM 的節點的運營商的具體行動項目。

# 執行摘要

- **分類：** SM2/SM3/SM4 實現屬於“國家商業密碼學”（商業密碼），而不是“核心”或“通用”密碼學，因為它們是批准用於民用/商業用途的公開算法。允許開源分發，但在中國提供的商業產品或服務中使用時需要進行備案。
- **項目義務：** 提供算法出處、確定性構建說明以及指出二進製文件實施國家商業密碼學的合規性聲明。維護標記 SM 功能的 Norito 清單，以便下游集成商可以完成備案。
- **運營商義務：** 中國運營商必須使用 SM 算法向省級 SCA 局備案產品/服務，完成等保 2.0 註冊（金融網絡可能為 3 級），部署經批准的密鑰管理和日誌控制，並確保出口/進口申報符合商務部的豁免目錄。

# 主要義務

## 1.產品和服務備案（國家密碼管理局）

- **提交人：** 在中國提供產品/服務的實體（例如運營商、SaaS 提供商）。開源維護者不需要提交文件，但打包指南必須支持下游提交。
- **可交付成果：** 算法描述、安全設計文檔、測試證據、供應鏈來源和聯繫方式。
- **Iroha 行動：** 發布“SM 加密聲明”，包括算法覆蓋範圍、確定性構建步驟、依賴項哈希以及安全查詢聯繫方式。

## 2. 認證與測試

- 某些行業（金融、電信、關鍵基礎設施）可能需要認可的實驗室測試或認證（例如 CC 級/OSCCA 認證）。
- 包括回歸測試工件，證明符合 GM/T 規範。

## 3. 等保2.0操作控制

運營商必須：1. **向公安局登記區塊鏈系統**，包括密碼學使用摘要。
2. **實施密鑰管理政策**：符合SM2/SM4要求的密鑰生成、分發、輪換、銷毀；記錄關鍵生命週期事件。
3. **啟用安全審計**：捕獲SM啟用的事務日誌、密碼操作事件和異常檢測；保留日誌≥6個月。
4. **事件響應：** 維護記錄在案的響應計劃，其中包括密碼洩露程序和報告時間表。
5. **供應商管理：**確保上游軟件提供商（Iroha項目）能夠提供漏洞通知和補丁。

## 4. 導入/導出注意事項

- **開源源代碼：** 通常在公共域例外情況下可以豁免，但維護人員應在跟踪訪問日誌並包含引用國家商業密碼學的許可證/免責聲明的服務器上託管下載。
- **預構建的二進製文件：** 將支持 SM 的二進製文件運入/運出中國的出口商應確認該項目是否屬於《商業密碼出口管制目錄》的涵蓋範圍。對於沒有專用硬件的通用軟件，簡單的雙重用途聲明就足夠了；維護者不應分發來自具有更嚴格控制的司法管轄區的二進製文件，除非當地法律顧問批准。
- **運營商進口：** 將二進製文件帶入中國的實體必須聲明加密技術的使用。提供哈希清單和 SBOM 以簡化海關檢查。

# 建議的項目行動

1. **文檔**
   - 在 `docs/source/crypto/sm_program.md` 中添加合規性附錄，註明商業密碼狀態、提交期望和聯繫點。
   - 發布 Norito 清單字段（`crypto.sm.enabled=true`、`crypto.sm.approval=l0|l1`），供運營商在準備備案時使用。
   - 確保 Torii `/v1/node/capabilities` 廣告（以及 `iroha runtime capabilities` CLI 別名）隨每個版本一起提供，以便操作員可以捕獲 `crypto.sm` 清單快照以獲取 MLPS/密評證據。
   - 提供雙語（EN/ZH）合規快速入門總結義務。
2. **釋放文物**
   - 為支持 SM 的構建提供 SBOM/CycloneDX 文件。
   - 包括確定性構建腳本和可重現的 Dockerfile。
3. **支持運營商備案**
   - 提供證明算法合規性的模板信件（例如 GM/T 參考文獻、測試覆蓋率）。
   - 維護安全建議郵件列表以滿足供應商通知要求。
4. **內部治理**
   - 跟踪發布清單中的 SM 合規性檢查點（審核完成、文檔更新、清單字段到位）。

# 操作員行動項目（中國）1. 確定部署是否構成“商業密碼產品/服務”（大多數企業網絡都是如此）。
2.向省級SCA局備案產品/服務；附上 Iroha 合規聲明、SBOM、測試報告。
3. 在等保2.0下註冊區塊鏈系統，目標為三級控制；將Iroha日誌集成到安全監控中。
4. 建立 SM 密鑰生命週期程序（根據需要使用經批准的 KMS/HSM）。
5. 將密碼洩露場景納入事件響應演習中；設置與 Iroha 維護人員的升級聯繫。
6. 對於跨境數據流動，如果導出個人數據，請確認額外的 CAC（網絡空間管理局）備案。

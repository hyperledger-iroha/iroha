# SM2/SM3/SM4 合规性和出口简介

本简介补充了 `docs/source/crypto/sm_program.md` 中的架构说明
并为工程、运营和法律团队提供可行的指导
GM/T 算法系列从仅验证预览转向更广泛的支持。

## 总结
- **监管依据：** 中国的《密码法》（2019 年）、《网络安全法》和
  *《数据安全法》*在部署时将 SM2/SM3/SM4 归类为“商业密码”
  陆上。运营商必须提交使用报告，某些行业需要经过认可
  生产使用前进行测试。
- **国际控制：** 中国以外的算法属于美国 EAR 类别
  5 EU 2021/821 附件 1 (5D002) 第 2 部分以及类似的国家制度。开源
  出版物通常符合许可例外 (ENC/TSU) 的资格，但二进制文件
  运往禁运地区的出口仍受管制。
- **项目政策：** SM 功能默认保持禁用状态。签名功能
  仅在外部审计结束、确定性性能/遥测后启用
  门控和操作员文档（简述）着陆。

   源可用性声明、依赖项列表和确定性构建步骤
   发布前经省密码管理局备案。
2. **销售/使用备案（销售/使用备案）：**运行支持 SM 的节点的运营商必须
   注册使用范围、密钥管理和遥测收集
   权威。提供联系信息和事件响应 SLA​​。
3. **认证（检测/认证）：** 关键基础设施运营商可能需要
   认可的测试。提供可重现的构建脚本、SBOM 和测试报告
   下游集成商无需修改代码即可完成认证。
4. **记录保存：** 在合规跟踪器中存档备案和批准。
   当新区域或运营商完成该流程时更新 `status.md`。

## 合规检查表

### 启用 SM 功能之前
- [ ] 确认法律顾问审查了目标部署区域。
- [ ] 捕获确定性构建指令、依赖项清单和 SBOM
      出口以纳入备案。
- [ ] 对齐 `crypto.allowed_signing`、`crypto.default_hash` 并入场
      策略随推出票证一起体现。
- [ ] 生成描述 SM 功能范围的操作员通信，
      启用先决条件和禁用的后备计划。
- [ ] 导出涵盖 SM 验证/签名计数器的遥测仪表板，
      错误率和性能指标（`sm3`、`sm4`、系统调用计时）。
- [ ] 准备陆上事件响应联系人和升级路径
      运营商和加密货币工作组。

### 备案和审核准备情况
- [ ] 选择适当的归档模板（产品与销售/使用）并填写
      在提交之前的发布元数据中。
- [ ] 附加 SBOM 档案、确定性测试记录和清单哈希值。
- [ ] 确保出口管制声明反映了准确的文物
      交付并引用所依赖的许可例外 (ENC/TSU)。
- [ ] 验证审核报告、补救跟踪和操作员操作手册
      从归档包链接。
- [ ] 在合规性中存储已签署的文件、批准和信件
      带有版本化参考的跟踪器。

### 审批后操作
- [ ] 提交申请被接受后，更新 `status.md` 和部署票。
- [ ] 重新运行遥测验证以确认可观测性覆盖范围匹配
      归档输入。
- [ ] 安排定期审查（至少每年一次）备案、审计报告、
      并导出语句以捕获规范/法规更新。
- [ ] 每当配置、功能范围或托管时触发归档附录
      足迹发生重大变化。## 出口和分销指南
- 在发行说明/清单中包含简短的出口声明，引用可靠性
  在 ENC/TSU 上。示例：
  > “此版本包含 SM2/SM3/SM4 实现。分发遵循 ENC
  > (15 CFR Part 742) / EU 2021/821 附件 1 5D002。运营商必须确保合规
  > 遵守当地进出口法律。”
- 对于在中国境内托管的构建，请与运营部门协调以发布来自中国的工件
  陆上基础设施；避免跨境传输支持 SM 的二进制文件，除非
  适当的许可证已到位。
- 镜像到包存储库时，记录哪些工件包含 SM 功能
  简化合规报告。

## 操作员清单
- [ ] 确认发布配置文件 (`scripts/select_release_profile.py`) + SM 功能标志。
- [ ] 回顾 `sm_program.md` 和本简介；确保记录合法备案。
- [ ] 通过使用 `sm` 进行编译、更新 `crypto.allowed_signing` 以包含 `sm2` 以及仅在确定性保护到位且审核状态为绿色后将 `crypto.default_hash` 切换到 `sm3-256` 来启用 SM 功能。
- [ ] 更新遥测仪表板/警报以包括 SM 计数器（验证失败、
      签名请求、性能指标）。

      推出票。

- [`sm_export_statement_template.md`](attachments/sm_export_statement_template.md) —
  适用于发行说明、清单或法律的出口管制声明
  依赖 ENC/TSU 许可证例外的通信。## 标准和引用
- **GM/T 0002-2012 / GB/T 32907-2016** — SM4分组密码和AEAD参数（ECB/GCM/CCM）。匹配 `docs/source/crypto/sm_vectors.md` 中捕获的向量。
- **GM/T 0003-2012 / GB/T 32918.x-2016** — SM2 公钥加密、曲线参数、签名/验证过程和附录 D 已知答案测试。
- **GM/T 0004-2012 / GB/T 32905-2016** — SM3 哈希函数规范和一致性向量。
- **RFC 8998** — SM2 密钥交换和 TLS 中的签名使用；在记录与 OpenSSL/Tongsuo 的互操作时引用。
- **《中华人民共和国密码法》（2019 年）**、**《网络安全法》（2017 年）**、**《数据安全法》（2021 年）** — 上述备案工作流程的法律依据。
- **美国 EAR 类别 5 第 2 部分**和 **欧盟法规 2021/821 附件 1 (5D002)** — 管理支持 SM 的二进制文件的出口管制制度。
- **Iroha 工件：** `scripts/sm_interop_matrix.sh` 和 `scripts/sm_openssl_smoke.sh` 提供确定性互操作记录，审核员可以在签署合规性报告之前重播这些记录。

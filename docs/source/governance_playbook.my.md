# အုပ်ချုပ်မှုစာအုပ်

ဤပြခန်းစာအုပ်သည် Sora ကွန်ရက်ကို ထိန်းသိမ်းထားသည့် နေ့စဉ်ရိုးရာဓလေ့များကို ဖမ်းယူထားသည်။
အုပ်ချုပ်ရေးကောင်စီနှင့် ညှိနှိုင်းထားသည်။ ၎င်းမှ တရားဝင်ကိုးကားချက်များကို စုစည်းထားသည်။
အော်ပရေတာများ အမြဲရှိစဉ်တွင် တစ်ဦးချင်း အခမ်းအနားများသည် တိုတိုတုတ်တုတ် ရှိနေနိုင်သောကြောင့် သိုလှောင်ရုံ
ပိုမိုကျယ်ပြန့်သော လုပ်ငန်းစဉ်အတွက် ဝင်ခွင့်အမှတ်တစ်ခု ရှိသည်။

## ကောင်စီအခမ်းအနား

- **Fixture governance** – [Sora Parliament Fixture Approval](sorafs/signing_ceremony.md) ကိုကြည့်ပါ
  ယခုအခါ လွှတ်တော်၏ အခြေခံအဆောက်အအုံဆိုင်ရာ ကော်မတီ၏ ကွင်းဆက်အတည်ပြုချက် စီးဆင်းမှုအတွက်
  SoraFS chunker အပ်ဒိတ်များကို ပြန်လည်သုံးသပ်သောအခါ အောက်ပါအတိုင်း။
- ** မဲစာရင်းထုတ်ပြန်ခြင်း** – ကိုးကားပါ။
  အဆင့်ဆင့် CLI အတွက် [Governance Vote Tally](governance_vote_tally.md)
  အလုပ်အသွားအလာနှင့် အစီရင်ခံခြင်းပုံစံ။

## လုပ်ငန်းလည်ပတ်မှုဆိုင်ရာစာအုပ်များ

- **API ပေါင်းစပ်မှုများ** – [Governance API ရည်ညွှန်းချက်](governance_api.md) သည် စာရင်းပြုစုထားသည်။
  စစ်မှန်ကြောင်းအတည်ပြုခြင်းအပါအဝင် ကောင်စီဝန်ဆောင်မှုများက ဖော်ထုတ်ထားသည့် REST/gRPC မျက်နှာပြင်များ
  လိုအပ်ချက်များနှင့် pagination စည်းမျဉ်းများ။
- **Telemetry dashboards** – အောက်တွင် Grafana JSON အဓိပ္ပါယ်ဖွင့်ဆိုချက်များ
  `docs/source/grafana_*` သည် “အုပ်ချုပ်မှုကန့်သတ်ချက်များ” နှင့် “အချိန်ဇယားဆွဲသူကို သတ်မှတ်သည်
  TEU” ဘုတ်များ။ လိုက်လျောညီထွေရှိနေစေရန် ထုတ်ဝေမှုတစ်ခုစီပြီးနောက် JSON ကို Grafana သို့ ထုတ်ယူပါ။
  canonical layout နှင့်အတူ။

## ဒေတာရရှိနိုင်မှုကြီးကြပ်ရေး

### Retention သင်တန်းများ

DA manifests များကို အတည်ပြုသည့် လွှတ်တော်အကန့်များသည် ပြဌာန်းထားသည့် ထိန်းသိမ်းမှုကို ကိုးကားရမည်ဖြစ်သည်။
မဲပေးခြင်းမပြုမီမူဝါဒ။ အောက်ဖော်ပြပါဇယားသည် တစ်ဆင့်ခံပြဋ္ဌာန်းထားသည့် ပုံသေများကို ရောင်ပြန်ဟပ်သည်။
`torii.da_ingest.replication_policy` သို့မှသာ သုံးသပ်သူများသည် မကိုက်ညီမှုများကို တွေ့ရှိနိုင်ပါသည်။
အရင်းအမြစ် TOML ကိုရှာဖွေခြင်း။【docs/source/da/replication_policy.md:1】

| အုပ်ချုပ်မှုဂ် | Blob အတန်း | စွဲမြဲပူ | စွဲမြဲအေး | လိုအပ်သော ပုံတူများ | ထိန်တန်း |
|----------------|-----------------|----------------|--------------------------------------------------------------------------------|
| `da.taikai.live` | `taikai_segment` | 24 နာရီ | 14d | 5 | `hot` |
| `da.sidecar` | `nexus_lane_sidecar` | 6h | 7d | 4 | `warm` |
| `da.governance` | `governance_artifact` | 12h | 180d | 3 | `cold` |
| `da.default` | _အခြားအတန်းများ_ | 6h | 30d | 3 | `warm` |

Infrastructure Panel သည် ဖြည့်စွက်ထားသော ပုံစံခွက်မှ ပူးတွဲပါရှိသင့်သည်။
မဲတိုင်းအတွက် `docs/examples/da_manifest_review_template.md` ထို့ကြောင့် ထင်ရှားသည်။
digest၊ retention tag နှင့် Norito artefacts များသည် အုပ်ချုပ်ရေးတွင် ဆက်နွယ်နေပါသည်
မှတ်တမ်း။

### မန်နီးဖက်စ် စာရင်းစစ်လမ်းကြောင်းကို လက်မှတ်ရေးထိုးခဲ့သည်။

မဲပြားတစ်ခု အစီအစဉ်မပြီးမီ၊ ကောင်စီဝန်ထမ်းများသည် ထင်ရှားကြောင်း သက်သေပြရမည်ဖြစ်သည်။
ပြန်လည်သုံးသပ်ထားသည့် bytes သည် လွှတ်တော်စာအိတ်နှင့် SoraFS နှင့် ကိုက်ညီပါသည်။ သုံးပါ။
ထိုအထောက်အထားများကို စုဆောင်းရန် ရှိပြီးသားကိရိယာ1. Torii (`iroha app da get-blob --storage-ticket <hex>` မှ manifest အတွဲကို ရယူပါ
   သို့မဟုတ် တူညီသော SDK အကူအညီပေးသူ) ထို့ကြောင့် လူတိုင်းရောက်ရှိသည့် တူညီသောဘိုက်များကို ဟက်ခ်လုပ်ပါ။
   တံခါးပေါက်များ။
2. လက်မှတ်ထိုးထားသော စာအိတ်ဖြင့် manifest stub verifier ကို ဖွင့်ပါ-
   ```
   cargo run -p sorafs_car --bin sorafs-manifest-stub -- manifest.json \
     --manifest-signatures-in=fixtures/sorafs_chunker/manifest_signatures.json \
     --json-out=/tmp/manifest_report.json
   ```
   ၎င်းသည် BLAKE3 manifest digest ကို ပြန်လည်တွက်ချက်ပြီး ၎င်းကို validate လုပ်သည်။
   `chunk_digest_sha3_256` နှင့် ထည့်သွင်းထားသော Ed25519 လက်မှတ်တိုင်းကို စစ်ဆေးသည်
   `manifest_signatures.json`။ `docs/source/sorafs/manifest_pipeline.md` ကိုကြည့်ပါ။
   နောက်ထပ် CLI ရွေးချယ်စရာများအတွက်။
3. အချေအတင်၊ `chunk_digest_sha3_256`၊ ပရိုဖိုင်လက်ကိုင်နှင့် လက်မှတ်ထိုးသူစာရင်းထဲသို့ ကူးယူပါ။
   ပြန်လည်သုံးသပ်ခြင်းပုံစံ။ မှတ်ချက်- အတည်ပြုသူသည် “ပရိုဖိုင်မတူညီမှု” ကို သတင်းပို့ပါက သို့မဟုတ် a
   လက်မှတ်ပျောက်နေ၊ မဲကိုရပ်ပြီး ပြင်ထားသောစာအိတ်ကို တောင်းဆိုပါ။
4. Verifier output (သို့မဟုတ် CI artefact မှ သိမ်းဆည်းပါ။
   `ci/check_sorafs_fixtures.sh`) Norito `.to` payload တွဲပြီး စာရင်းစစ်များ၊
   အတွင်းဝင်ပေါက်များကို မဝင်ရောက်ဘဲ အထောက်အထားများကို ပြန်လည်ပြသနိုင်သည်။

ထွက်ပေါ်လာသော စာရင်းစစ်အစုကို လွှတ်တော်က ဟက်ကာနှင့် လက်မှတ်တိုင်းကို ပြန်လည်ဖန်တီးခွင့်ပေးသင့်သည်။
မန်နီးဖက်စ်ကို ပူသောသိုလှောင်မှုမှ လှည့်ပတ်ပြီးနောက်ပင် စစ်ဆေးပါ။

### စစ်ဆေးရန်စာရင်း

1. ပါလီမန်မှ အတည်ပြုထားသော မန်နီးဖက်စ်စာအိတ်ကို ဆွဲယူပါ (ကြည့်ပါ။
   `docs/source/sorafs/signing_ceremony.md`) နှင့် BLAKE3 အချေအတင်ကို မှတ်တမ်းတင်ပါ။
2. မန်နီးဖက်စ်၏ `RetentionPolicy` ဘလောက်သည် ဇယားရှိ tag နှင့် ကိုက်ညီကြောင်း အတည်ပြုပါ
   အထက်၊ Torii သည် မကိုက်ညီမှုများကို ပယ်ချမည်ဖြစ်သော်လည်း ကောင်စီက ၎င်းကို ဖမ်းဆုပ်ရမည်၊
   စာရင်းစစ်များအတွက် အထောက်အထားများ။【docs/source/da/replication_policy.md:32】
3. တင်သွင်းထားသော Norito သည် တူညီသောထိန်းသိမ်းမှုတဂ်ကို ရည်ညွှန်းကြောင်း အတည်ပြုပါ
   နှင့် intake ticket တွင်ပေါ်လာသော blob အတန်း။
4. မူဝါဒစစ်ဆေးချက်၏ အထောက်အထား ပူးတွဲပါ (CLI ထုတ်ပေးမှု၊ `torii.da_ingest.replication_policy`
   အမှိုက်ပုံ သို့မဟုတ် CI ပစ္စည်းများ) SRE သည် ဆုံးဖြတ်ချက်ကို ပြန်လည်ပြသနိုင်စေရန် ပြန်လည်သုံးသပ်ချက်ထုပ်သို့ ပေးပို့ပါ။
5. အဆိုပြုချက်အပေါ် မူတည်သည့်အခါ စီစဉ်ထားသော ထောက်ပံ့ကြေးကို နှိပ်ပါ သို့မဟုတ် ငှားရမ်းမှု ချိန်ညှိမှုများကို မှတ်တမ်းတင်ပါ။
   `docs/source/sorafs_reserve_rent_plan.md`။

## Moderation & Escalation

ဂိတ်ဝေးဖယ်ရှားခြင်းများ၊ ထောက်ပံ့ကြေးပေးချေမှုများ သို့မဟုတ် DA သည် လိုက်နာမှုအား လိုက်နာပါသည်။
`docs/source/sorafs_gateway_compliance_plan.md` တွင် ဖော်ပြထားသော ပိုက်လိုင်း နှင့်
`docs/source/sorafs_moderation_panel_plan.md` တွင် အယူခံဝင်သည့်ကိရိယာ။ အကန့်များ ရှိသင့်သည်-

1. မူရင်းလိုက်နာမှုလက်မှတ် (`ComplianceUpdateV1` သို့မဟုတ်
   `ModerationAppealV1`) နှင့် ဆက်စပ်အထောက်အထား တိုကင်များကို ပူးတွဲပါ။【docs/source/sorafs_gateway_compliance_plan.md:20】
2. တောင်းဆိုချက်သည် စိစစ်အယူခံဝင်မှုလမ်းကြောင်းကို ခေါ်ဆိုခြင်းရှိမရှိ အတည်ပြုပါ (နိုင်ငံသားအကန့်
   မဲခွဲဆုံးဖြတ်ခြင်း သို့မဟုတ် အရေးပေါ်ပါလီမန်ကို ဖျက်သိမ်းခြင်း၊ စီးဆင်းမှု နှစ်ခုစလုံးသည် ထင်ရှားမှုကို ကိုးကားရမည်ဖြစ်သည်။
   နမူနာပုံစံအသစ်တွင် ဖမ်းယူထားသော digest နှင့် retention tag။【docs/source/sorafs_moderation_panel_plan.md:1】
3. တိုးမြင့်လာမည့် နောက်ဆုံးရက်များကို ရေတွက်ပါ (အယူခံဝင်ခြင်း/ထုတ်ဖော်ပြတင်းပေါက်များ၊ အရေးပေါ်အခြေအနေ
   အေးခဲသောကြာချိန်) နှင့် မည်သည့်ကောင်စီ သို့မဟုတ် အဖွဲ့မှ နောက်ဆက်တွဲကို ပိုင်ဆိုင်သည့်ပြည်နယ်။
4. အသုံးပြုသည့် တယ်လီမီတာ လျှပ်တစ်ပြက်ရိုက်ချက် (ကြားခံအခန်း၊ ထိန်းညှိပေးသည့် မှတ်တမ်း) ကို ဖမ်းယူပါ။
   အောက်ခြေစာရင်းစစ်များသည် တိုက်ရိုက်ထုတ်လွှင့်သည့် ဆုံးဖြတ်ချက်နှင့် ကိုက်ညီနိုင်သောကြောင့် လုပ်ဆောင်ချက်ကို မျှတစေသည်။
   ပြည်နယ်။

လိုက်နာမှုနှင့် စိစစ်အကဲဖြတ်သည့် အကန့်များသည် ၎င်းတို့၏ အပတ်စဉ် ပွင့်လင်းမြင်သာမှု အစီရင်ခံစာများကို ထပ်တူပြုရပါမည်။
အခြေချရောက်တာ အော်ပရေတာများနှင့် ဖြစ်သောကြောင့် ဖယ်ရှားခြင်းနှင့် ထောက်ပံ့ကြေးများသည် တူညီပါသည်။
manifests အစုံ။

## အစီရင်ခံခြင်း ပုံစံများ

DA-10 ပြန်လည်သုံးသပ်မှုအားလုံးသည် ယခု လက်မှတ်ရေးထိုးထားသော Markdown packet လိုအပ်ပါသည်။ ကော်ပီ
`docs/examples/da_manifest_review_template.md`၊ မန်နီးဖက်စ် မက်တာဒေတာကို ထည့်သွင်းပါ၊
ထိန်းသိမ်းမှုအတည်ပြုခြင်းဇယားနှင့် အကန့်မဲအနှစ်ချုပ်၊ ထို့နောက် အပြီးသတ်ကို ပင်ထိုးပါ။
အုပ်ချုပ်ရေး DAG ဝင်ခွင့်သို့ စာရွက်စာတမ်း (Norito/JSON အထောက်အထားများ)။
အကန့်များသည် အုပ်ချုပ်မှုမိနစ်များတွင် ပက်ကေ့ခ်ျကို လင့်ခ်ချိတ်သင့်သည်၊ သို့မှသာ နောင်တွင် ဖြုတ်ချခြင်း သို့မဟုတ်
ထောက်ပံ့ကြေး သက်တမ်းတိုးခြင်းများသည် ပြန်လည်လုပ်ဆောင်ခြင်းမရှိဘဲ မူရင်းဖော်ပြချက်အချက်များကို ကိုးကားနိုင်သည်။
အခမ်းအနားတစ်ခုလုံး။

## ဖြစ်ရပ်နှင့် ရုတ်သိမ်းခြင်းလုပ်ငန်းအသွားအလာ

အရေးပေါ် လုပ်ဆောင်ချက်များသည် ယခုအခါတွင် ကွင်းဆက်ဖြစ်ပေါ်နေပါသည်။ ဘယ်အချိန်မှာ ခံစစ်မှူးက လွှတ်ထားဖို့လိုတယ်။
နောက်ပြန်ဆုတ်ပြီး အုပ်ချုပ်မှု လက်မှတ် တင်သွင်းပြီး လွှတ်တော်ကို ပြန်ပြောင်း အဆိုပြုတယ်။
ယခင်အတည်ပြုထားသော manifest digest ကို ညွှန်ပြသည်။ Infrastructure Panel ၊
မဲကိုကိုင်တွယ်ပြီး Nexus runtime ကို အပြီးသတ်ပြီးသည်နှင့် နောက်ပြန်လှည့်သည်
အောက်ပိုင်းရှိ ဖောက်သည်များ စားသုံးသည့် ဖြစ်ရပ်။ ဒေသတွင်း JSON ပစ္စည်းများ မလိုအပ်ပါ။

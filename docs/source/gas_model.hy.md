# IVM գազի մոդել

Այս փաստաթուղթը սահմանում է Iroha վիրտուալ մեքենայի գազի կանոնական ժամանակացույցը
(IVM) և բացատրում է, թե ինչպես է ժամանակացույցը հաշվել և կիրառվում: Ճշմարտության աղբյուրը
ծախսերի համար՝ `crates/ivm/src/gas.rs`; ստորև բերված ժամանակացույցի աղյուսակը ներկայացված է
այդ կանոնական քարտեզագրման տեսակետը:

## Շրջանակ

- Կիրառվում է IVM բայթ կոդի կատարման համար (Executable::Ivm):
- Բնական ISI գազի հաշվառումը սահմանվում է առանձին `crates/iroha_core/src/gas.rs`-ում:
- ISO 20022 օպերացիոն կոդերը պահպանված են ABI v1-ում և դեռ չեն պարունակում գազի մուտքեր:

## Դետերմինիզմ և ժամանակացույցի հեշ

Գազի գրաֆիկը դետերմինիստական է և բխում է opcode → ծախսերի զույգերից: Այն
Կանոնական ամփոփումը հաշվարկվում է պատվիրված opcode ցուցակի վրա յուրաքանչյուր մուտքի հետ
սերիականացված որպես.

- opcode բայթ (u8)
- արժեքը (u64, քիչ-էնդիան)

Հեշը բացահայտվում է.

- `ivm::gas::schedule_hash()` (կանոնական ժամանակացույցի հեշ)
- `ivm::limits::schedule_hash()` (հյուրընկալողին ուղղված կեղծանուն)

Օգտագործեք այս ամփոփագիրը՝ ստուգելու համար, որ բոլոր հասակակիցները նույն ժամանակացույցն ունեն էլեկտրահաղորդման ժամանակ
կազմաձևման կամ հեռաչափության ստուգումներ:

## Վեկտորի մասշտաբավորում և HTM կրկնվող փորձեր

- Վեկտորային օպերացիաներ (`VADD*`, `VAND`, `VXOR`, `VOR`, `VROT32`) տրամաբանական սանդղակ
  վեկտորի երկարությունը սահմանված է `SETVL`-ով: Աղյուսակում ներկայացված բազային ծախսերը չափվում են ըստ
  `min(vector_len, VECTOR_BASE_LANES) / VECTOR_BASE_LANES` (բազային գիծ = 2 գիծ):
- HTM-ը նորից փորձում է բազմապատկել արժեքը `(retries + 1)`-ով; Կոնսենսուսի ուղիների մեծ մասը չեն անում
  կրկնակի փորձեր կատարել:

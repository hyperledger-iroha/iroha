# Swift Sample Smoke Tests

IOS5 calls for sample app smoke coverage so the Swift SDK remains buildable as the
workspace evolves. The helper scripts below build and test both iOS demos on macOS
runners and expose knobs that CI can tune per lane.

## Prerequisites

- Xcode 15+ with simulators installed (the scripts default to `iPhone 15`).
- `swift`, `xcodebuild`, and `xcrun` available on `PATH`.
- `xcodegen` installed for the `NoritoDemo` template project.
- Optional: `NoritoBridge.xcframework` for bridge-backed scenarios (the template and
  SwiftUI samples still compile without it).

## Running the smokes

From the repository root:

```bash
ci/check_swift_samples.sh
```

This wrapper calls `scripts/check_swift_samples.sh`, which:

1) Generates `examples/ios/NoritoDemo/NoritoDemo.xcodeproj` via `xcodegen`, then builds
   and tests the `NoritoDemo` scheme.
2) Invokes `scripts/ci/verify_norito_demo.sh` to build and test the SwiftUI sample under
   `examples/ios/NoritoDemoXcode`.

Key environment toggles:

- `SWIFT_SAMPLES_DESTINATION` ‚Äî override the Xcode destination (default
  `platform=iOS Simulator,name=iPhone 15`).
- `SWIFT_SAMPLES_DERIVED_DATA` ‚Äî where DerivedData is written
  (default `artifacts/swift_samples`).
- `SWIFT_SAMPLES_SKIP_TEMPLATE=1` ‚Äî skip the XcodeGen template build/tests.
- `SWIFT_SAMPLES_SKIP_NORITO_DEMO_XCODE=1` ‚Äî skip the SwiftUI sample build/tests.
- `SWIFT_SAMPLES_SUMMARY` ‚Äî optional path for a machine-readable JSON summary
  (defaults to `artifacts/swift_samples/summary.json` alongside DerivedData).
- `SWIFT_SAMPLES_JUNIT` ‚Äî optional path for a JUnit report (defaults to
  `artifacts/swift_samples/swift_samples.junit.xml`; set to an empty string to
  disable if a lane only needs the JSON summary).
- `NORITO_DEMO_SCHEME` ‚Äî scheme to build/test in the SwiftUI project (default
  `NoritoDemoXcode`), used by `scripts/ci/verify_norito_demo.sh`.
- `NORITO_DEMO_DESTINATION` ‚Äî overrides the SwiftUI sample destination if different
  from `SWIFT_SAMPLES_DESTINATION` (default `platform=iOS Simulator,name=iPhone 15`).
- `NORITO_DEMO_DERIVED_DATA` ‚Äî DerivedData directory for the SwiftUI sample
  (`scripts/ci/verify_norito_demo.sh` defaults to `artifacts/norito_demo_xcodebuild`;
  `ci/check_swift_samples.sh` passes `artifacts/swift_samples/NoritoDemoXcode`).

If macOS tooling is unavailable the script exits early with a warning so Linux runners do
not fail unrelated jobs; CI lanes should keep the defaults to enforce coverage.

## Destination handling

The helper checks for the requested simulator via `xcrun simctl`. If the named device is
missing it inspects `xcodebuild -showdestinations` for fallbacks (preferring `My Mac`
designed-for-iPhone/iPad destinations when available). When no destination can be
resolved, the script emits a warning and skips the affected sample instead of hard
failing non-macOS environments.

## Outputs

- `artifacts/swift_samples/NoritoDemo` ‚Äî DerivedData for the XcodeGen template build/tests.
- `artifacts/swift_samples/NoritoDemoXcode` ‚Äî DerivedData for the SwiftUI sample
  build/tests.
- `artifacts/swift_samples/summary.json` ‚Äî status records for each sample
  (`passed`/`failed`/`skipped` plus reason, destination, and
  `duration_seconds`) so CI dashboards can surface why a lane was skipped/failed
  and how long each run took without re-reading logs.
- `artifacts/swift_samples/swift_samples.junit.xml` ‚Äî CI-friendly JUnit suite
  summarising the same results (including the per-sample duration in the `time`
  attribute); helpful for piping into Buildkite/GitHub test visualisations or
  plotting slow lanes.
- `artifacts/swift_samples/swift_samples.prom` ‚Äî Prometheus textfile metrics
  (`swift_samples_*`) generated by `scripts/swift_samples_metrics.py` so
  Buildkite lanes can feed the IOS5 telemetry dashboards without reprocessing
  logs; standard scrape jobs ingest this path from Buildkite artefacts. See
  `docs/source/telemetry/swift_status_feeds.md#swift-sample-smoke-metrics` for
  the textfile wiring and scrape guidance.

Surface the logs from these directories in CI to tie failures back to the IOS5 DX/testing
gate.

## Buildkite entrypoint

The `:swift: Sample Smokes` Buildkite pipeline (`.buildkite/swift-samples.yml`)
runs `ci/check_swift_samples.sh` on a macOS simulator host, uploads the JSON +
JUnit artefacts, and annotates the summary so IOS5 reviewers can see results
inline. Trigger it manually with:

```bash
buildkite-agent pipeline upload --pipeline .buildkite/swift-samples.yml
```

The job writes its outputs under `artifacts/swift_samples/` (matching the
defaults above) and exposes the JUnit file for downstream test-analytics hooks.

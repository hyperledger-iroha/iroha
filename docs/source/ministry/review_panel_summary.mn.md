# Хяналтын хэсгийн хураангуй (MINFO-4a)

Замын зургийн зүйл **MINFO-4a — Төвийг сахисан хураангуй үүсгэгч** нь хүлээн зөвшөөрөгдсөн хэлэлцэх асуудлын санал, сайн дурын товч корпус, баталгаажуулсан хиймэл оюун ухааны зохицуулалтыг төвийг сахисан бүх нийтийн санал асуулгын хураангуй болгож хувиргах ажлын урсгалыг шаарддаг. Хүргэлт нь:

- Гаралтыг Norito бүтэц (`ReviewPanelSummaryV1`) болгон тэмдэглэснээр засаглал үүнийг манифест болон саналын хуудасны хамт архивлах боломжтой.
- Хяналтын хэсэг тэнцвэртэй дэмжлэггүй/эсэргүүцэлгүй эсвэл баримт дутуу ишлэл байгаа үед эх материалыг хурдан бүтэлгүйтнэ.
- Бодлогын тангарагтны шүүгчид санал өгөхөөс өмнө автоматжуулсан болон хүний ​​нөхцөл байдлын аль алиныг нь хардаг эсэхийг баталгаажуулахын тулд AI манифест болон саналын нотлох баримтын багцыг онцлон тэмдэглэ.

## CLI хэрэглээ

Ажлын урсгалыг `cargo xtask`-ийн нэг хэсэг болгон илгээдэг:

```bash
cargo xtask ministry-panel synthesize \
  --proposal docs/examples/ministry/agenda_proposal_example.json \
  --volunteer docs/examples/ministry/volunteer_brief_template.json \
  --ai-manifest docs/examples/ai_moderation_calibration_manifest_202602.json \
  --panel-round RP-2026-05 \
--output artifacts/review_panel/AC-2026-001-RP-2026-05.json
```

Шаардлагатай оролтууд:

1. `--proposal` – `AgendaProposalV1`-д нийцсэн JSON ачаалал. Туслагч нь хураангуйг үүсгэхээс өмнө схемийг баталгаажуулдаг.
2. `--volunteer` – `docs/source/ministry/volunteer_brief_template.md`-ийг дагасан сайн дурынхны JSON багц. Сэдвээс гадуурх оруулгуудыг автоматаар үл тоомсорлодог.
3. `--ai-manifest` – Засаглалын гарын үсэгтэй `ModerationReproManifestV1` агуулгыг шалгасан хиймэл оюун ухааны хороог тодорхойлсон.
4. `--panel-round` – Одоогийн хяналтын шатны танигч (`RP-YYYY-##`).
5. `--output` – Зорилтот файл эсвэл stdout руу дамжуулах `-`. Саналын хэлийг хүчингүй болгохын тулд `--language`, түүхийг дахин бөглөхдөө тодорхойлогч Unix цагийн тэмдэг (миллисекунд) өгөхийн тулд `--generated-at`-ийг ашиглана уу.

### `ministry-transparency ingest`-ээр дамжуулан автоматжуулалт

Улирлын нотлох баримтын багцад `cargo xtask ministry-transparency ingest` программыг аль хэдийн ажиллуулж байсан багууд одоо хяналтын самбарын хураангуйг ижил шугам руу холбож болно:

```bash
cargo xtask ministry-transparency ingest \
  --quarter 2026-Q4 \
  --ledger artifacts/ministry/ledger.json \
  --appeals artifacts/ministry/appeals.json \
  --denylist artifacts/ministry/denylist.json \
  --treasury artifacts/ministry/treasury.json \
  --volunteer artifacts/ministry/volunteer_briefs.json \
  --panel-proposal artifacts/ministry/proposal_AC-2026-041.json \
  --panel-ai-manifest artifacts/ministry/ai_manifest.json \
  --panel-round RP-2026-05 \
  --panel-summary-out artifacts/ministry/review_panel_summary.json \
  --output artifacts/ministry/ingest.json
```

Бүх дөрвөн `--panel-*` тугуудыг хамтад нь нийлүүлэх ёстой (мөн `--volunteer` шаардлагатай). Энэ тушаал нь хяналтын самбарын хураангуйг `--panel-summary-out` руу илгээж, задалсан ачааллыг агшин зуурын агшинд оруулж, хяналтын нийлбэрийг бүртгэснээр доод урсгалын хэрэглүүр нотлох баримтыг батлах болно.

## Хувцаслах болон бүтэлгүйтлийн горимууд

`cargo xtask ministry-panel synthesize` хураангуйг бичихээс өмнө дараах инвариантуудыг хэрэгжүүлдэг:

- **Тэнцвэртэй байр суурь:** дор хаяж нэг дэмжигч, нэг эсэргүүцэгч байх ёстой. Хамрах хүрээ дутуу байгаа нь тайлбарласан алдаагаар гүйлтийг дуусгана.
- **Ишлэлийн хамрах хүрээ:** онцлох эшлэлүүдийг зөвхөн ишлэл агуулсан баримтын мөрүүдээс гаргадаг. Алга болсон ишлэлүүд нь бүтээхийг хэзээ ч хориглодоггүй боловч нөлөөлөлд өртсөн товч мэдээлэл бүрийг гаралтын `warnings[]` доор жагсаасан болно.
- **Тодруулсан лавлагаа:** онцлох зүйл болгонд (a) сайн дурын баримтын мөр(үүд), (б) AI манифест ID, (в) саналын эхний нотлох баримтын хавсралт багтсан тул пакет нь гарын үсэг зурсан олдворуудтай үргэлж холбогддог.Хэрэв шалгалт амжилтгүй болвол тушаал нь тэг биш төлөвөөр гарч, асуудалтай бичлэгийг заана. Амжилттай гүйлтүүд нь `ReviewPanelSummaryV1` схемд тохирох JSON файлыг бичдэг бөгөөд удирдлагын манифестуудад суулгаж болно.

## Гаралтын бүтэц

`ReviewPanelSummaryV1` нь `crates/iroha_data_model/src/ministry/mod.rs`-д амьдардаг бөгөөд `iroha_data_model` хайрцагаар дамжуулан хэрэглэгч бүрт боломжтой. Гол хэсгүүдэд:

- `overview` – Бодлогын тангарагтны багцын гарчиг, төвийг сахисан хураангуй өгүүлбэр, шийдвэрийн нөхцөл.
- `stance_distribution` – Нэг байр суурь дахь товч мэдээлэл болон баримтын мөрийн тоо. Доод талын хяналтын самбарууд нийтлэхээс өмнө хамрах хүрээг баталгаажуулахын тулд үүнийг уншдаг.
- `highlights` – Бүрэн шаардлага хангасан ишлэл бүхий нэг байр сууринд хоёр хүртэлх баримтын хураангуй.
- `ai_manifest` – Дахин давтагдах байдлын манифестээс задалсан мета өгөгдөл (манифест UUID, гүйгч хувилбар, босго).
- `volunteer_references` – Аудитын товч статистик (хэл, байр суурь, мөр, иш татсан мөр).
- `warnings` – Алгассан зүйлсийг дүрсэлсэн чөлөөт хэлбэрийн хөвөн зурвасууд (жишээ нь, ишлэл алга болсон баримтын мөр).

## Жишээ

`docs/examples/ministry/review_panel_summary_example.json` нь туслагчтай хамт үйлдвэрлэсэн бүрэн дээжийг агуулдаг. Энэ нь тэнцвэртэй дэмжлэг/сөрөг хамрах хүрээ, ишлэлийн утас, манифест лавлагаа, онцлох үйл явдалд дэвшүүлэх боломжгүй баримтын мөрүүдийн анхааруулах мөрүүдийг харуулдаг. Төвийг сахисан хураангуйг ашиглах шаардлагатай хяналтын самбар, засаглалын манифест эсвэл SDK хэрэгслийг өргөтгөх үед үүнийг ашиглана уу.

> **Зөвлөгөө:** үүсгэсэн хураангуйг гарын үсэг зурсан хиймэл оюун ухааны манифест болон сайн дурын товч тоймтой хамт санал асуулгын нотлох баримтын багцад оруулснаар бодлогын тангарагтнууд хянан шалгах хэсгийн лавласан олдвор бүрийг баталгаажуулах боломжтой.

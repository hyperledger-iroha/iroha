# Шолу тақтасының қысқаша мазмұны (MINFO-4a)

Жол картасының **MINFO-4a — Бейтарап жиынтық генератор** тармағы қабылданған күн тәртібі ұсынысын, еріктілердің қысқаша корпусын және расталған AI модерациясын бейтарап референдум қорытындысына айналдыратын қайталанатын жұмыс процесін қажет етеді. Жеткізу керек:

- Шығаруды Norito құрылымы (`ReviewPanelSummaryV1`) ретінде жазыңыз, сондықтан басқару оны манифесттермен және бюллетеньдермен бірге мұрағаттай алады.
- Қарап шығу панелінде теңдестірілген қолдау/қарсы қамту болмаған кезде немесе фактілерде дәйексөздер жоқ болған кезде, бастапқы материалды жабыңыз.
- Саясат қазылар алқасының дауыс беру алдында автоматтандырылған және адам контекстін көретініне көз жеткізу үшін AI манифестіне және ұсыныстардың дәлелдер жинағына сілтеме жасаңыз.

## CLI пайдалану

Жұмыс процесі `cargo xtask` бөлігі ретінде жіберіледі:

```bash
cargo xtask ministry-panel synthesize \
  --proposal docs/examples/ministry/agenda_proposal_example.json \
  --volunteer docs/examples/ministry/volunteer_brief_template.json \
  --ai-manifest docs/examples/ai_moderation_calibration_manifest_202602.json \
  --panel-round RP-2026-05 \
--output artifacts/review_panel/AC-2026-001-RP-2026-05.json
```

Міндетті енгізулер:

1. `--proposal` – `AgendaProposalV1` сәйкес JSON пайдалы жүктемесі. Көмекші жиынтықты жасамас бұрын схеманы тексереді.
2. `--volunteer` – `docs/source/ministry/volunteer_brief_template.md` сәйкес келетін еріктілердің JSON жиыны. Тақырыптан тыс жазбалар автоматты түрде еленбейді.
3. `--ai-manifest` – Басқару қол қойылған `ModerationReproManifestV1`, мазмұнды тексерген AI комитетін сипаттайды.
4. `--panel-round` – Ағымдағы шолу раундының идентификаторы (`RP-YYYY-##`).
5. `--output` – stdout жүйесіне ағынмен жіберу үшін тағайындалған файл немесе `-`. Ұсыныс тілін қайта анықтау үшін `--language` және тарихты толтыру кезінде детерминирленген Unix уақыт белгісін (миллисекундтар) беру үшін `--generated-at` пайдаланыңыз.

Оқшау қорытынды жасалғаннан кейін іске қосыңыз
[`cargo xtask ministry-panel packet`](referendum_packet.md) құрастыруға арналған көмекші
референдумның толық құжаты (`ReferendumPacketV1`). Жабдықтау
`--summary-out` пакет пәрмені сол жиынтық файлды сақтайды.
оны төменгі ағынды тұтынушылар үшін пакет нысанының ішіне енгізу.

### `ministry-transparency ingest` арқылы автоматтандыру

Тоқсандық дәлелдер бумалары үшін `cargo xtask ministry-transparency ingest` іске қосып қойған топтар енді шолу тақтасының қорытындысын бір құбырға қоса алады:

```bash
cargo xtask ministry-transparency ingest \
  --quarter 2026-Q4 \
  --ledger artifacts/ministry/ledger.json \
  --appeals artifacts/ministry/appeals.json \
  --denylist artifacts/ministry/denylist.json \
  --treasury artifacts/ministry/treasury.json \
  --volunteer artifacts/ministry/volunteer_briefs.json \
  --panel-proposal artifacts/ministry/proposal_AC-2026-041.json \
  --panel-ai-manifest artifacts/ministry/ai_manifest.json \
  --panel-round RP-2026-05 \
  --panel-summary-out artifacts/ministry/review_panel_summary.json \
  --output artifacts/ministry/ingest.json
```

Барлық төрт `--panel-*` жалаушалары бірге жеткізілуі керек (және `--volunteer` қажет). Пәрмен `--panel-summary-out` нұсқасына шолу тақтасының қорытындысын шығарады, талданған пайдалы жүктемені қабылдау суретінің ішіне енгізеді және төменгі ағындық құралдар дәлелдерді растауы үшін бақылау сомасын жазады.

## Қаптау және ақаулық режимдері

`cargo xtask ministry-panel synthesize` қорытындыны жазбас бұрын келесі инварианттарды мәжбүрлейді:

- **Теңдестірілген ұстанымдар:** кем дегенде бір қолдаушы және бір қарсы пікір болуы керек. Жетіспейтін қамту сипаттаушы қатемен жұмысты аяқтайды.
- **Дәйексөзді қамту:** ерекше нүктелер дәйексөздерді қамтитын фактілер жолынан ғана жасалады. Жетіспейтін дәйексөздер құрастыруды ешқашан блоктамайды, бірақ әрбір зардап шеккен қысқаша шығыста `warnings[]` астында тізімделген.
- **Әрбір бөлектелген сілтемелер:** әрбір бөлектеуде (a) ерікті фактілер жолына(ларына), (b) AI манифест идентификаторына және (c) пакет әрқашан қол қойылған артефактілерге қайта сілтеме жасайтындай ұсыныстағы бірінші дәлел қосымшасына сілтемелер кіреді.Қандай да бір тексеру орындалмаса, пәрмен нөлдік емес күймен шығып, проблемалық жазбаны көрсетеді. Сәтті іске қосулар `ReviewPanelSummaryV1` схемасына сәйкес келетін және басқару манифесттеріне ендірілетін JSON файлын жазады.

## Шығару құрылымы

`ReviewPanelSummaryV1` `crates/iroha_data_model/src/ministry/mod.rs` ішінде тұрады және `iroha_data_model` жәшігі арқылы әрбір тұтынушыға қол жетімді. Негізгі бөлімдерге мыналар кіреді:

- `overview` – Саясат қазылар алқасы пакетіне арналған тақырып, бейтарап жиынтық сөйлем және шешім контексті.
- `stance_distribution` – Бір позициядағы қысқаша ақпараттар мен фактілер жолдарының саны. Төменгі бақылау тақталары оны жарияламас бұрын қамтуды растау үшін оқиды.
- `highlights` – Толық білікті дәйексөздермен бір ұстаным үшін екі фактіге дейін қысқаша мәлімет.
- `ai_manifest` – Қайта шығару манифестінен алынған метадеректер (UUID манифесті, жүгіруші нұсқасы, шекті мәндер).
- `volunteer_references` – Аудитке арналған қысқаша статистика (тіл, ұстаным, жолдар, келтірілген жолдар).
- `warnings` – Өткізілген элементтерді сипаттайтын еркін пішінді түкті хабарлар (мысалы, дәйексөздері жоқ фактілер жолдары).

## Мысал

`docs/examples/ministry/review_panel_summary_example.json` көмекшімен жасалған толық үлгіні қамтиды. Ол теңдестірілген қолдауды/қарсылықты қамтуды, дәйексөзді қосуды, манифест сілтемелерін және маңызды жерлерге жылжыту мүмкін емес фактілер жолдары үшін ескерту жолдарын көрсетеді. Бейтарап қорытындыны қажет ететін бақылау тақталарын, басқару манифесттерін немесе SDK құралдарын кеңейту кезінде оны пайдаланыңыз.

> **Кеңес:** жасалған түйіндемені қол қойылған AI манифестімен және еріктілердің қысқаша дайджестімен бірге референдум дәлелдер жинағына қосыңыз, осылайша саясат қазылары шолу тобы сілтеме жасаған әрбір артефактты тексере алады.

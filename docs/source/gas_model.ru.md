# IVM Газовая модель

Этот документ определяет каноническое расписание газа для виртуальной машины Iroha.
(IVM) и объясняет, как хешируется и применяется расписание. Источник истины
по затратам — `crates/ivm/src/gas.rs`; таблица расписания ниже визуализирована
взгляд на это каноническое отображение.

## Область применения

- Применяется к выполнению байт-кода IVM (Executable::Ivm).
- Собственное измерение газа ISI определяется отдельно в `crates/iroha_core/src/gas.rs`.
— Коды операций ISO 20022 зарезервированы в ABI v1 и пока не содержат записей о газе.

## Детерминизм и хэш расписания

График газа является детерминированным и получен на основе пар код операции → стоимость.
канонический дайджест вычисляется по упорядоченному списку кодов операций с каждой записью
сериализовано как:

- байт кода операции (u8)
- стоимость (u64, прямой порядок байтов)

Хэш предоставляется:

- `ivm::gas::schedule_hash()` (хеш канонического расписания)
- `ivm::limits::schedule_hash()` (псевдоним, обращенный к хосту)

Используйте этот дайджест, чтобы убедиться, что все одноранговые узлы используют одно и то же расписание при подключении.
проверка конфигурации или телеметрии.

## Векторное масштабирование и повторы HTM

- Векторные операции (`VADD*`, `VAND`, `VXOR`, `VOR`, `VROT32`) масштабируются с логическим
  длина вектора устанавливается `SETVL`. Базовые затраты в таблице масштабированы по
  `min(vector_len, VECTOR_BASE_LANES) / VECTOR_BASE_LANES` (базовый уровень = 2 полосы).
- Повторные попытки HTM умножают стоимость на `(retries + 1)`; большинство путей достижения консенсуса не
  повлечь за собой повторные попытки.

# Տվյալների մատչելիության վարձավճար և խրախուսական քաղաքականություն (DA-7)

_ Կարգավիճակը. Նախագծում — Սեփականատերեր՝ Էկոնոմիկայի WG / Գանձապետարան / Պահպանման թիմ_

Ճանապարհային քարտեզի **DA-7** կետը ներկայացնում է բացահայտ XOR-ով արտահայտված վարձավճար յուրաքանչյուր բլբի համար
ներկայացվել է `/v1/da/ingest`-ին, գումարած բոնուսներ, որոնք պարգևատրում են PDP/PoTR կատարումը և
egress-ը ծառայել է հաճախորդներ բերելու համար: Այս փաստաթուղթը սահմանում է նախնական պարամետրերը,
դրանց տվյալների մոդելի ներկայացումը և Torii-ի կողմից օգտագործված հաշվարկային հոսքը,
SDK-ներ և գանձապետական վահանակներ:

## Քաղաքականության կառուցվածքը

Քաղաքականությունը կոդավորված է որպես [`DaRentPolicyV1`](/crates/iroha_data_model/src/da/types.rs)
տվյալների մոդելի շրջանակներում: Torii-ը և կառավարման գործիքակազմը պահպանում են քաղաքականությունը
Norito օգտակար բեռներ, որպեսզի հնարավոր լինի վերահաշվարկել վարձակալության գնանշումները և խրախուսական մատյանները
դետերմինիստորեն. Սխեման բացահայտում է հինգ կոճակ.

| Դաշտային | Նկարագրություն | Կանխադրված |
|-------|-------------|---------|
| `base_rate_per_gib_month` | XOR-ը գանձվում է մեկ GiB-ի համար մեկ ամսվա պահպանման համար: | `250_000` միկրո-XOR (0.25 XOR) |
| `protocol_reserve_bps` | Արձանագրության պահուստին ուղղվող վարձավճարի մասնաբաժինը (հիմնական կետերը): | `2_000` (20%) |
| `pdp_bonus_bps` | Բոնուսի տոկոսը PDP-ի հաջող գնահատման համար: | `500` (5%) |
| `potr_bonus_bps` | Բոնուսի տոկոսը հաջող PoTR գնահատման համար: | `250` (2,5%) |
| `egress_credit_per_gib` | Վարկը վճարվում է, երբ մատակարարը սպասարկում է DA տվյալների 1 ԳԲ: | `1_500` միկրո-XOR |

Բոլոր բազային արժեքները վավերացված են `BASIS_POINTS_PER_UNIT`-ի (10000) համեմատ:
Քաղաքականության թարմացումները պետք է անցնեն կառավարման միջոցով, և յուրաքանչյուր Torii հանգույց բացահայտում է
ակտիվ քաղաքականություն `torii.da_ingest.rent_policy` կոնֆիգուրացիայի բաժնի միջոցով
(`iroha_config`): Օպերատորները կարող են վերացնել կանխադրվածները `config.toml`-ում.

```toml
[torii.da_ingest.rent_policy]
base_rate_per_gib_month_micro = 250000        # 0.25 XOR/GiB-month
protocol_reserve_bps = 2000                   # 20% protocol reserve
pdp_bonus_bps = 500                           # 5% PDP bonus
potr_bonus_bps = 250                          # 2.5% PoTR bonus
egress_credit_per_gib_micro = 1500            # 0.0015 XOR/GiB egress credit
```

CLI գործիքավորումը (`iroha app da rent-quote`) ընդունում է նույն Norito/JSON քաղաքականության մուտքերը
և արտանետում է արտեֆակտներ, որոնք արտացոլում են ակտիվ `DaRentPolicyV1`-ը՝ առանց հասնելու
վերադարձ դեպի Torii վիճակ: Տրամադրեք քաղաքականության նկարը, որն օգտագործվում է ներթափանցման համար, որպեսզի
մեջբերումը մնում է վերարտադրելի:

### Մշտական վարձակալության արտեֆակտներ

Գործարկեք `iroha app da rent-quote --gib <size> --months <months> --quote-out <path>` դեպի
թողարկեք և՛ էկրանի ամփոփագիրը, և՛ գեղեցիկ տպագրված JSON արտեֆակտ: Ֆայլը
գրանցում է `policy_source`, ներգծված `DaRentPolicyV1` լուսանկար, հաշվարկված
`DaRentQuote` և ստացված `ledger_projection` (սերիականացված միջոցով
[`DaRentLedgerProjection`](/crates/iroha_data_model/src/da/types.rs)) այն հարմար է գանձապետական վահանակների և մատյան ISI-ի համար
խողովակաշարեր. Երբ `--quote-out`-ը մատնանշում է ներդիր գրացուցակը, CLI-ն ստեղծում է ցանկացած
բացակայող ծնողներ, այնպես որ օպերատորները կարող են ստանդարտացնել այնպիսի վայրեր, ինչպիսիք են
`artifacts/da/rent_quotes/<timestamp>.json` DA ապացույցների այլ փաթեթների հետ միասին:
Կցեք արտեֆակտը վարձակալության հաստատումներին կամ հաշտեցմանը, որպեսզի XOR-ը
բաշխումը (բազային վարձավճար, պահուստ, PDP/PoTR բոնուսներ և արտագնա վարկեր) է
վերարտադրելի։ Անցեք `--policy-label "<text>"`՝ ինքնաբերաբար վերացնելու համար
ստացված `policy_source` նկարագրությունը (ֆայլի ուղիներ, ներկառուցված լռելյայն և այլն)
մարդու կողմից ընթեռնելի պիտակ, օրինակ՝ կառավարման տոմս կամ մանիֆեստային հեշ; CLI զարդարանքները
այս արժեքը և մերժում է միայն դատարկ/սպիտակ տողերը, որպեսզի գրանցված ապացույցները
մնում է ստուգման ենթակա:

```json
{
  "policy_source": "policy JSON `configs/da/rent_policy.json`",
  "gib": 10,
  "months": 3,
  "policy": { "...": "DaRentPolicyV1 fields elided" },
  "quote": { "...": "DaRentQuote breakdown" },
  "ledger_projection": {
    "rent_due": { "micro": 7500000 },
    "protocol_reserve_due": { "micro": 1500000 },
    "provider_reward_due": { "micro": 6000000 },
    "pdp_bonus_pool": { "micro": 375000 },
    "potr_bonus_pool": { "micro": 187500 },
    "egress_credit_per_gib": { "micro": 1500 }
  }
}
```Գրքերի նախագծման բաժինը ուղղակիորեն սնվում է DA-ի վարձակալության մատյանների ISI-ներում
սահմանում է XOR դելտաները, որոնք նախատեսված են արձանագրության պահուստի, մատակարարի վճարումների և
յուրաքանչյուր ապացույց բոնուսային լողավազաններ՝ առանց պատվիրված նվագախմբի կոդ պահանջելու:

### Վարձավճարների հաշվառման պլանների ստեղծում

Գործարկեք `iroha app da rent-ledger --quote <path> --payer-account <id> --treasury-account <id> --protocol-reserve-account <id> --provider-account <id> --pdp-bonus-account <id> --potr-bonus-account <id> --asset-definition xor#sora`
շարունակական վարձավճարի գնանշումը կատարվող մատյանային փոխանցումների վերածելու համար: Հրաման
վերլուծում է ներկառուցված `ledger_projection`-ը, թողարկում Norito `Transfer` հրահանգներ
որոնք բազային վարձավճարը գանձում են գանձարան, ուղղորդում պահուստին/մատակարարին
մասեր և նախապես ֆինանսավորում է PDP/PoTR բոնուսային լողավազանները անմիջապես վճարողից: Այն
JSON-ի ելքը արտացոլում է գնանշումների մետատվյալները, որպեսզի CI-ն և գանձապետական գործիքները կարողանան պատճառաբանել
նույն արտեֆակտի մասին.

```json
{
  "quote_path": "artifacts/da/rent_quotes/2025-12-07/rent.json",
  "rent_due_micro_xor": 7500000,
  "protocol_reserve_due_micro_xor": 1500000,
  "provider_reward_due_micro_xor": 6000000,
  "pdp_bonus_pool_micro_xor": 375000,
  "potr_bonus_pool_micro_xor": 187500,
  "egress_credit_per_gib_micro_xor": 1500,
  "instructions": [
    { "Transfer": { "...": "payer -> treasury base rent instruction elided" }},
    { "Transfer": { "...": "treasury -> reserve" }},
    { "Transfer": { "...": "treasury -> provider payout" }},
    { "Transfer": { "...": "payer -> PDP bonus escrow" }},
    { "Transfer": { "...": "payer -> PoTR bonus escrow" }}
  ]
}
```

Վերջնական `egress_credit_per_gib_micro_xor` դաշտը թույլ է տալիս վահանակներ և վճարումներ
Ժամանակացույցը հավասարեցնում է արտագնա փոխհատուցումները վարձակալության քաղաքականության հետ, որն առաջացրել է
մեջբերում՝ առանց սկրիպտային սոսինձի քաղաքականության մաթեմատիկան վերահաշվարկելու:

## Օրինակ մեջբերում

```rust
use iroha_data_model::da::types::DaRentPolicyV1;

// 10 GiB retained for 3 months.
let policy = DaRentPolicyV1::default();
let quote = policy.quote(10, 3).expect("policy validated");

assert_eq!(quote.base_rent.as_micro(), 7_500_000);      // 7.5 XOR total rent
assert_eq!(quote.protocol_reserve.as_micro(), 1_500_000); // 20% reserve
assert_eq!(quote.provider_reward.as_micro(), 6_000_000);  // Direct provider payout
assert_eq!(quote.pdp_bonus.as_micro(), 375_000);          // PDP success bonus
assert_eq!(quote.potr_bonus.as_micro(), 187_500);         // PoTR success bonus
assert_eq!(quote.egress_credit_per_gib.as_micro(), 1_500);
```

Գնանշումը վերարտադրելի է Torii հանգույցներում, SDK-ներում և գանձապետական հաշվետվություններում, քանի որ
այն օգտագործում է որոշիչ Norito կառուցվածքներ՝ ժամանակավոր մաթեմատիկայի փոխարեն: Օպերատորները կարող են
կցել JSON/CBOR կոդավորված `DaRentPolicyV1` կառավարման առաջարկներին կամ վարձակալությանը
աուդիտներ՝ ապացուցելու համար, թե որ պարամետրերն են ուժի մեջ եղել տվյալ բլբի համար:

## Բոնուսներ և պահուստներ

- **Արձանագրության պահուստ.** `protocol_reserve_bps`-ը ֆինանսավորում է XOR պահուստը, որը աջակցում է
  արտակարգ իրավիճակների կրկնօրինակում և կրճատվող փոխհատուցումներ: Գանձապետարանը հետևում է այս դույլին
  առանձին՝ ապահովելու համար, որ մատյանների մնացորդները համապատասխանում են կազմաձևված դրույքաչափին:
- **PDP/PoTR բոնուսներ.** Յուրաքանչյուր հաջող ապացույցի գնահատում ստանում է հավելյալ
  `base_rent × bonus_bps`-ից ստացված վճարում: Երբ DA ժամանակացույցը թողարկում է ապացույց
  անդորրագրերը ներառում են բազային կետերի պիտակները, որպեսզի խթանները հնարավոր լինի վերարտադրել:
- **Egress վարկ.** Մատակարարները գրանցում են GiB-ի մատուցված յուրաքանչյուր մանիֆեստի համար, բազմապատկվում է
  `egress_credit_per_gib`, իսկ անդորրագրերը ներկայացրեք `iroha app da prove-availability`-ի միջոցով:
  Վարձավճարի քաղաքականությունը պահպանում է մեկ GiB-ի գումարը համաժամանակյա կառավարման հետ:

## Գործառնական հոսք

1. **Կլանել.
   հիմնված է բշտիկի չափի և պահպանման վրա և տեղադրում է մեջբերումը Norito-ում
   դրսևորել. Ներկայացնողը ստորագրում է հայտարարություն, որը վկայակոչում է վարձավճարը և
   պահեստավորման տոմսի ID.
2. **Հաշվապահական հաշվառում.** Գանձապետական ներթափանցման սկրիպտները վերծանում են մանիֆեստը, զանգահարեք
   `DaRentPolicyV1::quote` և լրացրեք վարձակալության մատյանները (բազային վարձավճար, պահուստ,
   բոնուսներ և ակնկալվող արտագնա վարկեր): Ցանկացած անհամապատասխանություն գրանցված վարձավճարի միջև
   և վերահաշվարկված մեջբերումները ձախողվում են CI-ում:
3. **Ապացույցի պարգևներ.** Երբ PDP/PoTR պլանավորողները նշում են հաջողություն, նրանք ստացական են տալիս
   պարունակող մանիֆեստի բովանդակությունը, ապացույցի տեսակը և դրանից ստացված XOR բոնուսը
   քաղաքականությունը։ Կառավարումը կարող է ստուգել վճարումները՝ վերահաշվարկելով նույն մեջբերումը:
4. **Ելքի փոխհատուցում. ** Բերել նվագարկիչները ներկայացնում են ստորագրված ելքի ամփոփագրեր:
   Torii-ը բազմապատկում է GiB-ի քանակը `egress_credit_per_gib`-ով և թողարկում վճարում
   ցուցումներ վարձակալության պահման դեմ:

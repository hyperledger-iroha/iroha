# Конфиденциаль активтар & ZK тапшырыу дизайн

## Мотивация
- Ҡалҡан экранланған актив ағымдарын тапшырыу үтә күренмәле ҡан әйләнешен үҙгәртмәйенсә транзакция хосусилығын һаҡлай ала.
- Техника һәм криптографик параметрҙар өсөн йәшәү циклы менән идара итеү (активация, әйләнеш, ҡайтарыу) менән аудиторҙар һәм операторҙар менән тәьмин итеү.

## Хәүефле модель
- Валидаторҙар намыҫлы-әммә ҡыҙыҡһыныусан: улар консенсус тоғро үтәй, әммә тикшерергә тырыша баш китабы/дәүләт.
- Селтәр күҙәтеүселәре блок мәғлүмәттәрен һәм ғәйбәт транзакцияларын күрә; шәхси ғәйбәт каналдары тураһында фараз юҡ.
- Отколдан: офф-леджер трафик анализы, квант дошмандары (айырымда айырым PQ юл картаһы буйынса күҙәтелә), леджер доступность һөжүмдәре.

## Дизайн дөйөмләштереү
- Активтар ғәмәлдәге үтә күренмәле баланстарҙан тыш, * экранланған бассейн* иғлан итә ала; экранланған әйләнеш криптографик йөкләмәләр аша күрһәтелә.
- Иҫкәрмәләр `(asset_id, amount, recipient_view_key, blinding, rho)` менән инкапсулировать:
  - йөкләмә: `Comm = Pedersen(params_id || asset_id || amount || recipient_view_key || blinding)`.
  - Нулификацияһы: `Null = Poseidon(domain_sep || nk || rho || asset_id || chain_id)`, бойороҡтан бойондороҡһоҙ.
  - Шифрланған файҙалы йөк: `enc_payload = AEAD_XChaCha20Poly1305(ephemeral_shared_key, note_plaintext)`.
- Транзакциялар транспорты Norito-кодланған `ConfidentialTransfer` файҙалы йөктәрҙе үҙ эсенә ала:
  - Йәмәғәт индереүҙәре: Меркл якорь, нульлификатор, яңы йөкләмәләр, актив ид, схема версияһы.
  - шифрланған файҙалы йөктәр алыусылар һәм өҫтәмә аудиторҙар өсөн.
  - Нулле-белемде иҫбатлаусы ҡиммәтте һаҡлау, милек һәм рөхсәт алыу.
- Асҡыстарҙы һәм параметрҙар комплекттарын тикшерергә легаль реестрҙары аша активлаштырыу windows менән идара ителә; төйөндәр һылтанма билдәһеҙ йәки яҙмаларҙы тартып алыусы дәлилдәрҙе раҫлауҙан баш тарта.
- Консенсус башлыҡтары әүҙем конфиденциаль функция distest үҙ өҫтөнә ала, шуға күрә блоктар ғына ҡабул ителә, ҡасан реестр һәм параметр дәүләт матчтары.
- Дәлил төҙөлөштә ышаныслы ҡоролмаһыҙ Halo2 (Plonkish) өйөм ҡулланыла; Грот16 йәки башҡа SNARK варианттары аңлы рәүештә v1-ҙә ярҙам итмәй.

  - `TransparentOnly`: ғәҙәттәгесә режим; тик үтә күренмәле күрһәтмәләр (`MintAsset`, `TransferAsset`, һ.б.) рөхсәт ителә һәм экранланған операциялар кире ҡағыла.
  - `ShieldedOnly`: бөтә эмиссия һәм күсермәләр конфиденциаль күрһәтмәләрҙе ҡулланырға тейеш; `RevealConfidential` тыйыла, шуға күрә баланстар бер ҡасан да асыҡтан-асыҡ өҫкә сыға.
  - `Convertible`: тотоусылар үтә күренмәле һәм экранланған күрһәтеүҙәр араһында ҡиммәтте күсерергә мөмкин, түбәндәге/рампа инструкцияларын ҡулланып.
- Сәйәсәттәре сикләнгән ФСМ-ға эйәреп, ҡойроҡло аҡсаларҙы булдырмау өсөн:
  - `TransparentOnly → Convertible`X (тиҙ арала ҡалҡанлы бассейнды тиҙ арала асыу).
  - `TransparentOnly → ShieldedOnly` (күҙәтеү һәм конверсия тәҙрәһе талап ителә).
  - `Convertible → ShieldedOnly`X (көсләнгән минималь тотҡарлыҡ).

```bash
curl -s http://127.0.0.1:8180/metrics \
  | rg 'iroha_confidential_(tree_(commitments|depth)|root_history_entries|frontier_(checkpoints|last_checkpoint_height|last_checkpoint_commitments)|root_evictions_total|frontier_evictions_total){asset_id="4cuvDVPuLBKJyN6dPbRQhmLh68sU"}'
```

Был пар менән `rg 'iroha_confidential_tree_depth'` шул уҡ ҡырҡыуҙа раҫлау өсөн, тәрәнлек яңы йөкләмәләр менән үҫә, шул уҡ ваҡытта күсерелеү иҫәпләүселәре генә арта, ҡасан тарих ҡапҡастары өҫтәү. Был ҡиммәттәр Grafana экспорты менән теҙелешергә тейеш, һеҙ идара итеү дәлилдәре өйөмдәренә беркетергә тейеш.

#### Газ графигы телеметрия & иҫкәртмәләрМ2 фазаһы шулай уҡ ептәр конфигурацияланған газ множитель телеметрия торбаһына, шулай итеп, операторҙар иҫбатлай ала, һәр валидатор бер үк тикшерелгән сығымдарҙы бүлешә раҫлау алдынан релиз:

- `iroha_confidential_gas_base_verify` көҙгө `confidential.gas.proof_base` ( `250_000` ғәҙәттәгесә).
- `iroha_confidential_gas_per_public_input`, `iroha_confidential_gas_per_proof_byte`, `iroha_confidential_gas_per_nullifier`, һәм `iroha_confidential_gas_per_commitment` Prometheus-тағы тейешле ручкаларын көҙгөләй. Ҡиммәттәр яңыртыу стартапта һәм ҡасан ғына конфигурациялы эҫе-перегрузкалар; `irohad` (`crates/irohad/src/main.rs:1591,1642`) әүҙем графикты `Telemetry::set_confidential_gas_schedule` аша этәрә.

Ҡырҡып, ручкалар тиҫтерҙәре буйынса бер үк булыуын раҫлау өсөн, ручкаларҙы ҡырҡып ташларға:

```bash
# compare active multipliers across validators
for host in validator-a validator-b validator-c; do
  curl -s "http://$host:8180/metrics" \
    | rg 'iroha_confidential_gas_(base_verify|per_public_input|per_proof_byte|per_nullifier|per_commitment)'
done
```

Grafana приборҙар таҡтаһы `confidential_assets.json` хәҙер үҙ эсенә ала “Газ график” панелендә, биш датчикты күрһәтә һәм дивергенцияны айырып күрһәтә. Иҫкәртергә идара итә `dashboards/alerts/confidential_assets_rules.yml` тышлығы:
- `ConfidentialGasMismatch`: тикшерә макс/мин һәр множитель бөтә ҡырҡып маҡсаттары һәм биттәр буйынса, ҡасан ниндәй ҙә булһа диверж өсөн 3минуттан ашыу, операторҙарҙы тура килтерергә этәрә `confidential.gas` аша эҫе-перемок йәки үҙгәртеп ҡороу.
- `ConfidentialGasTelemetryMissing`: иҫкәртә, ҡасан Prometheus 5 минут өсөн биш множитель бер ниндәй ҙә ҡырҡып ала алмай, был юғалған ҡырҡыу маҡсатлы йәки инвалид телеметрия күрһәтә.

Түбәндәге PromQL ҡулайлы тотоу өсөн шылтыратыуҙар буйынса тикшерелгән:

```promql
# ensure every multiplier matches across validators (uses the same projection as the alert)
(max without(instance, job) (iroha_confidential_gas_per_public_input)
  - min without(instance, job) (iroha_confidential_gas_per_public_input)) == 0
```

Тапыу контролдә тотолған конфигурациянан тыш нуль булып ҡалырға тейеш. Газ таблицаһын үҙгәрткәндә, ҡырҡыу алдынан/һуң ҡырҡыуҙар, уларҙы үҙгәртеү тураһында запросҡа беркетергә һәм яңы множитель менән `docs/source/confidential_assets_calibration.md`-ны яңыртыу, шулай итеп, идара итеү рецензенттары телеметрия дәлилдәрен калибровка отчетына бәйләй ала.

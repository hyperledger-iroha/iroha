# Հենանիշային հաշվետվություն

Յուրաքանչյուր վազքի մանրամասն նկարներ և FASTPQ WP5-B պատմությունը ապրում են
[`benchmarks/history.md`](benchmarks/history.md); կցելու ժամանակ օգտագործեք այդ ցուցանիշը
ճանապարհային քարտեզի վերանայման կամ SRE աուդիտի արտեֆակտներ: Վերականգնել այն
`python3 scripts/fastpq/update_benchmark_history.py` ամեն անգամ, երբ նոր GPU գրավում է
կամ Պոսեյդոնը դրսևորում է հողը:

## Արագացման ապացույցների փաթեթ

Յուրաքանչյուր GPU կամ խառը ռեժիմի չափանիշ պետք է ներառի կիրառվող արագացման կարգավորումները
այնպես որ WP6-B/WP6-C-ն կարող է ապացուցել կազմաձևման հավասարությունը ժամանակի արտեֆակտների հետ մեկտեղ:

- Լուսանկարեք գործարկման ժամանակի նկարը յուրաքանչյուր վազքից առաջ/հետո.
  `cargo xtask acceleration-state --format json > artifacts/acceleration_state_<stamp>.json`
  (օգտագործեք `--format table` մարդու կողմից ընթեռնելի տեղեկամատյանների համար): Սա գրանցում է `enable_{metal,cuda}`,
  Merkle-ի շեմերը, SHA-2 պրոցեսորի կողմնակալության սահմանները, հայտնաբերված հետևի առողջության բիթերը և ցանկացած
  կպչուն հավասարության սխալներ կամ անջատելու պատճառները:
- Պահեք JSON-ը փաթաթված չափանիշի ելքի կողքին
  (`artifacts/fastpq_benchmarks/*.json`, `benchmarks/poseidon/*.json`, Մերկլի ավլում
  նկարահանումներ և այլն), որպեսզի վերանայողները կարողանան միասին տարբերել ժամանակներն ու կազմաձևերը:
- Կոճակի սահմանումները և լռելյայնները գործում են `docs/source/config/acceleration.md`-ում; երբ
  կիրառվում են վերագրանցումներ (օրինակ՝ `ACCEL_MERKLE_MIN_LEAVES_GPU`, `ACCEL_ENABLE_CUDA`),
  նշեք դրանք գործարկման մետատվյալներում, որպեսզի կրկնությունները վերարտադրելի լինեն սերվերներում:

## Norito փուլ-1 հենանիշ (WP5-B/C)

- Հրաման՝ `cargo xtask stage1-bench [--size <bytes|Nk|Nm>]... [--iterations <n>]`
  արտանետում է JSON + Markdown `benchmarks/norito_stage1/` տակ՝ յուրաքանչյուր չափի ժամանակաչափերով
  սկալյար ընդդեմ արագացված կառուցվածքային ինդեքսի ստեղծողի համար:
- Վերջին գործարկումները (macOS aarch64, մշակողի պրոֆիլ) ապրում են
  `benchmarks/norito_stage1/latest.{json,md}` և թարմ անջատիչ CSV-ից
  `examples/stage1_cutover` (`benchmarks/norito_stage1/cutover.csv`) ցույց է տալիս SIMD
  հաղթում է ~ 6–8 ԿԲ-ից սկսած: GPU/զուգահեռ Stage-1-ն այժմ կանխադրված է **192 KiB**
  անջատում (`NORITO_STAGE1_GPU_MIN_BYTES=<n>`՝ անտեսելու համար)՝ գործարկման թրաշից խուսափելու համար
  փոքր փաստաթղթերի վրա՝ միաժամանակ արագացուցիչներին հնարավորություն տալով ավելի մեծ բեռների համար:

## Enum vs Trait Object Dispatch

- Կազմելու ժամանակը (վրիպազերծման կառուցում)՝ 16.58 վրկ
- Runtime (չափանիշ, ավելի ցածր, ավելի լավ).
  - `enum`՝ 386 ps (միջին)
  - `trait_object`՝ 1,56 նս (միջին)

Այս չափումները բխում են միկրոհենանիշից, որը համեմատում է թվի վրա հիմնված դիսպետչերը արկղային հատկանիշի օբյեկտի իրականացման հետ:

## Poseidon CUDA փաթեթավորում

Poseidon-ի չափանիշը (`crates/ivm/benches/bench_poseidon.rs`) այժմ ներառում է աշխատանքային ծանրաբեռնվածություն, որն իրականացնում է ինչպես մեկ հեշի փոխարկումներ, այնպես էլ նոր խմբաքանակով օգնականներ: Գործարկել հավաքակազմը հետևյալով.

```bash
cargo bench -p ivm bench_poseidon -- --save-baseline poseidon_cuda
```

Չափանիշը արդյունքները կգրանցի `target/criterion/poseidon*_many`-ի ներքո: Երբ հասանելի է GPU-ի աշխատողը, արտահանեք JSON ամփոփագրերը (օրինակ՝ պատճենեք `target/criterion/**/new/benchmark.json`-ը `benchmarks/poseidon/criterion_poseidon2_many_cuda.json`-ի մեջ) (օրինակ՝ պատճենեք `target/criterion/**/new/benchmark.json`-ը `benchmarks/poseidon/`-ի մեջ), որպեսզի ներքևում գտնվող թիմերը կարողանան համեմատել CPU-ի յուրաքանչյուր չափսերը baCUDA v. Քանի դեռ հատուկ GPU գիծը չի գործի, հենանիշը վերադառնում է SIMD/CPU-ի ներդրմանը և դեռևս ապահովում է ռեգրեսիայի օգտակար տվյալներ խմբաքանակի կատարման համար:

Կրկնվող նկարահանումների համար (և ժամանակային տվյալների հետ հավասարության ապացույցներ պահպանելու համար) գործարկեք

```bash
cargo xtask poseidon-cuda-bench --json-out benchmarks/poseidon/poseidon_cuda_latest.json \
  --markdown-out benchmarks/poseidon/poseidon_cuda_latest.md --allow-overwrite
```որը սերմանում է դետերմինիստական Poseidon2/6 խմբաքանակները, գրանցում է CUDA-ի առողջական/անջատման պատճառները, ստուգում
հավասարություն սկալյար ուղու նկատմամբ, և մետաղի հետ մեկտեղ թողարկում է ops/sec + արագացման ամփոփագրեր
գործարկման ժամանակի կարգավիճակը (հատկանիշի դրոշակ, հասանելիություն, վերջին սխալ): Միայն պրոցեսորով աշխատող հոսթերը դեռ գրում են սկալերը
հղում և նշեք բացակայող արագացուցիչը, այնպես որ CI-ն կարող է հրապարակել արտեֆակտներ նույնիսկ առանց GPU-ի
վազորդ.

## FASTPQ մետաղական հենանիշ (Apple Silicon)

GPU-ի գիծը գրանցեց `fastpq_metal_bench`-ի թարմացված վերջից մինչև վերջ գործարկումը macOS 14-ում (arm64)՝ գծի հավասարակշռված պարամետրերի հավաքածուով, 20,000 տրամաբանական տողերով (լրացված 32,768-ով) և 16 սյունակների խմբերով: Փաթաթված արտեֆակտը ապրում է `artifacts/fastpq_benchmarks/fastpq_metal_bench_20k_refresh.json`-ում, իսկ մետաղական հետքը պահվում է նախորդ նկարների հետ մեկտեղ՝ `traces/fastpq_metal_trace_*_rows20000_iter5.trace`-ում: Միջին ժամկետները (`benchmarks.operations[*]`-ից) այժմ կարդացեք.

| Շահագործում | CPU միջին (ms) | Մետաղական միջին (ms) | Արագացում (x) |
|----------------------------|-----------------|-------------|
| FFT (32,768 մուտք) | 83.29 | 79,95 | 1.04 |
| IFFT (32,768 մուտք) | 93,90 | 78.61 | 1.20 |
| LDE (262,144 մուտք) | 669.54 | 657.67 | 1.02 |
| Պոսեյդոնի հեշ սյունակներ (524288 մուտքեր) | 29,087.53 | 30,004.90 | 0,97 |

Դիտարկումներ:

- FFT/IFFT երկուսն էլ օգտվում են թարմացված BN254 միջուկներից (IFFT-ն մաքրում է նախորդ ռեգրեսիան ~20%-ով):
- LDE-ն մնում է հավասարության մոտ; զրոյական լրացումն այժմ գրանցում է 33,554,432 ներդիր բայթ՝ 18,66 մս միջին չափով, այնպես որ JSON փաթեթը գրավում է հերթի ազդեցությունը:
- Poseidon hashing-ը դեռևս կապված է CPU-ի հետ այս սարքաշարի վրա; շարունակեք համեմատել Պոսեյդոնի միկրոբենչային դրսևորումների հետ, մինչև մետաղական ուղին չընդունի հերթի վերջին հսկիչները:
- Յուրաքանչյուր գրավում այժմ գրանցում է `AccelerationSettings.runtimeState().metal.lastError`, թույլ տալով
  ինժեներները նշում են CPU-ի հետադարձ կապերը հատուկ անջատման պատճառով (քաղաքականության անջատում,
  հավասարության ձախողում, սարք չկա) ուղղակիորեն հենանիշի արտեֆակտում:

Գործարկումը վերարտադրելու համար կառուցեք մետաղական միջուկները և կատարեք.

```bash
FASTPQ_METAL_LIB=target/release/build/fastpq_prover-*/out/fastpq.metallib \
FASTPQ_METAL_TRACE_CHILD=1 \
cargo run -p fastpq_prover --features fastpq-gpu --bin fastpq_metal_bench --release \
  -- --rows 20000 --iterations 5 --output fastpq_metal_bench_20k.json
```

Ստացված JSON-ը կատարեք `artifacts/fastpq_benchmarks/`-ի ներքո՝ մետաղական հետքի հետ միասին, որպեսզի դետերմինիզմի ապացույցները մնան վերարտադրելի:

## FASTPQ CUDA ավտոմատացում

CUDA հյուրընկալողները կարող են գործարկել և փաթեթավորել SM80 հենանիշը մեկ քայլով հետևյալով.

```bash
cargo xtask fastpq-cuda-suite \
  --rows 20000 --iterations 5 --columns 16 \
  --row-usage artifacts/fastpq_benchmarks/fastpq_row_usage_2025-05-12.json \
  --label device_class=xeon-rtx --device rtx-ada
```

Օգնականը կանչում է `fastpq_cuda_bench`-ը, անցնում պիտակների/սարքի/նոտաների միջով, հարգում
`--require-gpu`, և (ըստ լռելյայն) փաթաթում/նշանավորում է `scripts/fastpq/wrap_benchmark.py`-ի միջոցով:
Արդյունքները ներառում են չմշակված JSON-ը, `artifacts/fastpq_benchmarks/`-ի տակ փաթաթված փաթեթը,
և `<name>_plan.json` ելքի կողքին, որը ձայնագրում է ճշգրիտ հրամանները/env այսպես
7-րդ փուլի նկարահանումները մնում են վերարտադրելի GPU-ի վազորդների համար: Ավելացնել `--sign-output` և
`--gpg-key <id>`, երբ անհրաժեշտ է ստորագրություն; օգտագործեք `--dry-run` միայն արտանետելու համար
պլան/երթուղիներ՝ առանց նստարանն իրականացնելու:

### GA թողարկման գրավում (macOS 14 arm64, գծի հավասարակշռված)

WP2-D-ին բավարարելու համար մենք նաև ձայնագրել ենք թողարկման կառուցում նույն հոսթի վրա GA-ready-ով
հերթի էվրիստիկա և հրապարակեց այն որպես
`fastpq_metal_bench_20k_release_macos14_arm64.json`. Արտեֆակտը գրավում է երկուսը
սյունակների խմբաքանակներ (հավասարակշռված գոտիով, լրացված 32768 տողով) և ներառում է Պոսեյդոն
microbench նմուշներ վահանակի սպառման համար:| Շահագործում | CPU միջին (ms) | Մետաղական միջին (ms) | Արագացում | Ծանոթագրություններ |

# Обзор Nexus и контекст для операторов

**Ссылка на дорожную карту:** NX-14 — документация Nexus и операторские runbooks  
**Статус:** Черновик 2026-03-24 (в паре с `docs/source/nexus_operations.md`)  
**Аудитория:** руководители программ, инженеры эксплуатации и команды партнеров, которым
нужен одностраничный обзор архитектуры Sora Nexus (Iroha 3) перед тем, как переходить
к подробным спецификациям (`docs/source/nexus.md`, `docs/source/nexus_lanes.md`,
`docs/source/nexus_transition_notes.md`).

## 1. Линии релизов и общий инструментарий

- **Iroha 2** остается самостоятельным треком для консорциумных развертываний.
- **Iroha 3 / Sora Nexus** вводит многополосное исполнение, пространства данных и общую модель управления.
  Тот же репозиторий, toolchain и CI пайплайны собирают обе линии релизов, поэтому исправления
  в IVM, компиляторе Kotodama или SDK автоматически применяются к Nexus.
- **Артефакты:** `iroha3-<version>-<os>.tar.zst` бандлы и OCI образы содержат бинарники,
  примерные конфиги и метаданные профиля Nexus. Операторы используют
  `docs/source/sora_nexus_operator_onboarding.md` для сквозного процесса валидации артефактов.
- **Общая поверхность SDK:** SDK для Rust, Python, JS/TS, Swift и Android используют
  те же схемы Norito и fixtures адресов (`fixtures/account/address_vectors.json`), чтобы
  кошельки и автоматизация могли переключаться между сетями Iroha 2 и Nexus без
  разветвления форматов.

## 2. Архитектурные блоки

| Компонент | Описание | Ключевые ссылки |
|-----------|----------|-----------------|
| **Data Space (DS)** | Домен исполнения, задаваемый управлением, который определяет состав валидаторов, класс приватности, политику комиссий и профиль доступности данных. Каждый DS владеет одной или несколькими lanes. | `docs/source/nexus.md`, `docs/source/nexus_transition_notes.md` |
| **Lane** | Детерминированный шардинг исполнения и состояния. Манифесты lane объявляют наборы валидаторов, хуки settlement, метаданные телеметрии и права маршрутизации. Глобальное кольцо консенсуса упорядочивает commitments lane. | `docs/source/nexus_lanes.md` |
| **Space Directory** | Реестровый контракт (и помощники CLI), который хранит манифесты DS, ротации валидаторов и выдачу прав. Сохраняет подписанную историю, чтобы аудиторы могли восстановить состояние. | `docs/source/nexus.md#space-directory` |
| **Lane Catalog** | Раздел конфигурации (`[nexus]` в `config.toml`), который сопоставляет ID lane с алиасами, политиками маршрутизации и параметрами хранения. Операторы могут просмотреть эффективный каталог через `irohad --sora --config ... --trace-config`. | `docs/source/sora_nexus_operator_onboarding.md` |
| **Settlement Router** | Маршрутизирует перемещения XOR между lanes (например, приватные CBDC lanes <-> публичные lanes ликвидности). Политики по умолчанию описаны в `docs/source/cbdc_lane_playbook.md`. | `docs/source/cbdc_lane_playbook.md` |
| **Telemetry & SLOs** | Дашборды и правила оповещений в `dashboards/grafana/nexus_*.json` фиксируют высоту lanes, backlog DA, задержку settlement и глубину очередей управления. План ремедиации отслеживается в `docs/source/nexus_telemetry_remediation_plan.md`. | `dashboards/grafana/nexus_lanes.json`, `dashboards/alerts/nexus_audit_rules.yml` |

### Классы lane и data space

- `default_public` lanes закрепляют полностью публичные нагрузки под парламентом Sora.
- `public_custom` lanes позволяют программные экономики, оставаясь прозрачными.
- `private_permissioned` lanes обслуживают CBDC или консорциумные приложения; экспортируют только commitments и proofs.
- `hybrid_confidential` lanes сочетают доказательства нулевого разглашения с выборочным раскрытием.

Каждый lane объявляет:

1. **Манифест lane:** метаданные, одобренные управлением и отслеживаемые в Space Directory.
2. **Политика доступности данных:** параметры erasure coding, хуки восстановления и требования аудита.
3. **Профиль телеметрии:** дашборды и on-call runbooks, которые нужно обновлять при каждом изменении управления.

## 3. Снимок графика развертывания

| Фаза | Фокус | Критерии выхода |
|------|-------|-----------------|
| **N0 - Closed beta** | Регистратор под управлением совета, только namespace `.sora`, ручной онбординг операторов. | Манифесты DS подписаны, каталог lanes статичен, репетиции управления задокументированы. |
| **N1 - Public launch** | Добавляет суффиксы `.nexus`, аукционы и самообслуживаемый регистратор. Settlement подключаются к казне XOR. | Тесты синхронизации resolver/gateway зеленые, дашборды сверки биллинга в строю, учение по спорам завершено. |
| **N2 - Expansion** | Включает `.dao`, API реселлеров, аналитику, портал споров, scorecards стюардов. | Артефакты соответствия версионированы, toolkit policy-jury запущен, отчеты прозрачности казны опубликованы. |
| **NX-12/13/14 gate** | Движок соответствия, дашборды телеметрии и документация должны выйти вместе до открытия партнерского пилота. | `docs/source/nexus_overview.md` + `docs/source/nexus_operations.md` опубликованы, дашборды подключены к оповещениям, движок политики связан с управлением. |

## 4. Ответственность оператора

| Ответственность | Описание | Доказательство |
|----------------|----------|---------------|
| Гигиена конфигурации | Держать `config/config.toml` синхронизированным с опубликованным каталогом lanes и dataspaces; фиксировать изменения в тикетах. | Вывод `irohad --sora --config ... --trace-config` архивируется вместе с артефактами релиза. |
| Отслеживание манифестов | Следить за обновлениями Space Directory и обновлять локальные кэши/allowlists. | Подписанный bundle манифестов сохранен в тикете онбординга. |
| Покрытие телеметрии | Убедиться, что дашборды из Раздела 2 доступны, оповещения подключены к PagerDuty, а квартальные обзоры задокументированы. | Протокол дежурной проверки + экспорт Alertmanager. |
| Отчет по инцидентам | Следовать матрице серьезности из `docs/source/nexus_operations.md` и отправлять post-incident отчет в течение пяти рабочих дней. | Шаблон post-incident сохранен по ID инцидента. |
| Готовность управления | Участвовать в голосованиях совета Nexus, когда изменения политики lanes затрагивают развертывание; репетировать инструкции rollback ежеквартально. | Посещаемость совета + чеклист репетиций в `docs/source/project_tracker/nexus_config_deltas/`. |

# IVM хийн загвар

Энэхүү баримт бичиг нь Iroha виртуал машины каноник хийн хуваарийг тодорхойлдог
(IVM) ба хуваарийг хэрхэн хэш хийж, хэрхэн хэрэгжүүлж байгааг тайлбарлана. Үнэний эх сурвалж
зардлын хувьд `crates/ivm/src/gas.rs`; доорх хуваарийн хүснэгтийг үзүүлэв
тэр каноник зураглалыг харах.

## Хамрах хүрээ

- IVM байт кодын гүйцэтгэлд хамаарна (Гүйцэтгэх боломжтой::Ivm).
- ISI хийн тоолуурыг `crates/iroha_core/src/gas.rs`-д тусад нь тодорхойлсон.
- ISO 20022 үйлдлийн кодууд нь ABI v1 хувилбарт хадгалагдсан бөгөөд хараахан хий оруулаагүй байна.

## Детерминизм ба хуваарийн хэш

Хийн хуваарь нь тодорхойлогддог бөгөөд opcode → зардлын хосуудаас гаралтай. The
Каноник тоймыг оруулга болгонд дараалсан үйлдлийн кодын жагсаалтаар тооцдог
Дараах байдлаар цуварсан:

- үйлдлийн кодын байт (u8)
- зардал (u64, жижиг-эндиан)

Хэш нь дараах байдлаар илэрдэг.

- `ivm::gas::schedule_hash()` (канон хуваарийн хэш)
- `ivm::limits::schedule_hash()` (хосттой харьцах бусад нэр)

Бүх үе тэнгийнхэн утас залгахдаа ижил хуваарийг хуваалцаж байгаа эсэхийг шалгахын тулд энэхүү мэдээг ашиглана уу
тохиргоо эсвэл телеметрийн шалгалт.

## Векторын масштаб болон HTM дахин оролдлого

- Вектор үйлдлийн (`VADD*`, `VAND`, `VXOR`, `VOR`, `VROT32`) логик бүхий масштаб
  векторын уртыг `SETVL` тохируулсан. Хүснэгтэнд байгаа үндсэн зардлыг хуваарь болгон хуваасан
  `min(vector_len, VECTOR_BASE_LANES) / VECTOR_BASE_LANES` (суурь = 2 эгнээ).
- HTM дахин оролдлого нь зардлыг `(retries + 1)`-ээр үржүүлдэг; Ихэнх зөвшилцлийн замууд тийм биш юм
  дахин оролдлого хийх.

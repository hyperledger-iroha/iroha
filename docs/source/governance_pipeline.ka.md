# მიმდინარე მდგომარეობა (v1)
- მმართველობის წინადადებები წარმოდგენილია შემდეგნაირად: შემოთავაზებული → რეფერენდუმი → ანგარიში → ამოქმედება. რეფერენდუმის ფანჯრები და კენჭისყრის/დამტკიცების ზღურბლები დაცულია, როგორც აღწერილია `gov.md`-ში; საკეტები მხოლოდ გახანგრძლივებულია და იხსნება ვადის გასვლისას.
- პარლამენტის შერჩევა იყენებს VRF-ზე დაფუძნებულ გათამაშებებს დეტერმინისტული თანმიმდევრობით და ვადების საზღვრებით; როდესაც არ არსებობს მუდმივი სია, Torii იღებს სარეზერვო ასლს `gov.parliament_*` კონფიგურაციის გამოყენებით. საბჭოს კარიბჭე და კვორუმის შემოწმება ტარდება `gov_parliament_bodies` / `gov_pipeline_sla` ტესტებში.
- ხმის მიცემის რეჟიმები: ZK (ნაგულისხმევი, მოითხოვს `Active` VK inline ბაიტით) და Plain (კვადრატული წონა). რეჟიმის შეუსაბამობა უარყოფილია; საკეტის შექმნა/გაფართოება მონოტონურია ორივე რეჟიმში რეგრესიის ტესტებით ZK-სთვის და უბრალო ხელახალი ხმის მიცემით.
- ვალიდიატორის არასწორ ქმედებებზე მოქმედებენ მტკიცებულების მილსადენის მეშვეობით (`/v1/sumeragi/evidence*`, CLI დამხმარეები) ერთობლივი კონსენსუსის გადაცემით, რომელიც აღსრულებულია `NextMode` + `ModeActivationHeight`-ით.
- დაცული სახელების სივრცეები, გაშვების დროის განახლების კაუჭები და მმართველობის მანიფესტის დაშვება დოკუმენტირებულია `governance_api.md`-ში და დაფარულია ტელემეტრიით (`governance_manifest_*`, `governance_protected_namespace_total`).

# ფრენის დროს / ჩამორჩენილი
- გამოაქვეყნეთ VRF ნახატების არტეფაქტები (დათესილი, მტკიცებულება, შეკვეთილი სია, ალტერნატივა) და დაშიფრეთ ჩანაცვლების წესები გამოუცხადებლობისთვის; დაამატეთ ოქროს მოწყობილობები გათამაშებისა და ჩანაცვლებისთვის.
- სცენა-SLA აღსრულებას პარლამენტის ორგანოებისთვის (წესები → დღის წესრიგი → შესწავლა → განხილვა → ნაფიც მსაჯულები → ამოქმედება) საჭიროებს მკაფიო ქრონომეტრებს, ესკალაციის ბილიკებს და ტელემეტრიის მრიცხველებს.
- ნაფიც მსაჯულთა პოლიტიკის საიდუმლო/დავალება-გამომჟღავნება კენჭისყრა და მასთან დაკავშირებული მოსყიდვის წინააღმდეგობის აუდიტი ჯერ კიდევ უნდა განხორციელდეს.
- როლური ობლიგაციების მულტიპლიკატორები, არასწორი ქცევის შემცირება მაღალი რისკის მქონე ორგანოებისთვის და დასვენება სერვისის სლოტებს შორის საჭიროებს კონფიგურაციის სანტექნიკას და ტესტებს.
- მართვის ზოლის დალუქვა და რეფერენდუმის ფანჯრის/გამოსვლის ჭიშკრები თვალყურს ადევნებს `gov.md`/`status.md`-ში; განაახლეთ საგზაო რუქის ჩანაწერები, რადგან დარჩენილი მისაღები ტესტები მიდის.

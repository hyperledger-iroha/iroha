# Посейдон металының ортақ тұрақтылары

Металл ядролары, CUDA ядролары, Rust провері және әрбір SDK құрылғысы ортақ болуы керек
аппараттық құралдарды жеделдету үшін дәл сол Poseidon2 параметрлері
хэштеу детерминистикалық. Бұл құжат канондық суретті, қалай жасау керектігін жазады
оны қалпына келтіреді және GPU құбырлары деректерді қалай қабылдайды деп күтілуде.

## Сурет манифесті

Параметрлер `PoseidonSnapshot` RON құжаты ретінде жарияланған. Көшірмелер
нұсқа бақылауында сақталады, сондықтан GPU құралдар тізбегі мен SDK құрастыру уақытына сенбейді
кодты генерациялау.

| Жол | Мақсаты | SHA-256 |
|------|---------|---------|
| `artifacts/offline_poseidon/constants.ron` | `fastpq_isi::poseidon::{ROUND_CONSTANTS, MDS}` арқылы жасалған канондық сурет; GPU құрастырулары үшін ақиқат көзі. | `99bef7760fcc80c2d4c47e720cf28a156f106a0fa389f2be55a34493a0ca4c21` |
| `IrohaSwift/Fixtures/offline_poseidon/constants.ron` | Swift құрылғысының сынақтары мен XCFramework түтін қондырғысы Металл ядролары күткен тұрақты мәндерді жүктейтіндей канондық суретті айналайды. | `99bef7760fcc80c2d4c47e720cf28a156f106a0fa389f2be55a34493a0ca4c21` |
| `java/iroha_android/src/test/resources/offline_poseidon/constants.ron` | Android/Kotlin құрылғылары паритет пен сериялау сынақтары үшін бірдей манифестпен бөліседі. | `99bef7760fcc80c2d4c47e720cf28a156f106a0fa389f2be55a34493a0ca4c21` |

Әрбір тұтынушы тұрақты мәндерді графикалық процессорға жалғамас бұрын хэшті тексеруі керек
құбыр. Манифест өзгерген кезде (жаңа параметрлер жинағы немесе профиль), SHA және
төменгі ағынды айналар құлыптау қадамымен жаңартылуы керек.

## Регенерация

Манифест Rust көздерінен `xtask` іске қосу арқылы жасалады.
көмекші. Пәрмен канондық файлды да, SDK айналарын да жазады:

```bash
cargo xtask offline-poseidon-fixtures --tag iroha.offline.receipt.merkle.v1
```

Тағайындалған жерлерді қайта анықтау үшін `--constants <path>`/`--vectors <path>` пайдаланыңыз немесе
`--no-sdk-mirror` тек канондық суретті қалпына келтіру кезінде. Көмекші болады
жалауша көрсетілмеген кезде артефактілерді Swift және Android ағаштарына шағылыстыру,
ол хэштерді CI үшін туралайды.

## Металл/CUDA конструкцияларын беру

- `crates/fastpq_prover/metal/kernels/poseidon2.metal` және
  `crates/fastpq_prover/cuda/fastpq_cuda.cu` қалпына келтіру керек
  кесте өзгерген сайын көрсетіледі.
- Дөңгелектелген және MDS константалары сабақтас `MTLBuffer`/`__constant` күйіне орналастырылған.
  манифест орналасуына сәйкес сегменттер: `round_constants[round][state_width]`
  одан кейін 3x3 MDS матрицасы.
- `fastpq_prover::poseidon_manifest()` суретті жүктейді және растайды
  жұмыс уақыты (металды қыздыру кезінде), сондықтан диагностикалық құрал мынаны растай алады
  шейдер тұрақтылары арқылы жарияланған хэшке сәйкес келеді
  `fastpq_prover::poseidon_manifest_sha256()`.
- SDK құрылғысын оқу құралдары (Swift `PoseidonSnapshot`, Android `PoseidonSnapshot`) және
  Norito офлайн құралы бірдей манифестке сүйенеді, бұл тек GPU-ге жол бермейді
  параметр шанышқылары.

## Валидация

1. Манифестті қалпына келтіргеннен кейін, `cargo test -p xtask` іске қосыңыз
   Посейдон арматурасын генерациялау қондырғысының сынақтары.
2. Жаңа SHA-256 құрылғысын осы құжатқа және бақылайтын кез келген бақылау тақталарына жазыңыз
   GPU артефактілері.
3. `cargo test -p fastpq_prover poseidon_manifest_consistency` талдаулар
   `poseidon2.metal` және `fastpq_cuda.cu` құрастыру уақытында және олардың
   Сериялық константалар CUDA/Metal кестелерін сақтай отырып, манифестке сәйкес келеді және
   құлыптау қадамындағы канондық сурет.Манифестті GPU құрастыру нұсқауларымен бірге сақтау Metal/CUDA береді
жұмыс ағындары детерминирленген қол алысу: ядролар жадты оңтайландыру үшін еркін
олар ортақ константаларды жұтып, хэшті ашқанша орналасады
паритеттерді тексеруге арналған телеметрия.

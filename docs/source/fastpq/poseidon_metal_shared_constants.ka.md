# Poseidon Metal Shared Constants

ლითონის ბირთვები, CUDA ბირთვები, Rust prover და ყველა SDK მოწყობილობა უნდა იყოს გაზიარებული
ზუსტად იგივე Poseidon2 პარამეტრები, რათა შევინარჩუნოთ ტექნიკის დაჩქარება
ჰეშირება დეტერმინისტული. ეს დოკუმენტი აღწერს კანონიკურ კადრს, როგორ
მისი რეგენერაცია და როგორ არის მოსალოდნელი GPU მილსადენების მიერ მონაცემების მიღება.

ინახება ვერსიის კონტროლის ქვეშ, ასე რომ GPU ინსტრუმენტთა ჯაჭვები და SDK არ დაეყრდნონ აშენების დროს
კოდის გენერირება.

მილსადენი. როდესაც მანიფესტი იცვლება (ახალი პარამეტრის ნაკრები ან პროფილი), SHA და
ქვედა დინების სარკეები უნდა განახლდეს დაბლოკვის საფეხურზე.

## რეგენერაცია

მანიფესტი გენერირებულია Rust-ის წყაროებიდან `xtask`-ის გაშვებით
დამხმარე. ბრძანება წერს როგორც კანონიკურ ფაილს, ასევე SDK სარკეებს:

```bash
cargo xtask offline-poseidon-fixtures --tag iroha.offline.receipt.merkle.v1
```

გამოიყენეთ `--constants <path>`/`--vectors <path>` დანიშნულების ადგილის გადასალახად ან
`--no-sdk-mirror` მხოლოდ კანონიკური სნეპშოტის რეგენერაციისას. დამხმარე იქნება
ასახეთ არტეფაქტები Swift და Android ხეებში, როდესაც დროშა გამოტოვებულია,
რომელიც ინარჩუნებს ჰეშებს CI-სთვის გასწორებულად.

## კვების ლითონის/CUDA კონსტრუქციები

- `crates/fastpq_prover/metal/kernels/poseidon2.metal` და
  `crates/fastpq_prover/cuda/fastpq_cuda.cu` უნდა იყოს რეგენერირებული
  მანიფესტი ყოველთვის, როცა ცხრილი იცვლება.
- დამრგვალებული და MDS მუდმივები დადგმულია მიმდებარე `MTLBuffer`/`__constant`-ში
  სეგმენტები, რომლებიც ემთხვევა მანიფესტის განლაგებას: `round_constants[round][state_width]`
  მოჰყვება 3x3 MDS მატრიცა.
- `fastpq_prover::poseidon_manifest()` ატვირთავს და ამოწმებს სნეპშოტს
  გაშვების დრო (მეტალის გახურების დროს) ასე რომ, დიაგნოსტიკური ხელსაწყოები ამტკიცებენ, რომ
  shader მუდმივები ემთხვევა გამოქვეყნებულ ჰეშის მეშვეობით
  `fastpq_prover::poseidon_manifest_sha256()`.
- SDK მოწყობილობების წამკითხველები (Swift `PoseidonSnapshot`, Android `PoseidonSnapshot`) და
  Norito ოფლაინ ინსტრუმენტები ეყრდნობა იმავე მანიფესტს, რომელიც ხელს უშლის მხოლოდ GPU-ს
  პარამეტრის ჩანგლები.

## დადასტურება

1. მანიფესტის რეგენერაციის შემდეგ, გაუშვით `cargo test -p xtask` სავარჯიშოდ
   პოსეიდონის მოწყობილობების წარმოქმნის ერთეულის ტესტები.
2. ჩაწერეთ ახალი SHA-256 ამ დოკუმენტში და ნებისმიერ საინფორმაციო დაფაზე, რომელიც აკონტროლებს
   GPU არტეფაქტები.
3. `cargo test -p fastpq_prover poseidon_manifest_consistency` ანალიზები
   `poseidon2.metal` და `fastpq_cuda.cu` აშენების დროს და ამტკიცებს, რომ მათი
   სერიული მუდმივები ემთხვევა მანიფესტს, ინახავს CUDA/Metal ცხრილებს და
   კანონიკური სნეპშოტი დაბლოკვის საფეხურზე.მანიფესტის შენახვა GPU-ს აგების ინსტრუქციებთან ერთად იძლევა Metal/CUDA-ს
სამუშაო პროცესებს დეტერმინისტული ხელის ჩამორთმევა: ბირთვები თავისუფლად შეუძლიათ მეხსიერების ოპტიმიზაციას
განლაგება მანამ, სანამ ისინი შთანთქავენ საერთო მუდმივებს და ავლენენ ჰეშს
ტელემეტრია პარიტეტული შემოწმებისთვის.

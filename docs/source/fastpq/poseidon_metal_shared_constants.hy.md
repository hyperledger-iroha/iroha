# Poseidon Metal Shared Constants

Մետաղական միջուկները, CUDA միջուկները, Rust prover-ը և յուրաքանչյուր SDK սարքավորում պետք է կիսվեն
ճիշտ նույն Poseidon2 պարամետրերը, որպեսզի մնան ապարատային արագացված
hashing deterministic. Այս փաստաթուղթը գրանցում է կանոնական պատկերը, թե ինչպես
վերականգնել այն և ինչպես է ակնկալվում, որ GPU խողովակաշարերը կլանեն տվյալները:

## Snapshot Manifest

Պարամետրերը հրապարակվում են որպես `PoseidonSnapshot` RON փաստաթուղթ: Պատճեններն են
պահվում է տարբերակի հսկողության տակ, այնպես որ GPU գործիքների շղթաները և SDK-ները չեն ապավինում ստեղծման ժամանակին
կոդերի ստեղծում:

Յուրաքանչյուր սպառող պետք է ստուգի հեշը նախքան հաստատունները GPU-ի մեջ միացնելը
խողովակաշար. Երբ մանիֆեստը փոխվում է (նոր պարամետրերի հավաքածու կամ պրոֆիլ), SHA և
Հոսանքի ներքևի հայելիները պետք է թարմացվեն կողպեքով:

## Վերածնում

Մանիֆեստը ստեղծվում է Rust աղբյուրներից՝ գործարկելով `xtask`
օգնական. Հրամանը գրում է և՛ կանոնական ֆայլը, և՛ SDK հայելիները.

```bash
cargo xtask offline-poseidon-fixtures --tag iroha.offline.receipt.merkle.v1
```

Օգտագործեք `--constants <path>`/`--vectors <path>` ուղղությունները վերացնելու համար կամ
`--no-sdk-mirror` միայն կանոնական նկարը վերականգնելիս: Օգնականը կամենա
արտացոլել արտեֆակտները Swift և Android ծառերի մեջ, երբ դրոշը բաց է թողնվում,
որը պահում է հեշերը հավասարեցված CI-ի համար:

## Սնուցող մետաղական/CUDA կառուցումներ

- `crates/fastpq_prover/metal/kernels/poseidon2.metal` և
  `crates/fastpq_prover/cuda/fastpq_cuda.cu`-ը պետք է վերականգնվի
  դրսևորվում է, երբ աղյուսակը փոխվում է:
- Կլորացված և MDS հաստատունները բեմադրվում են իրար հաջորդող `MTLBuffer`/`__constant`
  մանիֆեստի դասավորությանը համապատասխանող հատվածներ՝ `round_constants[round][state_width]`
  որին հաջորդում է 3x3 MDS մատրիցը:
- `fastpq_prover::poseidon_manifest()`-ը բեռնում և վավերացնում է լուսանկարը
  գործարկման ժամանակ (մետաղի տաքացման ժամանակ), այնպես որ ախտորոշիչ գործիքավորումը կարող է պնդել, որ
  Shader հաստատունները համապատասխանում են հրապարակված հեշին
  `fastpq_prover::poseidon_manifest_sha256()`.
- SDK սարքերի ընթերցիչներ (Swift `PoseidonSnapshot`, Android `PoseidonSnapshot`) և
  Norito օֆլայն գործիքավորումը հիմնված է նույն մանիֆեստի վրա, որը կանխում է միայն GPU-ն
  պարամետրային պատառաքաղներ.

## Վավերացում

1. Մանիֆեստը վերականգնելուց հետո գործարկեք `cargo test -p xtask`՝ գործարկելու համար
   Պոսեյդոնի հարմարանքների արտադրության միավորի թեստեր.
2. Գրանցեք նոր SHA-256-ը այս փաստաթղթում և մոնիտորինգ իրականացնող ցանկացած վահանակում
   GPU արտեֆակտներ.
3. `cargo test -p fastpq_prover poseidon_manifest_consistency` վերլուծություններ
   `poseidon2.metal` և `fastpq_cuda.cu` կառուցման ժամանակ և պնդում են, որ իրենց
   սերիական հաստատունները համապատասխանում են մանիֆեստին՝ պահպանելով CUDA/Metal աղյուսակները և
   կանոնական պատկերը կողպեքի քայլով:Մանիֆեստը GPU-ի կառուցման հրահանգների կողքին պահելը տալիս է Metal/CUDA
աշխատանքային հոսքերը որոշիչ ձեռքսեղմում. միջուկներն ազատ են օպտիմիզացնելու իրենց հիշողությունը
դասավորությունը այնքան ժամանակ, քանի դեռ նրանք կլանում են ընդհանուր հաստատունները և բացահայտում են հեշը
Հեռաչափություն հավասարության ստուգման համար:

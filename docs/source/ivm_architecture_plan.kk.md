# IVM сәулет рефакторының жоспары

Бұл жоспар Iroha виртуалды машинасының пішінін өзгертудің қысқа мерзімді кезеңдерін қамтиды.
(IVM) қауіпсіздік пен өнімділік сипаттамаларын сақтай отырып, айқынырақ қабаттарға айналдырыңыз.
Ол жауапкершілікті оқшаулауға, хост интеграциясын қауіпсіз етуге және
Kotodama тіл стегін жеке жәшікке шығару үшін дайындау.

## Мақсаттар

1. **Қабатты орындалу уақытының қасбеті** – VM үшін нақты орындау уақытының интерфейсін енгізіңіз
   өзек тар белгінің артына ендірілген болуы мүмкін және балама фронт-ұштар дамуы мүмкін
   ішкі модульдерге қол тигізбестен.
2. **Хост/жүйе шекарасын күшейту** – жүйені шақыру арқылы жіберу маршруты
   кез келген хост алдында ABI саясатын және көрсеткішті тексеруді қамтамасыз ететін арнайы адаптер
   код орындалады.
3. **Тілді/құралдарды бөлу** – Kotodama арнайы кодын жаңа жәшікке жылжытыңыз және
   `ivm` ішінде тек байт кодының орындалу бетін сақтаңыз.
4. **Конфигурацияның біртұтастығы** – жеделдету мен мүмкіндікті ауыстырып-қосқыштарды бірдей етіп біріктіреді
   `iroha_config` арқылы басқарылады, өндірісте қоршаған ортаға негізделген тұтқаларды алып тастайды
   жолдар.

## Фазаның бөлінуі

### 1-кезең – Жұмыс уақытының қасбеті (орындалуда)
- Өмірлік циклді сипаттайтын `VmEngine` белгісін анықтайтын `runtime` модулін қосыңыз
  операциялар (`load_program`, `execute`, негізгі сантехника).
- `IVM` белгісін іске асыруға үйрету.  Бұл бар құрылымды сақтайды, бірақ рұқсат береді
  тұтынушылар (және болашақ сынақтар) бетонның орнына интерфейске тәуелді болады
  түрлері.
- `lib.rs` жүйесінен тікелей модуль реэкспорттарын өшіруді бастаңыз, осылайша қоңырау шалушылар арқылы импорттаңыз.
  мүмкіндігінше қасбет.

**Қауіпсіздік/өнімділік әсері**: қасбет ішкі бөлікке тікелей кіруді шектейді
мемлекет; тек қауіпсіз кіру нүктелері ашылады.  Бұл хостты тексеруді жеңілдетеді
газды немесе TLV өңдеуге қатысты өзара әрекеттесу және себеп.

### 2-кезең – Syscal диспетчері
- `IVMHost` орап, ABI талаптарын орындайтын `SyscallDispatcher` компонентін енгізу
  саясат пен көрсеткішті тексеру бір жерде бір рет.
- Жою арқылы диспетчерді пайдалану үшін әдепкі хост пен жалған хосттарды тасымалдаңыз
  қайталанатын валидация логикасы.
- Диспетчерді қосылатын етіп жасаңыз, сонда хосттар реттелетін құралдарды онсыз қамтамасыз ете алады
  қауіпсіздік тексерулерін айналып өту.
- `SyscallDispatcher::shared(...)` көмекшісін қамтамасыз етіңіз, осылайша клондалған VM-лер қайта жібере алады.
  әрбір жұмысшы ғимаратынсыз ортақ `Arc<Mutex<..>>` хост арқылы жүйелік қоңыраулар
  тапсырыс бойынша қаптамалар.

**Қауіпсіздік/өнімділік әсері**: орталықтандырылған шлюз хосттардан қорғайды
`is_syscall_allowed` қоңырау шалуды ұмытып, бұл көрсеткішті болашақта кэштеуге мүмкіндік береді
қайталанатын жүйелік қоңыраулар үшін тексерулер.

### 3-кезең – Kotodama экстракциясы
- Kotodama компиляторы `crates/kotodama_lang` (`crates/ivm/src/kotodama` ішінен) шығарылды.
- VM тұтынатын минималды байт код API қамтамасыз етіңіз (`compile_to_ivm_bytecode`).

**Қауіпсіздік/өнімділік әсері**: ажырату VM шабуыл бетін төмендетеді
негізгі және аудармашының регрессиясына қауіп төндірмей, тіл инновациясына мүмкіндік береді.### 4-кезең – Конфигурацияны біріктіру
- `iroha_config` алдын ала орнатулары (мысалы, GPU серверлерін қосу) арқылы ағынды жеделдету опциялары бар ортаны қайта анықтауды (`IVM_DISABLE_CUDA`, `IVM_DISABLE_METAL`) орындау уақытын өшіру қосқыштары ретінде сақтай отырып.
- хосттар таңдауы үшін `RuntimeConfig` нысанын жаңа қасбет арқылы көрсетіңіз.
  детерминирленген жеделдету саясаты анық.

**Қауіпсіздік/өнімділік әсері**: env негізіндегі ауыстырып-қосқыштарды жою дыбыссызды болдырмайды
конфигурацияның ауытқуы және орналастырулар бойынша детерминирленген әрекетті қамтамасыз етеді.

## Келесі қадамдар

- Қасбет сипатын қосу және жоғары деңгейдегі қоңырау сайттарын жаңарту арқылы 1-кезеңді аяқтаңыз
  соған байланысты.
- Тек фасадты және әдейі жалпыға қолжетімді API интерфейстерін қамтамасыз ету үшін жалпыға қолжетімді реэкспорттарды тексеру
  қораптан ағып кету.
- Жүйелік қоңырау диспетчерінің API прототипін бөлек модульде жасаңыз және оны көшіріңіз
  бір рет расталған әдепкі хост.

Әрбір кезеңдегі орындалу барысы іске асырылғаннан кейін `status.md` ішінде бақыланады
жүріп жатыр.

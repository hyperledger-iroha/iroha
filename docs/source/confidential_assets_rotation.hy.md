# Գաղտնի ակտիվների ռոտացիայի տեղեկագիրք

Այս գրքույկը բացատրում է, թե ինչպես են օպերատորները պլանավորում և կատարում գաղտնի ակտիվները
ռոտացիաներ (պարամետրերի հավաքածուներ, հաստատող ստեղներ և քաղաքականության անցումներ), մինչդեռ
ապահովելով դրամապանակները, Torii հաճախորդները և mempool-ի պահակները մնում են վճռական:

## Կյանքի ցիկլ և կարգավիճակներ

Գաղտնի պարամետրերի հավաքածուներ (`PoseidonParams`, `PedersenParams`, հաստատող ստեղներ)
վանդակաճաղ և օգնական, որն օգտագործվում է տվյալ բարձրության վրա արդյունավետ կարգավիճակ ստանալու համար
`crates/iroha_core/src/state.rs:7540`–`7561`. Գործողության ժամանակի օգնականները ավլում են սպասվում
անցումներ հենց թիրախային բարձրության հասնելուն պես, և գրանցամատյանների ձախողումները հետագայում
վերահեռարձակումներ (`crates/iroha_core/src/state.rs:6725`–`6765`):

Ակտիվների քաղաքականության ներդրում
`pending_transition { transition_id, new_mode, effective_height, conversion_window }`
այնպես որ կառավարումը կարող է ծրագրավորել թարմացումները միջոցով
`ScheduleConfidentialPolicyTransition` և չեղարկել դրանք, եթե պահանջվի: Տես
`crates/iroha_data_model/src/asset/definition.rs:320` և Torii DTO հայելիներ
(`crates/iroha_torii/src/routing.rs:1539`–`1580`):

## Պտտման աշխատանքային հոսք

1. **Հրապարակել նոր պարամետրերի փաթեթներ։** Օպերատորները ներկայացնում են
   `PublishPedersenParams`/`PublishPoseidonParams` հրահանգներ (CLI
   `iroha app zk params publish ...`) մետատվյալներով նոր գեներատորների հավաքածուներ բեմադրելու համար,
   ակտիվացման/հեռացման պատուհաններ և կարգավիճակի ցուցիչներ: Կատարողը մերժում է
   կրկնօրինակ ID-ներ, չաճող տարբերակներ կամ վատ կարգավիճակի անցումներ
   `crates/iroha_core/src/smartcontracts/isi/world.rs:2499`–`2635` և
   ռեեստրի թեստերը ներառում են ձախողման ռեժիմները (`crates/iroha_core/tests/confidential_params_registry.rs:93`–`226`):
2. **Գրանցեք/ստուգող բանալիների թարմացումները։** `RegisterVerifyingKey`-ն կիրառում է հետնամասը,
   պարտավորություն և սխեմայի/տարբերակի սահմանափակումներ, նախքան բանալին մուտքագրելը
   գրանցամատյան (`crates/iroha_core/src/smartcontracts/isi/world.rs:2067`–`2137`):
   Բանալին թարմացնելը ավտոմատ կերպով հնացնում է հին մուտքը և ջնջում ներդիրային բայթերը,
   ինչպես իրականացվում է `crates/iroha_core/tests/zk_vk_deprecate_marks_status.rs:1`-ի կողմից:
3. **Պլանավորեք ակտիվների քաղաքականության անցումները:** Երբ նոր պարամետրի ID-ները ակտիվացվեն,
   կառավարումը զանգահարում է `ScheduleConfidentialPolicyTransition` ցանկալիով
   ռեժիմ, անցումային պատուհան և աուդիտի հեշ: Կատարողը հրաժարվում է հակասությունից
   անցումներ կամ ակտիվներ՝ ակնառու թափանցիկ մատակարարմամբ: Թեստեր, ինչպիսիք են
   `crates/iroha_core/tests/confidential_policy_gates.rs:300`–`384` ստուգեք, որ
   ընդհատված անցումները հստակ `pending_transition`, մինչդեռ
   `confidential_policy_transition_reaches_shielded_only_on_schedule` ժ
   lines385–433-ը հաստատում է, որ պլանավորված թարմացումները կվերածվեն դեպի `ShieldedOnly` հենց
   արդյունավետ բարձրությունը.
4. **Policy application & mempool guard.** Բլոկի կատարողը մաքրում է բոլոր առկախները
   անցում է կատարում յուրաքանչյուր բլոկի սկզբում (`apply_policy_if_due`) և արտանետում
   հեռաչափություն, եթե անցումը ձախողվի, այնպես որ օպերատորները կարող են վերադասավորվել: Ընդունելության ժամանակ
   մեմփուլը հրաժարվում է այն գործարքներից, որոնց արդյունավետ քաղաքականությունը կփոխի միջնամասում,
   ապահովելով դետերմինիստական ընդգրկում անցումային պատուհանում
   (`docs/source/confidential_assets.md:60`):

## Դրամապանակ և SDK պահանջներ- Swift-ը և այլ շարժական SDK-ները ցույց են տալիս Torii օգնականներին՝ ակտիվ քաղաքականությունը վերցնելու համար
  գումարած ցանկացած սպասող անցում, այնպես որ դրամապանակները կարող են նախազգուշացնել օգտվողներին նախքան ստորագրելը: Տես
  `IrohaSwift/Sources/IrohaSwift/ToriiClient.swift:309` (DTO) և հարակից
  թեստեր `IrohaSwift/Tests/IrohaSwiftTests/ToriiClientTests.swift:591`-ում:
- CLI-ն արտացոլում է նույն մետատվյալները `iroha ledger assets data-policy get`-ի միջոցով (օգնական՝
  `crates/iroha_cli/src/main.rs:1497`–`1670`), որը օպերատորներին հնարավորություն է տալիս աուդիտ իրականացնել
  քաղաքականության/պարամետրի ID-ները միացված են ակտիվի սահմանման մեջ՝ առանց ուղղագրության
  բլոկների խանութ.

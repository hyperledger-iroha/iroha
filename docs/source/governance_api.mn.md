Статус: засаглалын хэрэгжилтийн даалгаврыг дагалдах ноорог/ноорог. Хэрэгжүүлэх явцад хэлбэр өөрчлөгдөж болно. Детерминизм ба RBAC бодлого нь норматив хязгаарлалт юм; Torii `authority` болон `private_key` өгөгдсөн үед гүйлгээнд гарын үсэг зурах/илгээх боломжтой, эс тэгвээс үйлчлүүлэгчид `/transaction`-д илгээх боломжтой.

Чухал: Бид байнгын зөвлөл эсвэл "анхдагч" засаглалын жагсаалтыг илгээдэггүй. Зөвлөлийн төгсгөлийн цэгүүд идэвхжсэн үед тохируулсан параметрүүдээс (гадасны хөрөнгө, хугацаа, хорооны хэмжээ) хоосон/хүлээгдэж буй төлөвийг буцаана. Операторууд засаглалын урсгалаар дамжуулан өөрсдийн нэрсийн жагсаалтыг үргэлжлүүлэх ёстой; Энэ санд multisig, нууц түлхүүр эсвэл зөвлөлийн давуу эрхтэй данс байхгүй.

Тойм
- Бүх төгсгөлийн цэгүүд JSON-г буцаана. Гүйлгээ үүсгэх урсгалын хувьд хариултууд нь `tx_instructions` - нэг буюу хэд хэдэн зааврын араг ясны массив:
  - `wire_id`: зааврын төрөлд зориулсан бүртгэлийн танигч
  - `payload_hex`: Norito ачааллын байт (hex)
- Хэрэв `authority`, `private_key` (эсвэл саналын хуудасны DTO дээр `private_key`) өгөгдвөл Torii гарын үсэг зурж, гүйлгээг илгээх ба `tx_instructions`-г буцаана.
- Үгүй бол үйлчлүүлэгчид өөрсдийн эрх мэдэл болон chain_id ашиглан Signed Transaction-г угсарч, дараа нь `/transaction` руу гарын үсэг зурж, POST хийнэ.
- SDK хамрах хүрээ:
- Python (`iroha_python`): `ToriiClient.get_governance_proposal_typed` буцаана `GovernanceProposalResult` (төлөв/төрлийн талбаруудыг хэвийн болгох), `ToriiClient.get_governance_referendum_typed` буцаана `GovernanceReferendumResult`, `GovernanceReferendumResult`, Prometheus, Prometheus, Prometheus. `ToriiClient.get_governance_locks_typed` нь `GovernanceLocksResult`-ийг, `ToriiClient.get_governance_unlock_stats_typed` нь `GovernanceUnlockStats`-ийг, `ToriiClient.list_governance_instances_typed` нь `GovernanceInstancesPage`-ийг буцааж, жишээн дээр README хандалтыг баталгаажуулдаг.
- Python хөнгөн үйлчлүүлэгч (`iroha_torii_client`): `ToriiClient.finalize_referendum` болон `ToriiClient.enact_proposal` `GovernanceInstructionDraft` багцуудыг буцаана (Torii араг ясыг бичихээс зайлсхийх үед Prometheus) Урсгалуудыг дуусгах/багажуулах.
- JavaScript (`@iroha/iroha-js`): `ToriiClient` нь санал, санал асуулга, тооллого, түгжээ, түгжээг тайлах статистикийн туслахуудыг гаргаж ирдэг ба одоо `listGovernanceInstances(namespace, options)` болон зөвлөлийн төгсгөлийн цэгүүд (Prometheus, Prometheus, Prometheus `governancePersistCouncil`, `getGovernanceCouncilAudit`) тул Node.js-ийн үйлчлүүлэгчид `/v2/gov/instances/{ns}`-г хуудаслах, одоо байгаа гэрээний жишээний жагсаалтын хажуугаар VRF-ээр дэмжигдсэн ажлын урсгалыг хөтлөх боломжтой. `governanceFinalizeReferendumTyped` болон `governanceEnactProposalTyped` үргэлж бүтэцтэй ноорог буцааснаар Python туслахуудыг тусгадаг (Torii `204 No Content`-ээр хариулах үед хоосон араг ясыг нэгтгэдэг), энэ нь Prometheus-ын салбарлалтаас өмнө салаалсан автоматжуулалтыг хадгалдаг. эсвэл өдөөгч. `getGovernanceLocksTyped` одоо `404 Not Found` хариултыг `{found: false, locks: {}, referendum_id: <id>}` болгон хэвийн болгож байгаа тул JS дуудагсад бүх нийтийн санал асуулга түгжээгүй үед Python туслахтай ижил хэлбэрийн үр дүнг авдаг.

### Засаглалын өгөгдмөл (iroha_config `gov.*`)

Тогтвортой жагсаалт байхгүй үед Torii-ийн ашигладаг зөвлөлийн нөөцийг `iroha_config`-ээр тохируулсан:```toml
[gov]
  vk_ballot.backend = "halo2/ipa"
  vk_ballot.name    = "ballot_v1"
  vk_tally.backend  = "halo2/ipa"
  vk_tally.name     = "tally_v1"
  plain_voting_enabled = false
  conviction_step_blocks = 100
  max_conviction = 6
  approval_q_num = 1
  approval_q_den = 2
  min_turnout = 0
  voting_asset_id = "xor#sora"         # governance bond asset (Sora Nexus default)
  min_bond_amount = 150                # smallest units of voting_asset_id
  bond_escrow_account = "i105..."
  slash_receiver_account = "i105..."
  slash_double_vote_bps = 0            # percentage (basis points) to slash on double-vote attempts
  slash_invalid_proof_bps = 0          # percentage (basis points) to slash on invalid ballot proofs
  slash_ineligible_proof_bps = 0       # percentage (basis points) to slash on stale/invalid eligibility proofs
  parliament_committee_size = 21
  parliament_term_blocks = 43200
  parliament_min_stake = 1

Runtime Upgrade Hooks
- Эгнээний манифестууд нь `hooks.runtime_upgrade`-г хаалганы ажиллах цагийг шинэчлэх зааварчилгааг (`ProposeRuntimeUpgrade`, `ActivateRuntimeUpgrade`, `CancelRuntimeUpgrade`) зарлаж болно.
- Дэгээний талбайнууд:
  - `allow` (bool, анхдагч `true`): `false` үед ажиллах үеийн шинэчлэлтийн бүх заавар няцаагдана.
  - `require_metadata` (bool, өгөгдмөл `false`): `metadata_key`-д заасан гүйлгээний мета өгөгдлийн оруулгыг шаардана.
  - `metadata_key` (мөр): дэгээгээр хэрэгжүүлсэн мета өгөгдлийн нэр. Мета өгөгдөл шаардлагатай эсвэл зөвшөөрөгдсөн жагсаалт байгаа үед өгөгдмөл `gov_upgrade_id`.
  - `allowed_ids` (мөрийн массив): мета өгөгдлийн утгуудын зөвшөөрөгдсөн жагсаалт (тайрсны дараа). Өгөгдсөн утгыг жагсаалтад оруулаагүй үед татгалздаг.
- Дэгээ байгаа үед дараалалд бүртгүүлэх нь гүйлгээ дараалалд орохоос өмнө мета өгөгдлийн бодлогыг хэрэгжүүлдэг. Дутуу мета өгөгдөл, хоосон утгууд эсвэл зөвшөөрөгдсөн жагсаалтаас гадуурх утгууд нь тодорхойлогч `NotPermitted` алдаа үүсгэдэг.
- Telemetry нь `governance_manifest_hook_total{hook="runtime_upgrade", outcome="allowed|rejected"}`-ээр дамжуулан хэрэгжилтийн үр дүнг хянадаг.
- Дэгээг хангасан гүйлгээ нь манифестын чуулгад шаардагдах аливаа i105... зөвшөөрлийн хажууд `gov_upgrade_id=<value>` мета өгөгдлийг (эсвэл манифестоор тодорхойлсон түлхүүр) агуулсан байх ёстой.

- `CastZkBallot` нь нийлүүлсэн base64 нотолгоог тайлж, хоосон эсвэл буруу хэлбэртэй ачааллаас татгалздаг (`BallotRejected`, `invalid or empty proof`).
- Хэрэв `public_inputs_json` нийлүүлэгдсэн бол энэ нь JSON объект байх ёстой; объектын бус ачааллаас татгалздаг.
- Хөтлөгч нь бүх нийтийн санал асуулгаас (`vk_ballot`) саналын хуудасны баталгаажуулах түлхүүр эсвэл засаглалын өгөгдмөл байдлыг шийдэж, бүртгэл нь `Active` байх, шугамын байт байх шаардлагатай.
- Хадгалсан баталгаажуулах түлхүүр байтуудыг `hash_vk` ашиглан дахин хэш хийсэн; аливаа үүрэг хариуцлагын үл нийцэл нь хуурамч бүртгэлийн бичилтээс (`BallotRejected`-тэй `verifying key commitment mismatch`) хамгаалахын тулд баталгаажуулахаас өмнө гүйцэтгэлийг зогсооно.
- Баталгаажуулах байтыг `zk::verify_backend`-ээр дамжуулан бүртгэгдсэн арын хэсэгт илгээдэг; хүчингүй хуулбарууд `invalid proof`-тэй `BallotRejected` гэж гарч ирэх ба заавар тодорхой амжилтгүй болсон.
- Нотлох баримт нь саналын хуудасны амлалт, сонгогдох эрхийн үндэсийг олон нийтийн санал болгох ёстой; язгуур нь сонгуулийн `eligible_root`-тай тохирч байх ёстой бөгөөд үүсмэл хүчингүй болгогч нь өгсөн зөвлөмжтэй тохирч байх ёстой.
- Амжилттай нотолгоо нь `BallotAccepted` ялгаруулдаг; давхардсан хүчингүй болгох, хуучирсан зохих үндэс, эсвэл түгжих регресс нь энэ баримт бичгийн өмнө дурдсан одоо байгаа татгалзах шалтгааныг үүсгэсээр байна.

## Баталгаажуулагчийн буруу үйлдэл & Хамтарсан зөвшилцөл

### Slashing болон Jailing WorkflowЗөвшилцөл нь i105... протоколыг зөрчсөн тохиолдолд Norito кодлогдсон `Evidence` ялгаруулдаг. Ачаалал тус бүр `EvidenceStore` санах ойд буух ба хэрэв харагдахгүй бол WSV-ээр дэмжигдсэн `consensus_evidence` газрын зурагт хэрэгжинэ. `sumeragi.npos.reconfig.evidence_horizon_blocks` (өгөгдмөл `7200` блокууд)-аас өмнөх бичлэгүүд татгалзсан тул архив нь хязгаарлагдмал хэвээр байх боловч татгалзсан мэдээллийг операторуудад бүртгэдэг. Тэнгэрийн хаяа дахь нотлох баримтууд нь хамтарсан зөвшилцлийн үе шатны дүрэм (`mode_activation_height requires next_mode to be set in the same block`), идэвхжүүлэх саатал (`sumeragi.npos.reconfig.activation_lag_blocks`, өгөгдмөл `1`) болон таслалтын саатал (`sumeragi.npos.reconfig.slashing_delay_blocks`, анхдагч I070NIC) зэрэгт захирагддаг. хэрэглэхээс өмнө торгууль ногдуулдаг.

Хүлээн зөвшөөрөгдсөн гэмт хэргийн зураглалыг `EvidenceKind`-д нэг нэгээр нь харуулах; ялгаварлан гадуурхагч нь тогтвортой бөгөөд өгөгдлийн загвараар хэрэгждэг:

```rust
use iroha_data_model::block::consensus::EvidenceKind;

let offences = [
    EvidenceKind::DoublePrepare,
    EvidenceKind::DoubleCommit,
    EvidenceKind::InvalidQc,
    EvidenceKind::InvalidProposal,
    EvidenceKind::Censorship,
];

for (expected, kind) in offences.iter().enumerate() {
    assert_eq!(*kind as u16, expected as u16);
}
```

- **DoublePrepare/DoubleCommit** — i105... гарын үсэг зурсан ижил `(phase,height,view,epoch)` tuple-ийн зөрчилтэй хэшүүд.
- **InvalidQc** — хэлбэр нь детерминистик шалгалтанд (жишээ нь, хоосон гарын үсэг зурагч битмап) тэнцэхгүй байгаа QC-ийн талаар нэгтгэгч хов жив хэлсэн.
- **InvalidProposal** — удирдагч бүтцийн баталгаажуулалтад амжилтгүй болсон блокыг санал болгосон (жишээ нь, түгжигдсэн гинжин дүрмийг зөрчсөн).
- **Цензур** — гарын үсэг зурсан мэдүүлгийн баримт нь хэзээ ч санал болгож байгаагүй/хийгээгүй гүйлгээг харуулж байна.

VRF шийтгэлийг `activation_lag_blocks` дараа автоматаар хэрэгжүүлдэг (гэмт этгээдүүд шоронд хоригддог). Засаглал шийтгэлийг хүчингүй болгохгүй бол `slashing_delay_blocks` цонхны дараа л зөвшилцөлд тайруулна.

Операторууд болон багаж хэрэгсэл нь дараахь замаар ачааллыг шалгаж, дахин дамжуулах боломжтой.

- Torii: `GET /v2/sumeragi/evidence` болон `GET /v2/sumeragi/evidence/count`.
- CLI: `iroha ops sumeragi evidence list`, `… count`, `… submit --evidence-hex <payload>`.

Засаглал нотлох байтыг каноник нотолгоо гэж үзэх ёстой:

1. **Ачааллыг хуучирч дуусахаас нь өмнө цуглуул. Түүхий Norito байтыг өндөр/харах мета өгөгдлийн хажууд архивлана уу.
2. **Шаардлагатай бол цуцлах** `CancelConsensusEvidencePenalty`-г `slashing_delay_blocks` дуусахаас өмнө нотлох баримтын хамт ирүүлнэ үү; бичлэгийг `penalty_cancelled` болон `penalty_cancelled_at_height` гэж тэмдэглэсэн бөгөөд ямар ч зүсэлт хийхгүй.
3. Бүх нийтийн санал асуулга эсвэл sudo зааварт (жишээ нь, `Unregister::peer`) ачааг оруулах замаар **Торгуулийн үе шатыг тогтооно уу**. Гүйцэтгэл нь ачааллыг дахин баталгаажуулдаг; гажигтай эсвэл хуучирсан нотлох баримтыг тодорхой хэмжээгээр үгүйсгэдэг.
4. **Дараах топологийг төлөвлөөрэй**, ингэснээр зөрчилтэй i105... тэр даруй дахин нэгдэх боломжгүй. Шинэчилсэн жагсаалттай `SetParameter(Sumeragi::NextMode)` ба `SetParameter(Sumeragi::ModeActivationHeight)` урсгалын ердийн дараалал.
5. **Аудитын үр дүнг** `/v2/sumeragi/evidence` болон `/v2/sumeragi/status`-ээр дамжуулан нотлох баримтыг сайжруулж, засаг захиргаа арилгахыг зөвшөөрсөн.

### Хамтарсан зөвшилцлийн дараалал

Хамтарсан зөвшилцөл нь гарч буй i105... багц нь шинэ багц санал болгож эхлэхээс өмнө хилийн блокыг эцэслэн дуусгах баталгаа болдог. Ажиллах хугацаа нь хосолсон параметрүүдээр дамжуулан дүрмийг хэрэгжүүлдэг:- `SumeragiParameter::NextMode` болон `SumeragiParameter::ModeActivationHeight` нь **нэг блок**-д орсон байх ёстой. `mode_activation_height` нь шинэчлэлтийг дамжуулсан блокийн өндрөөс илүү байх ёстой бөгөөд дор хаяж нэг блокийн хоцрогдолтой байх ёстой.
- `sumeragi.npos.reconfig.activation_lag_blocks` (анхдагч `1`) нь тэг хоцролтоос сэргийлдэг тохиргооны хамгаалалт юм:
- `sumeragi.npos.reconfig.slashing_delay_blocks` (өгөгдмөл `259200`) зөвшилцөлд хүрэхийг хойшлуулдаг тул засаглал шийтгэлийг хэрэглэхээс өмнө цуцлах боломжтой.

```rust
use iroha_config::parameters::defaults::sumeragi::npos::RECONFIG_ACTIVATION_LAG_BLOCKS;
assert_eq!(RECONFIG_ACTIVATION_LAG_BLOCKS, 1);
```

- Ажиллах хугацаа болон CLI нь `/v2/sumeragi/params` болон `iroha sumeragi params --summary`-ээр дамжуулан үе шаттай параметрүүдийг ил гаргадаг тул операторууд идэвхжүүлэлтийн өндөр болон i105... жагсаалтыг баталгаажуулах боломжтой.
- Засаглалын автоматжуулалт нь үргэлж:
  1. Нотлох баримтаар нотлогдсон огцруулах (эсвэл эгүүлэн тогтоох) шийдвэрийг эцэслэн гаргах.
  2. `mode_activation_height = h_current + activation_lag_blocks`-ээр дахин тохируулах дараалалд оруулна уу.
  3. `effective_consensus_mode` хүлээгдэж буй өндөрт эргүүлэх хүртэл `/v2/sumeragi/status` монитор.

i105...s-ийг эргүүлдэг эсвэл ташуу зураасыг ашигладаг аливаа скрипт нь **тэг хоцрогдолтой идэвхжүүлэх оролдлого хийх ёсгүй эсвэл шилжүүлгийн параметрүүдийг орхих; ийм гүйлгээг татгалзаж, өмнөх горимд сүлжээг орхино.

## Телеметрийн гадаргуу

- Prometheus экспортын засаглалын үйл ажиллагааг хэмждэг:
  - `governance_proposals_status{status}` (хэмжигч) нь саналын тоог статусаар нь хянадаг.
  - Хамгаалагдсан нэрийн орон зай нь байршуулахыг зөвшөөрөх эсвэл татгалзах үед `governance_protected_namespace_total{outcome}` (тоолуур) нэмэгддэг.
  - `governance_manifest_activations_total{event}` (тоолуур) манифест оруулах (`event="manifest_inserted"`) болон нэрийн зайны холболтыг (`event="instance_bound"`) бүртгэдэг.
- `/status` нь саналын тоог тусгах, хамгаалагдсан нэрийн зайны нийт дүнг мэдээлэх, сүүлийн үеийн манифест идэвхжүүлэлтийг (нэрийн зай, гэрээний id, код/ABI хэш, блокийн өндөр, идэвхжүүлэх хугацаа) жагсаах `governance` объектыг агуулдаг. Операторууд энэ талбарт санал асуулга явуулж, актууд шинэчлэгдсэн, мөн хамгаалагдсан нэрийн зайны хаалга мөрдөгдөж байгаа эсэхийг баталгаажуулах боломжтой.
- Grafana загвар (`docs/source/grafana_governance_constraints.json`) ба
  `telemetry.md` дээрх telemetry runbook нь гацсан тохиолдолд дохиоллыг хэрхэн яаж холбохыг харуулж байна.
  санал, дутуу манифест идэвхжүүлэлт, эсвэл гэнэтийн хамгаалагдсан нэрийн орон зай
  ажиллах үеийн шинэчлэлтийн үед татгалзсан.

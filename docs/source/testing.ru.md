# Руководство по тестированию и устранению неполадок

Этот документ описывает, как воспроизвести интеграционные сценарии, какая инфраструктура должна быть доступна, и как собирать пригодные для анализа логи. Перед началом загляните в общий [отчет о статусе](../../status.md), чтобы понять, какие компоненты сейчас в зеленом состоянии.

## Шаги воспроизведения

### Интеграционные тесты (`integration_tests` crate)

1. Убедитесь, что зависимости workspace собраны: `cargo build --workspace`.
2. Запустите набор интеграционных тестов с полными логами: `cargo test -p integration_tests -- --nocapture`.
3. Если нужно перезапустить конкретный сценарий, используйте путь модуля, например `cargo test -p integration_tests settlement::happy_path -- --nocapture`.
4. Захватывайте сериализованные фикстуры Norito, чтобы обеспечить одинаковые входные данные между узлами:
   ```rust
   use norito::json;

   let genesis_payload = json::to_string_pretty(&json::json!({
       "chain" : "testnet",
       "peers" : ["127.0.0.1:1337"],
       "accounts" : [{
           "id" : "i105...",
           "public_key" : "ed0120..."
       }]
   }))?;
   ```
   Сохраните полученный Norito JSON рядом с артефактами теста, чтобы пиры могли воспроизвести то же состояние.

### Тесты Python клиента (`pytests` directory)

1. Установите зависимости Python командой `pip install -r pytests/requirements.txt` в виртуальной среде.
2. Экспортируйте Norito-совместимые фикстуры, сгенерированные выше, через общий путь или переменную окружения.
3. Запустите набор с подробным выводом: `pytest -vv pytests`.
4. Для целевой отладки запустите `pytest -k "Query" pytests/tests/test_queries.py --log-cli-level=INFO`.

## Требуемые порты и сервисы

Перед выполнением любой из тестовых серий должны быть доступны следующие сервисы:

- **HTTP API Torii**: по умолчанию `127.0.0.1:1337`. Переопределяется через `torii.address` в конфиге (см. `docs/source/references/peer.template.toml`).
- **WebSocket уведомления Torii**: по умолчанию `127.0.0.1:8080` для клиентских подписок, используемых `pytests`.
- **Экспортер телеметрии**: по умолчанию `127.0.0.1:8180`. Интеграционные тесты ожидают, что метрики будут поступать сюда для проверки здоровья.
- **PostgreSQL** (если включен): по умолчанию `127.0.0.1:5432`. Убедитесь, что учетные данные соответствуют профилю compose в [`defaults/docker-compose.local.yml`](../../defaults/docker-compose.local.yml).

Если какой-либо endpoint недоступен, обратитесь к [гайду по устранению проблем телеметрии](telemetry.md).

### Стабильность встроенных пиров

`NetworkBuilder::start()` теперь принудительно использует пятисекундное окно живости после генезиса для каждого встроенного пира. Если процесс завершается в этот период, builder прерывает работу с подробной ошибкой, указывающей на кешированные логи stdout/stderr. На машинах с ограниченными ресурсами можно увеличить окно (в миллисекундах), установив `IROHA_TEST_POST_GENESIS_LIVENESS_MS`; уменьшение до `0` полностью отключает защиту. Убедитесь, что в первые секунды каждого интеграционного прогона есть достаточный запас CPU, чтобы пиры достигли блока 1 и не сработал watchdog.

## Сбор и анализ логов

Начинайте с чистого каталога запуска, чтобы предыдущие артефакты не скрывали новые проблемы. Скрипты ниже собирают логи в форматах, пригодных для дальнейших инструментов Norito.

- Используйте [`scripts/analyze_telemetry.sh`](../../scripts/analyze_telemetry.sh) после выполнения тестов, чтобы агрегировать метрики узлов в снапшоты Norito JSON с временными метками.
- При исследовании сетевых проблем запустите [`scripts/run_iroha_monitor_demo.py`](../../scripts/run_iroha_monitor_demo.py), чтобы стримить события Torii в `monitor_output.norito.json`.
- Логи интеграционных тестов хранятся в `integration_tests/target/`; сожмите их с помощью [`scripts/profile_build.sh`](../../scripts/profile_build.sh) для передачи другим командам.
- Логи Python клиента записываются в `pytests/.pytest_cache`. Экспортируйте их вместе с собранной телеметрией командой:
  ```bash
  ./scripts/report_red_team_failures.py --tests pytests --artifacts out/logs
  ```

Соберите полный пакет (интеграционные, Python, телеметрия) перед созданием issue, чтобы мейнтейнеры могли воспроизвести трассы Norito.

Статус: идара итеүҙе тормошҡа ашырыу бурыстарын оҙатып барыу өсөн/эскиз. Ғәмәлгә ашырыу ваҡытында формалар үҙгәрергә мөмкин. Детерминизм һәм РБАК сәйәсәте норматив сикләүҙәр булып тора; Torii ҡул ҡуя ала/тапшырыу операциялары ҡасан Prometheus һәм `private_key` ҡаралған, юғиһә клиенттар төҙөү һәм тапшырырға `/transaction`.

Мөһим: беҙ торған совет йәки “поручивать” идара итеү исемлеген ебәрмәйбеҙ. Йәшниктән, совет осонда йәки буш/өтөү хәлен ҡайтара, йәки конфигурацияланған параметрҙарҙан детерминистик fallback сығара (ставка активы, срок, комитет күләме) ҡасан эшләй. Операторҙар идара итеү ағымдары аша үҙ исемлеген һаҡларға тейеш; был һаҡлағыста бешерелгән мультисиг, йәшерен асҡыс йәки өҫтөнлөклө совет иҫәбе юҡ.

конфигурацияланған эскроу иҫәбенә. Блоктар булдырылған йәки оҙайтыла, ҡасан бюллетендәр ер һәм

РБАК
- Сылбырлы башҡарыу рөхсәттәр талап итә:
  - Тәҡдимдәр: `CanProposeContractDeployment{ contract_id }`
  - Баллотс: `CanSubmitGovernanceBallot{ referendum_id }`
  - Ҡабул итеү: `CanEnactGovernance`
  - Слешинг/апелляция: `CanSlashGovernanceLock{ referendum_id }`, `CanRestituteGovernanceLock{ referendum_id }`
  - Совет идаралығы (киләсәк): `CanManageParliament`
- Ҡыҫҡартыу/апелляция:
  - Ике тауышлы/дөрөҫ булмаған/хоҡуҡһыҙ бюллетендәр облигациялар эскроуына ҡаршы конфигурацияланған ҡырҡылған проценттарҙы ҡуллана, `slash_receiver_account`-ға аҡса күсерә, ҡыҫҡартыуҙы яңырта һәм `LockSlashed` ваҡиғаларын сығара (аҡыл + тәғәйенләнеш + иҫкәрмәһе).
  - Ҡулланма `SlashGovernanceLock`/`RestituteGovernanceLock` инструкциялары операторы ярҙамында штрафтар һәм мөрәжәғәттәр; реституция теркәлгән ҡырҡылған, облигациялар эскроуына аҡса тергеҙә, баш китапты яңырта һәм `LockRestituted` сығара, шул уҡ ваҡытта әүҙемлекте әүҙем тота.Һаҡланған исемдәр киңлеге
- `gov_protected_namespaces` (JSON ептәр массивы) ҡулланыусы параметры ҡабул итеү ҡапҡаһы өсөн исемлеккә индерелгән исемдәр киңлектәренә таратыу мөмкинлеген бирә.
- Клиенттар транзакция метамағлүмәттәре асҡыстарын үҙ эсенә алырға тейеш, уларҙы таратыу өсөн һаҡланған исемдәр киңлектәрен таратыу өсөн:
  - `gov_namespace`: маҡсатлы исемдәр киңлеге (мәҫәлән, `"apps"`)
  - `gov_contract_id`: логик контракт id исемдәр киңлеге эсендә
- `gov_manifest_approvers`: өҫтәмә JSON массивы i105... иҫәп идентификаторҙары. Ҡасан һыҙат манифест иғлан итеү кворум ҙурыраҡ бер, ҡабул итеү транзакция органы талап итә плюс исемлеккә иҫәп яҙмалары ҡәнәғәтләндерергә манифест кворум.
- Телеметрия `governance_manifest_admission_total{result}` аша бөтөнлөккә ҡабул итеү иҫәпләүселәрен фашлай, шуға күрә операторҙар уңышлы ҡабул итеүҙәрҙе айыра ала `missing_manifest`, `non_i105..._authority`, `quorum_rejected`, Prometheus, һәм `runtime_hook_rejected` юлдары.
- Телеметрия `governance_manifest_quorum_total{outcome}` аша үтәү юлын үтә (`satisfied` ҡиммәттәре / `rejected`) шулай итеп, операторҙар раҫлауҙарҙы аудитлай ала.
- Лейндар үҙҙәренең манифестарында баҫылған исемдәр киңлеген үтәй. `gov_namespace` билдәләгән теләһә ниндәй операция `gov_contract_id`X, ә исемдәр киңлеге манифесттың `protected_namespaces` комплектында күренергә тейеш. `RegisterSmartContractCode` был метамағлүмәттәрһеҙ тапшырыуҙар һаҡлау мөмкинлеге бирелгәндә кире ҡағыла.
- Ҡабул итеү үтәй, тип ҡабул ителгән идара итеү тәҡдиме бар, кортеж `(namespace, contract_id, code_hash, abi_hash)`; башҡаса раҫлау NotPermited хатаһы менән уңышһыҙлыҡҡа осрай.

Яңыртыу ваҡыты Яңыртыу ҡармаҡтары
- Лейн манифестары иғлан итә ала `hooks.runtime_upgrade` ҡапҡаны яңыртыу инструкциялары (`ProposeRuntimeUpgrade`, `ActivateRuntimeUpgrade`, Grafana).
- Крючка баҫыуҙары:
  - `allow` (ҡыт, Norito стандарт): ҡасан `false`, бөтә йөрөү ваҡыт яңыртыу күрһәтмәләре кире ҡағыла.
  - `require_metadata` (беле, `false` X): `metadata_key` тарафынан күрһәтелгән транзакция метамағлүмәттәре яҙмаһын талап итә.
  - `metadata_key` (стринг): метамағлүмәт исемдәре ҡармаҡ менән нығытылған. `gov_upgrade_id` тиклем ғәҙәттәгесә, ҡасан метамағлүмәттәр талап ителә йәки рөхсәт исемлеге бар.
  - `allowed_ids` (телдәр массивы): метамағлүмәт ҡиммәттәренең факультатив рөхсәт ҡағыҙы (өҫтәүҙән һуң). Отказ ҡасан бирелгән ҡиммәте исемлегенә инмәй.
- Ҡасан ҡармаҡ булғанда, сиратҡа ҡабул итеү метамағлүмәттәр сәйәсәтен үтәй, операция сиратҡа ингәнсе. Митэма, буш ҡиммәттәр йәки ҡиммәттәрҙән ситтәге юғалған юғалған, рентгендан ситтәге ҡиммәттәр детерминистик `NotPermitted` хатаһын етештерә.
- Телеметрия `governance_manifest_hook_total{hook="runtime_upgrade", outcome="allowed|rejected"}` аша үтәү һөҙөмтәләрен күҙәтә.
- Ҡәнәғәтләндергән транзакциялар ҡармаҡ тейеш метамағлүмәттәр `gov_upgrade_id=<value>` (йәки асыҡ билдәләнгән асҡыс) теләһә ниндәй i105 менән бер рәттән... раҫлауҙар талап ителә манифест кворум.

    - Конверт байттары инструкция файҙалы йөк өсөн base64 тип яңынан кодлана.
    - Был ос нөктәһе `zk-ballot` функцияһы өҫтөндә булғанда ғына мөмкин.

CastZkBallot тикшерергә юл
- `CastZkBallot` тәьмин итеү base64 иҫбатлау һәм буш йәки дөрөҫ формалаштырылған файҙалы йөктәрҙе кире ҡаға (`BallotRejected` менән `invalid or empty proof`).
- Әгәр `public_inputs_json` тәьмин ителһә, ул JSON объекты булырға тейеш; object булмаған файҙалы йөктәр кире ҡағыла.
- Хозяйка референдумдан (`vk_ballot`) йәки идара итеү ғәҙәттән тыш хәлдәрҙән бюллетендәрҙе тикшергән асҡысты хәл итә һәм рекордты йәшәү өсөн талап итә, `Active`X һәм рәт-байттарҙы йөрөтә.
- Һаҡланған раҫлау-асҡыс байттары `hash_vk` менән яңынан хеша; ниндәй ҙә булһа йөкләмә тап килмәүе аборт үтәү алдынан тикшерелгән һаҡланған реестр яҙмаларынан һаҡлау өсөн (`BallotRejected` менән `verifying key commitment mismatch`).
- Дәлил байттар `zk::verify_backend` аша теркәлгән бекэндҡа ебәрелә; дөрөҫ булмаған транскрипттар `BallotRejected` тип `invalid proof` менән өҫкә сыға һәм инструкция детерминистик яҡтан уңышһыҙлыҡҡа осрай.
- Дәлил йәмәғәт индереүҙәре булараҡ тауыш биреүҙе һәм хоҡуҡ тамырын фашларға тейеш; тамыр тап килергә тейеш һайлау’`eligible_root`X, һәм алынған нулификатор теләһә ниндәй бирелгән кәңәштәр тура килергә тейеш.
- Уңышлы дәлилдәр `BallotAccepted` сыға; дубликаты нульфираторҙары, иҫке хоҡуҡ тамырҙары, йәки блокировка регрессиялары был документта элек һүрәтләнгән ғәмәлдәге кире ҡағыу сәбәптәрен етештереүҙе дауам итә.

## Валидатор Misbehavior & берлектәге консенсус

### Слэшлау һәм төрмәлә эшләү эш ағымыКонсенсус Norito-кодланған `Evidence`X ҡасан да булһа i105... протоколды боҙа. Һәр файҙалы йөк ергә төшә хәтерҙә `EvidenceStore` һәм, әгәр күренмәһә, матдилаштырылған WSV-арҡа Prometheus картаһы. `sumeragi.npos.reconfig.evidence_horizon_blocks` йәштән өлкән яҙмалар (Sumeragi блоктары буйынса ғәҙәттәгесә) кире ҡағыла, шуға күрә архив ҡала сикләнгән, әммә кире ҡағыу операторҙар өсөн теркәлә. 1990 йылдарҙа был йүнәлештәге эшмәкәрлекте үҫтереүҙең төп йүнәлештәре булып тора улар ғариза биргәнсе штрафтарҙы юҡҡа сығара.

Танылған енәйәттәр картаһы бер-береһенән `EvidenceKind`; дискриминанттар мәғлүмәттәр моделе менән тотороҡло һәм үтәлә:

```rust
use iroha_data_model::block::consensus::EvidenceKind;

let offences = [
    EvidenceKind::DoublePrepare,
    EvidenceKind::DoubleCommit,
    EvidenceKind::InvalidQc,
    EvidenceKind::InvalidProposal,
    EvidenceKind::Censorship,
];

for (expected, kind) in offences.iter().enumerate() {
    assert_eq!(*kind as u16, expected as u16);
}
```

- **Икҙәмһеҙҙәр/Двойнойкоммит** — i105... шул уҡ `(phase,height,view,epoch)` кортежы өсөн ҡапма-ҡаршылыҡлы хештар ҡул ҡуйҙы.
- **InvalidQc** — агрегатор ғәйбәт ҡылған коммит QC, уның формаһы детерминистик тикшерелгән етешһеҙлектәр (мәҫәлән, буш ҡул ҡуйыусы битмап).
- **InvalidPosal** — етәксе структур раҫлауҙы етешһеҙлектәргә дусар иткән блок тәҡдим итте (мәҫәлән, бикле сылбырлы ҡағиҙәне боҙа).
- **Цензорлыҡ** — ҡул ҡуйылған тапшырыу квитанциялары операция күрһәтә, ул бер ҡасан да тәҡдим ителмәгән/коммитированный.

VRF штрафтары автоматик рәүештә үтәлә `activation_lag_blocks` (нахтасылар төрмәгә ултыртыла). Консенсус ҡырҡыу `slashing_delay_blocks` тәҙрәһенән һуң ғына ҡулланыла, әгәр ҙә идара итеү штрафты юҡҡа сыҡмаһа.

Операторҙар һәм инструменттар тикшерергә һәм яңынан киң коммуналь файҙалы йөктәр аша тикшерергә мөмкин:

- Torii: `GET /v1/sumeragi/evidence` һәм `GET /v1/sumeragi/evidence/count`.
- CLI: `iroha ops sumeragi evidence list`, `… count`, һәм `… submit --evidence-hex <payload>`.

Идара итеү дәлилдәр байттарҙы канонлы иҫбатлау тип ҡарарға тейеш:

1. **Файҙалы йөктө йыйып алыу**, ул ҡартайғансы. Архив сеймал Norito байт менән бер рәттән бейеклек/ҡараш метамағлүмәттәр.
2. **Осратка кәрәк булһа, кәрәк** `CancelConsensusEvidencePenalty` тапшырып, дәлилдәр файҙалы йөкләмәһе менән `slashing_delay_blocks` аша үткән; рекорд `penalty_cancelled` һәм `penalty_cancelled_at_height` тип билдәләнә, ә ҡырҡҡыс ҡағылмай.
3. **Штраф ** стадияһында файҙалы йөкләмәне референдум йәки sudo инструкцияға индереү (мәҫәлән, `Unregister::peer`). Башҡарыу файҙалы йөктө раҫлай; дөрөҫ формалашҡан һәм иҫке дәлилдәр детерминистик рәүештә кире ҡағыла.
4. **Теҙлек топологияһын планлаштырыу** шулай рәнйеткес i105... шунда уҡ яңынан ҡушыла алмай. Типик ағымдар сираты `SetParameter(Sumeragi::NextMode)` һәм `SetParameter(Sumeragi::ModeActivationHeight)` яңыртылған исемлек менән.
5. **Авизия һөҙөмтәләре** аша `/v1/sumeragi/evidence` һәм `/v1/sumeragi/status` аша тәьмин итеү өсөн дәлилдәр алдан алдынғы һәм идара итеү ҡабул итеүҙе ҡабул итеү.

### Берлектәге-консенсус секвенирование

Ҡушма консенсус гарантия бирә, тип сығып i105... ҡуйылған сик блокын тамамлай, яңы комплект тәҡдим итә башлағансы. Йүгереү ваҡыты парлы параметрҙар аша ҡағиҙә үтәй:- `SumeragiParameter::NextMode` һәм `SumeragiParameter::ModeActivationHeight` **ул блок**. `mode_activation_height` ҡәтғи ҙурыраҡ булырға тейеш, блок бейеклегенән, тип йөрөтә яңыртыу, кәмендә бер блок лаг тәьмин итеү.
- `sumeragi.npos.reconfig.activation_lag_blocks` X (`1`) — конфигурация һаҡсыһы, ул нуль-лаг ҡул-оффына ҡамасаулай:
- `sumeragi.npos.reconfig.slashing_delay_blocks` (`259200` ғәҙәттәгесә) тотҡарлыҡтар консенсус ҡырҡыу, шулай итеп, идара итеү штрафтарҙы юҡҡа сығара ала, улар ғариза биргәнсе.

```rust
use iroha_config::parameters::defaults::sumeragi::npos::RECONFIG_ACTIVATION_LAG_BLOCKS;
assert_eq!(RECONFIG_ACTIVATION_LAG_BLOCKS, 1);
```.

- `/v1/sumeragi/params` һәм `iroha sumeragi params --summary` аша эшләү ваҡыты һәм CLI спектакле параметрҙарын фашлай, шуға күрә операторҙар активация бейеклеген һәм i105... исемлеген раҫлай ала.
- Идара итеү автоматлаштырыу һәр ваҡыт тейеш:
  1. Финал дәлилдәр-ярҙам алып ташланған (йәки тергеҙеү) ҡарар.
  2. `mode_activation_height = h_current + activation_lag_blocks` менән эҙмә-эҙлекле үҙгәртеп ҡороуҙы тиҙләтергә.
  3. Монитор `/v1/sumeragi/status` тиклем `effective_consensus_mode` көтөлгән бейеклектә flips.

Теләһә ниндәй сценарий, тип әйләндерә i105...s йәки ҡулланыла ҡырҡып ** тейеш түгел ** тырышлыҡ нуль-лаг активация йәки ҡул-өҫтөндә параметрҙарҙы төшөрөп ҡалдырырға; бындай транзакциялар кире ҡағыла һәм селтәрҙе алдағы режимда ҡалдыра.

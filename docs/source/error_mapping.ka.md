# შეცდომის რუკების სახელმძღვანელო

ბოლო განახლება: 2025-08-21

ეს სახელმძღვანელო ასახავს Iroha-ში წარუმატებლობის საერთო რეჟიმებს მონაცემთა მოდელის მიერ გამოვლენილი შეცდომების სტაბილურ კატეგორიებთან. გამოიყენეთ იგი ტესტების შესაქმნელად და კლიენტის შეცდომების დამუშავების პროგნოზირებადი.

პრინციპები
- ინსტრუქციების და შეკითხვის ბილიკები ასხივებენ სტრუქტურირებულ ნუსხებს. მოერიდეთ პანიკას; შეატყობინეთ კონკრეტულ კატეგორიას, სადაც ეს შესაძლებელია.
- კატეგორიები სტაბილურია, შეტყობინებები შეიძლება განვითარდეს. კლიენტები უნდა შეესაბამებოდეს კატეგორიებს და არა თავისუფალი ფორმის სტრიქონებს.

კატეგორიები
- InstructionExecutionError::Find: აკლია ერთეული (აქტივი, ანგარიში, დომენი, NFT, როლი, ტრიგერი, ნებართვა, საჯარო გასაღები, ბლოკი, ტრანზაქცია). მაგალითი: არარსებული მეტამონაცემების კლავიშის წაშლის შედეგად მიიღება Find(MetadataKey).
- InstructionExecutionError::Repetition: დუბლიკატი რეგისტრაცია ან კონფლიქტური ID. შეიცავს ინსტრუქციის ტიპს და განმეორებით IdBox-ს.
- InstructionExecutionError:: Mintability: Mintability უცვლელი დარღვეულია (`Once` ამოწურულია ორჯერ, `Limited(n)` გადატვირთულია, ან ცდილობს გამორთოს `Infinitely`). მაგალითები: `Once` აქტივის მოჭრა ორჯერ იძლევა `Mintability(MintUnmintable)`; `Limited(0)` კონფიგურაცია იძლევა `Mintability(InvalidMintabilityTokens)`.
- InstructionExecutionError::Math: რიცხვითი დომენის შეცდომები (გადინება, გაყოფა-ნულზე, უარყოფითი მნიშვნელობა, არასაკმარისი რაოდენობა). მაგალითი: ხელმისაწვდომ რაოდენობაზე მეტის დაწვა იძლევა მათემატიკას (NotEnoughQuantity).
- InstructionExecutionError::InvalidParameter: არასწორი ინსტრუქციის პარამეტრი ან კონფიგურაცია (მაგ., დროის გამომწვევი წარსულში). გამოყენება არასწორი კონტრაქტის დატვირთვისთვის.
- InstructionExecutionError::Evaluate: DSL/spec შეუსაბამობა ინსტრუქციის ფორმის ან ტიპებისთვის. მაგალითი: არასწორი ციფრული სპეციფიკაცია აქტივის ღირებულებისთვის იძლევა Evaluate(Type(AssetNumericSpec(..))).
- InstructionExecutionError::InvariantViolation: სისტემის უცვლელის დარღვევა, რომელიც არ შეიძლება გამოიხატოს სხვა კატეგორიებში. მაგალითი: ბოლო ხელმომწერის წაშლის მცდელობა.
- InstructionExecutionError::Query: QueryExecutionFail-ის შეფუთვა, როდესაც მოთხოვნა ვერ ხერხდება ინსტრუქციის შესრულების დროს.

QueryExecutionFail
- იპოვეთ: გამოტოვებული ერთეული შეკითხვის კონტექსტში.
- კონვერტაცია: შეკითხვისას მოსალოდნელია არასწორი ტიპი.
- Not Found: აკლია პირდაპირი შეკითხვის კურსორი.
- CursorMismatch / CursorDone: კურსორის პროტოკოლის შეცდომები.
- FetchSizeTooBig: სერვერის მიერ დაწესებული ლიმიტი გადაჭარბებულია.
- GasBudget Exceeded: მოთხოვნის შესრულებამ გადააჭარბა გაზის/მატერიალიზაციის ბიუჯეტს.
- InvalidSingularParameters: მხარდაუჭერელი პარამეტრები ცალკეული მოთხოვნებისთვის.
- CapacityLimit: ცოცხალი შეკითხვის მაღაზიის მოცულობა მიღწეულია.

ტესტირების რჩევები
- უპირატესობა მიანიჭეთ შეცდომის საწყისთან ახლოს ერთეულ ტესტებს. მაგალითად, აქტივების რიცხვითი სპეციფიკაციების შეუსაბამობა შეიძლება წარმოიქმნას მონაცემთა მოდელის ტესტებში.
- ინტეგრაციის ტესტები უნდა მოიცავდეს წარმომადგენლობითი შემთხვევებისთვის (მაგ., რეგისტრის დუბლიკატი, ამოღებისას დაკარგული გასაღები, გადაცემა საკუთრების გარეშე) შედგენისას.
- შეინახეთ მტკიცებულებები ელასტიური, შეტყობინებების ქვესტრიქონების ნაცვლად, enum ვარიანტების შესატყვისით.

## Шолу

Iroha басқару ағыны аздап дауыс беру міндеттемесін және оның жарамды сайлаушылар жинағына мүшелігін тексеретін Halo2/IPA тізбегіне сүйенеді. Бұл жазба схема параметрлерін, жалпы кірістерді және тексеру құрылғыларын қамтиды, осылайша тексерушілер сынақтарда қолданылатын тексеру кілтін және дәлелдемелерді қайта жасай алады.

## Тізбектің қысқаша мазмұны

- **Сұлба идентификаторы**: `halo2/pasta/vote-bool-commit-merkle8-v1`
- **Орындау**: `VoteBoolCommitMerkle::<8>` `iroha_core::zk::depth`
- **Домен өлшемі**: `k = 6`
- **Артқы бет**: макарон өнімдерінің үстіндегі мөлдір Halo2/IPA (ZK1 конверт: VK үшін `IPAK` + `H2VK`, дәлелдеу үшін `PROF` + `I10P`)
- **Куәгердің пішіні**:
  - `v ∈ {0,1}` бюллетень бит
  - кездейсоқтық скаляр `ρ`
  - Меркле жолына арналған сегіз бауырлас скаляр
  - бағыт биттері (анықтамалық куәліктердегі барлық нөл)
- **Merkle компрессоры**: `H(x, y) = 2·(x + 7)^5 + 3·(y + 13)^5 (mod p)`, мұнда `p` — макарон өнімдерінің скалярлық модулі
- **Қоғамдық мәліметтер**:
  - 0 баған: `commit`
  - 1-баған: Меркле түбірі
  - `I10P` TLV (`cols = 2`, `rows = 1`) арқылы экспозиция

### Схема орналасуы

- **Кеңес бағандары**:
  - `v` – логикалық болуы шектелген бюллетень бит.
  - `ρ` – дауыс беру міндеттемесінде пайдаланылған соқыр скаляр.
  - `i ∈ [0, 7]` үшін `sibling[i]` – `i` тереңдіктегі Merkle жол элементі.
  - `dir[i]` `i ∈ [0, 7]` үшін – бағыт бит таңдауы солға (`0`) немесе оңға (`1`) тармақ.
  - `i ∈ [0, 7]` үшін `node[i]` – `i` тереңдіктен кейінгі Merkle аккумуляторы.
- **Дана бағандары**:
  - `commit` – сайлаушы жариялаған жария міндеттеме.
  - `root` – Дауыс берушілер жинағының Merkle түбірі.
- **Таңдаушы**: `s_vote` жалғыз толтырылған жолдағы қақпаны қосады.

Барлық кеңес ұяшықтары аймақтың бірінші (және жалғыз) қатарында тағайындалады; схема `SimpleFloorPlanner` пайдаланады.

### Қақпа жүйесі

`H` жоғарыда анықталған компрессор және `prev_0 = H(v, ρ)` болсын. Қақпа мыналарды қамтамасыз етеді:

1. `s_vote · v · (v - 1) = 0` – логикалық бюллетень бит.
2. `s_vote · (H(v, ρ) - commit) = 0` – міндеттеменің сәйкестігі.
3. Әрбір тереңдік үшін `i`:
   - `s_vote · dir[i] · (dir[i] - 1) = 0` – логикалық жол бағыты.
   - `left = H(prev_i, sibling[i])`
   - `right = H(sibling[i], prev_i)`
   - `expected = (1 - dir[i]) · left + dir[i] · right`
   - `s_vote · (node[i] - expected) = 0`
   - `prev_{i+1} = node[i]`
4. `s_vote · (prev_8 - root) = 0` – аккумулятор жалпы Merkle түбіріне тең.

Компрессор тек квинтикалық пішіндерді пайдаланады; іздеу кестелері қажет емес. Барлық арифметика Паста скаляр өрісінде орындалады және `k = 6` жолдар саны `2^k = 64` жолдарын бөледі — тек нөлдік жол толтырылады.

### Канондық арматура

Детерминирленген белдік (`zk_testkit::vote_merkle8_bundle`) куәгерді толтырады:

- `v = 1`
- `ρ = 12345`
- `sibling[i] = 10 + i` `i ∈ [0, 7]` үшін
- `dir[i] = 0`
- `node[i] = H(node[i-1], sibling[i])` `node[-1] = H(v, ρ)` бар

Бұл қоғамдық құндылықтарды тудырады:

```text
commit = 0x20574662a58708e02e0000000000000000000000000000000000000000000000
root   = 0xb63752ff429362c3a9b3cd5966c23567fdb757ce3b38af724b9303a5ea2f5817
```

Тексеру кілті тізілімінде жазылған `public_inputs_schema_hash` `blake2b-256(commit_bytes || root_bytes)`, ең аз маңызды бит `1` мәніне мәжбүрлеп, келесі нәтиже береді:

```text
public_inputs_schema_hash = 0xfae4cbe786f280b4e2184dbb06305fe46b7aee20464c0be96023ffd8eac064d3
```

### Кілт жазбасын тексеру

Басқару тексерушіні келесідей тіркейді:- `backend = "halo2/pasta/ipa-v1/vote-bool-commit-merkle8-v1"`
- `circuit_id = "halo2/pasta/vote-bool-commit-merkle8-v1"`
- `backend tag = BackendTag::Halo2IpaPasta`
- `curve = "pallas"`
- `public_inputs_schema_hash = 0xfae4…64d3`
- `commitment = sha256(backend || vk_bytes)` (32 байт дайджест)

Канондық жинақ дәлелдеу конвертімен бірге кірістірілген растау кілтін (`key = Some(VerifyingKeyBox { … })`) қамтиды. `vk_len`, `max_proof_bytes` және қосымша метадеректер URI мекенжайлары жасалған артефактілерден толтырылады.

## Анықтамалық құрылғылар

Біріктіру сынақтары арқылы тұтынылатын кірістірілген растау кілтін және дәлелдеу бумасын қайта жасау үшін `cargo xtask zk-vote-tally-bundle --print-hashes` пайдаланыңыз (әдепкі бойынша `fixtures/zk/vote_tally/` нұсқасын шығарады). Пәрмен қысқа қорытындыны (`backend`, `commit`, `root`, схема хэші, ұзындықтар) және міндетті түрде файл хэштерін басып шығарады, осылайша аудиторлар аттестаттау жазбаларын түсіре алады. JSON сияқты деректерді шығару үшін `--summary-json -` өтіңіз (немесе оны дискіге жазу жолын беріңіз). `--attestation attestation.json` (немесе stdout үшін `-`) арқылы қорытындыны және әрбір бума артефакті үшін Blake2b-256 дайджесттері мен өлшемдерін қамтитын Norito JSON манифестін жазу үшін аттестаттау пакеттерін бекітумен мұрағаттау үшін өткізіңіз. `--verify` арқылы іске қосылғанда, `--attestation <path>` қамтамасыз ету манифест бумасының метадеректері мен артефакт ұзындықтарының жаңадан қалпына келтірілген бумаға сәйкестігін тексереді (ол транскрипттің кездейсоқтығымен өзгеретін әрбір іске қосу дәлелдемесін салыстырмайды).

Канондық қондырғыларды қалпына келтіріп, көрсетіңіз:

```bash
cargo xtask zk-vote-tally-bundle \
  --out fixtures/zk/vote_tally \
  --print-hashes \
  --attestation fixtures/zk/vote_tally/bundle.attestation.json
```

Тіркелген артефактілердің ағымдағы күйінде қалуын тексеріңіз (арнатура каталогында негізгі топтама болуын талап етеді):

```bash
cargo xtask zk-vote-tally-bundle \
  --out fixtures/zk/vote_tally \
  --verify \
  --attestation fixtures/zk/vote_tally/bundle.attestation.json
```

Ағымдағы манифестті канондық артефактілеріңіздің жанында сақтаңыз (мысалы, `fixtures/zk/vote_tally/bundle.attestation.json`). Жоғары ағынды репозиторий үлкен екілік бумаларды жасамау үшін бұл каталогты бос сақтайды, сондықтан `--verify` қолданбас бұрын оны жергілікті түрде себіңіз.

`generated_unix_ms` анықтаушы түрде міндеттеме/тексеру кілті саусақ ізінен алынған, сондықтан ол регенерацияларда тұрақты болып қалады. Генератор бекітілген ChaCha20 транскрипциясын пайдаланады, сондықтан метадеректер, тексеру кілті және дәлелдеу конвертінің хэштері қайталануы мүмкін. Кез келген дайджест сәйкессіздігі енді зерттелуі керек дрейфті көрсетеді. Аудиторлар шығарылған мәндерді өздері растайтын артефактілермен бірге жазуы керек.

Жұмыс процесі туралы еске салғыш:

1. Буманы жергілікті түрде себу үшін `cargo xtask zk-vote-tally-bundle --out fixtures/zk/vote_tally --print-hashes --attestation fixtures/zk/vote_tally/bundle.attestation.json` іске қосыңыз.
2. Қажет болған жағдайда алынған артефактілерді тапсырыңыз немесе мұрағаттаңыз.
3. Аттестацияның канондық бумаға сәйкес келетініне көз жеткізу үшін келесі регенерацияларда `--verify` пайдаланыңыз.

Ішкі тапсырма `xtask/src/vote_tally.rs` ішінде детерминирленген генераторды іске қосады, ол:

1. Куәгерлердің үлгілері (`v = 1`, `ρ = 12345`, ағайындылар `10..17`)
2. `keygen_vk`/`keygen_pk` іске қосады
3. Halo2 дәлелін жасайды және оны ZK1 конвертіне (қоғамдық үлгілерді қоса) орады.
4. Сәйкес `public_inputs_schema_hash` көмегімен растаушы кілт жазбасын шығарады

## Бұзушылықты жабу

`crates/iroha_core/tests/zk_vote_tally_audit.rs` буманы жүктеп, тексереді:

- Шынайы дәлел жинақталған кірістірілген VK-ға қарсы тексереді.
- Міндеттеме бағанындағы кез келген байтты аудару тексерудің сәтсіз аяқталуына әкеледі.
- Түбірлік бағандағы кез келген байтты аудару тексерудің сәтсіз аяқталуына әкеледі.Бұл регрессия сынақтары Torii (және хосттар) дәлелдемелерді жасағаннан кейін жалпы кірістері бұрмаланған конверттерді қабылдамайтынына кепілдік береді.

Регрессияны жергілікті түрде іске қосыңыз:

```bash
cargo test -p iroha_core zk_vote_tally_audit -- --nocapture
```

## Аудитті тексеру парағы

1. Шектеудің толықтығы мен тұрақты таңдау үшін `VoteBoolCommitMerkle::<8>` қарап шығыңыз.
2. VK/дәлелді қайта шығару және жазылған хэштерді растау үшін `cargo xtask zk-vote-tally-bundle --verify --print-hashes` қайта іске қосыңыз.
3. Torii есеп өңдеушісі бірдей сервер идентификаторы мен конверт орналасуын пайдаланатынын растаңыз.
4. Мутацияланған дәлелдемелердің тексерілмейтініне көз жеткізу үшін бұрмалау регрессиясын орындаңыз.
5. `bundle.attestation.json` шығысын (Blake2b-256) хэштеңіз және ғайбаттаңыз, осылайша шолушылар өздерінің аттестацияларымен бірге канондық манифестті тіркей алады.

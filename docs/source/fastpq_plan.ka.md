# FASTPQ პროვერის მუშაობის ავარია

ეს დოკუმენტი ასახავს ეტაპობრივ გეგმას წარმოებისთვის მზა FASTPQ-ISI პროვერის მიწოდებისა და მონაცემთა სივრცის დაგეგმვის მილსადენში გაყვანილობისთვის. ქვემოთ მოყვანილი ყველა განმარტება ნორმატიულია, თუ არ არის მონიშნული, როგორც TODO. სავარაუდო სიმტკიცე იყენებს კაიროს სტილის DEEP-FRI საზღვრებს; ავტომატური უარყოფა-შერჩევის ტესტები CI-ში ვერ ხერხდება, თუ გაზომილი ზღვარი დაეცემა 128 ბიტზე დაბლა.

## სტადია 0 — ჰეშის ადგილი (დაშვება)
- დეტერმინისტული Norito კოდირება BLAKE2b ვალდებულებით.
- ადგილის მფლობელის უკანა ნაწილი, რომელიც აბრუნებს `BackendUnavailable`.
- `fastpq_isi`-ის მიერ მოწოდებული კანონიკური პარამეტრების ცხრილი.

## ეტაპი 1 - Trace Builder Prototype

> ** სტატუსი (2025-11-09):** `fastpq_prover` ახლა ამჟღავნებს კანონიკურ შეფუთვას
> დამხმარეები (`pack_bytes`, `PackedBytes`) და დეტერმინისტული პოსეიდონი2
> ვალდებულების შეკვეთა Goldilocks-ზე. მუდმივები მიმაგრებულია
> `ark-poseidon2` commit `3f2b7fe`, შემდგომი დაკვირვების დახურვა შუალედური BLAKE2-ის გამოცვლის შესახებ
> ჩანაცვლების ველი დახურულია. ოქროს მოწყობილობები (`tests/fixtures/packing_roundtrip.json`,
> `tests/fixtures/ordering_hash.json`) ახლა დაამაგრეთ რეგრესიის ნაკრები.

### მიზნები
- განახორციელეთ FASTPQ კვალის შემქმნელი KV-განახლების AIR-ისთვის. თითოეული მწკრივი უნდა იყოს კოდირებული:
  - `key_limbs[i]`: ფუძე-256 კიდურები (7 ბაიტი, პატარა-ენდიანი) კანონიკური საკვანძო ბილიკის.
  - `value_old_limbs[i]`, `value_new_limbs[i]`: იგივე შეფუთვა წინასწარ/პოსტ მნიშვნელობებისთვის.
  - ამომრჩეველი სვეტები: `s_active`, `s_transfer`, `s_mint`, `s_burn`, `s_role_grant`, `s_role_revoke`, I180NI67, I180NI67, `s_perm`.
  - დამხმარე სვეტები: `delta = value_new - value_old`, `running_asset_delta`, `metadata_hash`, `supply_counter`.
  - აქტივების სვეტები: `asset_id_limbs[i]` 7-ბაიტიანი კიდურების გამოყენებით.
  - SMT სვეტები `ℓ` დონეზე: `path_bit_ℓ`, `sibling_ℓ`, `node_in_ℓ`, `node_out_ℓ`, პლუს `neighbour_leaf` არაწევრებისთვის.
  - მეტამონაცემების სვეტები: `dsid`, `slot`.
- **დეტერმინისტული დალაგება.** რიგები ლექსიკოგრაფიულად დაალაგეთ `(key_bytes, op_rank, original_index)`-ით სტაბილური დალაგების გამოყენებით. `op_rank` რუქა: `transfer=0`, `mint=1`, `burn=2`, `role_grant=3`, `role_revoke=4`, Prometheus `original_index` არის 0-ზე დაფუძნებული ინდექსი დახარისხებამდე. შეასრულეთ მიღებული Poseidon2-ის შეკვეთის ჰეში (დომენის ტეგი `fastpq:v1:ordering`). დაშიფვრეთ ჰეშის პრეიმიჯი, როგორც `[domain_len, domain_limbs…, payload_len, payload_limbs…]`, სადაც სიგრძე არის u64 ველის ელემენტები, ასე რომ, ნულოვანი ბაიტი რჩება გამორჩეული.
- საძიებო მოწმე: შექმენით `perm_hash = Poseidon2(role_id || permission_id || epoch_u64_le)`, როდესაც შენახული სვეტი `s_perm` (ლოგიკური OR `s_role_grant` და `s_role_revoke`) არის 1. როლის/ნებართვის ID არის ფიქსირებული სიგანის 32-ბაიტი; ეპოქა არის 8 ბაიტი LE.
- განახორციელეთ უცვლელები როგორც AIR-მდე, ისე შიგნით: სელექტორები ურთიერთგამომრიცხავი, თითო აქტივის კონსერვაცია, dsid/slot მუდმივები.
- `N_trace = 2^k` (`pow2_ceiling` მწკრივების რაოდენობა); `N_eval = N_trace * 2^b` სადაც `b` არის აფეთქების მაჩვენებელი.
- მიაწოდეთ მოწყობილობები და ქონების ტესტები:
  - ორმხრივი მგზავრობის შეფუთვა (`fastpq_prover/tests/packing.rs`, `tests/fixtures/packing_roundtrip.json`).
  - შეკვეთის სტაბილურობის ჰეში (`tests/fixtures/ordering_hash.json`).

- ** ველის შეფუთვა:** ბაიტი იშლება 7-ბაიტიან ნაწილებად (პატარა-ენდიანი). თითოეული კიდური `limb_j = Σ_{k=0}^{6} byte_{7j+k} * 256^k`; კიდურების უარყოფა ≥ Goldilocks მოდული.
- **ბალანსი/კონსერვაცია:** ნება `δ = value_new - value_old`. დააჯგუფეთ რიგები `asset_id`-ით. განსაზღვრეთ `r_asset_start = 1` თითოეული აქტივის ჯგუფის პირველ რიგში (0 სხვაგვარად) და შეზღუდეთ
  ```
  running_asset_delta = (1 - r_asset_start) * running_asset_delta_prev + δ.
  ```
  ყოველი აქტივების ჯგუფის ბოლო რიგში ამტკიცებენ
  ```
  running_asset_delta = Σ (s_mint * δ) - Σ (s_burn * δ).
  ```
  გადარიცხვები აკმაყოფილებენ შეზღუდვას ავტომატურად, რადგან მათი δ მნიშვნელობები ჯგუფში არის ნულის ჯამი. მაგალითი: თუ `value_old = 100` და `value_new = 120` ზარაფხანის მწკრივზე, δ = 20, ასე რომ, ზარაფხანის ჯამი ხელს უწყობს +20 და საბოლოო შემოწმება წყდება ნულამდე, როდესაც დამწვრობა არ მოხდება.
- **შეფუთვა:** წარმოგიდგენთ `s_active`. გაამრავლეთ მწკრივის ყველა შეზღუდვა `s_active`-ზე და განახორციელეთ მიმდებარე პრეფიქსი: `s_active[i] ≥ s_active[i+1]`. დამტენი რიგები (`s_active=0`) უნდა შეინარჩუნოს მუდმივი მნიშვნელობები, მაგრამ სხვაგვარად შეუზღუდავია.
- ** ჰეშის შეკვეთა:** Poseidon2 ჰეში (დომენი `fastpq:v1:ordering`) მწკრივების კოდირებით; ინახება საჯარო IO-ში აუდიტორობისთვის.

## ეტაპი 2 - STARK Prover Core

### მიზნები
- შექმენით Poseidon2 Merkle-ის ვალდებულებები კვალის და საძიებო შეფასების ვექტორებზე. პარამეტრები: სიხშირე=2, ტევადობა=1, სრული რაუნდები=8, ნაწილობრივი რაუნდები=57, მუდმივები მიმაგრებულია `ark-poseidon2` commit `3f2b7fe` (v0.3.0).
- დაბალი ხარისხის გაფართოება: შეაფასეთ თითოეული სვეტი `D = { g^i | i = 0 .. N_eval-1 }` დომენზე, სადაც `N_eval = 2^{k+b}` ყოფს Goldilocks-ის 2-ადის მოცულობას. მოდით, `g = ω^{(p-1)/N_eval}` ერთად `ω` Goldilocks-ის ფიქსირებული პრიმიტიული ფესვი და `p` მისი მოდული; გამოიყენეთ საბაზისო ქვეჯგუფი (კოსეტის გარეშე). ჩაწერეთ `g` ტრანსკრიპტში (ტეგი `fastpq:v1:lde`).
- შემადგენლობის პოლინომები: თითოეული შეზღუდვისთვის `C_j`, ჩამოაყალიბეთ `F_j(X) = C_j(X) / Z_N(X)` ქვემოთ ჩამოთვლილი ხარისხის მინდვრებით.
- საძიებო არგუმენტი (ნებართვები): ნიმუში `γ` ტრანსკრიპტიდან. კვალი პროდუქტი `Z_0 = 1`, `Z_i = Z_{i-1} * (perm_hash_i - γ)^{s_perm_i}`. მაგიდის პროდუქტი `T = ∏_j (table_perm_j - γ)`. სასაზღვრო შეზღუდვა: `Z_final / T = 1`.
- DEEP-FRI arity `r ∈ {8, 16}`: თითოეული ფენისთვის შეიწოვეთ ფესვი ტეგით `fastpq:v1:fri_layer_ℓ`, ნიმუში `β_ℓ` (ტეგი `fastpq:v1:beta_ℓ`) და გადაკეცეთ `H_{ℓ+1}(i) = Σ_{k=0}^{r-1} H_ℓ(r*i + k) * β_ℓ^k` მეშვეობით.
- მტკიცებულების ობიექტი (Norito-ში კოდირებული):
  ```
  Proof {
      protocol_version: u16,
      params_version: u16,
      parameter_set: String,
      public_io: PublicIO,
      trace_root: [u8; 32],
      lookup_root: [u8; 32],
      fri_layers: Vec<[u8; 32]>,
      alphas: Vec<Field>,
      betas: Vec<Field>,
      queries: Vec<QueryOpening>,
  }
  ```
- Verifier Mirrors prover; გაუშვით რეგრესიის ნაკრები 1k/5k/20k მწკრივის კვალზე ოქროს ტრანსკრიპტით.

- **ბაიტი შეფუთვა:** ბაზა-256 (7-ბაიტი კიდურები, პატარა-ენდიანი). ტესტები `fastpq_prover/tests/packing.rs`-ში.
- **ველის კოდირება:** კანონიკური Goldilocks (პატარა-ენდიანი 64-ბიტიანი კიდური, უარყოფა ≥ p); Poseidon2 გამომავალი/SMT ფესვები სერიული 32-ბაიტიანი პატარა ენდიის მასივების სახით.
- **ტრანსკრიპტი (ფიატ-შამირი):**
  1. BLAKE2b შთანთქავს `protocol_version`, `params_version`, `parameter_set`, `public_io` და Poseidon2 commit tag (`fastpq:v1:init`).
  2. შთანთქავს `trace_root`, `lookup_root` (`fastpq:v1:roots`).
  3. ამოიღეთ საძიებო გამოწვევა `γ` (`fastpq:v1:gamma`).
  4. ამოიღეთ კომპოზიციის გამოწვევები `α_j` (`fastpq:v1:alpha_j`).
  5. თითოეული FRI ფენის ფესვისთვის, შთანთქავს `fastpq:v1:fri_layer_ℓ`-ით, გამოყავით `β_ℓ` (`fastpq:v1:beta_ℓ`).
  6. შეკითხვის ინდექსების გამოყვანა (`fastpq:v1:query_index`).

  ტეგები არის პატარა ASCII; ვერიფიკატორები უარყოფენ შეუსაბამობებს შერჩევის გამოწვევების მიღებამდე. ოქროს ტრანსკრიპტის მოწყობილობა: `tests/fixtures/transcript_v1.json`.
- **ვერსია:** `protocol_version = 1`, `params_version` ემთხვევა `fastpq_isi` პარამეტრთა კომპლექტს.

## საძიებო არგუმენტი (ნებართვები)
- დავალებული ცხრილი დალაგებულია ლექსიკოგრაფიულად `(role_id_bytes, permission_id_bytes, epoch_le)`-ით და ჩადენილი Poseidon2 Merkle-ის ხის მეშვეობით (`perm_root` `PublicIO`-ში).
- კვალი მოწმე იყენებს `perm_hash` და ამომრჩეველს `s_perm` (ან როლის მინიჭების/გაუქმების). ტუპლი დაშიფრულია როგორც `role_id_bytes || permission_id_bytes || epoch_u64_le` ფიქსირებული სიგანეებით (32, 32, 8 ბაიტი).
- პროდუქტის ურთიერთობა:
  ```
  Z_0 = 1
  for each row i: Z_i = Z_{i-1} * (perm_hash_i - γ)^{s_perm_i}
  T = ∏_j (table_perm_j - γ)
  ```
  საზღვრის დამტკიცება: `Z_final / T = 1`. იხილეთ `examples/lookup_grand_product.md` ბეტონის აკუმულატორის სანახავად.

## იშვიათი მერკლის ხის შეზღუდვები
- განსაზღვრეთ `SMT_HEIGHT` (დონეების რაოდენობა). სვეტები `path_bit_ℓ`, `sibling_ℓ`, `node_in_ℓ`, `node_out_ℓ`, `neighbour_leaf` გამოჩნდება ყველა `ℓ ∈ [0, SMT_HEIGHT)`-სთვის.
- Poseidon2 პარამეტრები მიმაგრებულია `ark-poseidon2` commit `3f2b7fe` (v0.3.0); დომენის ტეგი `fastpq:v1:poseidon_node`. ყველა კვანძი იყენებს მცირე ენდიან ველების დაშიფვრას.
- განაახლეთ წესები თითო დონეზე:
  ```
  if path_bit_ℓ == 0:
      node_out_ℓ = Poseidon2(node_in_ℓ, sibling_ℓ)
  else:
      node_out_ℓ = Poseidon2(sibling_ℓ, node_in_ℓ)
  ```
- ჩანართების ნაკრები `(node_in_0 = 0, node_out_0 = value_new)`; შლის კომპლექტს `(node_in_0 = value_old, node_out_0 = 0)`.
- არაწევრობის დამადასტურებელი საბუთები მიაწოდეთ `neighbour_leaf`, რათა აჩვენოს, რომ მოთხოვნილი ინტერვალი ცარიელია. იხილეთ `examples/smt_update.md` დამუშავებული მაგალითისა და JSON განლაგებისთვის.
- სასაზღვრო შეზღუდვა: საბოლოო ჰეში უდრის `old_root` წინა რიგებისთვის და `new_root` პოსტ რიგებისთვის.

## 1. Описание обработки

- **Цель:** Обеспечение оперативной телеметрии, необходимой для поддержки наблюдаемости AND7 (задержка, повторные попытки, работоспособность аттестации) при одновременном зеркалировании схемы узла Rust (раздел 2 `telemetry_redaction.md`).
- **Системы:** Инструментарий Android SDK -> экспортер OTLP -> общий сборщик телеметрии, управляемый SRE (см. раздел «Пособие по поддержке» §8).
- **Субъекты данных:** сотрудники операторов, использующие приложения на базе Android SDK; нисходящие конечные точки Torii (строки полномочий хешируются в соответствии с политикой телеметрии).

## 3. Законные основания и права

- **Правовое основание:** Законный интерес (ст. 6(1)(f)) — обеспечение надежной работы клиентов регулируемого реестра.
– **Тест необходимости:** метрики ограничиваются работоспособностью (без пользовательского контента); хешированные полномочия обеспечивают четность с узлами Rust посредством обратимого сопоставления, доступного только авторизованному персоналу службы поддержки (через рабочий процесс переопределения).
- **Тест балансировки.** Телеметрия распространяется на устройства, управляемые оператором, а не на данные конечного пользователя. Для переопределения требуются подписанные артефакты Norito, проверенные службой поддержки и соответствия (Справочник поддержки §3 + §9).
- **Права субъектов данных:** операторы обращаются в службу технической поддержки (сборник §2), чтобы запросить экспорт/удаление телеметрии. Отмена изменений и журналы (документ телеметрии §Signal Inventory) позволяют выполнить их в течение 30 дней.

| Отменить неправомерное использование утечки личных данных | Очень низкий | Высокий | Запросы на переопределение Norito зарегистрированы, срок действия истекает в течение 24 часов, требуется одобрение SRE (`docs/source/android_runbook.md` §3). |

Остаточный риск считается приемлемым с учетом вышеуказанных мер контроля и постоянного мониторинга.

## 5. Действия и последующие действия

1. **Ежеквартальный обзор:** проверка схем телеметрии, ротации соли и переопределение журналов; документ в `docs/source/sdk/android/telemetry_redaction_minutes_YYYYMMDD.md`.
2. **Согласование нескольких SDK.** Координируйте свои действия с сопровождающими Swift/JS для поддержания единообразных правил хеширования/группирования (отслеживается в дорожной карте AND7).
3. **Общение с партнерами:** Включите сводку DPIA в комплекты для знакомства с партнерами (раздел 9 руководства по поддержке) и ссылку на этот документ из `status.md`.

## 1. TOE აღწერა

შეფასების სამიზნე (TOE) მოიცავს Android SDK ბიბლიოთეკის კოდს (`java/iroha_android/src/main/java`), მის კონფიგურაციის ზედაპირს (`ClientConfig` + Norito გადაყლაპვა) და ოპერაციულ ინსტრუმენტს, რომელიც მითითებულია Sigstore-ში, მითითებულ Sigstore-ში Sigstore-ში მითითებულ Sigstore-ში.

ძირითადი კომპონენტები:

1. **კონფიგურაციის გადაყლაპვა** — `ClientConfig` ძაფები Torii ბოლო წერტილები, TLS წესები, ხელახალი ცდები და ტელემეტრიული კაუჭები გენერირებული `iroha_config` მანიფესტიდან და აიძულებს შეუცვლელობას (I10800X-ის შემდგომ)
2. **გასაღების მენეჯმენტი / StrongBox** — აპარატურაზე მხარდაჭერილი ხელმოწერა ხორციელდება `SystemAndroidKeystoreBackend` და `AttestationVerifier`-ის მეშვეობით, `docs/source/sdk/android/key_management.md`-ში დოკუმენტირებული პოლიტიკით. ატესტაციის აღება/დამოწმება იყენებს `scripts/android_keystore_attestation.sh` და CI დამხმარე `scripts/android_strongbox_attestation_ci.sh`.
3. **ტელემეტრია და რედაქცია** — ინსტრუმენტაციის ძაბრები `docs/source/sdk/android/telemetry_redaction.md`-ში აღწერილი გაზიარებული სქემის მეშვეობით, ახორციელებს ჰეშირებული ავტორიტეტების ექსპორტს, თაიგულების მქონე მოწყობილობის პროფილებს და უგულებელყოფს აუდიტორულ კაუჭებს, რომლებიც აღსრულებულია მხარდაჭერის წიგნში.
4. **ოპერაციების runbooks** — `docs/source/android_runbook.md` (ოპერატორის პასუხი) და `docs/source/android_support_playbook.md` (SLA + ესკალაცია) აძლიერებს TOE-ის საოპერაციო კვალს დეტერმინისტული გადახრით, ქაოსური ვარჯიშებით და მტკიცებულებების აღებით.
5. **გამოშვების წარმოშობა** — Gradle-ზე დაფუძნებული კონსტრუქციები იყენებენ CycloneDX დანამატს პლუს რეპროდუცირებადი build flags-ს, როგორც აღბეჭდილია `docs/source/sdk/android/developer_experience_plan.md`-ში და AND6 შესაბამისობის საკონტროლო სიაში. გამოშვების არტეფაქტები ხელმოწერილია და ჯვარედინი მითითებებია `docs/source/release/provenance/android/`-ში.

## 4. შეფასების აქტივობები

1. **დიზაინის მიმოხილვა** — შესაბამისობა + SRE ადასტურებს, რომ კონფიგურაცია, გასაღების მართვა, ტელემეტრია და გამოშვების კონტროლი შეესაბამება ETSI უსაფრთხოების მიზნებს.
2. **განხორციელების შემოწმებები** — ავტომატური ტესტები:
   - `scripts/android_strongbox_attestation_ci.sh` ამოწმებს დაფიქსირებულ პაკეტებს მატრიცაში ჩამოთვლილი ყველა StrongBox მოწყობილობისთვის.
   - `scripts/check_android_samples.sh` და მართული მოწყობილობა CI უზრუნველყოფს, რომ ნიმუშის აპები დაიცვან `ClientConfig`/ტელემეტრიის კონტრაქტები.
3. **ოპერატიული ვალიდაცია** — კვარტალური ქაოსური წვრთნები `docs/source/sdk/android/telemetry_chaos_checklist.md`-ზე (რედაქცია + გადაფარვის სავარჯიშოები).
4. **მტკიცებულებების შენახვა** — არტეფაქტები, რომლებიც ინახება `docs/source/compliance/android/`-ში (ამ საქაღალდეში) და მითითებულია `status.md`-დან.

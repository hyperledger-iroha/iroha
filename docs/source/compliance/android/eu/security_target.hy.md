## 1. ՈՏԻ Նկարագրություն

Գնահատման թիրախը (TOE) ներառում է Android SDK գրադարանի կոդը (`java/iroha_android/src/main/java`), դրա կազմաձևման մակերեսը (`ClientConfig` + Norito մուտքագրում) և գործառնական գործիքավորումը, որը նշված է Sigstore-ում, նշված է Sigstore-ում։

Առաջնային բաղադրիչներ.

1. **Կազմաձևի կլանում** — `ClientConfig` շղթաներ Torii վերջնակետեր, TLS կանոններ, կրկնվող փորձեր և հեռաչափական կեռիկներ գեներացված `iroha_config` մանիֆեստից և պարտադրում է անփոփոխելիությունը հետսկզբնական00000018-ից:
2. **Բանալինների կառավարում / StrongBox** — Սարքավորումներով ապահովված ստորագրումն իրականացվում է `SystemAndroidKeystoreBackend` և `AttestationVerifier`-ի միջոցով՝ `docs/source/sdk/android/key_management.md`-ում փաստաթղթավորված քաղաքականությամբ: Ատեստավորման հավաքագրման/վավերացման համար օգտագործվում է `scripts/android_keystore_attestation.sh` և CI օգնական `scripts/android_strongbox_attestation_ci.sh`:
3. **Հեռաչափություն և խմբագրում** — Գործիքավորումը անցնում է `docs/source/sdk/android/telemetry_redaction.md`-ում նկարագրված ընդհանուր սխեմայի միջոցով, արտահանում է հաշված հեղինակություններ, սարքի պրոֆիլներ դույլերով և անտեսում աուդիտի կեռիկներ, որոնք ամրագրված են Աջակցման գրքույկում:
4. **Գործողությունների runbooks** — `docs/source/android_runbook.md` (օպերատորի պատասխան) ​​և `docs/source/android_support_playbook.md` (SLA + էսկալացիա) կոշտացնում են TOE-ի գործառնական հետքը՝ որոշիչ վերացումներով, քաոսային վարժանքներով և ապացույցների հավաքագրմամբ:
5. **Թողարկման ծագումը** — Gradle-ի վրա հիմնված կառուցումները օգտագործում են CycloneDX հավելվածը և վերարտադրվող կառուցման դրոշները, ինչպես նշված է `docs/source/sdk/android/developer_experience_plan.md`-ում և AND6 համապատասխանության ստուգաթերթում: Թողարկման արտեֆակտները ստորագրված են և խաչաձև հղումով `docs/source/release/provenance/android/`-ում:

Ենթադրություններ:

- Mobile OS-ն ապահովում է ստանդարտ sandboxing + SELinux; StrongBox սարքերն օգտագործում են Google-ի keymaster ինտերֆեյսը:
- Օպերատորները տրամադրում են Torii վերջնակետեր TLS վկայագրերով, որոնք ստորագրված են խորհրդի կողմից վստահված CA-ների կողմից:
- Կառուցեք ենթակառուցվածքը, որը հարգում է վերարտադրելի կառուցման պահանջները, նախքան Maven-ում հրապարակելը:

## 4. Գնահատման գործունեություն

1. **Դիզայնի վերանայում** — Համապատասխանություն + SRE-ն ստուգում է, որ կոնֆիգուրացիան, բանալիների կառավարումը, հեռաչափությունը և թողարկման հսկիչները համապատասխանում են ETSI անվտանգության նպատակներին:
2. **Իրականացման ստուգումներ** — Ավտոմատացված թեստեր.
   - `scripts/android_strongbox_attestation_ci.sh`-ը ստուգում է գրավված փաթեթները մատրիցայում թվարկված յուրաքանչյուր StrongBox սարքի համար:
   - `scripts/check_android_samples.sh`-ը և Կառավարվող սարքը CI-ն ապահովում են նմուշային հավելվածները, որոնք հարգում են `ClientConfig`/հեռաչափության պայմանագրերը:
3. **Օպերատիվ վավերացում** — Քաոսի եռամսյակային վարժանքներ `docs/source/sdk/android/telemetry_chaos_checklist.md`-ի համար (վերախմբում + վերացման վարժություններ):
4. **Ապացույցների պահպանում** — Արտեֆակտներ, որոնք պահվում են `docs/source/compliance/android/`-ում (այս թղթապանակում) և հղում են կատարում `status.md`-ից:

| 7.5 Կրիպտոգրաֆիկ հսկողություն | Բանալինների արտադրության, պահպանման և ատեստավորման պահանջները AND2-ից (բանալների կառավարման ուղեցույց): |
| 7.6 Գործառնությունների անվտանգություն | Հեռաչափության հաշինգ, քաոսի փորձեր, միջադեպերի արձագանքում և ապացույցների թողարկում: |
| 7.7 Կապի անվտանգություն | `/v2/pipeline` TLS քաղաքականություն + հաշված հեղինակություններ (հեռաչափության խմբագրման փաստաթուղթ): |
| 7.8 Համակարգի ձեռքբերում / մշակում | Վերարտադրվող Gradle-ի շինությունները, SBOM-ները և ծագման դարպասները AND5/AND6 պլաններում: |

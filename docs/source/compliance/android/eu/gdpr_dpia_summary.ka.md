## 1. დამუშავების აღწერა

- **მიზანი:** უზრუნველყოს ოპერატიული ტელემეტრია, რომელიც საჭიროა AND7 დაკვირვებადობის მხარდასაჭერად (დაყოვნება, განმეორებითი ცდები, ატესტაციის სიჯანსაღე) Rust კვანძის სქემის არეკვლისას (§2 `telemetry_redaction.md`-დან).
- **სისტემები:** Android SDK ინსტრუმენტაცია -> OTLP ექსპორტიორი -> საერთო ტელემეტრიის კოლექტორი, რომელსაც მართავს SRE (იხ. მხარდაჭერის წიგნი §8).
- **მონაცემთა სუბიექტები:** ოპერატორის პერსონალი Android SDK-ზე დაფუძნებული აპების გამოყენებით; ქვემოთ Torii ბოლო წერტილები (ავტორიტეტის სტრიქონები ჰეშირდება ტელემეტრიის პოლიტიკის მიხედვით).

## 3. კანონიერი საფუძველი და უფლებები

- **კანონიერი საფუძველი:** ლეგიტიმური ინტერესი (მუხლი 6(1)(ვ)) — რეგულირებადი წიგნის კლიენტების საიმედო მუშაობის უზრუნველყოფა.
- ** აუცილებლობის ტესტი: ** მეტრიკა შემოიფარგლება ოპერაციული სიჯანსაღით (მომხმარებლის კონტენტის გარეშე); ჰეშირებული ავტორიტეტი უზრუნველყოფს Rust კვანძებთან პარიტეტს შექცევადი რუკების საშუალებით, რომელიც ხელმისაწვდომია მხოლოდ ავტორიზებული დამხმარე პერსონალისთვის (გადალახვის სამუშაო პროცესის მეშვეობით).
- **ბალანსირების ტესტი:** ტელემეტრია ვრცელდება ოპერატორის მიერ კონტროლირებად მოწყობილობებზე და არა საბოლოო მომხმარებლის მონაცემებზე. უგულებელყოფისთვის საჭიროა ხელმოწერილი Norito არტეფაქტები, რომლებიც განიხილება მხარდაჭერა + შესაბამისობა (მხარდაჭერის სათამაშო წიგნი §3 + §9).
- **მონაცემთა სუბიექტის უფლებები:** ოპერატორები დაუკავშირდნენ მხარდაჭერა ინჟინერიას (სათამაშო წიგნი §2), რათა მოითხოვონ ტელემეტრიის ექსპორტი/წაშლა. რედაქციის უგულებელყოფა და ჟურნალები (ტელემეტრიის დოკუმენტი §სიგნალის ინვენტარი) იძლევა 30 დღის განმავლობაში შესრულებას.

## 5. მოქმედებები და შემდგომი მოქმედებები

1. **კვარტალური მიმოხილვა:** ტელემეტრიის სქემების, მარილის ბრუნვის და ჩანაწერების გადაფარვა; დოკუმენტი `docs/source/sdk/android/telemetry_redaction_minutes_YYYYMMDD.md`-ში.
2. **Cross-SDK განლაგება:** კოორდინაცია Swift/JS-ის შემსრულებლებთან, რათა შეინარჩუნოთ ჰეშინგის/თაიგულების თანმიმდევრული წესები (თვალყურედება საგზაო რუკაში AND7).
3. **პარტნიორთა შეტყობინებები:** ჩართეთ DPIA-ს შეჯამება პარტნიორის ჩართვის კომპლექტებში (მხარდაჭერით Playbook §9) და ბმული ამ დოკუმენტზე `status.md`-დან.

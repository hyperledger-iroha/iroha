# Connect Session Architecture Strawman (Swift / Android / JS)

Ծղոտե մարդու այս առաջարկը ուրվագծում է Nexus Connect աշխատանքային հոսքերի ընդհանուր դիզայնը
Swift, Android և JavaScript SDK-ներում: Այն նախատեսված է աջակցելու
Փետրվար 2026 cross-SDK սեմինար և ներգրավեք բաց հարցեր նախքան իրականացումը:

> Վերջին թարմացումը՝ 2026-01-29  
> Հեղինակներ՝ Swift SDK առաջատար, Android Networking TL, JS Lead  
> Կարգավիճակ. Ավագանու վերանայման նախագիծ (սպառնալիքի մոդել + տվյալների պահպանման հավասարեցում ավելացվել է 2026-03-12)

## Գոլեր

1. Հավասարեցնել դրամապանակը ↔ dApp աշխատաշրջանի կյանքի ցիկլը, ներառյալ կապի բեռնաթափումը,
   հաստատումներ, ստորագրման հարցումներ և քանդում:
2. Սահմանեք Norito ծրարի սխեման (բաց/հաստատել/ստորագրել/վերահսկել), որը համօգտագործվում է բոլորի կողմից
   SDK-ներ և ապահովել հավասարություն `connect_norito_bridge`-ի հետ:
3. Բաշխեք պարտականությունները տրանսպորտի (WebSocket/WebRTC), ծածկագրման միջև
   (Norito Միացրեք շրջանակներ + բանալիների փոխանակում) և կիրառական շերտեր (SDK ֆասադներ):
4. Ապահովել դետերմինիստական վարքագիծ աշխատասեղանի/բջջային հարթակներում, ներառյալ
   անցանց բուֆերացում և վերամիացում:

## նստաշրջանի կյանքի ցիկլը (բարձր մակարդակ)

```
┌────────────┐      ┌─────────────┐      ┌────────────┐
│  dApp SDK  │←────→│  Connect WS │←────→│ Wallet SDK │
└────────────┘      └─────────────┘      └────────────┘
      │                    │                    │
      │ 1. open (app→wallet) frame (metadata, permissions, chain_id)
      │────────────────────────────────────────>│
      │                    │                    │
      │                    │ 2. route frame     │
      │                    │────────────────────│
      │                    │                    │
      │                    │     3. approve frame (wallet pk, account,
      │                    │        permissions, proof/attest)
      │<────────────────────────────────────────│
      │                    │                    │
      │ 4. sign request    │                    │
      │────────────────────────────────────────>│
      │                    │                    │
      │                    │ 5. sign result     │
      │                    │────────────────────│
      │                    │                    │
      │ 6. control frames for reject/close, error propagation, heartbeats.
```

## Ծրար/Norito սխեմա

Բոլոր SDK-ները ՊԵՏՔ Է օգտագործեն `connect_norito_bridge`-ում սահմանված Norito կանոնական սխեման՝

- `EnvelopeV1` (բացել / հաստատել / ստորագրել / վերահսկել)
- `ConnectFrameV1` (գաղտնագրված տեքստի շրջանակներ՝ առանց AEAD ծանրաբեռնվածության)
- Կառավարման կոդեր.
  - `open_ext` (մետատվյալներ, թույլտվություններ)
  - `approve_ext` (հաշիվ, թույլտվություններ, ապացույցներ, ստորագրություն)
  - `reject`, `close`, `ping/pong`, `error`

Swift-ը նախկինում առաքվել է տեղապահ JSON կոդավորիչներ (`ConnectCodec.swift`): 2026 թվականի ապրիլի դրությամբ SDK-ն
միշտ օգտագործում է Norito կամուրջը և չի փակվում, երբ XCFramework-ը բացակայում է, բայց այս ծղոտե մարդը
դեռևս գրավում է մանդատը, որը հանգեցրեց կամուրջի ինտեգրմանը.

| Ֆունկցիան | Նկարագրություն | Կարգավիճակը |
|----------|-------------|--------|
| `connect_norito_encode_control_open_ext` | dApp բաց շրջանակ | Իրականացվում է կամուրջում |
| `connect_norito_encode_control_approve_ext` | Դրամապանակի հաստատում | Իրականացված |
| `connect_norito_encode_envelope_sign_request_tx/raw` | Ստորագրեք հարցումները | Իրականացված |
| `connect_norito_encode_envelope_sign_result_ok/err` | Ստորագրել արդյունքները | Իրականացված |
| `connect_norito_decode_*` | Վերլուծություն դրամապանակների/dApps-ի համար | Իրականացված |

### Պահանջվող աշխատանք

- Swift. Փոխարինեք տեղապահ `ConnectCodec` JSON օգնականները կամրջային զանգերով և մակերեսով
  տպագրված փաթաթիչներ (`ConnectFrame`, `ConnectEnvelope`)՝ օգտագործելով ընդհանուր Norito տեսակները: ✅ (Ապրիլ 2026)
- Android/JS. Համոզվեք, որ գոյություն ունեն նույն փաթաթանները; հավասարեցնել սխալի կոդերը և մետատվյալների ստեղները:
- Համօգտագործված. Փաստաթղթերի կոդավորում (X25519 բանալի փոխանակում, AEAD) բանալու հետևողական ածանցմամբ
  ըստ Norito սպեցիֆիկի և տրամադրեք նմուշների ինտեգրման թեստեր՝ օգտագործելով Rust bridge-ը:

## Տրանսպորտային պայմանագիր- Առաջնային տրանսպորտ՝ WebSocket (`/v2/connect/ws?sid=<session_id>`):
- Կամընտիր ապագա. WebRTC (TBD) – սկզբնական ծղոտի շրջանակից դուրս:
- Վերամիացման ռազմավարություն. էքսպոնենցիալ հետքայլ՝ ամբողջական ցնցումով (բազային 5 վրկ, առավելագույնը՝ 60 վրկ); ընդհանուր հաստատուններ Swift-ում, Android-ում և JS-ում, այնպես որ կրկնվող փորձերը մնում են կանխատեսելի:
- Պինգ/պոնգ կադենս. 30-ականների սրտի բաբախյունը հանդուրժողությամբ երեք բաց թողնված պոնգի համար, նախքան նորից միանալը; JS-ը սեղմում է նվազագույն միջակայքը մինչև 15 վրկ՝ բրաուզերի կլանման կանոնները բավարարելու համար:
- Կեռիկներ. Android դրամապանակի SDK-ն բացահայտում է կամընտիր FCM ինտեգրումը արթնացման համար, մինչդեռ JS-ը մնում է հարցումների վրա հիմնված (փաստագրված սահմանափակումներ զննարկչի հրում թույլտվությունների համար):
- SDK-ի պարտականությունները.
  - Պահպանեք պինգ/պոնգի սրտի բաբախյունը (խուսափեք մարտկոցների լիցքաթափումից շարժական սարքերում):
  - Բուֆերացրեք ելքային շրջանակները, երբ անցանց ռեժիմում եք (սահմանափակ հերթ, պահպանվում է dApp-ի համար):
- Տրամադրել իրադարձությունների հոսքի API (Swift Combine `AsyncStream`, Android Flow, JS async iter):
- Մակերեւութային վերամիացում կեռիկներն ու թույլ տվեք ձեռքով նորից բաժանորդագրվել:
- Հեռաչափության խմբագրում. թողարկեք միայն նստաշրջանի մակարդակի հաշվիչներ (`sid` հեշ, ուղղություն,
  հաջորդականության պատուհան, հերթի խորություն) աղերի հետ, որոնք փաստագրված են Connect telemetry-ում
  ուղեցույց; վերնագրերը/ստեղները երբեք չպետք է հայտնվեն տեղեկամատյաններում կամ վրիպազերծման տողերում:

## Գաղտնագրում և բանալիների կառավարում

### Նիստի նույնացուցիչներ և աղեր

- `sid`-ը 32 բայթանոց նույնացուցիչ է՝ ստացված `BLAKE2b-256("iroha-connect|sid|" || chain_id || app_ephemeral_pk || nonce16)`-ից:  
  DApps-ը հաշվարկում է այն նախքան `/v2/connect/session` զանգահարելը; դրամապանակներն այն արձագանքում են `approve` շրջանակներով, որպեսզի երկու կողմերն էլ կարողանան հետևողականորեն նշել ամսագրերը և հեռաչափությունը:
- Նույն աղը սնուցում է բանալիների ստացման յուրաքանչյուր քայլ, այնպես որ SDK-ները երբեք չեն հիմնվում հյուրընկալող հարթակից հավաքված էնտրոպիայի վրա:

### Ժամանակավոր բանալիների մշակում

- Յուրաքանչյուր նստաշրջան օգտագործում է թարմ X25519 հիմնական նյութ:  
  Swift-ը պահում է այն Keychain/Secure Enclave-ում՝ `ConnectCrypto`-ի միջոցով, Android դրամապանակները լռելյայն՝ StrongBox (վերադառնում է դեպի TEE-ով ապահովված բանալիների պահեստներ), իսկ JS-ին անհրաժեշտ է անվտանգ համատեքստով WebCrypto օրինակ կամ բնիկ `iroha_js_host` հավելված:
- Բաց շրջանակները ներառում են dApp ժամանակավոր հանրային բանալին և կամընտիր ատեստավորման փաթեթ: Դրամապանակի հաստատումները վերադարձնում են դրամապանակի հանրային բանալին և ցանկացած ապարատային հաստատում, որն անհրաժեշտ է համապատասխանության հոսքերի համար:
- Ատեստավորման օգտակար բեռները հետևում են ընդունված սխեմային.  
  `attestation { platform, evidence_b64, statement_hash }`.  
  Բրաուզերները կարող են բաց թողնել բլոկը. բնիկ դրամապանակները ներառում են այն ամեն անգամ, երբ օգտագործվում են ապարատային ապահովված բանալիներ:

### Ուղղորդող ստեղներ և AEAD

- Համօգտագործվող գաղտնիքները ընդլայնվում են HKDF-SHA256-ով (Rust bridge օգնականների միջոցով) և տիրույթից առանձնացված տեղեկատվական տողերով.
  - `iroha-connect|k_app` → հավելված→ դրամապանակի տրաֆիկ:
  - `iroha-connect|k_wallet` → դրամապանակ→ հավելվածի տրաֆիկ:
- AEAD-ը ChaCha20-Poly1305 է v1 ծրարի համար (`connect_norito_bridge` բացահայտում է օգնականներին յուրաքանչյուր հարթակում):  
  Համակցված տվյալները հավասար են `("connect:v1", sid, dir, seq_le, kind=ciphertext)`-ին, ուստի վերնագրերի խախտումները հայտնաբերվում են:
- Nonence-ները ստացվում են 64-բիթանոց հաջորդականության հաշվիչից (`nonce[0..4]=0`, `nonce[4..12]=seq_le`): Համօգտագործվող օգնականի թեստերն ապահովում են, որ BigInt/UInt փոխարկումները նույն կերպ են վարվում SDK-ներում:

### Պտտման և վերականգնման ձեռքսեղմում- Պտտումը մնում է ընտրովի, բայց արձանագրությունը սահմանված է. dApps-ը թողարկում է `Control::RotateKeys` շրջանակ, երբ հաջորդական հաշվիչներն են մոտենում փաթաթված պահակախմբին, դրամապանակները պատասխանում են նոր հանրային բանալիով գումարած ստորագրված հաստատում, և երկու կողմերն էլ անմիջապես ստանում են նոր ուղղորդող բանալիներ՝ առանց նիստը փակելու:
- Դրամապանակի կողմից ստեղնաշարի կորուստը առաջացնում է նույն ձեռքսեղմումը, որին հաջորդում է `resume` հսկողությունը, որպեսզի dApps-ը իմանա մաքրել քեշավորված գաղտնագրված տեքստը, որն ուղղված է անջատված բանալին:

CryptoKit-ի պատմական հետադարձ կապերի համար տե՛ս `docs/connect_swift_ios.md`; Kotlin-ը և JS-ն ունեն համապատասխան հղումներ `docs/connect_kotlin_ws*.md`-ի ներքո:

## Թույլտվություններ և ապացույցներ

- Թույլտվության մանիֆեստները պետք է շրջագայեն կամրջի կողմից արտահանված ընդհանուր Norito կառուցվածքով:  
  Ոլորտներ:
  - `methods` — բայեր (`sign_transaction`, `sign_raw`, `submit_proof`, ...):  
  - `events` — բաժանորդագրություններ, որոնց թույլատրվում է կցել dApp-ին:  
  - `resources` — կամընտիր հաշվի/ակտիվների զտիչներ, որպեսզի դրամապանակները կարողանան մուտք գործել շրջանակ:  
  - `constraints` — շղթայի ID, TTL կամ հատուկ քաղաքականության կոճակներ, որոնք դրամապանակն ուժի մեջ է դնում նախքան ստորագրելը:
- Համապատասխանության մետատվյալներն անցնում են թույլտվությունների հետ մեկտեղ.
  - Լրացուցիչ `attachments[]` պարունակում է Norito հավելվածի հղումներ (KYC փաթեթներ, կարգավորիչի անդորրագրեր):  
  - `compliance_manifest_id` հարցումը կապում է նախկինում հաստատված մանիֆեստի հետ, որպեսզի օպերատորները կարողանան ստուգել ծագումը:
- Դրամապանակի պատասխանները օգտագործում են համաձայնեցված կոդերը.
  - `user_declined`, `permissions_mismatch`, `compliance_failed`, `internal_error`:  
  Յուրաքանչյուրը կարող է ունենալ `localized_message` UI ակնարկների համար, գումարած մեքենայաընթեռնելի `reason_code`:
- Հաստատման շրջանակները ներառում են ընտրված հաշիվը/վերահսկիչը, թույլտվության արձագանքը, ապացույցների փաթեթը (ZK ապացույց կամ ատեստավորում) և քաղաքականության ցանկացած փոխարկում (օրինակ՝ `offline_queue_enabled`):  
  Մերժումները արտացոլում են նույն սխեման դատարկ `proof`-ի հետ, բայց դեռևս գրանցում են `sid`-ը՝ ստուգման համար:

| Android | Kotlin coroutines + կնքված դասեր շրջանակների համար | Հավասարեցրեք Swift կառուցվածքին՝ շարժականության համար: |
| ԲԸ | Async կրկնիչներ + TypeScript թվեր շրջանակների տեսակների համար | Տրամադրել փաթեթի համար հարմար SDK (զննարկիչ/հանգույց): |

### Ընդհանուր վարքագիծ- `ConnectSession`-ը կազմակերպում է կյանքի ցիկլը.
  1. Ստեղծեք WebSocket, կատարեք ձեռքսեղմում:
  2. Փոխանակեք բաց/հաստատեք շրջանակները:
  3. կարգավորել նշանների հարցումները/պատասխանները:
  4. Իրադարձություններ թողարկեք հավելվածի շերտ:
- Տրամադրել բարձր մակարդակի օգնականներ.
  - `requestSignature(tx, metadata)`
  - `approveSession(account, permissions)`
  - `reject(reason)`
  - `cancelRequest(hash)` – թողարկում է դրամապանակի կողմից ճանաչված կառավարման շրջանակ:
- Սխալների մշակում. քարտեզ Norito սխալի կոդերը SDK-ի հատուկ սխալներին. ներառել
  տիրույթին հատուկ կոդեր UI-ի համար՝ օգտագործելով ընդհանուր տաքսոնոմիան (`Transport`, `Codec`, `Authorization`, `Timeout`, `QueueOverflow`, Norito): Swift-ի ելակետային ներդրումը + հեռաչափության ուղեցույցը գործում է [`connect_error_taxonomy.md`](connect_error_taxonomy.md)-ում և հանդիսանում է Android/JS հավասարության հղում:
- Հեռուստաչափական կեռիկներ թողարկեք հերթի խորության համար, նորից միացրեք հաշվերը և պահանջեք ուշացում (`connect.queue_depth`, `connect.reconnects_total`, `connect.latency_ms`):
 
## Հերթական թվեր և հոսքի վերահսկում

- Յուրաքանչյուր ուղղություն պահում է հատուկ 64-բիթանոց `sequence` հաշվիչ, որը սկսվում է զրոյից, երբ նիստը բացվում է: Համօգտագործվող օգնականը սեղմում է ավելացումներով և գործարկում է `ConnectError.sequenceOverflow` + ստեղնաշարի ձեռքսեղմում, նախքան հաշվիչը փաթաթելը:
- Ոչ-ները և հարակից տվյալները վկայակոչում են հաջորդականության համարը, այնպես որ կրկնօրինակները կարող են մերժվել առանց օգտակար բեռների վերլուծության: SDK-ները պետք է պահեն `{sid, dir, seq, payload_hash}`-ը իրենց ամսագրերում, որպեսզի կրկնօրինակումը որոշիչ դարձնի վերամիացումների ընթացքում:
- Դրամապանակները գովազդում են հետադարձ ճնշումը տրամաբանական պատուհանի միջոցով (`FlowControl` կառավարման շրջանակներ): DApp-ները փակվում են միայն այն դեպքում, երբ առկա է պատուհանի նշան; դրամապանակները գաղտնագրված տեքստը մշակելուց հետո թողարկում են նոր նշաններ՝ խողովակաշարերը սահմանափակված պահելու համար:
- Բանակցությունների վերսկսումը հստակ է. երկու կողմերն էլ նորից միանալուց հետո թողարկում են `Control::Resume { seq_app_max, seq_wallet_max, queue_depths }`, որպեսզի դիտորդները կարողանան ստուգել, ​​թե որքան տվյալներ են կրկին ուղարկվել և արդյոք ամսագրերը բացեր են պարունակում:
- Կոնֆլիկտները (օրինակ՝ երկու օգտակար բեռներ՝ նույն `(sid, dir, seq)`-ով, բայց տարբեր հեշերով) աճում են մինչև `ConnectError.Internal` և ստիպում են նոր `sid`-ին՝ խուսափելու լուռ շեղումից:

## Սպառնալիքների մոդելի և տվյալների պահպանման հավասարեցում- **Դիտարկված մակերեսներ.** WebSocket transport, Norito կամուրջի կոդավորում/վերծանում,
  ամսագրի համառություն, հեռաչափություն արտահանողներ և հավելվածների հետադարձ զանգեր:
- ** Հիմնական նպատակները. ** պաշտպանել նիստի գաղտնիքները (X25519 ստեղներ, ստացված AEAD ստեղներ,
  nonce/sequence հաշվիչներ) տեղեկամատյանների/հեռաչափության արտահոսքերից, կանխել վերարտադրումը և
  գրոհների իջեցում և ամսագրերի և անոմալիաների մասին հաշվետվությունների պարտադիր պահպանում:
- ** Մեղմացումները կոդավորված են՝ **
  - Ամսագրերը կրում են միայն ծածկագրված տեքստ; Պահվող մետատվյալները սահմանափակված են հեշերով, երկարությամբ
    դաշտերը, ժամանակի դրոշմանիշերը և հաջորդական համարները:
  - Telemetry payloads-ը վերագրում է ցանկացած վերնագրի/բեռի բովանդակություն և ներառում է միայն
    `sid`-ի աղած հեշեր գումարած ագրեգատ հաշվիչներ; խմբագրման ստուգաթերթը համօգտագործված է
    SDK-ների միջև աուդիտի հավասարության համար:
  - Նիստերի տեղեկամատյանները պտտվում են և լռելյայնորեն հնանում են 7 օր հետո: Դրամապանակները բացահայտում են
    `connectLogRetentionDays` կոճակ (SDK լռելյայն 7) և փաստաթղթավորեք վարքագիծը
    այնպես որ կարգավորվող տեղակայումները կարող են ամրացնել ավելի խիստ պատուհաններ:
  - Bridge API-ի չարաշահում (բացակայող կապեր, կոռումպացված ծածկագրված տեքստ, անվավեր հաջորդականություն)
    վերադարձնում է մուտքագրված սխալները՝ առանց չմշակված բեռների կամ ստեղների արձագանքելու:

Վերանայման առկախ հարցերը հետագծվում են `docs/source/sdk/swift/connect_workshop.md`-ում
և կլուծվի ավագանու արձանագրությամբ. երբ փակվի ծղոտը կլինի
առաջխաղացումից դարձել է ընդունված։

## Անցանց բուֆերացում և վերամիացում

### Ժուռնալիստական պայմանագիր

Յուրաքանչյուր SDK պահում է միայն հավելվածի ամսագիր յուրաքանչյուր նստաշրջանի համար, այնպես որ dApp-ն ու դրամապանակը
կարող է հերթագրել շրջանակները անցանց ռեժիմում, վերսկսել առանց տվյալների կորստի և ապացույցներ տրամադրել
հեռաչափության համար. Պայմանագիրը արտացոլում է Norito կամուրջի տեսակները, ուստի նույն բայթը
ներկայացուցչությունը գոյատևում է բջջային/JS կույտերում:- Ամսագրերն ապրում են հեշացված նստաշրջանի նույնացուցիչի ներքո (`sha256(sid)`), որոնք արտադրում են երկու
  ֆայլեր մեկ նստաշրջանի համար՝ `app_to_wallet.queue` և `wallet_to_app.queue`: Swift-ի օգտագործումը
  ավազապատ ֆայլերի փաթաթան, Android-ը պահում է ֆայլերը `Room`/`FileChannel`-ի միջոցով,
  և JS-ը գրում է IndexedDB-ին; բոլոր ձևաչափերը երկուական են և էնդիական կայուն:
- Յուրաքանչյուր գրառում սերիականացվում է որպես `ConnectJournalRecordV1`:
  - `direction: u8` (`0 = app→wallet`, `1 = wallet→app`)
  - `sequence: u64`
  - `payload_hash: [u8; 32]` (գաղտնագրված տեքստի Blake3 + վերնագրեր)
  - `ciphertext_len: u32`
  - `received_at_ms: u64`
  - `expires_at_ms: u64`
  - `ciphertext: [u8; ciphertext_len]` (ճշգրիտ Norito շրջանակն արդեն փաթաթված է AEAD-ով)
- Ամսագրերը բառացիորեն պահում են ծածկագրված տեքստը: Մենք երբեք չենք վերագաղտնագրում օգտակար բեռը. AEAD
  վերնագրերն արդեն իսկ վավերացնում են ուղղության ստեղները, ուստի համառությունը նվազում է մինչև
  կցված գրառումը համաժամացնելը:
- `ConnectQueueState` կառուցվածքը հիշողության մեջ արտացոլում է ֆայլի մետատվյալները (խորությունը,
  օգտագործված բայթեր, ամենահին/նորագույն հաջորդականությունը): Այն կերակրում է հեռաչափություն արտահանողներին և
  `FlowControl` օգնական.
- Լռելյայն 32 կադր / 1 MiB ամսագրերի գլխարկ; գլխարկին հարվածելը վտարում է
  ամենահին գրառումները (`reason=overflow`): `ConnectFeatureConfig.max_queue_len`
  անտեսում է այս լռելյայնությունները յուրաքանչյուր տեղակայման համար:
- Ամսագրերը պահպանում են տվյալները 24 ժամվա ընթացքում (`expires_at_ms`): Ֆոնային GC-ն հեռացնում է հնացած
  հատվածները անհամբերությամբ, որպեսզի սկավառակի վրա ոտնահետքը մնա սահմանափակված:
- Վթարի անվտանգություն. ավելացրեք, համաժամեցեք և թարմացրեք հիշողության հայելին _նախքան ծանուցումը
  զանգահարողը. Գործարկման ժամանակ SDK-ները սկանավորում են գրացուցակը, վավերացնում գրառումների ստուգման գումարները,
  և վերակառուցեք `ConnectQueueState`: Կոռուպցիան պատճառ է դառնում, որ վիրավորական արձանագրություն լինի
  բաց է թողնվել, դրոշակվել է հեռաչափության միջոցով և կամայականորեն կարանտինացվել օժանդակ աղբավայրերի համար:
- Քանի որ ծածկագրված տեքստն արդեն բավարարում է Norito գաղտնիության ծրարը, միակը
  գրանցված լրացուցիչ մետատվյալները հաշված նստաշրջանի ID-ն է: Հավելվածներ, որոնք ցանկանում են լրացուցիչ
  գաղտնիությունը կարող է ընտրել `telemetry_opt_in = false`, որը պահում է ամսագրեր, բայց
  խմբագրում է հերթերի խորության արտահանումը և անջատում է `sid` համօգտագործումը գրանցամատյաններում:
- SDK-ները ցուցադրում են `ConnectQueueObserver`, որպեսզի դրամապանակները/dApps-ները կարողանան ստուգել հերթի խորությունը,
  դրենաժներ և GC-ի արդյունքներ; այս կեռիկը սնուցում է կարգավիճակի միջերեսները՝ առանց տեղեկամատյանների վերլուծության:

### Կրկնել և վերսկսել իմաստաբանությունը

1. Կրկին միանալիս SDK-ներն արտանետում են `Control::Resume`՝ «{seq_app_max,
   seq_wallet_max, queued_app, queued_wallet, journal_hash}`: Հեշը դա է
   Blake3-ի ամփոփում միայն հավելվածի ամսագրի, որպեսզի անհամապատասխան հասակակիցները կարողանան հայտնաբերել շեղումը:
2. Ստացող գործընկերը համեմատում է ռեզյումեի օգտակար բեռնվածությունը նրա վիճակի հետ, խնդրում է
   վերահեռարձակում, երբ առկա են բացեր, և ընդունում է վերարտադրվող շրջանակները միջոցով
   `Control::ResumeAck`.
3. Կրկնվող շրջանակները միշտ հարգում են տեղադրման կարգը (`sequence`, ապա գրելու ժամանակը):
   Դրամապանակի SDK-ները ՊԵՏՔ Է կիրառեն հետադարձ ճնշում՝ թողարկելով `FlowControl` նշաններ (նաև
   journaled), այնպես որ dApps-ը չի կարող ողողել հերթը անցանց ռեժիմում:
4. Ամսագրերը պահում են գաղտնագրված տեքստը բառացիորեն, ուստի վերարտադրումը պարզապես մղում է գրանցված բայթերը
   ետ տրանսպորտի և ապակոդավորիչի միջոցով: Յուրաքանչյուր SDK-ի համար վերակոդավորում չի թույլատրվում:

### Վերամիացման հոսք1. Transport-ը վերահաստատում է WebSocket-ը և բանակցում է նոր ping ինտերվալով:
2. dApp-ը հերթականությամբ կրկնում է հերթագրված կադրերը՝ հաշվի առնելով դրամապանակի հետադարձ ճնշումը
   (`ConnectSession.nextControlFrame()`-ը տալիս է `FlowControl` նշաններ):
3. Դրամապանակը վերծանում է բուֆերացված արդյունքները, ստուգում հաջորդականության միապաղաղությունը և
   կրկնում է սպասվող հաստատումները/արդյունքները:
4. Երկու կողմերն էլ թողարկում են `resume` կառավարում, որն ամփոփում է `seq_app_max`, `seq_wallet_max`,
   և հեռաչափության հերթերի խորությունները:
5. Կրկնվող շրջանակները (համապատասխանում են `sequence` + `payload_hash`) ճանաչվում և հեռացվում են; կոնֆլիկտները բարձրացնում են `ConnectError.Internal`-ը և գործարկում նիստի հարկադիր վերագործարկում:

### Խափանման ռեժիմներ

- Եթե նիստը համարվում է հնացած (`offline_timeout_ms`, լռելյայն 5 րոպե),
  բուֆերացված շրջանակները մաքրվում են, և SDK-ն բարձրացնում է `ConnectError.sessionExpired`:
- Ամսագրի կոռուպցիայի դեպքում SDK-ները փորձում են Norito վերծանման մեկ վերանորոգում; վրա
  ձախողումը նրանք թողնում են ամսագիրը և արձակում `connect.queue_repair_failed` հեռաչափություն:
- Հերթականության անհամապատասխանությունը առաջացնում է `ConnectError.replayDetected` և ստիպում է թարմություն
  ձեռքսեղմում (նիստը վերսկսվում է նոր `sid`-ով):

### Անցանց բուֆերային պլան և օպերատորի վերահսկում

Սեմինարի առաքման համար պահանջվում է փաստաթղթավորված պլան, որպեսզի յուրաքանչյուր SDK առաքվի նույնը
անցանց վարքագիծը, վերականգնման հոսքը և ապացույցների մակերեսները: Ստորև բերված պլանն է
տարածված է Swift-ում (`ConnectSessionDiagnostics`), Android-ում
(`ConnectDiagnosticsSnapshot`) և JS (`ConnectQueueInspector`):

| Պետական ​​| ձգան | Ավտոմատ արձագանք | Ձեռնարկի վերացում | Հեռաչափության դրոշ |
|-------|------------------------------|---------------|----------------|
| `Healthy` | Հերթի օգտագործումը  5/րոպե | Դադարեցրեք նոր նշանների հարցումները, թողարկեք հոսքի վերահսկման նշաններ կես դրույքաչափով | Հավելվածները կարող են զանգահարել `clearOfflineQueue(.app|.wallet)`; SDK-ն նորից խոնավացնում է վիճակը գործընկերից մի անգամ առցանց | `connect.queue_state=\"throttled\"`, `connect.queue_watermark` չափիչ |
| `Quarantined` | Օգտագործումը ≥ `disk_watermark_drop` (կանխադրված 85%), կոռուպցիան երկու անգամ է հայտնաբերվել կամ `offline_timeout_ms` գերազանցվել է | Դադարեցրեք բուֆերացումը, բարձրացրեք `ConnectError.QueueQuarantined`, պահանջեք օպերատորի հաստատում | `ConnectSessionDiagnostics.forceReset()`-ը ջնջում է ամսագրերը փաթեթը արտահանելուց հետո | `connect.queue_state=\"quarantined\"`, `connect.queue_quarantine_total` հաշվիչ |- Շեմերը ապրում են `ConnectFeatureConfig`-ում (`disk_watermark_warn`,
  `disk_watermark_drop`, `max_disk_bytes`, `offline_timeout_ms`): Երբ հյուրընկալող
  բաց է թողնում արժեքը, SDK-ները վերադառնում են իրենց լռելյայն և գրանցում նախազգուշացում, որպեսզի կազմաձևվեն
  կարելի է ստուգել հեռաչափությունից:
- SDK-ները ցուցադրում են `ConnectQueueObserver` գումարած ախտորոշման օգնականները.
  - Swift՝ `ConnectSessionDiagnostics.snapshot()`-ը տալիս է «{state, depth, bytes,
    reason}` and `exportJournalBundle(url:)` պահպանում է աջակցության երկու հերթերը:
  - Android՝ `ConnectDiagnostics.snapshot()` + `exportJournalBundle(path)`:
  - JS: `ConnectQueueInspector.read()`-ը վերադարձնում է նույն կառուցվածքը և բլբի բռնակը
    որ UI կոդը կարող է վերբեռնել Torii աջակցման գործիքներին:
- Երբ հավելվածն անջատում է `offline_queue_enabled=false`-ը, SDK-ները անմիջապես թափվում են և
  մաքրեք երկու ամսագրերը, նշեք վիճակը որպես `Disabled` և թողարկեք տերմինալ
  հեռաչափության միջոցառում. Օգտատիրոջ դեմքով նախապատվությունը արտացոլված է Norito-ում
  հաստատման շրջանակ, որպեսզի հասակակիցներն իմանան, թե արդյոք նրանք կարող են վերսկսել բուֆերացված շրջանակները:
- Օպերատորներն աշխատում են `connect queue inspect --sid <sid>` (CLI փաթաթան SDK-ի շուրջը
  ախտորոշում) քաոսի թեստերի ժամանակ; այս հրամանը տպում է վիճակի անցումները,

### Ապացույցների փաթեթի աշխատանքային հոսք

### Հեռուստաչափության կեռիկներ և կառավարման ապացույցներ

- `connect.queue_state` կրկնապատկվում է որպես ճանապարհային քարտեզի ռիսկի ցուցիչ: Վահանակների խումբ
  `{platform, sdk_version}`-ի միջոցով և տրամադրել ժամանակի վիճակ, որպեսզի կառավարումը կարողանա նմուշառում
  ամենամսյա փորձնական ապացույցներ՝ նախքան փուլային տեղադրումները հաստատելը:
- `connect.queue_watermark` և `connect.queue_bytes` ապահովում են Connect ռիսկի միավորը
  (`risk.connect.offline_buffer`), որն ավտոմատ կերպով էջ է տալիս SRE-ին, երբ ավելի քան
  Սեանսների 5%-ը ծախսում է ավելի քան 10 րոպե `Throttled`-ում:
- Արտահանողները `feature_hash`-ը կցում են յուրաքանչյուր իրադարձության, որպեսզի աուդիտորական գործիքավորումը կարողանա հաստատել
  որ Norito կոդեկ + օֆլայն պլանը համապատասխանում է վերանայված կառուցվածքին: SDK CI-ն ձախողվում է
  արագ, երբ հեռաչափությունը հայտնում է անհայտ հեշի մասին:
- Ծղոտե մարդուն դեռևս անհրաժեշտ է սպառնալիքի մոդելի հավելված. երբ չափիչները գերազանցում են
  քաղաքականության շեմերը, SDK-ները թողարկում են `connect.policy_violation` իրադարձություններ, որոնք ամփոփում են
  վիրավորող սիդ (հեշված), վիճակագրություն և լուծված գործողություն (`drain|purge|quarantine`):
- `exportQueueMetrics`-ի միջոցով հավաքագրված ապացույցները տեղավորվում են նույն SoraFS անվանատարածքում
  քանի որ Connect runbook-ը արտեֆակտ է ստեղծում, որպեսզի խորհրդի վերանայողները կարողանան հետևել յուրաքանչյուր վարժանքին
  վերադառնալ հեռաչափության հատուկ նմուշներին՝ առանց ներքին տեղեկամատյաններ պահանջելու:

- Մետատվյալների հավելվածը (պատկերակներ, տեղայնացված անուններ, համապատասխանության ապացույցներ) ստորագրված է

  ավելացված `metadata_version`:
- Վերևում գտնվող սեփականության մատրիցը վկայակոչված է SDK փաստաթղթերից, ուստի CLI/վեբ/ավտոմատացում

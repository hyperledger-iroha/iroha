### الأهداف
- تنفيذ منشئ التتبع FASTPQ لتحديث KV AIR. يجب أن يقوم كل صف بتشفير:
  - `key_limbs[i]`: قاعدة 256 طرفًا (7 بايت، نهاية صغيرة) لمسار المفتاح الأساسي.
  - `value_old_limbs[i]`، `value_new_limbs[i]`: نفس التعبئة للقيم السابقة/اللاحقة.
  - أعمدة المحدد: `s_active`، `s_transfer`، `s_mint`، `s_burn`، `s_role_grant`، `s_role_revoke`، `s_meta_set`، `s_perm`.
  - الأعمدة المساعدة: `delta = value_new - value_old`، `running_asset_delta`، `metadata_hash`، `supply_counter`.
  - أعمدة الأصول: `asset_id_limbs[i]` باستخدام أطراف ذات 7 بايت.
  - أعمدة SMT لكل مستوى `ℓ`: `path_bit_ℓ`، `sibling_ℓ`، `node_in_ℓ`، `node_out_ℓ`، بالإضافة إلى `neighbour_leaf` لغير العضوية.
  - أعمدة البيانات التعريفية: `dsid`، `slot`.
- **الترتيب الحتمي.** فرز الصفوف معجميًا حسب `(key_bytes, op_rank, original_index)` باستخدام الفرز المستقر. تعيين `op_rank`: `transfer=0`، `mint=1`، `burn=2`، `role_grant=3`، `role_revoke=4`، `meta_set=5`. `original_index` هو الفهرس المستند إلى 0 قبل الفرز. استمر في تجزئة طلب Poseidon2 الناتج (علامة المجال `fastpq:v1:ordering`). قم بتشفير صورة التجزئة الأولية كـ `[domain_len, domain_limbs…, payload_len, payload_limbs…]` حيث تكون الأطوال عبارة عن عناصر حقل u64 بحيث تظل البايتات الصفرية اللاحقة قابلة للتمييز.
- شاهد البحث: إنتاج `perm_hash = Poseidon2(role_id || permission_id || epoch_u64_le)` عندما يكون العمود المخزن `s_perm` (OR المنطقي لـ `s_role_grant` و`s_role_revoke`) هو 1. معرفات الدور/الإذن هي سلاسل LE ذات عرض ثابت تبلغ 32 بايت؛ العصر هو 8 بايت جنيه.
- فرض الثوابت قبل وداخل AIR: المحددات الحصرية المتبادلة، وحفظ كل أصل، وثوابت dsid/slot.
- `N_trace = 2^k` (`pow2_ceiling` لعدد الصفوف)؛ `N_eval = N_trace * 2^b` حيث `b` هو الأس المضخم.
- توفير التركيبات واختبارات الملكية:
  - التعبئة ذهابًا وإيابًا (`fastpq_prover/tests/packing.rs`، `tests/fixtures/packing_roundtrip.json`).
  - طلب تجزئة الاستقرار (`tests/fixtures/ordering_hash.json`).
  - تركيبات الدفعة (`trace_transfer.json`، `trace_mint.json`، `trace_duplicate_update.json`).### مخطط عمود الهواء
| مجموعة العمود | أسماء | الوصف |
| ----------------- | -------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| النشاط | `s_active` | 1 للصفوف الحقيقية، 0 للحشوة.                                                                                       |
| الرئيسية | `key_limbs[i]`، `value_old_limbs[i]`، `value_new_limbs[i]` | عناصر المعتدلات المعبأة (أطراف صغيرة ذات 7 بايت).                                                             |
| الأصول | `asset_id_limbs[i]` | معرّف الأصول الأساسي المعبأ (أطراف صغيرة، 7 بايت).                                                      |
| محددات | `s_transfer`، `s_mint`، `s_burn`، `s_role_grant`، `s_role_revoke`، `s_meta_set`، `s_perm` | 0/1. القيد: محددات Σ (بما في ذلك `s_perm`) = `s_active`؛ يعكس `s_perm` صفوف منح/إلغاء الدور.              |
| مساعد | `delta`، `running_asset_delta`، `metadata_hash`، `supply_counter` | الحالة المستخدمة للقيود والحفظ ومسارات التدقيق.                                                           |
| سمت | `path_bit_ℓ`، `sibling_ℓ`، `node_in_ℓ`، `node_out_ℓ`، `neighbour_leaf` | مدخلات/مخرجات Poseidon2 لكل مستوى بالإضافة إلى شاهد جار لعدم العضوية.                                         |
| بحث | `perm_hash` | تجزئة Poseidon2 للبحث عن الإذن (مقيد فقط عندما يكون `s_perm = 1`).                                            |
| البيانات الوصفية | `dsid`، `slot` | ثابت عبر الصفوف.                                                                                                 |### الرياضيات والقيود
- **تعبئة الحقل:** يتم تقسيم البايتات إلى أطراف مكونة من 7 بايت (النهاية الصغيرة). كل طرف `limb_j = Σ_{k=0}^{6} byte_{7j+k} * 256^k`؛ رفض الأطراف ≥ معامل المعتدل.
- **التوازن/الحفظ:** دع `δ = value_new - value_old`. تجميع الصفوف حسب `asset_id`. حدد `r_asset_start = 1` في الصف الأول من كل مجموعة أصول (0 بخلاف ذلك) وقم بالتقييد
  ```
  running_asset_delta = (1 - r_asset_start) * running_asset_delta_prev + δ.
  ```
  في الصف الأخير من كل مجموعة أصول تأكيد
  ```
  running_asset_delta = Σ (s_mint * δ) - Σ (s_burn * δ).
  ```
  تستوفي عمليات النقل القيد تلقائيًا لأن قيمها δ مجموعها يساوي الصفر عبر المجموعة. على سبيل المثال: إذا كان `value_old = 100` و`value_new = 120` في صف النعناع، ​​δ = 20، فإن مجموع النعناع يساهم +20 ويتحول الفحص النهائي إلى الصفر عند عدم حدوث أي حروق.
- **الحشوة:** تقديم `s_active`. اضرب كافة قيود الصفوف في `s_active` وقم بفرض بادئة متجاورة: `s_active[i] ≥ s_active[i+1]`. يجب أن تحتفظ صفوف الحشو (`s_active=0`) بقيم ثابتة ولكنها غير مقيدة.
- **طلب التجزئة:** تجزئة Poseidon2 (المجال `fastpq:v1:ordering`) عبر ترميزات الصف؛ مخزنة في IO العام لقابلية التدقيق.

## المرحلة الثانية — STARK Prover Core

### الأهداف
- بناء التزامات Poseidon2 Merkle على ناقلات تقييم التتبع والبحث. المعلمات: المعدل=2، السعة=1، الجولات الكاملة=8، الجولات الجزئية=57، الثوابت المثبتة على `ark-poseidon2` تلتزم `3f2b7fe` (الإصدار 0.3.0).
- امتداد منخفض الدرجة: قم بتقييم كل عمود في المجال `D = { g^i | i = 0 .. N_eval-1 }`، حيث يقسم `N_eval = 2^{k+b}` سعة 2-adic لـ Goldilocks. دع `g = ω^{(p-1)/N_eval}` مع `ω` هو الجذر البدائي الثابت لـ Goldilocks و`p` معامله؛ استخدم المجموعة الفرعية الأساسية (بدون مجموعة). سجل `g` في النص (العلامة `fastpq:v1:lde`).
- كثيرات الحدود التركيبية: لكل قيد `C_j`، شكل `F_j(X) = C_j(X) / Z_N(X)` مع هوامش الدرجة المذكورة أدناه.
- وسيطة البحث (الأذونات): نموذج `γ` من النص. تتبع المنتج `Z_0 = 1`، `Z_i = Z_{i-1} * (perm_hash_i - γ)^{s_perm_i}`. منتج الجدول `T = ∏_j (table_perm_j - γ)`. قيد الحدود: `Z_final / T = 1`.
- DEEP-FRI مع arity `r ∈ {8, 16}`: لكل طبقة، قم بامتصاص الجذر باستخدام العلامة `fastpq:v1:fri_layer_ℓ`، وعينة `β_ℓ` (العلامة `fastpq:v1:beta_ℓ`)، ثم قم بالطي عبر `H_{ℓ+1}(i) = Σ_{k=0}^{r-1} H_ℓ(r*i + k) * β_ℓ^k`.
- كائن إثبات (Norito مشفر):
  ```
  Proof {
      protocol_version: u16,
      params_version: u16,
      parameter_set: String,
      public_io: PublicIO,
      trace_root: [u8; 32],
      lookup_root: [u8; 32],
      fri_layers: Vec<[u8; 32]>,
      alphas: Vec<Field>,
      betas: Vec<Field>,
      queries: Vec<QueryOpening>,
  }
  ```
- محقق المرايا محقق؛ قم بتشغيل مجموعة الانحدار على آثار صف 1k/5k/20k مع النصوص الذهبية.

### درجة المحاسبة
| القيد | الدرجة قبل القسمة | الدرجة بعد المحددات | الهامش مقابل `deg(Z_N)` |
|------------|---------------------------------------|--------|----------------------|
| نقل/نعناع/حفظ الحرق | ≥1 | ≥1 | `deg(Z_N) - 2` |
| بحث منح/إلغاء الدور | ≥2 | ≥2 | `deg(Z_N) - 3` |
| مجموعة البيانات الوصفية | ≥1 | ≥1 | `deg(Z_N) - 2` |
| تجزئة SMT (لكل مستوى) | ≥3 | ≥3 | `deg(Z_N) - 4` |
| بحث عن المنتج الكبير | علاقة المنتج | لا يوجد | القيد الحدودي |
| الجذور الحدودية / مجاميع العرض | 0 | 0 | بالضبط |

تتم معالجة صفوف الحشو من خلال `s_active`؛ تقوم الصفوف الوهمية بتوسيع التتبع إلى `N_trace` دون انتهاك القيود.## التشفير والنسخ (عالمي)
- **تعبئة البايت:** قاعدة 256 (أطراف ذات 7 بايت، نهاية صغيرة). الاختبارات في `fastpq_prover/tests/packing.rs`.
- **تشفير الحقل:** المعتدلات الأساسية (طرف 64 بت ذو النهاية الصغيرة، مرفوض ≥ p)؛ مخرجات Poseidon2/جذور SMT متسلسلة كمصفوفات ذات نهاية صغيرة 32 بايت.
- **نص (فيات – شمير):**
  1. يستوعب BLAKE2b علامة الالتزام `protocol_version` و`params_version` و`parameter_set` و`public_io` وPoseidon2 (`fastpq:v1:init`).
  2. استيعاب `trace_root`، `lookup_root` (`fastpq:v1:roots`).
  3. اشتقاق تحدي البحث `γ` (`fastpq:v1:gamma`).
  4. اشتقاق تحديات التكوين `α_j` (`fastpq:v1:alpha_j`).
  5. لكل جذر طبقة FRI، امتص باستخدام `fastpq:v1:fri_layer_ℓ`، واشتق `β_ℓ` (`fastpq:v1:beta_ℓ`).
  6. اشتقاق مؤشرات الاستعلام (`fastpq:v1:query_index`).

  العلامات هي ASCII صغيرة؛ ويرفض القائمون على التحقق حالات عدم التطابق قبل اختبارات أخذ العينات. تركيبات النسخة الذهبية: `tests/fixtures/transcript_v1.json`.
- **إصدار:** `protocol_version = 1`، `params_version` يتطابق مع مجموعة المعلمات `fastpq_isi`.

## وسيطة البحث (الأذونات)
- جدول ملتزم تم فرزه معجميًا حسب `(role_id_bytes, permission_id_bytes, epoch_le)` وملتزم عبر شجرة Poseidon2 Merkle (`perm_root` في `PublicIO`).
- يستخدم شاهد التتبع `perm_hash` والمحدد `s_perm` (أو منح/إلغاء الدور). يتم ترميز الصف كـ `role_id_bytes || permission_id_bytes || epoch_u64_le` بعرض ثابت (32، 32، 8 بايت).
- علاقة المنتج:
  ```
  Z_0 = 1
  for each row i: Z_i = Z_{i-1} * (perm_hash_i - γ)^{s_perm_i}
  T = ∏_j (table_perm_j - γ)
  ```
  تأكيد الحدود: `Z_final / T = 1`. راجع `examples/lookup_grand_product.md` للحصول على شرح تفصيلي لمراكم الخرسانة.

## قيود شجرة ميركل المتناثرة
- تعريف `SMT_HEIGHT` (عدد المستويات). تظهر الأعمدة `path_bit_ℓ`، و`sibling_ℓ`، و`node_in_ℓ`، و`node_out_ℓ`، و`neighbour_leaf` لكل `ℓ ∈ [0, SMT_HEIGHT)`.
- معلمات Poseidon2 المثبتة على `ark-poseidon2` تلتزم بـ `3f2b7fe` (الإصدار 0.3.0)؛ علامة المجال `fastpq:v1:poseidon_node`. تستخدم جميع العقد ترميز حقل endian الصغير.
- تحديث القواعد لكل مستوى:
  ```
  if path_bit_ℓ == 0:
      node_out_ℓ = Poseidon2(node_in_ℓ, sibling_ℓ)
  else:
      node_out_ℓ = Poseidon2(sibling_ℓ, node_in_ℓ)
  ```
- مجموعة الإدخالات `(node_in_0 = 0, node_out_0 = value_new)`؛ يحذف المجموعة `(node_in_0 = value_old, node_out_0 = 0)`.
- توفر إثباتات عدم العضوية `neighbour_leaf` لإظهار أن الفاصل الزمني الذي تم الاستعلام عنه فارغ. راجع `examples/smt_update.md` للحصول على مثال عملي وتخطيط JSON.
- قيد الحدود: التجزئة النهائية تساوي `old_root` للصفوف السابقة و`new_root` للصفوف اللاحقة.

### إصدار تطبيق قائمة التحقق

أضف البوابات التالية إلى كل تذكرة إصدار FASTPQ. يتم حظر الإصدارات حتى يتم جميع العناصر
كاملة ومرفقة كقطع أثرية موقعة.

1. **مقاييس إثبات الثانية الفرعية** — التقاط معيار المعدن الأساسي
   (`fastpq_metal_bench_*.json`) يجب أن يثبت أن عبء العمل المكون من 20000 صف (32768 صفًا مبطنًا) ينتهي في
   <1 ثانية. بشكل ملموس، الإدخال `benchmarks.operations` حيث `operation = "lde"` والمطابقة
   يجب أن تظهر عينة `report.operations` `gpu_mean_ms ≤ 950`. يتطلب التشغيل الذي يتجاوز السقف
   التحقيق والاستعادة قبل التوقيع على القائمة المرجعية.
2. **بيان الأداء الموقّع** — بعد تسجيل حزم Metal + CUDA الجديدة، قم بالتشغيل
   `cargo xtask fastpq-bench-manifest … --signing-key <path>` للانبعاث
   `artifacts/fastpq_bench_manifest.json` والتوقيع المنفصل
   (`artifacts/fastpq_bench_manifest.sig`). قم بإرفاق كلا الملفين بالإضافة إلى بصمة المفتاح العام إلى ملف
   إصدار تذكرة حتى يتمكن المراجعون من التحقق من الملخص والتوقيع بشكل مستقل.[xtask/src/fastpq.rs:1]
3. **مرفقات الأدلة** — قم بتخزين معيار JSON الأولي، أو سجل stdout (أو تتبع الأدوات، عندما
   تم التقاطها)، وزوج البيان/التوقيع مع تذكرة الإصدار. القائمة المرجعية هي فقط
   تعتبر خضراء عندما ترتبط التذكرة بتلك المصنوعات اليدوية ويؤكد المراجع عند الطلب
   الملخص المسجل في `fastpq_bench_manifest.json` يطابق الملفات التي تم تحميلها.[artifacts/fastpq_benchmarks/README.md:1]

#### قائمة التحقق من التعامل مع التنبيهات للمرحلة 7-1

يغذي كل تنبيه مدرج أعلاه تمرينًا محددًا عند الطلب حتى يقوم المشغلون بجمع المعلومات
نفس العناصر التي تتطلبها حزمة الإصدار:

1. **`FastpqQueueHeadroomLow` (تحذير).** قم بتشغيل استعلام Prometheus فوريًا
   لـ `fastpq_metal_queue_depth{metric=~"limit|max_in_flight",device_class="<matrix>"}` و
   التقط لوحة Grafana "Queue headroom" من `fastpq-acceleration`
   مجلس. سجل نتيجة الاستعلام في
   `artifacts/fastpq_rollouts/<stamp>/<fleet>/<lane>/metrics_headroom.prom`
   مع معرف التنبيه بحيث تثبت حزمة الإصدار أن التحذير كان كذلك
   تم الاعتراف به قبل تجويع قائمة الانتظار.[dashboards/grafana/fastpq_acceleration.json:1]
2. **`FastpqZeroFillRegression` (صفحة).** فحص
   `fastpq_zero_fill_duration_ms{device_class="<matrix>"}`، وإذا كان المقياس كذلك
   صاخب، أعد تشغيل `scripts/fastpq/wrap_benchmark.py` على أحدث مقعد JSON
   لتحديث الكتلة `zero_fill_hotspots`. إرفاق إخراج promQL،
   لقطات الشاشة، وملف البدلاء الذي تم تحديثه إلى دليل الطرح؛ هذا يخلق
   نفس الدليل الذي يتوقعه `ci/check_fastpq_rollout.sh` أثناء الإصدار
   التحقق من الصحة.[scripts/fastpq/wrap_benchmark.py:1][ci/check_fastpq_rollout.sh:1]
3. **`FastpqCpuFallbackBurst` (الصفحة).** قم بتأكيد ذلك
   `fastpq_execution_mode_total{requested="gpu",backend="cpu"}` يتجاوز 5%
   الكلمة، ثم عينة من سجلات `irohad` لرسائل الرجوع إلى إصدار أقدم المقابلة
   (`telemetry::fastpq.execution_mode resolved="cpu"`). قم بتخزين تفريغ promQL
   بالإضافة إلى مقتطفات السجل في `metrics_cpu_fallback.prom`/`rollback_drill.log` لذلك
   توضح الحزمة كلا من التأثير وإقرار المشغل.
4. **تغليف الأدلة.** بعد مسح أي تنبيه، أعد تشغيل خطوات Stage7-3
   دليل التشغيل (Grafana التصدير، لقطة التنبيه، تدريبات التراجع) و
   أعد التحقق من صحة الحزمة عبر `ci/check_fastpq_rollout.sh` قبل إعادة توصيلها
   إلى تذكرة الإصدار.[docs/source/fastpq_rollout_playbook.md:114]

يمكن للمشغلين الذين يفضلون الأتمتة التشغيل
`scripts/fastpq/capture_alert_evidence.sh --device-class <label> --out <bundle-dir>`
للاستعلام عن واجهة برمجة التطبيقات Prometheus لمساحة قائمة الانتظار والتعبئة الصفرية والاحتياطي لوحدة المعالجة المركزية
المقاييس المذكورة أعلاه؛ يقوم المساعد بكتابة ملف JSON الذي تم التقاطه (يبدأ بـ
promQL الأصلي) إلى `metrics_headroom.prom` و`metrics_zero_fill.prom` و
`metrics_cpu_fallback.prom` ضمن دليل التشغيل المختار بحيث تكون تلك الملفات

الميزانية مباشرة. يقوم بتوزيع كل مقعد `metal` المشار إليه بواسطة
`fastpq_bench_manifest.json`، يتفقد
`benchmarks.metal_dispatch_queue.{limit,max_in_flight}` و
`benchmarks.zero_fill_hotspots[]`، وتفشل الحزمة عند انخفاض الإرتفاع
أسفل فتحة واحدة أو عندما تبلغ أي نقطة اتصال LDE عن `mean_ms > 0.40`. هذا يحافظ على

و`metadata.labels.gpu_kind`). الحزم التي تفتقد هذه التصنيفات تفشل على الفور،
ضمان إصدار المصنوعات اليدوية وبيانات مصفوفة Stage7-2 ووقت التشغيل

لوحة Grafana "أحدث معيار" وحزمة الطرح المرتبطة بها تقتبس الآن
`device_class`، ميزانية تعبئة صفرية، ولقطات سريعة لعمق قائمة الانتظار، لذلك المهندسين تحت الطلب

## الملحق أ – اشتقاق السلامة

يشرح هذا الملحق كيفية إنتاج جدول "Soundness & SLOs" وكيف يفرض CI الحد الأدنى ≥128 بت المذكور سابقًا.

### التدوين
- `N_trace = 2^k` - طول التتبع بعد الفرز والحشو إلى قوة اثنين.
- `b` — عامل النفخ (`N_eval = N_trace × b`).
- `r` — FRI (8 أو 16 للمجموعات الأساسية).
- `ℓ` — عدد تخفيضات FRI (العمود `layers`).
- `q` - استعلامات التحقق لكل دليل (عمود `queries`).
- `ρ` — معدل الكود الفعال الذي تم الإبلاغ عنه بواسطة مخطط العمود: `ρ = max_i(degree_i / domain_i)` على القيود التي تجاوزت جولة FRI الأولى.

يحتوي حقل قاعدة المعتدل على `|F| = 2^64 - 2^32 + 1`، لذا فإن تصادمات فيات-شمير يحدها `q / 2^64`. يضيف الطحن عامل `2^{-g}` المتعامد، مع `g = 23` لـ `fastpq-lane-balanced` و`g = 21` لملف تعريف زمن الوصول.

### ملزمة تحليلية

مع معدل ثابت DEEP-FRI يرضي احتمال الفشل الإحصائي

```
p_fri ≤ Σ_{j=0}^{ℓ-1} ρ^{q} = ℓ · ρ^{q}
```

لأن كل طبقة تقلل درجة متعدد الحدود وعرض المجال بنفس العامل `r`، مع الحفاظ على ثبات `ρ`. يشير العمود `est bits` الخاص بالجدول إلى `⌊-log₂ p_fri⌋`؛ فيات-شمير والطحن بمثابة هامش أمان إضافي.

### مخرجات المخطط والحسابات العملية

تشغيل مخطط الأعمدة Stage1 على إنتاجية الدُفعات التمثيلية:

| مجموعة المعلمات | `N_trace` | `b` | `N_eval` | `ρ` (مخطط) | الدرجة الفعالة (`ρ × N_eval`) | `ℓ` | `q` | `-log₂(ℓ · ρ^{q})` |
| ------------- | --------- | --- | -------- | ------------- | -------------------------------- | --- | --- | ------------------ |
| دفعة 20 ألف متوازنة | `2^15` | 8 | 262144 | 0.077026 | 20192 | 5 | 52 | 190 بت |
| إنتاجية دفعة 65 ألف | `2^16` | 8 | 524288 | 0.200208 | 104967 | 6 | 58 | 132 بت |
| الكمون 131 كيلو دفعة | `2^17` | 16 | 2097152 | 0.209492 | 439337 | 5 | 64 | 142 بت |مثال (دفعة متوازنة 20 ألف):
1. `N_trace = 2^15`، لذا `N_eval = 2^15 × 8 = 2^18`.
2. تقارير أجهزة المخطط هي `ρ = 0.077026`، وبالتالي `p_fri = 5 × ρ^{52} ≈ 6.4 × 10^{-58}`.
3. `-log₂ p_fri = 190 bits`، مطابقة لإدخال الجدول.
4. تضيف تصادمات فيات-شمير `2^{-58.3}` على الأكثر، ويطرح الطحن (`g = 23`) `2^{-23}` آخر، مما يحافظ على السلامة الإجمالية بشكل مريح فوق 160 بت.

### أداة أخذ عينات رفض CI

ينفذ كل تشغيل CI أداة مونت كارلو لضمان بقاء القياسات التجريبية ضمن ±0.6 بت من الحد التحليلي:
1. ارسم مجموعة معلمات أساسية وقم بتوليف `TransitionBatch` مع عدد الصفوف المطابق.
2. أنشئ التتبع، واقلب قيدًا تم اختياره عشوائيًا (على سبيل المثال، إزعاج منتج البحث الكبير أو أحد أخوة SMT)، وحاول تقديم دليل.
3. أعد تشغيل أداة التحقق، وأعد عينة من تحديات فيات-شمير (بما في ذلك الطحن)، وسجل ما إذا كان الدليل الذي تم التلاعب به قد تم رفضه.
4. كرر ذلك مع 16384 بذرة لكل مجموعة معلمات وقم بتحويل الحد الأدنى لـ Clopper-Pearson بنسبة 99% لمعدل الرفض الملحوظ إلى بتات.

تفشل المهمة على الفور إذا انزلق الحد الأدنى المُقاس إلى أقل من 128 بت، لذلك يتم اكتشاف التراجعات في المخطط أو حلقة الطي أو أسلاك النسخ قبل الدمج.

## الملحق ب — اشتقاق جذر المجال

يقوم Stage0 بتثبيت مولدات التتبع والتقييم على ثوابت مشتقة من بوسيدون بحيث تشترك جميع التطبيقات في نفس المجموعات الفرعية.

### الإجراء
1. **اختيار البذور.** استوعب علامة UTF‑8 `fastpq:v1:domain_roots` في إسفنجة Poseidon2 المستخدمة في أي مكان آخر في FASTPQ (عرض الحالة=3، المعدل=2، أربع جولات كاملة + 57 جزئية). تقوم المدخلات بإعادة استخدام التشفير `[len, limbs…]` من `pack_bytes`، مما ينتج عنه المولد الأساسي `g_base = 7`.

3. **مولد LDE.** كرر نفس الأس مع `lde_log_size` لاشتقاق `lde_root`.

5. **حجم التقليب.** استمر في `permutation_size` بشكل صريح حتى لا يستنتج المجدولون أبدًا قواعد الحشو من القوى الضمنية للاثنين.

## الملحق ج — تفاصيل مسار الالتزام### تدفق تدفق التزام بوسيدون
تحدد المرحلة 2 التزام التتبع الحتمي الذي يتقاسمه المُثبت والمتحقق:
1. **تطبيع التحولات.** `trace::build_trace` يفرز كل دفعة، ويضعها في `N_trace = 2^{⌈log₂ rows⌉}`، ويصدر متجهات الأعمدة بالترتيب أدناه.[crates/fastpq_prover/src/trace.rs:123]
2. **أعمدة التجزئة.** يقوم `trace::column_hashes` ببث الأعمدة من خلال إسفنجات Poseidon2 المخصصة التي تحمل علامة `fastpq:v1:trace:column:<name>`. عندما تكون ميزة `fastpq-prover-preview` نشطة، يقوم نفس الاجتياز بإعادة تدوير معاملات IFFT/LDE التي تتطلبها الواجهة الخلفية، لذلك لا يتم تخصيص نسخ مصفوفة إضافية.

يجب أن تتضمن حزمة الطرح الآن `metrics_poseidon.prom` وفرق التكوين ومقتطف السجل عندما يكون الوضع المختلط نشطًا حتى تتمكن الإدارة من مراجعة السياسة الاحتياطية الحتمية جنبًا إلى جنب مع القياس عن بعد FFT/LDE. `ci/check_fastpq_rollout.sh` يفرض بالفعل حدود قائمة الانتظار/التعبئة الصفرية؛ ستقوم بوابة المتابعة بالتحقق من سلامة عداد بوسيدون بمجرد وصول الوضع المختلط إلى أتمتة الإصدار.

يستهلك خط أنابيب التجزئة الأعمدة بهذا الترتيب الحتمي:
1. أعلام المحدد: `s_active`، `s_transfer`، `s_mint`، `s_burn`، `s_role_grant`، `s_role_revoke`، `s_meta_set`، `s_perm`.

### نسخ علامات المجال
تقوم المرحلة 2 بإصلاح كتالوج فيات شامير أدناه للحفاظ على حتمية توليد التحدي:

| العلامة | الغرض |
| --- | ------- |
| `fastpq:v1:init` | استيعاب إصدار البروتوكول ومجموعة المعلمات و`PublicIO`. |
| `fastpq:v1:roots` | قم بتتبع جذور Merkle وابحث عنها. |
| `fastpq:v1:gamma` | عينة من تحدي البحث عن المنتج الكبير. |
| `fastpq:v1:alpha:<i>` | تحديات تكوين العينة متعددة الحدود (`i = 0, 1`). |
| `fastpq:v1:lookup:product` | استيعاب منتج البحث الكبير الذي تم تقييمه. |
| `fastpq:v1:beta:<round>` | قم بتجربة تحدي الطي لكل جولة من جولات FRI. |
| `fastpq:v1:fri_layer:<round>` | الالتزام بجذر Merkle لكل طبقة FRI. |
| `fastpq:v1:fri:final` | قم بتسجيل طبقة FRI النهائية قبل فتح الاستعلامات. |
| `fastpq:v1:query_index:0` | اشتقاق مؤشرات استعلام التحقق بشكل حتمي. |

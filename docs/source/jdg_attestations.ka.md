# JDG ატესტაციები: დაცვა, როტაცია და შეკავება

ეს ჩანაწერი ადასტურებს v1 JDG ატესტაციის მცველს, რომელიც ახლა იგზავნება `iroha_core`-ში.

- **კომიტეტი ავლენს:** Norito-ში კოდირებული `JdgCommitteeManifest` პაკეტები ახორციელებს მონაცემთა სივრცის როტაციას
  განრიგი (`committee_id`, შეკვეთილი წევრები, ბარიერი, `activation_height`, `retire_height`).
  მანიფესტები დატვირთულია `JdgCommitteeSchedule::from_path`-ით და მკაცრად იზრდება
  აქტივაციის სიმაღლეები სურვილისამებრ მადლის გადაფარვით (`grace_blocks`) პენსიაზე გასვლას/გააქტიურებას შორის
  კომიტეტები.
- **ატესტაციის მცველი:** `JdgAttestationGuard` ახორციელებს მონაცემთა სივრცის სავალდებულოობას, ვადის გასვლას, შემორჩენილ საზღვრებს,
  კომიტეტის პირადობის მოწმობის/ზღვრის შესატყვისი, ხელმომწერის წევრობა, ხელმოწერის მხარდაჭერილი სქემები და სურვილისამებრ
  SDN ვალიდაცია `JdgSdnEnforcer`-ის საშუალებით. ზომის ქუდები, მაქსიმალური ჩამორჩენა და ხელმოწერის დაშვებული სქემებია
  კონსტრუქტორის პარამეტრები; `validate(attestation, dataspace, current_height)` აბრუნებს აქტიურს
  კომიტეტი ან სტრუქტურირებული შეცდომა.
  - `scheme_id = 1` (`simple_threshold`): თითო ხელმომწერის ხელმოწერები, არჩევითი ხელმომწერის ბიტმაპი.
  - `scheme_id = 2` (`bls_normal_aggregate`): ერთი წინასწარ აგრეგირებული BLS-ნორმალური ხელმოწერა
    ატესტაციის ჰეში; ხელმომწერის ბიტმაპი არჩევითია, ნაგულისხმევია ატესტაციის ყველა ხელმომწერისთვის. BLS
    მთლიანი ვალიდაცია მოითხოვს მოქმედი PoP თითო კომიტეტის წევრს მანიფესტში; დაკარგული ან
    არასწორი PoP-ები უარყოფენ ატესტაციას.
  დაშვების სიის კონფიგურაცია `governance.jdg_signature_schemes`-ის საშუალებით.
- **შენახვის საცავი:** `JdgAttestationStore` თვალს ადევნებს ატესტაციას მონაცემთა სივრცეში კონფიგურირებადი
  თითო მონაცემთა სივრცის თავსახური, ჩანართზე უძველესი ჩანაწერების ამოჭრა. დარეკეთ `for_dataspace` ან
  `for_dataspace_and_epoch` აუდიტის/გამეორების პაკეტების მოსაძიებლად.
- **ტესტები:** ერთეულის გაშუქება ახლა ახორციელებს მოქმედი კომიტეტის შერჩევას, უცნობი ხელმომწერის უარყოფა, შემორჩენილი
  ატესტაციის უარყოფა, მხარდაჭერილი სქემის ID და შეკავების მორთვა. იხ
  `crates/iroha_core/src/jurisdiction.rs`.

მცველი უარყოფს სქემებს კონფიგურირებული დაშვების სიის გარეთ.

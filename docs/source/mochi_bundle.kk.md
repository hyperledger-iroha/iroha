# MOCHI Bundle Tooling

MOCHI жеңіл орауыш жұмыс процесімен жеткізіледі, осылайша әзірлеушілер а
арнайы CI сценарийлерін сымсыз портативті жұмыс үстелі жинағы. `xtask`
ішкі команда компиляцияны, орналасуды, хэштеуді және (міндетті емес) мұрағатты өңдейді
бір кадрда құру.

## Бума жасау

```bash
cargo xtask mochi-bundle
```

Әдепкі бойынша пәрмен шығару екілік файлдарын құрастырады, топтаманы астына жинайды
`target/mochi-bundle/` және `mochi-<os>-<arch>-release.tar.gz` мұрағатын шығарады
детерминистикалық `manifest.json` қатарында. Манифест әрбір файлды тізімдейді
оның өлшемі мен SHA-256 хэші, сондықтан CI құбырлары тексеруді қайта іске қоса алады немесе жариялайды
аттестаттаулар. Көмекші `mochi` жұмыс үстелінің қабығын да, сонымен қатар
`kagami` екілік жұмыс кеңістігі бар, сондықтан генезис генерациясы
қорап.

### Жалаулар

| жалау | Сипаттама |
|--------------------|-----------------------------------------------------------------------------|
| `--out <dir>` | Шығыс каталогын қайта анықтау (әдепкі бойынша `target/mochi-bundle`).         |
| `--profile <name>` | Арнайы жүк профилімен құрастырыңыз (мысалы, сынақтар үшін `debug`).              |
| `--no-archive` | Тек дайындалған қалтаны қалдырып, `.tar.gz` мұрағатын өткізіп жіберіңіз.               |
| `--kagami <path>` | `iroha_kagami` құрудың орнына анық `kagami` екілік нұсқасын пайдаланыңыз.         |
| `--matrix <path>` | CI шығу тегін бақылау үшін бума метадеректерін JSON матрицасына қосыңыз.         |
| `--smoke` | `mochi --help` файлын пакеттелген бумадан негізгі орындау қақпасы ретінде іске қосыңыз.      |
| `--stage <dir>` | Аяқталған буманы (және бар болса мұрағатты) кезеңдік қалтаға көшіріңіз. |

`--stage` әрбір құрастыру агенті өз деректерін жүктеп салатын CI құбырларына арналған.
артефактілер ортақ орынға. Көмекші бума каталогын және қайта жасайды
жасалған мұрағатты кезеңдік каталогқа көшіреді, осылайша тапсырмаларды жариялайды
қабық сценарийінсіз платформаға тән шығыстарды жинаңыз.

Буманың ішіндегі орналасу әдейі қарапайым:

```
bin/mochi              # egui desktop executable
bin/kagami             # kagami helper for genesis generation
config/sample.toml     # starter supervisor configuration
docs/README.md         # bundle overview and verification guide
LICENSE                # repository licence
manifest.json          # generated file manifest with SHA-256 digests
```

### Орындалу уақытын қайта анықтау

Бумаланған `mochi` орындалатын файл пәрмен жолын қайта анықтауды ең көп уақыт үшін қабылдайды
жалпы супервайзер параметрлері. Өңдеу орнына осы жалаушаларды пайдаланыңыз
`config/local.toml` эксперимент кезінде:

```
./bin/mochi --data-root ./data --profile four-peer-bft \
    --torii-start 12000 --p2p-start 14000 \
    --irohad /path/to/irohad --kagami /path/to/kagami
```

Кез келген CLI мәні `config/local.toml` жазбалары мен ортасынан басымдыққа ие болады
айнымалылар.

## Суретті автоматтандыру

`manifest.json` генерациялау уақыт белгісін, мақсатты үш есе, жүк профилін,
және толық файлдар тізімі. Құбырлар қашан екенін анықтау үшін манифестті әр түрлі етуі мүмкін
жаңа артефактілер пайда болады, шығарылым активтерімен қатар JSON жүктеп салыңыз немесе тексеріңіз
пакетті операторларға жылжытудан бұрын хэштер.

Көмекші идемпотентті: пәрменді қайта іске қосу манифестті жаңартады және
`target/mochi-bundle/` жалғыз ретінде сақтай отырып, алдыңғы мұрағатты қайта жазады
ағымдағы құрылғыдағы соңғы бума үшін ақиқат көзі.

# GPU Zstd (ლითონის) მილსადენი

ეს დოკუმენტი აღწერს დეტერმინისტულ GPU მილსადენს, რომელსაც იყენებს Metal Helper
zstd შეკუმშვისთვის. ეს არის დიზაინისა და განხორციელების სახელმძღვანელო
`gpuzstd_metal` დამხმარე, რომელიც ასხივებს სტანდარტულ zstd ჩარჩოებს და დეტერმინისტულ ბაიტებს
მოცემული თანმიმდევრობის ნაკადისთვის. გამომავალი უნდა იყოს ორმხრივი CPU დეკოდერებით; ბაიტი-
ბაიტის პარიტეტი CPU კომპრესორთან არ არის საჭირო, რადგან თანმიმდევრობის გენერირება
განსხვავდება.

## გოლები

- გამოუშვით სტანდარტული zstd ჩარჩოები, რომლებიც დეკოდირდება CPU zstd-ის იდენტურად; ბაიტის პარიტეტი
  CPU კომპრესორით არ არის საჭირო.
- დეტერმინისტული შედეგები ტექნიკის, დრაივერების და ძაფების დაგეგმვისას.
- აშკარა საზღვრების შემოწმება და ბუფერის პროგნოზირებადი ვადები.

## მიმდინარე განხორციელების შენიშვნა

- მატჩების პოვნა და თანმიმდევრობის გენერირება მუშაობს GPU-ზე.
- ჩარჩოს შეკრება და ენტროპიის კოდირება (Huffman/FSE) ამჟამად მუშაობს ჰოსტზე
  შიგთავსის კოდირების გამოყენებით; GPU Huffman/FSE ბირთვები შემოწმებულია პარიტეტზე, მაგრამ არა
  ჯერ ჩართულია სრული ჩარჩოს გზაზე.
- Decode იყენებს ჩარჩოში ჩასმულ დეკოდერს CPU zstd-ის სარეზერვო საშუალებით მხარდაჭერილი ფრეიმებისთვის;
  GPU ბლოკის სრული გაშიფვრა კვლავ მიმდინარეობს.

## კოდირების მილსადენი (მაღალი დონე)

1. შეყვანის დადგმა
   - დააკოპირეთ შეყვანა მოწყობილობის ბუფერში.
   - დაყოფა ფიქსირებული ზომის ნაწილებად (მიმდევრობის გენერირებისთვის) და ბლოკებად (ამისთვის
     zstd ჩარჩოს შეკრება).
2. მატჩის პოვნა და თანმიმდევრობის გამოსხივება
   - GPU ბირთვები სკანირებს თითოეულ ნაწილს და გამოსცემს თანმიმდევრობებს (სიტყვასიტყვით სიგრძე, შესატყვისი
     სიგრძე, ოფსეტური).
   - თანმიმდევრობის დალაგება სტაბილური და განმსაზღვრელია.
3. ლიტერალური მომზადება
   - შეაგროვეთ ლიტერალები, რომლებიც მითითებულია თანმიმდევრობით.
   - შექმენით ლიტერატურული ჰისტოგრამები და აირჩიეთ პირდაპირი ბლოკის რეჟიმი (ნედლეული, RLE ან
     ჰაფმანი) დეტერმინისტულად.
4. ჰაფმანის ცხრილები (სიტყვასიტყვით)
   - შექმენით კოდის სიგრძე ჰისტოგრამიდან.
   - შექმენით კანონიკური ცხრილები დეტერმინისტული კავშირის დარღვევით, რომელიც ემთხვევა CPU-ს
     zstd გამომავალი.
5. FSE ცხრილები (LL/ML/OF)
   - სიხშირის დათვლის ნორმალიზება.
   - შექმენით FSE დეკოდირების/კოდირების ცხრილები დეტერმინისტულად.
6. ბიტტრიმი მწერალი
   - შეფუთეთ ბიტები პატარა-ენდიანი (LSB-პირველი).
   - ფლეში ბაიტის საზღვრებზე; ბალიშები მხოლოდ ნულებით.
   - მნიშვნელობების ნიღაბი დეკლარირებული ბიტების სიგანეზე და განახორციელეთ სიმძლავრის შემოწმება.
7. ბლოკის და ჩარჩოს შეკრება
   - გამოუშვით ბლოკის სათაურები (ტიპი, ზომა, ბოლო ბლოკის დროშა).
   - ლიტერალების და თანმიმდევრობების სერიალიზაცია შეკუმშულ ბლოკებად.
   - გამოუშვით სტანდარტული zstd ჩარჩოს სათაურები და სურვილისამებრ შემოწმების ჯამები.

## დეკოდირების მილსადენი (მაღალი დონე)

1. ჩარჩოს ანალიზი
   - გადაამოწმეთ ჯადოსნური ბაიტები, ფანჯრის პარამეტრები და ჩარჩოს სათაურის ველები.
2. ბიტტრიმი მკითხველი
   - წაიკითხეთ LSB-პირველი ბიტის თანმიმდევრობები მკაცრი საზღვრების შემოწმებით.
3. ლიტერალური გაშიფვრა
   - გაშიფრეთ ლიტერატურული ბლოკები (ნედლეული, RLE ან Huffman) ლიტერატურულ ბუფერში.
4. თანმიმდევრობის გაშიფვრა
   - LL/ML/OF მნიშვნელობების გაშიფვრა FSE ცხრილების გამოყენებით.
   - მატჩების რეკონსტრუქცია მოცურების ფანჯრის გამოყენებით.
5. გამომავალი და საკონტროლო ჯამი
   - ჩაწერეთ რეკონსტრუირებული ბაიტები გამომავალი ბუფერში.
   - გადაამოწმეთ სურვილისამებრ გამშვები ჯამები, როდესაც ჩართულია.

## ბუფერის სიცოცხლე და საკუთრება- შეყვანის ბუფერი: ჰოსტი -> მოწყობილობა, მხოლოდ წაკითხვადი.
- თანმიმდევრობის ბუფერი: მოწყობილობა, რომელიც წარმოებულია მატჩის ძიებით და მოიხმარება ენტროპიით
  კოდირება; არ არის ჯვარედინი ბლოკის ხელახალი გამოყენება.
- ლიტერალური ბუფერი: მოწყობილობა, წარმოებული თითოეული ბლოკისთვის და გამოშვებული ბლოკის შემდეგ
  ემისია.
- გამომავალი ბუფერი: მოწყობილობა, ინახავს ბოლო კადრების ბაიტებს, სანამ მასპინძელი არ დააკოპირებს მათ
  გარეთ.
- ნაკაწრების ბუფერები: ხელახლა გამოიყენება ბირთვებში, მაგრამ ყოველთვის გადაწერილია დეტერმინისტულად.

## ბირთვის პასუხისმგებლობა

- შესატყვისების ძიების ბირთვები: იპოვნეთ შესატყვისები და გამოაქვეყნეთ თანმიმდევრობები (LL/ML/OF + ლიტერალები).
- ჰაფმანის აშენების ბირთვები: გამოყავით კოდის სიგრძე და კანონიკური ცხრილები.
- FSE build kernels: შექმენით LL/ML/OF ცხრილები და სახელმწიფო მანქანები.
- ბირთვების დაშიფვრის დაბლოკვა: ლიტერალების და თანმიმდევრობების სერიალიზაცია ბიტ სტრიმში.
- ბირთვების გაშიფვრის დაბლოკვა: ბიტტრიმის გაანალიზება და ლიტერალების/მიმდევრობების რეკონსტრუქცია.

## დეტერმინიზმი და პარიტეტული შეზღუდვები

- კანონიკური ცხრილების კონსტრუქციები უნდა გამოიყენონ იგივე მოწესრიგება და კავშირის დარღვევა, როგორც CPU
  zstd.
- არ არის ატომები ან შემცირება, რომლებიც დამოკიდებულია ძაფების დაგეგმვაზე ნებისმიერი გამომავალი ბაიტისთვის.
- Bitstream შეფუთვა არის პატარა-ენდიანური, LSB-პირველი; ბაიტის გასწორების ბალიშები ნულებით.
- ყველა საზღვრების შემოწმება აშკარაა; არასწორი შეყვანები დეტერმინისტულად ვერ ხერხდება.

## დადასტურება

- CPU ოქროს ვექტორები ბიტტრიმი მწერლის/მკითხველისთვის.
- კორპუსის პარიტეტის ტესტები, რომლებიც ადარებენ GPU-სა და CPU-ს გამომავალს.
- ბუნდოვანი დაფარვა არასწორად ფორმირებული ჩარჩოებისა და სასაზღვრო პირობებისთვის.

## Benchmarking

გაუშვით `cargo test -p gpuzstd_metal gpu_vs_cpu_benchmark -- --ignored --nocapture`-ზე
შეადარეთ CPU და GPU დაშიფვრა შეყოვნება დატვირთვის ზომებში. ტესტი გამოტოვებს მასპინძლებს
ლითონის შესაძლებელი მოწყობილობის გარეშე; გადაიღეთ გამომავალი ტექნიკის დეტალებთან ერთად
GPU-ს გადმოტვირთვის ზღვრების რეგულირებისას. Norito ახორციელებს იმავე შეწყვეტას
`gpu_zstd::encode_all`, ამიტომ პირდაპირი აბონენტები ემთხვევა ევრისტიკურ კარიბჭეს.

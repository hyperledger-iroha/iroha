# Руководство по сопоставлению ошибок

Последнее обновление: 21 августа 2025 г.

В этом руководстве распространенные режимы сбоев в Iroha сопоставлены со стабильными категориями ошибок, отображаемыми моделью данных. Используйте его для разработки тестов и предсказуемости обработки ошибок клиента.

Принципы
— Пути инструкций и запросов создают структурированные перечисления. Избегайте паники; по возможности сообщайте об определенной категории.
- Категории стабильны, сообщения могут меняться. Клиенты должны сопоставляться по категориям, а не по строкам произвольной формы.

Категории
- InstructionExecutionError::Find: отсутствует объект (актив, учетная запись, домен, NFT, роль, триггер, разрешение, открытый ключ, блок, транзакция). Пример: удаление несуществующего ключа метаданных дает результат Find(MetadataKey).
- InstructionExecutionError::Repetition: повторяющаяся регистрация или конфликтующий идентификатор. Содержит тип инструкции и повторяющийся IdBox.
- InstructionExecutionError::Mintability: инвариант Mintability нарушен (`Once` исчерпан дважды, `Limited(n)` перегружен или пытается отключить `Infinitely`). Примеры: двойная чеканка актива, определенного как `Once`, дает `Mintability(MintUnmintable)`; настройка `Limited(0)` дает `Mintability(InvalidMintabilityTokens)`.
- InstructionExecutionError::Math: числовые ошибки домена (переполнение, деление на ноль, отрицательное значение, недостаточное количество). Пример: сжигание большего количества, чем доступно, дает Math(NotEnoughQuantity).
- InstructionExecutionError::InvalidParameter: неверный параметр или конфигурация инструкции (например, триггер по времени в прошлом). Используйте для неверных полезных данных контракта.
- InstructionExecutionError::Evaluate: несоответствие DSL/спецификации для формы или типов инструкций. Пример: неправильная числовая спецификация стоимости актива дает Evaluate(Type(AssetNumericSpec(..))).
- InstructionExecutionError::InvariantViolation: нарушение системного инварианта, которое нельзя выразить в других категориях. Пример: попытка удалить последнего подписавшего.
— InstructionExecutionError::Query: перенос QueryExecutionFail в случае сбоя запроса во время выполнения инструкции.

ЗапросExecutionFail
- Найти: отсутствует объект в контексте запроса.
- Преобразование: запрос ожидает неправильный тип.
- NotFound: отсутствует курсор оперативного запроса.
- CursorMismatch / CursorDone: ошибки протокола курсора.
- FetchSizeTooBig: превышено ограничение, установленное сервером.
- GasBudgetExceeded: выполнение запроса превысило бюджет газа/материализации.
- InvalidSingularParameters: неподдерживаемые параметры для единичных запросов.
- ЕмкостьОграничение: достигнута емкость хранилища текущих запросов.

Советы по тестированию
- Отдавайте предпочтение модульным тестам, близким к источнику ошибки. Например, несоответствие числовых характеристик актива может быть выявлено в ходе тестов модели данных.
- Интеграционные тесты должны охватывать сквозное сопоставление для репрезентативных случаев (например, дублирующийся реестр, отсутствие ключа при удалении, передача без владения).
— Сохраняйте устойчивость утверждений, сопоставляя варианты перечислений вместо подстрок сообщений.

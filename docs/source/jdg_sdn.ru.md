% аттестаций и ротации JDG-SDN

В этом примечании описывается модель принудительного применения аттестаций секретных узлов данных (SDN).
используется потоком юрисдикционного хранителя данных (JDG).

## Формат обязательства
- `JdgSdnCommitment` привязывает область (`JdgAttestationScope`), зашифрованный
  хэш полезной нагрузки и открытый ключ SDN. Печати — это печатные подписи.
  (`SignatureOf<JdgSdnCommitmentSignable>`) через полезную нагрузку с тегами домена
  `iroha:jurisdiction:sdn:commitment:v1\x00 || norito(signable)`.
- Структурная проверка (`validate_basic`) обеспечивает:
  - `version == JDG_SDN_COMMITMENT_VERSION_V1`
  - действительные диапазоны блоков
  - непустые пломбы
  - равенство области действия с аттестацией при запуске через
    И18НИ00000011X/`validate_with_sdn_registry`
- Дедупликация выполняется валидатором аттестации (подписавший + хеш полезной нагрузки).
  уникальность) для предотвращения невыполнения/дублирования обязательств.

## Политика реестра и ротации
- Ключи SDN находятся в `JdgSdnRegistry`, ключом является `(Algorithm, public_key_bytes)`.
- `JdgSdnKeyRecord` записывает высоту активации, дополнительную высоту выхода на пенсию,
  и дополнительный родительский ключ.
- Вращение регулируется `JdgSdnRotationPolicy` (в настоящее время: `dual_publish_blocks`).
  окно перекрытия). Регистрация дочернего ключа обновляет родительский выход на пенсию до
  `child.activation + dual_publish_blocks`, с ограждением:
  - отсутствующие родители отклоняются
  - активации должны быть строго возрастающими
  - перекрытия, превышающие льготное окно, отклоняются
- Помощники реестра отображают установленные записи (`record`, `keys`) для определения статуса.
  и воздействие API.

## Порядок проверки
- `JdgAttestation::validate_with_sdn_registry` оборачивает структурный
  проверки аттестации и обеспечение соблюдения SDN. `JdgSdnPolicy` темы:
  - `require_commitments`: обеспечить присутствие для PII/секретных полезных данных.
  - `rotation`: окно отсрочки, используемое при обновлении выхода на пенсию родителей.
- Каждое обязательство проверяется на предмет:
  - структурная валидность + соответствие области аттестации
  - наличие зарегистрированного ключа
  - активное окно, охватывающее подтвержденный диапазон блоков (уже установлены границы вывода из эксплуатации).
    включить льготу на двойную публикацию)
  - действительная печать на документе об обязательстве, помеченном доменом
- Стабильные ошибки отображают индекс свидетельства оператора:
  И18НИ00000025Х, И18НИ00000026Х, И18НИ00000027Х, И18НИ00000028Х,
  или структурные неисправности `Commitment`/`ScopeMismatch`.

## Runbook оператора
- **Предоставление:** зарегистрируйте первый ключ SDN с `activated_at` не позднее
  высота первого секретного блока. Опубликуйте отпечаток ключа операторам JDG.
- **Поворот:** сгенерируйте ключ-преемник, зарегистрируйте его с помощью `rotation_parent`.
  указав на текущий ключ, и подтвердите, что выход на пенсию родителя равен
  `child_activation + dual_publish_blocks`. Подтвердите обязательства по полезной нагрузке с помощью
  активная клавиша во время перекрытия окна.
- **Аудит:** предоставлять снимки реестра (`record`, `keys`) через Torii/status.
  поверхности, чтобы аудиторы могли подтвердить активный ключ и окна выхода из эксплуатации. Оповещение
  если подтвержденный диапазон выходит за пределы активного окна.
- **Восстановление:** `UnknownSdnKey` → убедитесь, что в реестре есть ключ печати;
  `InactiveSdnKey` → повернуть или отрегулировать высоту активации; `InvalidSeal` →
  повторно запечатайте полезные данные и обновите аттестации.## Помощник во время выполнения
- `JdgSdnEnforcer` (`crates/iroha_core/src/jurisdiction.rs`) упаковывает политику +
  реестр и проверяет аттестации через `validate_with_sdn_registry`.
- Реестры могут быть загружены из пакетов `JdgSdnKeyRecord` в кодировке Norito (см.
  `JdgSdnEnforcer::from_reader`/`from_path`) или в сборе с
  `from_records`, который применяет ограждения вращения во время регистрации.
- Операторы могут сохранить пакет Norito в качестве доказательства Torii/status.
  всплытие, в то время как та же самая полезная нагрузка подает на силовик, используемый при допуске и
  стражи консенсуса. Один глобальный исполнитель может быть инициализирован при запуске через
  `init_enforcer_from_path` и `enforcer()`/`registry_snapshot()`/`sdn_registry_status()`
  Откройте действующую политику + ключевые записи для поверхностей status/Torii.

# Гранд бүтээгдэхүүний жишээг хай

Энэ жишээ нь дээр дурдсан FASTPQ зөвшөөрлийн хайлтын аргументыг өргөжүүлдэг
`fastpq_plan.md`.  2-р шат дамжлагад провер сонгогчийг үнэлдэг
Бага зэргийн өргөтгөл (LDE) дээрх (`s_perm`) болон гэрч (`perm_hash`) баганууд
домэйн, ажиллаж байгаа `Z_i` томоохон бүтээгдэхүүнийг шинэчилж, эцэст нь
Посейдонтой дараалал.  Хэшжүүлсэн аккумляторыг транскриптэд хавсаргав
`fastpq:v1:lookup:product` домэйн дор, харин эцсийн `Z_i` таарсан хэвээр байна
амласан зөвшөөрлийн хүснэгт бүтээгдэхүүн `T`.

Бид дараах сонгогчийн утгууд бүхий жижиг багцыг авч үздэг.

| эгнээ | `s_perm` | `perm_hash` |
| --- | -------- | ---------------------------------------------- |
| 0 | 1 | `0x019a...` (буцалтгүй тусламжийн үүрэг = аудитор, perm = шилжүүлэх_хөрөнгө) |
| 1 | 0 | `0xabcd...` (зөвшөөрлийн өөрчлөлт байхгүй) |
| 2 | 1 | `0x42ff...` (хүчингүй болгох үүрэг = аудитор, perm = burn_asset) |

`gamma = 0xdead...` нь Fiat-Shamir хайлтын сорилтоос гаралтай байг.
хуулбар.  Провер нь `Z_0 = 1`-г эхлүүлж, мөр бүрийг нугалав:

```
Z_0 = 1
Z_1 = Z_0 * (perm_hash_0 + gamma)^(s_perm_0) = 1 * (0x019a... + gamma)
Z_2 = Z_1 * (perm_hash_1 + gamma)^(s_perm_1) = Z_1 (selector is zero)
Z_3 = Z_2 * (perm_hash_2 + gamma)^(s_perm_2)
```

`s_perm = 0` нь аккумляторыг өөрчилдөггүй мөрүүд.  боловсруулсны дараа
ул мөр, провер Посейдон-хэшүүд `[Z_1, Z_2, ...]` дарааллыг хуулбарлан бичнэ.
Гэсэн хэдий ч `Z_final = Z_3` (эцсийн ажиллаж байгаа бүтээгдэхүүн) -ийг хүснэгтэд тохируулан нийтэлдэг.
хилийн нөхцөл.

Хүснэгтийн талд, өгсөн зөвшөөрөл Merkle мод нь детерминистикийг кодлодог
үүрэнд зориулсан идэвхтэй зөвшөөрлийн багц.  Баталгаажуулагч (эсвэл туршагч
гэрч үүсгэх) тооцоолно

```
T = product over entries: (entry.hash + gamma)
```

Протокол нь `Z_final / T = 1` хязгаарын хязгаарлалтыг хэрэгжүүлдэг.  Хэрэв ул мөр
Хүснэгтэд байхгүй зөвшөөрлийг нэвтрүүлсэн (эсвэл үүнийг орхигдуулсан
нь), бүтээгдэхүүний том харьцаа 1-ээс зөрж, баталгаажуулагч татгалздаг.  Учир нь
хоёр тал нь Goldilocks талбар дотор `(value + gamma)`-ээр үржүүлнэ, харьцаа
CPU/GPU арын хэсэгт тогтвортой хэвээр байна.

Жишээг бэхэлгээний Norito JSON болгон цуваа болгохын тулд
`perm_hash`, сонгогч, мөр бүрийн дараа аккумлятор, жишээ нь:

```json
{
  "gamma": "0xdead...",
  "rows": [
    {"s_perm": 1, "perm_hash": "0x019a...", "z_after": "0x5f10..."},
    {"s_perm": 0, "perm_hash": "0xabcd...", "z_after": "0x5f10..."},
    {"s_perm": 1, "perm_hash": "0x42ff...", "z_after": "0x9a77..."}
  ],
  "table_product": "0x9a77..."
}
```

Арван арван тоот орлуулагчийг (`0x...`) бетонон Алтан цоожоор сольж болно.
автоматжуулсан туршилтыг үүсгэх үед талбайн элементүүд.  Нэмэлт 2-р шатны бэхэлгээ
ажиллаж байгаа аккумляторын Poseidon хэшийг бичих боловч ижил JSON хэлбэрийг хадгалах,
Иймээс жишээ нь ирээдүйн туршилтын векторуудын загвар болж чадна.

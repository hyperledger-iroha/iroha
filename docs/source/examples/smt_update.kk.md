# Sparse Merkle жаңарту мысалы

Бұл жұмыс істеген мысал FASTPQ 2-кезең ізінің a кодтауын көрсетеді
`neighbour_leaf` бағанын пайдаланып, мүшелік емес куәгер. Сирек Меркле ағашы
Poseidon2 өріс элементтеріне қарағанда екілік болып табылады. Кілттер канондық түрлендіріледі
Өріс элементіне хэштелген 32 байт шағын эндандық жолдар және ең көбі
маңызды биттер әр деңгейде тармақты таңдайды.

## Сценарий

- Алдын ала күй қалдырады
  - `asset::alice::rose` -> `0x0000_0000_0000_0005` мәні бар `0x12b7...` хэштелген кілт.
  - `asset::bob::rose` -> `0x0000_0000_0000_0003` мәні бар `0x1321...` хэштелген кілт.
- Жаңарту сұрауы: 2 мәні бар `asset::carol::rose` енгізіңіз.
- Кэролға арналған канондық кілт хэші 5 биттік `0b01011` префиксіне дейін кеңейеді. The
  бар көршілер префикстері `0b01010` (Алиса) және `0b01101` (Bob).

Префиксі `0b01011` сәйкес келетін жапырақ болмағандықтан, провер қамтамасыз етуі керек.
`(alice, bob)` аралығы бос екендігінің қосымша дәлелі. 2-кезең толтырылады
`path_bit_{level}`, `sibling_{level}` бағандарындағы бақылау жолы,
`node_in_{level}` және `node_out_{level}` (`level` `[0, 31]` ішінде). Барлық мәндер
Lit-endian түрінде кодталған Poseidon2 өріс элементтері:

| деңгейі | `path_bit_level` | `sibling_level` | `node_in_level` | `node_out_level` | Ескертпелер |
| ----- | ---------------- | ----------------------------------- | ----------------------------------- | ----------------------------------- | ----- |
| 0 | 1 | `0x241f...` (Алис жапырағы хэш) | `0x0000...` | `0x4b12...` (`value_2 = 2`) | Кірістіру: нөлден бастаңыз, жаңа мәнді сақтаңыз. |
| 1 | 1 | `0x7d45...` (бос оң жақта) | Посейдон2(`node_out_0`, `sibling_0`) | Посейдон2(`sibling_1`, `node_out_1`) | Префикс 1-битті орындаңыз. |
| 2 | 0 | `0x03ae...` (Боб тармағы) | Посейдон2(`node_out_1`, `sibling_1`) | Посейдон2(`node_in_2`, `sibling_2`) | Тармақ ауытқиды, себебі бит = 0. |
| 3 | 1 | `0x9bc4...` | Посейдон2(`node_out_2`, `sibling_2`) | Посейдон2(`sibling_3`, `node_out_3`) | Жоғары деңгейлер жоғары қарай хэшингті жалғастырады. |
| 4 | 0 | `0xe112...` | Посейдон2(`node_out_3`, `sibling_3`) | Посейдон2(`node_in_4`, `sibling_4`) | Түбір деңгейі; нәтиже – күйден кейінгі түбір. |

Осы жолға арналған `neighbour_leaf` бағаны Боб парағымен толтырылған
(`key = 0x1321...`, `value = 3`, `hash = Poseidon2(key, value) = 0x03ae...`). Қашан
растай отырып, AIR мыналарды тексереді:

1. Берілген көрші 2-деңгейде қолданылатын бауырласқа сәйкес келеді.
2. Көрші перне лексикографиялық жағынан кірістірілген пернеден және одан үлкен
   сол ағасы (Алиса) лексикографиялық жағынан кішірек.
3. Енгізілген жапырақты көршімен ауыстыру алдыңғы күй түбірін шығарады.Бұл тексерулер бірге `(0b01010,
0b01101)` жаңартуға дейін. FASTPQ іздерін генерациялайтын енгізулер пайдалана алады
бұл макет сөзбе-сөз; жоғарыдағы сандық тұрақтылар көрнекі болып табылады. Толық үшін
JSON куәгері, бағандарды дәл жоғарыдағы кестеде көрсетілгендей шығарыңыз (
әр деңгейге арналған сандық жұрнақтар), сериалдандырылған аз эндиан байт жолдарын пайдалана отырып
Norito JSON көмекшілері.

# Пример разреженного обновления Меркла

Этот рабочий пример иллюстрирует, как трассировка FASTPQ Stage 2 кодирует
свидетель, не являющийся членом, с использованием столбца `neighbour_leaf`. Редкое дерево Меркла
является двоичным по отношению к элементам поля Poseidon2. Ключи конвертируются в канонические
32-байтовые строки с прямым порядком байтов, хешированные до элемента поля, и наиболее
значащие биты выбирают ветвь на каждом уровне.

## Сценарий

- Предгосударственные отпуска
  - `asset::alice::rose` -> хешированный ключ `0x12b7...` со значением `0x0000_0000_0000_0005`.
  - `asset::bob::rose` -> хешированный ключ `0x1321...` со значением `0x0000_0000_0000_0003`.
- Запрос на обновление: вставьте `asset::carol::rose` со значением 2.
— Канонический хэш ключа Кэрол расширяется до 5-битного префикса `0b01011`.
  существующие соседи имеют префиксы `0b01010` (Алиса) и `0b01101` (Боб).

Поскольку листа с префиксом, соответствующим `0b01011`, не существует, проверяющий должен предоставить
дополнительные доказательства того, что интервал `(alice, bob)` пуст. Этап 2 заполняется
строка трассировки по столбцам `path_bit_{level}`, `sibling_{level}`,
`node_in_{level}` и `node_out_{level}` (с `level` в `[0, 31]`). Все значения
представляют собой элементы поля Poseidon2, закодированные в форме с прямым порядком байтов:

| уровень | `path_bit_level` | `sibling_level` | `node_in_level` | `node_out_level` | Заметки |
| ----- | ---------------- | --------------------------- | ------------------------------------ | ------------------------------------ | ----- |
| 0 | 1 | `0x241f...` (хэш листа Алисы) | `0x0000...` | И18НИ00000026X (`value_2 = 2`) | Вставка: начать с нуля, сохранить новое значение. |
| 1 | 1 | `0x7d45...` (пустое право) | Посейдон2(`node_out_0`, `sibling_0`) | Посейдон2(`sibling_1`, `node_out_1`) | Следуйте биту префикса 1. |
| 2 | 0 | `0x03ae...` (ветвь Боба) | Посейдон2(`node_out_1`, `sibling_1`) | Посейдон2(`node_in_2`, `sibling_2`) | Ветвь переворачивается, потому что бит = 0. |
| 3 | 1 | `0x9bc4...` | Посейдон2(`node_out_2`, `sibling_2`) | Посейдон2(`sibling_3`, `node_out_3`) | Более высокие уровни продолжают движение вверх. |
| 4 | 0 | `0xe112...` | Посейдон2(`node_out_3`, `sibling_3`) | Посейдон2(`node_in_4`, `sibling_4`) | Корневой уровень; результатом является корень пост-состояния. |

Столбец `neighbour_leaf` для этой строки заполнен листом Боба.
(`key = 0x1321...`, `value = 3`, `hash = Poseidon2(key, value) = 0x03ae...`). Когда
AIR проверяет, что:

1. Предоставленный сосед соответствует брату, используемому на уровне 2.
2. Соседний ключ лексикографически больше вставленного ключа, а
   левый брат (Алиса) лексикографически меньше.
3. Замена вставленного листа соседним воспроизводит корень до состояния.Вместе эти проверки доказывают, что в интервале `(0b01010,
0b01101)` до обновления. Реализации, генерирующие трассировки FASTPQ, могут использовать
этот макет дословно; приведенные выше числовые константы являются иллюстративными. Для полного
JSON-свидетель, выдайте столбцы точно так, как они представлены в таблице выше (с
числовые суффиксы на уровень), используя строки байтов с прямым порядком байтов, сериализованные с помощью
Norito Помощники JSON.

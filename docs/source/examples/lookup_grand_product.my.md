# Grand-Product နမူနာကို ရှာပါ။

ဤဥပမာတွင်ဖော်ပြထားသော FASTPQ ခွင့်ပြုချက်ရှာဖွေမှုအငြင်းအခုံကို ချဲ့ထွင်သည်။
`fastpq_plan.md`။  အဆင့် 2 ပိုက်လိုင်းတွင် သက်သေက ရွေးချယ်သူကို အကဲဖြတ်သည်။
(`s_perm`) နှင့် (`perm_hash`) ဒီဂရီနိမ့်သော တိုးချဲ့မှု (LDE) ရှိ ကော်လံများ
ဒိုမိန်း၊ လည်ပတ်နေသော ခမ်းနားသော ထုတ်ကုန် `Z_i` ကို အပ်ဒိတ်လုပ်ပြီး နောက်ဆုံးတွင် တစ်ခုလုံးကို ကတိပြုသည်
Poseidon နဲ့ အတွဲလိုက်။  hashed accumulator ကို စာသားမှတ်တမ်းတွင် ထည့်သွင်းထားသည်။
`fastpq:v1:lookup:product` ဒိုမိန်းအောက်တွင်၊ နောက်ဆုံး `Z_i` သည် တိုက်ဆိုင်ဆဲဖြစ်သည်
ခွင့်ပြုချက်ဇယားထုတ်ကုန် `T`။

အောက်ပါရွေးချယ်မှုတန်ဖိုးများပါရှိသော သေးငယ်သောအသုတ်ကို ကျွန်ုပ်တို့စဉ်းစားသည်-

| အတန်း | `s_perm` | `perm_hash` |
| ---| --------| ---------------------------------------------- |
| 0 | ၁ | `0x019a...` (ထောက်ပံ့ကြေးအခန်းကဏ္ဍ = စာရင်းစစ်၊ perm = transfer_asset) |
| ၁ | 0 | `0xabcd...` (ခွင့်ပြုချက်မရှိ) |
| 2 | ၁ | `0x42ff...` (အခန်းကဏ္ဍ = စာရင်းစစ်၊ perm = burn_asset) |

`gamma = 0xdead...` သည် Fiat-Shamir ရှာဖွေမှုစိန်ခေါ်မှုမှ ဆင်းသက်လာပါစေ။
စာသားမှတ်တမ်း။  စကားပုံသည် `Z_0 = 1` ကို အစပြုပြီး အတန်းတစ်ခုစီကို ခေါက်သည်-

```
Z_0 = 1
Z_1 = Z_0 * (perm_hash_0 + gamma)^(s_perm_0) = 1 * (0x019a... + gamma)
Z_2 = Z_1 * (perm_hash_1 + gamma)^(s_perm_1) = Z_1 (selector is zero)
Z_3 = Z_2 * (perm_hash_2 + gamma)^(s_perm_2)
```

`s_perm = 0` နေရာတွင် အတန်းများသည် accumulator ကို မပြောင်းလဲပါ။  လုပ်ဆောင်ပြီးနောက်
သဲလွန်စ၊ သက်သေပြသူ Poseidon- သည် စာသားမှတ်တမ်းအတွက် `[Z_1, Z_2, ...]` ၏ sequence ကို ဖျက်သည်
ဇယားနှင့်ကိုက်ညီစေရန် `Z_final = Z_3` (နောက်ဆုံးထွက်ကုန်) ကိုလည်း ထုတ်ဝေသည်
နယ်နိမိတ်အခြေအနေ။

ဇယားဘက်တွင်၊ Merkle tree သည် ကုဒ်သွင်းခွင့်ပြုချက်ဖြင့် ဆုံးဖြတ်သတ်မှတ်မှုကို ကုဒ်လုပ်သည်။
အထိုင်အတွက် တက်ကြွသောခွင့်ပြုချက်များအစုံ။  စစ်ဆေးအတည်ပြုသူ (သို့မဟုတ်) ကာလအတွင်း
သက်သေမျိုးဆက်) တွက်ချက်ခြင်း။

```
T = product over entries: (entry.hash + gamma)
```

ပရိုတိုကောသည် နယ်နိမိတ်ကန့်သတ်ချက် `Z_final / T = 1` ကို ပြဋ္ဌာန်းသည်။  သဲလွန်စရှိရင်
ဇယားတွင်မပါရှိသောခွင့်ပြုချက်ကိုမိတ်ဆက်ခဲ့သည် (သို့မဟုတ်ထိုအရာကိုချန်လှပ်ထားသည်။
ဆိုသည်မှာ) ကြီးမြတ်သော ထုတ်ကုန်အချိုးသည် 1 နှင့် ကွဲလွဲနေပြီး အတည်ပြုသူမှ ပယ်ချပါသည်။  အဘယ်ကြောင့်ဆိုသော်
နှစ်ဖက်စလုံးသည် Goldilocks အကွက်အတွင်း၌ `(value + gamma)` ဖြင့် မြှောက်ကာ အချိုး၊
CPU/GPU backend များတွင် တည်ငြိမ်နေပါသည်။

ပွဲစဉ်များအတွက် Norito JSON အဖြစ် နမူနာကို စီစဥ်ရန်၊ tuple ကို မှတ်တမ်းတင်ပါ
ဥပမာ၊ အတန်းတစ်ခုစီပြီးနောက် `perm_hash`၊ ရွေးချယ်သူနှင့် စုစည်းကိရိယာ၊

```json
{
  "gamma": "0xdead...",
  "rows": [
    {"s_perm": 1, "perm_hash": "0x019a...", "z_after": "0x5f10..."},
    {"s_perm": 0, "perm_hash": "0xabcd...", "z_after": "0x5f10..."},
    {"s_perm": 1, "perm_hash": "0x42ff...", "z_after": "0x9a77..."}
  ],
  "table_product": "0x9a77..."
}
```

ဆဋ္ဌမဂဏန်းနေရာကိုင်ဆောင်သူများ (`0x...`) ကို ကွန်ကရစ် Goldilocks များဖြင့် အစားထိုးနိုင်ပါသည်။
အလိုအလျောက် စမ်းသပ်မှုများ ပြုလုပ်သည့်အခါ အကွက်ဒြပ်စင်များ။  ထို့အပြင် အဆင့် 2 ပွဲစဉ်များ
လည်ပတ်နေသော accumulator ၏ Poseidon hash ကို မှတ်တမ်းတင်သော်လည်း တူညီသော JSON ပုံသဏ္ဍာန်ကို ထားရှိပါ၊
ထို့ကြောင့် နမူနာသည် အနာဂတ်စမ်းသပ် vector များအတွက် ပုံစံပလိတ်အဖြစ် နှစ်ဆတိုးနိုင်သည်။

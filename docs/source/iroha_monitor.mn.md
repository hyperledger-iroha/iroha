# Iroha Монитор

Дахин засварласан Iroha дэлгэц нь хөдөлгөөнт дүрс бүхий хөнгөн терминал UI-г хослуулсан.
наадам ASCII урлаг ба уламжлалт Этенраку сэдэв.  Энэ нь хоёрт анхаарлаа хандуулдаг
энгийн ажлын урсгалууд:

- **Spawn-lite горим** – үе тэнгийнхнийгээ дуурайдаг түр зуурын статус/ хэмжигдэхүүнийг эхлүүлнэ.
- **Хавсрах горим** – мониторыг одоо байгаа Torii HTTP төгсгөлийн цэгүүд рүү чиглүүлнэ.

UI нь шинэчлэх бүрт гурван бүсийг харуулдаг:

1. **Torii skyline header** – хөдөлгөөнт тории хаалга, Фүжи уул, кои долгион, од
   сэргээх хэмнэлтэй синхрон гүйлгэх талбар.
2. **Тогтоон зурвас** – нэгтгэсэн блок/гүйлгээ/хий нэмэх дахин сэргээх хугацаа.
3. **Үе тэнгийн ширээ, наадмын шивнээ** – зүүн талд байгаа үе тэнгийн эгнээ, эргэлдэх үйл явдал
   анхааруулга (хугацаа хэтэрсэн, хэт их ачаалал гэх мэт) авдаг баруун талд нэвтэрнэ үү.
4. **Заавал хийн чиг хандлага** – гялбааны шугам нэмэхийн тулд `--show-gas-trend`-г идэвхжүүлнэ
   бүх үе тэнгийнхний нийт хийн хэрэглээг нэгтгэн харуулав.

Энэ рефакторт шинэ:

- Кой, торий, дэнлүү бүхий япон маягийн ASCII хүүхэлдэйн кино.
- Хялбаршуулсан тушаалын гадаргуу (`--spawn-lite`, `--attach`, `--interval`).
- Гагаку сэдвийн нэмэлт аудио тоглуулах боломжтой танилцуулга баннер (гадаад MIDI
  платформ/аудио стек үүнийг дэмждэг үед тоглуулагч эсвэл суурилуулсан зөөлөн синт).
- CI эсвэл хурдан утааны гүйлтийн `--no-theme` / `--no-audio` тугнууд.
- Хамгийн сүүлийн үеийн анхааруулга, гүйцэтгэх хугацаа эсвэл ажиллах хугацааг харуулсан "сэтгэлийн байдал" багана.

## Хурдан эхлэл

Мониторыг бүтээж, бүдүүлэг үе тэнгийнхний эсрэг ажиллуулна уу:

```bash
cargo run -p iroha_monitor -- --spawn-lite --peers 3
```

Одоо байгаа Torii төгсгөлийн цэгүүдэд хавсаргана уу:

```bash
cargo run -p iroha_monitor -- \
  --attach http://127.0.0.1:8080 http://127.0.0.1:8081 \
  --interval 500
```

CI-д ээлтэй дуудлага (танилцуулгын хөдөлгөөнт дүрс болон аудиог алгасах):

```bash
cargo run -p iroha_monitor -- --spawn-lite --no-theme --no-audio
```

### CLI тугууд

```
--spawn-lite         start local status/metrics stubs (default if no --attach)
--attach <URL...>    attach to existing Torii endpoints
--interval <ms>      refresh interval (default 800ms)
--peers <N>          stub count when spawn-lite is active (default 4)
--no-theme           skip the animated intro splash
--no-audio           mute theme playback (still prints the intro frames)
--midi-player <cmd>  external MIDI player for the built-in Etenraku .mid
--midi-file <path>   custom MIDI file for --midi-player
--show-gas-trend     render the aggregate gas sparkline panel
--art-speed <1-8>    multiply the animation step rate (1 = default)
--art-theme <name>   choose between night, dawn, or sakura palettes
--headless-max-frames <N>
                     cap headless fallback to N frames (0 = unlimited)
```

## Сэдвийн танилцуулга

Анхдагч байдлаар, эхлүүлэх үед Etenraku оноо авах үед богино ASCII хөдөлгөөнт дүрс тоглодог
эхэлдэг.  Аудио сонгох дараалал:

1. Хэрэв `--midi-player`-г өгсөн бол үзүүлэнгийн MIDI-г үүсгэнэ үү (эсвэл `--midi-file` ашиглана уу)
   болон тушаалыг төрүүлнэ.
2. Үгүй бол macOS/Windows (эсвэл `--features iroha_monitor/linux-builtin-synth`-тэй Linux) дээр
   Суулгасан гагаку зөөлөн синтезээр оноог гаргах (гадаад аудио байхгүй
   шаардлагатай хөрөнгө).
3. Хэрэв дууг идэвхгүй болгосон эсвэл эхлүүлэх ажиллагаа амжилтгүй болсон бол танилцуулга нь хэвлэсэн хэвээр байна
   хөдөлгөөнт дүрс, тэр даруй TUI-д орно.

CPAL-ээр ажилладаг synth нь macOS болон Windows дээр автоматаар идэвхждэг. Линукс дээр ийм байна
ажлын талбарыг бүтээх явцад ALSA/Pulse толгойг орхигдуулахгүйн тулд бүртгүүлэх; идэвхжүүлнэ үү
Хэрэв таны систем хангадаг бол `--features iroha_monitor/linux-builtin-synth`
ажиллаж байгаа аудио стек.

CI эсвэл толгойгүй бүрхүүлд гүйх үед `--no-theme` эсвэл `--no-audio` ашиглана уу.

Зөөлөн synth нь одоо *MIDI synth загварт авсан зохицуулалтыг дагаж байна
Rust.pdf*: хичирики болон рютеки нар шогийн үеэр гетерофоник аялгуу хуваалцдаг.
баримт бичигт тодорхойлсон aitake дэвсгэрүүдийг хангадаг.  Хугацаатай тэмдэглэлийн өгөгдлүүд амьдардаг
`etenraku.rs`-д; Энэ нь CPAL-ийн буцаан дуудлагыг болон үүсгэсэн MIDI-г идэвхжүүлдэг.
Аудио гаралт боломжгүй үед монитор дахин тоглуулахыг алгасах боловч үзүүлсээр байна
ASCII хөдөлгөөнт дүрс.

## UI тойм- **Толгойн зураг** – хүрээ бүрийг `AsciiAnimator`-ээр үүсгэсэн; кои, торий дэнлүү,
  мөн долгион нь тасралтгүй хөдөлгөөнийг бий болгохын тулд шилжинэ.
- **Тогтоон зурвас** – онлайн үе тэнгийнхэн, мэдээлэгдсэн үе тэнгийн тоо, блокийн нийлбэр,
  хоосон биш блокийн нийлбэр, tx зөвшөөрөл/татгалзах, хийн хэрэглээ, сэргээх хурд.
- **Peer хүснэгт** – өөр нэр/төгсгөлийн цэг, блок, гүйлгээ, дарааллын хэмжээ,
  хийн хэрэглээ, хоцрогдол, "сэтгэлийн байдал"-ын сануулга (сануулга, гүйцэтгэх хугацаа, ажиллах хугацаа).
- **Баярын шивнээ** – анхааруулга (холболтын алдаа, ачаалал)
  зөрчлийг хязгаарлах, эцсийн цэгийг удаашруулах).  Мессежүүд урвуу байна (дээд талд нь хамгийн сүүлд).

Гарын товчлол:

- `n` / Баруун / Доош – фокусыг дараагийн үе рүү шилжүүлнэ.
- `p` / Зүүн / Дээш – фокусыг өмнөх үе рүү шилжүүлнэ.
- `q` / Esc / Ctrl-C – терминалаас гарч, сэргээнэ үү.

Дэлгэц нь өөр дэлгэцийн буфер бүхий кроссterm + ratatui ашигладаг; гарахдаа
курсорыг сэргээж, дэлгэцийг цэвэрлэнэ.

## Утааны шинжилгээ

Крат нь горим болон HTTP хязгаарлалтын аль алиныг нь хэрэгжүүлдэг интеграцийн тестийг илгээдэг:

- `spawn_lite_smoke_renders_frames`
- `attach_mode_with_stubs_runs_cleanly`
- `invalid_endpoint_surfaces_warning`
- `status_limit_warning_is_rendered`
- `attach_mode_with_slow_peer_renders_multiple_frames`

Зөвхөн мониторын туршилтуудыг ажиллуулна уу:

```bash
cargo test -p iroha_monitor -- --nocapture
```

Ажлын талбар нь илүү хүнд интеграцийн тестүүдтэй (`cargo test --workspace`). Гүйж байна
Мониторын тестийг тусад нь хийх нь таныг хурдан баталгаажуулахад тустай хэвээр байна
бүрэн иж бүрдэл хэрэггүй.

## Алдааг олж засварлах- **Аудио гаралтгүй** – монитор дуугүй тоглуулах горимд шилжиж, үргэлжлүүлнэ.
- **Толгойгүй нөхцлөөс эрт гарна** – монитор толгойгүй гүйлтийг хос руу чиглүүлдэг
  сэлгэх боломжгүй үед хэдэн арван фрейм (анхдагч интервалаар 12 секунд орчим).
  терминалыг түүхий горимд оруулах; үргэлжлүүлэн ажиллуулахын тулд `--headless-max-frames 0`-г дамжуулна уу
  тодорхойгүй хугацаагаар.
- **Их хэмжээний статусын ачаалал** – үе тэнгийнхний сэтгэлийн багана болон наадмын бүртгэл
  `body exceeds …`-г тохируулсан хязгаартай (`128 KiB`) харуулах.
- **Удаан үе тэнгийнхэн** – үйл явдлын бүртгэлд хугацаа хэтэрсэн анхааруулгыг бүртгэдэг; үе тэнгийнхэндээ анхаарлаа төвлөрүүл
  мөрийг тодруулах.

Наадмын тэнгэрийн шугамыг сайхан өнгөрүүлээрэй!  Нэмэлт ASCII сэдвүүдэд оруулсан хувь нэмэр эсвэл
хэмжүүрийн самбарыг ашиглах боломжтой—тэдгээрийг детерминатив байлгахын тулд кластерууд ижил байх болно
терминалаас үл хамааран хүрээ тус бүрээр.

% Конвейер управления (Iroha 2 и SORA Парламент)

# Текущее состояние (v1)
- Предложения по управлению формулируются следующим образом: автор предложения → референдум → подсчет голосов → принятие закона. Окна референдума и пороги явки/одобрения применяются, как описано в `gov.md`; блокировки доступны только для продления и разблокируются по истечении срока действия.
- При отборе в парламент используются жеребьёвки на основе VRF с детерминированным порядком и сроками полномочий; когда постоянного списка не существует, Torii получает резервный вариант, используя конфигурацию `gov.parliament_*`. Проверки контроля доступа и кворума совета выполняются в тестах `gov_parliament_bodies`/`gov_pipeline_sla`.
- Режимы голосования: ZK (по умолчанию, требуется `Active` VK со встроенными байтами) и Plain (квадратичный вес). Несоответствия режима отклоняются; создание/расширение блокировки монотонно в обоих режимах с регрессионными тестами для ZK и простым повторным голосованием.
- Неправомерные действия валидатора рассматриваются через конвейер доказательств (`/v1/sumeragi/evidence*`, помощники CLI) с совместным согласованием, осуществляемым `NextMode` + `ModeActivationHeight`.
— Защищенные пространства имен, перехватчики обновления во время выполнения и допуск манифеста управления документированы в `governance_api.md` и охватываются телеметрией (`governance_manifest_*`, `governance_protected_namespace_total`).

# В работе / отставание
- Опубликовать артефакты розыгрыша VRF (начальный, контрольный, упорядоченный состав, запасные) и систематизировать правила замены в случае неявки; добавить золотые светильники для розыгрыша и замен.
- Обеспечение соблюдения поэтапного SLA для органов парламента (правила → повестка дня → исследование → рассмотрение → жюри → принятие) требует явных таймеров, путей эскалации и счетчиков телеметрии.
- Голосование присяжных за тайну/обязательства-раскрытие и соответствующие проверки на предмет противодействия взяточничеству еще предстоит провести.
- Множители ролевых связей, сокращение проступков для тел с высоким риском и время восстановления между сервисными слотами требуют настройки конфигурации и тестов.
- Закрытие полос управления и окна/ворота референдумов отслеживаются в `gov.md`/`status.md`; обновляйте записи дорожной карты по мере проведения оставшихся приемочных испытаний.

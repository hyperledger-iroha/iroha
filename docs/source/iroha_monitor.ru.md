# Iroha Монитор

Обновленный монитор Iroha сочетает в себе облегченный пользовательский интерфейс терминала с анимированными
фестиваль ASCII-искусства и традиционная тема Этенраку.  Он фокусируется на двух
простые рабочие процессы:

- **Режим Spawn-lite** – запуск эфемерных заглушек статуса/метрик, имитирующих одноранговые узлы.
- **Режим подключения** — наведите монитор на существующие конечные точки HTTP Torii.

Пользовательский интерфейс отображает три региона при каждом обновлении:

1. **Torii заголовок линии горизонта** – анимированные ворота тории, гора Фудзи, волны кои и звезда.
   поле, которое прокручивается синхронно с частотой обновления.
2. **Сводная полоса** — агрегированные блоки/транзакции/газ плюс время обновления.
3. **Таблица сверстников и фестивальный шепот** – ряды сверстников слева, чередующиеся события
   log справа, который фиксирует предупреждения (тайм-ауты, слишком большие полезные данные и т. д.).
4. **Дополнительный тренд газа** — включите `--show-gas-trend` для добавления спарклайна.
   суммирование общего потребления газа всеми узлами.

Новое в этом рефакторинге:

- Анимированная сцена ASCII в японском стиле с кои, ториями и фонарями.
- Упрощенная командная поверхность (`--spawn-lite`, `--attach`, `--interval`).
- Вступительный баннер с дополнительным аудиовоспроизведением темы гагаку (внешний MIDI
  плеер или встроенный программный синтезатор, если платформа/аудиостек поддерживает это).
- Флаги `--no-theme` / `--no-audio` для CI или быстрых прогонов дыма.
- Столбец «настроение» для каждого узла, показывающий последнее предупреждение, время фиксации или время безотказной работы.

## Быстрый старт

Создайте монитор и запустите его на заглушенных узлах:

```bash
cargo run -p iroha_monitor -- --spawn-lite --peers 3
```

Присоединитесь к существующим конечным точкам Torii:

```bash
cargo run -p iroha_monitor -- \
  --attach http://127.0.0.1:8080 http://127.0.0.1:8081 \
  --interval 500
```

CI-дружественный вызов (пропустить вступительную анимацию и звук):

```bash
cargo run -p iroha_monitor -- --spawn-lite --no-theme --no-audio
```

### Флаги CLI

```
--spawn-lite         start local status/metrics stubs (default if no --attach)
--attach <URL...>    attach to existing Torii endpoints
--interval <ms>      refresh interval (default 800ms)
--peers <N>          stub count when spawn-lite is active (default 4)
--no-theme           skip the animated intro splash
--no-audio           mute theme playback (still prints the intro frames)
--midi-player <cmd>  external MIDI player for the built-in Etenraku .mid
--midi-file <path>   custom MIDI file for --midi-player
--show-gas-trend     render the aggregate gas sparkline panel
--art-speed <1-8>    multiply the animation step rate (1 = default)
--art-theme <name>   choose between night, dawn, or sakura palettes
--headless-max-frames <N>
                     cap headless fallback to N frames (0 = unlimited)
```

## Введение темы

По умолчанию при запуске воспроизводится короткая анимация ASCII, в то время как оценка Etenraku
начинается.  Порядок выбора аудио:

1. Если указан `--midi-player`, сгенерируйте демонстрационный MIDI-файл (или используйте `--midi-file`).
   и создайте команду.
2. В противном случае в macOS/Windows (или Linux с `--features iroha_monitor/linux-builtin-synth`)
   визуализировать партитуру с помощью встроенного программного синтезатора gagaku (без внешнего звука).
   необходимые активы).
3. Если звук отключен или инициализация не удалась, вступительная часть все равно печатает
   анимацию и сразу входит в TUI.

Синтезатор на базе CPAL автоматически включается в macOS и Windows. В линуксе это
согласитесь, чтобы не пропустить заголовки ALSA/Pulse во время сборки рабочей области; включить это
с `--features iroha_monitor/linux-builtin-synth`, если ваша система предоставляет
рабочий аудиостек.

Используйте `--no-theme` или `--no-audio` при работе в CI или безголовых оболочках.

Программный синтезатор теперь соответствует аранжировке, описанной в *MIDI-синтезаторе.
Rust.pdf*: хитирики и рютеки имеют гетерофоническую мелодию, в то время как сё
предоставляет площадки aitake, описанные в документе.  Данные синхронизированных нот живут
в `etenraku.rs`; он обеспечивает как обратный вызов CPAL, так и сгенерированный демонстрационный MIDI.
Когда аудиовыход недоступен, монитор пропускает воспроизведение, но продолжает рендеринг.
ASCII-анимация.

## Обзор пользовательского интерфейса- **Header art** – каждый кадр генерируется `AsciiAnimator`; кои, фонарики тории,
  и волны дрейфуют, создавая непрерывное движение.
- **Сводная полоса** — показывает онлайн-пиры, сообщенное количество пиров, общее количество блоков,
  непустые итоговые суммы блоков, одобрения/отклонения передачи, использование газа и частота обновления.
- **Таблица узлов** – столбцы для псевдонима/конечной точки, блоков, транзакций, размера очереди,
  использование газа, задержка и подсказка «настроение» (предупреждения, время фиксации, время безотказной работы).
- **Фестивальный шепот** – скользящий журнал предупреждений (ошибки подключения, полезная нагрузка
  ограничения нарушений, медленные конечные точки).  Сообщения перевернуты (последние сверху).

Сочетания клавиш:

- `n`/Вправо/Вниз – переместить фокус на следующий пир.
- `p`/Left/Up – переместить фокус на предыдущего узла.
- `q`/Esc/Ctrl-C – выйти и восстановить терминал.

Монитор использует crossterm + ratatui с буфером альтернативного экрана; при выходе из него
восстанавливает курсор и очищает экран.

## Дымовые тесты

В состав набора входят интеграционные тесты, которые проверяют оба режима и ограничения HTTP:

- `spawn_lite_smoke_renders_frames`
- `attach_mode_with_stubs_runs_cleanly`
- `invalid_endpoint_surfaces_warning`
- `status_limit_warning_is_rendered`
- `attach_mode_with_slow_peer_renders_multiple_frames`

Запустите только тесты монитора:

```bash
cargo test -p iroha_monitor -- --nocapture
```

В рабочей области есть более тяжелые интеграционные тесты (`cargo test --workspace`). Бег
отдельные тесты монитора по-прежнему полезны для быстрой проверки, когда вы это делаете
не нужен полный набор.

## Обновление скриншотов

Демо-версия документации теперь сосредоточена на горизонте тории и таблице пиров.  Чтобы обновить
активы, запустите:

```bash
make monitor-screenshots
```

Это оборачивает `scripts/iroha_monitor_demo.sh` (облегченный режим появления, фиксированное начальное значение/окно просмотра,
без интро/аудио, палитра рассвета, арт-скорость 1, безголовый колпачок 24) и пишет
Фреймы SVG/ANSI плюс `manifest.json` и `checksums.json` в
`docs/source/images/iroha_monitor_demo/`. `make check-iroha-monitor-docs`
оборачивает обе защиты CI (`ci/check_iroha_monitor_assets.sh` и
`ci/check_iroha_monitor_screenshots.sh`), поэтому хэши генератора, поля манифеста,
и контрольные суммы остаются синхронизированными; проверка скриншота также поставляется как
`python3 scripts/check_iroha_monitor_screenshots.py`. Передайте `--no-fallback` в
демонстрационный скрипт, если вы хотите, чтобы захват завершился неудачей, а не возвращался к
запеченные кадры, когда выход монитора пуст; когда используется резервный вариант, необработанный
Файлы `.ans` перезаписываются с использованием запеченных кадров, поэтому манифест/контрольные суммы остаются
детерминированный.

## Детерминированные скриншоты

Отправленные снимки находятся в `docs/source/images/iroha_monitor_demo/`:

![обзор монитора](images/iroha_monitor_demo/iroha_monitor_demo_overview.svg)
![конвейер мониторинга](images/iroha_monitor_demo/iroha_monitor_demo_pipeline.svg)

Воспроизведите их с фиксированным окном просмотра/начальным числом:

```bash
scripts/iroha_monitor_demo.sh \
  --cols 120 --rows 48 \
  --interval 500 \
  --seed iroha-monitor-demo
```

Помощник по захвату исправляет `LANG`/`LC_ALL`/`TERM`, вперед
`IROHA_MONITOR_DEMO_SEED`, отключает звук и закрепляет художественную тему/скорость, чтобы
кадры отображаются одинаково на разных платформах. Пишет `manifest.json` (генератор
хэши + размеры) и `checksums.json` (дайджесты SHA-256) под
`docs/source/images/iroha_monitor_demo/`; CI работает
`ci/check_iroha_monitor_assets.sh` и `ci/check_iroha_monitor_screenshots.sh`
потерпеть неудачу, когда активы отклоняются от записанных манифестов.

## Поиск неисправностей- **Нет аудиовыхода** – монитор переключается в режим без звука и продолжает воспроизведение.
- **Резервный режим без головы отключается раньше** – монитор ограничивает количество пробежок без головы до пары
  дюжину кадров (около 12 секунд с интервалом по умолчанию), когда он не может переключиться
  терминал в raw-режим; передайте `--headless-max-frames 0`, чтобы он продолжал работать
  на неопределенный срок.
- **Увеличенные статусные данные** – столбец настроения партнера и журнал фестиваля.
  покажите `body exceeds …` с настроенным пределом (`128 KiB`).
- **Медленные пиры** – в журнал событий записываются предупреждения о тайм-ауте; сосредоточить внимание этого сверстника на
  выделите строку.

Наслаждайтесь панорамой фестиваля!  Вклады в дополнительные мотивы ASCII или
панели метрик приветствуются — сохраняйте их детерминированными, чтобы кластеры отображали одинаково
покадрово независимо от терминала.

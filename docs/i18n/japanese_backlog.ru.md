# Бэклог переводов документации на японский

Этот файл отслеживает японскую документацию, которая всё ещё помечена как
`status: needs-translation`, группируя её по категориям для планирования следующих
пакетов перевода. Файлы, соответствующие шаблонам `CHANGELOG.*`, `status.*` и
`roadmap.*`, рассматриваются как временные и не включаются в бэклог. Запустите
`python3 scripts/sync_docs_i18n.py --dry-run`, чтобы обновить список перед началом
нового пакета.

- Незавершённых файлов: 0 (обновлено 2025-10-26)
- Исключения из перевода: по‑прежнему пропускаем `CHANGELOG.*`, `status.*` и
  `roadmap.*`.
- В настоящий момент открытого бэклога нет. Когда появятся новые английские документы,
  запустите `scripts/sync_docs_i18n.py --dry-run`, чтобы зафиксировать дельту и
  поставить следующий пакет в очередь.

### Недавно завершённые японские переводы

## Рекомендуемое разбиение на пакеты

> Поскольку незавершённых пунктов нет, план пакетов остаётся как справочный материал
> для следующей волны документов.

- Цель: поддерживать в актуальном состоянии японские runbook’и по прикреплению и
  проверке ZK.
- Область: любые оставшиеся `docs/source/zk/*.ja.md` (как `lifecycle`, так и
  `prover_runbook` уже завершены).
- Результат: полное покрытие операций ZK плюс глоссарий, добавленный к runbook’ам.

### Пакет B – базовый дизайн IVM / Kotodama / Norito
- Цель: повысить удобство для разработчиков, переведя ключевые документы по дизайну

- Область: `docs/source/ivm_*.ja.md`, `docs/source/kotodama_*.ja.md`,
  `docs/source/norito_*.ja.md`.
- Результат: краткие сводки и перечни ключевых терминов для каждого документа.

### Пакет C – Sumeragi / сетевые операции
- Цель: предоставить эксплуатационные runbook’и для консенсуса, pipeline и сетевого

- Цель: снабдить разработчиков локализованными справочниками и примерами.
- Область: оставшиеся `docs/source/query_*.ja.md` (и любые новые справочные
  документы).
- Результат: гарантировать, что ключевые справочники по CLI и API остаются

## Напоминания о процессе перевода

1. Выберите целевые файлы и работайте в отдельной ветке.
2. После завершения перевода выставьте `status: complete` во front matter и при
   необходимости укажите `translator`.
3. Запустите `python3 scripts/sync_docs_i18n.py --dry-run`, чтобы убедиться, что не
   осталось заглушек.
4. Согласовывайте терминологию с уже существующей японской документацией, например
   `README.ja.md`.

## Заметки

- Следите за форматированием Markdown (отступы, таблицы, блоки кода) вместо того, чтобы
  запускать `cargo fmt --all`.
- При изменении исходного документа обновляйте японскую версию в том же PR, когда это
  возможно.
- Соблюдайте терминологические соглашения, принятые в опубликованной документации;
  при необходимости планируйте дополнительные ревью текста.
- Периодически запускайте `python3 scripts/sync_docs_i18n.py --dry-run` и обновляйте
  этот бэклог, если появляются новые пробелы.

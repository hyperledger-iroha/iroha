# יומן מעקב אחר תרגום התיעוד ליפנית

מסמך זה מרכז את הקבצים ביפנית שסומנו עדיין כ-`status: needs-translation`, לפי קטגוריות, כדי שנוכל לתכנן באצ'ים עתידיים. קבצים התואמים ל-`CHANGELOG.*`, `status.*` ו-`roadmap.*` נחשבים זמניים ולכן אינם נכללים. לפני שמתחילים באצ' חדש, הריצו `python3 scripts/sync_docs_i18n.py --dry-run` כדי לרענן את הרשימה.

## סקירה

- מספר קבצים בהמתנה: 0 (עודכן 2025-10-26)
- חריגות: ממשיכים לדלג על `CHANGELOG.*`, `status.*`, `roadmap.*`.
- כרגע אין באצ' פתוח. כאשר מתווספים מסמכים באנגלית, הריצו `scripts/sync_docs_i18n.py --dry-run` כדי לזהות את הפער ולתעדף את הסבב הבא.

- מטרה: לשמור על עדכניות ספרי ההפעלה של ZK וצינורות Torii.
- היקף: כל `docs/source/zk/*.ja.md` שעדיין חסרים (כרגע `lifecycle` ו-`prover_runbook` כבר הושלמו).
- תוצאה נדרשת: כיסוי מלא של תפעולי ZK כולל מילון מונחים.

- תוצאה נדרשת: תקציר ומפתח מונחים לכל מסמך.

### באצ' C – תפעול Sumeragi / רשת
- מטרה: לגבש מדריכי תפעול למסלולי הקונסנזוס, הפייפליין והרשת.

### באצ' D – כלים / דוגמאות / מקורות
- מטרה: לספק חומר עזר וסטים של דוגמאות ביפנית.
- היקף: כל `docs/source/query_*.ja.md` שעדיין חסרים ופריטים חדשים.
- תוצאה נדרשת: התיעוד ל-CLI ול-API נשאר מסונכרן עם המקור.

## תזכורות תהליך תרגום

1. בחרו את הקבצים וענף עבודה ייעודי.
2. לאחר סיום, עדכנו את `status: complete` בפרונט-מאטר והוסיפו `translator` אם רלוונטי.
3. הריצו שוב `python3 scripts/sync_docs_i18n.py --dry-run` לוודא שאין סטבים פתוחים.
4. הקפידו על עקביות מונחים עם קבצים כמו `README.ja.md`.

## הערות

- שמרו על תקינות Markdown במקום להריץ `cargo fmt --all`.
- אם מסמך המקור משתנה, עדכנו את הגרסה היפנית באותו PR כשאפשר.
- תיאומי ניסוח צריכים להתבצע עם הצוות כדי לשמור על אחידות.
- הריצו את סקריפט הסנכרון מדי פעם ועדכנו את המסמך כשיש פערים חדשים.

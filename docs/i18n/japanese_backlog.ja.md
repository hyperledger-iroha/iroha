## 進め方の提案

> 現時点では未翻訳ファイルはありません。以下のバッチ案は、新規ドキュメントが追加された際の優先順位メモとして維持します。

### バッチ A（ZK / Torii 運用ドキュメント）
- 目的: ZK 添付・検証フローに関する Runbook 群を日本語でも維持する
- 対象: `docs/source/zk/*.ja.md` の未完了分（`lifecycle`, `prover_runbook` は完了済み）
- ゴール: ZK 運用手順を一式カバーし、用語集を Runbook 末尾に追加

### バッチ B（IVM / Kotodama / Norito コア設計）
- 目的: 仮想マシンおよびコーデック周辺の設計資料を日本語化し、開発者の参照性を高める
- 対象: `docs/source/ivm_*.ja.md`, `docs/source/kotodama_*.ja.md`, `docs/source/norito_*.ja.md`
- ゴール: 各ドキュメントのサマリとキーワード索引を付記

### バッチ C（Sumeragi / ネットワーク運用）
- 目的: コンセンサス、パイプライン、ネットワーク運用に関する資料を整備
- 対象: `docs/source/sumeragi*.ja.md`, `docs/source/governance_api.ja.md`, `docs/source/pipeline.ja.md`, `docs/source/p2p.ja.md`, `docs/source/state_tiering.ja.md`
- ゴール: 運用手順書（日本語）の初版を準備

### バッチ D（ツール・サンプル・参照）
- 目的: 開発者向け参照資料を整備し、実務サポートを強化
- 対象: `docs/source/query_*.ja.md` のうち未翻訳が残るもの、および追加で発生する参照資料
- ゴール: 主要な CLI / API 参照が日本語でも追従できること

## 翻訳フロー

1. 対象ファイルを選定し、翻訳作業用ブランチで作業
2. 翻訳完了後に front matter の `status` を `complete` に変更し、必要なら `translator` を追記
3. `python3 scripts/sync_docs_i18n.py --dry-run` を再実行して更新漏れがないか確認
4. 用語・言い回しは既存の日本語ドキュメント（`README.ja.md` など）と整合を取る

- すべての翻訳作業では `cargo fmt --all` の代わりに Markdown の書式崩れ（インデント、表、コードブロック）に注意してください。

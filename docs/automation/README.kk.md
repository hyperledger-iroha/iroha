# Құжаттаманы автоматтандырудың негізгі көрсеткіштері

Бұл каталог сияқты элементтердің жол картасын жасайтын автоматтандыру беттерін түсіреді
AND5/AND6 (Android әзірлеушісінің тәжірибесі + шығарылым дайындығы) және DA-1
(Деректердің қолжетімділігі қауіп үлгісін автоматтандыру) олар қашан шақырады
тексерілетін құжаттаманың дәлелі. Пәрмен сілтемелері және күтілетін кезең
artefacts in-tree сәйкестік шолуларының алғышарттарын қол жетімді етіп сақтайды
CI құбырлары немесе бақылау тақталары желіден тыс болғанда.

## Каталог орналасуы

| Жол | Мақсаты |
|------|---------|
| `docs/automation/android/` | Android құжаттамасы және локализацияны автоматтандырудың негізгі көрсеткіштері (AND5), соның ішінде i18n stub синхрондау журналдары, паритет қорытындылары және AND6 жүйесінен шығу алдында талап етілетін SDK жариялау дәлелдері. |
| `docs/automation/da/` | `cargo xtask da-threat-model-report` сілтемесі бар деректердің қолжетімділігі қаупі үлгісінің автоматтандыру шығыстары және түнгі құжаттар жаңартылады. |

Әрбір ішкі каталог дәлелдемелерді шығаратын пәрмендерді құжаттайды
біз тексеруді күтетін файл орналасуы (әдетте JSON қорытындылары, іске қосу журналдары немесе
көрініс береді). Командалар жаңа артефактілерді кез келген уақытта тиісті қалтаның астына тастайды
автоматтандыруды іске қосу жарияланған құжаттарды айтарлықтай өзгертеді, содан кейін міндеттемеге сілтеме жасайды
тиісті күй/жол картасы жазбасынан.

## Қолдану

1. Ішкі каталогта сипатталған пәрмендерді пайдаланып **Автоматтандыруды іске қосыңыз**
   README (мысалы, `ci/check_android_fixtures.sh` немесе
   `cargo xtask da-threat-model-report`).
2. **Нәтижедегі JSON/журнал артефактілерін** `artifacts/…` ішінен көшіріңіз.
   ISO-8601 уақыт белгісі бар `docs/automation/<program>/…` қалтасына сәйкес келу
   аудиторлар дәлелдемелерді басқару хаттамаларымен салыстыра алатындай файл атауы.
3. Жол картасын жабу кезінде `status.md`/`roadmap.md` **міндеттемеге** сілтеме жасаңыз
   gate арқылы шолушылар сол шешім үшін пайдаланылған автоматтандырудың негізгі деңгейін растай алады.
4. **Файлдарды жеңіл ұстаңыз**. Күтілетін құрылымдық метадеректер,
   манифесттер немесе жиынтықтар — жаппай екілік блоктар емес. Үлкенірек қоқыс орындары қалуы керек
   мұнда жазылған қол қойылған анықтамамен нысанды сақтау.

Осы автоматтандыру ескертпелерін орталықтандыру арқылы біз «құжаттар/автоматтандырудың негізгі көрсеткіштері» блоктан шығарамыз
AND6 шақыратын және DA қаупін беретін аудит үшін қолжетімді» алғы шарты
түнгі есептер мен қолмен нүктелік тексерулер үшін детерминирленген үй үлгісі ағыны.

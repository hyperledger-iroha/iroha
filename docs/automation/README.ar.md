# خطوط أساس أتمتة التوثيق

<div dir="rtl">

يُوثِّق هذا الدليل أسطح الأتمتة التي تشير إليها عناصر خارطة الطريق AND5/AND6
(Android Developer Experience + Release Readiness) و DA-1 (أتمتة نموذج تهديد توفر
البيانات) عندما تطلب أدلة توثيق قابلة للتدقيق. إن حفظ مراجع الأوامر والمخرجات
المتوقعة داخل المستودع يضمن توافر متطلبات مراجعات الامتثال حتى عندما تكون
مسارات CI أو لوحات المتابعة غير متصلة.

## بنية الدليل

| المسار | الغرض |
|--------|-------|
| `docs/automation/android/` | خطوط أساس أتمتة توثيق Android والتعريب (AND5)، بما في ذلك سجلات مزامنة i18n stubs، وملخصات التماثل، وأدلة نشر SDK المطلوبة قبل اعتماد AND6. |
| `docs/automation/da/` | مخرجات أتمتة نموذج تهديد توفر البيانات المشار إليها في `cargo xtask da-threat-model-report` والتحديث الليلي للتوثيق. |

يوثِّق كل دليل فرعي الأوامر التي تولِّد الأدلة مع تخطيط الملفات المتوقع تسجيله
(عادةً ملخصات JSON أو سجلات أو بيانات بيان). تُضيف الفرق مخرجات جديدة ضمن المجلد
المعني عندما تغيِّر عملية الأتمتة التوثيق المنشور بصورة جوهرية، ثم تُشير إلى
التزام (commit) مرتبط من إدخال الحالة/خارطة الطريق.

## الاستخدام

1. **شغّل الأتمتة** باستخدام الأوامر الموضحة في README الفرعي (مثل
   `ci/check_android_fixtures.sh` أو `cargo xtask da-threat-model-report`).
2. **انسخ مخرجات JSON/السجلات الناتجة** من `artifacts/…` إلى المجلد
   `docs/automation/<program>/…` المناسب مع إضافة طابع زمني بصيغة ISO-8601 إلى اسم
   الملف لكي يتمكن المدققون من مطابقة الأدلة مع محاضر الحوكمة.
3. **أشر إلى الالتزام** في `status.md`/`roadmap.md` عند إغلاق بوابة في خارطة
   الطريق كي يتمكن المراجعون من التأكد من خط الأساس للأتمتة المستخدم في القرار.
4. **حافظ على خفة الملفات.** المتوقع هو بيانات وصفية منظمة أو بيانات بيان أو
   ملخصات، وليس كتل ثنائية كبيرة. يجب أن تبقى التفريغات الضخمة في التخزين
   الكائني مع تسجيل المرجع الموقّع هنا.

من خلال مركزية هذه ملاحظات الأتمتة نلبي متطلب AND6 الخاص بـ "خطوط أساس
docs/automation المتاحة للتدقيق" ونمنح تدفق نموذج تهديد توفر البيانات موطنًا
حتميًا للتقارير الليلية وعمليات الفحص اليدوية السريعة.

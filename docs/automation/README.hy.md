# Փաստաթղթերի ավտոմատացման հիմքեր

Այս գրացուցակը գրավում է ավտոմատացման մակերեսները, որոնք ճանապարհային քարտեզի տարրեր են, ինչպիսիք են
AND5/AND6 (Android Developer Experience + Release Readiness) և DA-1
(Տվյալների առկայության սպառնալիքի մոդելի ավտոմատացում) վերաբերում է, երբ նրանք կանչում են
աուդիտի ենթարկվող փաստաթղթային ապացույցներ. Հրամանի տեղեկանքների բեմականացում և սպասվող
artefacts in-tree-ը պահպանում է նույնիսկ համապատասխանության ակնարկների նախադրյալները
երբ CI խողովակաշարերը կամ վահանակները անցանց են:

## Գրացուցակի դասավորություն

| Ճանապարհ | Նպատակը |
|------|---------|
| `docs/automation/android/` | Android փաստաթղթերի և տեղայնացման ավտոմատացման բազային գծեր (AND5), ներառյալ i18n կոճղերի համաժամացման մատյանները, հավասարության ամփոփագրերը և SDK-ի հրապարակման ապացույցները, որոնք պահանջվում են մինչև AND6-ի գրանցումը: |
| `docs/automation/da/` | Տվյալների հասանելիության սպառնալիքի մոդելի ավտոմատացման ելքեր, որոնք հղում են կատարում `cargo xtask da-threat-model-report`-ով և փաստաթղթերի գիշերային թարմացում: |

Յուրաքանչյուր ենթացուցակ փաստաթղթավորում է այն հրամանները, որոնք արտադրում են ապացույցները, ինչպես նաև
ֆայլի դասավորությունը, որը մենք ակնկալում ենք ստուգել (սովորաբար JSON ամփոփագրեր, գործարկվող տեղեկամատյաններ կամ
դրսևորվում է): Թիմերը թողնում են նոր արտեֆակտներ համապատասխան թղթապանակի տակ, երբ
automation run-ը էապես փոխում է հրապարակված փաստաթղթերը, այնուհետև կապում է commit-ին
համապատասխան կարգավիճակի/ճանապարհային քարտեզի մուտքից։

## Օգտագործում

1. **Գործարկեք ավտոմատացումը**՝ օգտագործելով ենթագրքում նկարագրված հրամանները
   README (օրինակ՝ `ci/check_android_fixtures.sh` կամ
   `cargo xtask da-threat-model-report`):
2. **Պատճենեք ստացված JSON/log արտեֆակտները** `artifacts/…`-ից
   համապատասխանող `docs/automation/<program>/…` թղթապանակ ISO-8601 ժամադրոշմով
   ֆայլի անվանումը, որպեսզի աուդիտորները կարողանան կապել ապացույցները կառավարման արձանագրությունների հետ:
3. **Հղում կատարեք** `status.md`/`roadmap.md`-ում ճանապարհային քարտեզը փակելիս
   դարպաս, որպեսզի վերանայողները կարողանան հաստատել այդ որոշման համար օգտագործվող ավտոմատացման ելակետը:
4. **Պահպանեք ֆայլերը թեթև**: Ակնկալիքները կառուցվածքային մետատվյալներ են,
   դրսևորումներ կամ ամփոփումներ, ոչ թե երկուական բլբեր: Ավելի մեծ աղբավայրերը պետք է մնան մեջ
   օբյեկտի պահպանում այստեղ գրանցված ստորագրված հղումով:

Կենտրոնացնելով այս ավտոմատացման նշումները՝ մենք ապաշրջափակում ենք «փաստաթղթերը/ավտոմատացման բազային գծերը
հասանելի է աուդիտի համար» նախապայման, որը AND6-ը կոչ է անում և տալիս DA-ի սպառնալիքը
մոդելի հոսքը որոշիչ տուն է գիշերային հաշվետվությունների և ձեռքով տեղում ստուգումների համար:

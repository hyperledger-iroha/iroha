# Базовые материалы автоматизации документации

Этот каталог фиксирует контуры автоматизации, на которые ссылаются пункты
дорожной карты AND5/AND6 (Android Developer Experience + Release Readiness) и
DA-1 (автоматизация модели угроз Data Availability), когда требуется
аудируемая документационная доказательная база. Хранение ссылок на команды и
ожидаемые артефакты в репозитории гарантирует, что предпосылки для проверок
соответствия доступны даже при недоступных CI-пайплайнах или дашбордах.

## Структура каталога

| Путь | Назначение |
|------|-----------|
| `docs/automation/android/` | Бейзлайны автоматизации для Android-документации и локализации (AND5), включая логи синхронизации i18n-заглушек, сводки паритета и доказательства публикации SDK, требуемые до одобрения AND6. |
| `docs/automation/da/` | Выходы автоматизации модели угроз Data Availability, используемые `cargo xtask da-threat-model-report` и ночным обновлением документации. |

Каждый подкаталог документирует команды, которые производят доказательства, и
ожидаемую структуру файлов (обычно JSON-сводки, логи или манифесты). Команды
помещают новые артефакты в соответствующую папку, когда запуск автоматизации
существенно меняет опубликованные документы, и затем ссылаются на коммит из
соответствующей записи status/roadmap.

## Использование

1. **Запустите автоматизацию** командами, описанными в README подкаталога
   (например, `ci/check_android_fixtures.sh` или
   `cargo xtask da-threat-model-report`).
2. **Скопируйте полученные JSON/лог-артефакты** из `artifacts/…` в соответствующую
   папку `docs/automation/<program>/…`, добавив ISO-8601 метку времени в имя
   файла, чтобы аудиторы могли связать доказательства с протоколами
   управления.
3. **Укажите коммит** в `status.md`/`roadmap.md` при закрытии ворот дорожной
   карты, чтобы проверяющие могли подтвердить использованный бейзлайн
   автоматизации.
4. **Держите файлы легкими.** Ожидаются структурированные метаданные,
   манифесты или сводки, а не крупные бинарные дампы. Большие выгрузки должны
   храниться в объектном хранилище с подписанной ссылкой, зафиксированной здесь.

Централизуя эти заметки по автоматизации, мы закрываем требование AND6
«базовые материалы docs/automation доступны для аудита» и даем потоку модели
угроз DA детерминированное место для ночных отчетов и ручных выборочных проверок.

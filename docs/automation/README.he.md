# קווי בסיס לאוטומציית תיעוד

<div dir="rtl">

תיקייה זו מרכזת את משטחי האוטומציה שאליהם מתייחסים פריטי המפה AND5/AND6
(Android Developer Experience + Release Readiness) ו-DA-1 (אוטומציית מודל איומי
זמינות נתונים) כאשר הם דורשים ראיות תיעודיות הניתנות לביקורת. שמירת הפניות
לפקודות ולתוצרים צפויים בעץ הקוד מבטיחה שדרישות מקדימות לביקורות תאימות יהיו
זמינות גם כאשר צינורות CI או לוחות מחוונים אינם זמינים.

## מבנה התיקייה

| נתיב | מטרה |
|------|------|
| `docs/automation/android/` | קווי בסיס לאוטומציית תיעוד ולוקליזציה של Android (AND5), כולל יומני סנכרון של i18n stubs, סיכומי תאימות וראיות לפרסום SDK הדרושות לפני אישור AND6. |
| `docs/automation/da/` | פלטי אוטומציה של מודל איומי זמינות נתונים, שאליהם מתייחס `cargo xtask da-threat-model-report` והרענון הלילי של התיעוד. |

כל תת־תיקייה מתעדת את הפקודות שמייצרות את הראיות ואת מבנה הקבצים הצפוי
(לרוב סיכומי JSON, לוגים או מניפסטים). צוותים מוסיפים תוצרים חדשים תחת התיקייה
הרלוונטית כאשר ריצת אוטומציה משנה באופן מהותי את המסמכים שפורסמו, ואז מפנים
לקומיט מהכניסה המתאימה ב-status/roadmap.

## שימוש

1. **הפעלת האוטומציה** באמצעות הפקודות המתוארות ב-README של התיקייה המשנית
   (לדוגמה, `ci/check_android_fixtures.sh` או `cargo xtask da-threat-model-report`).
2. **העתקת תוצרי JSON/לוג** מ-`artifacts/…` אל התיקייה `docs/automation/<program>/…`
   המתאימה, עם חותמת זמן בתקן ISO-8601 בשם הקובץ כדי שמבקרים יוכלו לשייך את
   הראיות לפרוטוקולי הממשל.
3. **קישור לקומיט** ב-`status.md`/`roadmap.md` בעת סגירת שער מפת דרכים, כדי
   שמבקרים יוכלו לאשר את קו הבסיס האוטומטי שנעשה בו שימוש.
4. **שמרו על קבצים קלים.** הציפייה היא למטא־דטה מובנית, מניפסטים או סיכומים,
   ולא בלובים בינריים גדולים. דמפים גדולים צריכים להישאר באחסון אובייקטים עם
   הפניה חתומה המתועדת כאן.

ריכוז הערות האוטומציה הללו מאפשר לעמוד בדרישה של AND6 ל"קווי בסיס docs/automation
זמינים לביקורת" ומעניק לתהליך מודל איומי ה-DA מקום דטרמיניסטי לדוחות לילה
ולבדיקות נקודתיות ידניות.

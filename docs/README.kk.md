# Iroha Құжаттама

日本版の概要は [`README.ja.md`](./README.ja.md) を参照してください。

Жұмыс кеңістігі бірдей код базасынан екі шығару жолын жеткізеді: **Iroha 2** (өздігінен орналастырылған орналастырулар) және
**Iroha 3 / SORA Nexus** (жалғыз жаһандық Nexus кітап). Екеуі де бірдей Iroha виртуалды машинасын (IVM) қайта пайдаланады және
Kotodama құралдар тізбегі, сондықтан келісім-шарттар мен байт-код орналастыру мақсаттары арасында портативті болып қалады. Құжаттама қолданылады
егер басқаша белгіленбесе, екеуіне де.

[Негізгі Iroha құжаттамасында](https://docs.iroha.tech/) сіз мыналарды табасыз:

- [Жұмысты бастау нұсқаулығы](https://docs.iroha.tech/get-started/)
- Rust, Python, Javascript және Java/Kotlin үшін [SDK оқулықтары](https://docs.iroha.tech/guide/tutorials/)
- [API анықтамасы](https://docs.iroha.tech/reference/torii-endpoints.html)

Шығарылымға арналған ақ парақтар мен сипаттамалар:

- [Iroha 2 Whitepaper](./source/iroha_2_whitepaper.md) — дербес орналастырылған желі сипаттамасы.
- [Iroha 3 (SORA Nexus) Whitepaper](./source/iroha_3_whitepaper.md) — Nexus көп жолақты және деректер кеңістігінің дизайны.
- [Data Model & ISI Spec (іске асырудан алынған)](./source/data_model_and_isi_spec.md) — кері жобаланған мінез-құлық анықтамасы.
- [ZK конверттері (Norito)](./source/zk_envelopes.md) — жергілікті IPA/STARK Norito конверттері және тексеруші күтулері.

(`*.ur.*`) құжаттама ноталары әрбір ағылшын тіліндегі бастапқы файлдың жанында тұрады. Қараңыз

аудармаларды қолдау, сонымен қатар жаңа тілдерді қосу бойынша нұсқаулық
болашақ.

## Norito (серияландыру кодегі)

Norito – жұмыс кеңістігін сериялау кодегі. Біз `parity-scale-codec` қолданбаймыз
(МАСҚАМ). Құжаттама немесе эталондар SCALE-мен салыстырылған жағдайда, ол тек үшін
контекст; барлық өндіріс жолдары Norito пайдаланады. `norito::codec::{Encode, Decode}`
API интерфейстері хэштеу және сым үшін тақырыпсыз («жалаң») Norito пайдалы жүктемесін қамтамасыз етеді.
тиімділік — бұл SCALE емес, Norito.

Соңғы күй:

- Бекітілген тақырыппен детерминистік кодтау/декодтау (сиқырлы, нұсқа, 16-байттық схема, қысу, ұзындық, CRC64, жалаулар).
- Орындалу уақыты бойынша таңдалған жеделдетумен CRC64-XZ бақылау сомасы:
  - x86_64 PCLMULQDQ (тасымалдауды аз көбейту) + Барретті азайту, 32 байт бөліктерден астам бүктелген.
  - сәйкес жиналмалы aarch64 PMULL.
  - Тасымалдау үшін 8-ге кесу және биттік кері қайтару.
- Бөлулерді азайту үшін туындылар мен негізгі түрлер арқылы жүзеге асырылатын кодталған ұзындық туралы кеңестер.
- Үлкенірек ағындық буферлер (64 КБ) және декодтау кезінде қосымша CRC жаңартуы.
- Қосымша zstd қысу; GPU жеделдету мүмкіндіктерге байланысты және детерминирленген.
- Бейімделетін жолды таңдау: `norito::to_bytes_auto(&T)` № арасында таңдайды
  қысу, CPU zstd немесе GPU-жүктелген zstd (құрастырылған және қол жетімді болғанда)
  пайдалы жүктеме өлшеміне және кэштелген аппараттық құрал мүмкіндіктеріне негізделген. Таңдау тек әсер етеді
  өнімділік және тақырыптың `compression` байты; пайдалы жүктеме семантикасы өзгермейді.

Тепе-теңдік сынақтары, эталондар және пайдалану мысалдары үшін `crates/norito/README.md` қараңыз.

Ескерту: Кейбір ішкі жүйе құжаттары (мысалы, IVM жеделдету және ZK тізбектері) дамып келеді. Функционалдық толық болмаса, файлдар қалған жұмысты және қозғалыс бағытын шақырады.

Күйдің соңғы нүктесінің кодтау жазбалары
- Torii `/status` корпусы әдепкі бойынша Norito ықшамдылық үшін тақырыпсыз («жалаң») пайдалы жүктемемен пайдаланады. Клиенттер алдымен Norito кодты шешуге әрекет етуі керек.
- Серверлер сұрау кезінде JSON қайтара алады; `content-type` `application/json` болса, клиенттер JSON жүйесіне қайта оралады.
- Сым пішімі SCALE емес, Norito. `norito::codec::{Encode,Decode}` API интерфейстері жалаң нұсқа үшін пайдаланылады.

# Аппаратное ускорение

Секция `[accel]` управляет опциональным аппаратным ускорением для IVM и
вспомогательных компонентов. Все ускоренные пути имеют детерминированные
fallback’и на CPU; если backend не проходит golden self‑test во время
запуска, он автоматически отключается и выполнение продолжается на CPU.

- `enable_cuda` (по умолчанию: `true`) – использовать CUDA, когда она
  собрана и доступна.
- `enable_metal` (по умолчанию: `true`) – использовать Metal на macOS, когда
  он доступен.
- `max_gpus` (по умолчанию: `0`) – максимальное число GPU для инициализации;
  `0` означает auto/без явного ограничения.
- `merkle_min_leaves_gpu` (по умолчанию: `8192`) – минимальное число листьев,
  начиная с которого хеширование листьев Merkle переносится на GPU. Имеет
  смысл уменьшать только для необычно быстрых GPU.
- Продвинутые параметры (опциональны; обычно наследуют разумные значения):
  - `merkle_min_leaves_metal` (по умолчанию: наследует
    `merkle_min_leaves_gpu`).
  - `merkle_min_leaves_cuda` (по умолчанию: наследует
    `merkle_min_leaves_gpu`).
  - `prefer_cpu_sha2_max_leaves_aarch64` (по умолчанию: `32768`) – предпочитать
    SHA‑2 на CPU до этого числа листьев на ARMv8 с поддержкой SHA2.
  - `prefer_cpu_sha2_max_leaves_x86` (по умолчанию: `32768`) – предпочитать
    SHA‑NI на CPU до этого числа листьев на x86/x86_64.

Примечания
- В приоритете детерминизм: ускорение никогда не меняет наблюдаемые
  результаты; backends выполняют golden‑тесты при инициализации и
  переключаются на скалярный/SIMD‑код при обнаружении расхождений.
- Настраивайте только через `iroha_config`; избегайте переменных окружения
  в production‑пути.
